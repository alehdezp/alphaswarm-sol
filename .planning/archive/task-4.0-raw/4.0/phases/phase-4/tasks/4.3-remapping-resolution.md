## Objective

Resolve Solidity import paths to actual file paths using project-specific remappings.

## CRITICAL CONTEXT

### The Hard Problem

Resolving imports is HARD because:
1. Different project types use different conventions
2. Remappings can be nested (remapping points to another remapping)
3. OpenZeppelin versions differ between projects
4. Some imports are relative, some are absolute

### What This Is NOT

This is NOT about compiling Solidity. Slither already handles that. This is about:
- Knowing what import statement to put in test files
- Resolving `@openzeppelin/contracts/token/ERC20/ERC20.sol` to actual path
- Making generated test files more likely to compile

### Realistic Expectations

**We will NOT achieve 100% resolution.** Target: 70% of common patterns.

Common patterns we CAN handle:
- `@openzeppelin/contracts/...` -> `lib/openzeppelin-contracts/contracts/...` (Foundry)
- `@openzeppelin/contracts/...` -> `node_modules/@openzeppelin/contracts/...` (Hardhat)
- `forge-std/...` -> `lib/forge-std/src/...`

Patterns we likely CANNOT handle:
- Custom remappings with version-specific paths
- Transitive dependencies
- Circular remappings

```python
"""Import path resolution for test scaffold generation."""

from __future__ import annotations

from pathlib import Path
from typing import Dict, List, Optional
import re

from true_vkg.testing.detection import ProjectConfig, ProjectType

class ImportResolver:
    """Resolve Solidity import paths to actual file paths."""

    # Common patterns we handle
    COMMON_REMAPPINGS = {
        # OpenZeppelin (Foundry-style)
        "@openzeppelin/contracts/": [
            "lib/openzeppelin-contracts/contracts/",
            "lib/@openzeppelin/contracts/",
            "node_modules/@openzeppelin/contracts/",
        ],
        "@openzeppelin/contracts-upgradeable/": [
            "lib/openzeppelin-contracts-upgradeable/contracts/",
            "node_modules/@openzeppelin/contracts-upgradeable/",
        ],
        # Forge std
        "forge-std/": [
            "lib/forge-std/src/",
        ],
        # Solmate
        "solmate/": [
            "lib/solmate/src/",
        ],
        # Solady
        "solady/": [
            "lib/solady/src/",
        ],
    }

    def __init__(self, config: ProjectConfig):
        """Initialize resolver with project config.

        Args:
            config: Project configuration from detection
        """
        self.config = config
        self.root = config.root

    def resolve(self, import_path: str) -> Optional[str]:
        """Resolve import path to actual path.

        Args:
            import_path: The import path from Solidity (e.g., "@openzeppelin/contracts/token/ERC20/ERC20.sol")

        Returns:
            Resolved path relative to project root, or None if cannot resolve
        """
        # First try project-specific remappings
        for prefix, replacement in self.config.remappings.items():
            if import_path.startswith(prefix):
                candidate = import_path.replace(prefix, replacement, 1)
                full_path = self.root / candidate
                if full_path.exists():
                    return candidate
                # Try anyway - might still work
                return candidate

        # Try common patterns
        for prefix, candidates in self.COMMON_REMAPPINGS.items():
            if import_path.startswith(prefix):
                for candidate_base in candidates:
                    candidate = import_path.replace(prefix, candidate_base, 1)
                    full_path = self.root / candidate
                    if full_path.exists():
                        return candidate

        # For Hardhat, try node_modules directly
        if self.config.project_type == ProjectType.HARDHAT:
            if import_path.startswith("@"):
                candidate = f"node_modules/{import_path}"
                if (self.root / candidate).exists():
                    return candidate

        # Cannot resolve
        return None

    def resolve_for_test(self, import_path: str) -> str:
        """Resolve import path for test file, with fallback.

        Always returns a string suitable for import statement.
        If cannot resolve, returns original with comment.

        Args:
            import_path: The import path

        Returns:
            Import statement string
        """
        resolved = self.resolve(import_path)
        if resolved:
            return f'import "{resolved}";'
        else:
            return f'// TODO: Cannot resolve import, adjust manually\n// import "{import_path}";'

    def suggest_forge_std_import(self) -> str:
        """Suggest the correct forge-std import for this project.

        Returns:
            Import statement for Test.sol
        """
        if self.config.project_type == ProjectType.FOUNDRY:
            # Check if forge-std exists
            if "forge-std/" in self.config.remappings:
                return 'import "forge-std/Test.sol";'
            if (self.root / "lib/forge-std").exists():
                return 'import "forge-std/Test.sol";'

        # For non-Foundry or missing forge-std
        return (
            '// TODO: Install forge-std or use alternative test framework\n'
            '// forge install foundry-rs/forge-std\n'
            '// import "forge-std/Test.sol";'
        )

def extract_imports_from_source(source_code: str) -> List[str]:
    """Extract import paths from Solidity source code.

    Args:
        source_code: Solidity source file content

    Returns:
        List of import paths
    """
    # Match: import "path"; or import {Foo} from "path";
    pattern = r'import\s+(?:{[^}]+}\s+from\s+)?["\']([^"\']+)["\']'
    return re.findall(pattern, source_code)
```

```python
# tests/test_testing_remappings.py

import tempfile
from pathlib import Path
from true_vkg.testing.detection import detect_project_structure
from true_vkg.testing.remappings import ImportResolver, extract_imports_from_source

def test_resolve_openzeppelin_foundry():
    """Resolve @openzeppelin in Foundry project."""
    with tempfile.TemporaryDirectory() as tmpdir:
        root = Path(tmpdir)
        (root / "foundry.toml").write_text(
            '[profile.default]\nremappings = ["@openzeppelin/=lib/openzeppelin-contracts/"]\n'
        )
        (root / "lib/openzeppelin-contracts/contracts/token/ERC20").mkdir(parents=True)
        (root / "lib/openzeppelin-contracts/contracts/token/ERC20/ERC20.sol").touch()

        config = detect_project_structure(root)
        resolver = ImportResolver(config)

        result = resolver.resolve("@openzeppelin/contracts/token/ERC20/ERC20.sol")
        assert result is not None
        assert "openzeppelin" in result.lower()

def test_resolve_forge_std():
    """Resolve forge-std imports."""
    with tempfile.TemporaryDirectory() as tmpdir:
        root = Path(tmpdir)
        (root / "foundry.toml").write_text('[profile.default]\n')
        (root / "lib/forge-std/src").mkdir(parents=True)
        (root / "lib/forge-std/src/Test.sol").touch()

        config = detect_project_structure(root)
        resolver = ImportResolver(config)

        result = resolver.resolve("forge-std/Test.sol")
        assert result == "lib/forge-std/src/Test.sol"

def test_resolve_returns_none_when_not_found():
    """Return None for unresolvable imports."""
    with tempfile.TemporaryDirectory() as tmpdir:
        root = Path(tmpdir)
        (root / "foundry.toml").write_text('[profile.default]\n')

        config = detect_project_structure(root)
        resolver = ImportResolver(config)

        result = resolver.resolve("nonexistent/Contract.sol")
        assert result is None

def test_extract_imports():
    """Extract import paths from source code."""
    source = '''
    import "@openzeppelin/contracts/token/ERC20/ERC20.sol";
    import {Ownable} from "@openzeppelin/contracts/access/Ownable.sol";
    import "./interfaces/IVault.sol";
    '''
    imports = extract_imports_from_source(source)
    assert len(imports) == 3
    assert "@openzeppelin/contracts/token/ERC20/ERC20.sol" in imports
    assert "@openzeppelin/contracts/access/Ownable.sol" in imports
    assert "./interfaces/IVault.sol" in imports
```

## Edge Cases to Handle

1. **Missing dependencies**: Return helpful TODO comment
2. **Multiple possible resolutions**: Try each in priority order
3. **Relative imports**: Pass through unchanged (they're relative to test file)
4. **Version mismatches**: Document but don't block

## Validation Criteria

- [ ] Resolves @openzeppelin imports in Foundry projects
- [ ] Resolves @openzeppelin imports in Hardhat projects
- [ ] Handles forge-std imports
- [ ] Returns None for unresolvable imports
- [ ] `resolve_for_test()` always returns usable string
- [ ] All tests pass

## Known Limitations (Document, Don't Fix)

1. Cannot verify OpenZeppelin version compatibility
2. Cannot resolve transitive dependencies
3. Cannot handle monorepo structures
4. Relative imports not resolved (passed through)
