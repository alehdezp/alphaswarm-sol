## Research Questions

### 1. How Does PageRank Actually Work?

**Core Formula:**
```
PR(v) = (1-d) + d * Σ(PR(u) / out_degree(u)) for all u linking to v
```

Where:
- `d` = damping factor (typically 0.85)
- `1-d` = teleport probability (0.15)

**Critical:** The original Task 9.1 pseudocode was WRONG. It divided by nothing.

### 2. What Makes PPR Different From PageRank?

**Personalized PageRank** modifies the teleport distribution:
- Standard PageRank: teleport uniformly to any node
- PPR: teleport ONLY to seed nodes

```
PPR(v, seeds) = (1-d) * (1 if v in seeds else 0) / |seeds|
             + d * Σ(PPR(u, seeds) / out_degree(u))
```

### 3. How Does HippoRAG Use PPR?

From the [HippoRAG paper](https://arxiv.org/abs/2405.14831):
- Seeds = query entities
- Edges = entity co-occurrence in passages
- PPR identifies passages most relevant to query

**Key Insight:** HippoRAG uses UNWEIGHTED edges. BSKG needs WEIGHTED edges for security relevance.

### 4. How Should BSKG Weight Edges?

**Proposed Weighting Strategy:**

| Edge Type | Base Weight | Rationale |
|-----------|-------------|-----------|
| CALLS | 1.0 | Neutral baseline |
| CALLS_EXTERNAL | 1.5 | Security-relevant |
| READS_STATE | 0.8 | Less direct impact |
| WRITES_STATE | 1.2 | State changes important |
| HAS_MODIFIER | 0.5 | Guards reduce risk |

**Risk Adjustment:**
```
final_weight = base_weight * (1 + risk_score)
```

Where `risk_score` is 0.0-1.0 from `src/true_vkg/kg/rich_edge.py`.

**Guard Penalty:**
```
if edge.guards_at_source:
    final_weight *= 0.5  # Guarded edges less likely on attack path
```

**CRITICAL:** Weights must be NORMALIZED to prevent explosion:
```
weights_out = {e: w / sum(all_weights_from_node) for e, w in edges}
```

### 5. What Alpha Value for Security Graphs?

**Tradeoff:**
- Higher alpha (0.2-0.3) = more teleport = stays closer to seeds = less context
- Lower alpha (0.1-0.15) = more walk = explores further = more context

**Recommendation for VKG:**
- alpha = 0.20 for STRICT mode (stay close to finding)
- alpha = 0.15 for STANDARD mode (balanced exploration)
- alpha = 0.10 for RELAXED mode (explore widely)

**Justification:** Security graphs are smaller than text knowledge graphs. Higher teleport prevents walking too far from the vulnerability.

### 6. Convergence Criteria

**Standard Criterion:**
```
max_i |PPR_new[i] - PPR_old[i]| < epsilon
```

**Recommended:**
- epsilon = 1e-4 for fast convergence
- max_iter = 50 for safety
