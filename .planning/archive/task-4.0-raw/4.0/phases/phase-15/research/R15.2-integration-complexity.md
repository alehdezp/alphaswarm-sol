## Objective

---

## Complexity Dimensions

### 1. Code Quality Assessment

For each solution, check:

```bash
# Lines of code
wc -l src/true_vkg/{solution}/*.py

# Test coverage ratio
ls tests/test_{solution}.py 2>/dev/null && wc -l tests/test_{solution}.py

# Type hint coverage
grep -c "def.*:" src/true_vkg/{solution}/*.py
grep -c "def.*:.*->" src/true_vkg/{solution}/*.py
```

### 2. Test Health

```bash
# Run existing tests
uv run pytest tests/test_{solution}.py -v --tb=short

# Check for skipped/xfail markers
grep -r "pytest.mark.skip\|pytest.mark.xfail" tests/test_{solution}.py
```

### 3. Documentation Quality

Check each solution for:
- [ ] Module docstring
- [ ] Class docstrings
- [ ] Function docstrings with examples
- [ ] README or user guide

### 4. CLI Integration Points

Identify:
- Where in CLI would this be exposed?
- What command structure makes sense?
- Any existing CLI hooks?

## Complexity Scoring Matrix

| Factor | Weight | Description |
|--------|--------|-------------|
| Lines of Code | 10% | More code = more to review |
| External Dependencies | 20% | External deps = installation friction |
| API Costs | 15% | Ongoing cost = adoption barrier |
| Test Coverage | 15% | Low coverage = risky integration |
| Documentation | 10% | Poor docs = hard to maintain |
| Core Coupling | 15% | Tight coupling = fragile integration |
| User Value | 15% | High value justifies complexity |

**Scoring:**
- 1-3: Simple (< 4h integration)
- 4-6: Moderate (4-8h integration)
- 7-10: Complex (> 8h integration)

## Quick Assessment Commands

```bash
# For each solution, run this assessment block:
SOLUTION="evolution"

echo "=== $SOLUTION Assessment ==="
echo "LOC:"
find src/true_vkg/$SOLUTION -name "*.py" -exec wc -l {} + | tail -1

echo "Test LOC:"
wc -l tests/test_$SOLUTION.py 2>/dev/null || echo "No tests found"

echo "External imports:"
grep -hr "^import\|^from" src/true_vkg/$SOLUTION/ 2>/dev/null | grep -v "true_vkg" | sort -u | head -10

echo "Test run:"
uv run pytest tests/test_$SOLUTION.py -v --tb=line -q 2>&1 | tail -10
```
