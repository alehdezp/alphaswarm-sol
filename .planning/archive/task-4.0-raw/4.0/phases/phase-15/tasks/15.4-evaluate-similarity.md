## What This Solution Does

**Location:** `src/true_vkg/similarity/`
**Tests:** 47 tests in `tests/test_similarity.py`

The Semantic Similarity solution uses AST-based comparison to find similar code:
1. Parses Solidity AST
2. Extracts structural features
3. Computes similarity scores between functions/contracts
4. Clusters similar vulnerabilities
5. Finds "this code looks like known vulnerable code"

**Claimed Benefits:**
- Find similar vulnerable patterns
- Cluster exploit families
- "Have we seen this before?" detection

### Phase 2: Test Similarity Detection (2h)

Test on known similar contracts:

```python
from true_vkg.similarity import SimilarityEngine

engine = SimilarityEngine()

# Test 1: Identical contracts should be 100% similar
score1 = engine.compare("contract1.sol", "contract1.sol")
assert score1 == 1.0

# Test 2: Similar reentrancy patterns
score2 = engine.compare(
    "tests/contracts/ReentrancyVuln1.sol",
    "tests/contracts/ReentrancyVuln2.sol"
)
print(f"Reentrancy similarity: {score2}")

# Test 3: Different vulnerability types should be less similar
score3 = engine.compare(
    "tests/contracts/ReentrancyVuln.sol",
    "tests/contracts/AccessControlVuln.sol"
)
print(f"Cross-type similarity: {score3}")
```
