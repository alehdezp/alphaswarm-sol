## CRITICAL CONTEXT

**Why this task matters:**
DVDeFi alone (13 challenges) risks overfitting. BSKG might be excellent at detecting DVDeFi-style vulnerabilities while failing on real-world code. SmartBugs provides:
- Different vulnerability distribution
- Academic validation baseline
- Prevents pattern tuning to specific exploits

**What failure looks like:**
- BSKG reports 84% on DVDeFi but only 40% on SmartBugs = overfitting
- Patterns only work on CTF-style code, not production code

### Step 2: Curate 69 Annotated Vulnerabilities (1.5h)

SmartBugs has multiple categories. Curate files with clear ground truth:

**Categories to Include:**
- `reentrancy/` - All files
- `access_control/` - All files
- `arithmetic/` - Select overflow/underflow examples
- `bad_randomness/` - All files
- `unchecked_low_level_calls/` - All files

**Categories to EXCLUDE (not in BSKG scope):**
- `front_running/` - Needs mempool analysis (not static)
- `denial_of_service/` - Some need gas estimation
- `short_addresses/` - ABI-level, not contract logic
