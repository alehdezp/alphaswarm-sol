## CRITICAL CONTEXT

**Why this task matters:**
Without a safe set, the false positive (FP) rate is UNKNOWN. BSKG could be flagging 50% of safe code as vulnerable and we wouldn't know. A high FP rate destroys auditor trust.

**What failure looks like:**
- BSKG flags OpenZeppelin's ERC20 as vulnerable = zero credibility
- Auditor spends 80% of time dismissing false positives = tool abandoned

**Real-world impact:**
- 15% FP = 15 false alarms per 100 findings = manageable
- 50% FP = 50 false alarms per 100 findings = unusable

### Step 6: Implement FP Rate Calculation (1h)

```python
# src/true_vkg/benchmark/fp_calculator.py

def calculate_fp_rate(safe_set_results: List[SafeContractResult]) -> FPMetrics:
    """Calculate false positive rate from safe set results."""
    total_contracts = len(safe_set_results)
    contracts_with_findings = sum(1 for r in safe_set_results if r.findings)
    total_findings = sum(len(r.findings) for r in safe_set_results)

    fp_rate = contracts_with_findings / total_contracts if total_contracts else 0

    # Categorize FPs by severity
    critical_fps = sum(1 for r in safe_set_results
                       for f in r.findings if f.severity == 'critical')
    high_fps = sum(1 for r in safe_set_results
                   for f in r.findings if f.severity == 'high')

    return FPMetrics(
        fp_rate=fp_rate,
        total_contracts=total_contracts,
        contracts_with_fp=contracts_with_findings,
        total_fp_findings=total_findings,
        critical_fp_count=critical_fps,
        high_fp_count=high_fps,
    )
```

### Step 7: Add Safe Set to Benchmark Runner (30 min)

```python
# In runner.py
def run_suite(self, suite_name: str) -> BenchmarkResults:
    suite = load_suite(suite_name)

    if suite.type == "safe-set":
        return self._run_safe_set(suite)
    else:
        return self._run_vuln_suite(suite)

def _run_safe_set(self, suite) -> BenchmarkResults:
    """For safe sets, any finding is a false positive."""
    results = []
    for contract in suite.contracts:
        # Analyze
        graph = build_graph(contract.path)
        findings = analyze(graph)

        # All findings are FPs
        result = SafeContractResult(
            contract_id=contract.id,
            findings=findings,  # Should be empty
            is_clean=len(findings) == 0,
        )
        results.append(result)

    fp_metrics = calculate_fp_rate(results)
    return BenchmarkResults(fp_metrics=fp_metrics)
```

## 7. WHAT TO DO IF FP RATE IS HIGH

**If FP rate > 15%:**

1. **Identify offending patterns:**
   ```bash
   vkg benchmark run --suite safe-set --verbose
   # Look for which patterns are firing
   ```

2. **Analyze false positives:**
   - Is the pattern too broad?
   - Missing `none` condition?
   - Legitimate edge case?

3. **Options:**
   - Refine pattern in Phase 1 (create backlog item)
   - Add exclusion to pattern
   - Document as known FP

4. **Do NOT:**
   - Ignore and ship with high FP rate
   - Remove "troublesome" safe contracts
   - Weaken benchmark targets
