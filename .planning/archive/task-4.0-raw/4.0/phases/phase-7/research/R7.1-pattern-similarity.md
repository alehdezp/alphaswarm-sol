## Research Objective

Define what makes two findings "similar" for learning purposes. This determines when FP warnings apply and how learning events transfer.

---

## The Problem

### Pattern ID Alone is Too Broad

```
Pattern: vm-001 (reentrancy)

Finding A: withdraw() with nonReentrant modifier -> FP
Finding B: withdraw() without any guard -> TP

Both are vm-001, but A's FP verdict shouldn't affect B's confidence.
```

### Function Signature Alone is Too Narrow

```
Finding A: withdraw(uint256) in VaultA.sol -> FP
Finding B: withdraw(uint256) in VaultB.sol -> TP

Same signature, but different implementations.
```

---

## Research Questions

1. What features best predict if findings will have same verdict?
2. How to balance precision vs recall of similarity?
3. Should similarity be binary or graduated?
4. How to handle cross-contract similarities?

## Candidate Similarity Features

### Code-Level Features

| Feature | Example | Pros | Cons |
|---------|---------|------|------|
| Modifiers present | [nonReentrant] | Strong indicator | May miss equivalent guards |
| State variables touched | [balance, totalSupply] | Shows behavior | Complex to extract |
| External call targets | [msg.sender, token] | Attack surface | May be dynamic |
| Inheritance chain | [ReentrancyGuard] | Standard patterns | Naming varies |
| AST structure hash | Hash of function AST | Code similarity | Sensitive to formatting |

### Pattern-Level Features

| Feature | Example | Pros | Cons |
|---------|---------|------|------|
| Pattern ID | vm-001 | Simple | Too broad |
| Matched properties | [state_write_after_call] | Specific | Pattern dependent |
| Severity | critical | Groups similar | Too broad |
| Evidence lines | [45, 50] | Exact match | Brittle to changes |

### Context Features

| Feature | Example | Pros | Cons |
|---------|---------|------|------|
| Contract type | DEX, Lending | Domain similarity | Classification hard |
| Function role | withdraw, deposit | Semantic meaning | Naming varies |
| Protocol | Aave-fork | Common patterns | Detection hard |

## Proposed Similarity Function

### Tier 1: Exact Match (Strong Similarity)

```python
def exact_match(f1, f2) -> bool:
    """Two findings are exactly similar."""
    return (
        f1.pattern_id == f2.pattern_id and
        f1.modifier_set == f2.modifier_set and
        f1.inheritance_hash == f2.inheritance_hash
    )
```

**Use for:** Direct FP warning transfer

### Tier 2: Structural Match (Moderate Similarity)

```python
def structural_match(f1, f2) -> bool:
    """Findings have similar structure."""
    return (
        f1.pattern_id == f2.pattern_id and
        f1.modifier_set.intersection(f2.modifier_set) and
        levenshtein_ratio(f1.function_signature, f2.function_signature) > 0.8
    )
```

**Use for:** Soft warnings, lower confidence

### Tier 3: Pattern Match (Weak Similarity)

```python
def pattern_match(f1, f2) -> bool:
    """Findings match same pattern."""
    return f1.pattern_id == f2.pattern_id
```

**Use for:** Aggregate statistics only

## Implementation Approach

### Similarity Key Generation

```python
@dataclass
class SimilarityKey:
    """Key for finding similarity lookup."""
    pattern_id: str
    modifier_signature: str  # Sorted, joined modifiers
    guard_hash: str          # Hash of guard-related code

    @classmethod
    def from_finding(cls, finding) -> "SimilarityKey":
        modifiers = sorted(finding.modifiers or [])
        mod_sig = "|".join(modifiers)

        # Extract guard-related code patterns
        guard_patterns = extract_guards(finding.code)
        guard_hash = hash(guard_patterns)

        return cls(
            pattern_id=finding.pattern_id,
            modifier_signature=mod_sig,
            guard_hash=guard_hash,
        )

    def matches(self, other: "SimilarityKey", tier: int = 1) -> bool:
        if tier == 1:
            return self == other
        if tier == 2:
            return (self.pattern_id == other.pattern_id and
                    self.modifier_signature == other.modifier_signature)
        return self.pattern_id == other.pattern_id
```

### Guard Extraction

```python
def extract_guards(code: str) -> str:
    """Extract guard patterns from code."""
    guards = []

    # Check for reentrancy guard
    if "nonReentrant" in code or "ReentrancyGuard" in code:
        guards.append("REENTRANCY_GUARD")

    # Check for access control
    if any(g in code for g in ["onlyOwner", "onlyRole", "require(msg.sender"]):
        guards.append("ACCESS_CONTROL")

    # Check for pausable
    if "whenNotPaused" in code or "Pausable" in code:
        guards.append("PAUSABLE")

    return "|".join(sorted(guards))
```
