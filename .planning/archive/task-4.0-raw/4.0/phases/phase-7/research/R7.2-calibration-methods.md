## Research Objective

Determine the best approach for calibrating pattern confidence based on empirical data, balancing simplicity with accuracy.

---

## The Calibration Problem

### Current State
Patterns have arbitrary confidence values (e.g., 0.7, 0.85) not based on actual performance.

### Desired State
Confidence = actual probability that a match is a true positive.

### Calibration Goal
When we say "85% confidence", 85% of those findings should be TPs.

---

## Candidate Methods

### 1. Simple Precision Mapping

**Approach:** Set confidence = observed precision

```python
confidence = TP / (TP + FP)
```

**Pros:**
- Simple to understand
- Easy to implement
- Interpretable

**Cons:**
- Doesn't account for sample size uncertainty
- No smoothing for edge cases
- Brittle with small samples

**Verdict:** Good starting point, needs bounds

### 2. Platt Scaling

**Approach:** Fit logistic regression to calibrate probabilities

```python
P(y=1|s) = 1 / (1 + exp(A*s + B))
```

Where s is the raw score, A and B are learned from data.

**Pros:**
- Standard ML approach
- Handles score distributions
- Well-studied

**Cons:**
- Requires fitting procedure
- Needs enough data points
- Assumes logistic relationship

**Verdict:** Overkill for our use case

### 3. Isotonic Regression

**Approach:** Learn monotonic mapping from scores to probabilities

**Pros:**
- Non-parametric
- Preserves ordering
- Flexible

**Cons:**
- Can overfit with small data
- More complex implementation
- Harder to interpret

**Verdict:** Too complex for initial implementation

### 4. Bayesian Updating (Selected)

**Approach:** Start with prior, update with evidence

```python
# Prior: P(TP) = 0.5 (uninformed)
# After n TPs and m FPs:
P(TP|data) = (n + prior_strength) / (n + m + 2*prior_strength)
```

**Pros:**
- Handles small samples gracefully
- Principled uncertainty handling
- Prior prevents extreme values
- Easy to update incrementally

**Cons:**
- Need to choose prior strength
- Slightly more complex than simple precision

**Verdict:** RECOMMENDED APPROACH

## Bayesian Calibration Details

### Formula

```python
def bayesian_confidence(
    true_positives: int,
    false_positives: int,
    prior_strength: float = 2.0,  # Equivalent to 2 pseudo-observations
    prior_probability: float = 0.5,  # Uninformed prior
) -> float:
    """Calculate calibrated confidence using Bayesian updating."""
    # Add pseudo-observations
    effective_tp = true_positives + prior_strength * prior_probability
    effective_fp = false_positives + prior_strength * (1 - prior_probability)

    total = effective_tp + effective_fp
    return effective_tp / total
```

### Properties

| Samples | Precision | Bayesian Confidence |
|---------|-----------|---------------------|
| 0 | N/A | 0.50 (prior) |
| 2 TP, 0 FP | 1.00 | 0.75 |
| 5 TP, 0 FP | 1.00 | 0.83 |
| 10 TP, 0 FP | 1.00 | 0.91 |
| 8 TP, 2 FP | 0.80 | 0.75 |
| 80 TP, 20 FP | 0.80 | 0.80 |

**Observation:** With more data, Bayesian approaches raw precision.

### Prior Strength Selection

| Strength | Effect | Use When |
|----------|--------|----------|
| 1 | Weak prior | Lots of data expected |
| 2 | Moderate | Default |
| 5 | Strong | Conservative, safety-critical |
| 10 | Very strong | Very few data points expected |

**Recommendation:** Use 2.0 as default, allow configuration.

## Learning Decay Integration

### Time-Weighted Bayesian Update

```python
def time_weighted_confidence(events: List[Event], half_life_days: float = 30):
    """Calculate confidence with time decay."""
    now = datetime.now()
    effective_tp = prior_strength * prior_probability
    effective_fp = prior_strength * (1 - prior_probability)

    for event in events:
        age_days = (now - event.timestamp).days
        weight = math.exp(-0.693 * age_days / half_life_days)

        if event.type == CONFIRMED:
            effective_tp += weight
        elif event.type == REJECTED:
            effective_fp += weight

    return effective_tp / (effective_tp + effective_fp)
```

### Properties

- Recent events have full weight
- Old events fade to zero
- Prior always contributes (stability)

## Bounds Integration

### Calibrated Confidence with Bounds

```python
def bounded_bayesian_confidence(
    events: List[Event],
    bounds: PatternBounds,
    half_life_days: float = 30,
) -> float:
    """Calculate bounded, time-weighted Bayesian confidence."""
    raw = time_weighted_confidence(events, half_life_days)
    return bounds.clamp(raw)
```

### Bound Calculation from Calibration

```python
def calculate_bounds(
    observed_precision: float,
    sample_size: int,
    confidence_interval: float = 0.95,
) -> Tuple[float, float]:
    """Calculate confidence bounds from observed data."""
    if sample_size < 5:
        # Too few samples - use wide defaults
        return (0.3, 0.9)

    # Wilson score interval
    z = 1.96  # 95% CI
    n = sample_size
    p = observed_precision

    denominator = 1 + z*z/n
    center = (p + z*z/(2*n)) / denominator
    spread = z * math.sqrt((p*(1-p) + z*z/(4*n))/n) / denominator

    lower = max(0.15, center - spread)
    upper = min(0.98, center + spread)

    return (lower, upper)
```

4. **Clamp all outputs** to bounds
5. **Start conservative** - easier to loosen than tighten
