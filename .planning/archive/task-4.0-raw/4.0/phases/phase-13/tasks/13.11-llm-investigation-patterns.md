## 1. OBJECTIVE

Create a new class of patterns called **LLM Investigation Patterns** - patterns designed purely for LLM logical reasoning that cannot be expressed as deterministic property checks. These patterns guide the LLM to use tools (LSP, graph queries, PPR) to intelligently explore code and find vulnerabilities through reasoning.

**Key Distinction:**
- **Tier A Patterns:** Deterministic, same code = same finding
- **LLM Investigation Patterns:** Reasoning-based, LLM explores and concludes

**Why This Matters:**
Some vulnerabilities require understanding business logic, cross-contract reasoning, or economic attack vectors that cannot be captured in property-based patterns. LLMs excel at this kind of analysis when given proper guidance.

---

## 2. PROBLEM STATEMENT

### Current Gap

Tier A patterns detect:
- Reentrancy (property: `state_write_after_external_call`)
- Access control (property: `has_access_gate`)
- Oracle staleness (property: `has_staleness_check`)

**What Tier A CANNOT detect:**
- Business logic flaws (incorrect accounting, wrong assumptions)
- Cross-contract attack paths (chained vulnerabilities)
- Economic attacks (price manipulation, arbitrage)
- Privilege escalation paths (indirect ownership transfer)
- Invariant violations (protocol-specific logic errors)
- Configuration errors (wrong parameters, addresses)

### Solution: LLM Investigation Patterns

Patterns that describe WHAT to investigate and HOW to reason, rather than WHAT properties to check.

## 3. ARCHITECTURE

### 3.1 Investigation Pattern Structure

```yaml
# Investigation Pattern Schema
id: inv-[category]-[number]
name: Human-readable name
type: investigation  # NEW: Distinguishes from detection patterns
category: [business-logic | cross-contract | economic | privilege | invariant | config]

# What makes this worth investigating
trigger:
  description: "When this investigation should be triggered"
  graph_signals:
    - signal: "Function writes critical state"
      property: writes_privileged_state
    - signal: "Function has external calls"
      property: calls_external

# The investigation procedure (for LLM to follow)
investigation:
  hypothesis: "What we're trying to prove or disprove"

  steps:
    - id: 1
      action: explore_graph
      description: "Find all functions that modify [relevant state]"
      graph_query: "FIND functions WHERE modifies_state AND target_state"
      interpretation: |
        List all state-modifying functions. Look for:
        - Multiple entry points to same state
        - Inconsistent access control
        - Missing state transitions

    - id: 2
      action: lsp_references
      description: "Find all references to the owner variable"
      target: "${state_variable_name}"
      interpretation: |
        Check if owner can be modified via:
        - Direct assignment
        - Delegatecall
        - Unprotected setter

    - id: 3
      action: ppr_expand
      description: "Find related functions via PPR"
      seed: "${function_name}"
      depth: 2
      interpretation: |
        Examine functions within 2 hops for:
        - Shared state access
        - Call chains that bypass protection

    - id: 4
      action: reason
      description: "Synthesize findings"
      prompt: |
        Given the above exploration results:
        1. Is there a privilege escalation path?
        2. What's the attack sequence?
        3. Confidence level?

  verdict_criteria:
    vulnerable: |
      - Clear attack path exists
      - Requires only attacker-controlled inputs
      - No time-of-check/time-of-use protection
    uncertain: |
      - Possible path but requires specific conditions
      - Needs manual verification
    safe: |
      - No viable attack path
      - All state changes properly protected

# Expected output
output:
  verdict: VULNERABLE | UNCERTAIN | SAFE
  attack_path: "Description of attack sequence"
  confidence: 0-100
  evidence: ["List of code locations supporting verdict"]
  recommendation: "How to fix if vulnerable"
```

### 3.2 Investigation Actions

| Action | Description | Tool Used |
|--------|-------------|-----------|
| `explore_graph` | Query the knowledge graph | Graph Query Engine |
| `lsp_references` | Find all references to a symbol | LSP findReferences |
| `lsp_definition` | Go to symbol definition | LSP goToDefinition |
| `lsp_call_hierarchy` | Trace call hierarchy | LSP incomingCalls/outgoingCalls |
| `ppr_expand` | Expand context via PPR | PPR Algorithm (Phase 9) |
| `read_code` | Read specific file/function | Read Tool |
| `reason` | LLM reasoning step | LLM Provider |
| `synthesize` | Combine multiple findings | LLM Provider |

### 3.3 Investigation Flow

```
┌─────────────────────────────────────────────────────────────────┐
│  INVESTIGATION PATTERN EXECUTION                                 │
│                                                                 │
│  1. TRIGGER CHECK                                               │

│     │ LLM produces:                                           │ │
│     │   - Verdict (VULNERABLE/UNCERTAIN/SAFE)                 │ │

└─────────────────────────────────────────────────────────────────┘
```

## 4. INVESTIGATION PATTERNS

### 4.1 Business Logic Patterns

#### INV-BL-001: Accounting Invariant Violation

```yaml
id: inv-bl-001
name: Accounting Invariant Violation
type: investigation
category: business-logic
severity_range: [HIGH, CRITICAL]

trigger:
  description: "Function modifies balance-like state"
  graph_signals:
    - property: writes_user_balance
    - property: writes_critical_state

investigation:
  hypothesis: "Balance tracking may not match actual token holdings"

  steps:
    - id: 1
      action: explore_graph
      description: "Find all functions that modify balances"
      graph_query: "FIND functions WHERE writes_state AND (modifies balance OR modifies totalSupply)"
      interpretation: |
        For each balance-modifying function, note:
        - What triggers the modification (deposit, withdraw, transfer)?
        - Is the modification atomic with token movement?
        - Are there edge cases (zero amounts, self-transfers)?

    - id: 2
      action: lsp_references
      description: "Find all balance reads"
      target: "balances"
      interpretation: |
        Check where balances are READ:
        - Are reads always after writes settle?
        - Any TOCTOU risks between read and use?

    - id: 3
      action: reason
      description: "Analyze invariant preservation"
      prompt: |
        Analyzing the balance tracking invariant:

        INVARIANT: sum(balances) == actualTokenBalance OR totalSupply == sum(balances)

        Step 1 findings (balance modifications): {step_1_results}
        Step 2 findings (balance reads): {step_2_results}

        Check for:
        1. Can an external call between balance check and update break invariant?
        2. Are there rounding errors that accumulate?
        3. Can fee-on-transfer tokens break accounting?
        4. What happens with zero-amount operations?

  verdict_criteria:
    vulnerable: |
      - Clear path to break invariant
      - Leads to fund extraction or inflation
    uncertain: |
      - Possible edge case but unclear impact
      - Depends on token type
    safe: |
      - Invariant preserved in all paths
      - Proper guards against edge cases
```

#### INV-BL-002: State Machine Violation

```yaml
id: inv-bl-002
name: State Machine Violation
type: investigation
category: business-logic

trigger:
  description: "Contract has state-dependent behavior"
  graph_signals:
    - property: has_state_machine
    - property: checks_state

investigation:
  hypothesis: "State transitions may be bypassable or in wrong order"

  steps:
    - id: 1
      action: explore_graph
      description: "Map all state transitions"
      graph_query: "FIND functions WHERE writes_state AND state_like_variable"
      interpretation: |
        Build state machine diagram:
        - What states exist?
        - What transitions are allowed?
        - Who can trigger transitions?

    - id: 2
      action: lsp_call_hierarchy
      description: "Find all callers of state-changing functions"
      target: "${state_changing_function}"
      interpretation: |
        For each state transition:
        - Can it be called in wrong state?
        - Can external contracts influence state?
        - Are there indirect paths?

    - id: 3
      action: reason
      prompt: |
        Given the state machine mapping from step 1 and call analysis from step 2:

        1. Can any state be skipped?
        2. Can any transition be triggered out of order?
        3. Are there reentrancy concerns in state transitions?
        4. Can emergency/admin functions bypass normal flow?

  verdict_criteria:
    vulnerable: "State can be manipulated to bypass intended flow"
    uncertain: "Complex state logic needs manual review"
    safe: "All transitions properly gated and ordered"
```

### 4.2 Cross-Contract Patterns

#### INV-CC-001: Privilege Escalation via External Call

```yaml
id: inv-cc-001
name: Privilege Escalation via External Call
type: investigation
category: cross-contract

trigger:
  description: "Contract makes external calls that could be hijacked"
  graph_signals:
    - property: calls_external
    - property: uses_delegatecall

investigation:
  hypothesis: "External call could lead to unauthorized privilege escalation"

  steps:
    - id: 1
      action: explore_graph
      description: "Find all external call targets"
      graph_query: "FIND external_calls WHERE source = current_function"
      interpretation: |
        For each external call:
        - Is the target address controlled by user input?
        - Can the target call back into this contract?
        - What permissions does the call have?

    - id: 2
      action: ppr_expand
      description: "Find functions reachable during callback"
      seed: "${function_name}"
      depth: 3
      interpretation: |
        During a callback, which functions could be called?
        - Do any modify privileged state?
        - Do any have weaker access control?

    - id: 3
      action: lsp_references
      description: "Find all modifications to owner/admin"
      target: "owner"
      interpretation: |
        Can ownership be transferred during callback?
        Look for:
        - setOwner, transferOwnership functions
        - Direct assignment to owner variable

    - id: 4
      action: reason
      prompt: |
        Analyzing privilege escalation paths:

        External calls (step 1): {step_1_results}
        Reachable functions during callback (step 2): {step_2_results}
        Owner modifications (step 3): {step_3_results}

        Is there a path where:
        1. Attacker triggers external call
        2. Callback calls privileged function
        3. Attacker gains elevated permissions

  verdict_criteria:
    vulnerable: "Clear escalation path exists"
    uncertain: "Path exists but requires specific conditions"
    safe: "No viable escalation path"
```

## 5. IMPLEMENTATION

### 5.1 Investigation Pattern Executor

```python
# src/true_vkg/investigation/executor.py

from dataclasses import dataclass
from typing import Dict, Any, List, Optional
from enum import Enum

class InvestigationAction(Enum):
    EXPLORE_GRAPH = "explore_graph"
    LSP_REFERENCES = "lsp_references"
    LSP_DEFINITION = "lsp_definition"
    LSP_CALL_HIERARCHY = "lsp_call_hierarchy"
    PPR_EXPAND = "ppr_expand"
    READ_CODE = "read_code"
    REASON = "reason"
    SYNTHESIZE = "synthesize"

@dataclass
class InvestigationStep:
    id: int
    action: InvestigationAction
    description: str
    params: Dict[str, Any]
    interpretation: str

@dataclass
class StepResult:
    step_id: int
    action: InvestigationAction
    raw_output: Any
    llm_interpretation: str
    evidence: List[str]

@dataclass
class InvestigationResult:
    pattern_id: str
    verdict: str  # VULNERABLE | UNCERTAIN | SAFE
    confidence: int
    attack_path: Optional[str]
    evidence: List[str]
    recommendation: str
    step_results: List[StepResult]
    tokens_used: int
    cost_usd: float

class InvestigationExecutor:
    """Execute LLM investigation patterns."""

    def __init__(
        self,
        graph,
        lsp_client,
        ppr_engine,
        llm_provider,
        knowledge_manager,
    ):
        self.graph = graph
        self.lsp = lsp_client
        self.ppr = ppr_engine
        self.llm = llm_provider
        self.knowledge = knowledge_manager

    async def execute(
        self,
        pattern: Dict[str, Any],
        context: Dict[str, Any],
    ) -> InvestigationResult:
        """
        Execute an investigation pattern.

        Args:
            pattern: Parsed investigation pattern YAML
            context: Context including function, graph node, code

        Returns:
            InvestigationResult with verdict and evidence
        """
        # Check trigger conditions
        if not self._check_triggers(pattern, context):
            return InvestigationResult(
                pattern_id=pattern["id"],
                verdict="SKIPPED",
                confidence=100,
                attack_path=None,
                evidence=["Trigger conditions not met"],
                recommendation="",
                step_results=[],
                tokens_used=0,
                cost_usd=0.0,
            )

        # Execute investigation steps
        step_results = []
        accumulated_evidence = []

        for step_def in pattern["investigation"]["steps"]:
            step = self._parse_step(step_def, context)
            result = await self._execute_step(step, context, step_results)
            step_results.append(result)
            accumulated_evidence.extend(result.evidence)

        # Final synthesis
        verdict_result = await self._synthesize_verdict(
            pattern=pattern,
            step_results=step_results,
            context=context,
        )

        return InvestigationResult(
            pattern_id=pattern["id"],
            verdict=verdict_result["verdict"],
            confidence=verdict_result["confidence"],
            attack_path=verdict_result.get("attack_path"),
            evidence=accumulated_evidence + verdict_result.get("evidence", []),
            recommendation=verdict_result.get("recommendation", ""),
            step_results=step_results,
            tokens_used=sum(r.llm_interpretation_tokens for r in step_results),
            cost_usd=sum(r.cost for r in step_results),
        )

    def _check_triggers(
        self,
        pattern: Dict[str, Any],
        context: Dict[str, Any],
    ) -> bool:
        """Check if trigger conditions are met."""
        signals = pattern.get("trigger", {}).get("graph_signals", [])

        for signal in signals:
            property_name = signal.get("property")
            if property_name:
                node = context.get("node")
                if node and hasattr(node, property_name):
                    if not getattr(node, property_name, False):
                        return False
        return True

    async def _execute_step(
        self,
        step: InvestigationStep,
        context: Dict[str, Any],
        previous_results: List[StepResult],
    ) -> StepResult:
        """Execute a single investigation step."""

        if step.action == InvestigationAction.EXPLORE_GRAPH:
            raw = await self._action_explore_graph(step.params, context)
        elif step.action == InvestigationAction.LSP_REFERENCES:
            raw = await self._action_lsp_references(step.params, context)
        elif step.action == InvestigationAction.LSP_CALL_HIERARCHY:
            raw = await self._action_lsp_call_hierarchy(step.params, context)
        elif step.action == InvestigationAction.PPR_EXPAND:
            raw = await self._action_ppr_expand(step.params, context)
        elif step.action == InvestigationAction.READ_CODE:
            raw = await self._action_read_code(step.params, context)
        elif step.action == InvestigationAction.REASON:
            raw = await self._action_reason(step, context, previous_results)
        else:
            raw = {"error": f"Unknown action: {step.action}"}

        # Get LLM interpretation of results
        interpretation = await self._interpret_results(
            step=step,
            raw_output=raw,
            context=context,
        )

        return StepResult(
            step_id=step.id,
            action=step.action,
            raw_output=raw,
            llm_interpretation=interpretation["interpretation"],
            evidence=interpretation.get("evidence", []),
        )

    async def _action_explore_graph(
        self,
        params: Dict[str, Any],
        context: Dict[str, Any],
    ) -> Any:
        """Execute graph exploration."""
        query = params.get("graph_query", "")
        # Substitute context variables
        query = self._substitute_variables(query, context)
        return self.graph.query(query)

    async def _action_lsp_references(
        self,
        params: Dict[str, Any],
        context: Dict[str, Any],
    ) -> Any:
        """Find references using LSP."""
        target = self._substitute_variables(params.get("target", ""), context)
        return await self.lsp.find_references(target, context.get("file_path"))

    async def _action_lsp_call_hierarchy(
        self,
        params: Dict[str, Any],
        context: Dict[str, Any],
    ) -> Any:
        """Get call hierarchy using LSP."""
        target = self._substitute_variables(params.get("target", ""), context)
        return await self.lsp.incoming_calls(target, context.get("file_path"))

    async def _action_ppr_expand(
        self,
        params: Dict[str, Any],
        context: Dict[str, Any],
    ) -> Any:
        """Expand context using PPR."""
        seed = self._substitute_variables(params.get("seed", ""), context)
        depth = params.get("depth", 2)
        return self.ppr.expand(seed_node=seed, max_depth=depth)

    async def _action_read_code(
        self,
        params: Dict[str, Any],
        context: Dict[str, Any],
    ) -> Any:
        """Read code from file."""
        target = self._substitute_variables(params.get("target", ""), context)
        # Implementation would read the file
        return {"code": "...", "path": target}

    async def _action_reason(
        self,
        step: InvestigationStep,
        context: Dict[str, Any],
        previous_results: List[StepResult],
    ) -> Any:
        """Execute reasoning step with LLM."""
        prompt = step.params.get("prompt", "")

        # Substitute previous step results
        for prev in previous_results:
            prompt = prompt.replace(
                f"{{step_{prev.step_id}_results}}",
                prev.llm_interpretation
            )

        prompt = self._substitute_variables(prompt, context)

        response = await self.llm.analyze(
            context="",
            prompt=prompt,
        )
        return response

    async def _interpret_results(
        self,
        step: InvestigationStep,
        raw_output: Any,
        context: Dict[str, Any],
    ) -> Dict[str, Any]:
        """Get LLM interpretation of step results."""
        interpretation_prompt = f"""
        Step: {step.description}

        Raw Output:
        {raw_output}

        Interpretation Guide:
        {step.interpretation}

        Provide:
        1. A concise interpretation of what the output shows
        2. Any evidence of vulnerability or safety
        3. Key findings relevant to the investigation hypothesis
        """

        response = await self.llm.analyze(
            context="",
            prompt=interpretation_prompt,
        )

        return {
            "interpretation": response.get("content", ""),
            "evidence": self._extract_evidence(response.get("content", "")),
        }

    async def _synthesize_verdict(
        self,
        pattern: Dict[str, Any],
        step_results: List[StepResult],
        context: Dict[str, Any],
    ) -> Dict[str, Any]:
        """Synthesize final verdict from all step results."""

        # Get relevant knowledge
        knowledge_block = self.knowledge.get_for_pattern(pattern["id"])

        synthesis_prompt = f"""
        ## Investigation: {pattern['name']}
        ## Hypothesis: {pattern['investigation']['hypothesis']}

        ## Step Results:
        {"".join(f"Step {r.step_id}: {r.llm_interpretation}" for r in step_results)}

        ## Verdict Criteria:
        VULNERABLE if: {pattern['investigation']['verdict_criteria']['vulnerable']}
        UNCERTAIN if: {pattern['investigation']['verdict_criteria']['uncertain']}
        SAFE if: {pattern['investigation']['verdict_criteria']['safe']}

        ## Relevant Knowledge:
        {knowledge_block.content if knowledge_block else "No specific knowledge available"}

        Based on all the above, provide your verdict as JSON:
        {{
            "verdict": "VULNERABLE" | "UNCERTAIN" | "SAFE",
            "confidence": 0-100,
            "attack_path": "Description of attack if vulnerable",
            "evidence": ["List of supporting evidence"],
            "recommendation": "Fix if vulnerable, otherwise why safe"
        }}
        """

        response = await self.llm.analyze(
            context="",
            prompt=synthesis_prompt,
        )

        import json
        try:
            return json.loads(response.get("content", "{}"))
        except json.JSONDecodeError:
            return {
                "verdict": "UNCERTAIN",
                "confidence": 0,
                "evidence": ["Failed to parse LLM response"],
                "recommendation": "Manual review required",
            }

    def _substitute_variables(self, text: str, context: Dict[str, Any]) -> str:
        """Substitute ${variable} placeholders with context values."""
        for key, value in context.items():
            text = text.replace(f"${{{key}}}", str(value))
        return text

    def _extract_evidence(self, text: str) -> List[str]:
        """Extract evidence statements from LLM response."""
        # Simple implementation - look for bullet points or numbered items
        lines = text.split("\n")
        evidence = []
        for line in lines:
            line = line.strip()
            if line.startswith(("-", "*", "•")) or line[:2].isdigit():
                evidence.append(line.lstrip("-*• 0123456789."))
        return evidence
```
