## 1. Overview

This document defines **routing rules for tool disagreement** in BSKG multi-tool analysis. When BSKG integrates outputs from multiple tools (VKG, Slither, Aderyn, etc.), disagreements must be detected, classified, and routed appropriately.

**Philosophy Foundation (from `docs/PHILOSOPHY.md`):**
- "VKG integrates and normalizes outputs from multiple tools"
- "Deduplication and disagreement tracking are first-class concerns"
- "If tools disagree on a finding, mark it as disputed"
- "Disputed beads automatically enter multi-agent debate"
- "Evidence from each tool is preserved and attributed"

**Key Constraint:** This specification is documentation only. No changes to `builder.py` are allowed. Disagreement detection and routing is a post-graph operation applied during finding assembly.

## 2. What Constitutes Tool Disagreement

### 2.1 Disagreement Definition

**Tool disagreement** occurs when two or more tools analyze the same code location but produce conflicting outputs. A disagreement is NOT the same as "one tool found it, another didn't"--that's a coverage gap, not a disagreement.

### 2.2 Disagreement Types

| Type | ID | Definition | Example |
|------|-----|------------|---------|
| **Location Match, Verdict Conflict** | DIS-001 | Same location, one tool flags, another explicitly clears | BSKG flags reentrancy at L42; Slither marks L42 as "safe" |
| **Location Match, Class Mismatch** | DIS-002 | Same location, different vulnerability classifications | VKG: reentrancy; Aderyn: access-control at same function |
| **Location Match, Severity Conflict** | DIS-003 | Same location and class, different severity levels | VKG: critical; Slither: medium |
| **Partial Overlap** | DIS-004 | Overlapping line ranges with different conclusions | VKG: L40-50 vulnerable; Slither: L45-55 safe |
| **Semantic Disagreement** | DIS-005 | Same pattern matched, different semantic interpretation | VKG: state-write-after-call; Slither: no reentrancy risk |
| **Guard Interpretation** | DIS-006 | Tools disagree on whether a guard is effective | VKG: guard insufficient; Aderyn: guard protects |

### 2.3 Non-Disagreements (Coverage Gaps)

These are NOT disagreements and have different routing:

| Scenario | Classification | Routing |
|----------|---------------|---------|
| BSKG finds issue, Slither has no output for location | Coverage gap | Use BSKG finding as-is |
| Slither finds issue, BSKG has no match | External finding | Import with `tool_sources: [slither]` |
| Tool failed to run | Tool unavailable | Record in `missing_tools`, lower confidence |
| Different locations, same pattern | Independent findings | Keep distinct beads |

### 2.4 Location Matching Algorithm

Two findings are at the "same location" if:

```python
def locations_match(loc_a: dict, loc_b: dict, tolerance: int = 3) -> bool:
    """
    Determine if two locations refer to the same code region.

    Args:
        loc_a: Location dict with file, contract, function, lines
        loc_b: Location dict from another tool
        tolerance: Line number tolerance for partial overlap

    Returns:
        True if locations match within tolerance
    """
    # Must be same file
    if normalize_path(loc_a.get("file")) != normalize_path(loc_b.get("file")):
        return False

    # Must be same contract (if specified)
    if loc_a.get("contract") and loc_b.get("contract"):
        if loc_a["contract"] != loc_b["contract"]:
            return False

    # Must be same function (if specified)
    if loc_a.get("function") and loc_b.get("function"):
        if loc_a["function"] != loc_b["function"]:
            return False

    # Check line overlap
    lines_a = loc_a.get("lines", [0, 0])
    lines_b = loc_b.get("lines", [0, 0])

    # Expand ranges by tolerance
    range_a = range(max(0, lines_a[0] - tolerance), lines_a[1] + tolerance + 1)
    range_b = range(max(0, lines_b[0] - tolerance), lines_b[1] + tolerance + 1)

    # Check if ranges overlap
    return bool(set(range_a) & set(range_b))


def normalize_path(path: str) -> str:
    """Normalize file path for comparison."""
    if not path:
        return ""
    # Remove leading ./ or /
    path = path.lstrip("./")
    # Convert to lowercase for case-insensitive comparison
    return path.lower()
```

## 3. Tool Agreement Scoring

### 3.1 Agreement Score Model

Each finding receives an **agreement score** based on tool consensus:

| Scenario | Agreement Score | Interpretation |
|----------|-----------------|----------------|
| All tools agree | +1.0 | Strong consensus |
| Majority agree, minority silent | +0.75 | Good agreement |
| Majority agree, minority disagree | +0.25 | Partial agreement, dispute |
| Split (50/50) | 0.0 | Disputed, needs debate |
| Majority disagree with BSKG | -0.25 | BSKG finding questioned |
| All tools explicitly disagree | -0.5 | Strong counter-evidence |

### 3.2 Tool Weights

Not all tools are equal. Weight by tool capability and accuracy:

| Tool | Weight | Rationale |
|------|--------|-----------|
| BSKG | 1.0 | Primary system with semantic analysis |
| Slither | 0.9 | High accuracy, comprehensive detectors |
| Aderyn | 0.8 | Good accuracy, newer tool |
| Mythril | 0.7 | Symbolic execution, more FPs |
| Custom | 0.5 | Unknown accuracy, trust less |

### 3.3 Agreement Score Calculation

```python
def compute_agreement_score(
    finding: dict,
    tool_results: dict[str, list[dict]]
) -> tuple[float, list[str]]:
    """
    Compute tool agreement score for a finding.

    Args:
        finding: The BSKG finding to score
        tool_results: Dict of tool_name -> list of findings from that tool

    Returns:
        (score, list_of_disagreeing_tools)
    """
    TOOL_WEIGHTS = {
        "vkg": 1.0,
        "slither": 0.9,
        "aderyn": 0.8,
        "mythril": 0.7,
    }

    loc = finding.get("location", {})
    vuln_class = finding.get("vulnerability_class", "")

    # Track tool verdicts
    agree_weight = TOOL_WEIGHTS.get("vkg", 1.0)  # BSKG always agrees with itself
    disagree_weight = 0.0
    total_weight = agree_weight
    disagreeing_tools = []

    for tool, results in tool_results.items():
        if tool == "vkg":
            continue

        tool_weight = TOOL_WEIGHTS.get(tool, 0.5)
        total_weight += tool_weight

        # Find matching location in tool results
        matching = [r for r in results if locations_match(loc, r.get("location", {}))]

        if not matching:
            # Tool has no opinion on this location
            continue

        # Check for agreement or disagreement
        for match in matching:
            match_class = match.get("vulnerability_class", "")
            match_verdict = match.get("verdict", "")

            if match_verdict == "safe" or match_verdict == "false_positive":
                # Tool explicitly disagrees
                disagree_weight += tool_weight
                disagreeing_tools.append(tool)
                break
            elif match_class and match_class != vuln_class:
                # Different classification = partial disagreement
                disagree_weight += tool_weight * 0.5
                disagreeing_tools.append(f"{tool}:class_mismatch")
            else:
                # Tool agrees
                agree_weight += tool_weight

    # Normalize to [-0.5, 1.0] range
    if total_weight == 0:
        return 0.0, []

    score = (agree_weight - disagree_weight) / total_weight

    return round(score, 3), disagreeing_tools
```

## 4. The `disputed` Flag Behavior

### 4.1 When to Set `disputed: true`

The `disputed` flag is set on an evidence packet when:

```python
def should_mark_disputed(
    finding: dict,
    tool_results: dict[str, list[dict]]
) -> tuple[bool, str]:
    """
    Determine if a finding should be marked as disputed.

    Returns:
        (should_dispute, reason)
    """
    agreement_score, disagreeing_tools = compute_agreement_score(finding, tool_results)
    loc = finding.get("location", {})

    # Check for explicit disagreement types
    for tool, results in tool_results.items():
        if tool == "vkg":
            continue

        for result in results:
            if not locations_match(loc, result.get("location", {})):
                continue

            # DIS-001: Verdict conflict
            if result.get("verdict") in ("safe", "false_positive", "cleared"):
                return True, f"Verdict conflict: {tool} marks as {result.get('verdict')}"

            # DIS-003: Severity conflict (2+ levels difference)
            our_severity = finding.get("severity", "medium")
            their_severity = result.get("severity", "medium")
            if severity_distance(our_severity, their_severity) >= 2:
                return True, f"Severity conflict: VKG={our_severity}, {tool}={their_severity}"

            # DIS-005: Semantic disagreement (same class, different conclusion)
            if (result.get("vulnerability_class") == finding.get("vulnerability_class") and
                result.get("risk_assessment", "") != "" and
                "no risk" in result.get("risk_assessment", "").lower()):
                return True, f"Semantic disagreement: {tool} finds no risk"

            # DIS-006: Guard interpretation
            if ("guard" in result.get("notes", "").lower() and
                "protected" in result.get("notes", "").lower()):
                if not finding.get("properties", {}).get("has_guard"):
                    return True, f"Guard interpretation: {tool} sees protection, BSKG does not"

    # Agreement score threshold
    if agreement_score <= 0.25:
        return True, f"Low agreement score: {agreement_score}"

    return False, ""


def severity_distance(sev_a: str, sev_b: str) -> int:
    """Calculate severity level distance."""
    LEVELS = {"critical": 4, "high": 3, "medium": 2, "low": 1, "info": 0}
    return abs(LEVELS.get(sev_a, 2) - LEVELS.get(sev_b, 2))
```

### 4.2 `disputed` Flag Effects

When `disputed: true` is set on an evidence packet:

| Effect | Description |
|--------|-------------|
| **Bucket Override** | Finding is forced to `uncertain` regardless of score (per P0.P.2) |
| **Auto-Debate Routing** | Finding is automatically queued for multi-agent debate |
| **Evidence Preservation** | All tool outputs are preserved in `tool_evidence` field |
| **UI Indicator** | Report displays "DISPUTED" badge next to finding |
| **SLA Escalation** | If not resolved within debate SLA, escalates to human review |

### 4.3 Clearing the `disputed` Flag

The `disputed` flag can only be cleared by:

1. **Debate Resolution**: Attacker/Defender/Verifier reach consensus
2. **Test Verification**: Exploit test passes or fails definitively
3. **Human Override**: Auditor manually resolves the dispute
4. **Tool Update**: Re-running tools produces agreement


## 5. Routing Targets

### 5.1 Routing Decision Matrix

Based on disagreement type and agreement score, findings are routed to specific targets:

| Agreement Score | Disagreement Type | Routing Target | Priority |
|-----------------|-------------------|----------------|----------|
| >= 0.75 | None | `auto-confirm` | LOW |
| 0.50 - 0.74 | None | `uncertain` | NORMAL |
| 0.25 - 0.49 | Any | `debate` | HIGH |
| < 0.25 | DIS-001 (verdict conflict) | `escalate` | CRITICAL |
| < 0.25 | DIS-003 (severity conflict) | `debate` | HIGH |
| < 0.25 | DIS-005 (semantic) | `debate` | HIGH |
| < 0.25 | DIS-006 (guard) | `debate` | HIGH |
| Any | Multiple disagreements | `convoy-debate` | HIGH |

### 5.2 Routing Target Definitions

#### `auto-confirm`
- **Description**: Strong tool agreement; proceed without debate
- **Action**: Set `confidence_bucket: likely` (Tier A maximum)
- **Next Step**: Queue for test generation if severity >= high
- **SLA**: None (immediate)

#### `uncertain`
- **Description**: Moderate confidence; manual review recommended
- **Action**: Set `confidence_bucket: uncertain`, `disputed: false`
- **Next Step**: Include in report with verification flag
- **SLA**: Review within 24 hours

#### `debate`
- **Description**: Tools disagree; multi-agent debate required
- **Action**: Set `disputed: true`, queue for debate convoy
- **Next Step**: Attacker constructs exploit, Defender searches guards
- **SLA**: Debate completion within 4 hours, then escalate

#### `escalate`
- **Description**: Critical disagreement requiring human attention
- **Action**: Set `disputed: true`, add to human review queue
- **Next Step**: Human auditor reviews all tool evidence
- **SLA**: Human review within 8 hours for critical severity

#### `convoy-debate`
- **Description**: Multiple related disagreements; batch processing
- **Action**: Create or join convoy, debate as group
- **Next Step**: Convoy-level debate with shared context
- **SLA**: Convoy completion within 8 hours

### 5.3 Routing Decision Function

```python
from enum import Enum
from dataclasses import dataclass

class RouteTarget(Enum):
    AUTO_CONFIRM = "auto-confirm"
    UNCERTAIN = "uncertain"
    DEBATE = "debate"
    ESCALATE = "escalate"
    CONVOY_DEBATE = "convoy-debate"

class RoutePriority(Enum):
    LOW = 1
    NORMAL = 2
    HIGH = 3
    CRITICAL = 4

@dataclass
class RoutingDecision:
    target: RouteTarget
    priority: RoutePriority
    reason: str
    disagreement_types: list[str]
    disputed: bool
    convoy_id: str | None = None

def route_finding(
    finding: dict,
    tool_results: dict[str, list[dict]],
    convoy_context: dict | None = None
) -> RoutingDecision:
    """
    Determine routing target for a finding based on tool agreement.

    Args:
        finding: The BSKG finding to route
        tool_results: Dict of tool_name -> list of findings
        convoy_context: Optional convoy context for batch processing

    Returns:
        RoutingDecision with target, priority, and metadata
    """
    # Compute agreement and check for disputes
    agreement_score, disagreeing_tools = compute_agreement_score(finding, tool_results)
    should_dispute, dispute_reason = should_mark_disputed(finding, tool_results)

    # Identify disagreement types
    disagreement_types = identify_disagreement_types(finding, tool_results)

    # Check for convoy-level patterns
    if convoy_context:
        similar_disputes = convoy_context.get("similar_disputes", 0)
        if similar_disputes >= 3:
            return RoutingDecision(
                target=RouteTarget.CONVOY_DEBATE,
                priority=RoutePriority.HIGH,
                reason=f"Multiple related disagreements ({similar_disputes}) in same category",
                disagreement_types=disagreement_types,
                disputed=True,
                convoy_id=convoy_context.get("convoy_id")
            )

    # Strong agreement - auto-confirm
    if agreement_score >= 0.75 and not should_dispute:
        return RoutingDecision(
            target=RouteTarget.AUTO_CONFIRM,
            priority=RoutePriority.LOW,
            reason="Strong tool agreement",
            disagreement_types=[],
            disputed=False
        )

    # Moderate agreement - uncertain
    if agreement_score >= 0.50 and not should_dispute:
        return RoutingDecision(
            target=RouteTarget.UNCERTAIN,
            priority=RoutePriority.NORMAL,
            reason="Moderate tool agreement; verification recommended",
            disagreement_types=[],
            disputed=False
        )

    # Critical verdict conflict - escalate immediately
    if "DIS-001" in disagreement_types and agreement_score < 0.25:
        severity = finding.get("severity", "medium")
        if severity in ("critical", "high"):
            return RoutingDecision(
                target=RouteTarget.ESCALATE,
                priority=RoutePriority.CRITICAL,
                reason=f"Critical verdict conflict: {dispute_reason}",
                disagreement_types=disagreement_types,
                disputed=True
            )

    # Any disagreement with low agreement - debate
    if should_dispute or agreement_score < 0.50:
        return RoutingDecision(
            target=RouteTarget.DEBATE,
            priority=RoutePriority.HIGH,
            reason=dispute_reason or f"Low agreement score ({agreement_score})",
            disagreement_types=disagreement_types,
            disputed=True
        )

    # Default fallback
    return RoutingDecision(
        target=RouteTarget.UNCERTAIN,
        priority=RoutePriority.NORMAL,
        reason="Unable to determine clear agreement; defaulting to uncertain",
        disagreement_types=disagreement_types,
        disputed=should_dispute
    )


def identify_disagreement_types(
    finding: dict,
    tool_results: dict[str, list[dict]]
) -> list[str]:
    """Identify all applicable disagreement types for a finding."""
    types = []
    loc = finding.get("location", {})

    for tool, results in tool_results.items():
        if tool == "vkg":
            continue

        for result in results:
            if not locations_match(loc, result.get("location", {})):
                continue

            # DIS-001: Verdict conflict
            if result.get("verdict") in ("safe", "false_positive"):
                types.append("DIS-001")

            # DIS-002: Class mismatch
            if (result.get("vulnerability_class") and
                result["vulnerability_class"] != finding.get("vulnerability_class")):
                types.append("DIS-002")

            # DIS-003: Severity conflict
            if severity_distance(finding.get("severity", ""), result.get("severity", "")) >= 2:
                types.append("DIS-003")

            # DIS-004: Partial overlap (checked in locations_match)
            # This is implicit if we get here with line tolerance

            # DIS-005: Semantic disagreement
            if ("no risk" in result.get("notes", "").lower() or
                "not vulnerable" in result.get("notes", "").lower()):
                types.append("DIS-005")

            # DIS-006: Guard interpretation
            if ("protected" in result.get("notes", "").lower() or
                "guarded" in result.get("notes", "").lower()):
                types.append("DIS-006")

    return list(set(types))  # Deduplicate
```

## 6. Convoy Routing for Graph-Quality Issues

### 6.1 Convoy Definition

A **convoy** is a batch of related beads that share context and can be processed together. Convoy routing applies when:

1. Multiple findings in the same contract have similar disagreements
2. Multiple findings share the same vulnerability class with disputes
3. Systematic tool disagreement pattern is detected (tool A always disagrees with BSKG on class X)

### 6.2 Convoy Creation Triggers

| Trigger | Threshold | Convoy Type | Description |
|---------|-----------|-------------|-------------|
| Same-contract disputes | >= 3 | `contract-dispute` | Multiple disputes in one contract |
| Same-class disputes | >= 5 | `class-dispute` | Many disputes in one vuln class |
| Tool-specific pattern | >= 10 | `tool-calibration` | One tool systematically disagrees |
| Graph quality issue | >= 5 | `graph-quality` | Missing evidence across findings |

### 6.3 Convoy Data Model

```python
@dataclass
class DisputeConvoy:
    """A convoy for batch processing disputed findings."""

    id: str                          # convoy-dispute-{timestamp}
    convoy_type: str                 # See triggers above
    beads: list[str]                 # List of finding IDs
    shared_context: dict             # Common context for all beads
    status: str                      # pending, in_debate, resolved, escalated
    created_at: str
    resolution: dict | None = None   # Resolution details when complete

    @classmethod
    def create(cls, convoy_type: str, findings: list[dict]) -> "DisputeConvoy":
        """Create a new dispute convoy."""
        import hashlib
        from datetime import datetime

        timestamp = datetime.utcnow().strftime("%Y%m%dT%H%M%SZ")
        id_hash = hashlib.sha256(
            f"{convoy_type}:{timestamp}".encode()
        ).hexdigest()[:8]

        return cls(
            id=f"convoy-dispute-{id_hash}",
            convoy_type=convoy_type,
            beads=[f.get("id") for f in findings],
            shared_context=extract_shared_context(findings),
            status="pending",
            created_at=timestamp,
            resolution=None
        )

def extract_shared_context(findings: list[dict]) -> dict:
    """Extract shared context across findings for convoy."""
    contracts = set()
    vuln_classes = set()
    tools_involved = set()

    for f in findings:
        loc = f.get("location", {})
        contracts.add(loc.get("contract", "unknown"))
        vuln_classes.add(f.get("vulnerability_class", ""))
        tools_involved.update(f.get("tool_sources", ["vkg"]))

    return {
        "contracts": list(contracts),
        "vulnerability_classes": list(vuln_classes),
        "tools_involved": list(tools_involved),
        "finding_count": len(findings),
        "common_patterns": identify_common_patterns(findings),
    }

def identify_common_patterns(findings: list[dict]) -> list[str]:
    """Identify patterns common to disputed findings."""
    patterns = []

    # Check for common vulnerability types
    vuln_classes = [f.get("vulnerability_class", "") for f in findings]
    from collections import Counter
    class_counts = Counter(vuln_classes)
    for cls, count in class_counts.most_common():
        if count >= 2 and cls:
            patterns.append(f"common_class:{cls}")

    # Check for common disagreement types
    all_dispute_types = []
    for f in findings:
        all_dispute_types.extend(f.get("disagreement_types", []))
    type_counts = Counter(all_dispute_types)
    for dtype, count in type_counts.most_common():
        if count >= 2:
            patterns.append(f"common_disagreement:{dtype}")

    return patterns
```

### 6.4 Convoy Routing Function

```python
def check_convoy_routing(
    new_finding: dict,
    pending_disputes: list[dict],
    active_convoys: list[DisputeConvoy]
) -> DisputeConvoy | None:
    """
    Check if a finding should be routed to an existing or new convoy.

    Args:
        new_finding: The newly disputed finding
        pending_disputes: List of pending disputed findings not in convoy
        active_convoys: List of active dispute convoys

    Returns:
        DisputeConvoy to route to, or None if individual routing
    """
    # Check if should join existing convoy
    for convoy in active_convoys:
        if convoy.status == "pending" and should_join_convoy(new_finding, convoy):
            return convoy

    # Check if should create new convoy
    similar_pending = find_similar_disputes(new_finding, pending_disputes)

    # Same-contract trigger
    same_contract = [
        f for f in similar_pending
        if f.get("location", {}).get("contract") ==
           new_finding.get("location", {}).get("contract")
    ]
    if len(same_contract) >= 2:  # 3 with new = threshold
        convoy = DisputeConvoy.create(
            "contract-dispute",
            [new_finding] + same_contract
        )
        return convoy

    # Same-class trigger
    same_class = [
        f for f in similar_pending
        if f.get("vulnerability_class") == new_finding.get("vulnerability_class")
    ]
    if len(same_class) >= 4:  # 5 with new = threshold
        convoy = DisputeConvoy.create(
            "class-dispute",
            [new_finding] + same_class
        )
        return convoy

    # Graph-quality trigger (missing evidence pattern)
    if new_finding.get("request_more_context"):
        incomplete = [
            f for f in similar_pending
            if f.get("request_more_context")
        ]
        if len(incomplete) >= 4:
            convoy = DisputeConvoy.create(
                "graph-quality",
                [new_finding] + incomplete
            )
            return convoy

    return None


def should_join_convoy(finding: dict, convoy: DisputeConvoy) -> bool:
    """Check if finding should join an existing convoy."""
    context = convoy.shared_context

    # Contract match for contract-dispute convoys
    if convoy.convoy_type == "contract-dispute":
        finding_contract = finding.get("location", {}).get("contract", "")
        if finding_contract in context.get("contracts", []):
            return True

    # Class match for class-dispute convoys
    if convoy.convoy_type == "class-dispute":
        finding_class = finding.get("vulnerability_class", "")
        if finding_class in context.get("vulnerability_classes", []):
            return True

    # Graph-quality match
    if convoy.convoy_type == "graph-quality":
        if finding.get("request_more_context"):
            return True

    return False


def find_similar_disputes(
    finding: dict,
    pending: list[dict]
) -> list[dict]:
    """Find pending disputes similar to the given finding."""
    similar = []

    finding_loc = finding.get("location", {})
    finding_class = finding.get("vulnerability_class", "")

    for p in pending:
        p_loc = p.get("location", {})
        p_class = p.get("vulnerability_class", "")

        # Same contract or same class is similar
        if (p_loc.get("contract") == finding_loc.get("contract") or
            p_class == finding_class):
            similar.append(p)

    return similar
```

## 7. Escalation Rules to Multi-Agent Debate

### 7.1 Debate Entry Criteria

A disputed finding enters multi-agent debate when:

| Criterion | Routing Target | Debate Type |
|-----------|---------------|-------------|
| `disputed: true` from routing | `debate` | Standard debate |
| Convoy formed | `convoy-debate` | Batch debate |
| SLA timeout on uncertain | `debate` | Escalation debate |
| Human request | Any | Priority debate |

### 7.2 Debate Protocol Integration

From `docs/PHILOSOPHY.md`, the debate protocol requires:

1. **Attacker**: Construct exploit path and attack narrative
2. **Defender**: Search for guards, invariants, mitigating logic
3. **Verifier**: Cross-check reasoning and evidence consistency
4. **Test Builder**: Generate PoC if applicable

```python
@dataclass
class DebateInput:
    """Input package for multi-agent debate."""

    finding_id: str
    finding: dict                    # Full finding with evidence packet
    disputed_reason: str             # Why this is disputed
    disagreement_types: list[str]    # DIS-001, DIS-002, etc.
    tool_evidence: dict[str, dict]   # Evidence from each tool
    agreement_score: float
    convoy_id: str | None
    shared_context: dict | None      # If part of convoy

    def to_debate_prompt(self) -> dict:
        """Prepare debate input for agents."""
        return {
            "finding_summary": {
                "id": self.finding_id,
                "location": self.finding.get("location"),
                "vulnerability_class": self.finding.get("vulnerability_class"),
                "severity": self.finding.get("severity"),
                "behavioral_signature": self.finding.get("behavioral_signature"),
            },
            "dispute_context": {
                "reason": self.disputed_reason,
                "disagreement_types": self.disagreement_types,
                "agreement_score": self.agreement_score,
            },
            "evidence": {
                "vkg": self.tool_evidence.get("vkg", {}),
                "other_tools": {
                    k: v for k, v in self.tool_evidence.items() if k != "vkg"
                },
            },
            "instructions": self._generate_instructions(),
        }

    def _generate_instructions(self) -> dict:
        """Generate role-specific instructions."""
        instructions = {
            "attacker": (
                "Analyze the BSKG finding and construct a concrete exploit path. "
                "Consider the disagreement from other tools and explain why "
                "the vulnerability is still exploitable despite their assessment."
            ),
            "defender": (
                "Search for guards, invariants, or mitigating logic that the "
                "other tools may have detected. Explain whether these protections "
                "are sufficient to prevent exploitation."
            ),
            "verifier": (
                "Cross-check the evidence from all tools. Identify any "
                "reasoning errors or missing evidence. Determine if the "
                "disagreement stems from different analysis scope or actual conflict."
            ),
        }

        # Customize based on disagreement type
        if "DIS-006" in self.disagreement_types:
            instructions["defender"] = (
                "Another tool claims this code is protected by a guard. "
                "Verify if this guard is effective. Consider edge cases, "
                "bypass conditions, and incomplete protection scenarios."
            )

        if "DIS-003" in self.disagreement_types:
            instructions["verifier"] = (
                "Tools disagree on severity. Analyze the impact and "
                "exploitability to determine the correct severity level. "
                "Consider real-world attack scenarios."
            )

        return instructions


def prepare_debate_input(
    finding: dict,
    routing_decision: RoutingDecision,
    tool_results: dict[str, list[dict]]
) -> DebateInput:
    """Prepare input for multi-agent debate."""
    # Extract evidence from each tool
    tool_evidence = {"vkg": finding}
    loc = finding.get("location", {})

    for tool, results in tool_results.items():
        if tool == "vkg":
            continue
        matching = [r for r in results if locations_match(loc, r.get("location", {}))]
        if matching:
            tool_evidence[tool] = matching[0]  # Use first match

    return DebateInput(
        finding_id=finding.get("id", ""),
        finding=finding,
        disputed_reason=routing_decision.reason,
        disagreement_types=routing_decision.disagreement_types,
        tool_evidence=tool_evidence,
        agreement_score=compute_agreement_score(finding, tool_results)[0],
        convoy_id=routing_decision.convoy_id,
        shared_context=None  # Populated if convoy
    )
```

### 7.4 Debate SLA and Escalation

| Severity | Debate SLA | Escalation Trigger |
|----------|------------|-------------------|
| Critical | 2 hours | Auto-escalate to human |
| High | 4 hours | Auto-escalate to human |
| Medium | 8 hours | Auto-escalate or mark inconclusive |
| Low | 24 hours | Mark inconclusive if no consensus |

```python
def check_debate_sla(
    debate_started: datetime,
    finding_severity: str
) -> tuple[bool, str]:
    """
    Check if debate has exceeded SLA.

    Returns:
        (exceeded, action)
    """
    SLA_HOURS = {
        "critical": 2,
        "high": 4,
        "medium": 8,
        "low": 24,
    }

    sla_hours = SLA_HOURS.get(finding_severity, 8)
    deadline = debate_started + timedelta(hours=sla_hours)

    if datetime.utcnow() > deadline:
        if finding_severity in ("critical", "high"):
            return True, "escalate_human"
        else:
            return True, "mark_inconclusive"

    return False, "continue"
```
