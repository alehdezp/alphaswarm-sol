## 1. Overview

This document specifies the **test requirements** for deduplication rule validation. These tests ensure that findings from multiple sources (patterns, tools) are correctly deduplicated according to the rules defined in `docs/PHILOSOPHY.md`.

**Key Principle:** These are TEST REQUIREMENTS (what to test), not test implementations. Actual test code will be written when the deduplication system is implemented.

**Philosophy Reference (from docs/PHILOSOPHY.md):**

```
Deduplication rules are explicit and reproducible:
- Normalize by file + contract + function + line range + vulnerability class
- Same location and class -> merge into one bead
- Same location, different class -> link as a cluster
- Different locations, same pattern -> keep distinct beads
```

**Test Scope:**
- Location normalization tests
- Same location/same class merge tests
- Same location/different class cluster tests
- Different location/same pattern distinct tests
- Cross-tool deduplication tests
- Edge case tests (fuzzy matching, overlapping ranges)

## 3. Deduplication Rules (from PHILOSOPHY.md)

### 3.1 Normalization Key

The deduplication key is computed from:

```
normalize_key = {
    file: string (normalized path),
    contract: string (contract name),
    function: string (function name),
    line_range: [int, int] (start, end lines),
    vulnerability_class: string (class identifier)
}
```

### 3.2 Rule Summary

| Condition | Action | Result |
|-----------|--------|--------|
| Same location + same class | **MERGE** | Single bead with combined evidence |
| Same location + different class | **CLUSTER** | Linked beads in same cluster |
| Different location + same pattern | **DISTINCT** | Separate beads |
| Different location + different class | **DISTINCT** | Separate beads (no relation) |

### 3.3 Normalization Rules

| Component | Normalization Rule |
|-----------|-------------------|
| `file` | Absolute path, lowercase, normalized separators |
| `contract` | Exact match (case-sensitive) |
| `function` | Exact match (case-sensitive) |
| `line_range` | Canonical form [min, max] |
| `vulnerability_class` | Canonical class ID (e.g., `reentrancy`, `access-control`) |

## 15. Appendix: Deduplication Algorithm Pseudocode

```python
def deduplicate(findings: list[Finding]) -> list[Bead]:
    """
    Deduplicate findings according to PHILOSOPHY.md rules.

    1. Normalize all locations
    2. Group by normalized location
    3. Within each location group:
       - Same class -> MERGE into single bead
       - Different classes -> CREATE cluster of beads
    4. Return deduplicated beads
    """
    # Step 1: Normalize
    normalized = [(f, normalize_location(f)) for f in findings]

    # Step 2: Group by location
    location_groups = group_by(normalized, key=lambda x: x[1])

    beads = []

    for loc_key, findings_at_loc in location_groups.items():
        # Step 3: Within location, group by class
        class_groups = group_by(findings_at_loc, key=lambda x: x[0].vuln_class)

        if len(class_groups) == 1:
            # Same class at same location -> MERGE
            merged_bead = merge_findings(list(class_groups.values())[0])
            beads.append(merged_bead)
        else:
            # Different classes at same location -> CLUSTER
            cluster_id = generate_cluster_id()
            for cls, cls_findings in class_groups.items():
                merged_bead = merge_findings(cls_findings)
                merged_bead.cluster_id = cluster_id
                beads.append(merged_bead)

    return beads


def merge_findings(findings: list[Finding]) -> Bead:
    """Merge multiple findings into a single bead."""
    # Take highest confidence
    confidence = max(f.confidence for f in findings)
    # Take highest severity
    severity = max(f.severity for f in findings)
    # Combine tool sources
    tool_sources = list(set(t for f in findings for t in f.tool_sources))
    # Combine evidence
    evidence = combine_evidence([f.evidence for f in findings])
    # Disputed if any is disputed
    disputed = any(f.disputed for f in findings)

    return Bead(
        confidence=confidence,
        severity=severity,
        tool_sources=tool_sources,
        evidence=evidence,
        disputed=disputed,
        source_findings=[f.id for f in findings]
    )
```
