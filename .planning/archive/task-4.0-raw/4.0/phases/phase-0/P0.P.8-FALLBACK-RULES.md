## 1. Overview

This document specifies the **fallback rules** for evidence packet assembly when required data is missing, tools are unavailable, or context budgets are constrained. These rules ensure graceful degradation rather than hard failure, while maintaining audit trails and confidence penalties.

**Philosophy Foundation (from `docs/PHILOSOPHY.md`):**
- "If required fields are missing, set `request_more_context: true`"
- "If tools are unavailable, record them in `missing_tools` and lower confidence"
- "If sources disagree, set `disputed: true` and route to debate"
- "If context budget is tight, drop to evidence packet and skip full bead"

**Key Constraint:** This specification is documentation only. No changes to `builder.py` are allowed. Fallback logic is a post-graph operation applied during finding assembly.

---

## 2. Fallback Philosophy

### 2.1 Core Principles

1. **Graceful Degradation Over Hard Failure**: The system should always produce *some* output, even if incomplete. Silent failures are worse than flagged incomplete data.

2. **Transparent Confidence Penalties**: Every fallback application results in a documented confidence penalty. Users see exactly why confidence is reduced.

3. **Best Available Data First**: Fallbacks follow a priority chain--use the best available source before falling back to defaults.

4. **Audit Trail Preservation**: Every fallback decision is logged with reason, timestamp, and penalty applied.

5. **Validation After Fallback**: After fallbacks are applied, the packet is re-validated to ensure it meets minimum completeness requirements.

### 2.2 Fallback vs. Error

| Scenario | Classification | Action |
|----------|---------------|--------|
| Field derivable from alternate source | Fallback | Use alternate source, log |
| Field has safe default value | Fallback | Apply default, penalty |
| Field is critical and completely unavailable | Soft Error | Flag `request_more_context`, continue |
| Finding ID missing | Hard Error | Cannot create packet, reject |
| System/infrastructure failure | Hard Error | Escalate, do not produce packet |

## 3. Fallback Values for Required Fields

### 3.1 Required Field Fallback Registry

From P0.P.1 Section 6, each required field has a defined fallback:

| Field | Primary Source | Fallback Chain | Final Default | Priority |
|-------|---------------|----------------|---------------|----------|
| `id` | Generated | N/A | Always computed | 1 |
| `finding_id` | Finding.id | N/A | **HARD ERROR** | 1 |
| `location.file` | Finding.location.file | Node.evidence[0].file -> Graph context | `"unknown"` | 2 |
| `location.contract` | Finding.location.contract | Node.label (parse) -> File name | `"Unknown"` | 3 |
| `location.function` | Finding.location.function | Node.label (parse) -> Pattern target | `"unknown"` | 3 |
| `location.lines` | Finding.location.{line, end_line} | Node.evidence[0].{line_start, line_end} | `[0, 0]` | 4 |
| `behavioral_signature` | Node.properties.behavioral_signature | compute_from_ops() -> derive_from_flags() | `""` | 2 |
| `semantic_ops` | Finding.evidence.operations | Node.properties.semantic_operations | `["UNKNOWN"]` | 2 |
| `properties` | Pattern match conditions | Node.properties (keys) | `["(pattern-based)"]` | 3 |
| `evidence_lines` | Finding.evidence.code_snippet | Read source file -> Evidence refs | `["(source unavailable)"]` | 4 |
| `tool_sources` | Computed from integrations | N/A | `["vkg"]` | 5 |
| `confidence_bucket` | Computed from score | N/A | `"uncertain"` | 5 |
| `rationale` | Finding.evidence.why_vulnerable | Pattern description | `"Matched pattern {id}"` | 4 |

### 3.2 Fallback Priority Levels

| Level | Description | Confidence Impact | Examples |
|-------|-------------|-------------------|----------|
| 1 | Critical - No fallback possible | Error or always computed | id, finding_id |
| 2 | High - Core behavioral data | -15 to -25 penalty | signature, semantic_ops |
| 3 | Medium - Context data | -10 to -15 penalty | contract, function, properties |
| 4 | Low - Supporting data | -5 to -10 penalty | lines, evidence_lines, rationale |
| 5 | Minimal - Defaults acceptable | No penalty | tool_sources, bucket |

### 3.3 Detailed Fallback Chains

#### 3.3.1 Location Fallback Chain

```
location.file:
  1. Finding.location.file
  2. Node.evidence[0].file (first evidence entry)
  3. Graph context (contract file from graph metadata)
  4. DEFAULT: "unknown"

location.contract:
  1. Finding.location.contract
  2. Parse from Node.label (e.g., "Vault.withdraw" -> "Vault")
  3. Derive from file name (e.g., "Vault.sol" -> "Vault")
  4. DEFAULT: "Unknown"

location.function:
  1. Finding.location.function
  2. Parse from Node.label (e.g., "Vault.withdraw" -> "withdraw")
  3. Pattern target function (from pattern metadata)
  4. DEFAULT: "unknown"

location.lines:
  1. [Finding.location.line, Finding.location.end_line or line]
  2. [Node.evidence[0].line_start, Node.evidence[0].line_end]
  3. [Pattern match line, Pattern match line] (single line)
  4. DEFAULT: [0, 0]
```

#### 3.3.2 Behavioral Signature Fallback Chain

```
behavioral_signature:
  1. Node.properties.behavioral_signature (pre-computed)
  2. Compute from Node.properties.operation_sequence:
     - Sort by order field
     - Map to operation codes via OP_CODES dict
     - Join with "->"
  3. Derive from individual operation flags:
     - Check reads_user_balance -> R:bal
     - Check transfers_value -> X:out
     - Check writes_user_balance -> W:bal
     - Check has_access_gate -> C:auth (prepend)
     - Sort by heuristic order
     - Join with "->"
  4. DEFAULT: ""
```

#### 3.3.3 Semantic Operations Fallback Chain

```
semantic_ops:
  1. Finding.evidence.operations (from pattern match)
  2. Node.properties.semantic_operations (derived list)
  3. Derive from Node.properties (individual flags):
     - transfers_value -> TRANSFERS_VALUE_OUT
     - receives_value -> RECEIVES_VALUE_IN
     - reads_user_balance -> READS_USER_BALANCE
     - writes_user_balance -> WRITES_USER_BALANCE
     - has_access_gate -> CHECKS_PERMISSION
     - modifies_owner -> MODIFIES_OWNER
     - (etc. for all 20 operations)
  4. DEFAULT: ["UNKNOWN"]
```

#### 3.3.4 Evidence Lines Fallback Chain

```
evidence_lines:
  1. Finding.evidence.code_snippet (split by newline)
  2. Finding.evidence.evidence_refs (formatted as "[ref]: value")
  3. Read from source file:
     - Open location.file
     - Extract lines [line-1:end_line]
     - Strip trailing whitespace
  4. DEFAULT: ["(source unavailable)"]
```

## 4. `request_more_context` Trigger Conditions

### 4.1 Trigger Definition

The `request_more_context` flag indicates that the evidence packet is incomplete and additional data would improve confidence. This is a **soft failure** mode--the packet is still produced but flagged for attention.

### 4.2 Trigger Conditions

| Condition ID | Trigger | Severity | Context Hint |
|--------------|---------|----------|--------------|
| RMC-001 | `behavioral_signature` is empty | HIGH | "No behavioral signature; CFG ordering unavailable" |
| RMC-002 | `semantic_ops` is `["UNKNOWN"]` | HIGH | "No semantic operations detected; operation analysis incomplete" |
| RMC-003 | `location.file` is `"unknown"` | MEDIUM | "Source file unknown; cannot verify code location" |
| RMC-004 | `location.lines` is `[0, 0]` | MEDIUM | "Line numbers unavailable; location imprecise" |
| RMC-005 | `evidence_lines` is `["(source unavailable)"]` | MEDIUM | "Source code unavailable; evidence incomplete" |
| RMC-006 | `properties` is `["(pattern-based)"]` | LOW | "No specific properties matched; pattern-only detection" |
| RMC-007 | Multiple HIGH triggers | CRITICAL | "Multiple critical fields missing: {list}" |
| RMC-008 | 3+ fallbacks applied | HIGH | "Extensive fallbacks applied: {count} fields" |

### 4.3 Trigger Priority and Aggregation

```python
def check_request_more_context(packet: dict, fallback_log: list[dict]) -> tuple[bool, str]:
    """
    Determine if request_more_context should be set.

    Args:
        packet: The assembled evidence packet
        fallback_log: List of fallback applications with severity

    Returns:
        (should_request, context_hint)
    """
    triggers = []

    # RMC-001: Empty signature
    if not packet.get("behavioral_signature"):
        triggers.append({
            "id": "RMC-001",
            "severity": "HIGH",
            "hint": "No behavioral signature; CFG ordering unavailable"
        })

    # RMC-002: Unknown ops
    ops = packet.get("semantic_ops", [])
    if ops == ["UNKNOWN"] or not ops:
        triggers.append({
            "id": "RMC-002",
            "severity": "HIGH",
            "hint": "No semantic operations detected"
        })

    # RMC-003: Unknown file
    loc = packet.get("location", {})
    if loc.get("file") == "unknown":
        triggers.append({
            "id": "RMC-003",
            "severity": "MEDIUM",
            "hint": "Source file unknown"
        })

    # RMC-004: Zero lines
    if loc.get("lines") == [0, 0]:
        triggers.append({
            "id": "RMC-004",
            "severity": "MEDIUM",
            "hint": "Line numbers unavailable"
        })

    # RMC-005: No evidence lines
    if packet.get("evidence_lines") == ["(source unavailable)"]:
        triggers.append({
            "id": "RMC-005",
            "severity": "MEDIUM",
            "hint": "Source code unavailable"
        })

    # RMC-006: Pattern-only properties
    if packet.get("properties") == ["(pattern-based)"]:
        triggers.append({
            "id": "RMC-006",
            "severity": "LOW",
            "hint": "No specific properties matched"
        })

    # RMC-008: Extensive fallbacks
    high_fallbacks = [f for f in fallback_log if f.get("severity") in ("HIGH", "CRITICAL")]
    if len(high_fallbacks) >= 3:
        triggers.append({
            "id": "RMC-008",
            "severity": "HIGH",
            "hint": f"Extensive fallbacks applied: {len(high_fallbacks)} critical fields"
        })

    # Determine if we should request more context
    high_triggers = [t for t in triggers if t["severity"] in ("HIGH", "CRITICAL")]

    if len(high_triggers) >= 2:
        # RMC-007: Multiple HIGH triggers
        hints = [t["hint"] for t in high_triggers[:3]]
        return True, f"Multiple critical fields missing: {'; '.join(hints)}"

    if high_triggers:
        return True, high_triggers[0]["hint"]

    # Medium triggers: only set if 2+ medium
    medium_triggers = [t for t in triggers if t["severity"] == "MEDIUM"]
    if len(medium_triggers) >= 2:
        hints = [t["hint"] for t in medium_triggers[:2]]
        return True, f"Incomplete: {'; '.join(hints)}"

    return False, ""
```

## 5. `missing_tools` Handling

### 5.1 Definition

The `missing_tools` field records tools that were expected to contribute but could not run or returned no data. This provides transparency about analysis coverage gaps.

### 5.2 When to Populate `missing_tools`

| Scenario | Tool Status | Action |
|----------|-------------|--------|
| Tool binary not found | Unavailable | Add to `missing_tools` |
| Tool returned error | Failed | Add to `missing_tools` with error code |
| Tool returned empty results | No findings | Do NOT add (not missing, just no findings) |
| Tool integration disabled | Skipped | Add to `missing_tools` with "disabled" |
| Tool timed out | Failed | Add to `missing_tools` with "timeout" |

### 5.3 `missing_tools` Data Model

```python
@dataclass
class MissingTool:
    """Record of a tool that could not contribute to analysis."""

    name: str              # Tool name (e.g., "slither", "mythril")
    reason: str            # Why tool is missing
    expected_contribution: str  # What the tool would have provided
    confidence_impact: float    # Penalty applied

    def to_dict(self) -> dict:
        return {
            "name": self.name,
            "reason": self.reason,
            "expected": self.expected_contribution,
            "penalty": self.confidence_impact
        }

# Reason codes
MISSING_TOOL_REASONS = {
    "not_installed": "Tool binary not found in PATH",
    "failed": "Tool execution failed with error",
    "timeout": "Tool execution exceeded time limit",
    "disabled": "Tool integration disabled in config",
    "unsupported": "Tool does not support this Solidity version",
    "license": "Tool requires license not available",
}

# Expected contributions by tool
TOOL_CONTRIBUTIONS = {
    "slither": "AST/IR analysis, detector findings, taint tracking",
    "aderyn": "Rule-based detectors, vulnerability classification",
    "mythril": "Symbolic execution, constraint solving",
    "medusa": "Fuzzing coverage, property violations",
    "halmos": "SMT-based formal verification",
}

# Confidence penalties per tool
TOOL_PENALTIES = {
    "slither": -0.10,   # Major tool, significant penalty
    "aderyn": -0.08,    # Good coverage tool
    "mythril": -0.05,   # Symbolic exec, nice-to-have
    "medusa": -0.03,    # Fuzzing optional
    "halmos": -0.03,    # Formal verification optional
    "custom": -0.02,    # Custom tools minimal impact
}
```

### 5.4 Handling Logic

```python
def record_missing_tools(
    expected_tools: list[str],
    available_results: dict[str, any],
    config: dict
) -> tuple[list[dict], float]:
    """
    Record missing tools and compute total penalty.

    Args:
        expected_tools: List of tools expected in this analysis
        available_results: Dict of tool_name -> results (or None if failed)
        config: Configuration with tool settings

    Returns:
        (list_of_missing_tool_records, total_confidence_penalty)
    """
    missing = []
    total_penalty = 0.0

    for tool in expected_tools:
        # Check if tool is disabled
        if config.get("tools", {}).get(tool, {}).get("enabled") == False:
            missing.append(MissingTool(
                name=tool,
                reason="disabled",
                expected_contribution=TOOL_CONTRIBUTIONS.get(tool, "unknown"),
                confidence_impact=0.0  # No penalty for intentionally disabled
            ).to_dict())
            continue

        # Check if tool produced results
        result = available_results.get(tool)

        if result is None:
            # Tool not found or failed
            reason = "not_installed"
            if tool in available_results and available_results[tool] == "error":
                reason = "failed"
            elif tool in available_results and available_results[tool] == "timeout":
                reason = "timeout"

            penalty = TOOL_PENALTIES.get(tool, -0.02)
            total_penalty += penalty

            missing.append(MissingTool(
                name=tool,
                reason=reason,
                expected_contribution=TOOL_CONTRIBUTIONS.get(tool, "unknown"),
                confidence_impact=penalty
            ).to_dict())

    return missing, total_penalty
```

### 5.5 Integration with Bucket Scoring

From P0.P.2, `missing_tools` affects confidence scoring:

```python
def apply_missing_tools_penalty(score: float, missing_tools: list[dict]) -> float:
    """
    Apply confidence penalty for missing tools.

    The penalty is the sum of individual tool penalties,
    capped at -0.20 to prevent excessive degradation.
    """
    if not missing_tools:
        return score

    total_penalty = sum(t.get("penalty", 0) for t in missing_tools)

    # Cap penalty at -0.20 (don't destroy confidence just because tools are missing)
    total_penalty = max(total_penalty, -0.20)

    return max(0.0, score + total_penalty)  # Floor at 0
```

## 6. Context Budget Fallback Logic (Packet vs. Full Bead)

### 6.1 Context Budget Definition

The **context budget** is the maximum token/character count allocated for LLM context in a given operation. When budget is constrained, the system must decide between producing a full bead or a minimal evidence packet.

### 6.2 Budget Thresholds

| Budget Level | Token Range | Output Mode | Description |
|--------------|-------------|-------------|-------------|
| Full | > 8000 tokens | Full Bead | Complete bead with all fields |
| Standard | 4000-8000 tokens | Standard Packet | Full evidence packet |
| Compact | 2000-4000 tokens | Compact Packet | Packet with compressed evidence |
| Minimal | 1000-2000 tokens | Minimal Packet | Core fields only |
| Critical | < 1000 tokens | Reference Only | ID and location only |

### 6.3 Field Inclusion by Budget Level

| Field | Full | Standard | Compact | Minimal | Critical |
|-------|------|----------|---------|---------|----------|
| id | YES | YES | YES | YES | YES |
| finding_id | YES | YES | YES | YES | YES |
| location | Full | Full | Full | Condensed | Line only |
| behavioral_signature | YES | YES | YES | YES | NO |
| semantic_ops | Full list | Full list | First 3 | First 1 | NO |
| properties | Full list | Full list | First 5 | First 2 | NO |
| evidence_lines | Full | First 5 | First 3 | First 1 | NO |
| tool_sources | YES | YES | YES | NO | NO |
| confidence_bucket | YES | YES | YES | YES | NO |
| rationale | Full | Full | Truncated | 50 chars | NO |
| evidence_links | YES | YES | NO | NO | NO |
| disputed | YES | YES | YES | YES | NO |
| missing_tools | YES | YES | NO | NO | NO |
| request_more_context | YES | YES | YES | YES | NO |
| context_hint | YES | YES | Truncated | NO | NO |

### 6.4 Budget Decision Algorithm

```python
@dataclass
class BudgetDecision:
    """Decision on context budget handling."""

    mode: str                # full_bead, standard, compact, minimal, critical
    token_estimate: int      # Estimated tokens for this mode
    fields_included: list[str]
    fields_excluded: list[str]
    truncation_applied: dict[str, str]  # field -> truncation description
    confidence_penalty: float

def decide_context_budget(
    available_tokens: int,
    packet_data: dict,
    priority: str = "normal"
) -> BudgetDecision:
    """
    Decide output mode based on available context budget.

    Args:
        available_tokens: Token budget available
        packet_data: Raw packet data to be serialized
        priority: Finding priority (critical, high, normal, low)

    Returns:
        BudgetDecision with mode and field handling
    """
    # Priority adjustment: high-priority findings get more budget
    priority_multiplier = {
        "critical": 1.5,
        "high": 1.2,
        "normal": 1.0,
        "low": 0.8
    }
    effective_budget = int(available_tokens * priority_multiplier.get(priority, 1.0))

    # Estimate full packet size
    full_estimate = estimate_token_count(packet_data)

    # Determine mode
    if effective_budget > 8000 and full_estimate <= effective_budget:
        return BudgetDecision(
            mode="full_bead",
            token_estimate=full_estimate,
            fields_included=list(packet_data.keys()),
            fields_excluded=[],
            truncation_applied={},
            confidence_penalty=0.0
        )

    if effective_budget > 4000:
        return create_standard_packet(packet_data, effective_budget)

    if effective_budget > 2000:
        return create_compact_packet(packet_data, effective_budget)

    if effective_budget > 1000:
        return create_minimal_packet(packet_data, effective_budget)

    return create_critical_packet(packet_data, effective_budget)


def create_compact_packet(packet_data: dict, budget: int) -> BudgetDecision:
    """Create compact packet with truncated fields."""
    excluded = ["evidence_links", "missing_tools"]
    truncation = {}

    # Truncate semantic_ops to first 3
    if len(packet_data.get("semantic_ops", [])) > 3:
        truncation["semantic_ops"] = f"Truncated from {len(packet_data['semantic_ops'])} to 3"

    # Truncate properties to first 5
    if len(packet_data.get("properties", [])) > 5:
        truncation["properties"] = f"Truncated from {len(packet_data['properties'])} to 5"

    # Truncate evidence_lines to first 3
    if len(packet_data.get("evidence_lines", [])) > 3:
        truncation["evidence_lines"] = f"Truncated from {len(packet_data['evidence_lines'])} to 3"

    # Truncate rationale to 200 chars
    rationale = packet_data.get("rationale", "")
    if len(rationale) > 200:
        truncation["rationale"] = f"Truncated from {len(rationale)} to 200 chars"

    # Truncate context_hint to 100 chars
    hint = packet_data.get("context_hint", "")
    if len(hint) > 100:
        truncation["context_hint"] = f"Truncated from {len(hint)} to 100 chars"

    included = [k for k in packet_data.keys() if k not in excluded]

    return BudgetDecision(
        mode="compact",
        token_estimate=estimate_token_count(packet_data) * 0.6,  # ~60% of full
        fields_included=included,
        fields_excluded=excluded,
        truncation_applied=truncation,
        confidence_penalty=-0.05  # Small penalty for truncation
    )


def create_minimal_packet(packet_data: dict, budget: int) -> BudgetDecision:
    """Create minimal packet with core fields only."""
    excluded = [
        "evidence_links", "missing_tools", "tool_sources",
        "context_hint", "artifacts"
    ]
    truncation = {
        "semantic_ops": "Limited to first operation",
        "properties": "Limited to first 2 properties",
        "evidence_lines": "Limited to first line",
        "rationale": "Limited to 50 characters"
    }

    # Condense location
    loc = packet_data.get("location", {})
    condensed_loc = f"{loc.get('contract', '?')}.{loc.get('function', '?')}:{loc.get('lines', [0])[0]}"
    truncation["location"] = f"Condensed to: {condensed_loc}"

    included = [
        "id", "finding_id", "location", "behavioral_signature",
        "semantic_ops", "properties", "evidence_lines",
        "confidence_bucket", "rationale", "disputed", "request_more_context"
    ]

    return BudgetDecision(
        mode="minimal",
        token_estimate=estimate_token_count(packet_data) * 0.35,  # ~35% of full
        fields_included=included,
        fields_excluded=excluded,
        truncation_applied=truncation,
        confidence_penalty=-0.10  # Moderate penalty for significant truncation
    )


def create_critical_packet(packet_data: dict, budget: int) -> BudgetDecision:
    """Create critical packet with reference data only."""
    # Only include absolute essentials
    included = ["id", "finding_id"]
    excluded = [k for k in packet_data.keys() if k not in included]

    # Add single-line location reference
    loc = packet_data.get("location", {})
    truncation = {
        "location": f"Line {loc.get('lines', [0])[0]} only",
        "all_other_fields": "Excluded due to budget constraints"
    }

    return BudgetDecision(
        mode="critical",
        token_estimate=50,  # Minimal tokens
        fields_included=included + ["location_line"],
        fields_excluded=excluded,
        truncation_applied=truncation,
        confidence_penalty=-0.20  # Significant penalty for minimal data
    )


def estimate_token_count(data: dict) -> int:
    """Estimate token count for packet data."""
    # Rough estimate: 4 chars per token for YAML/JSON
    import json
    json_str = json.dumps(data, indent=2)
    return len(json_str) // 4
```

### 6.5 When to Skip Full Bead

Per PHILOSOPHY.md: "If context budget is tight, drop to evidence packet and skip full bead."

| Condition | Action | Rationale |
|-----------|--------|-----------|
| Budget < 4000 AND finding is low severity | Skip bead, use packet | Low priority doesn't justify full bead |
| Budget < 2000 AND multiple findings | Skip bead for all | Batch requires compact format |
| Budget < 1000 | Skip bead always | Cannot fit meaningful bead |
| Bead would exceed budget by > 50% | Skip bead | Not worth truncation damage |

## 7. Confidence Penalty Rules

### 7.1 Penalty Registry

Each fallback application incurs a confidence penalty that affects the final bucket scoring:

| Fallback Category | Penalty | Cumulative Cap | Rationale |
|-------------------|---------|----------------|-----------|
| **Location Fallbacks** | | -0.15 | |
| - file unknown | -0.10 | | Cannot verify finding location |
| - contract unknown | -0.03 | | Context unclear |
| - function unknown | -0.03 | | Target unclear |
| - lines [0,0] | -0.05 | | No precise location |
| **Behavioral Fallbacks** | | -0.30 | |
| - signature empty | -0.20 | | No operation ordering (major) |
| - ops UNKNOWN | -0.15 | | No operation detection (major) |
| **Evidence Fallbacks** | | -0.15 | |
| - properties pattern-only | -0.05 | | Weak match evidence |
| - evidence_lines unavailable | -0.10 | | No code context |
| **Tool Fallbacks** | | -0.20 | |
| - per missing major tool | -0.05 to -0.10 | | Reduced coverage |
| **Context Budget** | | -0.20 | |
| - compact mode | -0.05 | | Minor truncation |
| - minimal mode | -0.10 | | Significant truncation |
| - critical mode | -0.20 | | Severe truncation |

### 7.2 Penalty Calculation Algorithm

```python
@dataclass
class FallbackApplication:
    """Record of a single fallback application."""

    field: str
    original_value: any
    fallback_value: any
    reason: str
    penalty: float
    timestamp: str

    def to_dict(self) -> dict:
        return {
            "field": self.field,
            "from": str(self.original_value)[:50] if self.original_value else "None",
            "to": str(self.fallback_value)[:50],
            "reason": self.reason,
            "penalty": self.penalty,
            "timestamp": self.timestamp
        }


class FallbackPenaltyCalculator:
    """Calculate cumulative fallback penalties with caps."""

    CATEGORY_CAPS = {
        "location": -0.15,
        "behavioral": -0.30,
        "evidence": -0.15,
        "tools": -0.20,
        "budget": -0.20,
    }

    PENALTY_MAP = {
        "location.file": ("location", -0.10),
        "location.contract": ("location", -0.03),
        "location.function": ("location", -0.03),
        "location.lines": ("location", -0.05),
        "behavioral_signature": ("behavioral", -0.20),
        "semantic_ops": ("behavioral", -0.15),
        "properties": ("evidence", -0.05),
        "evidence_lines": ("evidence", -0.10),
        "budget_compact": ("budget", -0.05),
        "budget_minimal": ("budget", -0.10),
        "budget_critical": ("budget", -0.20),
    }

    def __init__(self):
        self.applications: list[FallbackApplication] = []
        self.category_totals: dict[str, float] = {cat: 0.0 for cat in self.CATEGORY_CAPS}

    def apply_fallback(
        self,
        field: str,
        original_value: any,
        fallback_value: any,
        reason: str
    ) -> float:
        """
        Record a fallback application and return the penalty applied.

        Returns the actual penalty (may be reduced by cap).
        """
        from datetime import datetime

        category, base_penalty = self.PENALTY_MAP.get(field, ("unknown", -0.02))

        # Check category cap
        current = self.category_totals.get(category, 0.0)
        cap = self.CATEGORY_CAPS.get(category, -0.50)

        # Apply penalty up to cap
        remaining_capacity = cap - current
        actual_penalty = max(base_penalty, remaining_capacity)  # Both negative

        if actual_penalty < 0:
            self.category_totals[category] = current + actual_penalty

            self.applications.append(FallbackApplication(
                field=field,
                original_value=original_value,
                fallback_value=fallback_value,
                reason=reason,
                penalty=actual_penalty,
                timestamp=datetime.utcnow().isoformat()
            ))

        return actual_penalty

    def add_tool_penalty(self, tool_name: str, penalty: float, reason: str) -> float:
        """Add penalty for missing tool."""
        return self.apply_fallback(
            field=f"tool:{tool_name}",
            original_value=None,
            fallback_value="missing",
            reason=reason
        )

    def get_total_penalty(self) -> float:
        """Get total penalty across all categories."""
        return sum(self.category_totals.values())

    def get_audit_log(self) -> list[dict]:
        """Get audit log of all fallback applications."""
        return [app.to_dict() for app in self.applications]

    def get_summary(self) -> dict:
        """Get summary of penalties by category."""
        return {
            "total_penalty": self.get_total_penalty(),
            "by_category": dict(self.category_totals),
            "application_count": len(self.applications),
            "capped_categories": [
                cat for cat, total in self.category_totals.items()
                if total <= self.CATEGORY_CAPS.get(cat, 0) * 0.95  # Within 5% of cap
            ]
        }
```

### 7.3 Integration with P0.P.2 Bucket Scoring

The penalty calculator integrates with the bucket scoring system:

```python
def compute_tier_a_score_with_fallbacks(
    packet: dict,
    fallback_calculator: FallbackPenaltyCalculator
) -> float:
    """
    Compute Tier A score including fallback penalties.

    This extends P0.P.2 compute_tier_a_score() with fallback penalties.
    """
    # Base score from P0.P.2
    base_score = compute_tier_a_score(packet)

    # Apply fallback penalties
    fallback_penalty = fallback_calculator.get_total_penalty()

    # Combine (penalties are negative)
    final_score = base_score + fallback_penalty

    # Ensure score stays in valid range
    return max(0.0, min(1.0, final_score))
```

## 8. Validation After Fallback Application

### 8.1 Post-Fallback Validation

After all fallbacks are applied, the packet must be validated to ensure it meets minimum requirements:

```python
class ValidationLevel(Enum):
    COMPLETE = "complete"   # All fields present and valid
    PARTIAL = "partial"     # Core fields present, some fallbacks
    MINIMAL = "minimal"     # Minimum viable packet
    INVALID = "invalid"     # Cannot produce usable packet


def validate_after_fallbacks(
    packet: dict,
    fallback_calculator: FallbackPenaltyCalculator
) -> tuple[ValidationLevel, list[str]]:
    """
    Validate packet after fallback application.

    Returns:
        (validation_level, list_of_issues)
    """
    issues = []

    # Check absolute requirements
    if not packet.get("id"):
        issues.append("CRITICAL: Missing packet ID")

    if not packet.get("finding_id"):
        issues.append("CRITICAL: Missing finding ID")

    # Check location minimum
    loc = packet.get("location", {})
    if not loc:
        issues.append("CRITICAL: Missing location object")
    elif loc.get("file") == "unknown" and loc.get("lines") == [0, 0]:
        issues.append("SEVERE: Location completely unknown")

    # Check behavioral minimum
    sig = packet.get("behavioral_signature", "")
    ops = packet.get("semantic_ops", [])
    if not sig and (not ops or ops == ["UNKNOWN"]):
        issues.append("SEVERE: No behavioral evidence (signature or ops)")

    # Check confidence can be computed
    if not packet.get("confidence_bucket"):
        issues.append("MODERATE: Missing confidence bucket")

    # Check rationale
    rationale = packet.get("rationale", "")
    if not rationale or len(rationale) < 5:
        issues.append("MODERATE: Missing or trivial rationale")

    # Determine validation level
    critical_issues = [i for i in issues if i.startswith("CRITICAL")]
    severe_issues = [i for i in issues if i.startswith("SEVERE")]

    if critical_issues:
        return ValidationLevel.INVALID, issues

    if severe_issues:
        # Packet is usable but severely limited
        return ValidationLevel.MINIMAL, issues

    if issues:
        return ValidationLevel.PARTIAL, issues

    return ValidationLevel.COMPLETE, []
```

### 8.2 Validation Response Actions

| Validation Level | Action | Bucket Override | Routing |
|------------------|--------|-----------------|---------|
| COMPLETE | Proceed normally | None | Standard |
| PARTIAL | Set `request_more_context=true` | Force `uncertain` if score >= 0.75 | Standard |
| MINIMAL | Set `request_more_context=true` | Force `uncertain` always | Flag for review |
| INVALID | Reject packet | N/A | Route to graph-quality convoy |

### 10.2 Complete Pipeline Implementation

```python
def apply_fallbacks_complete(
    finding: "Finding",
    node: "Node",
    pattern: dict,
    config: dict,
    context_budget: int | None = None,
    tool_results: dict | None = None
) -> tuple[dict, dict]:
    """
    Complete fallback processing pipeline.

    Args:
        finding: The Finding object
        node: The graph Node
        pattern: Pattern that matched
        config: System configuration
        context_budget: Optional token budget for context
        tool_results: Optional dict of tool results

    Returns:
        (evidence_packet, audit_log)
    """
    calculator = FallbackPenaltyCalculator()

    # Step 1-3: Build packet with fallbacks
    packet = build_packet_with_fallbacks(finding, node, pattern, calculator)

    # Step 4: Apply context budget constraints
    if context_budget:
        budget_decision = decide_context_budget(context_budget, packet)
        packet = apply_budget_constraints(packet, budget_decision)
        if budget_decision.confidence_penalty < 0:
            calculator.apply_fallback(
                f"budget_{budget_decision.mode}",
                "full",
                budget_decision.mode,
                f"Context budget limited to {context_budget} tokens"
            )

    # Step 5: Record missing tools
    if tool_results is not None:
        expected_tools = config.get("expected_tools", ["slither", "aderyn"])
        missing, tool_penalty = record_missing_tools(expected_tools, tool_results, config)
        if missing:
            packet["missing_tools"] = missing
            for mt in missing:
                calculator.add_tool_penalty(
                    mt["name"],
                    mt["penalty"],
                    mt["reason"]
                )

    # Step 6: Check request_more_context triggers
    should_request, hint = check_request_more_context(packet, calculator.get_audit_log())
    if should_request:
        packet["request_more_context"] = True
        packet["context_hint"] = hint

    # Step 7: Validate
    validation = validate_after_fallbacks(packet, calculator)

    # Step 8: Compute final score and bucket
    if validation[0] == ValidationLevel.INVALID:
        # Cannot produce valid packet
        packet["confidence_bucket"] = "rejected"
        packet["bucket_rationale"] = f"Packet invalid: {'; '.join(validation[1])}"
    else:
        final_score = compute_tier_a_score_with_fallbacks(packet, calculator)
        bucket, rationale = compute_tier_a_bucket_with_validation(
            packet, final_score, validation[0]
        )
        packet["confidence_bucket"] = bucket
        packet["confidence_score"] = final_score
        packet["bucket_rationale"] = rationale

    # Step 9: Create audit log
    audit_log = log_fallback_audit(packet, calculator, validation)

    # Optionally embed audit reference in packet
    packet["_fallback_audit_id"] = audit_log["id"]

    return packet, audit_log


def compute_tier_a_bucket_with_validation(
    packet: dict,
    score: float,
    validation_level: ValidationLevel
) -> tuple[str, str]:
    """
    Compute bucket considering validation level.

    Extends P0.P.2 rules with validation-based overrides.
    """
    # Validation-based overrides
    if validation_level == ValidationLevel.MINIMAL:
        return "uncertain", "Packet at minimal validation level; extensive fallbacks applied"

    if validation_level == ValidationLevel.PARTIAL and score >= 0.75:
        return "uncertain", f"Packet has fallbacks (score={score}); verification recommended"

    # Standard P0.P.2 rules
    if packet.get("request_more_context"):
        return "uncertain", "Evidence packet incomplete; additional context needed"

    if packet.get("disputed"):
        return "uncertain", "Tool disagreement detected; requires debate resolution"

    if score >= 0.75:
        return "likely", f"Strong behavioral evidence (score={score})"

    return "uncertain", f"Moderate/weak evidence (score={score}); verification recommended"
```
