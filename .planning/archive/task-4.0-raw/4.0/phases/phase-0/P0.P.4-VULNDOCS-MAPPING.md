## 1. Overview

This document defines the mapping between **semantic operations** (from `operations.py`) and **VulnDocs categories** (from Phase 17). This mapping enables:

1. **Automatic VulnDocs retrieval** based on detected semantic operations
2. **Bead context enrichment** with relevant vulnerability knowledge
3. **Pattern-to-knowledge linking** for LLM analysis
4. **Evidence packet augmentation** with category-specific guidance

**Philosophy Alignment:** Semantic operations capture intent, not names. VulnDocs provides the knowledge backbone for those intents.

## 6. Retrieval Algorithm

### 6.1 Operation-Based Retrieval

```python
def get_vulndocs_categories(semantic_ops: List[str]) -> List[str]:
    """Get relevant VulnDocs categories from detected semantic operations.

    Args:
        semantic_ops: List of operation names (e.g., ["TRANSFERS_VALUE_OUT", "WRITES_USER_BALANCE"])

    Returns:
        List of category paths (e.g., ["reentrancy/classic", "access-control/missing-modifier"])
    """
    categories = set()

    # Primary mapping
    op_to_categories = {
        "TRANSFERS_VALUE_OUT": ["reentrancy", "mev/slippage", "token"],
        "RECEIVES_VALUE_IN": ["flash-loan", "logic/accounting"],
        "READS_USER_BALANCE": ["reentrancy", "logic/accounting"],
        "WRITES_USER_BALANCE": ["reentrancy", "logic/accounting"],
        "CHECKS_PERMISSION": ["access-control"],
        "MODIFIES_OWNER": ["access-control/role-escalation"],
        "MODIFIES_ROLES": ["access-control/role-escalation", "governance"],
        "CALLS_EXTERNAL": ["reentrancy", "dos/external-call-loop"],
        "CALLS_UNTRUSTED": ["reentrancy", "token/hooks"],
        "READS_EXTERNAL_VALUE": ["oracle", "flash-loan/price-oracle"],
        "MODIFIES_CRITICAL_STATE": ["access-control", "upgrade"],
        "INITIALIZES_STATE": ["upgrade/unprotected-initialize"],
        "READS_ORACLE": ["oracle", "flash-loan/price-oracle"],
        "LOOPS_OVER_ARRAY": ["dos/unbounded-loop"],
        "USES_TIMESTAMP": ["mev/deadline", "crypto/weak-randomness"],
        "USES_BLOCK_DATA": ["crypto/weak-randomness"],
        "PERFORMS_DIVISION": ["logic/arithmetic", "oracle"],
        "PERFORMS_MULTIPLICATION": ["logic/arithmetic"],
        "VALIDATES_INPUT": ["access-control"],
        "EMITS_EVENT": [],
    }

    for op in semantic_ops:
        if op in op_to_categories:
            categories.update(op_to_categories[op])

    return sorted(categories)
```

### 6.2 Signature-Based Retrieval

```python
def get_vulndocs_from_signature(signature: str) -> List[str]:
    """Get VulnDocs categories from behavioral signature.

    Args:
        signature: Behavioral signature (e.g., "R:bal->X:out->W:bal")

    Returns:
        List of category paths with risk assessment
    """
    # Known vulnerable patterns
    vulnerable_patterns = {
        "R:bal->X:out->W:bal": ["reentrancy/classic"],
        "R:bal->X:call->W:bal": ["reentrancy/classic"],
        "X:unk->W:bal": ["reentrancy/cross-contract"],
        "R:orc->A:div->X:out": ["oracle/price-manipulation"],
        "L:arr->X:call": ["dos/external-call-loop"],
        "U:blk->M:crit": ["crypto/weak-randomness"],
        "I:init->M:crit": ["upgrade/unprotected-initialize"],
    }

    categories = []

    # Direct match
    if signature in vulnerable_patterns:
        categories.extend(vulnerable_patterns[signature])

    # Partial match (check if pattern is substring)
    for pattern, cats in vulnerable_patterns.items():
        if pattern in signature and pattern != signature:
            categories.extend(cats)

    # Operation-level analysis for unmatched signatures
    ops = signature.replace("->", " ").split()
    code_to_op = {v: k.name for k, v in OP_CODES.items()}

    op_names = [code_to_op.get(code, "") for code in ops if code in code_to_op]
    categories.extend(get_vulndocs_categories(op_names))

    return sorted(set(categories))
```

### 6.3 Combined Retrieval for Findings

```python
def get_vulndocs_for_finding(finding: dict) -> dict:
    """Get comprehensive VulnDocs context for a finding.

    Args:
        finding: Finding dict with behavioral_signature and semantic_ops

    Returns:
        dict with categories, subcategories, and retrieval metadata
    """
    signature = finding.get("behavioral_signature", "")
    semantic_ops = finding.get("semantic_ops", [])

    # Get categories from both sources
    sig_categories = get_vulndocs_from_signature(signature)
    op_categories = get_vulndocs_categories(semantic_ops)

    # Merge and deduplicate
    all_categories = set(sig_categories) | set(op_categories)

    # Prioritize: signature matches are higher confidence
    prioritized = []
    for cat in sig_categories:
        prioritized.append({"category": cat, "source": "signature", "confidence": "high"})
    for cat in op_categories:
        if cat not in sig_categories:
            prioritized.append({"category": cat, "source": "operations", "confidence": "medium"})

    return {
        "categories": sorted(all_categories),
        "prioritized": prioritized,
        "retrieval_depth": "detection" if len(all_categories) <= 3 else "overview",
        "max_tokens": 2000 if len(all_categories) <= 3 else 4000,
    }
```

## 8. Mapping Maintenance Rules

### 8.1 Adding New Operations

When a new semantic operation is added to `operations.py`:

1. Add entry to primary mapping table (Section 4.1)
2. Add entry to reverse mapping (Section 4.2)
3. Update retrieval algorithm constants
4. Document any new behavioral signatures
5. Update this document's version

### 8.2 Adding New VulnDocs Categories

When a new VulnDocs category is added in Phase 17:

1. Add to category hierarchy (Section 2)
2. Map relevant operations to new category
3. Identify signature patterns for new category
4. Update reverse mapping table
5. Spawn task P0.P.4.x for tracking

### 8.3 Validation Tests

Tests to ensure mapping consistency:

```python
def test_all_operations_mapped():
    """Every semantic operation should have at least one category."""
    for op in SemanticOperation:
        categories = get_vulndocs_categories([op.name])
        # EMITS_EVENT is allowed to have no categories (informational)
        if op != SemanticOperation.EMITS_EVENT:
            assert len(categories) > 0, f"{op.name} has no VulnDocs mapping"

def test_all_categories_reachable():
    """Every VulnDocs category should be reachable from some operation."""
    all_categories = set()
    for op in SemanticOperation:
        all_categories.update(get_vulndocs_categories([op.name]))

    expected_categories = [
        "reentrancy", "access-control", "oracle", "flash-loan",
        "governance", "dos", "mev", "token", "crypto", "upgrade", "logic"
    ]
    for cat in expected_categories:
        assert any(cat in c for c in all_categories), f"{cat} unreachable"
```
