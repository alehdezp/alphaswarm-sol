## 1. Overview

This document defines the **Tier A bucket defaults** for BSKG findings. Tier A findings are purely deterministic--the same evidence always produces the same bucket assignment. No LLM reasoning is involved in Tier A bucket decisions.

**Key Constraints:**
- Tier A findings can ONLY default to `likely` or `uncertain`
- `confirmed` requires external verification (test/exploit proof or multi-agent consensus)
- `rejected` requires external counter-evidence (disproven by test or rebuttal)
- Determinism applies ONLY to Tier A; Tier B buckets may involve LLM judgment

---

## 2. Confidence Bucket Definitions

From `docs/PHILOSOPHY.md`:

| Bucket | Definition | Tier A Eligible |
|--------|------------|-----------------|
| `confirmed` | Verified by test or convergent multi-agent evidence | NO |
| `likely` | Strong behavioral evidence, but no exploit proof yet | YES |
| `uncertain` | Weak or conflicting signals, needs human review | YES |
| `rejected` | Disproven or explained as benign | NO |

### 2.1 Why Tier A Cannot Produce `confirmed` or `rejected`

1. **`confirmed`** requires evidence outside the graph:
   - A passing exploit test (Foundry, Hardhat)
   - Convergent multi-agent consensus (attacker + defender + verifier agree)
   - Human auditor verification

2. **`rejected`** requires counter-evidence:
   - A failing exploit test (attempted attack blocked)
   - Defender agent rebuttal with proof
   - Human auditor dismissal with rationale

Tier A operates solely on graph properties and pattern matches. It cannot execute tests or reason about counter-evidence.

## 3. Tier A Scoring Factors

The bucket assignment is computed from a deterministic scoring model. Each factor contributes to an evidence strength score.

### 3.1 Scoring Factor Registry

| Factor | Weight | Description | Score Contribution |
|--------|--------|-------------|-------------------|
| `properties_matched` | +15 each | Properties from pattern match | +15 per property (max 60) |
| `behavioral_signature` | +25 | Presence of operation sequence | +25 if non-empty |
| `semantic_ops_count` | +10 each | Number of semantic operations | +10 per op (max 40) |
| `operation_ordering` | +20 | CFG-verified operation ordering | +20 if ordering present |
| `evidence_lines` | +10 | Source code extracted | +10 if non-fallback |
| `tool_agreement` | +15 each | Additional tools agree | +15 per agreeing tool |
| `critical_severity` | +10 | Pattern severity is critical | +10 if critical |
| `high_severity` | +5 | Pattern severity is high | +5 if high |

### 3.2 Penalty Factors

| Factor | Penalty | Description |
|--------|---------|-------------|
| `fallback_signature` | -20 | Behavioral signature is empty/fallback |
| `fallback_ops` | -15 | semantic_ops is ["UNKNOWN"] |
| `fallback_location` | -10 | Location uses fallback values |
| `missing_evidence_lines` | -10 | evidence_lines is ["(source unavailable)"] |
| `tool_disagreement` | -25 | Tools disagree on this finding |
| `incomplete_packet` | -15 | `request_more_context` is true |

### 3.3 Score Calculation

```python
def compute_tier_a_score(packet: dict) -> float:
    """
    Compute Tier A evidence strength score.

    Returns a score from 0.0 to 1.0.
    Maximum raw score is ~200 points; normalized to [0, 1].
    """
    score = 0.0

    # Positive factors
    props = packet.get("properties", [])
    meaningful_props = [p for p in props if not p.startswith("(")]
    score += min(len(meaningful_props) * 15, 60)  # Cap at 60

    sig = packet.get("behavioral_signature", "")
    if sig and sig != "":
        score += 25

    ops = packet.get("semantic_ops", [])
    meaningful_ops = [o for o in ops if o != "UNKNOWN"]
    score += min(len(meaningful_ops) * 10, 40)  # Cap at 40

    # Check for ordering evidence in signature (contains "->")
    if "->" in sig:
        score += 20

    evidence = packet.get("evidence_lines", [])
    if evidence and evidence != ["(source unavailable)"]:
        score += 10

    tools = packet.get("tool_sources", [])
    additional_tools = [t for t in tools if t != "vkg"]
    score += len(additional_tools) * 15

    # Severity bonus (from pattern metadata, if available)
    severity = packet.get("severity", "medium")
    if severity == "critical":
        score += 10
    elif severity == "high":
        score += 5

    # Penalty factors
    if not sig:
        score -= 20

    if ops == ["UNKNOWN"] or not meaningful_ops:
        score -= 15

    loc = packet.get("location", {})
    if loc.get("file") == "unknown" or loc.get("lines") == [0, 0]:
        score -= 10

    if evidence == ["(source unavailable)"]:
        score -= 10

    if packet.get("disputed", False):
        score -= 25

    if packet.get("request_more_context", False):
        score -= 15

    # Normalize to [0, 1] range
    # Max theoretical score: 60 + 25 + 40 + 20 + 10 + 45 + 10 = 210
    # Typical strong finding: ~120-150
    normalized = max(0.0, min(score / 150.0, 1.0))

    return round(normalized, 3)
```

## 4. Bucket Assignment Rules

### 4.1 Score-to-Bucket Mapping

| Score Range | Bucket | Rationale |
|-------------|--------|-----------|
| >= 0.75 | `likely` | Strong behavioral evidence supports the finding |
| 0.40 - 0.74 | `uncertain` | Moderate evidence, needs verification |
| < 0.40 | `uncertain` | Weak evidence, high FP risk |

**Note:** Scores below 0.40 still map to `uncertain`, not `rejected`. Rejection requires active counter-evidence, not merely weak evidence.

### 4.2 Override Rules

These rules override the score-based bucket regardless of score:

| Condition | Override Bucket | Rationale |
|-----------|-----------------|-----------|
| `request_more_context == true` | `uncertain` | Incomplete packet cannot be `likely` |
| `disputed == true` | `uncertain` | Tool disagreement requires resolution |
| `fallback_location` used | `uncertain` | Cannot trust findings with unknown location |
| No properties AND no signature | `uncertain` | No behavioral evidence at all |

### 4.3 Bucket Computation Function

```python
def compute_tier_a_bucket(packet: dict) -> tuple[str, str]:
    """
    Compute Tier A confidence bucket.

    Returns:
        (bucket, rationale) tuple
    """
    # Check override conditions first
    if packet.get("request_more_context", False):
        return ("uncertain", "Evidence packet incomplete; additional context needed")

    if packet.get("disputed", False):
        return ("uncertain", "Tool disagreement detected; requires debate resolution")

    loc = packet.get("location", {})
    if loc.get("file") == "unknown" or loc.get("lines") == [0, 0]:
        return ("uncertain", "Location unknown; cannot verify finding")

    props = packet.get("properties", [])
    sig = packet.get("behavioral_signature", "")
    meaningful_props = [p for p in props if not p.startswith("(")]

    if not meaningful_props and not sig:
        return ("uncertain", "No behavioral evidence; pattern match only")

    # Compute score
    score = compute_tier_a_score(packet)

    # Map score to bucket
    if score >= 0.75:
        return ("likely", f"Strong behavioral evidence (score={score})")
    else:
        return ("uncertain", f"Moderate/weak evidence (score={score}); verification recommended")
```

## 5. Rationale Templates

Every bucket assignment MUST include a rationale explaining the decision. Use these templates for consistency.

### 5.1 `likely` Rationale Templates

| Context | Template |
|---------|----------|
| Strong signature | "Behavioral signature `{sig}` matches known vulnerability pattern; {n} properties confirm the match" |
| Multi-property match | "{n} properties matched pattern conditions with operation ordering evidence" |
| Tool agreement | "VKG finding corroborated by {tools}; strong behavioral evidence present" |
| Critical severity | "Critical severity pattern with strong evidence (score={score}); behavioral signature: {sig}" |

Example:
```
"Behavioral signature R:bal->X:out->W:bal matches classic reentrancy pattern; 4 properties confirm the match"
```

### 5.2 `uncertain` Rationale Templates

| Context | Template |
|---------|----------|
| Incomplete packet | "Evidence packet incomplete; missing: {missing_fields}" |
| Tool disagreement | "Tools disagree: BSKG detects {pattern}, {tool} reports {other}; requires debate" |
| Weak evidence | "Pattern matched but behavioral evidence is weak (score={score}); manual review recommended" |
| Missing signature | "No behavioral signature available; pattern match only without CFG ordering" |
| Location unknown | "Finding location could not be determined; source verification required" |
| Fallback data | "Evidence uses fallback values for {fields}; confidence reduced" |

Example:
```
"Pattern matched but behavioral evidence is weak (score=0.52); manual review recommended"
```

### 5.3 Rationale Construction Function

```python
def construct_bucket_rationale(
    bucket: str,
    packet: dict,
    score: float
) -> str:
    """Construct standardized rationale for bucket assignment."""
    sig = packet.get("behavioral_signature", "")
    props = packet.get("properties", [])
    meaningful_props = [p for p in props if not p.startswith("(")]
    tools = packet.get("tool_sources", ["vkg"])

    if bucket == "likely":
        if sig and len(meaningful_props) >= 3:
            return f"Behavioral signature `{sig}` matches vulnerability pattern; {len(meaningful_props)} properties confirm the match"
        elif len(tools) > 1:
            return f"VKG finding corroborated by {', '.join(tools[1:])}; strong behavioral evidence present"
        else:
            return f"Strong behavioral evidence (score={score}); signature: {sig or '(derived)'}"

    elif bucket == "uncertain":
        reasons = []

        if packet.get("request_more_context"):
            hint = packet.get("context_hint", "additional data")
            reasons.append(f"packet incomplete ({hint})")

        if packet.get("disputed"):
            reasons.append("tool disagreement detected")

        if not sig:
            reasons.append("no behavioral signature")

        if not meaningful_props:
            reasons.append("no property evidence")

        loc = packet.get("location", {})
        if loc.get("file") == "unknown":
            reasons.append("location unknown")

        if not reasons:
            reasons.append(f"moderate evidence score ({score})")

        return f"Verification needed: {'; '.join(reasons)}"

    return f"Bucket: {bucket}"
```

## 6. Bucket Transition Rules

Buckets can only transition via specific paths. Tier A findings start in their computed bucket and can only move via external verification.

### 6.1 Allowed Transitions

```
                  +-----------+
                  |  likely   |
                  +-----+-----+
                        |
         +--------------+--------------+
         |                             |
         v                             v
+--------+--------+           +--------+--------+
|    confirmed    |           |    rejected     |
| (test passes)   |           | (test fails OR  |
| OR (consensus)  |           |  counter-proof) |
+-----------------+           +-----------------+
         ^                             ^
         |                             |
         +--------------+--------------+
                        |
                  +-----+-----+
                  | uncertain |
                  +-----------+
```

### 6.2 Transition Requirements

| From | To | Requires |
|------|----|----------|
| `uncertain` | `likely` | Additional evidence raises score >= 0.75 AND resolves overrides |
| `uncertain` | `confirmed` | Exploit test passes OR multi-agent consensus confirms |
| `uncertain` | `rejected` | Counter-evidence verified OR defender rebuttal accepted |
| `likely` | `confirmed` | Exploit test passes OR multi-agent consensus confirms |
| `likely` | `rejected` | Counter-evidence verified OR defender rebuttal accepted |
| `likely` | `uncertain` | New contradicting evidence OR tool disagreement |
| `confirmed` | `rejected` | NOT ALLOWED without human override |
| `rejected` | `confirmed` | NOT ALLOWED without human override |

### 7.3 Complete Packet Assembly

```python
def assemble_packet_with_bucket(
    finding: Finding,
    node: Node,
    pattern: dict,
    external_results: dict = None
) -> dict:
    """
    Assemble evidence packet with Tier A bucket.

    References P0.P.1 for field mappings.
    """
    # Build base packet per P0.P.1
    packet = {
        "id": generate_packet_id(finding),
        "finding_id": finding.id,
        "location": {
            "file": finding.location.file,
            "contract": finding.location.contract,
            "function": finding.location.function,
            "lines": [finding.location.line, finding.location.end_line or finding.location.line]
        },
        "behavioral_signature": compute_signature_from_node(node),
        "semantic_ops": extract_semantic_ops(node, finding),
        "properties": extract_matched_properties(node, pattern),
        "evidence_lines": extract_evidence_lines(finding, finding.location.file),
        "tool_sources": get_tool_sources(finding, external_results),
        "rationale": construct_rationale(finding, pattern),
        "severity": finding.severity.value if finding.severity else "medium",
    }

    # Apply fallbacks per P0.P.1
    packet = apply_fallbacks(packet)

    # Check for tool disagreement
    if external_results:
        packet["disputed"] = check_tool_disagreement(
            finding, external_results
        )

    # Compute Tier A bucket
    bucket, bucket_rationale = compute_tier_a_bucket(packet)
    score = compute_tier_a_score(packet)

    packet["confidence_bucket"] = bucket
    packet["confidence_score"] = score
    packet["bucket_rationale"] = bucket_rationale

    return packet
```

## 9. Validation Requirements

### 9.1 Bucket Consistency Tests

Every bucket assignment must pass these validation checks:

```python
def validate_bucket_assignment(packet: dict) -> tuple[bool, list[str]]:
    """Validate bucket assignment consistency."""
    issues = []
    bucket = packet.get("confidence_bucket")
    score = packet.get("confidence_score", 0)

    # Check bucket is valid Tier A bucket
    if bucket not in ("likely", "uncertain"):
        issues.append(f"Invalid Tier A bucket: {bucket}")

    # Check score consistency (unless override)
    has_override = (
        packet.get("request_more_context") or
        packet.get("disputed") or
        packet.get("location", {}).get("file") == "unknown"
    )

    if not has_override:
        if bucket == "likely" and score < 0.75:
            issues.append(f"Bucket is 'likely' but score {score} < 0.75")
        if bucket == "uncertain" and score >= 0.75:
            issues.append(f"Bucket is 'uncertain' but score {score} >= 0.75 without override")

    # Check rationale present
    rationale = packet.get("bucket_rationale", "")
    if not rationale or len(rationale) < 10:
        issues.append("Bucket rationale missing or too short")

    # Check rationale matches bucket
    if bucket == "likely" and "verification" in rationale.lower():
        issues.append("'likely' rationale incorrectly mentions verification needed")
    if bucket == "uncertain" and "strong" in rationale.lower() and "evidence" in rationale.lower():
        issues.append("'uncertain' rationale incorrectly claims strong evidence")

    return (len(issues) == 0, issues)
```

### 9.2 Determinism Tests

Tier A bucket computation must be deterministic:

```python
def test_bucket_determinism():
    """Same input must always produce same bucket."""
    packet = create_test_packet()

    results = []
    for _ in range(100):
        bucket, rationale = compute_tier_a_bucket(packet)
        results.append((bucket, rationale))

    # All results must be identical
    assert all(r == results[0] for r in results), "Bucket computation is non-deterministic"
```
