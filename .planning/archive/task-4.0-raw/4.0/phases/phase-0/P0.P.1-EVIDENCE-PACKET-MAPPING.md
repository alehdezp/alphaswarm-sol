## 1. Overview

This document defines the **post-processing mapping** from BSKG graph properties, semantic operations, and findings model to Evidence Packet fields as specified in `docs/PHILOSOPHY.md`.

**Key Constraint:** No modifications to `builder.py` are allowed. All mappings are post-graph transformations applied during finding assembly.

## 2. Evidence Packet Contract (from PHILOSOPHY.md)

### 2.1 Required Fields

| Field | Type | Description |
|-------|------|-------------|
| `id` | string | Unique packet identifier |
| `finding_id` | string | Reference to parent finding |
| `location.file` | string | Source file path |
| `location.contract` | string | Contract name |
| `location.function` | string | Function name |
| `location.lines` | [int, int] | Start and end line numbers |
| `behavioral_signature` | string | Compact operation sequence (e.g., `R:bal->X:out->W:bal`) |
| `semantic_ops` | [string] | List of semantic operation names |
| `properties` | [string] | List of matched property names |
| `evidence_lines` | [string] | Relevant code snippets |
| `tool_sources` | [string] | Tools that contributed to detection |
| `confidence_bucket` | string | One of: confirmed, likely, uncertain, rejected |
| `rationale` | string | Why this is flagged |

### 2.2 Optional Fields

| Field | Type | Description |
|-------|------|-------------|
| `evidence_links` | [string] | Links to related findings/evidence |
| `disputed` | bool | True if tools disagree |
| `missing_tools` | [string] | Tools that could not run |
| `request_more_context` | bool | True if packet is incomplete |
| `context_hint` | string | What additional context would help |
| `artifacts` | [string] | Paths to related files (tests, reports) |

## 4. Field Mapping Rules

### 4.1 Required Field Mappings

| Evidence Packet Field | Source | Extraction Logic |
|----------------------|--------|------------------|
| `id` | Generated | `EP-{finding.id}-{timestamp_hash[:8]}` |
| `finding_id` | `Finding.id` | Direct copy |
| `location.file` | `Finding.location.file` | Direct copy |
| `location.contract` | `Finding.location.contract` | From Location or graph node |
| `location.function` | `Finding.location.function` | From Location or graph node |
| `location.lines` | `Finding.location.{line, end_line}` | `[line, end_line or line]` |
| `behavioral_signature` | `Finding.evidence.behavioral_signature` | From `compute_behavioral_signature()` |
| `semantic_ops` | `Finding.evidence.operations` | List of operation names |
| `properties` | `Finding.evidence.properties_matched` | List of matched property names |
| `evidence_lines` | `Finding.evidence.code_snippet` | Split by newline if multiline |
| `tool_sources` | Computed | `["vkg"]` + external tools if integrated |
| `confidence_bucket` | Computed | See Section 5 (Bucket Mapping) |
| `rationale` | `Finding.evidence.why_vulnerable` | Plain English explanation |

### 4.2 Detailed Mapping Logic

#### 4.2.1 `id` Generation

```python
def generate_packet_id(finding: Finding) -> str:
    """Generate unique evidence packet ID."""
    import hashlib
    from datetime import datetime

    timestamp = datetime.utcnow().isoformat()
    data = f"{finding.id}:{timestamp}"
    hash_suffix = hashlib.sha256(data.encode()).hexdigest()[:8].upper()
    return f"EP-{finding.id}-{hash_suffix}"
```

#### 4.2.2 `behavioral_signature` Derivation

The behavioral signature is computed from semantic operations via `compute_behavioral_signature()` in `operations.py`:

1. Derive all operations from function via `derive_all_operations(fn)`
2. Sort by CFG order
3. Map to short codes via `OP_CODES` dict
4. Join with arrow: `"->".join(codes)`

**Example:**
```
Operations: [READS_USER_BALANCE@0, TRANSFERS_VALUE_OUT@3, WRITES_USER_BALANCE@5]
Signature:  "R:bal->X:out->W:bal"
```

**Fallback:** If operations list is empty, set to empty string `""`.

#### 4.2.3 `semantic_ops` Population

Extract from graph node properties and pattern match:

```python
def extract_semantic_ops(node: Node, finding: Finding) -> list[str]:
    """Extract semantic operations for evidence packet."""
    ops = []

    # From finding evidence
    if finding.evidence.operations:
        ops.extend(finding.evidence.operations)

    # From graph node properties
    node_props = node.properties
    if node_props.get("semantic_operations"):
        ops.extend(node_props["semantic_operations"])

    # From operation_sequence if present
    if node_props.get("operation_sequence"):
        for entry in node_props["operation_sequence"]:
            ops.append(entry.get("op", ""))

    # Deduplicate while preserving order
    seen = set()
    unique_ops = []
    for op in ops:
        if op and op not in seen:
            seen.add(op)
            unique_ops.append(op)

    return unique_ops
```

**Fallback:** If no operations detected, return `["UNKNOWN"]`.

#### 4.2.4 `properties` Extraction

Extract properties that contributed to the pattern match:

```python
def extract_matched_properties(node: Node, pattern: dict) -> list[str]:
    """Extract properties matched by the pattern."""
    matched = []
    node_props = node.properties

    # From pattern 'all' conditions
    for condition in pattern.get("match", {}).get("all", []):
        prop_name = condition.get("property")
        if prop_name and prop_name in node_props:
            matched.append(f"{prop_name}={node_props[prop_name]}")

    # From pattern 'none' conditions (negated)
    for condition in pattern.get("match", {}).get("none", []):
        prop_name = condition.get("property")
        if prop_name and prop_name in node_props:
            matched.append(f"!{prop_name}={node_props[prop_name]}")

    return matched
```

**Fallback:** If no properties matched, return `["(no properties)"]`.

#### 4.2.5 `evidence_lines` Extraction

```python
def extract_evidence_lines(finding: Finding, source_file: str) -> list[str]:
    """Extract relevant source code lines."""
    lines = []

    # From code snippet
    if finding.evidence.code_snippet:
        lines.extend(finding.evidence.code_snippet.strip().split("\n"))

    # From evidence refs
    for ref in finding.evidence.evidence_refs:
        if ref.type == "code" and ref.value:
            lines.append(f"[{ref.ref}]: {ref.value}")

    # Fallback: read from source if available
    if not lines and source_file:
        try:
            with open(source_file, 'r') as f:
                file_lines = f.readlines()
                start = max(0, finding.location.line - 1)
                end = min(len(file_lines), (finding.location.end_line or finding.location.line))
                lines = [l.rstrip() for l in file_lines[start:end]]
        except Exception:
            pass

    return lines if lines else ["(source unavailable)"]
```

#### 4.2.6 `tool_sources` Population

```python
def get_tool_sources(finding: Finding, external_results: dict = None) -> list[str]:
    """Get list of tools that contributed to this finding."""
    sources = ["vkg"]  # BSKG always contributes

    # Check for external tool matches at same location
    if external_results:
        loc_key = f"{finding.location.file}:{finding.location.line}"
        for tool, locs in external_results.items():
            if loc_key in locs:
                sources.append(tool)

    return sources
```

#### 4.2.7 `rationale` Construction

```python
def construct_rationale(finding: Finding, pattern: dict) -> str:
    """Construct human-readable rationale."""
    parts = []

    # From why_vulnerable
    if finding.evidence.why_vulnerable:
        parts.append(finding.evidence.why_vulnerable)

    # From pattern description
    if pattern and pattern.get("description"):
        parts.append(f"Pattern: {pattern['description']}")

    # From guard analysis
    if finding.evidence.guard_analysis:
        parts.append(f"Guards: {finding.evidence.guard_analysis}")

    # From attack scenario
    if finding.evidence.attack_scenario:
        parts.append(f"Attack: {'; '.join(finding.evidence.attack_scenario[:2])}")

    if not parts:
        return f"Matched pattern {finding.pattern}"

    return " | ".join(parts)
```

## 5. Behavioral Signature Translation

### 5.1 Operation to Signature Code Mapping

The `OP_CODES` dictionary in `operations.py` provides the mapping:

```python
OP_CODES = {
    SemanticOperation.TRANSFERS_VALUE_OUT: "X:out",
    SemanticOperation.RECEIVES_VALUE_IN: "X:in",
    SemanticOperation.READS_USER_BALANCE: "R:bal",
    SemanticOperation.WRITES_USER_BALANCE: "W:bal",
    SemanticOperation.CHECKS_PERMISSION: "C:auth",
    SemanticOperation.MODIFIES_OWNER: "M:own",
    SemanticOperation.MODIFIES_ROLES: "M:role",
    SemanticOperation.CALLS_EXTERNAL: "X:call",
    SemanticOperation.CALLS_UNTRUSTED: "X:unk",
    SemanticOperation.READS_EXTERNAL_VALUE: "R:ext",
    SemanticOperation.MODIFIES_CRITICAL_STATE: "M:crit",
    SemanticOperation.INITIALIZES_STATE: "I:init",
    SemanticOperation.READS_ORACLE: "R:orc",
    SemanticOperation.LOOPS_OVER_ARRAY: "L:arr",
    SemanticOperation.USES_TIMESTAMP: "U:time",
    SemanticOperation.USES_BLOCK_DATA: "U:blk",
    SemanticOperation.PERFORMS_DIVISION: "A:div",
    SemanticOperation.PERFORMS_MULTIPLICATION: "A:mul",
    SemanticOperation.VALIDATES_INPUT: "V:in",
    SemanticOperation.EMITS_EVENT: "E:evt",
}
```

### 5.2 Signature Patterns and Meanings

| Signature Pattern | Vulnerability Type |
|-------------------|-------------------|
| `R:bal->X:out->W:bal` | Classic reentrancy (vulnerable) |
| `R:bal->W:bal->X:out` | CEI pattern (safe) |
| `C:auth->M:crit` | Protected privileged write (safe) |
| `M:crit` | Unprotected privileged write (vulnerable) |
| `R:orc->A:div->X:out` | Oracle manipulation risk |
| `X:unk->W:bal` | Untrusted call with balance write |
| `L:arr->X:call` | DoS risk (loop with external call) |

### 5.3 Signature Computation from Graph Node

```python
def compute_signature_from_node(node: Node) -> str:
    """Compute behavioral signature from graph node properties."""
    # Primary: use pre-computed signature
    if node.properties.get("behavioral_signature"):
        return node.properties["behavioral_signature"]

    # Secondary: compute from operation_sequence
    if node.properties.get("operation_sequence"):
        ops = node.properties["operation_sequence"]
        codes = []
        for entry in sorted(ops, key=lambda x: x.get("order", 0)):
            op_name = entry.get("op", "")
            for op_enum, code in OP_CODES.items():
                if op_enum.name == op_name:
                    codes.append(code)
                    break
        return "->".join(codes)

    # Fallback: derive from individual operation flags
    ops = []
    if node.properties.get("reads_user_balance"):
        ops.append(("R:bal", 0))
    if node.properties.get("transfers_value"):
        ops.append(("X:out", 1))
    if node.properties.get("writes_user_balance"):
        ops.append(("W:bal", 2))
    if node.properties.get("has_access_gate"):
        ops.insert(0, ("C:auth", -1))

    ops.sort(key=lambda x: x[1])
    return "->".join(code for code, _ in ops)
```

## 6. Fallback Values

When data is missing, apply these fallback rules:

| Field | Fallback Value | Trigger Condition |
|-------|---------------|-------------------|
| `id` | Generate new | Never missing (always computed) |
| `finding_id` | Required | Error if missing |
| `location.file` | `"unknown"` | Location not set |
| `location.contract` | `"Unknown"` | Contract not identified |
| `location.function` | `"unknown"` | Function not identified |
| `location.lines` | `[0, 0]` | Line numbers unavailable |
| `behavioral_signature` | `""` | No operations detected |
| `semantic_ops` | `["UNKNOWN"]` | No operations detected |
| `properties` | `["(pattern-based)"]` | No properties matched |
| `evidence_lines` | `["(source unavailable)"]` | Source not accessible |
| `tool_sources` | `["vkg"]` | No external tools |
| `confidence_bucket` | `"uncertain"` | Cannot compute bucket |
| `rationale` | `"Matched pattern {pattern_id}"` | No rationale available |

### 6.1 Fallback Processing Logic

```python
def apply_fallbacks(packet: dict) -> dict:
    """Apply fallback values to incomplete packet."""
    defaults = {
        "location": {
            "file": "unknown",
            "contract": "Unknown",
            "function": "unknown",
            "lines": [0, 0]
        },
        "behavioral_signature": "",
        "semantic_ops": ["UNKNOWN"],
        "properties": ["(pattern-based)"],
        "evidence_lines": ["(source unavailable)"],
        "tool_sources": ["vkg"],
        "confidence_bucket": "uncertain",
        "rationale": f"Matched pattern {packet.get('pattern', 'unknown')}"
    }

    # Apply location defaults
    loc = packet.get("location", {})
    for key, default in defaults["location"].items():
        if not loc.get(key):
            loc[key] = default
    packet["location"] = loc

    # Apply other defaults
    for key, default in defaults.items():
        if key != "location" and not packet.get(key):
            packet[key] = default

    # Set request_more_context if critical fields missing
    critical_missing = []
    if packet["behavioral_signature"] == "":
        critical_missing.append("behavioral_signature")
    if packet["semantic_ops"] == ["UNKNOWN"]:
        critical_missing.append("semantic_ops")
    if packet["evidence_lines"] == ["(source unavailable)"]:
        critical_missing.append("evidence_lines")

    if critical_missing:
        packet["request_more_context"] = True
        packet["context_hint"] = f"Missing: {', '.join(critical_missing)}"

    return packet
```

## 7. Validation Rules

### 7.1 Completeness Validation

A packet is **complete** if:

1. All required fields are present and non-empty
2. `location.lines[0] > 0` (valid line number)
3. `behavioral_signature` is non-empty OR `semantic_ops` has non-UNKNOWN entries
4. `confidence_bucket` is one of: `confirmed`, `likely`, `uncertain`, `rejected`
5. `rationale` explains why the finding was flagged

### 7.2 Validation Function

```python
def validate_packet(packet: dict) -> tuple[bool, list[str]]:
    """Validate evidence packet completeness.

    Returns:
        (is_valid, list_of_issues)
    """
    issues = []

    # Check required fields exist
    required = ["id", "finding_id", "location", "behavioral_signature",
                "semantic_ops", "properties", "evidence_lines",
                "tool_sources", "confidence_bucket", "rationale"]

    for field in required:
        if field not in packet or packet[field] is None:
            issues.append(f"Missing required field: {field}")

    # Validate location structure
    loc = packet.get("location", {})
    if not loc.get("file") or loc["file"] == "unknown":
        issues.append("Invalid location.file")
    if not loc.get("lines") or loc["lines"][0] <= 0:
        issues.append("Invalid location.lines")

    # Validate behavioral data
    sig = packet.get("behavioral_signature", "")
    ops = packet.get("semantic_ops", [])
    if not sig and (not ops or ops == ["UNKNOWN"]):
        issues.append("No behavioral evidence (signature or ops)")

    # Validate confidence bucket
    valid_buckets = {"confirmed", "likely", "uncertain", "rejected"}
    bucket = packet.get("confidence_bucket", "")
    if bucket not in valid_buckets:
        issues.append(f"Invalid confidence_bucket: {bucket}")

    # Validate rationale
    rationale = packet.get("rationale", "")
    if not rationale or len(rationale) < 10:
        issues.append("Rationale too short or missing")

    return (len(issues) == 0, issues)
```

### 7.3 Validation Levels

| Level | Criteria | Action |
|-------|----------|--------|
| **COMPLETE** | All validations pass | Proceed to bead creation |
| **PARTIAL** | Core fields present, some behavioral data missing | Set `request_more_context=true` |
| **INVALID** | Missing required fields or location | Reject, log for graph-quality debate |

## 8. Confidence Bucket Mapping

### 8.1 Tier A Default Buckets

For Tier A (deterministic) findings, the bucket is computed from evidence strength:

| Evidence Strength | Bucket | Criteria |
|-------------------|--------|----------|
| Strong | `likely` | 3+ properties matched AND behavioral signature present |
| Moderate | `uncertain` | 1-2 properties matched OR signature only |
| Weak | `uncertain` | Fallback values used |

**Note:** Tier A cannot produce `confirmed` (requires test/exploit proof) or `rejected` (requires counter-evidence).

### 8.2 Bucket Computation

```python
def compute_tier_a_bucket(packet: dict) -> str:
    """Compute confidence bucket for Tier A finding."""
    props = packet.get("properties", [])
    sig = packet.get("behavioral_signature", "")

    # Count meaningful properties (exclude fallbacks)
    meaningful_props = [p for p in props if not p.startswith("(")]

    # Strong evidence
    if len(meaningful_props) >= 3 and sig:
        return "likely"

    # Everything else is uncertain for Tier A
    return "uncertain"
```

## 10. Integration Points

### 10.1 Where Mapping Applies

The evidence packet mapping is applied at these points:

1. **Finding Assembly** (`findings/model.py` `to_dict()` calls)
2. **Bead Creation** (when finding becomes a bead)
3. **Report Generation** (for human-readable output)
4. **Tool Integration** (when merging external tool results)

### 10.2 Post-Processing Pipeline

```
Finding + Graph Node
       |
       v
 extract_semantic_ops()
 compute_signature_from_node()
 extract_matched_properties()
 extract_evidence_lines()
       |
       v
 Assemble raw packet
       |
       v
 apply_fallbacks()
       |
       v
 validate_packet()
       |
       v
 compute_tier_a_bucket()
       |
       v
 Evidence Packet
```
