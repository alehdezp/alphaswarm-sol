## 1. OBJECTIVE

Research how to spawn isolated Claude Code CLI sessions programmatically from Python. The goal is to use Claude Code as a "micro-agent" that can verify findings in isolation.

**IMPORTANT CLARIFICATION:** There is no standalone "Claude Agent SDK" library. What we're calling "Agent SDK" is actually:
1. The `claude` CLI from `@anthropic-ai/claude-code` npm package
2. Subprocess spawning from Python to invoke Claude Code
3. Structured prompts and result parsing

### 3.3 Key Question: CLI vs API?

**Option A: Claude Code CLI (subprocess)**
- Pros: Full tool access (Read, Bash, etc.), isolated sessions
- Cons: Requires npm install, subprocess overhead, harder to mock

**Option B: Direct Anthropic API (existing LLMClient)**
- Pros: Already implemented, easier to mock, no external dependency
- Cons: No tool access, just text completion

**Research should determine which approach is better for verification tasks.**

### Step 2: Subprocess Invocation Test (1h)

```python
import subprocess
import json

def test_claude_invocation():
    """Test basic Claude Code invocation."""
    result = subprocess.run(
        ["claude", "--prompt", "Say hello", "--no-interactive"],
        capture_output=True,
        text=True,
        timeout=30
    )
    print(f"Return code: {result.returncode}")
    print(f"Stdout: {result.stdout}")
    print(f"Stderr: {result.stderr}")
```
