## Research Objective

Determine the minimum and optimal context an LLM needs to accurately investigate each vulnerability class. This research directly informs the bead structure and investigation templates.

---

## Research Questions

1. What context does an LLM need to verify a reentrancy finding?
2. What context for access control?
3. What context for oracle manipulation?
4. What context for DoS?
5. What context for MEV?
6. What context for token issues?
7. What context for upgrade issues?
8. What is the minimum viable context per class?
9. What additional context improves accuracy?

---

## Methodology

### Step 1: Expert Knowledge Gathering
- Interview/survey security auditors
- Review audit report structures
- Analyze what experienced auditors look for

### Step 2: LLM Experimentation
- Create test findings with varying context levels
- Measure LLM accuracy at each level
- Identify minimum context for 80% accuracy

### Step 3: Context Ablation Study
- Remove context elements one at a time
- Measure impact on accuracy
- Rank context elements by importance

## Findings Per Vulnerability Class

### 1. Reentrancy

**Minimum Required Context:**
- [ ] The function with external call (full source)
- [ ] State variables being modified
- [ ] Order of operations (call before/after state update)
- [ ] Modifiers applied to function

**Optimal Additional Context:**
- [ ] Called functions (if internal)
- [ ] Contract inheritance (guards in parent)
- [ ] Visibility of function
- [ ] Value being transferred

**Key Questions LLM Must Answer:**
1. Is there an external call?
2. Is state modified after the call?
3. Is there a reentrancy guard?
4. Can the call target be malicious?

**Context Sufficiency Indicators:**
- LLM can identify call type (transfer/call/send)
- LLM can trace state modification order
- LLM can see modifier implementations

### 2. Access Control

## Context Size Analysis

### Experiment: Context Size vs Accuracy

| Context Level | Description | Estimated Accuracy |
|--------------|-------------|-------------------|
| Minimal | Function only | 50-60% |
| Basic | + modifiers/calls | 65-75% |
| Standard | + related funcs | 75-85% |
| Complete | + inheritance/contract | 85-95% |
| Full | Everything | 90-95% |

**Observation:** Diminishing returns after "Standard" level.

### Recommended Context Levels

| Vulnerability Class | Recommended Level | Reason |
|--------------------|------------------|--------|
| Reentrancy | Standard | Need call + state order |
| Access Control | Basic | Modifier visibility key |
| Oracle | Standard | Need validation logic |
| DoS | Basic | Loop structure primary |
| MEV | Basic | Parameter presence key |
| Token | Standard | Need transfer patterns |
| Upgrade | Complete | Need inheritance info |

## Context Extraction Strategy

### Priority Order for Context Elements

1. **Critical (Always Include):**
   - Function source code
   - Direct modifiers
   - Evidence lines from pattern

2. **Important (Include if Available):**
   - Called internal functions
   - Related state variables
   - Pattern-specific context

3. **Helpful (Include if Space Permits):**
   - Inheritance chain
   - Other callers
   - Full contract (if small)

4. **Optional (Include on Request):**
   - External dependencies
   - Deployment context
   - Historical changes

### Size Budgets

| Element | Target | Max | Notes |
|---------|--------|-----|-------|
| Function | 50 lines | 100 | Full function |
| Modifiers | 10 lines each | 30 total | All applied |
| Called funcs | 30 lines each | 150 total | Top 5 |
| State vars | 10 lines | 30 | Relevant only |
| Full contract | 200 lines | 500 | Skip if large |

**Total Target:** 300-400 lines of code context
**Maximum:** 800 lines

## Recommendations

### 1. Investigation Template Structure

Each template should provide:
- Class-specific context extraction rules
- Required code elements
- Questions mapping to evidence
- FP indicators to check

### 2. Dynamic Context Expansion

If initial context insufficient:
1. Start with minimum context
2. Track LLM confidence
3. If low confidence, expand
4. Max 2 expansion rounds

### 3. Context Quality Metrics

- Completeness: Required fields present
- Relevance: Context related to pattern
- Coherence: Code snippets connected
- Sufficiency: LLM doesn't ask for more
