```yaml
  - step_number: 1
    action: "Description of what to do"
    look_for: "Specific patterns to find"
    evidence_needed: "What confirms this step"
    red_flag: "Pattern indicating vulnerability"  # Optional
    safe_if: "Pattern indicating safety"          # Optional

# Key questions the investigator must answer

  - "Question 1?"
  - "Question 2?"

# Known patterns that trigger false positives
common_false_positives:
  - "Description of FP pattern 1"
  - "Description of FP pattern 2"

key_indicators:
  - "Strong indicator 1"
  - "Strong indicator 2"

# Patterns that would make the code safe
safe_patterns:
  - "Safe pattern 1"
  - "Safe pattern 2"
```

### 1. Reentrancy Template (`reentrancy.yaml`)

```yaml
vulnerability_class: reentrancy
version: 1.0
last_updated: 2026-01-07

investigation_steps:
  - step_number: 1
    action: "Identify the external call"
    look_for: |
      - .call{value:}()
      - .transfer()
      - .send()
      - High-level calls to external contracts
      - Token transfers (safeTransfer, transfer)
    evidence_needed: "Line number, target address source, value being sent"

  - step_number: 2
    action: "Check state changes relative to external call"
    look_for: |
      - Balance updates (balances[user] -= amount)
      - Mapping writes
      - State variable modifications
    evidence_needed: "List all state writes and their line numbers"
    red_flag: "State write occurs AFTER external call"
    safe_if: "All state writes occur BEFORE external call (CEI pattern)"

  - step_number: 3
    action: "Check for reentrancy protection"
    look_for: |
      - nonReentrant modifier
      - ReentrancyGuard inheritance
      - Custom mutex/lock variable
    evidence_needed: "Modifier name or lock mechanism code"
    safe_if: "Proper reentrancy guard is applied to this function"

  - step_number: 4
    action: "Determine if external call target is controlled"
    look_for: |
      - Is target address from user input?
      - Is target address from storage that user can modify?
      - Is it a hardcoded trusted address?
    evidence_needed: "Source of the call target"
    red_flag: "Target is user-controlled or from untrusted storage"
    safe_if: "Target is hardcoded, immutable, or from trusted source"

  - step_number: 5
    action: "Assess exploitability and impact"
    look_for: |
      - What can attacker gain from reentering?
      - Is there profit potential (drain funds, mint tokens)?
      - What's the maximum extractable value?
    evidence_needed: "Attack scenario with profit calculation"

questions_to_answer:
  - "Is there an external call in this function?"
  - "Does state change after the external call?"
  - "Is there a reentrancy guard protecting this function?"
  - "Can the external call target execute arbitrary code?"
  - "What can an attacker gain from reentering?"

common_false_positives:
  - "Function has nonReentrant modifier but pattern didn't detect it"
  - "External call is to a trusted, immutable contract (e.g., WETH)"
  - "State written after call is unrelated to the call's value"
  - "Function is internal/private and all public callers are protected"
  - "The 'state update' is just emitting an event"

key_indicators:
  - "Balance decremented AFTER value transfer"
  - "User-controlled callback address"
  - "No reentrancy protection"
  - "Significant value at risk"

safe_patterns:
  - "Checks-Effects-Interactions (CEI) pattern followed"
  - "nonReentrant modifier present"
  - "Pull payment pattern (user claims, not push)"
  - "External call to verified safe contract"
```

### 2. Access Control Template (`access_control.yaml`)

```yaml
vulnerability_class: access_control
version: 1.0
last_updated: 2026-01-07

investigation_steps:
  - step_number: 1
    action: "Identify the privileged operation"
    look_for: |
      - Ownership changes (transferOwnership, setOwner)
      - Administrative setters (setFee, setAdmin, pause)
      - Fund movements (withdraw, sweep, emergencyWithdraw)
      - Token minting/burning
      - Upgrade functions
    evidence_needed: "What sensitive operation does this function perform?"

  - step_number: 2
    action: "Check for access control mechanism"
    look_for: |
      - onlyOwner modifier
      - onlyRole(ADMIN_ROLE)
      - require(msg.sender == owner)
      - require(hasRole(role, msg.sender))
      - OpenZeppelin AccessControl or Ownable
    evidence_needed: "Access control code or modifier name"
    red_flag: "No access control check found"
    safe_if: "Proper access control modifier/check exists"

  - step_number: 3
    action: "Verify access control is correctly implemented"
    look_for: |
      - Is the modifier actually checking permissions?
      - Is the role/owner properly initialized?
      - Can the access control be bypassed?
    evidence_needed: "Modifier implementation code"
    red_flag: "Modifier exists but doesn't actually restrict access"

  - step_number: 4
    action: "Check function visibility"
    look_for: |
      - public vs external vs internal
      - Can this be called through another unprotected function?
    evidence_needed: "Function visibility and call paths"

  - step_number: 5
    action: "Assess impact of unauthorized access"
    look_for: |
      - What can attacker do?
      - What's the worst-case damage?
      - Is there value at risk?
    evidence_needed: "Impact description with severity"

questions_to_answer:
  - "What sensitive operation does this function perform?"
  - "Who should be allowed to call this function?"
  - "Is there any access control check?"
  - "Is the access control correctly implemented?"
  - "What's the impact if anyone can call this?"

common_false_positives:
  - "Function is intentionally public (e.g., user withdrawal)"
  - "Access control is in a base contract not shown"
  - "Function only reads state, doesn't modify"
  - "The 'privileged' operation has no security impact"
  - "Function is protected by a wrapper function"

key_indicators:
  - "public/external function modifies critical state"
  - "No onlyOwner or role check"
  - "Can change ownership or admin addresses"
  - "Can move funds or mint tokens"

safe_patterns:
  - "onlyOwner modifier from OpenZeppelin"
  - "AccessControl with proper role checks"
  - "msg.sender validation against stored admin"
  - "Multi-sig requirement"
```

### 3. Oracle Template (`oracle.yaml`)

```yaml
vulnerability_class: oracle
version: 1.0
last_updated: 2026-01-07

investigation_steps:
  - step_number: 1
    action: "Identify oracle usage"
    look_for: |
      - Chainlink latestRoundData()
      - Uniswap TWAP queries
      - Custom price feeds
      - getPrice(), getLatestPrice() patterns
    evidence_needed: "Oracle interface and price source"

  - step_number: 2
    action: "Check for staleness validation"
    look_for: |
      - updatedAt timestamp check
      - require(block.timestamp - updatedAt < MAX_DELAY)
      - Heartbeat validation
    evidence_needed: "Staleness check code and threshold value"
    red_flag: "No staleness check found"
    safe_if: "Staleness check with reasonable threshold (< 1 hour for most feeds)"

  - step_number: 3
    action: "Check for price sanity bounds"
    look_for: |
      - require(price > 0)
      - require(price < MAX_PRICE)
      - Circuit breaker on extreme values
    evidence_needed: "Price validation code"
    red_flag: "No validation that price is positive/reasonable"

  - step_number: 4
    action: "Check for L2-specific issues (if on L2)"
    look_for: |
      - Sequencer uptime check
      - Grace period after sequencer restart
      - L2 sequencer feed usage
    evidence_needed: "L2 handling code"
    red_flag: "On L2 without sequencer uptime check"
    safe_if: "Sequencer uptime validated before using price"

  - step_number: 5
    action: "Assess manipulation resistance"
    look_for: |
      - Is this a spot price (manipulable)?
      - Is TWAP used with sufficient window?
      - Can attacker profit from price manipulation?
    evidence_needed: "Price source analysis"

questions_to_answer:
  - "What oracle is being used?"
  - "Is there a staleness check?"
  - "Is the price validated (> 0, reasonable bounds)?"
  - "If L2, is sequencer uptime checked?"
  - "Can the price be manipulated for profit?"

common_false_positives:
  - "Oracle data is only used for display/logging"
  - "Staleness check exists but in a different function"
  - "Using trusted internal oracle with different validation"
  - "Price has no financial impact in this context"

key_indicators:
  - "No staleness validation"
  - "No price > 0 check"
  - "Using spot price without manipulation protection"
  - "L2 deployment without sequencer check"

safe_patterns:
  - "Full Chainlink validation (staleness, round, price > 0)"
  - "TWAP with >= 30 minute window"
  - "Multiple oracle fallback"
  - "L2 sequencer uptime validation"
```

### 4. DoS Template (`dos.yaml`)

```yaml
vulnerability_class: dos
version: 1.0
last_updated: 2026-01-07

investigation_steps:
  - step_number: 1
    action: "Identify loop or iteration"
    look_for: |
      - for loops
      - while loops
      - Array iterations
      - Mapping iterations (via auxiliary array)
    evidence_needed: "Loop code and iteration count source"

  - step_number: 2
    action: "Check if loop bound is controlled"
    look_for: |
      - Is array length user-controlled?
      - Is there a max iteration limit?
      - Can users grow the array unboundedly?
    evidence_needed: "Array growth mechanism and limits"
    red_flag: "Users can grow array without limit"
    safe_if: "Hardcoded max or admin-only array growth"

  - step_number: 3
    action: "Check for external calls in loop"
    look_for: |
      - .transfer() in loop
      - .call() in loop
      - External contract calls in loop
    evidence_needed: "External call code within loop"
    red_flag: "External call to user-provided addresses in loop"

  - step_number: 4
    action: "Check for gas griefing potential"
    look_for: |
      - Can one user block others?
      - Is there fallback to handle failures?
      - Are failures isolated?
    evidence_needed: "Failure handling analysis"
    red_flag: "Single revert blocks entire operation"
    safe_if: "Try/catch or pull pattern isolates failures"

  - step_number: 5
    action: "Assess DoS impact"
    look_for: |
      - What functionality is blocked?
      - How long would DoS last?
      - Can it be recovered from?
    evidence_needed: "Impact and recovery analysis"

questions_to_answer:
  - "Is there a loop in this function?"
  - "What controls the number of iterations?"
  - "Can an attacker make the loop very large?"
  - "Are there external calls inside the loop?"
  - "Can the function be made to revert permanently?"

common_false_positives:
  - "Array length is bounded by protocol design"
  - "Only admin can add to array"
  - "Loop has explicit max iterations"
  - "External calls are to trusted contracts"
  - "Function has explicit gas limit"

key_indicators:
  - "User-controlled array length"
  - "No maximum iteration limit"
  - "External calls in loop to untrusted addresses"
  - "Revert on any iteration fails entire transaction"

safe_patterns:
  - "Bounded arrays with explicit max"
  - "Pagination for large data sets"
  - "Pull pattern instead of push"
  - "Try/catch around external calls"
  - "Gas limit per iteration"
```

### Template Loader

Create `./src/true_vkg/beads/templates/loader.py`:

```python
"""Template loader for investigation guides."""
import os
from pathlib import Path
from typing import Dict, Optional
import yaml

from ..schema import InvestigationGuide, InvestigationStep


TEMPLATES_DIR = Path(__file__).parent


def load_template(vulnerability_class: str) -> Optional[InvestigationGuide]:
    """Load investigation template for a vulnerability class.

    Args:
        vulnerability_class: One of: reentrancy, access_control, oracle,
                            dos, mev, token, upgrade

    Returns:
        InvestigationGuide if template exists, None otherwise
    """
    # Normalize class name
    class_name = vulnerability_class.lower().replace("-", "_").replace(" ", "_")
    template_path = TEMPLATES_DIR / f"{class_name}.yaml"

    if not template_path.exists():
        return None

    with open(template_path, "r") as f:
        data = yaml.safe_load(f)

    return InvestigationGuide(
        steps=[
            InvestigationStep(
                step_number=s["step_number"],
                action=s["action"],
                look_for=s["look_for"],
                evidence_needed=s["evidence_needed"],
                red_flag=s.get("red_flag"),
                safe_if=s.get("safe_if"),
            )
            for s in data["investigation_steps"]
        ],
        questions_to_answer=data["questions_to_answer"],
        common_false_positives=data["common_false_positives"],
        key_indicators=data["key_indicators"],
        safe_patterns=data["safe_patterns"],
    )


def list_available_templates() -> list[str]:
    """List all available vulnerability class templates."""
    templates = []
    for path in TEMPLATES_DIR.glob("*.yaml"):
        if path.name != "__init__.yaml":
            templates.append(path.stem)
    return templates


def get_template_version(vulnerability_class: str) -> Optional[str]:
    """Get version of a template."""
    class_name = vulnerability_class.lower().replace("-", "_")
    template_path = TEMPLATES_DIR / f"{class_name}.yaml"

    if not template_path.exists():
        return None

    with open(template_path, "r") as f:
        data = yaml.safe_load(f)

    return data.get("version")
```
