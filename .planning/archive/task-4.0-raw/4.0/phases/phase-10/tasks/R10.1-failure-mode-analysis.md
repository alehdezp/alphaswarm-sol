## Deliverables

### 1. Failure Mode Catalog Document

Create `/Volumes/ex_ssd/home/projects/python/vkg-solidity/alphaswarm/task/4.0/phases/phase-10/research/failure-mode-catalog.md`

Organize by component:

```markdown
## Component: Slither

| Failure Mode | Trigger | Detection | Impact | Recovery Strategy |
|-------------|---------|-----------|--------|-------------------|
| Crash mid-analysis | Malformed Solidity | Non-zero exit code | No graph built | Retry with subset, or fail gracefully |
| Timeout | Large codebase | Timer exceeded | Partial graph | Kill process, report partial |
| OOM | Very large contracts | Memory exception | Process killed | Reduce batch size, retry |
| Missing solc version | Version constraint | Subprocess error | Cannot compile | Use fallback version or error clearly |

## Component: LLM API

| Failure Mode | Trigger | Detection | Impact | Recovery Strategy |
|-------------|---------|-----------|--------|-------------------|
| Rate limit | Too many requests | 429 status | Tier B blocked | Exponential backoff |
| API key invalid | Misconfiguration | 401 status | Tier B unavailable | Clear error message |
| Network unreachable | No internet | Connection exception | Tier B blocked | Offline mode |
| Timeout | Slow response | Timer exceeded | Request lost | Retry with backoff |

## Component: File System

| Failure Mode | Trigger | Detection | Impact | Recovery Strategy |
|-------------|---------|-----------|--------|-------------------|
| Disk full | Low disk space | Write exception | State corrupted | Clear cache, warn user |
| Permission denied | Wrong permissions | OS exception | Cannot write | Clear error with chmod hint |
| Directory deleted mid-op | External interference | FileNotFoundError | Operation fails | Recreate or fail gracefully |

## Component: External Tools (Aderyn, Medusa, Foundry)

| Failure Mode | Trigger | Detection | Impact | Recovery Strategy |
|-------------|---------|-----------|--------|-------------------|
| Not installed | Missing binary | subprocess.CalledProcessError | Tool unavailable | Skip, note in results |
| Version mismatch | Old version | Version check | Possible bugs | Warn, suggest update |
| Crash | Bug in tool | Non-zero exit | No results | Skip tool, continue |
```

### 2. Priority Classification

Classify each failure into:
- **CRITICAL:** BSKG cannot continue (must handle immediately)
- **HIGH:** Major feature degraded (handle in Phase 10)
- **MEDIUM:** Nice-to-have feature lost (handle if time permits)
- **LOW:** Edge case (log for future)

### 3. Recovery Strategy Matrix

For each failure mode, document:
1. **Immediate Action:** What happens right when failure occurs
2. **User Notification:** What user sees
3. **Recovery Command:** What command fixes it
4. **Prevention:** How to avoid it

---

## Research Method

1. **Code Analysis**
   - Search for `except Exception` in codebase to find existing error handling
   - Find subprocess calls (Slither, external tools)
   - Find network calls (LLM providers)
   - Find file operations

2. **Real-World Experience**
   - Review past issues/bugs related to failures
   - Consider common user mistakes

3. **Adversarial Thinking**
   - What would make BSKG crash?
   - What could corrupt state?
