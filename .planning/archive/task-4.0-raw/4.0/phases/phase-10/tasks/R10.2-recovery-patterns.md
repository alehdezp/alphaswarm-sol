## Research Topics

### 1. Circuit Breaker Pattern

**Reference:** https://martinfowler.com/bliki/CircuitBreaker.html

Research questions:
- When to "open" the circuit (stop trying)?
- How long to wait before "half-open" (try again)?
- What states to track?

**Application to VKG:**
- LLM API calls: After N failures, stop trying for M seconds
- External tools: Mark as unavailable after crash

### 2. Retry with Exponential Backoff

Research questions:
- Base delay and multiplier?
- Maximum retries?
- When to jitter to avoid thundering herd?

**Application to VKG:**
- LLM rate limits: Wait 1s, 2s, 4s, 8s...
- Network errors: Similar

### 3. Graceful Degradation

**Reference:** https://wiki.c2.com/?GracefulDegradation

Research questions:
- What functionality to preserve?
- How to communicate reduced capability?
- When to fail completely vs degrade?

**Application to VKG:**
```
Full Capability
    |
    +-- LLM fails --> Tier A only (still useful)
    +-- External tools fail --> BSKG patterns only
    +-- Slither fails --> Cannot continue (core dependency)
```

### 4. State Recovery Patterns

Research questions:
- Transactional writes (write-rename pattern)?
- Journaling / Write-Ahead Log?
- Checkpointing?

**Application to VKG:**
- Graph save: Write to temp file, rename on success
- Findings: Save incrementally to avoid total loss

### 5. User Communication Patterns

Research questions:
- Error message format?
- How to offer next steps?
- Progressive disclosure (simple message first, details on request)?

**Application to VKG:**
```bash
# Good
$ vkg analyze
Error: LLM API unavailable (rate limited)
  Next steps: Wait 60s and retry, or use --tier-a-only
  Details: Run vkg doctor for full diagnostics

# Bad
$ vkg analyze
urllib3.exceptions.ConnectionError: ('Connection aborted.', ...
```

```markdown
### BSKG Application

## 2. Retry with Backoff

### When to Use
- Transient failures (network blips, temporary unavailability)

### Implementation
from tenacity import retry, stop_after_attempt, wait_exponential

@retry(stop=stop_after_attempt(3), wait=wait_exponential(multiplier=1, max=30))
def call_llm(prompt):
    ...

### BSKG Application
- All LLM API calls
- External tool timeouts

## 3. Safe State Writes

### When to Use
- Any file write that must not corrupt

### Implementation
def safe_write(path: Path, data: bytes):
    temp = path.with_suffix('.tmp')
    temp.write_bytes(data)
    temp.rename(path)  # Atomic on POSIX

### BSKG Application
- graph.json writes
- findings.json writes
- config file updates

## 4. Error Message Template

### Format
[SEVERITY] Short description
  Context: What was happening
  Cause: Why it failed (if known)
  Next: What user can do
  Help: Where to get more info

### Example
[ERROR] Failed to build knowledge graph
  Context: Analyzing contracts/Vault.sol
  Cause: Slither version 0.9.0 not compatible with Solidity 0.8.20
  Next: Run 'pip install slither-analyzer>=0.10.0'
  Help: vkg doctor --verbose
```

### 2. Decision Tree

```
Failure Detected
    |
    +-- Is it transient? (network, timeout, rate limit)
    |       |
    |       +-- Yes: Retry with backoff (max 3 attempts)
    |       +-- No: Continue to degradation check
    |
    +-- Is core functionality? (Slither)
    |       |
    |       +-- Yes: Fail with clear error, suggest fix
    |       +-- No: Degrade gracefully
    |
    +-- Can we degrade?
    |       |
    |       +-- Yes: Continue with reduced capability, warn user
    |       +-- No: Fail gracefully with recovery command
    |
    +-- Is state corrupted?
            |
            +-- Yes: Offer 'vkg repair' or 'vkg reset'
            +-- No: Just report error
```
