## Context

### Why OpenCode SDK?

OpenCode provides a single SDK interface to 75+ LLM providers. Instead of maintaining separate integrations for each provider, BSKG can route through OpenCode for:
- Provider abstraction
- Automatic fallback
- Unified billing
- Session management

### Documentation

- OpenCode SDK Docs: https://opencode.ai/docs/sdk/
- OpenCode Providers: https://opencode.ai/docs/providers/

### 2. Provider Enumeration

```python
# List available providers
from opencode import OpenCode

client = OpenCode()
providers = client.list_providers()

for p in providers:
    print(f"{p.name}: {p.models}, {p.cost_per_1m_tokens}")
```

### 3. Session Management Test

```python
# Test session persistence
from opencode import OpenCode

client = OpenCode()

# Create session
session = client.create_session()

# Send messages
response1 = session.send("Analyze this function for reentrancy")
response2 = session.send("What about the modifier?")  # Should have context

# Resume session later
session_id = session.id
resumed = client.resume_session(session_id)
```

### 4. Structured Output Test

```python
# Test JSON schema enforcement
from opencode import OpenCode

VERDICT_SCHEMA = {
    "type": "object",
    "properties": {
        "verdict": {"enum": ["VULNERABLE", "SAFE", "UNCERTAIN"]},
        "confidence": {"type": "number", "minimum": 0, "maximum": 100}
    },
    "required": ["verdict", "confidence"]
}

client = OpenCode()
response = client.complete(
    prompt="Analyze this code...",
    output_schema=VERDICT_SCHEMA
)
```
