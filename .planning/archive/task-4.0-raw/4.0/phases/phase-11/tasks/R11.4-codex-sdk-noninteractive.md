### 2. Thread Persistence Test

```typescript
// Test thread resumption
import { Codex } from '@openai/codex';

const codex = new Codex();

// Create thread
const thread = await codex.createThread();

// Send messages
await thread.send("Analyze this function for reentrancy vulnerabilities");
await thread.send("Now check the state writes after external calls");

// Save thread ID
const threadId = thread.id;

// Later: resume thread
const resumed = await codex.resumeThread(threadId);
await resumed.send("What was the verdict on reentrancy?");
```

## Trade-off Analysis

### SDK vs Noninteractive

| Aspect | Codex SDK | Codex Noninteractive |
|--------|-----------|---------------------|
| **Language** | TypeScript (primary) | CLI (any language) |
| **State** | Thread-based | Per-invocation |
| **Control** | Fine-grained | Coarse-grained |
| **CI/CD** | Requires TypeScript wrapper | Direct CLI integration |
| **Debugging** | Full API access | Limited output |
| **Use Case** | Complex multi-stage analysis | Simple batch runs |

### Recommendation Matrix

| BSKG Use Case | Recommended Mode |
|--------------|------------------|
| Single finding verification | Noninteractive |
| Multi-stage audit pipeline | SDK with threads |
| CI/CD integration | Noninteractive |
| Interactive exploration | SDK |

### 2. Example Implementations

```python
# Python wrapper for Codex noninteractive
import subprocess
import json

def run_codex_verification(finding: dict) -> dict:
    """Run Codex verification in noninteractive mode."""
    prompt = f"""
    Verify this vulnerability finding:
    {json.dumps(finding, indent=2)}

    Return JSON with verdict, confidence, evidence.
    """

    result = subprocess.run(
        ["codex", "exec", "--noninteractive", "--json-output", prompt],
        capture_output=True,
        text=True,
        timeout=120
    )

    return json.loads(result.stdout)
```
