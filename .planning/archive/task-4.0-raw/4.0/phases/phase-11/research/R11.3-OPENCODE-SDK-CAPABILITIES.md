## Executive Summary

OpenCode is an open-source AI coding agent supporting 75+ LLM providers through the AI SDK and Models.dev integration. This research documents OpenCode SDK capabilities specifically for BSKG Tier B LLM integration, covering session management, provider abstraction, batch/CI operations, MCP tool integration, and cost tracking.

**Key Findings:**
1. OpenCode SDK provides type-safe programmatic control via `@opencode-ai/sdk`
2. Session management supports create/list/retrieve/update/delete with message history
3. Provider abstraction supports 75+ providers with unified interface and automatic fallback
4. Batch/CI operations supported via `opencode run` (non-interactive) and headless `serve` mode
5. MCP integration allows BSKG to expose tools via local stdio or remote HTTP servers
6. Cost tracking is NOT built into SDK but available via external tools (OCMonitor, Portkey)
7. Rate limiting handled at provider level, not SDK level

**Recommendation:** OpenCode is an excellent orchestration layer for BSKG Tier B when BSKG is used **by** AI agents. BSKG should expose MCP tools; OpenCode consumes them.

### 3.2 SDK Session Operations


## 4. Batch/CI Operation Support

### 4.1 Non-Interactive Mode

The `opencode run` command enables batch processing without the TUI:

```bash
# Single prompt execution
opencode run "Explain how closures work in JavaScript"

# With specific model
opencode run -m anthropic/claude-sonnet-4-5 "Analyze this code"

# With file attachment
opencode run -f contract.sol "Find vulnerabilities in this contract"

# JSON output for parsing
opencode run --format json "List all functions"
```

### 4.2 Headless Server Mode

```bash
# Start headless server
opencode serve --port 4096 --hostname 127.0.0.1

# Attach multiple clients
opencode attach --port 4096
```

**Benefits for CI/CD:**
- Reduce bootstrap overhead (single server, multiple requests)
- Shared session state across pipeline stages
- Resource pooling

### 4.3 GitHub Actions Integration

```bash
# Execute GitHub agent in Actions
opencode github run --event push --token $GITHUB_TOKEN
```

### 4.4 CI/CD Pipeline Pattern

```yaml
# .github/workflows/vkg-audit.yml
name: BSKG Security Audit
on: [push]

jobs:
  audit:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Install OpenCode
        run: curl -fsSL https://opencode.ai/install | bash

      - name: Start OpenCode Server
        run: opencode serve --port 4096 &

      - name: Run BSKG via OpenCode
        run: |
          opencode run --port 4096 \
            "Build BSKG graph and analyze contracts/*.sol for vulnerabilities"

      - name: Export Results
        run: |
          opencode run --port 4096 --format json \
            "Export findings as SARIF" > results.sarif
```

### 4.5 SDK Batch Pattern

```typescript
import { createOpencodeClient } from "@opencode-ai/sdk"

async function batchAnalyze(contracts: string[]) {
  const client = createOpencodeClient({ baseUrl: "http://localhost:4096" })
  const session = await client.session.create({})

  const results = []
  for (const contract of contracts) {
    // Inject contract context
    await client.session.message.send({
      sessionId: session.id,
      content: `Analyze contract: ${contract}`,
      noReply: false
    })

    // Wait for response
    const messages = await client.session.message.list({
      sessionId: session.id
    })

    results.push({
      contract,
      analysis: messages[messages.length - 1]
    })
  }

  return results
}
```

## 6. Cost and Rate Limiting Considerations

### 6.1 Cost Tracking Limitations

**OpenCode SDK does NOT include built-in cost tracking.**

The SDK provides:
- Session and message management
- Tool execution
- Event streams

But NOT:
- Token counting
- Cost estimation
- Budget enforcement
- Usage dashboards

### 6.2 External Cost Tracking Solutions

**Option 1: OpenCode Monitor (Community Tool)**

```bash
# Install OCMonitor
pip install ocmonitor

# Run alongside OpenCode
ocmonitor watch
```

**Features:**
- Real-time usage analytics
- Token consumption tracking per model
- Cost insights with model-specific pricing
- Budget alerts
- Session analysis

**Option 2: Portkey AI Gateway**

```json
{
  "provider": {
    "portkey": {
      "baseURL": "https://api.portkey.ai/v1",
      "options": {
        "apiKey": "{env:PORTKEY_API_KEY}"

```

**Portkey Features:**
- Unified API for multiple providers

### 6.3 Rate Limiting

**SDK Level:** No built-in rate limiting in OpenCode SDK.

**Provider Level:** Rate limits enforced by individual providers:

| Provider | Rate Limit | Strategy |
|----------|------------|----------|
| Anthropic | Tier-based (RPM, TPM) | Automatic backoff |
| OpenAI | Tier-based | 429 response + retry-after |
| OpenRouter | Per-API-key quotas | Automatic retry |

**Application-Level Rate Limiting:**

```typescript
import { RateLimiter } from "@rate-limiter/core"

const limiter = new RateLimiter({
  requestsPerMinute: 60,
  tokensPerMinute: 100000
})

async function rateLimitedCall(prompt: string) {
  await limiter.acquire()
  return client.session.message.send({ content: prompt })
}
```

### 6.4 Cost Management Strategies for VKG

| Strategy | Implementation |
|----------|----------------|
| **Tier routing** | Use cheaper models for simple tasks |
| **Context pruning** | Send minimal context to LLM |
| **Caching** | Cache LLM responses for identical queries |
| **Budget caps** | External tool (Portkey/OCMonitor) |
| **Batch grouping** | Combine multiple findings per LLM call |

## 7. Integration Architecture for BSKG Tier B

### 7.1 Architecture Options

**Option A: BSKG as Tool for OpenCode (Recommended)**

```
User --> OpenCode --> [LLM decides] --> BSKG MCP Tools --> [Results back to LLM]
```

- BSKG provides deterministic analysis (Tier A)
- OpenCode/LLM provides reasoning (Tier B)
- Clean separation of concerns
- LLM has context about user intent

**Option B: BSKG Orchestrates OpenCode**

```
User --> BSKG --> [Needs LLM?] --> OpenCode SDK --> [LLM reasoning]
```

- BSKG controls when/if LLM is used
- More complex integration
- BSKG manages cost/rate limits

**Option C: Hybrid (Recommended for Phase 11)**


### 7.2 Recommended Integration Pattern

```python
# src/true_vkg/llm/opencode_provider.py
"""OpenCode SDK provider for BSKG Tier B analysis."""

from dataclasses import dataclass
from typing import Optional
import httpx

@dataclass
class OpenCodeConfig:
    base_url: str = "http://localhost:4096"
    timeout: int = 120

class OpenCodeProvider:
    """Provider that uses OpenCode SDK for Tier B analysis."""

    def __init__(self, config: OpenCodeConfig):
        self.config = config
        self.client = httpx.AsyncClient(
            base_url=config.base_url,
            timeout=config.timeout
        )

    async def analyze_finding(
        self,
        finding: "Finding",
        context: str
    ) -> "TierBResult":
        """Send finding to OpenCode for Tier B analysis."""

        # Create or reuse session
        session = await self._get_or_create_session()

        # Inject BSKG context without triggering response
        await self._send_message(
            session_id=session["id"],
            content=f"VKG Tier A Finding:\n{finding.to_context()}",
            no_reply=True
        )

        # Request Tier B analysis
        response = await self._send_message(
            session_id=session["id"],
            content=self._tier_b_prompt(finding),
            no_reply=False
        )

        return self._parse_tier_b_response(response)

    def _tier_b_prompt(self, finding: "Finding") -> str:
        return f"""
        Analyze this potential vulnerability:

        ## Pattern
        {finding.pattern_id}: {finding.pattern_name}

        ## Location
        {finding.contract}:{finding.line}

        ## Evidence
        {finding.evidence_summary()}

        ## Task
        Determine if this is a TRUE POSITIVE or FALSE POSITIVE.

        Respond in JSON:
        {{
          "verdict": "VULNERABLE" | "SAFE" | "UNCERTAIN",
          "confidence": 0-100,
          "reasoning": "...",
          "evidence": ["..."]
        }}
        """
```

### 7.3 BSKG MCP Server Implementation

See Phase 3 research (R3.2) for complete MCP server implementation. Key additions for Tier B:

```python
# Additional MCP tool for Tier B context
@mcp.tool()
async def tier_b_context(finding_id: str) -> str:
    """Get enriched context for Tier B analysis.

    Returns finding with:
    - Full code context (function + callers + modifiers)
    - Pattern explanation
    - Investigation steps
    - Similar historical exploits

    Args:
        finding_id: Finding ID (e.g., VKG-001)

    Returns:
        JSON context optimized for LLM analysis
    """
    global _findings

    if _findings is None:
        return json.dumps({"error": "No findings available"})

    finding = _findings.get(finding_id)
    if finding is None:
        return json.dumps({"error": f"Finding {finding_id} not found"})

    # Use Bead system from Phase 6
    from true_vkg.beads import BeadCreator
    bead = BeadCreator().create(finding)

    return json.dumps({
        "status": "success",
        "context": bead.to_llm_context()
    })
```

## 8. Comparison with Other SDKs

### 8.1 SDK Comparison Matrix

| Feature | OpenCode SDK | Claude Agent SDK | Codex SDK |
|---------|-------------|------------------|-----------|
| **Providers** | 75+ | Claude only | OpenAI only |
| **Session Mgmt** | Full CRUD | Thread-based | Thread-based |
| **MCP Support** | Native | Limited | None |
| **Cost Tracking** | External | Built-in | Built-in |
| **Batch Mode** | CLI + SDK | SDK only | CLI + SDK |

### 8.2 When to Use Which

| Use Case | Recommended SDK |
|----------|-----------------|
| Multi-provider flexibility | OpenCode SDK |
| Claude-specific features | Claude Agent SDK |
| OpenAI ecosystem | Codex SDK |
| Local model development | OpenCode SDK |
| Strict cost control | Claude/Codex SDK |
| MCP tool ecosystem | OpenCode SDK |

### 8.3 Integration Strategy

For BSKG Phase 11, we recommend:

1. **Primary:** Expose BSKG as MCP tools for any AI agent (OpenCode, Claude Code, Codex)
2. **Optional:** Use OpenCode SDK as fallback provider for standalone BSKG Tier B
3. **Cost Tracking:** Implement via external tools (Portkey/OCMonitor) or BSKG internal metrics

### 9.3 Tier B Architecture Decision

**Decision:** BSKG remains a **tool provider**, not an LLM orchestrator.
