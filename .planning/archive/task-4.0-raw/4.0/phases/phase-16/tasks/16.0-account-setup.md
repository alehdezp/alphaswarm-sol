## Objective

Set up all external accounts and credentials needed for release before any publishing work begins.

## Why This Matters

Publishing will FAIL without proper credentials. This task ensures we don't discover missing access mid-release.

---

## Checklist

### PyPI Setup

- [ ] PyPI account created: https://pypi.org/account/register/
- [ ] 2FA enabled on PyPI account
- [ ] Trusted publisher configured (GitHub Actions OIDC):
  ```
  PyPI > Your projects > alphaswarm > Publishing > Add new trusted publisher
  - Repository: [org]/alphaswarm
  - Workflow: release.yml
  - Environment: release
  ```
- [ ] Test PyPI account created: https://test.pypi.org/
- [ ] Test PyPI trusted publisher configured (same process)

### Docker Hub Setup

- [ ] Docker Hub account: https://hub.docker.com/signup
- [ ] Organization created: `truevkg` (or similar available name)
- [ ] Repository created: `truevkg/alphaswarm`
- [ ] Access token generated for CI:
  ```
  Account Settings > Security > New Access Token
  Name: github-actions-push
  Permissions: Read, Write, Delete
  ```
- [ ] Token stored as GitHub secret: `DOCKERHUB_TOKEN`
- [ ] Username stored as GitHub secret: `DOCKERHUB_USERNAME`

### GitHub Setup

- [ ] Repository has correct org ownership
- [ ] GitHub Actions enabled
- [ ] Branch protection on `main`:
  - Require pull request reviews
  - Require status checks to pass
  - Require signed commits (optional but recommended)
- [ ] Release environment created:
  ```
  Settings > Environments > New environment: "release"
  - Required reviewers: [maintainer list]
  ```
- [ ] GITHUB_TOKEN has correct permissions for releases

---

## Credential Storage

Store credentials ONLY in these locations:

| Credential | Storage Location | Access |
|------------|-----------------|--------|
| PyPI API token | GitHub Secrets: `PYPI_API_TOKEN` | CI only |
| Docker Hub token | GitHub Secrets: `DOCKERHUB_TOKEN` | CI only |
| Docker Hub username | GitHub Secrets: `DOCKERHUB_USERNAME` | CI only |

**NEVER** store credentials in:
- Repository files
- Local env files committed to git
- Documentation

---

## Troubleshooting

| Problem | Cause | Solution |
|---------|-------|----------|
| "Invalid API token" | Token expired or wrong | Regenerate in PyPI settings |
| "403 Forbidden" on Docker push | Wrong token permissions | Create new token with write access |
| Trusted publisher fails | OIDC not configured | Check workflow name matches exactly |
| Test PyPI works, real PyPI fails | Different accounts | Verify same setup on both |
