## Research Questions

### Q1: Platt Scaling vs Isotonic Regression

**Question:** Which calibration method is better for our use case?

**What to Research:**
- Platt scaling: Fits logistic regression to map scores to probabilities
  - Pros: Parametric, works with small data
  - Cons: Assumes sigmoid shape
- Isotonic regression: Non-parametric, monotonic fitting
  - Pros: No shape assumption, handles arbitrary distributions
  - Cons: Needs more data, can overfit

**Decision Criteria:**
- How many labeled examples do we expect per pattern? If < 50, lean toward Platt
- Is our raw confidence already somewhat calibrated or completely off?

**Output Required:**
```markdown
## Calibration Method Decision
- Selected: [Platt | Isotonic | Both with fallback]
- Rationale: [Why this method fits VKG]
- Minimum data requirement: [N samples]
```

### Q2: Per-Pattern vs Global Calibration

**Question:** Should we train one calibrator per pattern or one global calibrator?

**What to Research:**
- Global: One calibrator for all patterns
  - Pros: More data, simpler
  - Cons: Ignores pattern-specific precision
- Per-pattern: Separate calibrator per pattern ID
  - Pros: Reflects actual pattern precision
  - Cons: Sparse data problem for rare patterns
- Hierarchical: Per-pattern with global fallback
  - Pros: Best of both
  - Cons: More complex

**Decision Criteria:**
- How many patterns do we have? (Check `patterns/core/*.yaml`)
- How many findings per pattern can we expect?
- Is there significant variance in pattern precision?

**Output Required:**
```markdown
## Calibration Scope Decision
- Selected: [Global | Per-pattern | Hierarchical]
- Minimum samples for per-pattern: [N]
- Fallback strategy: [Description]
```

## Common Pitfalls

   - **Fix:** Only choose if you have sufficient data

   - **Fix:** Always plan fallback to global
