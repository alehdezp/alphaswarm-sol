# Task 19.8: Real-World Evaluation Harness

**Status:** TODO
**Priority:** MUST
**Estimated Hours:** 16h
**Depends On:** R19.4

---

## Objective

Build a reproducible evaluation harness that compares baseline
pattern detection vs label-aware detection across real-world
protocols, producing precision/recall deltas and category metrics.

---

## Context

Relevant modules and data:
- `validation/ground-truth/` (audit-derived findings)
- `validation/scripts/validate_project.py` (baseline validation flow)
- `src/true_vkg/validation/metrics.py` (precision/recall/F1)
- `src/true_vkg/queries/label_patterns.py` (label-aware matcher)
- `benchmarks/` and `examples/` (local fixture pools)

---

## Implementation

1. Create a label-aware validator script:
   - `validation/scripts/validate_label_aware.py`
   - Accepts `--project`, `--all`, `--mode baseline|label`.
   - Generates findings via baseline patterns or label-aware patterns.

2. For each project in the corpus:
   - Load cached graph if available (`graphs/`), otherwise build once.
   - Run baseline patterns and save findings to:
     `validation/results/<project>/vkg-findings-baseline.json`
   - Run labeler + label-aware patterns and save findings to:
     `validation/results/<project>/vkg-findings-labels.json`
   - Validate both against ground truth using `FindingMatcher`.

3. Aggregate results:
   - Write `validation/results/phase-19/aggregate.json` with:
     - precision/recall/F1 per project
     - deltas (label-aware minus baseline)
     - per-category metrics (business logic, access control, state)

4. Provide a concise markdown report:
   - `validation/results/phase-19/REPORT.md`
   - Include table of deltas and notes on failure modes.
