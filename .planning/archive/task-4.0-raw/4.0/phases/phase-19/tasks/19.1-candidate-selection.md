# Task 19.1: Candidate Selection + Context Slicing

**Status:** TODO
**Priority:** MUST
**Estimated Hours:** 10h
**Depends On:** R19.1

---

## Objective

Generate a small, high-signal set of functions for LLM labeling by
using graph properties and category-aware slicing. This limits cost
and reduces hallucination.

---

## Context

Relevant modules:
- `src/true_vkg/kg/slicer.py` (category property filtering)
- `src/true_vkg/kg/property_sets.py` (category definitions)
- `src/true_vkg/learning/overlay.py` (label storage)

---

## Implementation

Create `src/true_vkg/learning/label_candidates.py`:

- `LabelCandidate` dataclass:
  - `node_id`, `category`, `score`, `graph_context`
- `CandidateSelector`:
  - Uses `GraphSlicer` to slice the graph
  - Scores candidates based on high-risk signals:
    - `has_external_calls` + `writes_state`
    - `writes_privileged_state`
    - `reads_oracle_price` + `writes_state`
    - `public_wrapper_without_access_gate`

Pseudo-logic:
```python
if node.properties.get("has_external_calls") and node.properties.get("writes_state"):
    score += 2
if node.properties.get("writes_privileged_state"):
    score += 2
if node.properties.get("public_wrapper_without_access_gate"):
    score += 2
```

Return top-N candidates per category (default 20).
