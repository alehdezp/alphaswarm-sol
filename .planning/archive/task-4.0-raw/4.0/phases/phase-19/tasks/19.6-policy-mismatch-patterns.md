**Status:** TODO
**Priority:** SHOULD

---

## Objective

Define label-aware patterns that catch authority/intent
mismatches and business logic risks.

---

## Context

Patterns live in `patterns/` and use the `labels` property
in label-aware matcher.

---

## Implementation

Create new directory:
- `patterns/semantic/`

Add patterns such as:
- `policy-privileged-action-public`
- `policy-upgrade-without-governance`
- `policy-oracle-update-no-auth`
- `policy-treasury-withdraw-no-gate`

Example:
```yaml
id: policy-treasury-withdraw-no-gate
name: Treasury Withdraw Without Authority
scope: Function
lens: [Authority]
severity: high
match:
  all:
    - property: labels
      op: contains_any
      value: ["INTENT:TREASURY_WITHDRAW", "CAPABILITY:TRANSFER_OUT"]
    - property: labels
      op: contains_any
      value: ["AUTH:PUBLIC"]
```
