# Task 19.7: Invariant + State Machine Patterns

**Status:** TODO
**Priority:** SHOULD
**Estimated Hours:** 10h
**Depends On:** 19.6

---

## Objective

Define label-aware patterns that detect invariant violations and
state machine logic errors.

---

## Context

Use labels such as:
- `CONTEXT:INVARIANT:*`
- `STATE:*`
- `REQUIRES_STATE:*`
- `TRANSITIONS_TO:*`

---

## Implementation

Create patterns in `patterns/semantic/`:

- `invariant-total-supply-mismatch`
- `state-transition-without-guard`
- `paused-state-missing-check`

Example:
```yaml
id: state-missing-paused-guard
name: Missing Paused State Guard
scope: Function
lens: [Authority]
severity: medium
match:
  all:
    - property: labels
      op: contains_any
      value: ["STATE:PAUSED"]
    - property: labels
      op: contains_any
      value: ["REQUIRES_STATE:ACTIVE"]
  none:
    - property: labels
      op: contains_any
      value: ["HAS_PAUSE_CHECK"]
```
