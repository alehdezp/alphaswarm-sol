## Objective

Select 6 real-world projects for validating BSKG detection accuracy. Projects must have:
1. Public audit reports with detailed findings
2. Solidity source code accessible
3. Known vulnerabilities documented
4. Diverse protocol types

## Project Selection Criteria

### MUST Have
- [ ] Public audit report from reputable auditor (Trail of Bits, OpenZeppelin, Consensys Diligence, Spearbit, etc.)
- [ ] Solidity source code available (not just bytecode)
- [ ] At least 5 findings in audit report
- [ ] Code compiles with Slither

### SHOULD Have
- [ ] Multiple severity levels in findings
- [ ] Mix of vulnerability types
- [ ] Active/maintained project
- [ ] Post-audit code available (for FP verification)

### Diversity Requirements
Cover at least 4 of these 6 categories:
1. **DeFi Lending** (Aave, Compound, Euler, Morpho)
2. **DEX/AMM** (Uniswap, Curve, Balancer, Sushiswap)
3. **NFT/Gaming** (Blur, Nouns, Loot)
4. **Governance/DAO** (Compound Governor, ENS)
5. **Infrastructure** (Chainlink, LayerZero, bridges)
6. **Yield/Vault** (Yearn, Convex, EigenLayer)

## Research Tasks

### 1. Gather Audit Reports

For each candidate:
- [ ] Find audit report PDF/link
- [ ] Count findings by severity
- [ ] Identify vulnerability types covered
- [ ] Note code version audited

### 2. Verify Code Availability

For each candidate:
- [ ] Clone repository
- [ ] Identify audited commit/tag
- [ ] Verify `forge build` or `npx hardhat compile` works
- [ ] Verify Slither runs without errors

### 3. Extract Ground Truth

Create ground truth file per project:

```yaml
# validation/morpho-blue/ground-truth.yaml
project: morpho-blue
audit_source: "Spearbit"
audit_date: "2024-03-15"
code_version: "v1.0.0"
commit: "abc123..."

findings:
  - id: MORPHO-001
    title: "Reentrancy in supply()"
    severity: critical
    category: reentrancy
    location:
      file: "src/Morpho.sol"
      function: "supply"
      line_range: [145, 180]
    vkg_should_find: true  # Expected BSKG detection

  - id: MORPHO-002
    title: "Business logic: incorrect interest calculation"
    severity: medium
    category: business_logic
    location:
      file: "src/libraries/Interest.sol"
      function: "computeInterest"
      line_range: [23, 45]
    vkg_should_find: false  # Business logic - BSKG won't catch

  # ... more findings
```

## What Constitutes a "Match"?

### Strict Match
- Same file
- Same function
- Same vulnerability category

### Fuzzy Match (acceptable)
- Same file, different function if related
- Related category (e.g., "reentrancy" vs "cross-function reentrancy")
- Different line number but same code block

### Not a Match
- Different file entirely
- Different vulnerability type
- Generic finding that doesn't match specific audit finding
