## The 8 Metrics with Data Sources

### Metric 1: Detection Rate

| Field | Value |
|-------|-------|
| **Name** | `detection_rate` |
| **Formula** | `detected_vulns / expected_vulns` |
| **Target** | >= 80% |
| **Alert Warning** | < 75% |
| **Alert Critical** | < 70% |
| **Unit** | percentage |

**Data Source:**
- **expected_vulns**: From `tests/projects/*/MANIFEST.yaml` files. Each manifest lists expected vulnerabilities with pattern IDs.
- **detected_vulns**: From BSKG scan output. Compare pattern matches to manifest.
- **Collection Point**: After `vkg scan` on benchmark projects.

**Example Manifest:**
```yaml
# tests/projects/defi-lending/MANIFEST.yaml
expected_vulnerabilities:
  - pattern: vm-001
    function: withdraw
    line: 42
  - pattern: auth-003
    function: setOwner
    line: 15
```

---

### Metric 2: False Positive Rate

| Field | Value |
|-------|-------|
| **Name** | `false_positive_rate` |
| **Formula** | `FP / (FP + TP)` |
| **Target** | < 15% |
| **Alert Warning** | > 18% |
| **Alert Critical** | > 20% |
| **Unit** | percentage |

**Data Source:**
- **TP (True Positive)**: Finding matches an expected vulnerability in MANIFEST.yaml
- **FP (False Positive)**: Finding does NOT match any expected vulnerability
- **Collection Point**: After comparing scan results to manifests

**Important:** Requires labeled benchmark data. Without manifests, FP rate is UNDEFINED.

---

### Metric 3: Pattern Precision (Per-Pattern)

| Field | Value |
|-------|-------|
| **Name** | `pattern_precision` |
| **Formula** | `TP / (TP + FP)` per pattern, then average |
| **Target** | avg > 85% |
| **Alert Warning** | avg < 80% |
| **Alert Critical** | avg < 75% |
| **Unit** | percentage |

**Data Source:**
- Same as False Positive Rate, but grouped by pattern_id
- **Collection Point**: After scan, group findings by pattern

**Output Format:**
```json
{
  "pattern_precision": {
    "average": 0.87,
    "by_pattern": {
      "vm-001": 0.95,
      "auth-003": 0.82,
      "dos-001": 0.75
    }
  }
}
```

---

### Metric 4: Scaffold Compile Rate

| Field | Value |
|-------|-------|
| **Name** | `scaffold_compile_rate` |
| **Formula** | `compiled_scaffolds / total_scaffolds` |
| **Target** | >= 60% |
| **Alert Warning** | < 50% |
| **Alert Critical** | < 40% |
| **Unit** | percentage |

**Data Source:**
- **total_scaffolds**: From Bead system (Phase 6). Each finding that enters Tier B verification generates a scaffold.
- **compiled_scaffolds**: Scaffolds that pass Foundry compilation
- **Collection Point**: `src/true_vkg/kg/scaffold.py::compile_scaffold()` - logs success/failure

**Dependency:** Phase 6 (Beads) must be complete. If Beads not available, this metric is SKIPPED.

---

### Metric 5: LLM Autonomy

| Field | Value |
|-------|-------|
| **Name** | `llm_autonomy` |
| **Formula** | `auto_resolved / total_tier_b_findings` |
| **Target** | >= 70% |
| **Alert Warning** | < 60% |
| **Alert Critical** | < 50% |
| **Unit** | percentage |

**Data Source:**
- **total_tier_b_findings**: Findings that require LLM verification (from Bead system)
- **auto_resolved**: Findings where LLM verdict was accepted without human escalation
- **Collection Point**: Bead verdict logging

**Dependency:** Phase 6 (Beads) + Phase 11 (LLM Integration) must be complete. If unavailable, metric is SKIPPED.

---

### Metric 6: Time to Detection

| Field | Value |
|-------|-------|
| **Name** | `time_to_detection` |
| **Formula** | `average(scan_duration_per_contract)` |
| **Target** | < 30 seconds |
| **Alert Warning** | > 45 seconds |
| **Alert Critical** | > 60 seconds |
| **Unit** | seconds |

**Data Source:**
- **scan_duration**: Wall-clock time from `vkg scan <contract>` start to findings output
- **Collection Point**: CLI timing wrapper, logged per contract

**Measurement Code:**
```python
# In src/true_vkg/cli.py scan command
import time
start = time.perf_counter()
# ... scan logic ...
duration = time.perf_counter() - start
tracker.record_timing("scan", contract_path, duration)
```

---

### Metric 7: Token Efficiency

| Field | Value |
|-------|-------|
| **Name** | `token_efficiency` |
| **Formula** | `average(tokens_per_finding_resolution)` |
| **Target** | < 10,000 tokens |
| **Alert Warning** | > 15,000 tokens |
| **Alert Critical** | > 20,000 tokens |
| **Unit** | tokens |

**Data Source:**
- **tokens_per_resolution**: Tokens sent to LLM for Tier B verification
- **Collection Point**: LLM client telemetry (`src/true_vkg/llm/telemetry.py`)

**Dependency:** Phase 11 (LLM Integration). If unavailable, metric is SKIPPED.

---

### Metric 8: Escalation Rate

| Field | Value |
|-------|-------|
| **Name** | `escalation_rate` |
| **Formula** | `human_escalations / total_tier_b_findings` |
| **Target** | < 20% |
| **Alert Warning** | > 25% |
| **Alert Critical** | > 30% |
| **Unit** | percentage |

**Data Source:**
- **human_escalations**: Findings where LLM was uncertain and requested human review
- **total_tier_b_findings**: Same as Metric 5
- **Collection Point**: Bead escalation logging

**Dependency:** Phase 6 (Beads). If unavailable, metric is SKIPPED.

```python
"""Metric definitions with thresholds and data sources."""
from dataclasses import dataclass
from typing import Optional, List
from .types import MetricName


@dataclass
class MetricDefinition:
    """Definition of a BSKG metric."""
    name: MetricName
    formula: str
    target: float
    threshold_warning: float
    threshold_critical: float
    unit: str
    higher_is_better: bool
    data_sources: List[str]
    dependencies: List[str]  # Phase dependencies


# The 8 metric definitions
METRIC_DEFINITIONS = {
    MetricName.DETECTION_RATE: MetricDefinition(
        name=MetricName.DETECTION_RATE,
        formula="detected_vulns / expected_vulns",
        target=0.80,
        threshold_warning=0.75,
        threshold_critical=0.70,
        unit="percentage",
        higher_is_better=True,
        data_sources=["tests/projects/*/MANIFEST.yaml", "scan output"],
        dependencies=[]
    ),
    MetricName.FALSE_POSITIVE_RATE: MetricDefinition(
        name=MetricName.FALSE_POSITIVE_RATE,
        formula="FP / (FP + TP)",
        target=0.15,
        threshold_warning=0.18,
        threshold_critical=0.20,
        unit="percentage",
        higher_is_better=False,
        data_sources=["tests/projects/*/MANIFEST.yaml", "scan output"],
        dependencies=[]
    ),

        name=MetricName.PATTERN_PRECISION,
        formula="avg(TP / (TP + FP) per pattern)",
        target=0.85,
        threshold_warning=0.80,
        threshold_critical=0.75,
        unit="percentage",
        higher_is_better=True,
        data_sources=["tests/projects/*/MANIFEST.yaml", "scan output"],

    ),
    MetricName.SCAFFOLD_COMPILE_RATE: MetricDefinition(
```

```python
from true_vkg.metrics.definitions import METRIC_DEFINITIONS, get_available_metrics, MetricName


class TestMetricDefinitions:
    """Test metric definitions."""

    def test_all_8_metrics_defined(self):
        """All 8 metrics have definitions."""
        assert len(METRIC_DEFINITIONS) == 8
        for name in MetricName:
            assert name in METRIC_DEFINITIONS

    def test_detection_rate_thresholds(self):
        """Detection rate has correct thresholds."""
        defn = METRIC_DEFINITIONS[MetricName.DETECTION_RATE]
        assert defn.target == 0.80
        assert defn.threshold_critical == 0.70
        assert defn.higher_is_better is True

    def test_fp_rate_lower_is_better(self):
        """FP rate is a lower-is-better metric."""
        defn = METRIC_DEFINITIONS[MetricName.FALSE_POSITIVE_RATE]
        assert defn.higher_is_better is False

    def test_available_metrics_excludes_dependencies(self):
        """Available metrics doesn't include those needing Phase 6/11."""
        available = get_available_metrics()
        # Should not include Bead-dependent metrics
        assert MetricName.SCAFFOLD_COMPILE_RATE not in available
        assert MetricName.LLM_AUTONOMY not in available
```
