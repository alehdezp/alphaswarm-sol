## CRITICAL CONTEXT

**Why this task matters:**
AI coding agents (Claude Code, Cursor, Cody, etc.) discover available tools by reading AGENTS.md files. Without this file, an LLM in a fresh session has NO IDEA that BSKG exists or how to use it. This is THE entry point for agentic automation.

**What success looks like:**
1. User starts fresh Claude Code session in project with VKG
2. User says: "Find security vulnerabilities in this codebase"
3. Claude Code reads AGENTS.md, discovers BSKG commands
4. Claude Code runs BSKG audit autonomously
5. User gets findings without ever typing a BSKG command

**What failure looks like:**
- User has to explain BSKG to Claude Code every session
- User has to manually type all BSKG commands
- Tool is effectively unusable for AI-assisted workflows

## 3. RESEARCH: WHAT MAKES AGENTS.MD EFFECTIVE

Before implementing, study these effective AGENTS.md patterns:

### Pattern 1: Quick Start First
```markdown
# ToolName - One-Line Description

## Quick Start (Copy-Paste Ready)
```bash
tool init
tool analyze
tool report
```
```

**Why:** LLMs scan for actionable commands. Put them at the top.

### Pattern 2: When to Use Each Command
```markdown
| Command | When to Use |
|---------|-------------|
| `tool build` | First step, always |
| `tool analyze` | After build |
```

**Why:** LLMs need decision trees, not just command lists.

### Pattern 3: Workflow, Not Reference
```markdown
## Workflow
1. Build: `vkg build contracts/`
2. Analyze: `vkg analyze`
3. Investigate: For each finding...
4. Report: `vkg report`
```

**Why:** LLMs work better with explicit sequences than inference.

### Pattern 4: Error Recovery
```markdown
## If Something Goes Wrong
| Error | Fix |
|-------|-----|
| "No contracts found" | Check path exists |
```

**Why:** LLMs need recovery paths, not just happy paths.

## 4. AGENTS.MD TEMPLATE

Create `src/true_vkg/templates/AGENTS.md.template`:

```markdown
# BSKG - Solidity Vulnerability Detector

Automated security analysis for Solidity smart contracts. Detects vulnerabilities in DeFi protocols, access control, reentrancy, oracle manipulation, and more.

## Quick Start

```bash
# Initialize BSKG in this project
vkg init

# Build knowledge graph from contracts
vkg build contracts/

# Run vulnerability analysis
vkg analyze

# See all findings
vkg findings list

# Get next priority finding to investigate
vkg findings next

# Generate report
vkg report --format sarif
```

## Available Commands

| Command | Purpose | When to Use |
|---------|---------|-------------|
| `vkg init` | Initialize project | Once per project |
| `vkg build <path>` | Build knowledge graph | First step, after code changes |
| `vkg analyze` | Run vulnerability detection | After build |
| `vkg findings list` | List all findings | Overview of issues |
| `vkg findings next` | Get priority finding | Start investigation |
| `vkg findings show <id>` | Show finding details | Investigate specific |
| `vkg findings update <id> --status <status>` | Update verdict | After investigation |
| `vkg report --format sarif` | Generate report | End of audit |

## Audit Workflow

### Step 1: Build Knowledge Graph
```bash
vkg build contracts/
```
Output: Graph built with X contracts, Y functions

### Step 2: Run Analysis
```bash
vkg analyze
```
Output: Found N potential vulnerabilities

### Step 3: Investigate Findings
```bash
# Get next priority finding
vkg findings next

# After investigation, update status
vkg findings update VKG-001 --status confirmed
# or
vkg findings update VKG-001 --status rejected --reason "False positive: protected by modifier"
```

### Step 4: Generate Report
```bash
vkg report --format sarif > security-report.sarif
```

## Understanding Output

### Finding Structure
```json
{
  "id": "VKG-001",
  "pattern": "reentrancy-classic",
  "severity": "critical",
  "location": "Vault.sol:45",
  "behavioral_signature": "R:bal->X:out->W:bal",
  "evidence": ["state_write_after_external_call = true"]
}
```

### Severity Levels
- **critical**: Direct fund loss possible
- **high**: Significant impact, exploitable
- **medium**: Limited impact or complex exploit
- **low**: Informational or edge case

### Status Values
- **pending**: Not yet reviewed
- **investigating**: Currently analyzing
- **confirmed**: Verified vulnerability
- **rejected**: False positive
- **fixed**: Issue resolved

## Common Issues

| Problem | Solution |
|---------|----------|
| "No contracts found" | Check path exists and contains .sol files |
| "Build failed" | Run `forge build` or `npx hardhat compile` first |
| "Slither not found" | Install: `pip install slither-analyzer` |
| Analysis incomplete | Check warnings for skipped contracts |

## Tips for AI Agents

1. **Always run `vkg build` before `vkg analyze`**
2. **Use `vkg findings next` for priority-based investigation**
3. **Read `vkg findings show <id>` for full context before judging**
4. **Update status after each investigation to track progress**
5. **Check analysis completeness in output (coverage percentage)**

## More Information

- Pattern documentation: `vkg patterns list`
- Query specific patterns: `vkg query "pattern:reentrancy-classic"`
- Natural language queries: `vkg query "functions that transfer ETH without access control"`

## 5. IMPLEMENTATION

### Step 1: Create Template System (1h)

```python
# src/true_vkg/templates/__init__.py

from pathlib import Path
from string import Template

TEMPLATE_DIR = Path(__file__).parent


def get_template(name: str) -> str:
    """Load template file."""
    path = TEMPLATE_DIR / f"{name}.template"
    if not path.exists():
        raise FileNotFoundError(f"Template not found: {name}")
    return path.read_text()


def render_agents_md(
    project_name: str = "Project",
    vkg_version: str = "4.0.0",
) -> str:
    """Render AGENTS.md with project-specific values."""
    template = get_template("AGENTS.md")
    return Template(template).safe_substitute(
        project_name=project_name,
        vkg_version=vkg_version,
    )
```

### Step 2: Add CLI Command (1h)

```python
# In src/true_vkg/cli.py

@click.command()
@click.option("--output", "-o", default=".vrs/AGENTS.md")
def init(output: str):
    """Initialize BSKG for this project."""
    output_path = Path(output)
    output_path.parent.mkdir(parents=True, exist_ok=True)

    agents_md = render_agents_md(
        project_name=Path.cwd().name,
        vkg_version=get_version(),
    )

    output_path.write_text(agents_md)
    click.echo(f"Created {output_path}")
    click.echo()
    click.echo("Next steps:")
    click.echo("  1. vkg build contracts/")
    click.echo("  2. vkg analyze")
```
