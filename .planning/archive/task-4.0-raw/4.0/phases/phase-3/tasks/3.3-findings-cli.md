## CRITICAL CONTEXT

**Why this task matters:**
The findings CLI is the primary interface for auditors (human or LLM) to interact with detected vulnerabilities. Without these commands, findings are just data in JSON files with no workflow support.

**What success looks like:**
```bash
$ vkg findings list
VKG-001 [CRITICAL] reentrancy-classic   Vault.withdraw      pending
VKG-002 [HIGH]     access-missing-gate  Admin.setFee        pending
VKG-003 [MEDIUM]   oracle-no-staleness  PriceFeed.getPrice  investigating

$ vkg findings next
VKG-001: Classic Reentrancy in Vault.withdraw
Severity: CRITICAL | Confidence: 92%
...

$ vkg findings update VKG-001 --status confirmed --note "Verified with PoC"
Updated VKG-001: pending -> confirmed
```

## 4. COMMAND SPECIFICATIONS

### 4.1 `vkg findings list`

**Purpose:** List all findings with optional filters

**Options:**
```
--status <status>     Filter by status (pending, investigating, confirmed, rejected)
--severity <severity> Filter by severity (critical, high, medium, low)
--pattern <pattern>   Filter by pattern ID
--limit <n>          Limit results (default: 50)
--format <format>    Output format (table, json, csv)
```

**Example Output (table):**
```
ID        SEVERITY   PATTERN              LOCATION              STATUS
VKG-001   CRITICAL   reentrancy-classic   Vault.sol:45          pending
VKG-002   HIGH       access-missing-gate  Admin.sol:23          pending
VKG-003   MEDIUM     oracle-no-staleness  Price.sol:67          investigating

Total: 3 findings | 2 pending | 1 investigating | 0 confirmed
```

**Example Output (json):**
```json
{
  "findings": [
    {"id": "VKG-001", "severity": "critical", ...}
  ],
  "summary": {
    "total": 3,
    "by_status": {"pending": 2, "investigating": 1}
  }
}
```

### 4.2 `vkg findings show <id>`

**Purpose:** Show complete finding details

**Arguments:**
- `id` (required): Finding ID (e.g., VKG-001)

**Example Output:**
```
Finding VKG-001: reentrancy-classic

Location: Vault.sol:45 (withdraw)
Severity: CRITICAL | Confidence: 92% | Tier: A
Status: pending

Behavioral Signature: R:bal->X:out->W:bal
  (Read balance, External call out, Write balance)

Evidence:
  1. [code] Vault.sol:45-52 - withdraw function body
  2. [property] state_write_after_external_call = true
  3. [call_graph] withdraw -> transfer (external, value)

Why Vulnerable:
  External call transfers ETH to msg.sender before balance
  is decremented. Attacker can re-enter via receive() and
  withdraw multiple times.

Minimal Reproduction:
  1. Deploy attacker contract with receive() calling withdraw
  2. Deposit funds
  3. Call withdraw - receive() triggers, re-enters
  4. Balance not decremented, full amount withdrawn again

Fix: Apply CEI pattern - update balance before external call

Investigation Notes:
  (none)

Actions:
  vkg findings update VKG-001 --status confirmed
  vkg findings update VKG-001 --status rejected --reason "Protected by nonReentrant"
```

### 4.3 `vkg findings next`

**Purpose:** Get next highest-priority finding to investigate

**Options:**
```
--status <status>  Filter by status (default: pending)
```

**Logic:**
1. Filter by status (default: pending)
2. Sort by severity (critical > high > medium > low)
3. Within same severity, sort by confidence (higher first)
4. Return first finding

**Example Output:**
Same as `vkg findings show <id>` for the selected finding.

### 4.4 `vkg findings update <id>`

**Purpose:** Update finding status and add notes

**Arguments:**
- `id` (required): Finding ID

**Options:**
```
--status <status>    New status (pending, investigating, confirmed, rejected, fixed)
--reason <reason>    Reason for status change (required for rejected)
--note <note>        Add investigation note
```

**Example:**
```bash
# Confirm vulnerability
vkg findings update VKG-001 --status confirmed --note "Verified with PoC test"

# Reject as false positive
vkg findings update VKG-002 --status rejected --reason "Protected by onlyOwner modifier"

# Add note without status change
vkg findings update VKG-003 --note "Needs further investigation of oracle source"
```

**Output:**
```
Updated VKG-001: pending -> confirmed
Note added: "Verified with PoC test"
```

### 4.5 `vkg findings refresh`

**Purpose:** Refresh findings after code changes

**Options:**
```
--stale-only    Only refresh stale findings
--clear         Clear all findings and re-analyze
```

**Logic:**
1. Compare current graph fingerprint with finding fingerprints
2. Mark mismatched findings as stale
3. Optionally re-run analysis

**Output:**
```
Checking 10 findings against current graph...

Stale: 3 findings (graph changed)
  VKG-001: Vault.sol modified
  VKG-003: PriceFeed.sol modified
  VKG-007: Function removed

Current: 7 findings (unchanged)

Run 'vkg analyze' to generate new findings.
```

### 4.6 `vkg findings export`

**Purpose:** Export findings to various formats

**Options:**
```
--format <format>   Output format (json, sarif, csv, markdown)
--output <path>     Output file (default: stdout)
--status <status>   Filter by status
```

### Step 1: Create Findings CLI Module (2h)

```python
# src/true_vkg/cli/findings.py

import click
from pathlib import Path
from true_vkg.findings.store import FindingStore
from true_vkg.findings.model import Status, Severity


@click.group()
def findings():
    """Manage vulnerability findings."""
    pass


@findings.command("list")
@click.option("--status", type=click.Choice([s.value for s in Status]))
@click.option("--severity", type=click.Choice([s.value for s in Severity]))
@click.option("--pattern", help="Filter by pattern ID")
@click.option("--limit", default=50, help="Limit results")
@click.option("--format", "fmt", type=click.Choice(["table", "json"]), default="table")
def list_findings(status, severity, pattern, limit, fmt):
    """List all findings."""
    store = FindingStore()

    # Apply filters
    findings = list(store.list())

    if status:
        findings = [f for f in findings if f.status.value == status]
    if severity:
        findings = [f for f in findings if f.severity.value == severity]
    if pattern:
        findings = [f for f in findings if f.pattern_id == pattern]

    findings = findings[:limit]

    if fmt == "json":
        output = {"findings": [f.to_dict() for f in findings]}
        click.echo(json.dumps(output, indent=2))
    else:
        _print_findings_table(findings)
        _print_summary(findings)


@findings.command("show")
@click.argument("finding_id")
def show_finding(finding_id: str):
    """Show finding details."""
    store = FindingStore()
    finding = store.read(finding_id)

    if not finding:
        click.echo(f"Finding not found: {finding_id}", err=True)
        raise SystemExit(1)

    click.echo(_format_finding_detail(finding))


@findings.command("next")
@click.option("--status", default="pending")
def next_finding(status: str):
    """Get next priority finding."""
    store = FindingStore()

    if status == "pending":
        finding = store.get_next_priority()
    else:
        findings = list(store.list(status=Status(status)))
        finding = findings[0] if findings else None

    if not finding:
        click.echo("No findings matching criteria.")
        return

    click.echo(_format_finding_detail(finding))


@findings.command("update")
@click.argument("finding_id")
@click.option("--status", type=click.Choice([s.value for s in Status]))
@click.option("--reason", help="Reason for status change")
@click.option("--note", help="Add investigation note")
def update_finding(finding_id: str, status: str, reason: str, note: str):
    """Update finding status or add note."""
    store = FindingStore()
    finding = store.read(finding_id)

    if not finding:
        click.echo(f"Finding not found: {finding_id}", err=True)
        raise SystemExit(1)

    old_status = finding.status.value

    if status:
        if status == "rejected" and not reason:
            click.echo("--reason required when rejecting", err=True)
            raise SystemExit(1)
        finding.set_status(Status(status), reason or "")
        click.echo(f"Updated {finding_id}: {old_status} -> {status}")

    if note:
        finding.add_note(note)
        click.echo(f"Note added: {note}")

    store.update(finding)


@findings.command("refresh")
@click.option("--stale-only", is_flag=True)
@click.option("--clear", is_flag=True)
def refresh_findings(stale_only: bool, clear: bool):
    """Refresh findings after code changes."""
    store = FindingStore()

    if clear:
        count = store.clear_all()
        click.echo(f"Cleared {count} findings.")
        click.echo("Run 'vkg analyze' to generate new findings.")
        return

    # Get current graph fingerprint
    from true_vkg.kg.store import GraphStore
    graph_store = GraphStore()
    current_fp = graph_store.get_fingerprint()

    stale_ids = store.mark_stale(current_fp)

    if stale_ids:
        click.echo(f"Stale: {len(stale_ids)} findings (graph changed)")
        for fid in stale_ids:
            click.echo(f"  {fid}")
    else:
        click.echo("All findings current.")


def _print_findings_table(findings: list) -> None:
    """Print findings as table."""
    header = f"{'ID':<10} {'SEVERITY':<10} {'PATTERN':<25} {'LOCATION':<25} {'STATUS':<12}"
    click.echo(header)
    click.echo("-" * len(header))

    for f in findings:
        loc = f"{f.location.file}:{f.location.line}"
        click.echo(
            f"{f.id:<10} {f.severity.value.upper():<10} "
            f"{f.pattern_id[:25]:<25} {loc[:25]:<25} {f.status.value:<12}"
        )


def _print_summary(findings: list) -> None:
    """Print findings summary."""
    by_status = {}
    for f in findings:
        by_status[f.status.value] = by_status.get(f.status.value, 0) + 1

    parts = [f"Total: {len(findings)} findings"]
    for status, count in by_status.items():
        parts.append(f"{count} {status}")

    click.echo()
    click.echo(" | ".join(parts))


def _format_finding_detail(finding) -> str:
    """Format full finding for display."""
    from true_vkg.findings.signature import explain_signature

    lines = []
    lines.append(f"Finding {finding.id}: {finding.pattern_id}")
    lines.append("")
    lines.append(f"Location: {finding.location.file}:{finding.location.line} ({finding.location.function})")
    lines.append(f"Severity: {finding.severity.value.upper()} | Confidence: {finding.confidence*100:.0f}% | Tier: {finding.tier.value[-1].upper()}")
    lines.append(f"Status: {finding.status.value}")
    lines.append("")

    if finding.behavioral_signature:
        lines.append(f"Behavioral Signature: {finding.behavioral_signature}")
        lines.append(f"  ({explain_signature(finding.behavioral_signature)})")
        lines.append("")

    if finding.evidence:
        lines.append("Evidence:")
        for i, evd in enumerate(finding.evidence, 1):
            lines.append(f"  {i}. [{evd.type}] {evd.summary}")
        lines.append("")

    if finding.why_vulnerable:
        lines.append("Why Vulnerable:")
        for line in finding.why_vulnerable.split("\n"):
            lines.append(f"  {line}")
        lines.append("")

    if finding.minimal_repro_steps:
        lines.append("Minimal Reproduction:")
        for step in finding.minimal_repro_steps:
            lines.append(f"  {step}")
        lines.append("")

    if finding.fix_recommendation:
        lines.append(f"Fix: {finding.fix_recommendation}")
        lines.append("")

    if finding.notes:
        lines.append("Investigation Notes:")
        for note in finding.notes:
            lines.append(f"  - {note}")
        lines.append("")

    lines.append("Actions:")
    lines.append(f"  vkg findings update {finding.id} --status confirmed")
    lines.append(f"  vkg findings update {finding.id} --status rejected --reason \"...\"")

    return "\n".join(lines)
```

def _severity_color(severity: Severity) -> str:

        Severity.HIGH: "yellow",
        Severity.MEDIUM: "blue",

import pytest

class TestFindingsList:
    def test_list_empty(self, runner):
        result = runner.invoke(main, ["findings", "list"])
        assert result.exit_code == 0
        assert "Total: 0" in result.output

    def test_list_with_findings(self, runner, store):
