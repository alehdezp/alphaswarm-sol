## Executive Summary

OpenCode is an open-source AI coding agent built for the terminal that supports 75+ LLM providers. This research documents how BSKG can be exposed as a tool for OpenCode, enabling AI-assisted vulnerability audits.

**Key Findings:**
1. OpenCode discovers tools via MCP (Model Context Protocol) servers or custom tool definitions
2. Configuration uses `opencode.json` with a well-defined schema
3. BSKG can integrate via either MCP server (recommended) or custom tool definitions
4. OpenCode supports local MCP servers via stdio transport (perfect for BSKG CLI)

## 1. OpenCode SDK Overview

### 1.1 What is OpenCode?

OpenCode is an open-source AI coding agent that provides:
- **Terminal UI (TUI)**: Interactive Bubble Tea-based interface
- **CLI mode**: Non-interactive execution via `opencode run`
- **Server mode**: HTTP API for programmatic control
- **75+ LLM providers**: Including Claude, GPT, Gemini, Groq, and more
- **Tool system**: Built-in and custom tools the LLM can invoke

**Official Resources:**
- Website: https://opencode.ai/
- Documentation: https://opencode.ai/docs/
- GitHub: https://github.com/opencode-ai/opencode (archived, now charmbracelet/opencode)
- SDK: `npm install @opencode-ai/sdk`

## 2. Tool Discovery Mechanism

### 2.1 How OpenCode Discovers Tools

OpenCode uses three mechanisms for tool discovery:

1. **Built-in Tools**: 13 tools always available (bash, edit, write, read, grep, glob, list, patch, lsp, skill, todowrite, todoread, webfetch)

2. **MCP Servers**: External tools via Model Context Protocol
   - Local servers (stdio transport)
   - Remote servers (HTTP transport)
   - Tools auto-registered with server name prefix

3. **Custom Tools**: User-defined tools via TypeScript/JavaScript files
   - Located in `.opencode/tool/` or `~/.config/opencode/tool/`
   - Automatically discovered and registered

### 2.2 MCP Tool Discovery Flow

```
┌─────────────────────────────────────────────────────────────────┐
│                    MCP Tool Discovery Flow                        │
├─────────────────────────────────────────────────────────────────┤
│                                                                   │
│  1. Config Load                                                   │
│     opencode.json → mcp: { "vkg": { ... } }                       │
│                              │                                     │
│  2. Server Start             │                                     │
│     spawn process ←──────────┘                                     │
│     (uv run alphaswarm mcp-serve)                                   │
│                              │                                     │
│  3. Tool Registration        │                                     │
│     MCP server sends ←───────┘                                     │
│     tools/list response                                           │
│                              │                                     │
│  4. Tool Available           │                                     │
│     LLM can call ←───────────┘                                     │
│     vkg_build, vkg_analyze, etc.                                  │
│                                                                   │
└─────────────────────────────────────────────────────────────────┘
```

### 2.3 Tool Registration Schema

When an MCP server starts, it responds to `tools/list` with:

```json
{
  "tools": [
    {
      "name": "build_kg",
      "description": "Build knowledge graph from Solidity contracts",
      "inputSchema": {
        "type": "object",
        "properties": {
          "path": {
            "type": "string",
            "description": "Path to contracts directory"
          }
        },
        "required": ["path"]
      }
    },
    {
      "name": "analyze",
      "description": "Run vulnerability analysis on built graph",
      "inputSchema": {
        "type": "object",
        "properties": {
          "patterns": {
            "type": "array",
            "items": { "type": "string" },
            "description": "Pattern IDs to check (optional, all if omitted)"
          }
        }
      }
    }
  ]
}
```

## 3. Configuration Format/Schema

### 3.1 Configuration File Locations

OpenCode loads configuration from multiple sources (in priority order):

| Priority | Location | Scope |
|----------|----------|-------|
| 1 | Remote `.well-known/opencode` | Organization |
| 2 | `~/.config/opencode/opencode.json` | Global user |
| 3 | `OPENCODE_CONFIG` env var path | Custom |
| 4 | `./opencode.json` | Project |
| 5 | `.opencode/` directory | Project-specific |
| 6 | `OPENCODE_CONFIG_CONTENT` env var | Inline |

**Note:** Configurations are **merged**, not replaced.

### 3.2 Full Configuration Schema

### 3.3 MCP Server Configuration Options

#### Local MCP Server (stdio transport)

```json
{
  "mcp": {
    "server-name": {
      "type": "local",
      "command": ["executable", "arg1", "arg2"],
      "environment": {
        "VAR_NAME": "value",
        "FROM_ENV": "{env:EXTERNAL_VAR}"
      },
      "enabled": true,
      "timeout": 5000
    }
  }
}
```

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `type` | `"local"` | Yes | Local server via stdio |
| `command` | `string[]` | Yes | Command and arguments |
| `environment` | `object` | No | Environment variables |
| `enabled` | `boolean` | No | Enable/disable (default: true) |
| `timeout` | `number` | No | Startup timeout in ms |

#### Remote MCP Server (HTTP transport)

```json
{
  "mcp": {
    "server-name": {
      "type": "remote",
      "url": "https://mcp-server.com",
      "headers": {
        "Authorization": "Bearer {env:API_KEY}"
      },
      "oauth": {
        "clientId": "{env:CLIENT_ID}",
        "clientSecret": "{env:CLIENT_SECRET}",
        "scope": "tools:read"
      },
      "enabled": true,
      "timeout": 10000
    }
  }
}
```

### 3.4 Variable Substitution

OpenCode supports variable substitution in config values:

| Pattern | Description | Example |
|---------|-------------|---------|
| `{env:VAR_NAME}` | Environment variable | `{env:API_KEY}` |
| `{file:path}` | File contents | `{file:~/.secrets/key}` |
| `{cwd}` | Current working directory | `{cwd}` |

## 4. Integration Architecture for VKG

### 4.1 Recommended Approach: MCP Server

The recommended integration is via MCP server because:
1. **Standard protocol**: MCP is the de-facto standard for AI tool integration
2. **Rich schema**: Full JSON Schema for parameters
3. **Async support**: Long-running operations supported
4. **Isolation**: Runs in separate process

### 5.2 BSKG MCP Server Implementation (Python)

```python
# src/true_vkg/mcp/server.py
"""VKG MCP Server for OpenCode integration."""

import sys
import json
import logging
from typing import Any

from mcp.server.fastmcp import FastMCP

from true_vkg.kg.builder import VKGBuilder
from true_vkg.findings.store import FindingsStore
from true_vkg.queries.executor import QueryExecutor

# Suppress stdout for stdio transport
logging.basicConfig(stream=sys.stderr, level=logging.INFO)
logger = logging.getLogger(__name__)

# Initialize MCP server
mcp = FastMCP("vkg")

# Global state
_builder: VKGBuilder | None = None
_findings: FindingsStore | None = None


@mcp.tool()
async def build_kg(path: str) -> str:
    """Build knowledge graph from Solidity contracts.

    Args:
        path: Path to contracts directory or file

    Returns:
        Summary of built graph (contracts, functions, properties)
    """
    global _builder

    try:
        _builder = VKGBuilder(path)
        graph = _builder.build()

        return json.dumps({
            "status": "success",
            "contracts": len(graph.contracts),
            "functions": len(graph.functions),
            "properties_extracted": sum(
                len(f.properties) for f in graph.functions
            )
        })
    except Exception as e:
        logger.error(f"Build failed: {e}")
        return json.dumps({
            "status": "error",
            "error": str(e),
            "suggestion": "Ensure path exists and contains .sol files"
        })


@mcp.tool()
async def analyze(patterns: list[str] | None = None) -> str:
    """Run vulnerability analysis on built knowledge graph.

    Args:
        patterns: Optional list of pattern IDs to check.
                  If omitted, all patterns are checked.

    Returns:
        Analysis summary with finding counts by severity
    """
    global _builder, _findings

    if _builder is None:
        return json.dumps({
            "status": "error",
            "error": "No graph built",
            "suggestion": "Run build_kg first"
        })

    try:
        findings = _builder.analyze(pattern_ids=patterns)
        _findings = FindingsStore()
        _findings.add_all(findings)

        return json.dumps({
            "status": "success",
            "total_findings": len(findings),
            "by_severity": {
                "critical": len([f for f in findings if f.severity == "critical"]),
                "high": len([f for f in findings if f.severity == "high"]),
                "medium": len([f for f in findings if f.severity == "medium"]),
                "low": len([f for f in findings if f.severity == "low"])
            }
        })
    except Exception as e:
        logger.error(f"Analysis failed: {e}")
        return json.dumps({
            "status": "error",
            "error": str(e)
        })


@mcp.tool()
async def query(query_text: str, format: str = "json") -> str:
    """Query the knowledge graph using natural language or VQL2.

    Args:
        query_text: Natural language or VQL2 query
        format: Output format (json, compact, explain)

    Returns:
        Query results
    """
    global _builder

    if _builder is None or _builder.graph is None:
        return json.dumps({
            "status": "error",
            "error": "No graph built",
            "suggestion": "Run build_kg first"
        })

    try:
        executor = QueryExecutor(_builder.graph)
        results = executor.execute(query_text)

        if format == "compact":
            return executor.format_compact(results)
        elif format == "explain":
            return executor.format_explain(results)
        else:
            return json.dumps(results, indent=2)
    except Exception as e:
        return json.dumps({
            "status": "error",
            "error": str(e)
        })


@mcp.tool()
async def findings_list(
    status: str | None = None,
    severity: str | None = None
) -> str:
    """List all findings, optionally filtered.

    Args:
        status: Filter by status (pending, investigating, confirmed, rejected)
        severity: Filter by severity (critical, high, medium, low)

    Returns:
        List of findings with summary info
    """
    global _findings

    if _findings is None:
        return json.dumps({
            "status": "error",
            "error": "No findings available",
            "suggestion": "Run analyze first"
        })

    findings = _findings.list(status=status, severity=severity)
    return json.dumps({
        "status": "success",
        "count": len(findings),
        "findings": [
            {
                "id": f.id,
                "pattern": f.pattern_id,
                "severity": f.severity,
                "location": f"{f.contract}:{f.line}",
                "status": f.status
            }
            for f in findings
        ]
    })


@mcp.tool()
async def findings_next() -> str:
    """Get the next priority finding to investigate.

    Returns the highest-priority pending finding.
    Priority is based on severity, confidence, and age.

    Returns:
        Full finding details or message if no pending findings
    """
    global _findings

    if _findings is None:
        return json.dumps({
            "status": "error",
            "error": "No findings available",
            "suggestion": "Run analyze first"
        })

    finding = _findings.next_priority()
    if finding is None:
        return json.dumps({
            "status": "success",
            "message": "No pending findings",
            "summary": _findings.summary()
        })

    return json.dumps({
        "status": "success",
        "finding": finding.to_dict()
    })


@mcp.tool()
async def findings_show(id: str) -> str:
    """Show full details for a specific finding.

    Args:
        id: Finding ID (e.g., VKG-001)

    Returns:
        Complete finding with evidence, code snippets, and verification steps
    """
    global _findings

    if _findings is None:
        return json.dumps({
            "status": "error",
            "error": "No findings available"
        })

    finding = _findings.get(id)
    if finding is None:
        return json.dumps({
            "status": "error",
            "error": f"Finding {id} not found"
        })

    return json.dumps({
        "status": "success",
        "finding": finding.to_dict()
    })


@mcp.tool()
async def findings_update(
    id: str,
    status: str,
    reason: str | None = None
) -> str:
    """Update the status of a finding.

    Args:
        id: Finding ID (e.g., VKG-001)
        status: New status (investigating, confirmed, rejected, fixed)
        reason: Reason for status change (required for confirmed/rejected)

    Returns:
        Updated finding summary
    """
    global _findings

    if _findings is None:
        return json.dumps({
            "status": "error",
            "error": "No findings available"
        })

    valid_statuses = ["investigating", "confirmed", "rejected", "fixed"]
    if status not in valid_statuses:
        return json.dumps({
            "status": "error",
            "error": f"Invalid status: {status}",
            "valid_statuses": valid_statuses
        })

    if status in ["confirmed", "rejected"] and not reason:
        return json.dumps({
            "status": "error",
            "error": f"Reason required for status '{status}'"
        })

    try:
        finding = _findings.update(id, status=status, reason=reason)
        return json.dumps({
            "status": "success",
            "finding": {
                "id": finding.id,
                "status": finding.status,
                "reason": finding.verdict_reason
            }
        })
    except Exception as e:
        return json.dumps({
            "status": "error",
            "error": str(e)
        })


@mcp.tool()
async def report(format: str = "sarif") -> str:
    """Generate audit report.

    Args:
        format: Report format (sarif, json, markdown)

    Returns:
        Report content in specified format
    """
    global _findings

    if _findings is None:
        return json.dumps({
            "status": "error",
            "error": "No findings available"
        })

    try:
        if format == "sarif":
            return _findings.export_sarif()
        elif format == "markdown":
            return _findings.export_markdown()
        else:
            return json.dumps(_findings.export_json(), indent=2)
    except Exception as e:
        return json.dumps({
            "status": "error",
            "error": str(e)
        })


@mcp.tool()
async def patterns_list(lens: str | None = None) -> str:
    """List available vulnerability patterns.

    Args:
        lens: Filter by lens (e.g., Reentrancy, AccessControl, Oracle)

    Returns:
        List of patterns with descriptions
    """
    from true_vkg.patterns.registry import PatternRegistry

    registry = PatternRegistry()
    patterns = registry.list(lens=lens)

    return json.dumps({
        "status": "success",
        "count": len(patterns),
        "patterns": [
            {
                "id": p.id,
                "name": p.name,
                "severity": p.severity,
                "lens": p.lens,
                "description": p.description[:100] + "..."
                    if len(p.description) > 100 else p.description
            }
            for p in patterns
        ]
    })


def main():
    """Run MCP server on stdio transport."""
    mcp.run(transport="stdio")


if __name__ == "__main__":
    main()
```

### 5.4 Alternative: Custom Tool Definitions

If MCP is not preferred, use custom tool definitions in `.opencode/tool/vkg.ts`:

```typescript
// .opencode/tool/vkg.ts
import { tool } from "@opencode-ai/plugin"
import { $ } from "bun"

export const build = tool({
  description: "Build BSKG knowledge graph from Solidity contracts",
  args: {
    path: tool.schema.string().describe("Path to contracts directory")
  },
  async execute({ path }) {
    const result = await $`uv run alphaswarm build-kg ${path} --json`.text()
    return result
  }
})

export const analyze = tool({
  description: "Run vulnerability analysis",
  args: {
    patterns: tool.schema.array(tool.schema.string()).optional()
      .describe("Pattern IDs to check")
  },
  async execute({ patterns }) {
    const args = patterns ? patterns.map(p => `--pattern ${p}`).join(" ") : ""
    const result = await $`uv run alphaswarm analyze ${args} --json`.text()
    return result
  }
})

export const query = tool({
  description: "Query the knowledge graph",
  args: {
    query: tool.schema.string().describe("Natural language or VQL2 query"),
    format: tool.schema.enum(["json", "compact", "explain"]).optional()
  },
  async execute({ query, format = "json" }) {
    const result = await $`uv run alphaswarm query ${query} --format ${format}`.text()
    return result
  }
```

```python
# src/true_vkg/templates/opencode.py
"""OpenCode configuration generator for BSKG integration."""

import json
from pathlib import Path
from typing import Any


def generate_opencode_config(
    project_path: Path,
    vkg_command: list[str] | None = None,
    additional_mcp: dict[str, Any] | None = None,
) -> dict[str, Any]:
    """Generate opencode.json configuration for BSKG integration.

    Args:
        project_path: Path to the project root
        vkg_command: Custom command for BSKG MCP server
        additional_mcp: Additional MCP servers to include

    Returns:
        Complete opencode.json configuration dict
    """
    if vkg_command is None:
        vkg_command = ["uv", "run", "alphaswarm", "mcp-serve"]

    config = {
        "$schema": "https://opencode.ai/config.json",

        # BSKG MCP Server
        "mcp": {
            "vkg": {
                "type": "local",
                "command": vkg_command,
                "environment": {
                    "VKG_PROJECT": str(project_path),
                    "VKG_LOG_LEVEL": "INFO"
                },
                "enabled": True,
                "timeout": 30000
            }
        },

        # Enable BSKG tools
        "tools": {
            "vkg_*": True
        },

        # Tool permissions
        "permission": {
            "vkg_build": "allow",
            "vkg_analyze": "allow",
            "vkg_query": "allow",
            "vkg_findings_list": "allow",
            "vkg_findings_next": "allow",
            "vkg_findings_show": "allow",
            "vkg_findings_update": "ask",  # Requires confirmation
            "vkg_report": "allow"
        },

        # Include AGENTS.md for context
        "instructions": [
            ".vrs/AGENTS.md"
        ],

        # Custom command for quick audit start
        "command": {
            "audit": {
                "template": "Run a complete security audit on this Solidity project",
                "description": "Start BSKG vulnerability audit"
            }
        }
    }

    # Add additional MCP servers if provided
    if additional_mcp:
        config["mcp"].update(additional_mcp)

    return config


def write_opencode_config(
    project_path: Path,
    output_path: Path | None = None,
    **kwargs
) -> Path:
    """Write opencode.json to project.

    Args:
        project_path: Path to the project root
        output_path: Custom output path (default: project_path/opencode.json)
        **kwargs: Additional arguments for generate_opencode_config

    Returns:
        Path to written config file
    """
    if output_path is None:
        output_path = project_path / "opencode.json"

    config = generate_opencode_config(project_path, **kwargs)

    output_path.write_text(json.dumps(config, indent=2))
    return output_path
```

## 6. Limitations and Considerations

### 6.1 Technical Limitations

| Limitation | Impact | Mitigation |
|------------|--------|------------|
| MCP context overhead | Many tools add tokens | Keep tool descriptions concise |
| Stdio transport only for local | No HTTP for local servers | Use remote MCP for hosted BSKG |
| No streaming tool output | Long operations block | Add progress indicators |
| Tool timeout limits | Large projects may timeout | Implement chunked processing |

### 6.2 Security Considerations

| Concern | Risk Level | Mitigation |
|---------|------------|------------|
| Arbitrary code execution | High | BSKG only analyzes, never executes |
| File system access | Medium | BSKG reads only, writes to .vrs/ |
| API key exposure | High | Use `{env:VAR}` not plaintext |
| Prompt injection via code | Medium | Sanitize code snippets in findings |

### 6.3 UX Considerations

| Consideration | Recommendation |
|---------------|----------------|
| Tool discovery | Include AGENTS.md in `instructions` |
| Permission prompts | Set common operations to `allow` |
| Error messages | Return structured JSON with suggestions |
| Long operations | Provide progress updates via logs |

### 6.4 Comparison: MCP vs Custom Tools

| Aspect | MCP Server | Custom Tools |
|--------|------------|--------------|
| Setup complexity | Higher (server impl) | Lower (shell wrappers) |
| Type safety | Full JSON Schema | TypeScript schemas |
| State management | Persistent server process | Stateless per-call |
| Debugging | Separate process logs | Inline with OpenCode |
| Portability | Standard protocol | OpenCode-specific |
| **Recommendation** | **Production** | **Quick prototyping** |
