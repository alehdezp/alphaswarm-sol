## Reordering Audit (11 Iterations)

This roadmap was re-audited with multi-agent review in eight rounds.

1. Iteration 1 found a major sequencing gap: benchmarks were planned before full workflow/agent/skill live validation and without deterministic replay/negative controls.
2. Iteration 2 added hard dependencies: marker and proof-token enforcement in 3.1/3.2, all-skills-tested gate in 4, and harness-first testing in 6.
3. Iteration 3 (strict review) rejected `... → 6 → 5 → 7 → 8` as too risky for vanity metrics; final sequence moves 7 before 5 so benchmark publication is gated by enforced hooks/evidence schema.
4. Iteration 4 (3-agent parallel analysis) found Phase 4.1 (workflow testing harness) was placed after agent-dependent features (3.2, 4) despite having zero technical dependency on them. Testing frameworks must come before the features they test. The harness core was extracted into Phase 3.1b and placed immediately after Phase 3.1. Phase 4.1 was reduced to test expansion (audit/debate scenarios needing 3.2/4).
5. Iteration 5 (4-agent parallel investigation + plan review) corrected Phase 3.1b architecture. Phase 3.1b expanded from 4 plans to 8: added 100 test projects corpus, skill invocation tests (30), agent behavioral tests (21), orchestrator flow test, regression baseline. Phase 3.1 corrected: dead code estimate from ~16K LOC to ~3-6K (125+ imports exist, import graph analysis now mandatory first step). Phase 3.2 corrected: re-verify break points before assuming all 12 broken, align markers with existing lowercase metadata keys, leverage existing replay.py (633 LOC).
6. Iteration 6: Phase 3.1c inserted between 3.1b and 3.2 — Reasoning-Based Evaluation Framework. Phase 3.1b builds harness infrastructure (transcript parser, assertions, hooks); 3.1c adds intelligent evaluation ON TOP of that harness. Key innovation: smart dynamic selection per workflow + safe prompt sandboxing + regression detection + blocking debrief protocol. 10 plans, ~43 new files, ~4500 LOC. Zero dependency on 3.2 or 4. Total plans increased from 61 to 71.
7. Iteration 7 (philosophy-first redesign) restructured Phase 3.1b from 8 plans to 7 (infrastructure-only) and Phase 3.1c from 10 plans to 12 (absorbs all skill/agent/orchestrator tests + regression baseline from 3.1b). Key insight: 3.1c IS the testing framework, not an add-on. Skill/agent/orchestrator tests belong in 3.1c where they are evaluated with the full pipeline (capability + reasoning + debrief), not in 3.1b where they would only do mechanical liveness checks.
