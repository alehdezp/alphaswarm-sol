<task type="auto">
  <name>Task 1: Run full test suite and categorize all failures</name>
  <files>.vrs/debug/phase-3.1.1/failure-triage.md</files>
  <action>
1. Run `uv run pytest tests/ --tb=line -q 2>&1 | tee /tmp/test-output.txt` to capture all failures with one-line tracebacks.

2. Parse failures and categorize each into one of 5 categories:

   | Category | Criteria | Action |
   |----------|----------|--------|
   | STALE_CODE | References removed infrastructure, deleted files, renamed APIs | Delete test or simple fix |
   | MISSING_PATTERN_ID | References deprecated `ext-001..ext-072` numbering | Delete test — dead code |
   | EXTERNAL_DEP | Requires external tool (Slither, Aderyn, Mythril) not installed | Add `@pytest.mark.skipUnless(...)` |
   | PATTERN_GAP | Pattern exists but doesn't detect the expected vulnerability | Add `@pytest.mark.xfail(reason="Known pattern quality gap: [id]")` |
   | OTHER | Doesn't fit above categories | Triage individually — fix if quick, xfail with specific reason |

3. Priority order for fixes:
   - STALE_CODE: Delete dead tests or apply simple fix (first priority)
   - MISSING_PATTERN_ID: Delete deprecated references (second)
   - EXTERNAL_DEP: Add skip markers (third)
   - PATTERN_GAP: Add xfail markers with tracking reason (fourth)
   - OTHER: Case-by-case (last)

4. Create triage report at `.vrs/debug/phase-3.1.1/failure-triage.md` with:
   - Summary counts per category
   - Per-file breakdown: file name, test count, category, action taken
   - Remaining unexplained failures (should be zero)

5. Apply markers to test files:
   - `@pytest.mark.xfail(reason="Known pattern quality gap: [pattern-id]")` for PATTERN_GAP
   - `@pytest.mark.skipUnless(shutil.which("slither"), "Slither not installed")` for EXTERNAL_DEP
   - Delete or fix for STALE_CODE and MISSING_PATTERN_ID

**Time-box:** Maximum 2 execution sessions. If not complete, document what's left and create a follow-up note.

**Stopping criterion:** Phase is complete when every failure has a category label AND the triage report exists. Fixing all failures is NOT required — categorization IS.

**Do NOT:**
- Attempt to fix PATTERN_GAP failures (those are pattern quality issues for 3.1c)
- Convert unittest.TestCase to pytest classes
- Run the full suite multiple times unnecessarily
