## Reviewer Corrections (Post-Execution)

Applied 2026-03-02 after reviewer alignment review against 3.1c.2-04-EXPECTED.md.
Commit: `6aad9238`

### Correction 1: Baseline manager must not overwrite integrity rejection

**Severity:** must_have (criterion 1 — "When effective_verdict == 'FAIL', code sets baseline_update_status = 'rejected'")

**Issue:** The baseline manager AND gate at the end of `run()` did not check `result.metadata.get("tainted")`. A session that integrity-rejected (tainted=True, baseline_update_status="rejected") could have its status overwritten to "updated" by `self._baseline_manager.update_baseline()` if session_valid and effective_score were both satisfied — which they would be since integrity runs after scoring.

**Fix:** Added `and not result.metadata.get("tainted")` as the fourth condition in the baseline manager gate in `tests/workflow_harness/lib/evaluation_runner.py`.

**Test added:** `test_integrity_rejection_not_overwritten_by_baseline_manager` — uses a FakeBaselineManager to verify `update_baseline()` is never called for tainted results, and that `baseline_update_status` remains "rejected".

### Correction 2: test_integrity_metadata_recorded did not test PASS behavior

**Severity:** must_have (alignment criterion — PASS verdict -> no rejection)

**Issue:** The original `test_integrity_metadata_recorded` could never produce a PASS verdict because `_check_worktree_isolation()` always fires critical in test environments (no agent git worktrees). The test verified metadata presence but could not assert absence of rejection.json, absence of "tainted" flag, or baseline_update_status != "rejected".

**Fix:** Renamed to `test_integrity_pass_no_rejection`. Monkeypatches `validate_batch` to return a clean `IntegrityReport(verdict="PASS", violations=[])`. Asserts:
- `rejection.json` does NOT exist in obs dir
- `result.metadata["integrity_check"]["verdict"] == "PASS"`
- `"tainted"` not in `result.metadata`
- `result.baseline_update_status != "rejected"`

### Correction 3: test_debrief_artifact_format had silent-pass risk

**Severity:** must_have (test correctness)

**Issue:** All format assertions in `test_debrief_artifact_format` were inside `if debrief_path.exists():`. If debrief.json was never written, the test passed silently without asserting anything.

**Fix:** Replaced the conditional with `assert debrief_path.exists(), "debrief.json must exist for format validation"` before the artifact content checks. The test now fails loudly if debrief persistence is broken.
