# Plan 04: Post-Pipeline Integration — Integrity + Debrief + Enrichment

<objective>
Wire the integrity check pipeline (agent_execution_validator with 13 checks from Plan 02) and debrief persistence (Stage 9) into evaluation_runner.py as post-pipeline stages. Add score tainting, DEGRADED escalation, stage timing telemetry, session correlation IDs, failure_mode classification on IntegrityViolation, and observation enrichment.

Purpose: D-5 mandates integrity check auto-runs after every Agent Teams evaluation session. EVAL-02/EVAL-10 require debrief capture. Currently the validator exists as standalone checks but is NOT wired into the pipeline, and debrief artifacts are not persisted. Without this wiring, integrity failures are silent (polluting baselines) and debrief data is lost after the session. The review recommendations add zero-risk improvements (timing, tainting, correlation IDs) that prevent downstream phases (3.1c.3, 3.1f) from needing to retrofit these signals.

Output: Integrity check + debrief wired into evaluation_runner.py, auto-reject on FAIL, DEGRADED escalation, score tainting, rejection.json + debrief.json artifacts, stage timing, session correlation ID, failure_mode classification, observation enrichment hook, integration tests.
</objective>

    ## 1.5: Wire integrity check after store_result (Stage 8.5)

    After the Stage 8 store_result block (after `stages_completed.append("store_result")`), add the integrity check. This is NOT embedded inside a stage — it runs as a post-pipeline validation gate.

    ```python
    # Stage 8.5: Integrity validation (D-5: auto-run after every session)
    t0 = time.monotonic()
    if effective_obs_dir and effective_obs_dir.exists():
        try:
            from alphaswarm_sol.testing.evaluation.agent_execution_validator import (
                validate_batch,
            )
            integrity_report = validate_batch(
                observation_dir=effective_obs_dir,
                batch_id=session_id or batch_id or "unknown",
                ground_truth_dir=self._ground_truth_dir,
            )

            # Record integrity result in metadata
            warning_count = sum(
                1 for v in integrity_report.violations
                if v.severity == "warning"
            )
            critical_count = sum(
                1 for v in integrity_report.violations
                if v.severity == "critical"
            )
            result.metadata["integrity_check"] = {
                "verdict": integrity_report.verdict,
                "total_files": integrity_report.total_files,
                "files_checked": integrity_report.files_checked,
                "critical_violations": critical_count,
                "warning_violations": warning_count,
                "summary": integrity_report.summary,
            }

            # DEGRADED escalation: 3+ warnings → FAIL (prevents dead zone)
            effective_verdict = integrity_report.verdict
            if effective_verdict == "DEGRADED" and warning_count >= 3:
                effective_verdict = "FAIL"
                result.metadata["integrity_check"]["escalated"] = True
                result.metadata["integrity_check"]["escalation_reason"] = (
                    f"DEGRADED with {warning_count} warnings (>= 3) escalated to FAIL"
                )
                logger.warning(
                    "Integrity DEGRADED escalated to FAIL: %d warnings for session %s",
                    warning_count, session_id,
                )

            # Auto-REJECT on FAIL verdict (original or escalated)
            if effective_verdict == "FAIL":
                result.baseline_update_status = "rejected"

                # Score tainting: mark result as polluted
                result.metadata["tainted"] = True
                result.metadata["taint_reason"] = (
                    f"integrity_check_failed: {integrity_report.summary}"
                )

                # Write rejection artifact
                rejection_path = effective_obs_dir / "rejection.json"
                rejection_path.write_text(
                    integrity_report.model_dump_json(indent=2)
                )
                result.metadata["integrity_rejection"] = {
                    "report_path": str(rejection_path),
                    "critical_count": critical_count,
                }
                logger.error(
                    "Integrity check FAILED for session %s: %s",
                    session_id, integrity_report.summary,
                )

            stages_completed.append("integrity_check")
            health.parsed_records = len(stages_completed)
            health.stages_completed = stages_completed

        except Exception as e:
            logger.warning("Integrity check failed to run: %s", e)
            health.errors += 1

    stage_durations["integrity_check"] = time.monotonic() - t0
    ```

    **Design constraints:**
    - Integrity runs AFTER store_result so results are persisted even if integrity fails (forensic analysis)
    - Auto-reject sets `baseline_update_status = "rejected"` unconditionally on FAIL — overrides any earlier value
    - Integrity check MUST NOT prevent Stage 9 (debrief) from running — debrief data is valuable even from FAIL sessions
    - DEGRADED escalation threshold is 3+ warnings (configurable via constant if desired)
    - Score tainting writes `tainted=True` and `taint_reason` into `result.metadata` — downstream consumers (regression detection, tier management in 3.1f) use these to exclude polluted data
    - `"integrity_check"` appended to stages_completed ONLY if the check ran successfully

    ## 1.7: Add _extract_intelligence_signals() helper

    Add a module-level helper function that extracts 3.1c.3-targeted intelligence signals from debrief answers:
    ```python
    def _extract_intelligence_signals(debrief: DebriefResponse) -> dict[str, Any]:
        """Extract intelligence-targeted signals from debrief for 3.1c.3 modules.

        Looks for answers to the 3 intelligence questions:
        1. Most informative graph query
        2. Unhelpful query + what to change
        3. Information needed but unavailable from graph

        These feed coverage_radar and query_coach in 3.1c.3.
        """
        signals: dict[str, Any] = {}
        questions = debrief.questions_asked
        answers = debrief.answers

        # Map known intelligence questions to signal keys
        INTELLIGENCE_KEYWORDS = {
            "most informative": "most_informative_query",
            "unhelpful results": "unhelpful_query_feedback",
            "could not get from the graph": "missing_information",
        }

        for i, question in enumerate(questions):
            if i >= len(answers):
                break
            q_lower = question.lower()
            for keyword, signal_key in INTELLIGENCE_KEYWORDS.items():
                if keyword in q_lower:
                    signals[signal_key] = answers[i]
                    break

        return signals
    ```

    **NOTE:** This function extracts from whatever questions were asked in Stage 6. The 3 intelligence-targeted questions ("Which graph query was most informative?", "Which query returned unhelpful results?", "What information did you need that you could not get from the graph?") must be added to the debrief prompt template in `debrief_protocol.py`. This is a one-line addition to the DEBRIEF_QUESTIONS list — do it as part of this task. Do NOT redesign the protocol.
