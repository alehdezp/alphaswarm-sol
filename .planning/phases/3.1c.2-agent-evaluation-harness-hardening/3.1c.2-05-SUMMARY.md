## Deviations from Plan

### 1. [Rule 3 - Blocking] Env-var propagation to Agent Teams teammates
- **Found during:** Task 1 preflight check 8
- **Issue:** `_GUARD_PROFILE=strict` set in main session does NOT propagate to spawned Agent teammates (separate Claude Code processes)
- **Fix:** Two-layer enforcement: (1) prompt-based tool restrictions in agent prompt, (2) post-hoc validate_batch() on observation files
- **Impact:** delegate_guard.py runtime hook cannot enforce in Agent Teams mode. Prompt-based restrictions are the primary enforcement layer.

### 2. [Rule 3 - Blocking] evaluation-auditor agent type not recognized mid-session
- **Found during:** Previous session attempt
- **Issue:** Custom agent type created at `.claude/agents/evaluation-auditor.md` but agent registry loads at session startup
- **Fix:** Used `evaluation-auditor` agent type (recognized in new session). Prompt includes strict tool restrictions.

### 3. [Rule 1 - Bug] Validator knowledge_leakage false positives
- **Found during:** Task 3 integrity check
- **Issue:** LEAKED_TERMS list includes pattern IDs that are output by the CLI tool
- **Fix:** Documented in failure-analysis.md. Recommended fix: update LEAKED_TERMS to exclude CLI output terms. Deferred to 3.1c.3.
