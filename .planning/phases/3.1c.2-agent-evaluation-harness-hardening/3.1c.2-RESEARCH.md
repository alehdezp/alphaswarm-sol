## Summary

Phase 3.1c.2 is primarily an infrastructure hardening phase, not new feature development. All the core mechanisms already exist in the codebase — `delegate_guard.py` blocks tools, `TranscriptParser` extracts JSONL events, `agent_execution_validator.py` checks integrity, `evaluation_runner.py` orchestrates the pipeline, and `debrief_protocol.py` handles agent debriefs. The work is: (1) configure and wire these mechanisms correctly for evaluation sessions, and (2) extend them with CLIAttemptState tracking and auto-reject behavior.

The primary risk is env-var propagation: `delegate_guard.py` currently uses a hardcoded config search path (`Path(__file__).parent / config_name`). To support per-session eval configs via `DELEGATE_GUARD_CONFIG` env var, the `_load_config()` function must be modified to check this env var FIRST before falling back to its current search paths. Verified from source: the function at `tests/workflow_harness/hooks/delegate_guard.py:19` does not currently read any env vars — this is the primary code change for Plan 01.

The JSONL transcript format is well-understood: official docs confirm `agent_transcript_path` in SubagentStop input (verified at `claude_code_hooks_docs.md:873`). The `TranscriptParser` in `tests/workflow_harness/lib/transcript_parser.py` already parses this format and extracts Bash tool calls with `get_bash_commands()`. CLIAttemptState (Plan 02) is a natural extension of the existing `has_bskg_query()` and `get_bskg_queries()` methods.

**Primary recommendation:** Do NOT redesign any existing modules. Extend `_load_config()` in delegate_guard.py, extend `agent_execution_validator.py` with check 13, add Stage 9 to `evaluation_runner.py`, and create the eval-specific YAML config. The existing architecture is sound — the gaps are narrow and targeted.

## Architecture Patterns

### Pattern 1: delegate_guard.py Env-Var Extension

**What:** Add env-var config path override to `_load_config()` so evaluation sessions can use a different config without symlinks.

**Current behavior** (verified from source at `delegate_guard.py:19-42`):
```python
def _load_config() -> dict:
    config_name = "delegate_guard_config.yaml"
    search_paths = [
        Path(__file__).parent / config_name,
        Path(os.getcwd()) / ".claude" / "hooks" / config_name,
        Path(os.getcwd()) / ".vrs" / "config" / config_name,
    ]
    for config_path in search_paths:
        if config_path.is_file():
            return _parse_simple_yaml(config_path)
    return {"blocked_tools": [], "blocked_patterns": [], "allowed_reads": []}
```

**Required change:** Add env-var check BEFORE the search paths:
```python
def _load_config() -> dict:
    # Eval mode: env-var config path override (ALPHASWARM_EVAL_MODE + DELEGATE_GUARD_CONFIG)
    if os.environ.get("ALPHASWARM_EVAL_MODE") == "1":
        config_override = os.environ.get("DELEGATE_GUARD_CONFIG", "")
        if config_override and Path(config_override).is_file():
            return _parse_simple_yaml(Path(config_override))
        elif config_override:
            # Config path set but file missing = fail-fast (exit 2)
            print(f"EVAL MODE: DELEGATE_GUARD_CONFIG not found: {config_override}", file=sys.stderr)
            sys.exit(2)
        # else: ALPHASWARM_EVAL_MODE=1 but no config path — use default search
    # ... existing search paths ...
```

**Why env-var gating works:** Each hook invocation is a fresh Python process. The hook reads the env var fresh every time. The `evaluation_runner.py` preflight must set these env vars in the subprocess environment before spawning Agent Teams.

**Env-var propagation to child processes:** Claude Code propagates env vars to subagent processes when `isolation: "worktree"` is used. Verified from CONTEXT.md Assumption 3 (GAP-03, confirmed): "Hooks propagate to Agent Teams child processes via project-level config sharing (v2.1.63+). Per-session scoping uses env-var gating."

**Confidence:** HIGH — verified from source code + CONTEXT.md confirmed resolution.

input_str = json.dumps(tool_input)

            if allowed in input_str:

### Pattern 3: CLIAttemptState Extraction from JSONL Transcripts

**What:** Use `TranscriptParser` to determine whether an agent tried CLI and succeeded/failed, or never tried.

**JSONL record structure** (verified from official docs `claude_code_hooks_docs.md:652-680` + `transcript_parser.py:186-252`):
- Record type `"assistant"`: contains `content` list with `type: "tool_use"` blocks
- Record type `"user"`: contains `content` list with `type: "tool_result"` blocks
- Tool call matched to result via `tool_use_id` / `tool_result.tool_use_id`

**SubagentStop hook provides `agent_transcript_path`** (verified from official docs `claude_code_hooks_docs.md:856-875`):
```json
{
  "session_id": "abc123",
  "transcript_path": "~/.claude/projects/.../abc123.jsonl",
  "agent_transcript_path": "~/.claude/projects/.../abc123/subagents/agent-def456.jsonl"
}
```

**Transcript accessibility for worktree-isolated agents:** Confirmed — `agent_transcript_path` is at user level (`~/.claude/projects/`), not inside the worktree. The `obs_agent_stop.py` hook already captures this field (verified from `tests/workflow_harness/hooks/obs_agent_stop.py:26-34`).

**CLIAttemptState enum logic** (implements D-CONTEXT Plan 02 design):
```python
from enum import Enum

class CLIAttemptState(Enum):
    ATTEMPTED_SUCCESS = "attempted_success"   # alphaswarm CLI called, non-empty result
    ATTEMPTED_FAILED  = "attempted_failed"    # alphaswarm CLI called, error result
    NOT_ATTEMPTED     = "not_attempted"       # No alphaswarm CLI calls in transcript
    TRANSCRIPT_UNAVAILABLE = "transcript_unavailable"  # JSONL not found

def compute_cli_attempt_state(transcript_path: Path | None) -> CLIAttemptState:
    if transcript_path is None or not transcript_path.exists():
        return CLIAttemptState.TRANSCRIPT_UNAVAILABLE
    parser = TranscriptParser(transcript_path)
    bskg_queries = parser.get_bskg_queries()
    if not bskg_queries:
        return CLIAttemptState.NOT_ATTEMPTED
    # Check if any query produced non-empty results
    successful = [q for q in bskg_queries if q.result_snippet and len(q.result_snippet.strip()) > 10]
    if successful:
        return CLIAttemptState.ATTEMPTED_SUCCESS
    return CLIAttemptState.ATTEMPTED_FAILED
```

**Integration point:** `agent_execution_validator.py`'s `validate_observation_file()` at line 107 iterates through checks. Check 13 is added to this function. The `observation_file` contains `agent_id` — use this to locate the corresponding transcript from the SubagentStop observation JSONL.

**Confidence:** HIGH — verified from transcript_parser.py source + official SubagentStop docs.

## Common Pitfalls

### Pitfall 1: Partial pattern matching blocks legitimate CLI calls

**What goes wrong:** `blocked_patterns: ["python"]` also blocks `Bash(command: "uv run alphaswarm ...")` if the command string contains "python" somewhere (e.g., in a path like `/usr/bin/python3`).

**Why it happens:** `delegate_guard.py` uses substring matching on `json.dumps(tool_input)`. If the alphaswarm CLI itself prints "python" in its output or path, it would trigger blocking.

**How to avoid:** Test each pattern against real alphaswarm CLI commands. The `allowed_reads` mechanism provides a whitelist escape hatch — but it checks the same `input_str`, so `allowed_reads: ["alphaswarm"]` would bypass the block for any command containing "alphaswarm". Use this pattern:
```yaml
allowed_reads:
  - "alphaswarm"   # Allow any Bash command containing "alphaswarm"
  - "contracts/"   # Allow Read of contract files
```

**Warning signs:** Canary test fails with exit 2 for a `Bash(uv run alphaswarm build-kg ...)` call.

**Confidence:** HIGH — verified from delegate_guard.py pattern matching logic.

### Pitfall 2: Transcript path lookup failure for isolated teammates

**What goes wrong:** The `agent_transcript_path` from SubagentStop hook points to a path inside `~/.claude/projects/...`. If the session binary is stale (auto-update), the path may not exist.

**Why it happens:** Claude Code auto-updates can delete old binary versions, causing Agent Teams to fail at spawn. If teammates never start, no transcript is written.

**How to avoid:** Preflight check before Plan 06 session spawn (from MEMORY.md):
```bash
SESSION_BIN=$(which claude) && $SESSION_BIN --version || echo "STALE SESSION — restart required"
```
Also: `CLIAttemptState.TRANSCRIPT_UNAVAILABLE` is the graceful degradation path — not a critical violation, just a warning.

**Confidence:** HIGH — documented in MEMORY.md from prior failure.

### Pitfall 3: Stage 9 debrief runs even when session is invalid

**What goes wrong:** If Stage 1 (session validity check) returns `valid=False`, the runner returns early with `status="invalid_session"`. Stage 9 must not run in this case.

**Why it happens:** Stage 9 is added after `store_result`. If the early-return path from Stage 1 doesn't have a parallel early-return for Stage 9, the code after the try/except block may still execute.

**How to avoid:** Gate Stage 9 on `status == "completed"`:
```python
if status == "completed" and self._run_mode == RunMode.INTERACTIVE:
    # Run Stage 9 debrief
    ...
```

**Confidence:** HIGH — verified from evaluation_runner.py early-return logic at lines 540-569.

### Pitfall 4: delegate_guard.py config not loaded by evaluation session hooks

**What goes wrong:** `ALPHASWARM_EVAL_MODE=1` is set in the orchestrator process, but hooks fire in a different subprocess context. If env vars are not propagated to hook subprocesses, the eval config is never loaded.

**Why it happens:** Claude Code spawns hook scripts as subprocesses via the `command:` field in settings.json. The subprocess inherits the parent's env. But `evaluation_runner.py` sets env vars using `os.environ.setdefault()` — this only affects the current process.

**How to avoid:** The evaluation_runner.py must launch the evaluation Agent Team session with env vars set at the Claude Code CLI invocation level, not just in Python. This means the runner must use `subprocess.run(["claude", "--agent", ...], env={**os.environ, "ALPHASWARM_EVAL_MODE": "1", "DELEGATE_GUARD_CONFIG": "/path/to/eval.yaml"})`.

**WARNING:** This is a key integration challenge. The evaluation_runner.py currently does NOT spawn Claude Code sessions — it's a post-session analysis pipeline. The ORCHESTRATOR (the Claude Code session that spawns Agent Teams) must set the env vars. Plan 01 must document this clearly: env vars must be set in the orchestrator's SessionStart hook or in the TeamCreate call.

**Confidence:** MEDIUM — env-var propagation in CC confirmed by CONTEXT.md, but the exact mechanism for orchestrator → teammate propagation needs to be tested in Plan 01 canary.

### Pitfall 5: Ground truth stats comparison uses wrong node count field

**What goes wrong:** `agent_execution_validator.py`'s `_check_graph_stats_against_ground_truth()` at line 648 expects `actual_stats.get("nodes", 0)` from the ground truth JSON. But the proposed `stats.json` format uses `node_count`, not `nodes`.

**Why it happens:** The existing validator was written before the ground truth format was decided.

**How to avoid:** Either (a) use `nodes` and `edges` as the field names in `stats.json` to match the validator's expectation, or (b) update the validator to use `node_count`. Recommendation: use `nodes`/`edges` to match the observation format already used in `cal-*.json` (`data.get("graph_stats", {})` with `nodes` and `edges` keys).

**Confidence:** HIGH — verified from both agent_execution_validator.py and cal-01-attacker.json format.

### Recommended Validation Strategy

Plan 01 must include a canary test before any Plan 06 evaluation session. The canary is a deterministic unit test: set `ALPHASWARM_EVAL_MODE=1` and `DELEGATE_GUARD_CONFIG=/path/to/eval.yaml`, call `delegate_guard.main()` with a synthetic `{"tool_name": "Bash", "tool_input": {"command": "python -c 'import os'"}}`, and assert exit code 2. This provides immediate, run-anywhere verification that does not require spawning actual agents.

For Plans 02-04, unit tests are sufficient: provide synthetic JSONL transcripts and observation files, verify the CLIAttemptState enum and auto-reject behavior. The 4 calibration contracts provide known node/edge counts (cal-01 = 12 nodes / 17 edges) for ground truth comparison.

Plan 05 (debrief Stage 9) requires integration testing against `RunMode.INTERACTIVE` — mocked debrief artifacts can verify the wiring without live agents. Only Plan 06 (Plan 12 retry) requires live Agent Teams.

**Key invariants to check:**
- CLIAttemptState.NOT_ATTEMPTED → severity "critical" violation (not warning)
- CLIAttemptState.ATTEMPTED_FAILED → severity "warning" (CLI tried but failed — FM-1 territory, not FM-3)
- CLIAttemptState.TRANSCRIPT_UNAVAILABLE → severity "warning" with graceful degradation
- VERDICT: FAIL → `baseline_update_status = "rejected"` + rejection.json artifact

### Edge Cases Requiring Validation

- **Pattern false positives:** `uv run alphaswarm build-kg` should NOT be blocked by any pattern in eval config. Add explicit test: verify exit code 0 for alphaswarm commands.
- **Empty transcript:** `TranscriptParser` with a 0-byte JSONL file should return `CLIAttemptState.TRANSCRIPT_UNAVAILABLE`, not crash.
- **Mixed BSKG results:** Some query calls succeed (non-empty result), some fail (empty). `CLIAttemptState` should be `ATTEMPTED_SUCCESS` if at least one succeeds.
- **Stage 9 with SIMULATED mode:** Debrief must be skipped (not fail) when `run_mode == SIMULATED`. Verify `debrief_agent_name is None` gate.
