# Plan 01: Evaluation delegate_guard Config + Env-Var Gating

<objective>
Create the evaluation-specific delegate_guard config and add env-var gating to delegate_guard.py so evaluation sessions enforce tool restrictions at runtime, not just via prompts.

Purpose: Plan 12 Batch 1 proved that prompt-based tool restrictions fail. Agents bypassed restrictions by running `Bash(python -c "import ...")` and reading project files directly. This plan creates the enforcement primitive: a PreToolUse hook that blocks at the tool-call level with exit code 2.

Critical fix: The original design had `allowed_reads: ["alphaswarm"]` which let `Bash(python -c "import alphaswarm")` through because "alphaswarm" is a substring match on the entire tool_input JSON. This revision separates `allowed_file_reads` (path-based exceptions for Read/Grep) from `allowed_bash_commands` (prefix-based exceptions for Bash), closing the bypass.

Output: `delegate_guard_config_eval.yaml` with blocked_tools/blocked_patterns/allowed_file_reads/allowed_bash_commands, env-var extension in `delegate_guard.py` with blocked-call logging, 9 canary tests proving enforcement works (including the critical bypass regression test).
</objective>

    ```python
    def _load_config() -> dict:
        # Eval mode: env-var config path override (D-6 from CONTEXT.md)
        if os.environ.get("_GUARD_PROFILE") == "strict":
            config_override = os.environ.get("DELEGATE_GUARD_CONFIG", "")
            if config_override:
                override_path = Path(config_override)

                        return _parse_simple_yaml(override_path)
                    except (OSError, ValueError):
                        pass
                # Config path set but file missing or unreadable = fail-fast

                    f"EVAL MODE: DELEGATE_GUARD_CONFIG not found or unreadable: {config_override}",
                    file=sys.stderr,
                )
                sys.exit(2)
        # ... existing search paths unchanged ...
    ```

    Key behaviors:
    - `_GUARD_PROFILE=strict` + valid `DELEGATE_GUARD_CONFIG` path -> use that config
    - `_GUARD_PROFILE=strict` + invalid/missing `DELEGATE_GUARD_CONFIG` path -> exit 2 (fail-fast, blocks ALL tool calls)
    - `_GUARD_PROFILE` not set -> existing behavior (search paths)
    - Do NOT change anything in the existing `main()` function EXCEPT the allow-list logic (see Part A2)

    **Part A2: Modify `main()` to distinguish file reads from bash commands**

    The current `main()` uses a single `allowed_reads` list for the escape hatch. This must be split to handle two separate exception lists from the config:

    ```python
    # In main(), after loading config:
    allowed_file_reads = config.get("allowed_file_reads", config.get("allowed_reads", []))
    allowed_bash_commands = config.get("allowed_bash_commands", [])

    # For Bash tool: check allowed_bash_commands (prefix match on the command string)
    if tool_name == "Bash":
        command = tool_input.get("command", "")
        for pattern in blocked_patterns:
            if pattern in input_str:
                for allowed_cmd in allowed_bash_commands:
                    if command.startswith(allowed_cmd):
                        _log_allowed(tool_name, pattern, input_str)
                        sys.exit(0)
                _log_blocked(tool_name, pattern, input_str)
                sys.exit(2)

    # For Read/Grep: check allowed_file_reads (substring match on file path)
    if tool_name in ("Read", "Grep"):
        file_path = tool_input.get("file_path", tool_input.get("path", ""))
        for pattern in blocked_patterns:
            if pattern in input_str:
                for allowed_path in allowed_file_reads:
                    if allowed_path in file_path:
                        _log_allowed(tool_name, pattern, input_str)
                        sys.exit(0)
                _log_blocked(tool_name, pattern, input_str)
                sys.exit(2)
    ```

    CRITICAL: This fixes the Gap 2.7 bypass where `allowed_reads: ["alphaswarm"]` would let `Bash(python -c "import alphaswarm")` through because `"alphaswarm"` appeared as a substring in the tool_input JSON. With the new design:
    - `allowed_bash_commands: ["uv run alphaswarm"]` uses PREFIX match on the command string
    - `python -c "import alphaswarm"` does NOT start with `"uv run alphaswarm"` -> BLOCKED
    - `uv run alphaswarm build-kg contracts/X.sol` DOES start with `"uv run alphaswarm"` -> ALLOWED

    Backward compatibility: When the config contains `allowed_reads` (old format) but not `allowed_file_reads`, fall back to `allowed_reads` for file-based tools. This preserves dev-config behavior.

    **Part A3: Add blocked-call logging**

    Add a `_log_blocked()` helper function that appends a JSON record to the file at `DELEGATE_GUARD_LOG` env var path (if set):

    ```python
    def _log_blocked(tool_name: str, matched_pattern: str, input_str: str) -> None:
        log_path = os.environ.get("DELEGATE_GUARD_LOG")
        if not log_path:
            return
        import datetime
        record = {
            "timestamp": datetime.datetime.utcnow().isoformat() + "Z",
            "tool_name": tool_name,
            "blocked_pattern": matched_pattern,
            "tool_input_preview": input_str[:200],
        }
        try:
            with open(log_path, "a") as f:
                f.write(json.dumps(record) + "\n")
        except OSError:
            pass  # Best-effort logging, do not block the guard decision
    ```

    Also add a corresponding `_log_allowed()` for audit trail of exception-path hits (same format but with `"action": "allowed"`).

    CRITICAL DESIGN NOTES:

    - **Gap 2.7 fix:** `allowed_file_reads` and `allowed_bash_commands` are SEPARATE lists.
      `allowed_file_reads` applies ONLY to Read/Grep tool path parameters.
      `allowed_bash_commands` applies ONLY to Bash tool command parameter via PREFIX match.
      The old `allowed_reads: ["alphaswarm"]` substring match that let `python -c "import alphaswarm"` through is ELIMINATED.

    - **`"python3"` has no trailing space (Gap fix):** This catches `python3`, `python3.12`, `python3.11`, etc.
      The pattern `"python "` (with space) catches `python -c ...` and `python script.py`.
      Together they cover all Python interpreter invocations.

    - **Glob, Grep, Edit blocked (Gaps 4.2-4.4):** These tools are added to `blocked_tools`.
      Read and Grep have a path parameter, so `allowed_file_reads: ["contracts/", "tests/contracts/"]`
      provides an exception path. Glob and Edit have NO exception path — they are fully blocked.

    - **git, base64, eval, exec, perl, ruby, node blocked (Gaps 2.1-2.3, 5.1):**
      These are added to `blocked_patterns` to prevent shell escape vectors and alternative interpreters.

    - **Env var discovery blocked:** `"EVAL_MODE"`, `"GUARD_PROFILE"`, `"ALPHASWARM_EVAL"` are
      added to `blocked_patterns` so agents cannot discover the guard env vars via `env | grep` or `printenv`.

    - **`_GUARD_PROFILE` env var (Gap 1.1):** Renamed from `ALPHASWARM_EVAL_MODE`.
      The underscore prefix makes it less discoverable via `env` listing (underscore-prefixed vars
      are conventionally internal). Value is `"strict"` instead of `"1"` for self-documentation.
