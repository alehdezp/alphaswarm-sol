## Why This Phase Exists

Even after 3.1c.1 fixes CLI path resolution, agents can still:

1. **Import Python modules directly** instead of using CLI — the secure-reviewer admitted: "Direct KnowledgeGraph inspection via Python was required for precise evidence"
2. **Read project context** (CLAUDE.md, .planning/, docs/) despite prompt-based "don't read" instructions — prompt restrictions are suggestions, not constraints
3. **Fabricate findings** — agents wrote JSON with fake node IDs, build hashes, and evidence references that looked plausible but were never produced by real tool execution
4. **Mark tasks "completed" without execution** — no verification that tools were actually run before task status was updated

The evaluation harness needs ENFORCEMENT mechanisms, not just instructions. The enforcement primitive is `delegate_guard.py`'s PreToolUse hook, which blocks at the tool-call level (exit code 2 = tool rejected). Prompt-based restrictions are advisory supplements, not enforcement.

## Assumptions

1. **Tool access in evaluation sessions is enforced via delegate_guard.py PreToolUse hook.** The hook blocks by tool name (`blocked_tools`) AND by Bash command patterns (`blocked_patterns`). `subagent_type` determines system prompt only — it does not restrict tools. Evidence: Plan 12 Batch 1 agents bypassed prompt-based restrictions via `Bash(python -c "import ...")`.
   - 1a. **Bash calls matching `blocked_patterns` are blocked at the hook layer, not the tool layer.** Patterns like `python`, `import`, `cat` must be in `blocked_patterns` — the current config `[".sol"]` only matches Solidity file paths and provides zero protection against Python imports. Evidence: GAP-01 confirmed `.sol` pattern does not match `python -c "import os"`.
2. **JSONL session transcripts are accessible for Agent Teams teammates.** Transcripts stored at `~/.claude/projects/{encoded-path}/{uuid}.jsonl` (user-level, always readable by orchestrator). The `SubagentStop` hook provides a direct `agent_transcript_path` field (available since v2.0.42). Worktree isolation does not prevent access. Evidence: GAP-02 confirmed. Primary method: capture path via SubagentStop hook. Fallback: filesystem scan with timestamp filtering.
3. **Claude Code hooks can capture tool_use events for post-session analysis.** Hooks are the SECONDARY verification method. JSONL transcripts are primary.
4. **The `agent_execution_validator.py` module (12 checks) is the foundation.** Plan 02 extends it with check 13 (CLIAttemptState), not replaces it.
5. **Ground truth graph builds can be done centrally before agent sessions.** Two-phase evaluation: orchestrator builds graph, agents reason over it.
6. **delegate_guard.py reads config at INVOCATION TIME (per PreToolUse call, no caching).** Each hook invocation is a fresh process. Hooks propagate to Agent Teams child processes via project-level config sharing (v2.1.63+). Per-session scoping uses env-var gating (`_GUARD_PROFILE=1`) + env-var config path override (`DELEGATE_GUARD_CONFIG=/path/to/eval.yaml`). No symlinks needed. Evidence: GAP-03 confirmed via source code analysis.
</assumptions>
