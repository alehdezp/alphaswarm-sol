## Decisions Made
- **Error detection heuristic in _is_error_result():** Short results (<50 chars) check all error indicators; longer results check only the first line. This prevents false positives where legitimate data contains error-like substrings in later sections.
- **Lazy imports for check 13:** `_check_cli_attempt_state()` and `_get_cli_state_messages()` use function-level imports to avoid circular import issues between validator and cli_attempt_state modules.
- **TranscriptParser as single source of truth (D-2):** Both `cli_attempt_state.py` and `reasoning_timeline.py` import from `tests.workflow_harness.lib.transcript_parser`. No duplicate JSONL parsing logic.
- **Conclusion reclassification:** The last text event in extract_reasoning_timeline() is always reclassified as "conclusion" to ensure every transcript has a conclusion event, even if the heuristic initially classified it differently.
- **_resolve_transcript_paths via JSONL scanning:** Scans all *.jsonl files in the observation directory for `agent_stop` event records, extracting `agent_transcript_path` and mapping by `agent_id`. This is the bridge between the hook system and the validator pipeline.
