# Plan 03: Ground Truth Graph Generation + Expected Findings

<objective>
Generate and persist ground truth graph stats for all 4 calibration contracts AND hand-author expected vulnerability findings for each, so that:

1. The agent_execution_validator can compare agent-reported stats against actual build output (stats.json with `nodes` and `edges` fields).
2. The stats.json includes semantic metadata (`expected_vulnerability_class`, `semantic_fingerprint`, `function_names[]`, `has_external_calls`, `has_state_modification_after_call`) enabling richer evaluation beyond just "did agent use CLI."
3. Per-contract `expected_findings.json` files provide ground truth for WHAT agents should find (not just whether they used the right tools), enabling 3.1c.3 intelligence modules (tier_manager auto-promote, coverage_radar gap tracking, semantic accuracy evaluation).

Purpose: D-4 (from CONTEXT.md) mandates pre-generated ground truth before any agent session. Without ground truth, `_check_graph_stats_against_ground_truth()` in the validator has nothing to compare against and silently skips the check. Without expected findings, the evaluation can only measure process compliance ("did agent use CLI?") but not outcome accuracy ("did agent find the right bug?").

Output: `stats.json` files (with semantic fields) for 4 calibration contracts + `expected_findings.json` for 4 calibration contracts + a repeatable generation script.
</objective>

```python
def _check_graph_stats_against_ground_truth(
    files: list[Path],
    ground_truth_dir: Path,
) -> list[IntegrityViolation]:
    # Expects ground_truth_dir / "{contract_id}-graph-stats.json"
    # with keys: "nodes", "edges"
    for f in files:
        data = json.loads(f.read_text())
        contract_id = data.get("contract_id", "unknown")
        gt_path = ground_truth_dir / f"{contract_id}-graph-stats.json"
        actual_stats = json.loads(gt_path.read_text())
        actual_nodes = actual_stats.get("nodes", 0)
```
