## Summary

Phase 3.1 is primarily a codebase cleanup and foundation-laying phase. The technical domain spans four areas: (1) Python AST-based import graph analysis to identify dead code in a 30,892 LOC testing module, (2) barrel export (`__init__.py`) rewriting to reduce 456 lines / 165 re-exported symbols down to ~8 CLI-consumed symbols, (3) JSON Schema definition for evaluation contracts using the already-available `jsonschema` 4.25.1 library, and (4) testing rules consolidation from ~25 rules to ~15 categories with stable machine-readable IDs.

The codebase is in a confirmed state where only `src/alphaswarm_sol/cli/main.py` imports from the testing module outside of testing itself. The 8 CLI-consumed symbols come from exactly 3 import sites (lines 820, 886, 968). Everything else in `testing/__init__.py` is barrel re-export bloat. The shipping manifest targets are confirmed: 30 skills, 21 agents.

**Primary recommendation:** Use Python `ast` module for import graph analysis (NOT vulture or pyflakes, which lack the __init__.py barrel-exclusion capability needed here). The import graph script should be a standalone `.py` file saved as an artifact, not ad-hoc commands -- it must be reproducible for the verification gate.

## Architecture Patterns

### Import Graph Analysis Script Structure
```python
# save as: scripts/analyze_testing_imports.py
import ast
import json
from pathlib import Path

def build_import_graph(src_root: Path, exclude_barrel: Path) -> dict:
    """
    For every .py file in src_root:
    1. Parse with ast
    2. Find all ImportFrom nodes
    3. If the file IS exclude_barrel, skip (don't count barrel re-exports as callers)
    4. Record: {symbol: [list of files that import it]}
    """
    # Step 1: Enumerate all barrel-exported symbols (to know what to track)
    barrel_symbols = extract_barrel_symbols(exclude_barrel)

    # Step 2: Walk all .py files, find who imports each symbol
    graph = {sym: [] for sym in barrel_symbols}
    for py_file in src_root.rglob("*.py"):
        if py_file == exclude_barrel:
            continue  # CRITICAL: skip the barrel itself
        imports = extract_imports(py_file)
        for module, name in imports:
            key = f"{module}.{name}"
            if key in graph:
                graph[key].append(str(py_file))

    return graph

def categorize(graph: dict) -> dict:
    """Categorize each symbol as TRULY_DEAD, TEST_ONLY, PRODUCTION_DEPENDENT, TRANSITIVE"""
    categories = {}
    for symbol, callers in graph.items():
        if not callers:
            categories[symbol] = "TRULY_DEAD"
        elif all("tests/" in c for c in callers):
            categories[symbol] = "TEST_ONLY"
        elif any("cli/" in c for c in callers):
            categories[symbol] = "PRODUCTION_DEPENDENT"
        else:
            categories[symbol] = "TRANSITIVE"
    return categories
```

**Key design choice:** The script must output JSON to `.vrs/debug/phase-3.1/import-graph.json` as a machine-readable artifact. This artifact is the gate proof -- deletion decisions trace back to it.

```python
def assert_matches_contract(result: dict, contract: dict) -> None:
    """Validate a workflow result against an evaluation contract."""
    for check in contract["capability_checks"]:
        check_id = check["id"]
        assert check_id in result.get("checks", {}), (
            f"Missing capability check '{check_id}' in result"
        )
        actual = result["checks"][check_id]
        assert actual.get("passed") is not None, (
            f"Check '{check_id}' has no pass/fail status"
        )

def assert_reasoning_dimensions_covered(
    transcript: str, contract: dict
) -> None:
    """Assert transcript covers the reasoning dimensions in the contract."""
    dims = contract.get("reasoning_dimensions", [])
    for dim in dims:
        assert dim.lower() in transcript.lower(), (
            f"Reasoning dimension '{dim}' not found in transcript"
        )
```

### Anti-Patterns to Avoid
- **Naive grep for imports:** `grep "from alphaswarm_sol.testing"` will miss dynamic imports and wrongly include comments. Use `ast.parse()` for accuracy.
- **Deleting before analyzing:** The import graph artifact MUST exist on disk before any `git rm`. This is a gate requirement in the context.md.
- **Bulk deletion of tests/:** Test files should be individually justified against the import graph, not batch-deleted by directory.
- **Rewriting __init__.py to empty:** After deletion, __init__.py must still export the 8 CLI-consumed symbols. An empty file breaks the CLI.

## Common Pitfalls

### Pitfall 1: False Liveness from __init__.py Barrel Exports
**What goes wrong:** Every symbol appears to have a caller (the `__init__.py` file), making nothing look dead.
**Why it happens:** The `__init__.py` re-exports 165 symbols from 24 modules. Any import analysis that counts `__init__.py` as a caller will report zero dead code.
**How to avoid:** The import graph script MUST exclude `__init__.py` as a caller source. It should only track imports FROM other files (cli/main.py, tests/*.py, other src/ modules).
**Warning signs:** If the import graph shows every symbol has >= 1 caller, the barrel exclusion is not working.

### Pitfall 2: Transitive Dependencies Creating Cascade Failures
**What goes wrong:** Deleting module A that is imported by module B (also dead) doesn't cause test failures, but deleting module B without deleting module A's import of B will cause an ImportError.
**Why it happens:** Dead code can form chains: A imports B imports C. All three are dead, but deletion order matters if __init__.py references any of them.
**How to avoid:** Delete ALL dead code in one atomic commit, then rewrite __init__.py in the same commit. Don't leave partial state.
**Warning signs:** `python -c "import alphaswarm_sol.testing"` raising ImportError after partial deletion.

### Pitfall 3: harness/ Directory False Positive
**What goes wrong:** The harness/ directory (1,789 LOC) is classified as "needs verification" because `ClaudeCodeRunner` MIGHT have external callers.
**Why it happens:** Without the import graph, it's unclear if anything outside testing/ and tests/ imports from harness/.
**How to avoid:** The import graph will resolve this definitively. Current evidence shows harness/ is only imported by dead directories (e2e/, unit/) and itself.
**Warning signs:** If the import graph shows a non-testing, non-test caller for harness, it should be kept.

**Research finding (HIGH confidence):** I verified via grep that harness/ is only imported by: itself (internal), e2e/ (dead), and unit/ (dead). No imports from cli/, no imports from non-testing src/. The import graph will confirm this, but harness/ is very likely TRULY_DEAD.

### Pitfall 4: Test File Deletion Breaking Test Discovery
**What goes wrong:** Pytest collection fails after deleting test files because conftest.py or other fixtures reference deleted modules.
**Why it happens:** Test files may be imported by conftest.py fixtures or by other test files for shared utilities.
**How to avoid:** Run `pytest tests/ --collect-only` after deletion to verify all remaining tests can be collected. Current baseline: 11,363 tests collected, 1 error.
**Warning signs:** Collection error count increasing from the baseline of 1.

### Pitfall 5: Rules Rewrite Dropping Anti-Fabrication Substance
**What goes wrong:** During consolidation from ~25 to ~15 rule categories, the specific anti-fabrication checks (nonce verification, transcript hashing, duration bounds) get lost in the abstraction.
**Why it happens:** Merging rules for readability can accidentally abstract away concrete enforcement mechanisms.
**How to avoid:** Each merged category must preserve the specific checks from its constituent rules. Word count of merged category should be >= 60% of sum of originals (per context.md drift detection).
**Warning signs:** Merged "Transcript Authenticity" category no longer mentions nonce, hash, or specific enforcement commands.

```python
# Verified working with project's ast module
import ast
from pathlib import Path

def extract_imports(py_file: Path) -> list[tuple[str, str]]:
    """Extract (module, symbol) pairs from a Python file."""
    try:
        with open(py_file) as f:
            tree = ast.parse(f.read())
    except SyntaxError:
        return []

    imports = []
    for node in ast.walk(tree):
        if isinstance(node, ast.ImportFrom) and node.module:
            for alias in node.names:
                imports.append((node.module, alias.name))
    return imports

# Verified: __init__.py has 165 symbol re-exports from 24 modules
# Verified: Only cli/main.py imports from testing outside testing itself
```
