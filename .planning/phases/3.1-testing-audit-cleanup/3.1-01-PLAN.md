Create `scripts/analyze_testing_imports.py` — a custom AST-based import graph analyzer. This script MUST:

1. Parse all Python files in `src/alphaswarm_sol/` and `tests/` using the `ast` module
2. For each file, extract all `import` and `from ... import` statements
3. Build a symbol-level caller map: for each symbol exported by any `testing/` module, list all files that import it
4. **CRITICAL: Exclude `src/alphaswarm_sol/testing/__init__.py` barrel re-exports from caller counts.** The __init__.py imports ~150 symbols solely for re-export — these are NOT real callers. The script must detect barrel-import patterns (imports that exist only to re-export) and exclude them.
5. Categorize each file in `testing/` into one of:
   - `TRULY_DEAD`: Zero callers anywhere (excluding __init__.py re-exports)
   - `TEST_ONLY`: Called only by files in `tests/`
   - `PRODUCTION_DEPENDENT`: Called by `cli/main.py` or other non-testing `src/` modules
   - `TRANSITIVE`: Only called by other testing modules (analyze callers recursively — if all transitive callers are TRULY_DEAD, this file is also TRULY_DEAD)
6. Output results to `.vrs/debug/phase-3.1/import-graph.json` with structure:
   ```json
   {
     "generated_at": "ISO timestamp",
     "files": {
       "src/alphaswarm_sol/testing/master_orchestrator.py": {
         "category": "TRULY_DEAD",
         "symbols_exported": ["MasterOrchestrator"],
         "real_callers": [],
         "barrel_callers_excluded": ["testing/__init__.py"]
       }
     },
     "summary": {
       "TRULY_DEAD": 30,
       "TEST_ONLY": 5,
       "PRODUCTION_DEPENDENT": 12,
       "TRANSITIVE": 3
     }
   }
   ```

Run the script: `python scripts/analyze_testing_imports.py`
