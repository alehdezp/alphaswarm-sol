# Expected Results: 3.1-02 Legacy Infrastructure Cleanup

---

## Philosophy

This plan exists to eliminate all remaining legacy infrastructure references (tmux, jujutsu-workspace, deleted runners) from production code, configs, registries, and skill/agent markdown files. It serves the phase goal by ensuring downstream plans (3.1-03 shipping manifest, 3.1-04 rules rewrite) start from a clean baseline with zero legacy noise. This plan explicitly does NOT delete dead source code from `testing/` (that is 3.1-01), does NOT rewrite testing rules (3.1-04), and does NOT touch test files in `tests/` (3.1-06). Its scope is strictly: sweep for legacy references, clean configs/registries, update workflow exports, archive originals.

## Implementation Reasoning

### Decision 1: Comprehensive grep sweep before any modifications

- **Selected:** Run grep sweeps for all legacy terms (tmux, jujutsu, jj-workspace, deleted runner names) across production code, configs, and skill/agent .md files before making any changes
- **Why:** Exploration indicated these should be zero in production, but "should be" is not "is." The sweep establishes the factual baseline and discovers any surprise references.
- **Anti-patterns:** Do NOT assume exploration findings are still accurate (files may have changed). Do NOT skip scanning `.claude/skills/` and `.claude/agents/` directories.

### Decision 2: Archive-then-modify workflow

- **Selected:** Copy original configs to `.planning/archive/testing-rules/legacy/` before any modification
- **Why:** Archives provide rollback capability and let reviewers diff before/after. If a modification is wrong, the original is recoverable without git archaeology.
- **Anti-patterns:** Do NOT modify configs without archiving first. Do NOT archive to a location that gets cleaned up by other plans.

### Decision 3: Verify exports match filesystem reality

- **Selected:** After cleaning `workflow/__init__.py`, verify that every exported symbol corresponds to a file that exists on disk
- **Why:** 3.1-01 deletes `self_improving_runner.py` and other workflow modules. If `workflow/__init__.py` still tries to import them, it breaks at runtime.
- **Anti-patterns:** Do NOT leave exports for modules that were deleted in 3.1-01. Do NOT add exports for modules that do not exist yet.

## Acceptable Deviations

### OK (preserves intent)
- Legacy references found in `.planning/` or `docs/` directories (those are documentation, not production code -- note but do not necessarily clean)
- Additional legacy terms discovered beyond tmux/jujutsu (e.g., old runner class names in comments)
- More skill/agent .md files need cleanup than initially expected
- Archive directory contains more files than just configs (e.g., deleted .md files)

### DRIFT (misalignment -- needs correction)
- Legacy references remain in `src/alphaswarm_sol/` .py or .yaml files after cleanup
- Registry entries for deleted skills not removed
- `workflow/__init__.py` still exports deleted modules
- Configs modified without being archived first

### STOP (fundamental failure -- needs replanning)
- Registry cleanup breaks skill loading (`uv run alphaswarm tools status` fails)
- Config cleanup breaks YAML parsing
- More than 10 production files contain legacy references (indicates systematic problem, not stragglers)
- `pytest tests/` discovers > 5 new failures after changes
