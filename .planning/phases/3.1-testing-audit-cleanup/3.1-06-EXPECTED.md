# Expected Results: 3.1-06 Test File Cleanup

---

## Philosophy

This plan exists to delete test files whose sole imports reference modules deleted in 3.1-01 and 3.1-02, and to verify the test suite remains intact for living modules. It serves the phase goal by removing dead test weight that causes import errors and false coverage metrics. This plan explicitly does NOT delete source code from `src/` (3.1-01 did that), does NOT clean configs or registries (3.1-02 did that), and does NOT create new tests (3.1c does that). Its scope is strictly: identify dead test files via import analysis, delete them, and verify the remaining test suite is healthy.

## Implementation Reasoning

### Decision 1: Import-based analysis to determine test file liveness

- **Selected:** Parse each test file's imports, check if ALL `alphaswarm_sol.testing.*` imports reference modules deleted in 3.1-01/02. Classify as DEAD_TEST, MIXED_TEST, or LIVE_TEST.
- **Why:** A test file is dead only if ALL of its testing imports target deleted modules. A file with even one living import may contain valuable test functions for that living module. Import analysis (not filename matching) is the reliable method.
- **Anti-patterns:** Do NOT delete test files based on filename pattern alone. Do NOT delete a MIXED_TEST file without first editing out the dead imports and keeping the live tests.

### Decision 2: Baseline/delta verification for test count accuracy

- **Selected:** Capture test count before deletion, capture after deletion, verify delta equals the exact number of test functions in deleted files
- **Why:** Silent test drops (where unrelated tests disappear) are a serious regression risk. Delta verification catches this. If the delta exceeds the count of test functions in deleted files, something unrelated was lost.
- **Anti-patterns:** Do NOT skip baseline capture. Do NOT accept "tests pass" as sufficient (tests can pass while losing coverage). Do NOT ignore delta mismatches.

### Decision 3: conftest.py as a special case

- **Selected:** Check conftest.py files (directory-level and root-level) for imports of deleted modules. Edit conftest if it imports dead modules (remove dead imports/fixtures) but never delete conftest.py.
- **Why:** conftest.py provides shared fixtures for multiple test files. Deleting conftest breaks all tests in its directory. Editing is the safe action.
- **Anti-patterns:** Do NOT delete any conftest.py. Do NOT leave conftest importing deleted modules (causes ImportError for all tests in that directory).

## Acceptable Deviations

### OK (preserves intent)
- Fewer test files deleted than the 6 candidates listed in the plan (some may have been already removed or may have living imports)
- Additional test files identified as dead beyond the listed candidates
- MIXED_TEST files edited rather than deleted (preserving live test functions)
- Test count delta is 0 (if all listed test files were already removed in prior plans or already had import errors being suppressed)
- conftest.py files edited to remove dead imports (correct behavior for shared fixtures)

### DRIFT (misalignment -- needs correction)
- Test files deleted that import living modules (false deletion)
- Delta mismatch: more tests lost than expected from deleted files
- No baseline count captured (cannot verify delta)
- Living-module test files (test_skills_package.py, evaluator tests) deleted

### STOP (fundamental failure -- needs replanning)
- `pytest tests/ --collect-only` shows import errors in remaining tests
- Full test suite fails with > 5 new errors after deletion
- Tests for WorkflowEvaluator or TrajectoryEvaluator deleted (kept runners lost their tests)
- conftest.py deleted (breaks all tests in its directory)
