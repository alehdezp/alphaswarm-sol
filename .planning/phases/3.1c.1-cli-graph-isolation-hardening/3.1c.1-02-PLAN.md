# Plan 02: Per-contract Graph Storage

**Goal:** Eliminate graph cross-contamination between concurrent agent builds by storing each contract's graph in a hash-based subdirectory using a canonical contract identity, with atomic writes, skip-if-exists caching, staleness detection, and backward compatibility.

**Trigger:** Plan 3.1c-12 Batch 1 calibration FAILED because sequential `build-kg` calls overwrote each other's graphs in a single flat file. Agent A's graph disappeared when Agent B built.

**Context decisions:** D-2a (contract identity specification), D-2b (per-contract graph storage), D-2c (read-path inventory), D-meta (canonical meta.json schema), D-4 (backward compatibility). All DECIDED and locked in CONTEXT.md.

**Note:** This plan uses the read-path inventory from Plan 01 Task 1 for graph callsites. Plan 01 Task 1 must complete before Task 3 of this plan begins (the inventory is shared). However, Tasks 1-2 of this plan can execute in parallel with Plan 01.

## Tasks

<task id="1">
  <name>Task 1: Implement Contract Identity Utility</name>
  <objective>Create `src/alphaswarm_sol/kg/identity.py` with the canonical `contract_identity()` function per specification D-2a.</objective>
  <what>
    Implement a single shared utility used by ALL components (build-kg, query, --graph resolution):

    **`contract_identity(paths: list[Path]) -> str`:**
    - Compute `os.path.realpath()` for each path (follows symlinks, canonicalizes)
    - Sort the realpath strings
    - Strip trailing slashes
    - Join with newline separator
    - SHA-256 hash, truncate to 12 hex chars
    - Single-file contracts degenerate to per-file hashing

    **`_resolve_project_root(inputs: list[Path]) -> tuple[Path, str]`:**
    - Run `git rev-parse --show-toplevel` from `inputs[0].parent`
    - On `CalledProcessError`: log warning to stderr, return `(os.getcwd(), "fallback_cwd")`
    - On success: return `(toplevel_path, "git_toplevel")`

    **`filter_dependency_paths(all_paths: list[Path], project_root: Path) -> list[Path]`:**
    - Primary filter: use Slither's `source_mapping.is_dependency` flag when available
    - Fallback for raw solc: keep only paths that start with `project_root` prefix
    - WARNING for each excluded dependency path
  </what>
  <done_criteria>
    - `contract_identity()` exists in `src/alphaswarm_sol/kg/identity.py`
    - Function is deterministic: same input paths always produce same hash regardless of order
    - `os.path.realpath()` used (not `os.path.abspath()`)
    - Trailing slashes stripped before hashing
    - UTF-8 encoding used
    - `_resolve_project_root()` handles non-git directories gracefully
    - `filter_dependency_paths()` excludes dependency paths using `is_dependency` flag
  </done_criteria>
  <verify>
    Unit tests:
    - `test_single_file_identity`: known file -> known hash (deterministic)
    - `test_multi_file_identity_order_independent`: paths in different order -> same hash
    - `test_trailing_slash_stripped`: path with/without trailing slash -> same hash
    - `test_symlink_resolved`: symlink and target -> same hash (via `os.path.realpath()`)
    - `test_non_git_fallback`: run from temp dir outside git -> returns fallback_cwd, no error
  </verify>
</task>

<task id="2">
  <name>Task 2: Implement Hash-Based Graph Storage with Atomic Writes</name>
  <objective>Refactor `GraphStore` to write graphs into hash-based subdirectories with atomic write semantics, skip-if-exists caching, meta.json sidecar per D-meta schema, and backward-compatible read fallback.</objective>
  <what>
    Modify `src/alphaswarm_sol/kg/store.py`:

    **`GraphStore.save(graph, *, identity: str, meta: dict, format="toon", force=False) -> Path`:**
    - Compute target directory: `self.root / identity`
    - Create directory: `hash_dir.mkdir(parents=True, exist_ok=True)`
    - Skip-if-exists: if `target.exists() and not force`, log INFO and return existing path
    - Atomic write: write to `graph.toon.tmp`, then `os.replace()` to `graph.toon`
    - Write meta.json atomically: write to `meta.json.tmp`, then `os.replace()`
    - meta.json follows D-meta schema: `schema_version`, `built_at`, `graph_hash`, `contract_paths`, `stem`, `source_contract`, `identity`, `slither_version`, `project_root_type`

    **`GraphStore.load(*, identity: str = None) -> KnowledgeGraph`:**
    - If `identity` provided: load from `self.root / identity / "graph.toon"`
    - If `.tmp` exists without `.toon`: raise `CorruptGraphError`
    - Backward compatibility: if hash-based subdirectory doesn't exist, try flat `self.root / "graph.toon"` with deprecation WARNING

    **`--force` flag:** Propagated from CLI `build-kg --force` through to `GraphStore.save(force=True)`

    **`--check-fresh` flag on build-kg:**
    - Compute SHA256 of current contract files (same algorithm as graph_hash in meta.json)
    - Compare to stored `meta.json.graph_hash`
    - Emit to stderr: `[graph-cache] skipped: <stem> (hash match)` if fresh
    - Emit to stderr: `[WARNING] cache mismatch: <stem> has changed on disk` if stale
    - Exit code 0 on fresh, 42 on stale

    **Pydantic model for meta.json:** Create `GraphMetadata` model validating all D-meta fields. All fields mandatory, string lengths bounded, no extra fields.
  </what>
  <done_criteria>
    - `GraphStore.save()` writes to `{root}/{identity}/graph.toon` via atomic tmp+rename
    - `GraphStore.save()` creates `meta.json` sidecar per D-meta schema
    - Skip-if-exists works: second save with same identity returns without overwriting
    - `--force` flag overwrites existing graph
    - `GraphStore.load()` raises `CorruptGraphError` on partial write detection
    - Backward-compatible fallback to flat `graph.toon` with deprecation warning
    - `--check-fresh` exits 0 on fresh, 42 on stale
    - `GraphMetadata` Pydantic model validates all D-meta fields
  </done_criteria>
  <verify>
    Unit tests:
    - `test_save_creates_hash_subdir`: verify directory structure `.vrs/graphs/{identity}/graph.toon`
    - `test_save_atomic_write`: verify no `graph.toon.tmp` remains after successful save
    - `test_save_skip_if_exists`: second save returns early, file mtime unchanged
    - `test_save_force_overwrites`: second save with force=True overwrites
    - `test_load_corrupt_detection`: create `.tmp` without `.toon`, verify CorruptGraphError
    - `test_load_backward_compat`: flat `graph.toon` at root, verify loads with deprecation warning
    - `test_meta_json_schema`: verify all D-meta fields present and types correct
    - `test_check_fresh_exit_codes`: fresh -> 0, stale -> 42
  </verify>
</task>

## must_haves

Derived from phase goal using goal-backward methodology:

1. **Per-contract graph isolation**: Each contract's graph stored in its own `{identity}` subdirectory. Two different contracts produce two different directories. Sequential builds never overwrite.
2. **Canonical contract identity**: `contract_identity()` is deterministic, order-independent, symlink-resolving. All callers use the same shared utility (no divergent implementations).
3. **Atomic writes**: `GraphStore.save()` uses tmp+rename. No corrupt partial files left on crash.
4. **meta.json sidecar per D-meta schema**: Every graph directory has a valid meta.json with all mandatory fields. This is the cross-plan interface contract consumed by Plan 03.
5. **Backward compatibility**: Pre-upgrade flat `graph.toon` files still load with deprecation warning. No operator workflow breaks.
6. **Skip-if-exists + --force**: Default idempotent; explicit --force to rebuild.
7. **--check-fresh staleness detection**: Exit 42 on stale for programmatic detection.
8. **Zero hardcoded graph paths**: All callsites migrated to GraphStore API with identity.
9. **Read-path inventory graph callsites all MIGRATED**: Complete audit trail.
