# Expected Results: Plan 01 — Absolute vulndocs Resolution

**Plan:** 3.1c.1-01
**Phase:** 3.1c.1-cli-graph-isolation-hardening
**Generated:** 2026-03-01

---

## Fundamental Deliverable

This plan delivers a centralized vulndocs resolution module (`resolution.py`) that replaces all cwd-relative `Path("vulndocs")` constructions with package-aware resolution functions, so that agents running in Jujutsu workspaces, non-project directories, and wheel installs can locate vulnerability pattern data without depending on their working directory. Without it, Plan 04 (CLI smoke tests) cannot validate CLI fixes because agents will continue to get `PatternDirectoryNotFoundError` from any non-project cwd — reproducing the exact Plan 3.1c-12 Batch 1 failure. Success means `vulndocs_read_path()` returns a valid `Traversable` from ANY cwd, all 15+ callsites are migrated to use it, and no consumer bypasses the `Traversable` API contract. Failure means bare `Path("vulndocs")` callsites remain in production code, or the resolution module exists but consumers cast its output to `Path` — defeating the purpose of layout-agnostic resolution and leaving the same class of failures latent for wheel and zip-backed installs.

## Drift Detection

### OK: Resolution uses Path.resolve() instead of os.path.realpath()
**Scenario:** The executor implements `vulndocs_write_path()` using `Path(__file__).resolve().parent` instead of `Path(os.path.realpath(__file__)).parent`. Both produce the same result for symlink resolution.
**Why OK:** `Path.resolve()` and `os.path.realpath()` are functionally equivalent for symlink resolution in modern Python. The CONTEXT.md D-1 specifies `Path(__file__).resolve().parent` as the pattern. This is an acceptable implementation choice that does not affect correctness.
**Action:** Verify the symlink-resolving behavior is present (either mechanism), then accept.

### OK: Task 2 finding leads to _vulndocs_data instead of vulndocs
**Scenario:** The namespace collision mini-POC (Task 2) discovers that `importlib.resources.files("alphaswarm_sol").joinpath("vulndocs")` collides with the Python subpackage. The executor uses `_vulndocs_data` as the bundled data path instead, and `vulndocs_read_path()` calls `.joinpath("_vulndocs_data")`.
**Why OK:** The plan explicitly defines this as a valid branch in Task 2's decision gate. The core contract (Traversable returned, works from any cwd) is preserved regardless of the internal path name.
**Action:** Verify the Task 2 research document exists with collision evidence, and that pyproject.toml and resolution.py both use the alternate path consistently.

### DRIFT: Resolution module exists but pattern loader bypasses it
**Scenario:** `resolution.py` is implemented with the correct dual-mode functions, but `queries/patterns.py` retains `_DEFAULT_VULNDOCS = Path("vulndocs")` and only calls `vulndocs_read_path()` when an explicit argument is passed. The default case still uses the cwd-relative path.
**Why DRIFT:** The module exists but the primary consumer that triggered the entire phase failure still has the cwd-relative default. Agents using the default code path will fail identically to Plan 12 Batch 1. The resolution module is dead code for the most critical path.
**Action:** Request correction. Verify `patterns.py` no longer defines its own default vulndocs path and imports from resolution.py unconditionally.

### DRIFT: Tests exist but Traversable mock test is missing
**Scenario:** `tests/test_vulndocs_resolution.py` has 10+ tests covering read/write path resolution, env var override, and dual-root detection. But the `test_traversable_mock_rejects_fspath` test is absent — there is no verification that consumers avoid casting to `Path`.
**Why DRIFT:** The Traversable API contract (must_haves.truths[2]) is one of the plan's core invariants. Without the mock test, there is no verification that the migration actually enforces the contract. A consumer could import `vulndocs_read_path()` and immediately wrap it in `Path()`, which would work in development (editable install) but fail in wheel/zip installs.
**Action:** Request the missing test. The `__fspath__` mock test is explicitly listed in Task 5 and is a CONTEXT.md D-1 done criterion.

### STOP: Resolution module uses cwd-relative paths internally
**Scenario:** `resolution.py` defines `vulndocs_read_path()` but internally constructs the path as `Path("vulndocs").resolve()` or `Path.cwd() / "vulndocs"` — still depending on the current working directory rather than using `importlib.resources.files()` or `__file__`-relative resolution.
**Why STOP:** This reproduces the exact bug the plan exists to fix. The module gives the illusion of centralized resolution while embedding the same cwd dependency. Every agent in a non-project directory will still fail. The plan's fundamental purpose — cwd-independent resolution — is not delivered.
**Action:** Escalate. The core mechanism is wrong, not just incomplete. All downstream plans (02, 03, 04) depend on this being genuinely cwd-independent.

## Reviewer Guidance

**Priorities:**

1. **Verify zero bare `Path("vulndocs")` in production code.** This is the single most mechanically verifiable criterion and the direct fix for the Plan 12 Batch 1 root cause. Run `grep -rn 'Path("vulndocs")' src/` and `grep -rn "Path('vulndocs')" src/` — zero matches outside `resolution.py` is the hard pass/fail gate.

2. **Confirm the Traversable return type and mock test.** The dual-mode architecture (D-1) is only meaningful if `vulndocs_read_path()` returns `Traversable` and consumers honor it. Check the function signature, the `test_traversable_mock_rejects_fspath` test, and at least 2-3 migrated consumers to verify they use `.iterdir()`/`.open()`/`.joinpath()` rather than Path-casting.

3. **Validate the read-path inventory completeness.** The inventory is a shared artifact with Plan 02. Verify it contains all RESEARCH.md callsites (15+ vulndocs, 9+ graph), each with the four required columns, and that every Plan 01-owned callsite is marked MIGRATED after Task 6.

4. **Check `pyproject.toml` for vulndocs packaging.** Wheel builds that omit vulndocs data will cause `importlib.resources.files()` to return an empty or missing traversable. Verify the force-include or package-data directive uses recursive `**` glob and references the path determined by Task 2.

5. **Verify Task 2 research exists with a clear finding.** Task 2 is a decision gate that determines the exact `joinpath()` argument used throughout the module. If the research is missing or ambiguous, all downstream implementation choices are ungrounded.
