# Plan 04: CLI Isolation Smoke Tests with Agent Teams

**Goal:** Validate that CLI fixes from Plans 01-03 actually work when REAL agents run in Jujutsu-isolated workspaces. Prove cross-contamination is eliminated. Establish the interactive testing approach for all downstream phases.

**Trigger:** Plans 01-03 fix the code, but code fixes don't prove agent behavior. The Plan 12 Batch 1 failure showed agents falling back to raw Python imports when CLI failed. We must prove agents USE the CLI and GET correct results.

**Scope boundary (from CONTEXT.md):** This plan performs BINARY smoke tests (CLI ran? Results non-empty? Correct graph?). It does NOT perform agent debriefs or structured post-session evaluation -- that is Phase 3.1c.2's responsibility. The interrogation protocol here is a DIAGNOSTIC tool for infrastructure failures, not an evaluation mechanism.

**Context decisions:** Testing Approach (refined) in CONTEXT.md. All specifications locked.

**Hard dependency:** Plans 01, 02, and 03 MUST be fully MERGED before Plan 04 execution begins. This plan validates the combined fix, not individual components.

**Execution constraint:** This plan spawns Agent Teams and MUST execute in a top-level Claude Code session, not via subagents (per TESTING-PHILOSOPHY.md Principle 15 and TESTING-FRAMEWORK.md Principle 14).

## Agent Teams Protocol References

This plan MUST follow:
- **TESTING-FRAMEWORK.md "Agent Team Spawning Protocol"**: Context isolation checklist, focused prompts, explicit tool restrictions
- **TESTING-FRAMEWORK.md "Context Control Checklist"**: Tools specified, project context prohibited, ground truth isolated
- **TESTING-FRAMEWORK.md "F1: Binary Staleness"**: Preflight check before TeamCreate -- verify session binary exists
- **TESTING-FRAMEWORK.md "F2: Context Isolation Violation"**: Explicitly restrict tools and prohibit reading project files
- **TESTING-PHILOSOPHY.md Principle 6**: Real execution only
- **TESTING-PHILOSOPHY.md Principle 15**: Sequential execution for Agent Teams

<task id="2">
  <name>Task 2: Establish Ground-Truth Patterns via Sequential Builds</name>
  <objective>Build graphs for 2 test contracts SEQUENTIALLY and identify discriminating patterns (a pattern present in Contract X but absent in Contract Y, and vice versa) to serve as ground truth for cross-contamination detection.</objective>
  <what>
    Select 2 test contracts from `tests/contracts/` with distinct vulnerability patterns (Claude's discretion -- e.g., a reentrancy contract and an access-control contract).

    **Sequential ground-truth establishment:**
    1. Build graph for Contract X from a Jujutsu workspace: `uv run alphaswarm build-kg tests/contracts/X.sol`
    2. Query for X's unique pattern: `uv run alphaswarm query "<x_only_pattern>" --graph X`
    3. Verify non-empty results for X's pattern
    4. Build graph for Contract Y from a DIFFERENT Jujutsu workspace: `uv run alphaswarm build-kg tests/contracts/Y.sol`
    5. Query for Y's unique pattern: `uv run alphaswarm query "<y_only_pattern>" --graph Y`
    6. Verify non-empty results for Y's pattern
    7. Cross-verify: `query "<x_only_pattern>" --graph Y` returns `matches=0`
    8. Cross-verify: `query "<y_only_pattern>" --graph X` returns `matches=0`

    Store ground-truth patterns in `.vrs/observations/plan04/ground-truth-patterns.json`:
    ```json
    {
      "contract_x": {"name": "...", "path": "...", "unique_pattern": "...", "expected_matches_min": 1},
      "contract_y": {"name": "...", "path": "...", "unique_pattern": "...", "expected_matches_min": 1},
      "cross_verified": true
    }
    ```

    This step MUST complete successfully before any concurrent or Agent Teams testing. Without ground truth, cross-contamination cannot be falsified.
  </what>
  <done_criteria>
    - 2 distinct contracts have graphs built in separate workspaces
    - Each contract has at least 1 discriminating pattern verified as non-empty
    - Cross-verification confirms each pattern is ABSENT in the other contract's graph
    - Ground-truth JSON file exists at `.vrs/observations/plan04/ground-truth-patterns.json`
    - `cross_verified: true` in the JSON
  </done_criteria>
  <verify>
    - `jq '.cross_verified' .vrs/observations/plan04/ground-truth-patterns.json` returns `true`
    - Manual inspection: unique patterns make semantic sense for their respective contracts
  </verify>
</task>

<task id="4">
  <name>Task 4: Cross-Contamination Verification with Ground-Truth Patterns</name>
  <objective>Verify that concurrent agent builds do not cross-contaminate by having each agent query the OTHER contract's discriminating pattern and confirming empty results.</objective>
  <what>
    After Task 3 agents complete their builds:

    **Cross-contamination test:**
    1. Agent-X queries Y's unique pattern against X's graph: `uv run alphaswarm query "{y_only_pattern}" --graph {stem_x}`
    2. Agent-Y queries X's unique pattern against Y's graph: `uv run alphaswarm query "{x_only_pattern}" --graph {stem_y}`
    3. Both queries MUST return `matches=0` in the `# result:` header
    4. If either returns matches > 0: FAIL -- cross-contamination detected

    **Concurrent same-contract race test:**
    1. Launch two `build-kg tests/contracts/X.sol` simultaneously from different workspaces
    2. After both complete, verify `meta.json` in `.vrs/graphs/{identity_x}/` is valid JSON
    3. Verify all D-meta schema fields present and types correct
    4. Verify `graph.toon` file is non-zero bytes and loadable

    Store results in `.vrs/observations/plan04/cross-contamination-results.json`:
    ```json
    {
      "cross_query_x_for_y_pattern": {"matches": 0, "passed": true},
      "cross_query_y_for_x_pattern": {"matches": 0, "passed": true},
      "concurrent_same_contract": {"meta_json_valid": true, "graph_loadable": true},
      "overall_verdict": "PASS"
    }
    ```
  </what>
  <done_criteria>
    - Cross-contamination queries both return `matches=0`
    - Concurrent same-contract race produces valid meta.json and loadable graph
    - Results JSON exists at `.vrs/observations/plan04/cross-contamination-results.json`
    - `overall_verdict` is "PASS"
  </done_criteria>
  <verify>
    - `jq '.overall_verdict' .vrs/observations/plan04/cross-contamination-results.json` returns `"PASS"`
    - `jq '.cross_query_x_for_y_pattern.matches' ...` returns `0`
    - `jq '.cross_query_y_for_x_pattern.matches' ...` returns `0`
  </verify>
</task>

## must_haves

Derived from phase goal using goal-backward methodology:

1. **Agents USE the CLI**: Every agent session has `alphaswarm build-kg` or `alphaswarm query` in bash tool calls. Zero Python imports of `alphaswarm_sol`. (This is the core validation -- Plan 12 agents fell back to Python.)
2. **Agents GET correct results**: `# result:` header shows `graph_nodes > 0`. Queries return pattern-relevant data, not empty or raw nodes.
3. **No cross-contamination**: Discriminating pattern queries return `matches=0` for the wrong contract's graph. Falsifiable by design: ground-truth patterns are pre-verified.
4. **Concurrent builds don't corrupt**: Same-contract race produces valid meta.json and loadable graph.
5. **Framework pre-validation passes**: WorkspaceManager, TeamManager, EvaluationRunner all work independently. Failure diagnosis is unambiguous.
6. **Persistent artifacts**: `.vrs/observations/plan04/` directory exists with all test results. Framework contribution requirement satisfied.

- **GOV-05 (No fake tests):** Task 1 pre-validation tests have no skip/xfail. Agent Teams tests are real execution per TESTING-PHILOSOPHY.md Principle 6.
