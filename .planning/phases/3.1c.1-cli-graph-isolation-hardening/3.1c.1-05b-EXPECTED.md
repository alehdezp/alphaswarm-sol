## Fundamental Deliverable

Without it, Plan 05a's taxonomy and template exist in isolation — no phase's CONTEXT.md references them, no governance document enforces artifact requirements, and the Snapshot Protocol has no downstream adopters. Success means every near-term phase has a Testing Gate section with the correct phase type and requirements, ROADMAP.md mandates testing gates for all future phases, PLAN-PHASE-GOVERNANCE.md enforces artifact paths in done criteria, 3.1c.1 itself is retroactively annotated as CODE+FRAMEWORK (proving the mixed-type rule works), and no premature CONTEXT.md files exist for undefined phases 4+. Failure means the testing mandate remains advisory — a design document nobody is required to follow — and phases 3.1c.2 and 3.1f proceed without testing gates, making Phase 3.1c.1's governance work wasted.
