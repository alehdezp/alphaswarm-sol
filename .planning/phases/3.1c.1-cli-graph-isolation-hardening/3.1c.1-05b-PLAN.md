# Plan 05b: Testing Mandate Rollout

**Goal:** Apply the phase-type taxonomy and testing gates (from Plan 05a) to all near-term phases (3.1c.2, 3.1f, 3.2), update project governance documents (ROADMAP.md, PLAN-PHASE-GOVERNANCE.md), and annotate Phase 3.1c.1 retroactively as `type: CODE+FRAMEWORK`.

**Trigger:** Plan 05a designed the taxonomy and templates. Without rollout, the governance design exists as an abstract document that no phase references. The testing mandate is not enforced.

**Context decisions:** Cross-Phase Testing Mandate (refined) in CONTEXT.md. Near-term phases get full CONTEXT.md updates. Far-future phases (4+) get ROADMAP.md template only. Snapshot Protocol inheritance for downstream phases. All DECIDED and locked.

**Depends on:** Plan 05a (design must complete before rollout begins).

## must_haves

Derived from phase goal using goal-backward methodology:

1. **Near-term phases have testing gates**: 3.1c.2 and 3.1f both have Testing Gate sections in their CONTEXT.md files with phase-type-specific requirements. Without this, the mandate is abstract.
2. **ROADMAP.md mandates testing gates for future phases**: Any new CONTEXT.md must include a Testing Gate section. This is the far-future enforcement mechanism.
3. **PLAN-PHASE-GOVERNANCE.md enforces artifact requirements**: Done criteria must include artifact paths. This integrates the testing mandate into the existing governance contract.
4. **3.1c.1 retroactively annotated as CODE+FRAMEWORK**: First test of the mixed-type rule, proving it works before rolling out to other phases.
5. **Snapshot Protocol inherited by downstream phases**: 3.1c.2 and 3.1f both reference the Snapshot Protocol. Without this, framework-type testing requirements cannot be satisfied.
6. **No premature CONTEXT.md files**: Phases 4+ get ROADMAP.md template only. No speculative context files for undefined phases.
