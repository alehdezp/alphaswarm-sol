## Pre-Execution Concerns (from Planning Documents)

Based on what I can infer from CLAUDE.md context about the project:

### Concern 1: Traversable vs Path Type Safety
The EXPECTED.md likely requires `vulndocs_read_path()` to return `Traversable` (not `Path`).
This is a strict type contract. When implementation happens, watch for:
- Consumers casting `Traversable` back to `Path()` (defeats the purpose)
- `_walk_pattern_files()` using `os.walk()` or `Path.glob()` instead of Traversable iteration
- Test mocks that are too permissive (using real Path objects instead of mock Traversable)

### Concern 2: importlib.resources Behavior with Editable Installs
`importlib.resources.files()` behaves differently for:
- Regular installs (returns zipfile-backed Traversable)
- Editable installs via `uv pip install -e .` (returns filesystem Path)
- Direct `uv run` without install (may not resolve package data at all)

The implementation must handle all three modes.

### Concern 3: Dual-Root Detection at Import Time
If `_assert_single_vulndocs_root()` runs at module import time, it could:
- Slow down imports (filesystem scanning)
- Fail in CI environments where vulndocs might not be installed
- Create circular import issues if resolution.py imports from modules that import it

### Concern 4: ALPHASWARM_VULNDOCS_DIR Override
The environment variable override must:
- Take precedence over BOTH importlib.resources AND __file__-relative paths
- Validate the directory exists AND contains expected structure (not just "is a directory")
- Be documented in a way that users can discover it
