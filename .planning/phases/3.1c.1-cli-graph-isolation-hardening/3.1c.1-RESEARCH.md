## Summary

Phase 3.1c.1 fixes the three root causes that caused Plan 3.1c-12 Batch 1 calibration to fail: (1) vulndocs resolution via cwd-relative paths that break in Jujutsu workspaces, (2) graph cross-contamination from single-file storage overwriting, and (3) silent query degradation to empty results. All three root causes are confirmed by source code analysis. The fixes use well-established Python patterns (importlib.resources, hash-based subdirectories, explicit error-when-ambiguous semantics). The CONTEXT.md is exceptionally thorough (48KB, improved status, 59 merged improvement items, 3/3 gaps resolved) and provides authoritative specifications for every implementation detail.

## User Constraints (from CONTEXT.md)

### Locked Decisions

**D-1: vulndocs Resolution (refined)**
Dual-mode resolution for vulndocs:
- **Read path:** `importlib.resources.files("alphaswarm_sol").joinpath("vulndocs")` for detection/query consumers (layout-agnostic, works across editable and non-editable installs, handles zip-safe packages). Returns a `Traversable` -- consumers MUST NOT cast to `pathlib.Path` directly. Use `Traversable` methods (`.iterdir()`, `.open()`, `.joinpath()`) or `importlib.resources.as_file()` context manager for operations requiring `os.stat()`, `glob()`, or YAML loading from path.
- **Write path:** `Path(__file__).resolve().parent / ...` for editable installs; `ALPHASWARM_VULNDOCS_DIR` env var override for installed packages. Write operations (`/vrs:add-vulnerability`, `/vrs:refine`, `vulndocs validate --fix`) need writable paths.
- **Conflict detection:** `_assert_single_vulndocs_root()` detects when both package-installed and editable vulndocs trees exist with `.yaml` files at different resolved paths. Raised at module import time (not per-call). Suppressible via `ALPHASWARM_SKIP_VULNDOCS_CONFLICT_CHECK=1`.
- `pyproject.toml` must declare vulndocs in `package_data` using `["vulndocs/**"]` recursive glob -- not single-level `*` which silently omits nested subdirectories in some setuptools versions.
- All vulndocs consumers (category traversal, pattern loaders, sub-path constructions) must be audited to ensure zero bare relative `Path()` constructions downstream of the resolved root. Test code must use the same resolution helpers.

**D-1a: ALPHASWARM_VULNDOCS_DIR Contract (new)**
When `ALPHASWARM_VULNDOCS_DIR` is set, it overrides BOTH `vulndocs_read_path()` and `vulndocs_write_path()`. The package-bundled vulndocs are completely bypassed.
- **Validation:** On first call to either path function. If path does not exist: raises `VulndocsConfigError('ALPHASWARM_VULNDOCS_DIR={path} does not exist')`. If path exists but is not a directory: raises `VulndocsConfigError('... is not a directory')`. If directory exists and is empty of YAML files: logs WARNING but does not raise -- empty override is a valid bootstrap state.

**D-2a: Contract Identity Specification (refined)**
A single canonical contract identity specification used by all components:
- **Canonical form:** `os.path.realpath()` of the primary .sol file (or sorted realpaths for multi-file compilations)
- **Encoding:** UTF-8 normalized, trailing slashes stripped
- **Hash algorithm:** SHA-256 of canonical form, truncated to 12 hex chars
- **Multi-file (compilation unit):** Slither treats compilation unit as atomic. Identity = SHA-256(sorted absolute realpaths of CLI-specified `.sol` inputs only). Compiled dependency paths from Slitter's `source_mapping` are excluded using `source_mapping.is_dependency` flag as primary filter, project-root prefix as fallback for raw solc. Single-file contracts degenerate to per-file hashing.
- **Project root resolution:** `git rev-parse --show-toplevel` from `input[0].parent`. On failure: log warning, use `os.getcwd()` as project_root, store `project_root_type` in meta.json.
- **Implementation:** Single shared utility `contract_identity(paths: List[Path]) -> str` in a shared module that all callers import (build-kg, query, --graph resolution). All included paths must be under project root; WARNING for excluded dependency paths.

**D-meta: Canonical meta.json Schema (new)**

## Standard Stack

### Core
| Library | Version | Purpose | Why Standard |
|---------|---------|---------|--------------|
| `importlib.resources` | stdlib (3.9+) | Package-safe vulndocs resolution | The standard Python mechanism for accessing package data from any working directory; returns `Traversable` objects that work even with zip-backed packages |
| `pathlib.Path` | stdlib | File path manipulation | Already used extensively in codebase; `__file__`-relative resolution for editable installs |
| `hashlib` (sha256) | stdlib | Contract identity hashing | Standard, fast, deterministic; 12-char hex truncation gives 48 bits = ~281 trillion unique identifiers |
| `os.replace()` | stdlib | Atomic file rename | POSIX guarantee of atomic rename on same filesystem; the standard pattern for crash-safe writes |
| `os.path.realpath()` | stdlib | Symlink-resolving canonical path | Required to canonicalize across Jujutsu workspace symlinks; `abspath()` only resolves relative-to-cwd |
| `typer` | >=0.12.3 | CLI framework | Already the CLI framework for `alphaswarm`; exit codes and stderr/stdout separation are built-in |
| `pydantic` | >=2.8.2 | meta.json schema validation | Already used for data models throughout the codebase |

### Supporting
| Library | Version | Purpose | When to Use |
|---------|---------|---------|-------------|
| `json` | stdlib | meta.json serialization | For writing/reading the per-graph metadata sidecar |
| `subprocess` | stdlib | `git rev-parse --show-toplevel` | For project root resolution in non-git fallback scenarios |
| `jj` CLI | >=0.38.0 | Jujutsu workspace management | For Plan 04 smoke tests; verified via GAP-01 research |
| `pytest` | >=9.0.2 | Unit/integration testing | For Plan 01-03 unit tests (NOT for Plan 04 Agent Teams tests) |

### Alternatives Considered
| Instead of | Could Use | Tradeoff |
|------------|-----------|----------|
| `importlib.resources.files()` | `pkg_resources` | Deprecated, slower, import-time overhead; `importlib.resources` is the modern replacement |
| SHA-256 truncated 12 chars | Full SHA-256 (64 chars) | Too long for filesystem paths; 12 chars sufficient for project-scale uniqueness |
| `os.replace()` | `shutil.move()` | `shutil.move()` is NOT atomic across filesystems; `os.replace()` guarantees atomicity on same fs |
| `os.path.realpath()` | `Path.resolve()` | `Path.resolve()` is equivalent in modern Python; either works, but `os.path.realpath()` is explicit about symlink resolution intent |

### Pattern 1: Dual-Mode vulndocs Resolution
**What:** Two functions (`vulndocs_read_path()`, `vulndocs_write_path()`) that resolve the vulndocs data directory correctly from any working directory.
**When to use:** Any code that needs to access vulnerability pattern YAML files.
**Why it matters:** The vulndocs YAML data lives at the PROJECT ROOT (`vulndocs/`), NOT inside the Python package (`src/alphaswarm_sol/vulndocs/` is a code module). For `importlib.resources.files()` to find the data, it must be declared in `package_data` and bundled into the wheel. For editable installs, `__file__`-relative resolution works because Jujutsu workspaces create real file copies (GAP-01 confirmed).
**Example:**
```python
# Source: Python stdlib docs + CONTEXT.md D-1
import importlib.resources
from pathlib import Path
from importlib.abc import Traversable

def vulndocs_read_path() -> Traversable:
    """Read-only path to vulndocs data. Works from any cwd."""
    env_override = os.environ.get("ALPHASWARM_VULNDOCS_DIR")
    if env_override:
        p = Path(env_override)
        if not p.exists():
            raise VulndocsConfigError(f"ALPHASWARM_VULNDOCS_DIR={env_override} does not exist")
        if not p.is_dir():
            raise VulndocsConfigError(f"ALPHASWARM_VULNDOCS_DIR={env_override} is not a directory")
        return p
    return importlib.resources.files("alphaswarm_sol").joinpath("vulndocs")

def vulndocs_write_path() -> Path:
    """Writable path to vulndocs data. For editable installs only."""
    env_override = os.environ.get("ALPHASWARM_VULNDOCS_DIR")
    if env_override:
        p = Path(env_override)
        # Same validation as read path
        return p
    return Path(__file__).resolve().parent.parent / "vulndocs"
```

### Pattern 2: Contract Identity Shared Utility
**What:** Single canonical function to compute a deterministic identity hash for any contract or compilation unit.
**When to use:** build-kg (to name the graph subdirectory), query (to look up graph by contract), --graph flag (to resolve stem to directory).
**Example:**
```python
# Source: CONTEXT.md D-2a
import hashlib
import os
from pathlib import Path

def contract_identity(paths: list[Path]) -> str:
    """Compute canonical 12-char hex identity for a contract/compilation unit."""
    canonical = sorted(os.path.realpath(str(p)).rstrip("/") for p in paths)
    joined = "\n".join(canonical)
    return hashlib.sha256(joined.encode("utf-8")).hexdigest()[:12]
```

### Pattern 3: Atomic Graph Write with Skip-if-Exists
**What:** Write graph to temp file, then `os.replace()` to final path. Skip by default if already exists.
**When to use:** `GraphStore.save()` -- every graph write operation.
**Example:**
```python
# Source: CONTEXT.md D-2b
import os

def save(self, graph, *, identity: str, format="toon", force=False):
    hash_dir = self.root / identity
    hash_dir.mkdir(parents=True, exist_ok=True)
    target = hash_dir / f"graph.{format}"

    if target.exists() and not force:
        logger.info("Graph already exists for %s, skipping. Use --force to rebuild.", identity)
        return target

    tmp = hash_dir / f"graph.{format}.tmp"
    tmp.write_text(encoded_content, encoding="utf-8")
    os.replace(str(tmp), str(target))

    # Also write meta.json atomically
    meta_tmp = hash_dir / "meta.json.tmp"
    meta_tmp.write_text(json.dumps(meta), encoding="utf-8")
    os.replace(str(meta_tmp), str(hash_dir / "meta.json"))

    return target
```

**What:** When multiple graphs exist, refuse to guess. Error with actionable guidance.
**When to use:** `alphaswarm query` when `--graph` is not specified and multiple graph subdirectories exist.

### Anti-Patterns to Avoid
- **Bare `Path("vulndocs")` anywhere in production code:** This resolves relative to CWD, breaking in workspaces. Use `vulndocs_read_path()` or `vulndocs_write_path()`.
- **`Path(".vrs/graphs/graph.json")` without explicit base:** Same CWD problem. Use `GraphStore` with project-root-anchored paths.
- **`os.path.abspath()` for identity hashing:** Does not resolve symlinks. Jujutsu workspaces may have symlinked paths. Use `os.path.realpath()`.
- **`target.write_text()` directly (non-atomic):** A kill during write leaves a corrupt file. Use temp file + `os.replace()`.
- **Silent fallback to "nodes" query kind when patterns fail to load:** This is the current bug in `intent.py` L226. Must error, not silently degrade.

## Don't Hand-Roll

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| Package data resolution from any cwd | Custom path walking / `sys.modules` introspection | `importlib.resources.files()` | Stdlib handles editable installs, wheel installs, zip packages; custom solutions miss edge cases |
| Atomic file writes | `try/except` around `write_text()` | `os.replace()` after writing to `.tmp` | `os.replace()` is POSIX-atomic; exception-based approaches still leave partial files |
| Contract identity hashing | Custom serialization per callsite | Shared `contract_identity()` utility | A single canonical function prevents hash divergence between build, query, and --graph commands |
| CLI exit code semantics | Ad-hoc exception handling | Defined exit code enum (0=success, 1=routing error, 2=graph error) | Agents need machine-parseable exit codes to distinguish "query ran but found nothing" from "graph is missing" |

**Key insight:** The root cause of Plan 12's failure was NOT a single bug -- it was 3 independent bugs that compound. Each fix is individually simple (standard Python patterns), but they must be delivered together (Plans 01-03) before testing (Plan 04) can validate the combined fix.

## Common Pitfalls

### Pitfall 1: importlib.resources Returns Traversable, Not Path
**What goes wrong:** Code does `str(vulndocs_read_path() / "reentrancy")` or passes the result to `glob.glob()` -- both fail because `Traversable` is not a `Path`.
**Why it happens:** `importlib.resources.files()` returns a `Traversable` object that may be backed by a zip file, not a filesystem directory. It looks like a Path but isn't.
**How to avoid:** Use only `Traversable` methods: `.iterdir()`, `.open()`, `.joinpath()`. For operations requiring a real filesystem path (e.g., YAML loaders that take path arguments), use `importlib.resources.as_file()` context manager.
**Warning signs:** `TypeError: expected str, bytes or os.PathLike object` when passing vulndocs path to functions expecting `Path`.
**Done criterion from CONTEXT.md:** category traversal and pattern loader code passes a `Traversable` mock that raises `TypeError` on `__fspath__`.

### Pitfall 2: Vulndocs Data Not in the Package
**What goes wrong:** `importlib.resources.files("alphaswarm_sol").joinpath("vulndocs")` returns an empty traversable or raises `FileNotFoundError`.
**Why it happens:** The vulndocs YAML data lives at the PROJECT ROOT (`vulndocs/`), not inside `src/alphaswarm_sol/`. For non-editable installs, `package_data` must be declared in `pyproject.toml` to include them in the wheel. The current `pyproject.toml` does NOT include vulndocs in package_data.
**How to avoid:** Add `vulndocs/**` to `[tool.hatch.build.targets.wheel.force-include]` in `pyproject.toml`. Use `["vulndocs/**"]` recursive glob, not `["vulndocs/*"]`.
**Warning signs:** Tests pass in editable mode but fail in wheel-installed mode. `importlib.resources.files("alphaswarm_sol").joinpath("vulndocs/reentrancy").iterdir()` returns empty.
**Done criterion from CONTEXT.md:** After building a wheel and installing into a clean venv, `importlib.resources.files("alphaswarm_sol").joinpath("vulndocs/reentrancy").iterdir()` returns at least one `.yaml` file.

### Pitfall 3: The 15+ Other `Path("vulndocs")` Callsites
**What goes wrong:** Plan 01 fixes `patterns.py` L32 but leaves 15+ other bare `Path("vulndocs")` in `core/availability.py`, `learning/bootstrap.py`, `investigation/loader.py`, `vulndocs/ingestion/categorizer.py`, `agents/context/extractor.py`, etc. These still break from non-project-root cwd.
**Why it happens:** The CONTEXT.md correctly identifies the primary root cause in `patterns.py` but the codebase analysis reveals the pattern is replicated across many files.
**How to avoid:** The read-path inventory (D-2c) is mandatory BEFORE implementation. Grep broadly for `Path("vulndocs")`, `Path('vulndocs')`, and any other bare relative construction referencing vulndocs.
**Warning signs:** Tests pass for `query` but other commands (vulndocs validate, bead generation, learning bootstrap) still fail from workspaces.

### Pitfall 4: Graph Path Hardcoded in 6+ CLI Commands
**What goes wrong:** Plan 02 changes `GraphStore` internals but leaves `Path(".vrs/graphs/graph.json")` in 6 places across `cli/main.py` (query, suggest, schema-snapshot, detect, plus label and beads commands).
**Why it happens:** The CONTEXT.md D-2c correctly mandates a read-path inventory for `GraphStore` callsites, but the pattern `Path(".vrs/graphs/graph.json")` is used directly in typer defaults, not through `GraphStore`.
**How to avoid:** Include these hardcoded default paths in the read-path inventory. Each CLI command that constructs `Path(".vrs/graphs/graph.json")` must be updated to use the new graph resolution logic (error-when-ambiguous or --graph flag).

### Pitfall 5: Concurrent Agent Builds and `os.replace()` Race
**What goes wrong:** Two agents build the same contract simultaneously. Both write `graph.toon.tmp`, then both call `os.replace()`. The file ends up with one agent's data, but `meta.json` might be the other agent's metadata.
**Why it happens:** `os.replace()` is atomic for a single file but not for a pair of files (graph.toon + meta.json).
**How to avoid:** Plan 02 specifies skip-if-exists semantics. If the graph already exists, the second build skips. For concurrent races, the first `os.replace()` wins; the second overwrites but both graph+meta are from the same build (same process writes both). The Plan 04 concurrent same-contract test validates this: assert `meta.json` is valid JSON with correct fields.
**Warning signs:** `meta.json` contains fields from a different contract than the graph file.

### Pitfall 6: Testing Framework Pre-Validation Not Disambiguating Failure Modes
**What goes wrong:** Plan 04 smoke test fails and the operator can't tell whether the CLI is broken or the testing infrastructure (WorkspaceManager, TeamManager, EvaluationRunner) is broken.
**Why it happens:** Agent Teams testing involves complex infrastructure. If WorkspaceManager fails to create a workspace, no CLI test can run.
**How to avoid:** Plan 04 specifies a framework pre-validation step: test WorkspaceManager, TeamManager, and EvaluationRunner independently BEFORE spawning agents. Only proceed to agent spawning if all three pass.
**Warning signs:** All agent sessions fail with infrastructure errors (workspace creation, team creation) rather than CLI errors.

## Source Code State Verification

### patterns.py -- Confirmed Root Cause
- `_DEFAULT_VULNDOCS = Path("vulndocs")` at L32 is a bare cwd-relative path
- `get_patterns()` at L534-550 uses `_DEFAULT_VULNDOCS` as default, raises `PatternDirectoryNotFoundError` on missing dir
- `PatternStore.load_vulndocs_patterns()` iterates subdirectories, loads YAML files
- **Complexity:** The `PatternStore` class and `get_patterns()` are the canonical entry points; all 466 patterns flow through here. The fix must maintain the same return type (`list[PatternDefinition]`).
- **Consumers:** `intent.py` L221 calls `get_patterns(pattern_dir)`, `executor.py` uses it, `schema_hints.py` uses it, VQL2 executor uses it. All consumers pass through `pattern_dir` parameter which defaults to None (triggering `_DEFAULT_VULNDOCS`).

### store.py -- Confirmed Root Cause
- `GraphStore` is a dataclass with `root: Path`
- `save()` writes to `self.root / f"graph{ext}"` (L52) -- flat file, no isolation
- `load()` tries `self.root / "graph.toon"` then `self.root / "graph.json"` (L83-84) -- single file
- **Complexity:** Simple dataclass, ~98 lines total. Clean refactor target.
- **No meta.json exists today.** The plan introduces it as a new sidecar file.
- **No atomic writes today.** Direct `target.write_text()` at L66.

### intent.py -- Confirmed Silent Degradation
- L220-224: If `patterns` is empty, calls `get_patterns(pattern_dir)` and `resolve_rule_matches()` -- if vulndocs can't be found, this raises an exception
- L226: But if `get_patterns()` succeeds with 0 matching rules, `query_kind` stays "nodes" -- the default
- **The silent degradation is at L226-241:** The query_kind cascade sets "pattern" only if `patterns` is non-empty. When pattern matching fails silently (e.g., generic query text that doesn't match any rule), the query falls to "nodes" which returns raw graph nodes -- meaningless for agents expecting vulnerability pattern results.
- **The fix:** When agents send queries like "find reentrancy vulnerabilities" and no patterns match, the system should report "0 pattern matches for this query" rather than falling back to raw node enumeration.

### cli/main.py -- Confirmed Hardcoded Paths
- `build_kg()` L270: `output_dir = Path(out) if out else project_root / ".vrs" / "graphs"` -- anchored to project root (OK)
- `query()` L576: `graph_path = Path(graph) if graph else Path(".vrs/graphs/graph.json")` -- cwd-relative (BROKEN)
- `suggest()` L659: Same pattern (BROKEN)
- `schema_snapshot` L771: Same pattern (BROKEN)
- `detect` L895: Same pattern (BROKEN)

### pyproject.toml -- Confirmed Missing package_data
- Current `[tool.hatch.build.targets.wheel]` at L48-53 only includes `packages = ["src/alphaswarm_sol"]` and force-includes `shipping/skills` and `shipping/agents`

### Recommended Validation Strategy

**Three-layer approach:**

1. **Unit tests (Plans 01-03):** pytest tests for each new function/method. These are deterministic, fast, and run in CI. Key invariants: `vulndocs_read_path()` works from any cwd, `contract_identity()` is deterministic and canonical, `GraphStore` atomic writes don't leave corrupt files, query routing errors with correct exit codes.

2. **Integration tests (Plans 01-03):** pytest tests that exercise the full CLI commands via `subprocess.run()` or typer testing utilities. Verify end-to-end behavior: build-kg from a temp directory, query with --graph flag, check stdout/stderr/exit code contracts.

3. **Agent Teams smoke tests (Plan 04):** The real validation. Real agents running in real Jujutsu workspaces using real CLI commands. This layer validates that the fixes work in the actual execution context (agents don't know they're being tested). Framework pre-validation runs first to disambiguate "CLI broken" from "infrastructure broken."

**Sampling frequency:** Unit/integration tests run on every commit. Agent Teams tests run once per Plan 04 execution (minimum: 2 contracts, 3 queries each, 1 concurrent scenario, 1 same-contract race). Post-phase regression: re-run Agent Teams tests if any subsequent phase changes CLI code.

### Edge Cases Requiring Validation
- **Zip-backed packages:** `vulndocs_read_path()` returns a Traversable backed by zipfile, not filesystem. Consumers must not call `os.stat()` or `glob.glob()` on it.
- **Dual-root conflict detection:** Both editable install and wheel-installed vulndocs exist. The `_assert_single_vulndocs_root()` must detect this at import time.
- **Non-git directories:** `contract_identity()` project root resolution falls back to CWD. `meta.json` must contain `project_root_type: "fallback_cwd"`.
- **Unicode contract filenames:** `contract_identity()` uses UTF-8 encoding. Filenames with non-ASCII characters must hash correctly.
- **Symlinked contract paths:** `os.path.realpath()` must resolve symlinks before hashing. Two different symlinks to the same file must produce the same identity.
- **Empty .vrs/graphs/ directory:** Query must say "no graphs found" with build suggestion, not crash.
- **Partial write detection:** `GraphStore.load()` must raise `CorruptGraphError` when `.tmp` exists without `.toon`.
- **Stale graph detection:** `--check-fresh` must correctly compute and compare hashes; exit 42 on mismatch.
- **Concurrent same-contract race:** Two agents build same contract simultaneously. After both finish, `meta.json` must be valid JSON with correct schema.

The `vulndocs_read_path()` function would then `importlib.resources.files("alphaswarm_sol").joinpath("_vulndocs_data")`. Alternatively, test whether hatchling correctly handles data directories alongside Python subpackages with the same name. This must be validated in Plan 01 implementation with a real wheel build.
   - **Confidence:** MEDIUM -- this is the one area where the CONTEXT.md assumes `importlib.resources.files("alphaswarm_sol").joinpath("vulndocs")` works, but may conflict with the existing `alphaswarm_sol.vulndocs` Python module.
