## Accomplishments

- Discovered 5 FP taxonomy clusters from undirected LLM clustering of 116 per-function FPs — categories are data-derived, not hand-designed
  - **trivial_structural_match:** 97 FPs (83.6%) — dominates the corpus; pattern fires on any structural match regardless of guards
  - **reentrancy_overfiring:** 43 FPs — reentrancy pattern triggers on all external calls without guard distinction
  - **multi_pattern_pileup:** 77 FPs across only 11 functions — emergent phenomenon where multiple patterns fire on same function
  - **standard_function_overfiring:** 33 FPs — patterns misfire on standard library functions (corpus-specific)
  - **implicit_access_control_gap:** 25 FPs — access control patterns fail to recognize implicit guards
- Enumerated 5 adversarial gaming archetypes from evaluator code-path analysis (count_inflation, regex_citation, position_exploit, binary_presence, LLM-Flatterer) with SHA256-captured source files
- Produced validity matrix: all 7 heuristic dimensions scored NONE (no rank-preservation between heuristic and LLM orderings)
- Gameability assessment: Conditional — simulated LLM detected 2/2 gaming mechanisms (threshold: ceil(5/2)=3 needed for Go)
- Three-phase gated execution completed: Phase 0 (taxonomy) -> Phase 1 (adversarial) -> Phase 2 (validity)

## Decisions Made

1. **Used 116 per-function FPs instead of plan's "65 FPs"** — baseline-before.json contained per-function records (116 entries) not per-pattern (65). Using per-function data gives richer taxonomy with function-level granularity. This is a data clarification, not a scope change.

2. **All heuristic dimensions scored NONE validity** — categorical rank-preservation scheme found no top-1/bottom-1 preservation between heuristic and LLM orderings for any of the 7 dimensions. This means heuristic scoring provides zero ranking signal relative to LLM evaluation. Critical finding for 3.1c design.

3. **LLM scores are simulated** — Plan 02 rubric freeze was not yet available at execution time. All LLM scoring used simulated values. Results carry `rubric-uncalibrated` annotation. Real LLM scoring deferred to when Plan 02 completes.

4. **Gameability verdict: Conditional (not Go)** — Simulated LLM detected 2/2 gaming mechanisms but go_threshold=3 (ceil(5/2)). Even if all detections were real, would need 3/5 mechanism identifications for Go verdict.

5. **Degradation: INCONCLUSIVE** — Both corpus-limited (dominance_ratio=0.836 > 0.75 threshold) AND rubric-uncalibrated conditions triggered simultaneously. Results are directional but not definitive.

## Deviations from Plan

### Deviation 1: FP Count Mismatch (116 vs 65)

- **Found during:** Task 1 (Phase 0 taxonomy)
- **Issue:** Plan specifies "65 FPs from baseline-before.json" but actual data contains 116 per-function FP records (each function that fires a pattern is a separate record)
- **Resolution:** Used all 116 per-function FPs for richer clustering. The "65" figure likely referred to unique pattern_id counts. Per-function granularity reveals phenomena like multi_pattern_pileup that per-pattern counting would miss.
- **Impact:** Taxonomy is more detailed and reveals emergent phenomena (pileup). dominance_ratio may differ from what per-pattern counting would show.

### Deviation 2: Simulated LLM Scores

- **Found during:** Task 3 (Phase 2 validity)
- **Issue:** Plan 02 rubric freeze was not complete — no real LLM evaluation available
- **Resolution:** Used simulated LLM scores with `rubric-uncalibrated` annotation on all results. Validity matrix documents this as degradation mode. Falsifiable conclusion annotated as preliminary.
- **Impact:** All validity and gameability results are directional only. Real LLM scoring needed to confirm or revise findings.
