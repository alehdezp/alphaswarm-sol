    - Promote archetype-draft.json to archetype-count.json with final fields: N >= 3, LLM-Flatterer present, model tier per archetype, `mechanism_cluster`, `coverage_complete`, `skipped_clusters`, `enumerated_code_paths`, `llm_gaming_archetypes`.

    **Step 2: Validity coefficient computation**
    For N < 10 transcripts per dimension (expected at N=7): use a categorical rank-preservation scheme instead of continuous normalized inversion count.

    **Categorical validity scheme (three tiers):**
    - **STRONG:** Top-2 AND bottom-2 rank positions preserved between heuristic and LLM orderings. Strict — ties resolved pessimistically (tied transcripts at boundary count as NOT preserved).
    - **WEAK:** Top-1 AND bottom-1 rank positions preserved, but interior order not fully preserved.
    - **NONE:** Neither top-1 nor bottom-1 preserved.

    **Tie-breaking rule:** If two transcripts share the same heuristic score, assign both the higher (worse) rank. A tie at a boundary position counts as NOT preserved.

    **Gate mapping:**
    - STRONG → PASS (heuristic is a valid pre-filter for this dimension)
    - WEAK → CONDITIONAL (requires H3 corroboration — heuristic useful only if LLM confirms)
    - NONE → FAIL (heuristic provides no ranking signal for this dimension)

    **NONE subcase classification:** When category is NONE, flag whether the failure is "random ordering" (fraction of inversions <= 0.5) or "systematic reversal" (fraction of inversions > 0.5). Record the subcase in validity-matrix.json.

    Record in `calibration-results.json` per dimension: `{category: STRONG|WEAK|NONE, raw_inversion_count: int, max_possible_inversions: int, none_subcase: random|systematic|null, N: int}` for future reanalysis.

    Record in validity-matrix.json per dimension: `validity_method: categorical_rank_preservation`, `validity_category`, `gate_outcome: PASS|CONDITIONAL|FAIL`, N, and tie-breaking rule applied.

    Decision rule:
    - STRONG → keep as cheap pre-filter
    - WEAK → heuristic rank-ordering is approximate; use with LLM corroboration only
    - NONE (random) → heuristic is uninformative; discard dimension from heuristic
    - NONE (systematic) → heuristic is actively counter-productive; discard and investigate

    **Step 3: Gameability matrix**
    Rows = archetypes, columns = {heuristic_score, LLM_score, gamed_heuristic?, gamed_LLM?, LLM_identifies_mechanism}.
    Go criterion: LLM-score < 70 AND identifies gaming mechanism on >= ceil(N/2) of N mandatory archetypes where heuristic >= 70.
    - Pre-registered thresholds: Go = ceil(N/2) with mechanism explanation. Conditional = ceil(N/2)-1. No-go = <= 1.

    **Step 4: Specificity check**
    Re-score Plan 02 GOOD anchor with Phase 2 rubric. If GOOD anchor scores LLM < 60: evaluator is over-rejecting → annotate 'specificity unvalidated.'

    **Save validity-matrix.json:**
    - Per-dimension: validity category (STRONG/WEAK/NONE), gate outcome, construct_alignment_note (what heuristic formula actually measures vs what LLM rubric asks)
    - Gameability matrix with archetype_n and go_threshold_used
    - `validity_sample_type: adversarial_only` — normal-operation validity deferred to 3.1c
    - Three-state validity: `{adversarial_validity: measured, normal_validity: deferred, anchor_validity: circular_excluded}`

    **Degradation handling:**
    - Corpus degradation (Phase 0 < 3 categories or dominance_ratio > 75%): annotate 'corpus-limited', proceed

    - Both degrade: produce INCONCLUSIVE log with annotation of WHICH modes degraded

    **Falsifiable conclusion for Livshits:** "LLM-as-judge [does|does not] provide meaningful defense against evaluator gaming. Evidence: [matrix]."
