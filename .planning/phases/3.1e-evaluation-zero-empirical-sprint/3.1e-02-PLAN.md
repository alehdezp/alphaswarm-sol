    **Step 3: Design 3 calibration anchors**
    All grounded in domain reasoning quality using real contract names from baseline:

    **GOOD anchor:** Correct CEI violation trace with node IDs, multi-step reasoning connecting graph observations to vulnerability hypothesis. Contains 3 tool calls in Claude Code format, >= 1 BSKG query with plausible syntax. unique_tool_count <= 3, bskg_query_count <= 3 (keeps heuristic below saturation of 100).

    - unique_tool_count = GOOD_unique_tool_count (REQUIRED for heuristic parity on reasoning_depth)
    - bskg_query_count = GOOD_bskg_query_count (PREFERRED, ±1 relaxation allowed)
    - Reasoning chain reads as plausibly real but shallow
    - Result interpretation lacks causal specificity
    - BSKG queries have genuine semantic content (not template/empty)

    **BAD anchor:** Correct conclusion reached by luck, zero graph-based reasoning.
    - unique_tool_count <= 3, bskg_query_count = 0
    - NO reasoning chain — jumps from finding to conclusion
    - Anti-strawman check: contains at least one correct element (function name, pattern class, or vulnerability consequence)

    3. Pass conditions: both primary dimensions separate AND composite score separates. Partial separation (one of two primary dimensions fails) results in CONDITIONAL status — record which dimension failed and proceed with a note.
    4. Saturation check: if any dimension scores 1.0 for both GOOD and BAD, flag that dimension as uninformative and exclude it from composite.
    5. If no anchor pair satisfies minimum separation: halt Task 2, emit `anchor-feasibility-FAIL.json` with per-dimension scores, escalate to human. Do not substitute different transcripts without recording the substitution.

    **LLM smoke test (first 5 min):**
    Invoke `claude -p --model {pre-registered model_id} --output-format json` with trivial prompt to verify invocation works. Must complete within 5 minutes. If fails: record `invocation_status: lm_invocation_failed` in element-preregistration.json, session pivots to debugging only (max 20 min). Record smoke test result as `smoke_test_passed: true/false` in element-preregistration.json — this is a Session B gate.
    Pre-scoring: implement `validate_llm_response(raw)` function: on `JSONDecodeError` or missing required fields, log `{"error": "invalid_json", "raw_output": raw[:500], "dimension": dim, "pair": pair}` to `llm-scores.jsonl`, count as unusable call (not a flip). If >= 2 of 6 pairwise calls per dimension fail validation: mark dimension `LLM_unreliable` with `reason: invocation_failure` before computing go/no-go. All scoring calls use `--model {model_id}` explicitly.

    **Step 0a-0c: Rubric calibration (max 3 revision rounds)**
    For each primary dimension (selected in Task 1 Step 0):
    1. Fill REASONING_PROMPT_TEMPLATE with rubric text (`{expected}`) and anchor `{observed}` content
    2. Score all 3 anchors via `claude -p --output-format json`
    3. Calibration succeeds when: GOOD > MEDIOCRE > BAD AND GOOD-BAD spread >= 30 AND GOOD-MEDIOCRE gap >= 10
    4. Max 3 targeted revisions (each must target specific failure mode from prior round)
    5. Freeze when calibration succeeds: record SHA256 immediately, no edits after
    6. If calibration not achieved in 3 revisions: use best version, record "rubric calibration incomplete"

    **Non-determinism protocol:**
    For each pairwise comparison (3 pairs per dimension: GOOD-BAD, MEDIOCRE-BAD, GOOD-MEDIOCRE):
    - Call twice per pair via `claude -p`. Record both winners.
    - GOOD-BAD flip = strong evidence → LLM_unreliable
    - MEDIOCRE-BAD flip = moderate → record in `flip_resolution` (default: proximity_artifact)
    - GOOD-MEDIOCRE flip = close pair variance → informative for 3.1c, NOT LLM_unreliable
    - If budget allows: upgrade GOOD-BAD and MEDIOCRE-BAD to majority-vote n=3

    **Instrument 1 (ordering):** For each of 3 pairs per dimension, `claude -p` asks "which demonstrates better {dimension} reasoning?" Record winner + confidence.

    **Instrument 2 (scale calibration):** Absolute scores per anchor per dimension. Retained for 3.1c data, NOT for go/no-go.

    **Instrument 3 (divergence evidence):** Does LLM explanation name something the heuristic formula cannot compute?

    **Hallucination check (after official scoring):**
    Check LLM explanations for fabricated entities across five entity classes, each with an authoritative source:
    1. **FUNCTION_NAME** — must appear in the transcript's contract source section
    2. **VARIABLE_NAME** — must appear in the transcript's contract source section
    3. **LINE_NUMBER** — must be within the contract's line range
    4. **TOOL_OUTPUT_VALUE** — must match a value in the transcript's tool output section
    5. **GRAPH_NODE_ID** — must match a node ID in the BSKG query results section

    Extraction step: parse LLM explanation text, extract all tokens matching each entity class pattern.
    Cross-reference step: for each extracted entity, look up in authoritative source. Not found = hallucinated.
    For ambiguous entity classes (e.g., short variable names like `x` or `i`), fall back to human review with `source: "HUMAN"`.

    Gate: 0 hallucinated entities = PASS. Any hallucinated entity = FAIL with list. If any dimension has >= 1 hallucinated entity: dimension is LLM_unreliable.

    Output: `hallucination-check.json` with schema:
    ```json
    {
      "verdict": "PASS" | "FAIL",
      "entity_counts": {"FUNCTION_NAME": int, "VARIABLE_NAME": int, "LINE_NUMBER": int, "TOOL_OUTPUT_VALUE": int, "GRAPH_NODE_ID": int},
      "hallucinated": [{"entity": str, "class": str, "context": str, "source": "AUTO" | "HUMAN"}]
    }
    ```

    **Step 0d: Score held-out anchor**
    Score with all three instruments (pairwise against each calibration anchor, absolute score, explanation content). Record unique_tool_count vs calibration anchors. If scores low as expected: rubric generalizes. If scores high: rubric overfit.

    **Go/No-Go Decision:**
    Score hypotheses:
    - H1 (primary dimension A, GOOD vs MEDIOCRE): Confirmed if pairwise ordering GOOD wins AND LLM cites causal chain quality. Weight: 1.0.
    - H2 (primary dimension B, GOOD vs MEDIOCRE): Confirmed if pairwise ordering GOOD wins AND LLM cites interpretation depth. Weight: 1.0.
    - H3 (either dimension, BAD vs MEDIOCRE): Confirmed if LLM cites ABSENCE of evidence chain, not just absence of graph queries. Weight: 0.5.

    After computing H1/H2/H3, explicitly check for the RIGHT_ANSWER_WRONG_REASON pattern before declaring go/no-go.

    Decision table (apply from CONTEXT.md, with additions):
    - H1=pass, H2=pass, H3=pass → Score >= 2.5 → **GO**
    - H1=pass, H2=pass, H3=fail → Score = 2.0 → **GO** (investigate H3 failure mode)
    - H1=pass, H2=fail, H3=pass → Score = 1.5 → **AMBIGUOUS**
    - H1=fail, H2=pass, H3=pass → Score = 1.5 → **AMBIGUOUS**
    - H1=pass, H3=fail (regardless of H2) → Score = 1.0 → **NO-GO: RIGHT_ANSWER_WRONG_REASON**. The LLM orders correctly but cannot articulate why the BAD anchor lacks evidence chain reasoning. Action: examine LLM explanations for spurious features; consider rubric revision (counts against 3-revision limit). Record which H3 criteria failed for downstream diagnosis.
    - Score < 1.0 → **NO-GO**

    Update element-preregistration.json with GATE C by adding a `gate_c` top-level key (never mutating `gate_a` fields — GATE A is immutable after pre-registration):
    `gate_c: { flip_resolution_observed: str, flip_count_by_pair: {pair: count}, diverges_from_preregistration: bool }`
    The automated verify for Task 2 must assert `gate_c` key exists.

    **Output:** `llm-recommendation.md` — verdict, hypothesis scores, forward actions, citing specific row from decision table, AND a required `verdict_tier` field: `"synthetic_preliminary"` if no real transcript was scored (pre-flight found none), `"confirmed"` if >= 1 real transcript was evaluated. Maximum achievable verdict without real transcript: `"Go (preliminary — requires real transcript confirmation before 3.1c Track B activation)"`. Plan 05 Track B activation gates on `verdict_tier == "confirmed"` or explicit human override documented in plan-02-verdict-final.json (written by Task 3 checkpoint).

    Log all scoring calls to `.vrs/experiments/plan-02/llm-scores.jsonl` with fields: `(prompt_sha256, dimension, expected_sha256, observed_sha256, workflow_id, persona, llm_score, llm_evidence_count, cost_tokens, timestamp)`.

    **Session time guard:** If Session B reaches 90 minutes with calibration incomplete: freeze at current rubric version, annotate 'calibration-incomplete,' proceed to scoring.
