## Decisions Made

### DECISION-07204-01: Coverage Dimensions

**Choice:** Three-dimensional coverage (taxonomy, operations, labels)

**Rationale:** Phase 7.2 CONTEXT explicitly requires:
- Taxonomy coverage with vuln/safe exemplars per category
- Operation coverage mapped to patterns
- Label coverage in Tier C tests

### DECISION-07204-02: Counterfactual Types

**Choice:** Five transformation types implemented (audit_mirror deferred)

**Rationale:** These cover all semantic edit scenarios from Innovation I1:
- Guard inversion (security check bypass)
- CEI order swap (reentrancy creation)
- Grace period (timing window manipulation)
- Helper depth (detection bypass via indirection)
- Chain condition (L1/L2 conditional bypass)

### DECISION-07204-03: Hard-Negative Suite Composition

**Choice:** 8 initial templates, one per major vulnerability class

**Rationale:** Covers the most common FP sources:
- Reentrancy with guards
- Access control with timelocks
- Oracle with staleness
- Flash loan with cooldowns
- Delegatecall with whitelists
- External calls with CEI
- Helper calls with inlined checks
- L2 sequencer with grace periods
