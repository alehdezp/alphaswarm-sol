# Phase 07.2 Plan 11: PCP v2 High-Risk Batch Summary

PCP v2 context packs created for 4 high-risk patterns covering reentrancy, delegatecall, oracle freshness, and tx.origin authentication vulnerabilities.

## Objective

Apply PCP v2 context packs to a prioritized batch of high-risk patterns, beginning the bulk PCP v2 migration with concrete, high-impact examples.

## Key Deliverables

### 1. Batch A - Reentrancy & Delegatecall (Task 1)

**cross-function-reentrancy.pcp.yaml**
- Full PCP v2 structure with schema_version "2.0"
- Required ops: CALLS_EXTERNAL, WRITES_STATE, READS_STATE
- Ordering variants for CEI violations
- Anti-signals: reentrancy-lock (critical), cei-pattern (high), pull-payment (high)
- Witness requirements: external_call_to_untrusted, writes_shared_state, multiple_public_functions_share_state
- Negative witnesses: reentrancy_guard_on_all_entry_points, state_written_before_external_call
- Risk envelope: mixed assets, user->contract boundary, out value movement, high severity floor

**delegatecall-public.pcp.yaml**
- Full PCP v2 structure with schema_version "2.0"
- Required ops: DELEGATECALL
- Anti-signals: owner-only (critical), hardcoded-target (critical), whitelist (high), proxy-pattern (high)
- Witness requirements: uses_delegatecall, public_or_external_visibility
- Negative witnesses: target_address_hardcoded, owner_only_access_control, whitelisted_targets_only
- Guard taxonomy covering access control and hardcoded/whitelisted targets
- Exploit modeling for storage corruption, selfdestruct, and fund drain

## PCP v2 Structure Applied

Each pack includes all required PCP v2 sections:
- **Identity**: id (pcp-prefixed), version, pattern_id, name, summary, scope
- **Determinism**: no_rag: true, no_name_heuristics: true, serialization_order, hash_seed
- **Budget**: pcp_max_tokens: 800, context_max_tokens: 2200, cheap/verify/deep pass tokens
- **Evidence weights**: required, strong, weak, optional categories
- **Witness requirements**: minimal_required, negative_required
- **Exploit modeling**: preconditions, exploit_steps, impact_invariants
- **Operation signatures**: required_ops, ordering_variants, forbidden_ops
- **Anti-signals**: guard_type, severity, expected_context, evidence_refs, bypass_notes
- **Counterfactuals**: if_removed, becomes_true, notes
- **Guard taxonomy**: type, description, expected_ops, bypass_notes
- **Risk envelope**: asset_type, trust_boundary, value_movement, severity_floor
- **Composition hints**: co_occurs_with, combine_with
- **Economic placeholders**: value_flows, incentive_hooks, role_assumptions, offchain_dependencies
- **Evidence requirements**: node_types, edge_types, path_constraints
- **Unknowns policy**: missing_required, missing_optional, missing_anti_signal (all "unknown")
- **Metadata**: created, updated, author
