#### Vulnerability Pattern: Missing Grace Period

**Root Cause:** When an L2 sequencer goes down and restarts, all queued Chainlink price updates become available nearly simultaneously. Users have no time to react, enabling unfair liquidations.

**Correct Implementation Pattern (from Sherlock audits):**

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.20;

interface ISequencerUptimeFeed {
    function latestRoundData() external view returns (
        uint80 roundId,
        int256 answer,
        uint256 startedAt,
        uint256 updatedAt,
        uint80 answeredInRound
    );
}

interface IChainlinkPriceFeed {
    function latestRoundData() external view returns (
        uint80 roundId,
        int256 answer,
        uint256 startedAt,
        uint256 updatedAt,
        uint80 answeredInRound
    );
}

contract L2OracleWithGracePeriod {
    ISequencerUptimeFeed public immutable sequencerUptimeFeed;
    IChainlinkPriceFeed public immutable priceFeed;

    uint256 public constant GRACE_PERIOD_TIME = 3600; // 1 hour

    error SequencerDown();
    error GracePeriodNotOver();
    error StalePrice();
    error InvalidRound();

    constructor(address _sequencer, address _price) {
        sequencerUptimeFeed = ISequencerUptimeFeed(_sequencer);
        priceFeed = IChainlinkPriceFeed(_price);
    }

    function getPrice() external view returns (int256) {
        // Step 1: Check sequencer status
        (, int256 sequencerAnswer, uint256 startedAt, ,) =
            sequencerUptimeFeed.latestRoundData();

        // answer == 0: Sequencer is UP
        // answer == 1: Sequencer is DOWN
        if (sequencerAnswer != 0) {
            revert SequencerDown();
        }

        // Step 2: Enforce grace period after recovery
        uint256 timeSinceUp = block.timestamp - startedAt;
        if (timeSinceUp <= GRACE_PERIOD_TIME) {
            revert GracePeriodNotOver();
        }

        // Step 3: Get price with staleness check
        (uint80 roundId, int256 price, , uint256 updatedAt, uint80 answeredInRound) =
            priceFeed.latestRoundData();

        if (updatedAt < block.timestamp - 1 hours) {
            revert StalePrice();
        }

        if (answeredInRound < roundId) {
            revert InvalidRound();
        }

        return price;
    }
}
```

**Vulnerable Pattern (from existing test contract `tests/contracts/OracleL2NoSequencerCheck.sol`):**

```solidity
// Missing sequencer check - VULNERABLE

    require(answeredInRound >= roundId, "bad round");
```

```python
                "access_control.no_restriction"
```

### Statistical Stability Testing

**Approach:** Run Tier C labeling N times and compute stability score.

```python
import statistics
from collections import Counter

def compute_stability(results: list[set[str]], n_runs: int = 50) -> float:
    """
    Compute stability score for Tier C pattern results.

    Returns: float between 0.0 and 1.0 where 1.0 = perfect stability
    """
    # Count frequency of each unique result set
    result_counts = Counter(frozenset(r) for r in results)

    # Most common result frequency
    most_common_count = result_counts.most_common(1)[0][1]

    # Stability = fraction of runs producing most common result
    return most_common_count / n_runs

# Example usage
results = [tier_c_matcher.match(contract, labels) for _ in range(50)]
stability = compute_stability(results)
assert stability >= 0.85, f"Stability {stability} below 0.85 threshold"
```

**Threshold:** >= 0.85 (85% consistent findings across 50 runs)

## 4. Counterfactual Generation Techniques

### AST Manipulation with Slither

**Slither Version:** 0.11.5 (Jan 2026)
**Python Version:** >=3.10

**How Slither Parses AST:**
1. Runs `solc` compiler to get AST
2. Converts AST to SlithIR (intermediate representation)
3. Builds Control Flow Graph (CFG) per function

**Using Slither for AST Access:**

```python
from slither import Slither
from slither.core.declarations import Function, Contract
from slither.slithir.operations import HighLevelCall, LowLevelCall

# Parse contract
slither = Slither("Contract.sol")

for contract in slither.contracts:
    for function in contract.functions:
        # Access function properties
        print(f"Function: {function.name}")
        print(f"Modifiers: {function.modifiers}")
        print(f"External calls: {function.external_calls_as_expressions}")

        # Access SlithIR for fine-grained analysis
        for node in function.nodes:
            for ir in node.irs:
                if isinstance(ir, HighLevelCall):
                    print(f"  External call: {ir}")
```

### Counterfactual Variant Types (from CONTEXT.md)

1. **Guard Inversion:** Flip `require()` or modifier order
   ```solidity
   // Original (vulnerable)
   function withdraw() external {
       uint amount = balances[msg.sender];
       (bool success,) = msg.sender.call{value: amount}("");
       balances[msg.sender] = 0;  // Write after call - vulnerable
   }

   // Counterfactual (safe)
   function withdraw() external {
       uint amount = balances[msg.sender];
       balances[msg.sender] = 0;  // Write before call - safe
       (bool success,) = msg.sender.call{value: amount}("");
   }
   ```

2. **CEI Reordering:** Swap Checks-Effects-Interactions order
   ```solidity
   // CEI violation (vulnerable)
   effect();
   interaction();
   check();

   // CEI correct (safe)
   check();
   effect();
   interaction();
   ```

3. **Window Tweak:** Modify time windows
   ```solidity
   // Grace period missing (vulnerable)
   require(sequencerAnswer == 0, "sequencer down");

   // Grace period present (safe)
   require(sequencerAnswer == 0, "sequencer down");
   require(block.timestamp - startedAt > GRACE_PERIOD, "grace period");
   ```

4. **Helper Delegation:** Move checks to internal helpers
   ```solidity
   // Direct check (detectable)
   function withdraw() external {
       require(msg.sender == owner);
       // ...
   }

   // Helper check (test depth detection)
   function withdraw() external {
       _onlyOwner();
       // ...
   }
   function _onlyOwner() internal view {
       require(msg.sender == owner);
   }
   ```

5. **L1/L2 Branch:** Conditional chain logic
   ```solidity
   // L2-only check
   if (block.chainid == 42161 || block.chainid == 10) {
       // Sequencer check for Arbitrum/Optimism
       _checkSequencer();
   }
   ```

**Counterfactual Acceptance Rule:** Must differ by single semantic edit; BSKG property diff must reflect intended change.

## Potential Pitfalls

- **Overwriting provenance:** URL logging must be append-only. Avoid scripts that rewrite or drop existing entries in `.vrs/corpus/metadata/urls.yaml`.
- **Inconsistent corpus ratios:** The 80/20 safe/vulnerable target should be enforced by the ingestion pipeline, not just by manual selection.
- **Tier C instability:** Label-driven patterns can oscillate with small prompt shifts. Make stability gating mandatory and treat failures as redesign signals, not tuning targets.
- **PCP drift:** Patterns without PCP v2 will break future validation; avoid partial migration that leaves high-risk patterns un-migrated.
