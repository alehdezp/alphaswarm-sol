## What Was Built

### 1. /vrs-validate-vulndocs Wrapper Skill

A user-controlled skill (`disable-model-invocation: true`) that orchestrates the mandatory VulnDocs validation pipeline with 8 sequential stages:

1. **vrs-test-conductor** - Pipeline orchestration
2. **vrs-prevalidator** - Fast provenance/schema/duplicate gate (NEW)
3. **vrs-corpus-curator** - Corpus integrity validation
4. **vrs-pattern-verifier** - Tier A/B/C evidence gates
5. **vrs-benchmark-runner** - Precision/recall metrics
6. **vrs-mutation-tester** - Pattern robustness (conditional)
7. **vrs-regression-hunter** - Accuracy degradation (conditional)
8. **vrs-gap-finder-lite** - Fast coverage scan (NEW), escalates to vrs-gap-finder if needed

### 2. vrs-prevalidator Agent (Haiku 4.5)

Fast prevalidation gate for VulnDocs changes:

- **URL provenance** - Verify all sources logged in `.vrs/corpus/metadata/urls.yaml`
- **Schema sanity** - Check required fields (id, name, severity, category, tier, sources)
- **Duplicate detection** - Find redundant or conflicting entries
- **Ledger rules** - Enforce append-only URL ledger constraints

Output format:
```json
{
  "prevalidation_result": {
    "status": "passed|failed|warnings",
    "provenance_ok": true,
    "schema_ok": true,
    "duplicates": [],
    "missing_urls": [],
    "offending_files": []
  }
}
```

### 3. vrs-gap-finder-lite Agent (Sonnet 4.5)

Fast coverage and FP hotspot scan:

- **Coverage scan** - Find under-tested vulnerability classes
- **FP hotspot detection** - Identify patterns generating false positives
- **Limited scope** - Max 10 gaps to avoid noise
- **Escalation decision** - Flag when deep Opus analysis needed

Output format:
```json
{
  "gap_analysis_result": {
    "status": "passed|failed|escalate",
    "summary": { "total_gaps": 5, "coverage_score": 0.78 },
    "gaps": [...],  // max 10
    "escalate_to_opus": false
  }
}
```

### 4. Adaptive Pipeline Rules

The pipeline adapts based on change type:

| Change Type | Stages Run |
|-------------|------------|
| New pattern | All 8 stages |
| Edit pattern | Skip corpus-curator if only pattern change |
| Metadata only | Prevalidator + schema only |

### 5. Quality Gates

| Gate | Target | Blocking |
|------|--------|----------|
| Precision | >= 85% | Yes |
| Recall (critical) | >= 95% | Yes |
| Recall (high) | >= 85% | Yes |
| Schema compliance | 100% | Yes |
| Provenance verified | 100% | Yes |

## Decisions Made

### VAL-001: 8-Stage Pipeline with Conditional Stages

**Decision:** Sequential pipeline with conditional mutation/regression stages.

**Rationale:** Adaptive execution based on change type optimizes cost while maintaining rigor.

### VAL-002: Append-Only URL Ledger

**Decision:** `.vrs/corpus/metadata/urls.yaml` is append-only - no modifications to existing entries.

**Rationale:** Immutable provenance ensures research integrity and audit trail.

### VAL-003: Gap-Finder-Lite Default with Opus Escalation

**Decision:** Use Sonnet gap-finder-lite by default, escalate to Opus only when coverage < 70% OR FP rate > 15%.

**Rationale:** Cost optimization - deep analysis only when thresholds fail.
