## What Was Built

### Exemplar Pattern: oracle-l2-sequencer-grace-missing

**Vulnerability:** L2 oracle reads that:
- CHECK sequencer uptime (answer == 0)
- FAIL to enforce grace period after sequencer restart
- Allow stale price exploitation in immediate post-restart window

**Key Distinction from oracle-004:**
- oracle-004: No sequencer check at all (more severe)
- This pattern: Sequencer check EXISTS but grace period MISSING

**Pattern Features:**
- Tier A match conditions (ops + properties)
- Tier C label refinement for higher confidence
- L2-specific with chain-agnostic detection
- Severity: High (Critical if liquidation/collateral)

## Decisions Made

### DEC-07.2-05-01: Pattern Distinction
**Decision:** Require sequencer check to exist for this pattern to match

**Rationale:**
- oracle-004 handles "no sequencer check at all"
- This pattern specifically targets "has sequencer check, lacks grace period"
- Creates actionable, non-overlapping detection

### DEC-07.2-05-02: Property-Based Testing
**Decision:** Use YAML properties for pattern matching in tests

**Rationale:**
- Declarative test case definition
- Separates test data from test logic
- Enables easy test case expansion

### DEC-07.2-05-03: 55 Test Cases
**Decision:** Exceed 50+ requirement with diverse categories

**Rationale:**
- Comprehensive coverage of edge cases
- Counterfactual variants for robustness testing
- Helper-depth variants for cross-function detection
