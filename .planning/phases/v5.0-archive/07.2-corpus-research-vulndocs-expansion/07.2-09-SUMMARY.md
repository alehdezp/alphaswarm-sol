# Phase 7.2 Plan 09: Innovation Outputs - Evidence Debt, TTL Decay, Plausibility Summary

**One-liner:** Implemented Phase 7.2 innovations I4/I5/I6 with evidence debt tracking (25% threshold), 30-day context TTL decay, and prioritization-only plausibility scoring.

## What Was Built

### Innovation I4: Evidence Debt Tracking

Evidence debt measures how many findings lack strong evidence references:
- `EvidenceDebtTracker` class with strength classification (strong/medium/weak/unknown)
- Threshold: >25% weak evidence blocks pattern promotion to ready/excellent
- Output: `.vrs/corpus/metadata/evidence_debt.yaml`

Key data structures:
- `EvidenceStrength` enum: strong, medium, weak, unknown
- `EvidenceDebtEntry`: Per-finding evidence record
- `PatternEvidenceDebt`: Aggregate debt per pattern

```python
# Evidence strength classification
strong_indicators = ["audit", "postmortem", "exploit", "poc", "cve"]
# If ref contains any strong indicator -> STRONG
# If has code_location AND graph_node_id -> MEDIUM
# Otherwise -> WEAK or UNKNOWN
```

### Innovation I5: Context TTL Decay

Economic context fields have 30-day TTL with decay to unknown:
- `ContextTTLManager` class for TTL tracking and enforcement
- Decay rule: expired -> "unknown", NEVER "safe"
- Output: `.vrs/context/context_ttl.yaml`

Key principle: **Stale context cannot assert safety.** When TVL, oracle source, or governance assumptions expire, they decay to unknown status - never safe.

### Innovation I6: Plausibility Scoring

Exploit plausibility scoring for prioritization only:
- `PlausibilityScorer` class with factor-based scoring
- Factors: liquidity_depth, capital_requirement, time_window, access_barrier, complexity
- Priority tiers: P0 (>=0.8), P1 (>=0.6), P2 (>=0.4), P3 (<0.4)
- Output: `.vrs/corpus/metadata/plausibility_scores.yaml`

**CRITICAL:** Scores are for PRIORITIZATION ONLY - they NEVER hide findings.

### Metrics Integration

`InnovationMetrics` dataclass integrates all three innovations:
- Added to `TestMetrics.innovation_metrics` field
- Included in `format_summary()` output
- Serializes via `to_dict()` / `from_dict()`

Factory method `InnovationMetrics.from_trackers()` creates metrics from tracker instances.

## Key Decisions

| Decision | Rationale |
|----------|-----------|
| 25% weak evidence blocks promotion | Per CONTEXT I4: findings still reported, just can't promote patterns |
| TTL decay to "unknown" | Stale context cannot assert safety - only unknown status is safe |
| Plausibility for prioritization only | Per CONTEXT I6: scores never suppress, only order |
| 30-day default TTL | Balance between freshness and maintenance burden |
