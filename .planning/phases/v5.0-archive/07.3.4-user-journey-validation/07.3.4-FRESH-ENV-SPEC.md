# Fresh Environment Specification

## Purpose

Define the isolation protocol for outsider testing. A fresh environment simulates a user who has:
- Never seen the codebase
- No developer context or CLAUDE.md
- Only public documentation and installed tools

---

## Isolation Rules

### What is EXCLUDED (Forbidden Context)

| Category | Exclusion | Rationale |
|----------|-----------|-----------|
| Planning | `.planning/` directory | Developer-only artifacts |
| Project config | `CLAUDE.md` | Developer instructions |
| Development tools | `scripts/e2e/` | Testing infrastructure |
| Internal docs | `docs/reference/` (internal) | Developer reference |
| Dev skills | Development-only skills | `/gsd:*`, `/pattern-forge`, etc. |
| Test fixtures | `tests/contracts/` | Not user-facing |
| Local state | `.vrs/` (if pre-existing) | Clean slate required |

### What is INCLUDED (Available Context)

| Category | Inclusion | Location |
|----------|-----------|----------|
| README | `README.md` | GitHub repo root |
| User docs | `docs/guides/` (public) | User-facing guides |
| CLI help | `alphaswarm --help` | Installed tool |
| Product skills | `/vrs-audit`, `/vrs-investigate` | Claude Code skills |
| Public PyPI | `uv tool install alphaswarm-sol` | Package registry |

## Fresh Workspace Protocol

### Location Requirements

```yaml
workspace:
  base: /tmp/fresh-user-test/
  format: /tmp/fresh-user-test/{persona}-{timestamp}/
  example: /tmp/fresh-user-test/novice-20260131-094500/

isolation:
  # NO symlinks to repo
  # NO copies of developer artifacts
  # NO pre-installed alphaswarm (test install process)
  symlinks_allowed: false
  developer_artifacts_allowed: false
  pre_installed_tools_allowed: false

cleanup:
  # Clean after each test run to ensure isolation
  auto_cleanup: true
  retention_hours: 24  # Keep for debugging failures
```

### Directory Structure

```
/tmp/fresh-user-test/{persona}-{timestamp}/
├── project/                  # User's simulated project
│   └── contracts/            # Test Solidity contracts
│       └── SimpleVault.sol   # Minimal test contract
├── evidence/                 # Captured evidence pack
│   ├── transcript.txt        # claude-code-agent-teams transcript
│   ├── timing.json           # Duration metrics
│   └── artifacts/            # Generated outputs
├── logs/                     # Debug logs
│   └── setup.log             # Setup script output
└── state.json                # Test run state
```

## claude-code-agent-teams Session Requirements

### Session Naming

```yaml
session:
  name: fresh-user-{persona}-{run_id}
  socket: /tmp/fresh-user-{run_id}.sock

panes:
  # Single-pane setup simulates real user (no orchestrator)
  user:
    role: user
    index: 0
    cwd: /tmp/fresh-user-test/{persona}-{timestamp}/project/
    shell: zsh
```

### Isolation from Developer Sessions

The fresh environment session MUST:
1. Use a dedicated socket (not shared with `alphaswarm-claude-code-agent-teams-session`)
2. Start with clean shell environment (no sourced CLAUDE.md aliases)
3. Have `$HOME` pointing to a clean directory (no `~/.claude/`)
4. Have `$PATH` reset to system defaults + uv

### Environment Variables

```bash
# REQUIRED: Clean environment
export HOME=/tmp/fresh-user-test/{persona}-{timestamp}/home
export XDG_CONFIG_HOME=$HOME/.config
export XDG_DATA_HOME=$HOME/.local/share

# FORBIDDEN: No developer context
unset CLAUDE_MD
unset VRS_DEV_MODE
unset ALPHASWARM_DEBUG

# OPTIONAL: Networking (for Exa MCP, etc.)
# Network access allowed but timeout-protected
```

## Evidence Pack Capture

### Required Evidence

Every fresh environment run MUST produce:

```yaml
evidence_pack:
  required:
    - transcript: "claude-code-agent-teams pane capture with timestamps"
    - timing: "start/end/duration metrics"
    - outcome: "success/failure/partial status"
    - friction_points: "documented blockers and confusion"

  optional:
    - screenshots: "terminal screenshots at key moments"
    - user_prompts: "what user typed (Claude input)"
    - tool_outputs: "alphaswarm command outputs"
```

### Transcript Format

```
[2026-01-31T09:45:00Z] SESSION_START persona=novice
[2026-01-31T09:45:01Z] CMD: cd /tmp/fresh-user-test/novice-20260131/project
[2026-01-31T09:45:05Z] CMD: uv tool install alphaswarm-sol
[2026-01-31T09:45:30Z] OUTPUT: Installing alphaswarm-sol...
[2026-01-31T09:45:45Z] STATUS: install_complete duration=40s
[2026-01-31T09:45:50Z] CMD: alphaswarm --version
[2026-01-31T09:45:51Z] OUTPUT: alphaswarm-sol 0.5.0
[2026-01-31T09:46:00Z] CMD: claude
[2026-01-31T09:46:05Z] CLAUDE_PROMPT: /vrs-audit contracts/
...
[2026-01-31T09:50:00Z] SESSION_END outcome=success duration=300s
```

### Friction Point Documentation

For each friction point encountered:

```yaml
friction_point:
  id: FP-001
  timestamp: "2026-01-31T09:45:10Z"
  category: blocker | confusion | annoyance | enhancement
  priority: P0 | P1 | P2 | P3
  description: "What happened"
  expected: "What user expected"
  actual: "What actually happened"
  impact: "How it affected the journey"
  proposed_fix: "How to resolve"
```

## Validation Rules

### Pre-Run Checks

Before starting a fresh environment test:

```bash
# 1. Verify clean workspace
[ ! -d "/tmp/fresh-user-test" ] || rm -rf /tmp/fresh-user-test

# 2. Verify no lingering sessions
claude-code-agent-teams list-sessions | grep -q "fresh-user" && \
  claude-code-agent-teams kill-session -t "fresh-user*"

# 3. Verify no developer context leakage
env | grep -iE "claude|vrs|alphaswarm" && exit 1
```

### Post-Run Checks

After completing a fresh environment test:

```bash
# 1. Evidence pack exists and is valid
[ -f "evidence/transcript.txt" ] || exit 1
[ -f "evidence/timing.json" ] || exit 1

# 2. Transcript shows real execution (non-zero duration)
jq '.duration_seconds > 0' evidence/timing.json || exit 1

# 3. No mock indicators in transcript
grep -qE "MOCK|SIMULATED|SKIP" evidence/transcript.txt && exit 1
```
