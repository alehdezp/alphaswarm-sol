- From within the development repository
- With CLAUDE.md loaded (500+ lines of context)
- With .planning/ knowledge available
- With developer understanding of internals
- With shortcuts and workarounds known

## Core Innovation: Complete Isolation Testing

### The Fresh Machine Protocol

```yaml
environment:
  location: /tmp/fresh-user-test/  # Completely outside repo
  context: NONE  # No CLAUDE.md, no .planning/
  knowledge: Only README and published docs
  tools: Only what installation provides

simulation:
  persona: "Developer who just found AlphaSwarm on GitHub"
  goal: "Audit my DeFi contracts for vulnerabilities"
  resources: Only public documentation
```

## Test 1: The README-Only Installation

### What User Has
```
README.md (from GitHub)
```

### What User Does
```bash
# Step 1: Install
uv tool install alphaswarm-sol

# Step 2: Verify
alphaswarm --version

# Step 3: Navigate to their project
cd ~/my-defi-project/

# Step 4: Run audit
claude
> /vrs-audit contracts/
```

### Failure Points to Document

| Step | Potential Failure | User Impact |
|------|-------------------|-------------|
| 1 | uv not installed | "What is uv?" |
| 1 | Package name wrong | Can't find package |
| 2 | Command not found | Install failed silently |
| 3 | No Foundry project | Slither fails |
| 4 | Claude Code not installed | "What is claude?" |
| 4 | Skill not found | /vrs-audit unknown |
| 4 | Missing dependencies | Cryptic error |

### Self-Improvement Action

For EACH failure point:
1. Document exact error message
2. Propose fix (better docs, better error, auto-install)
3. Implement fix
4. Re-test from Step 1

## Test 2: The Silent Mode Test

### Setup
- No CLAUDE.md in project
- No .vrs/ directory
- No prior AlphaSwarm usage
- Raw Solidity files only

### What Should Happen
```
user> /vrs-audit contracts/

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
 ALPHASWARM SECURITY AUDIT
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

[Stage 1/7] Setup & Validation
  ⚠ alphaswarm not installed. Installing...
  ✓ alphaswarm installed (v0.5.0)
  ✓ slither available
  ✓ Project type: raw (no framework detected)
  ✓ Contracts found: 5 files

... (continues successfully)
```

### What Might Actually Happen
```
user> /vrs-audit contracts/

Error: skill 'vrs:audit' not found

# OR

Error: alphaswarm command not found

# OR

Error: slither not installed

# OR

Error: No contracts found at 'contracts/'
```

### Failure Analysis Protocol

```yaml
failure: "skill 'vrs:audit' not found"
investigation:
  - Is skill file present in Claude Code skills directory?
  - Is skill properly registered?
  - Does skill name match invocation?
fix_options:
  - Auto-register skill on first use
  - Better error message with instructions
  - Package skill with tool

failure: "alphaswarm command not found"
investigation:
  - Was uv tool install run?
  - Is PATH configured correctly?
  - Did install succeed?
fix_options:
  - Skill should auto-install alphaswarm
  - Better install verification
  - Fallback to Python module invocation

failure: "slither not installed"
investigation:
  - Is slither a dependency?
  - Should it be auto-installed?
  - Is it optional?
fix_options:
  - Auto-install slither
  - Use bundled slither
  - Graceful degradation without slither
```

## Test 3: The Error Recovery Test

### Scenarios

| Scenario | What Breaks | Expected Recovery |
|----------|-------------|-------------------|
| Network timeout | Exa MCP hangs | Graceful degradation, skip context |
| Slither crash | Compiler error | Clear error, suggest fix |
| Large codebase | Graph build timeout | Progress indicator, resume |
| Invalid Solidity | Parse error | Point to bad file/line |
| No vulnerabilities | Empty findings | "Clean audit" message |
| Permission denied | Can't write .vrs/ | Ask for directory |

### Test Protocol

For each scenario:
1. Artificially create the failure condition
2. Run /vrs-audit
3. Record exact behavior
4. Evaluate: Crash? Hang? Cryptic error? Clear error?
5. Fix until behavior is graceful

### Graceful Degradation Requirements

```yaml
exa_timeout:
  behavior: "Skip Stage 3, warn user, continue"
  message: "⚠ Context research timed out. Continuing without economic context."

slither_fail:
  behavior: "Stop with clear error"
  message: |
    ✗ Slither failed to parse contracts.
    Error: {slither_error}

    Common fixes:
    - Check Solidity version in contracts matches compiler
    - Run 'forge build' first if using Foundry
    - Check for syntax errors in contracts

graph_timeout:
  behavior: "Show progress, allow cancel, offer resume"
  message: |
    ◆ Building knowledge graph... (245/500 functions)

    This is taking longer than expected.
    Press Ctrl+C to cancel, or wait to continue.
```

## Test 4: The First-Time User Journey

### Day-by-Day Simulation

**Day 1: Discovery**
```
User: "I heard about AlphaSwarm, let me try it"
Action: Find GitHub repo → Read README → Attempt install
Document: What confuses them? What's missing?
```

**Day 2: Installation**
```
User: "Okay, I have it installed... now what?"
Action: Run alphaswarm --help → Try commands
Document: Is help clear? Are commands obvious?
```

**Day 3: First Audit**
```
User: "Let me audit my contracts"
Action: Run /vrs-audit → See results
Document: Did it work? Was output understandable?
```

**Day 4: Understanding Results**
```
User: "What does this finding mean?"
Action: Read audit report → Try to understand findings
Document: Are findings actionable? Is evidence clear?
```

**Day 5: Verification**
```
User: "Is this a real bug or false positive?"
Action: Try to verify finding manually
Document: Can they trace evidence? Can they reproduce?
```

### Friction Point Categories

| Category | Example | Priority |
|----------|---------|----------|
| Blocker | Can't install | P0 - Fix immediately |
| Confusion | Unclear error | P1 - Fix before GA |
| Annoyance | Slow operation | P2 - Improve later |
| Enhancement | "Wish it could..." | P3 - Future feature |
