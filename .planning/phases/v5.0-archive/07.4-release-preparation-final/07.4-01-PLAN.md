Write `03-approach.md` (~2-3 pages):

1. **Problem with Name-Based Detection**
   - Example: `withdraw()` vs `processPayment()` - same behavior, different names
   - Reentrancy pattern requires detecting transfer-before-write regardless of names
   - Real-world case: Function renamed `emergencyWithdraw` to bypass detectors

2. **Semantic Operations**
   - Definition: Abstract operations describing code behavior, not syntax
   - Core operations from docs/reference/operations.md:

     | Category | Operation | Description |
     |----------|-----------|-------------|
     | Value | TRANSFERS_VALUE_OUT | Any value transfer (call, transfer, send) |
     | Value | READS_USER_BALANCE | Reads balance of user/address |
     | Value | WRITES_USER_BALANCE | Modifies user balance state |
     | Access | CHECKS_PERMISSION | Access control check before action |
     | Access | MODIFIES_OWNER | Changes contract owner |
     | External | CALLS_EXTERNAL | Any external call |
     | External | READS_EXTERNAL_VALUE | Reads from external contract |
     | State | MODIFIES_CRITICAL_STATE | Changes key contract state |
     | State | READS_ORACLE | Reads price/data oracle |

   - Extraction: Derived from Slither IR, not source text

3. **Behavioral Signatures**
   - Definition: Ordered sequences of semantic operations that characterize behavior
   - Vulnerable reentrancy signature: `R:bal → X:out → W:bal` (read balance, external transfer, write balance)
   - Safe CEI pattern signature: `R:bal → W:bal → X:out` (read balance, write balance, then transfer)
   - Diagram: Show side-by-side comparison

4. **Two-Tier Pattern System**
   - Tier A: Strict graph-only matches, high confidence (>= 95% precision target)
     - Example: Public function with TRANSFERS_VALUE_OUT before WRITES_USER_BALANCE, no reentrancy guard
   - Tier B: LLM-verified patterns, exploratory
     - Example: Complex business logic patterns requiring context
   - Tier C: Label-dependent patterns (semantic labels like "balance_update")
   - Why two tiers: Balances precision with coverage, admits uncertainty

5. **Evidence-First Principle**
   - Every finding links to: Source location, BSKG node ID, operation sequence, confidence score
   - No hallucination: Claims must cite evidence
