<decisions>
## Implementation Decisions

### D1: Release Experience = Three-Path Onboarding
Design three explicit onboarding paths with different time horizons:
1. **60 seconds:** "Try it now" one-liner (uvx or pipx run)
2. **5 minutes:** Guided tour (demo pack + sample contracts)
3. **30 minutes:** Full local workspace (project scaffold + context pack)

### D2: Private Distribution Channels (No Public Publish)
Ship the same version to private/local channels only:
- **Local wheel/SDist**: built artifacts in `dist/` for offline install
- **pipx**: install from local wheel or private artifact store
- **uvx**: run from local wheel for fast demos without public publishing
- **Docker**: local image build + export tar for air-gapped use
- **Private Release**: private GitHub/GitLab release or internal artifact store

### D3: "Fun" Means Visible Progress
The first run must produce a tangible artifact:
- A minimal BSKG file in `.toon`
- A human-readable "first finding" output
- A short log that shows the swarm flow (attacker/defender/verifier)

### D4: Release Ceremony Assets
The release bundle includes:
- A polished "Release Notes" page
- A one-page "Why AlphaSwarm" explainer
- A minimal "Hello Alpha" demo script

### D5: Strict GA Gate
**No release assets are published** until Phase 7.3 passes GA gate metrics.
If GA gates fail, Phase 7.4 pauses.

### D6: No Public Publishing (Top Secret)
Do not publish to public registries (PyPI, GHCR public, etc.).
All artifacts must be distributed privately or locally.

### D7: Release Integrity Checks
Add a pre-release verification gate:
- Local dry-run install from wheel/SDist
- `twine check` on dist artifacts
- Smoke installs via `pipx` and `uvx` from local artifacts

### D8: Provenance + Attestations (Private)
Attach provenance/attestations to artifacts stored in the private release
channel so origin is verifiable without public distribution.

### D9: Docs + Examples Must Execute
Every CLI example in the quickstart and tours must be executed once in a clean
environment before publishing release assets.

### D10: Private Release Bundle Spec (Single Source of Truth)
Define a single, minimal bundle that *always* ships:
- `dist/*.whl`, `dist/*.tar.gz`
- `checksums.txt`
- `MANIFEST.md` (contents + versions)
- `QUICKSTART.md` (1-page secure review guide)
- `examples/` (tiny vulnerable + safe contracts)

### D11: Secure Review Workflow (3-Step)
Document a concrete, repeatable flow for reviewers:
1. Build graph
2. Run 3 targeted queries
3. Export a concise findings bundle

### D12: Reviewer Output Contract (Concise)
Standardize a tiny output pack with:
- `findings.md` (human summary)
- `findings.toon` (evidence bundle)
- `log.txt` (first-run trace)

</decisions>

- Color-coded stages for swarm roles

- A single-line summary at the end (risk score + evidence IDs)

### Release Checklist (Fast, Deterministic)
- Build sdist + wheel, run `twine check`
- Validate local install via `pipx`/`uvx` from `dist/`
- Generate private release bundle with checksums + attestations
- Export Docker image tar for offline install if needed

alphaswarm-sol-0.5.0-private/
├── dist/
├── checksums.txt
├── MANIFEST.md
├── QUICKSTART.md
├── examples/

### One-Page Secure Review Quickstart (Condensed)

<implementation>
## Implementation Blueprint (Private Shipping, 2026-Ready)

### Private Distribution Topology (Pick 1 Primary, 1 Backup)
1. **Sealed Bundle (Air-gap)**: zip of the private release folder (wheel + checksums + quickstart + examples)
2. **Private Package Index**: GitHub Packages, GitLab Package Registry, Artifactory, or CodeArtifact
3. **Private Release**: GitHub/GitLab private release with checksums + attestations

### Index Hardening (Avoid Dependency Confusion)
- Use a single private index for installs and **disable fallback** to public PyPI.
- For uv/pip installs: set `--index-url` + `--no-index` to prevent cross-index confusion.
- If using GitLab Package Registry, disable package forwarding to public PyPI.

### Install Modes (Private, Fast)
- **pipx local wheel**: `pipx install dist/alphaswarm_sol-*.whl`
- **uvx ephemeral run**: `uvx alphaswarm --version` (runs tool without install)
- **Docker air-gap**: `docker load -i alphaswarm-sol-0.5.0.tar`

### Multi-Orchestration Profiles (One File, Many Adapters)
Define profiles so the CLI can flip orchestration backends without code changes:
```
~/.vrs/profiles/
  local.yaml        # default: claude_code adapter
  langgraph.yaml    # LangGraph adapter with JSONL checkpoints
  agents-sdk.yaml   # Agents SDK adapter for production
```
Each profile specifies: adapter name, evidence preservation mode, trace mode,
and optional tool gating policy.

### Reviewer Output Contract (Concrete)
Ensure the secure review produces **one folder**:
```
reports/<timestamp>/
├── findings.md
├── findings.toon
└── log.txt
```
This is the only artifact reviewers need to read and archive.

### Release Integrity (Private)
- `twine check` on every artifact
- Verify `checksums.txt` during install (air-gap)
- Attach attestations for private releases if supported by the host

### First-Run Determinism (No Surprises)
- The demo contract must always yield at least 1 known finding
- The 3 queries in QUICKSTART must always return results
- The summary line must always include evidence IDs
