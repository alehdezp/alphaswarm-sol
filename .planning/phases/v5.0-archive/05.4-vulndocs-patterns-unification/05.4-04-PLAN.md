  <files>
    src/true_vkg/vulndocs/scaffold.py
  </files>
  <action>
    Create scaffold.py with utilities for creating new vulndocs from templates.

    1. scaffold_category(root: Path, category_id: str, name: str) -> Path:
       - Create category folder at root/{category_id}/
       - Copy .meta/templates/category.yaml as overview.md
       - Replace placeholders with actual values
       - Return path to created folder

    2. scaffold_vulnerability(
         root: Path,
         category: str,
         subcategory: str,
         severity: str = "medium"
       ) -> Path:
       - Create subcategory folder at root/{category}/{subcategory}/
       - Copy all files from .meta/templates/subcategory/
       - Replace placeholders:
         - {{category}} -> category
         - {{subcategory}} -> subcategory
         - {{severity}} -> severity
         - {{vulndoc}} -> f"{category}/{subcategory}"
         - {{date}} -> current date (YYYY-MM-DD)
       - Create empty patterns/ subfolder
       - Return path to created folder

    3. scaffold_pattern(
         vuln_path: Path,
         pattern_id: str,
         name: str
       ) -> Path:
       - Create pattern YAML in vuln_path/patterns/
       - Copy .meta/templates/pattern.yaml
       - Replace placeholders:
         - {{pattern_id}} -> pattern_id
         - {{name}} -> name
         - {{vulndoc}} -> parent vulnerability path
       - Return path to created file

    4. TemplateRenderer class:
       - Load template file
       - Replace placeholders using jinja2 or simple string replace
       - Support custom variables dict

    5. validate_scaffold_inputs(category: str, subcategory: str) -> list[str]:
       - Check naming conventions (lowercase, hyphens)
       - Check for existing folder (error if exists)
       - Return list of validation errors
  </action>

  <files>
    src/true_vkg/vulndocs/cli.py
  </files>
  <action>
    Create cli.py with Click command group for vulndocs operations.

    1. vulndocs_app = typer.Typer() command group

    2. @vulndocs_app.command("validate")
       def validate(
           path: str = "vulndocs",
           strict: bool = False,
           json_output: bool = False
       ):
       - Call validate_framework(Path(path))
       - Display results with rich formatting:
         - Table of vulnerabilities with level icons
         - Errors in red, warnings in yellow, suggestions in cyan
         - Summary counts at end
       - Exit code 1 if errors (or warnings with --strict)
       - --json outputs JSON for CI parsing

    3. @vulndocs_app.command("scaffold")
       def scaffold(
           category: str,
           subcategory: str,
           severity: str = "medium"
       ):
       - Validate inputs
       - Call scaffold_vulnerability()
       - Display created files with tree structure
       - Suggest next steps

    4. @vulndocs_app.command("info")
       def info(path: str = "vulndocs"):
       - Show summary statistics:
         - Number of categories
         - Number of vulnerabilities by level
         - Number of patterns
         - Coverage gaps (categories without patterns)
       - Use rich tables for display

    5. @vulndocs_app.command("list")
       def list_vulns(
           category: str | None = None,
           level: str | None = None
       ):
       - List all vulnerabilities
       - Filter by category if provided
       - Filter by validation level if provided
       - Display as rich table

    Use rich for all CLI output (tables, colors, tree structures).
