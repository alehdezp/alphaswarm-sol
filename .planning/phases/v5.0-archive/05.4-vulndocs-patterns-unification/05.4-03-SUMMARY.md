## Decisions Made

### Validation Philosophy
- **Only index.yaml is strictly required** - All other files are progressive enhancements
- **Validation warns but doesn't block** - Errors only for missing/invalid index.yaml
- **Progressive levels unlock features** - More complete documentation enables better tooling

### Discovery Strategy
- **Only discover folders WITH index.yaml** - Explicit opt-in approach
- **Supports 2-3 level nesting** - category/subcategory or category/subcategory/variant
- **Skips special folders** - .meta/templates, .meta/instructions, and hidden folders ignored

### Validation Levels
1. **MINIMAL**: Just valid index.yaml with required fields (id, category, subcategory, severity, vulndoc)
2. **STANDARD**: index.yaml + at least one .md file (warns if overview.md missing)
3. **COMPLETE**: All recommended files present (overview.md, detection.md, verification.md)
4. **EXCELLENT**: Complete + patterns/ folder with test_coverage in index.yaml

### Research Suggestions
- Empty Phase 7 fields (semantic_triggers, vql_queries, graph_patterns, reasoning_template) trigger suggestions
- If 3+ fields empty, suggests comprehensive Exa research command
- Suggestions reference specific commands: `uv run alphaswarm vulndocs research '{topic}'`
