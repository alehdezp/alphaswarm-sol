## Decisions Made

**1. Research depth levels:**
- Quick (30min): Survey existing knowledge with 10-15 sources
- Standard (1-2hr): Comprehensive investigation with 20-30 sources
- Deep (2-4hr): Exhaustive analysis with 40+ sources including academic papers
- Rationale: User control over time investment vs. thoroughness

**2. Similarity scoring dimensions:**
- Semantic triggers overlap (concept-based, not string matching)
- VQL query similarity (semantic alignment)
- Graph pattern similarity (structural patterns)
- Reasoning template similarity (detection logic)
- Description similarity (lower weight, text-based)
- Rationale: Multi-dimensional analysis prevents false merges

**3. Test generation strategy:**
- Four test types: vulnerable (basic pattern), safe (mitigated), adversarial (obfuscated), edge (boundaries)
- Tests use semantic operations, not function names
- Both Solidity contracts and Python test files generated
- Rationale: Comprehensive validation across all three quality focuses

**4. Model tier assignments:**
- Research, merge-findings, generate-tests: Sonnet 4.5 (cost-effective for structured work)
- Refine: Opus 4.5 (quality-critical pattern improvement)
- Rationale: Optimize cost while maintaining quality where it matters
