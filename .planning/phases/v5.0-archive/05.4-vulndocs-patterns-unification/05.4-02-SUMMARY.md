## Decisions Made

**1. Extended schema.py rather than creating new file**
- **Rationale:** Keeps all schema definitions in one place, clear separation via comments
- **Tradeoff:** Larger file but better discoverability
- **Impact:** Future phases import from single schema module

**2. Validated semantic_triggers against VALID_SEMANTIC_OPERATIONS**
- **Rationale:** Prevents typos and ensures consistency with operations.md
- **Implementation:** field_validator checks each trigger against list of 20 valid operations
- **Impact:** Type-safe Phase 7 test generation

**3. Required vulndoc field in PatternRef**
- **Rationale:** Enforces bi-directional linkage between patterns and vulnerability folders
- **Implementation:** Required field, validated against format
- **Impact:** Patterns always link back to their vulnerability context

**4. Used pydantic 2.x patterns**
- **Rationale:** Modern API with field_validator and model_validator decorators
- **Implementation:** @field_validator for semantic_triggers, @model_validator for vulndoc path
- **Impact:** Clean, declarative validation logic
