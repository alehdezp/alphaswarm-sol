## Decisions Made

**1. CI Validation Strategy: Warn but Don't Block**
- Errors (missing required fields) → fail build
- Warnings (incomplete optional fields) → GitHub annotations
- Rationale: Allows draft entries to be committed while enforcing minimum quality

**2. Skill Invocation Model Explanation**
- Documented that skills are prompt templates, not separate executables
- Agents invoke CLI commands via Bash tool based on skill instructions
- Rationale: Clarifies mental model for users and agents

**3. Graph-First Enforcement in CLAUDE.md**
- Added explicit constraints requiring BSKG queries over manual code reading
- Emphasizes semantic operations in detection strategies
- Rationale: Ensures agents leverage framework philosophy
