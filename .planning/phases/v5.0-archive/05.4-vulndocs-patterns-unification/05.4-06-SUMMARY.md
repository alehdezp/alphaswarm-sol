**Refinement constraints:**
- Never reduce recall below 0.70 (vulnerability detection recall is critical)
- Always test refinements before committing (verify improvement)
- Mark as draft if precision < 0.70 (too many false positives)

**Testing approach:**
- BSKG queries via Bash tool for consistency with production detection
- Ground truth validation from DVDeFi challenges, Code4rena audits, labeled tests
- Manual verification prompt when ground truth doesn't exist

**Quality gates:**
- draft: precision < 0.70 OR recall < 0.50
- ready: precision ≥ 0.70 AND recall ≥ 0.50
- excellent: precision ≥ 0.90 AND recall ≥ 0.85
