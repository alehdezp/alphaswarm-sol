<task type="auto">
  <name>Task 2: Create validation module with progressive levels</name>
  <files>
    src/true_vkg/vulndocs/validation.py
  </files>
  <action>
    Create validation.py with progressive validation logic per 05.4-CONTEXT.md.

    1. ValidationResult dataclass:
       - path: Path
       - level: ValidationLevel | None  # None if failed minimum
       - errors: list[str]  # Blocking issues
       - warnings: list[str]  # Quality issues
       - suggestions: list[str]  # Improvement ideas

    2. FrameworkValidationResult dataclass:
       - vulnerabilities: list[ValidationResult]
       - has_errors: bool
       - has_warnings: bool
       - summary: dict with counts per level

    3. validate_vulnerability(path: Path) -> ValidationResult:
       Progressive validation:

       a) REQUIRED: index.yaml must exist with minimum fields
          - Parse with VulnDocIndex
          - Required: id, category, subcategory, severity, vulndoc
          - If missing/invalid: level=None, add error

       b) MINIMAL level (index.yaml valid):
          - Just the required fields present
          - No warnings needed

       c) STANDARD level (index.yaml + at least 1 md file):
          - Check for any .md file
          - Warn if missing recommended files

       d) COMPLETE level (all recommended files):
          - Check for overview.md, detection.md, verification.md
          - Warn if missing exploits.md

       e) EXCELLENT level (patterns with test coverage):
          - Check patterns/ folder exists with .yaml files
          - Suggest running tests if test_coverage empty

    4. validate_framework(root: Path) -> FrameworkValidationResult:
       - Discover all vulnerabilities
       - Validate each one
       - Aggregate results
       - Add suggestions for missing categories if known gaps

    5. suggest_research(result: ValidationResult) -> list[str]:
       - If Phase 7 fields empty, suggest Exa research
       - If no patterns, suggest pattern creation
       - Format: "Consider: uv run alphaswarm vulndocs research {topic}"

    Key principle: Errors for missing required fields, warnings for quality,
    suggestions for improvements. Never block CI on warnings.
  </action>
  <verify>
    python -c "

from src.true_vkg.vulndocs.validation import validate_framework, ValidationLevel

# Test with empty/nonexistent vulndocs
root = Path('vulndocs')

print(f'Validation result: {len(result.vulnerabilities)} vulns, {result.has_errors} errors')
"
  </verify>
  <done>

    - ValidationResult captures errors, warnings, suggestions
    - validate_vulnerability returns level based on content presence
    - validate_framework aggregates results for entire folder
    - Suggestions reference Exa research for gaps
  </done>
