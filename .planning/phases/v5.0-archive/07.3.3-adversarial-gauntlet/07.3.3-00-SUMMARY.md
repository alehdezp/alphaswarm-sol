# Phase 07.3.3 Plan 00: Gauntlet Specification + Scoring + Runner Summary

**One-liner:** Binary pass/fail gauntlet framework with 2x FP/FN penalties, claude-code-agent-teams-isolated runner, and evidence pack integration.

## What Was Built

### 1. Gauntlet Specification (07.3.3-GAUNTLET-SPEC.md)

Comprehensive specification for adversarial gauntlet challenges:

- **Binary scoring:** Pass or fail, no partial credit
- **Explicit ground truth:** Pre-defined correct answers per contract
- **Evidence requirements:** Proof tokens + evidence packs mandatory
- **No retry without fix:** Must address root cause before re-running

**Defined Suites:**
| Suite | Cases | Pass Threshold |
|-------|-------|----------------|
| gauntlet-reentrancy | 10 | 100% accuracy |
| gauntlet-access-oracle | 10 | 100% accuracy |
| gauntlet-mutation | 30 | 75% detection, 0 FP |

### 2. Scoring Rules (configs/gauntlet_scoring.yaml)

Machine-readable scoring configuration:

```yaml
scoring:
  correct_vulnerable: +1   # True positive
  correct_safe: +1         # True negative
  false_positive: -2       # Penalized 2x
  false_negative: -2       # Penalized 2x
```

**Gate Integration:**
- G4 (Mutation Robustness): Enforced by gauntlet-mutation
- G6 (Regression Baseline): No score regression allowed

**Anti-Perfect Metrics Rule:**
- 100% precision AND recall triggers SUSPICIOUS flag
- Prevents overfitting and silent regressions

### 3. Gauntlet Runner (scripts/e2e/run_gauntlet_suite.sh)

Executable runner with:

- **claude-code-agent-teams-controlled runs:** Session isolation per suite
- **Evidence collection:** Automatic manifest generation
- **--dry-run mode:** Lists cases without execution
- **--fail-fast:** Stops on first binary failure
- **--all:** Runs all registered suites

**Usage:**
```bash
./scripts/e2e/run_gauntlet_suite.sh --suite gauntlet-reentrancy
./scripts/e2e/run_gauntlet_suite.sh --all --dry-run
```
