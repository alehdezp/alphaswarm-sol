Create `web_fetcher.py` for discovering and fetching protocol documentation.

Per 03-CONTEXT.md: "Auto-discover docs (README, docs/, whitepaper links) + user can override/add sources"

```python
@dataclass
class FetchedDocument:
    """A fetched document with metadata."""
    content: str
    source_url: str
    source_type: str  # readme, whitepaper, docs, audit, forum
    source_tier: int  # 1=official, 2=audit, 3=community
    fetch_time: str
    content_hash: str  # For change detection

class WebFetcher:
    """Discover and fetch protocol documentation.

    Supports:
    - Local file discovery (README.md, docs/)
    - GitHub repository docs
    - Whitepaper links (PDFs converted to text)
    - External URLs provided by user

    Per 03-CONTEXT.md source tiers:
    - Tier 1: Official docs (README, docs/, whitepaper)
    - Tier 2: Audit reports
    - Tier 3: Community/forums
    """

    def __init__(self, project_path: Path):
        self.project_path = Path(project_path)

    async def discover_docs(self) -> List[str]:
        """Auto-discover documentation sources.

        Looks for:
        - README.md, README.txt
        - docs/ directory
        - whitepaper links in README
        - SECURITY.md
        """

    async def fetch_local(self, path: Path) -> FetchedDocument:
        """Fetch local file content."""

    async def fetch_url(self, url: str, source_tier: int = 3) -> Optional[FetchedDocument]:
        """Fetch remote URL content using httpx."""

    async def fetch_all(
        self,
        additional_sources: Optional[List[str]] = None
    ) -> List[FetchedDocument]:
        """Fetch all discovered + additional sources."""

    def _extract_whitepaper_links(self, readme_content: str) -> List[str]:
        """Extract whitepaper/docs links from README."""

    def _classify_source(self, path_or_url: str) -> Tuple[str, int]:
        """Classify source type and tier."""
```

Implementation details:

1. **Local discovery**:
   - Check for README.md, README.txt, README.rst
   - Check for docs/, documentation/, doc/ directories
   - Parse README for whitepaper/docs links (regex for common patterns)

2. **URL fetching**:
   - Use httpx (already in deps) for async HTTP requests
   - Handle timeouts gracefully (10s default)
   - Convert HTML to markdown if needed (basic stripping)
   - For PDFs, note limitation (text extraction not implemented, return placeholder)

3. **Source classification**:
   - README, docs/ -> Tier 1 (official)
   - audit-report, security-review -> Tier 2
   - forum, github-issues -> Tier 3

4. **Content hashing**:
   - SHA256 of content for change detection (per 03-CONTEXT.md incremental updates)

```python
@dataclass
class DocParseResult:
    """Result of document parsing."""
    roles: List[Role]
    assumptions: List[Assumption]
    invariants: List[Invariant]
    incentives: List[str]
    value_flows: List[ValueFlow]
    governance: Dict[str, Any]
    tokenomics_summary: str
    security_claims: List[str]  # Hypotheses to test
    questions: List[str]  # Gaps to investigate
    source: str

class DocParser:
    """LLM-driven document parser for protocol context extraction.

    Uses existing LLMClient infrastructure for structured extraction.
    Per 03-CONTEXT.md decisions:
    - Cross-validate doc claims against code
    - Flag conflicts between docs and code
    - Extract security claims as hypotheses to test
    - Generate questions about gaps
    """

    def __init__(self, llm_client: Optional[LLMClient] = None):
        self.llm = llm_client or LLMClient()

    async def parse(
        self,
        document: FetchedDocument,
        code_context: Optional[Dict[str, Any]] = None
    ) -> DocParseResult:
        """Parse document and extract protocol context."""

    async def extract_roles(self, content: str, source_tier: int) -> List[Role]:
        """Extract roles and capabilities from document.

        Prompt template:
        - List all roles mentioned (admin, owner, user, keeper, etc.)
        - For each role, list capabilities (what they can do)
        - List trust assumptions about each role
        - Mark confidence based on explicitness
        """

    async def extract_assumptions(self, content: str, source_tier: int) -> List[Assumption]:
        """Extract protocol assumptions from document.

        Per 03-CONTEXT.md: Active inference of unstated assumptions
        """

    async def extract_invariants(self, content: str) -> List[Invariant]:
        """Extract invariants (both formal and natural language)."""

    async def extract_economics(self, content: str) -> Tuple[List[str], List[ValueFlow], str]:
        """Extract incentives, value flows, and tokenomics summary."""

    async def extract_governance(self, content: str) -> Dict[str, Any]:
        """Extract governance details.

        Per 03-CONTEXT.md: voting thresholds, timelock durations, upgrade procedures
        """

    async def cross_validate(
        self,
        doc_result: DocParseResult,
        code_context: Dict[str, Any]
    ) -> Tuple[DocParseResult, List[str]]:
        """Cross-validate doc claims against code.

        Per 03-CONTEXT.md: Flag conflicts between docs and code explicitly
        Returns updated result and list of conflicts/warnings
        """

    async def identify_gaps(self, content: str) -> List[str]:
        """Generate questions about security-critical gaps.

        Per 03-CONTEXT.md: Generate questions about gaps for investigation
        """
```

LLM Prompt Templates (use analyze_json for structured output):

1. **Role extraction prompt**:
```
Analyze this protocol documentation and extract all roles:

{content}

Return JSON:
{
    "roles": [
        {
            "name": "role name",
            "capabilities": ["cap1", "cap2"],
            "trust_assumptions": ["assumption1"],
            "confidence": "certain|inferred|unknown"
        }
    ]
}
```

2. **Assumption extraction prompt**:
```
Extract protocol assumptions from this documentation.
Include both explicit and inferred assumptions.

{content}

Return JSON:
{
    "assumptions": [
        {
            "description": "assumption text",
            "category": "price|time|trust|economic|other",
            "explicit": true/false
        }
    ]
}
```

3. **Gap identification prompt**:
```
Identify security-critical gaps in this documentation.
What important security information is missing?

{content}

Return JSON:
{
    "questions": [
        "Question about missing information 1",
        "Question about missing information 2"
    ]
}
```

Use LLMClient.analyze_json() for all structured extraction.
Set confidence based on source_tier and explicitness.
