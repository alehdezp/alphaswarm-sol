## Decisions Made

1. **A/B Test Architecture:** Uses claude-code-controller to launch isolated Claude Code sessions, runs /vrs-audit with specified mode, captures transcript and metrics
2. **Metrics Aggregation:** Uses dataclasses (WorktreeMetrics, AggregateMetrics, ModeComparison) for type-safe metrics handling
3. **Report Format:** Markdown with executive summary table, FP/FN analysis, performance overhead section, per-worktree tables, and raw JSON blocks
4. **Solo Mode Validation:** Checks 5 stages - graph exists (Stage 2), pattern-matches (Stage 4), agent-investigations (Stage 5), verdicts ABSENT (Stage 6), report (Stage 7)
5. **Single Pattern Validation:** Verifies only specified pattern returned, no extraneous patterns, handles empty results gracefully
