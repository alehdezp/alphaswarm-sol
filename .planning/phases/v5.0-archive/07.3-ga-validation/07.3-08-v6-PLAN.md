## Execution Script

**File:** `scripts/run_dvd_naive_receiver_test.sh`

```bash
#!/bin/bash
# Run Naive Receiver audit test

set -e

SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
MAIN_REPO="$(dirname "$SCRIPT_DIR")"
WORKTREE_NAME="dvd-naive-receiver"
WORKTREE_PATH="/tmp/vrs-worktrees/$WORKTREE_NAME"
TIMEOUT=600

echo "========================================"
echo " DVD NAIVE RECEIVER TEST"
echo "========================================"

# Step 1: Create worktree
echo "[1/6] Creating worktree from fixture..."
"$SCRIPT_DIR/manage_worktrees.sh" create-from-fixture "$WORKTREE_NAME" "dvd/naive-receiver"

# Step 2: Launch Claude Code
echo "[2/6] Launching Claude Code session..."
PANE_ID=$(claude-code-controller launch "cd $WORKTREE_PATH && claude --dangerously-skip-permissions" 2>&1 | grep -oP 'pane_id:\s*\K\S+' || echo "")

if [ -z "$PANE_ID" ]; then
    echo "ERROR: Failed to launch session"
    exit 1
fi

sleep 15

# Step 3: Send audit command
echo "[3/6] Sending /vrs-audit command..."
claude-code-controller send "/vrs-audit contracts/ --mode swarm" --pane="$PANE_ID"

# Step 4: Wait for completion
echo "[4/6] Waiting for completion (timeout: ${TIMEOUT}s)..."
START_TIME=$(date +%s)
claude-code-controller wait_idle --pane="$PANE_ID" --idle-time=60 --timeout="$TIMEOUT" || true
END_TIME=$(date +%s)
DURATION=$((END_TIME - START_TIME))

# Step 5: Capture and validate
echo "[5/6] Capturing output..."
claude-code-controller capture --pane="$PANE_ID" --lines=5000 > "$WORKTREE_PATH/audit-transcript.txt"

echo "[6/6] Validating..."
"$SCRIPT_DIR/manage_worktrees.sh" validate "$WORKTREE_NAME" all

# Compare to ground truth
uv run python "$SCRIPT_DIR/compare_findings.py" \
    --worktree "$WORKTREE_PATH" \
    --output "$WORKTREE_PATH/metrics.json"

# Display results
echo ""
echo "========================================"
echo " RESULTS"
echo "========================================"
echo "Duration: ${DURATION}s"
cat "$WORKTREE_PATH/metrics.json"

# Cleanup
claude-code-controller kill --pane="$PANE_ID" 2>/dev/null || true

# Check success
RECALL=$(jq -r '.recall' "$WORKTREE_PATH/metrics.json")
if [ "$RECALL" = "1" ] || [ "$RECALL" = "1.0" ]; then
    echo ""
    echo "SUCCESS: All vulnerabilities detected (recall = 1.0)"
    exit 0
else
    echo ""
    echo "FAILURE: Missed vulnerabilities (recall = $RECALL)"
    exit 1
fi
```

## Notes

1. **Ground Truth Integrity:** The vulnerability description comes directly from Damn Vulnerable DeFi, not from our tool's detection. This ensures we're measuring real detection capability.

2. **Expected Detection Logic:** AlphaSwarm should detect this as an access-control vulnerability because:
   - `flashLoan` is public/external
   - It allows specifying an arbitrary `receiver` address
   - No check that `msg.sender == receiver` or similar authorization
   - Missing `has_access_gate` for receiver validation

3. **Pattern Match:** The most likely matching pattern is `unauthorized-action-on-behalf` or `weak-access-control`. The semantic operation should detect `CALLS_EXTERNAL` without proper authorization check.

4. **Multi-Agent Debate:** In swarm mode, the attacker agent should construct the exploit path (call flashLoan 10 times to drain receiver's 10 ETH), the defender should fail to find guards, and the verifier should confirm as critical.

5. **Flexible Workflow Reference:** See `FLEXIBLE-WORKFLOW-DESIGN.md` for all available stage control, pattern control, and workflow control commands.
