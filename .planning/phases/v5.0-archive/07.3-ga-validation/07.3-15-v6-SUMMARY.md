## Decisions Made

1. **Required vs Optional checks:** Metrics, baseline, core docs, install, tests, fixtures, CLI are required. Type checks, linting, guides, workflow features are optional (warnings only).

2. **Exit code semantics:** Script exits 0 if required checks pass (normal mode) or all checks pass (--strict mode).

3. **Warning-based continuation:** The run_ga_gate.sh wrapper continues even if metrics/baseline are missing, allowing incremental validation.

4. **Flexible workflow checks:** Included in GA gate but marked as optional since these features may not be complete for initial GA.
