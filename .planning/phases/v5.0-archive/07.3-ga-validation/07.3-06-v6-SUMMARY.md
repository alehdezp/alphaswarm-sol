## One-Liner

Stage 6 debate validation with claude-code-controller orchestration and verdict structure verification.

## Summary

Created infrastructure for validating the multi-agent debate stage of the /vrs-audit pipeline. The test script creates a worktree from the foundry-vault fixture, builds the BSKG graph, creates mock agent investigations, and launches Claude Code via claude-code-controller to execute a full Attacker/Defender/Verifier debate protocol. The validation script verifies that verdicts.json contains expected outcomes (CONFIRMED for both reentrancy and access control vulnerabilities).

## Artifacts Created

### Test Script: `scripts/test_stage_debate.sh`

**Size:** 8.1 KB, executable

**Features:**
- Creates worktree from foundry-vault fixture
- Builds BSKG graph as prerequisite
- Creates agent-investigations.json (Stage 5 mock output)
- Launches Claude Code via claude-code-controller
- Sends detailed debate protocol prompt
- Waits for completion with configurable timeout (default 900s)
- Captures transcript for audit trail
- Validates verdicts using Python script
- Checks transcript for debate phase evidence

**Usage:**
```bash
./scripts/test_stage_debate.sh
TIMEOUT=1200 ./scripts/test_stage_debate.sh
```

### Validation Script: `scripts/validate_verdicts.py`

**Size:** 12 KB, executable

**Features:**
- Loads verdicts from JSON (supports multiple formats)
- Validates verdict exists for each expected finding
- Validates verdict outcome (CONFIRMED/DISPUTED/REJECTED)
- Validates attacker claim presence with exploit path
- Validates defender claim presence (guard search)
- Validates verifier verdict with decision and reasoning
- Outputs structured JSON results for CI integration

**Usage:**
```bash
uv run python scripts/validate_verdicts.py --verdicts .vrs/findings/verdicts.json
uv run python scripts/validate_verdicts.py --verdicts verdicts.json --expected expected/verdict-checks.json --output results.json
```

### Expected Checks: `tests/fixtures/foundry-vault/expected/verdict-checks.json`

**Size:** 929 bytes

**Content:**
- 2 expected verdicts: reentrancy-classic@withdraw and weak-access-control@setOwner
- Both expected outcome: CONFIRMED
- Both expected severity: critical
- Debate requirements: exploit_path, damage_potential, guard search, verdict, reasoning

## Debate Protocol

The test validates the following multi-agent debate flow:

```
Finding → ATTACKER → DEFENDER → VERIFIER → Verdict
         (exploit)   (guards)   (arbitrate)
```

| Finding | Attacker | Defender | Verifier Expected |
|---------|----------|----------|-------------------|
| Reentrancy in withdraw() | Exploit path via fallback | No guard found | CONFIRMED CRITICAL |
| Access control in setOwner() | Ownership takeover | No modifier found | CONFIRMED CRITICAL |
