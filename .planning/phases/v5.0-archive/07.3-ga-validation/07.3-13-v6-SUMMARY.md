## One-liner

Created metrics aggregation scripts to compute precision/recall/F1 from Wave 2 test results with breakdown by vulnerability type and ground truth source.

## Summary

This plan created the infrastructure for aggregating metrics from all Wave 2 tests (07.3-08 through 07.3-12). The scripts are designed to:

1. **Load test results** from `.vrs/testing/results/` directory
2. **Calculate overall metrics**: precision, recall, F1 score
3. **Break down by vulnerability type**: Track TP/FP/FN per vulnerability category
4. **Break down by ground truth source**: Compare performance across DVD, SmartBugs, manual annotation
5. **Check GA gate**: Pass if precision >= 70%, recall >= 60%, F1 >= 65%

## Script Capabilities

### aggregate_metrics.py

- **Input**: JSON test result files from `.vrs/testing/results/`
- **Output formats**: JSON metrics, Markdown report
- **Metrics computed**:
  - Total TP/FP/FN
  - Overall precision, recall, F1
  - By vulnerability type breakdown
  - By ground truth source breakdown
  - Individual test results table
- **GA gate check**: Exit code 0 if passed, 1 if failed

### run_metrics_aggregation.sh

- **Pre-checks**: Validates results directory exists
- **Execution**: Runs aggregation via `uv run python`
- **Output**: Writes to `.vrs/ga-metrics/aggregated-metrics.json` and `.vrs/ga-metrics/ga-metrics-report.md`
- **Summary**: Prints summary table with PASS/FAIL status

## Usage

```bash
# Run aggregation (requires Wave 2 test results)
./scripts/run_metrics_aggregation.sh

# Or run Python script directly
uv run python scripts/aggregate_metrics.py \
    --results-dir .vrs/testing/results/ \
    --output .vrs/ga-metrics/aggregated-metrics.json \
    --report .vrs/ga-metrics/ga-metrics-report.md
```
