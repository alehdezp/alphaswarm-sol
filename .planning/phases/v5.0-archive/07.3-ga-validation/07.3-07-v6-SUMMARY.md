## One-liner

Full /vrs-audit E2E test with 7-stage validation plus 6 flexible workflow scenarios for mode/stage/tier filtering.

## What Was Done

### Task 1: Full Pipeline Test Script (scripts/test_full_pipeline.sh)

Created comprehensive end-to-end test for the complete /vrs-audit pipeline:

**Features:**
- Creates isolated worktree from foundry-vault fixture
- Launches Claude Code via claude-code-controller with --dangerously-skip-permissions
- Sends /vrs-audit contracts/ --mode swarm|solo
- Monitors all 7 pipeline stages for completion
- Validates all expected output artifacts exist
- Compares findings to external ground truth
- Calculates precision, recall, F1 metrics
- Generates comprehensive validation report

**Usage:**
```bash
./scripts/test_full_pipeline.sh                  # Default: swarm mode, 1800s timeout
./scripts/test_full_pipeline.sh --mode solo      # Fast mode, no debate
./scripts/test_full_pipeline.sh --timeout 2400   # Custom timeout
```

**Output Validation:**
| Stage | Output | Required |
|-------|--------|----------|
| 2 | .vrs/graphs/*.toon | YES |
| 3 | .vrs/context/protocol-pack.yaml | No |
| 4 | .vrs/findings/pattern-matches.json | YES |
| 5 | .vrs/findings/agent-investigations.json | No |
| 6 | .vrs/findings/verdicts.json | swarm only |
| 7 | .vrs/reports/audit-report.md | YES |

### Task 2: Flexible Workflow Test Script (scripts/test_flexible_workflows.sh)

Created test harness for 6 flexible workflow scenarios:

| Scenario | Command | Validation |
|----------|---------|------------|
| 1. Solo mode | --mode solo | Report exists, no verdicts |
| 2. Skip stages | --skip-stages 3,6 | No context, no verdicts |
| 3. Only stages | --only-stages 2,4 | Graph + patterns only |
| 4. Tier filter | --tier A | All patterns Tier A |
| 5. Stop after | --stop-after 4 | Stops at patterns |
| 6. Category filter | --category reentrancy | Only reentrancy patterns |

**Usage:**
```bash
./scripts/test_flexible_workflows.sh             # Run all scenarios
./scripts/test_flexible_workflows.sh --scenario 1  # Run only solo mode
./scripts/test_flexible_workflows.sh --timeout 300 # Custom per-scenario timeout
```

### Task 3: E2E Validation Expectations (tests/fixtures/foundry-vault/expected/e2e-checks.json)

Created comprehensive E2E validation specification:

**Pipeline Stage Definitions:**
- All 7 stages with expected outputs
- Required vs optional output specifications
- Mode-specific requirements (swarm vs solo)

**Expected Findings:**
- VAULT-001: reentrancy in withdraw (critical)
- VAULT-002: access-control in setOwner (critical)

**Expected NOT Flagged (False Positive Checks):**
- safeWithdraw (has nonReentrant)
- emergencyWithdraw (has onlyOwner)
- transferOwnership (has onlyOwner)
- deposit (no external calls)
- getBalance (view function)

**Metrics Thresholds:**
- precision_min: 0.70
- recall_min: 0.60
- f1_min: 0.65

### Task 4: Enhanced compare_findings.py

Added safe function false positive detection:

**New Features:**
- SafeFunction dataclass for expected-safe functions
- load_safe_functions() to parse ground-truth.yaml
- check_false_positive_on_safe() to detect FP on safe functions
- Enhanced calculate_metrics() with safe function tracking
- FP list now shows "(safe: functionName)" for safe violations
- Result includes safe_functions_count

**Usage:**
```bash
uv run python scripts/compare_findings.py --worktree /path/to/worktree --verbose
```
