## CONCEPT: AGENTIC SELF-TESTING

Agentic self-testing is a novel approach where:

1. **Controller Agent** (your Claude Code session) orchestrates tests
2. **Subject Agent** (Claude Code in claude-code-agent-teams) executes the actual workflows
3. **Evaluator** (your session) compares output to ground truth

This is the correct way to test an agentic system - you can't test agents with Python simulations.

```
┌────────────────────────────────────────────────────────────────────────┐
│                                                                        │
│  YOU (Human) ───▶ Controller Claude Code ───▶ /vrs-self-test           │
│                           │                                            │
│                           │ orchestrates                               │
│                           ▼                                            │
│                   ┌───────────────┐                                    │
│                   │ claude-code-agent-teams session  │                                    │
│                   │ ┌───────────┐ │                                    │
│                   │ │  Subject  │ │ ◀─── ISOLATED from controller      │
│                   │ │  Claude   │ │                                    │
│                   │ │  Code     │ │                                    │
│                   │ └───────────┘ │                                    │
│                   └───────┬───────┘                                    │
│                           │                                            │
│                           │ real execution                             │
│                           ▼                                            │
│                   ┌───────────────┐                                    │
│                   │ Anthropic API │                                    │
│                   │ (real calls)  │                                    │
│                   └───────────────┘                                    │
│                                                                        │
└────────────────────────────────────────────────────────────────────────┘
```

## COMPONENT 2: Test Execution Pattern

### Pattern 1: Single Workflow Test

```python
# Pseudocode for what /vrs-workflow-test should do

async def test_workflow(workflow: str, contract: str):
    # 1. Create isolated session
    session = await create_claude-code-agent-teams_session("vrs-test-subject")

    # 2. Launch Claude Code
    await execute_in_session(session, "claude --dangerously-skip-permissions")
    await wait_for_prompt(session, timeout=30)

    # 3. Send workflow command
    start_time = time.time()
    await send_keys(session, f"{workflow} {contract}")

    # 4. Wait for completion
    await wait_for_completion(session,
        success_markers=["Complete", "Audit finished"],
        failure_markers=["Error:", "Traceback"],
        timeout=180
    )
    duration = time.time() - start_time

    # 5. Capture output
    transcript = await capture_pane(session, lines=1000)

    # 6. Extract findings from output
    findings = parse_findings_from_transcript(transcript)

    # 7. Cleanup
    await kill_session(session)

    return TestResult(
        workflow=workflow,
        contract=contract,
        duration_seconds=duration,  # REAL duration
        transcript=transcript,
        findings=findings
    )
```

## COMPONENT 4: Metrics Calculation (REAL)

### True Positive / False Positive Calculation

```python
def calculate_metrics(detected_findings: List[Finding],
                     ground_truth: GroundTruth) -> Metrics:
    """Calculate metrics by comparing detected findings to ground truth."""

    true_positives = []
    false_positives = []
    false_negatives = []

    # For each detected finding, check if it matches ground truth
    for detected in detected_findings:
        matched = False
        for gt_finding in ground_truth.findings:
            if findings_match(detected, gt_finding):
                true_positives.append((detected, gt_finding))
                matched = True
                break

        if not matched:
            false_positives.append(detected)

    # For each ground truth finding, check if we detected it
    detected_gt_ids = {match[1].id for match in true_positives}
    for gt_finding in ground_truth.findings:
        if gt_finding.id not in detected_gt_ids:
            false_negatives.append(gt_finding)

    tp = len(true_positives)
    fp = len(false_positives)
    fn = len(false_negatives)

    precision = tp / (tp + fp) if (tp + fp) > 0 else 0.0
    recall = tp / (tp + fn) if (tp + fn) > 0 else 0.0
    f1 = 2 * (precision * recall) / (precision + recall) if (precision + recall) > 0 else 0.0

    return Metrics(
        true_positives=tp,
        false_positives=fp,
        false_negatives=fn,
        precision=precision,
        recall=recall,
        f1=f1,
        # These are MEASURED, not estimated
        tokens_used=None,  # Filled from API response
        duration_seconds=None  # Filled from actual timing
    )

def findings_match(detected: Finding, ground_truth: Finding) -> bool:
    """Determine if a detected finding matches a ground truth finding."""
    # Match by location (most reliable)
    if detected.location and ground_truth.location:
        if locations_overlap(detected.location, ground_truth.location):
            return True

    # Match by vulnerability type + contract
    if detected.pattern == ground_truth.vulnerability_type:
        if detected.contract == ground_truth.contract:
            # Same type, same contract - likely same issue
            return True

    return False
```

## COMPONENT 5: Token Measurement (REAL)

### From Claude Code Output

Claude Code outputs token usage. Parse it from the transcript:

```python
def extract_token_usage(transcript: str) -> int:
    """Extract actual token usage from Claude Code output."""
    # Look for cost display in Claude Code output
    # Format: "Cost: $X.XX (Y tokens)"
    match = re.search(r'Cost: \$[\d.]+.*?(\d+) tokens', transcript)
    if match:
        return int(match.group(1))

    # Alternative: look for /cost command output
    match = re.search(r'Total tokens: (\d+)', transcript)
    if match:
        return int(match.group(1))

    # If not found, return -1 to indicate unmeasured
    return -1
```

### Validation

```python
def validate_token_measurement(tokens: int, duration: float) -> bool:
    """Validate that token measurement is plausible."""
    if tokens <= 0:
        return False  # Must be positive

    if duration < 1.0 and tokens > 1000:
        return False  # Suspicious: high tokens, instant execution

    # Rough heuristic: 50-500 tokens/second is reasonable
    tokens_per_second = tokens / duration if duration > 0 else float('inf')
    if tokens_per_second > 1000:
        return False  # Suspiciously fast

    return True
```

## COMPONENT 6: Self-Test Loop Implementation

### The Correct Self-Test Flow

```python
async def run_self_test(suite: str, max_iterations: int = 3):
    """Run self-test with real execution and automatic improvement."""

    tests = load_test_suite(suite)
    results = []

    for iteration in range(1, max_iterations + 1):
        print(f"\n=== Iteration {iteration}/{max_iterations} ===\n")

        failures = []

        for test in tests:
            if test.status == "passed":
                continue  # Already passed in previous iteration

            # REAL EXECUTION via claude-code-agent-teams
            result = await run_test_in_claude-code-agent-teams(test)

            if result.status == "pass":
                test.status = "passed"
                print(f"  ✓ {test.name}")
            else:
                failures.append((test, result))
                print(f"  ✗ {test.name}: {result.error}")

        if not failures:
            print("\nAll tests passing!")
            break

        # Try to fix simple failures
        for test, result in failures:
            if result.error_type in FIXABLE_ERRORS:
                fix = suggest_fix(result)
                if fix and fix.complexity == "simple":
                    print(f"  Attempting fix: {fix.description}")
                    await apply_fix(fix)
                    test.status = "pending"  # Re-test next iteration
            else:
                # Log to backlog for manual review
                backlog.add(ImprovementItem(
                    source=test.name,
                    error=result.error,
                    category="deferred",
                    reason="Complex fix required"
                ))

    return SelfTestReport(
        suite=suite,
        iterations_run=iteration,
        total_tests=len(tests),
        passed=[t for t in tests if t.status == "passed"],
        failed=[t for t in tests if t.status != "passed"],
        backlog=backlog.items
    )
```

## IMPLEMENTATION CHECKLIST

### Before First Test Run

- [ ] claude-code-agent-teams MCP server is configured and working
- [ ] ANTHROPIC_API_KEY is set in environment
- [ ] Ground truth imported from external source
- [ ] Criteria files exist in `.vrs/testing/criteria/`
- [ ] Test budget approved (~$200 for full GA validation)

### For Each Test Run

- [ ] Create isolated claude-code-agent-teams session (NOT your main session)
- [ ] Launch Claude Code with `--dangerously-skip-permissions`
- [ ] Wait for initialization before sending commands
- [ ] Capture FULL transcript including token usage
- [ ] Parse REAL duration from timestamps
- [ ] Compare findings to EXTERNAL ground truth
- [ ] Cleanup claude-code-agent-teams session after test

### Report Validation

- [ ] `duration_seconds > 0` for all tests
- [ ] `tokens_used` is measured (not hardcoded)
- [ ] `ground_truth_source` is external
- [ ] No test has 100% precision AND 100% recall
- [ ] Error rate is documented (may be > 0)

## TROUBLESHOOTING

### "Session not found"
```bash
# Check existing sessions
claude-code-agent-teams list-sessions

# Create if needed
claude-code-agent-teams new-session -d -s vrs-test-subject
```

### "Claude Code not responding"
```bash
# Check if process is running
ps aux | grep claude

# Kill stale process
pkill -f "claude.*dangerously"

# Restart
claude-code-agent-teams send-keys -t vrs-test-subject "claude --dangerously-skip-permissions" Enter
```

### "Tokens not captured"
- Ensure Claude Code is run with default output (not `--quiet`)
- Check transcript capture includes full output
- Look for `/cost` command output in transcript

### "Duration is 0"
- Ensure timestamps are captured at start and end
- Check that wait_for_completion actually waits
- Verify timeout handling doesn't short-circuit
