## One-Liner

Worktree infrastructure enhanced with fixture-based creation, stage validation, and ground truth comparison for real /vrs-audit workflow testing.

## What Was Done

### Task 1: Enhanced manage_worktrees.sh

Added three new capabilities to the worktree management script:

1. **`create-from-fixture <name> <fixture>`** - Creates a worktree pre-populated with:
   - Contracts from `tests/fixtures/<fixture>/src/`
   - Expected outputs from `tests/fixtures/<fixture>/expected/`
   - Ground truth from `tests/fixtures/<fixture>/ground-truth.yaml`
   - Foundry configuration if present

2. **`validate <name> <stage>`** - Validates individual stage outputs:
   - `graph` - Checks for `.toon` or `.json` in `.vrs/graphs/`
   - `context` - Checks for `protocol-pack.yaml`
   - `patterns` - Checks for `pattern-matches.json`
   - `agents` - Checks for `agent-investigations.json`
   - `debate` - Checks for `verdicts.json`
   - `report` - Checks for `audit-report.md`
   - `all` - Validates all stages

3. **`list-fixtures`** - Lists available test fixtures with metadata

### Task 2: Created run_audit_test.sh

Complete test runner script that:
- Creates worktree from fixture
- Launches Claude Code via claude-code-controller
- Sends `/vrs-audit` command with configurable mode
- Waits for completion (configurable timeout)
- Captures transcript
- Validates all stage outputs
- Runs ground truth comparison
- Outputs metrics summary

Options:
- `--timeout N` - Wait timeout (default: 600s)
- `--mode swarm|solo` - Audit mode (default: swarm)
- `--skip-cleanup` - Preserve worktree after test
- `--name NAME` - Custom worktree name

### Task 3: Created compare_findings.py

Python script for comparing audit findings to ground truth:
- Loads ground truth from YAML (with fallback parser for no-PyYAML environments)
- Loads findings from verdicts.json, agent-investigations.json, or pattern-matches.json
- Matches findings using:
  - Function name matching (exact and partial)
  - Line number matching (with 5-line tolerance)
  - Line range containment
  - Type normalization (e.g., "reentrancy", "access-control")
- Calculates precision, recall, and F1 score
- Outputs JSON with TP/FP/FN lists
