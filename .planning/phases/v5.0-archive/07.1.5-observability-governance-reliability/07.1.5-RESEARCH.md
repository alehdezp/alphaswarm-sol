## Summary

Observability in multi-agent systems is converging on structured traces (handoffs, tool calls, guardrails), with policy enforcement and cost governance. The OpenAI Agents SDK establishes patterns for hierarchical span tracing with specialized span types for agents, LLMs, tools, and guardrails. OpenTelemetry GenAI semantic conventions (v1.37+) provide standard instrumentation for LLM applications, with Langfuse and others building on these conventions.

Claude Code ecosystems show early observability tooling based on hook events, with real-time WebSocket dashboards tracking multi-agent coordination. Chaos engineering research (ChaosEater, ReliabilityBench) demonstrates systematic fault injection for LLM agent systems, testing resilience against API failures, communication delays, and agent failures.

Production observability platforms track 15+ LLM calls per user request, with dashboards showing token usage, latency (P95/P99), cost attribution, and tool analytics. SLO patterns focus on workflow success rate, mean time to recovery (MTTR), and end-to-end latency including retries.

**Primary recommendation:** Build unified trace schema with OpenTelemetry-compatible spans, structured JSON audit logs with evidence lineage, guardrail policies enforced at tool/agent boundaries, and SLO tracking for pool completion rate, verdict accuracy, and cost per finding.

### Pattern 1: OpenTelemetry Trace Hierarchy

**What:** Hierarchical spans representing agent execution flow with GenAI semantic conventions

**When to use:** All agent invocations, LLM calls, tool executions, handoffs

**Example:**
```python
# Source: OpenTelemetry GenAI semantic conventions + Langfuse integration
from opentelemetry import trace
from opentelemetry.trace import Status, StatusCode

tracer = trace.get_tracer("alphaswarm.orchestration", version="0.5.1")

def execute_agent(pool_id: str, bead_id: str, agent_type: str):
    """Execute agent with full tracing."""
    with tracer.start_as_current_span(
        f"agent.{agent_type}",
        kind=trace.SpanKind.INTERNAL,
        attributes={
            # GenAI semantic conventions
            "gen_ai.system": "anthropic",
            "gen_ai.request.model": "claude-3-5-sonnet",
            "gen_ai.operation.name": "investigate",
            # Pool/bead context
            "pool.id": pool_id,
            "bead.id": bead_id,
            "agent.type": agent_type,
            "agent.role": "attacker",
        }
    ) as span:
        try:
            # LLM call creates child span
            result = call_llm(prompt, model="claude-3-5-sonnet")

            # Record usage metrics
            span.set_attribute("gen_ai.usage.input_tokens", result.input_tokens)
            span.set_attribute("gen_ai.usage.output_tokens", result.output_tokens)
            span.set_attribute("gen_ai.usage.total_tokens", result.total_tokens)

            # Record cost
            cost = calculate_cost(result.input_tokens, result.output_tokens, "claude-3-5-sonnet")
            span.set_attribute("gen_ai.usage.cost_usd", cost)

            # Record events (not attributes - payload size)
            span.add_event(
                "llm.request",
                attributes={
                    "llm.request.temperature": 0.7,
                    "llm.request.top_p": 0.9,
                }
            )
            span.add_event(
                "llm.response",
                attributes={
                    "llm.response.finish_reason": result.finish_reason,
                }
            )

            span.set_status(Status(StatusCode.OK))
            return result

        except Exception as e:
            span.set_status(Status(StatusCode.ERROR, str(e)))
            span.record_exception(e)
            raise
```

### Pattern 3: Guardrail Policy Enforcement

**What:** Runtime validation of agent inputs/outputs, tool access, and cost limits

**When to use:** Before agent execution (input), after agent execution (output), before tool calls

**Example:**
```python
# Source: OpenAI Agents SDK guardrails + AgentSpec patterns
from typing import Callable, Any, Dict
from dataclasses import dataclass
from enum import Enum

class PolicyAction(Enum):
    """Action to take on policy violation."""
    ALLOW = "allow"
    BLOCK = "block"
    SANITIZE = "sanitize"
    ALERT = "alert"

@dataclass
class PolicyViolation:
    """Policy violation details."""
    policy_id: str
    violation_type: str
    severity: str
    message: str
    suggested_action: PolicyAction

class PolicyEnforcer:
    """Runtime policy enforcement for agents and tools."""

    def __init__(self, policies: Dict[str, Any]):
        self.policies = policies
        self.logger = structlog.get_logger("alphaswarm.governance")

    def check_input_policy(
        self,
        agent_type: str,
        input_data: Dict[str, Any],
        pool_id: str,
        trace_id: Optional[str] = None,
    ) -> Optional[PolicyViolation]:
        """Check input against policies before agent execution."""

        # Check cost budget
        if "budget" in self.policies:
            pool_cost = self.get_pool_cost(pool_id)
            if pool_cost > self.policies["budget"]["hard_limit_usd"]:
                return PolicyViolation(
                    policy_id="cost_budget.hard_limit",
                    violation_type="budget_exceeded",
                    severity="critical",
                    message=f"Pool {pool_id} exceeded hard limit: ${pool_cost:.2f}",
                    suggested_action=PolicyAction.BLOCK,
                )

        # Check tool access policy (Phase 7.1.2 guardrails)
        requested_tools = input_data.get("tools", [])
        allowed_tools = self.policies.get("tool_access", {}).get(agent_type, [])

        forbidden = set(requested_tools) - set(allowed_tools)
        if forbidden:
            return PolicyViolation(
                policy_id="tool_access.forbidden",
                violation_type="unauthorized_tool",
                severity="high",
                message=f"Agent {agent_type} requested forbidden tools: {forbidden}",
                suggested_action=PolicyAction.BLOCK,
            )

        return None  # No violation

    def check_output_policy(
        self,
        agent_type: str,
        output_data: Dict[str, Any],
        pool_id: str,
        trace_id: Optional[str] = None,
    ) -> Optional[PolicyViolation]:
        """Check output against policies after agent execution."""

        # Check evidence requirement (Phase 5.11-03 integrity gates)
        verdict = output_data.get("verdict")
        confidence = output_data.get("confidence")
        evidence_refs = output_data.get("evidence_refs", [])

        if confidence in ["LIKELY", "CONFIRMED"] and not evidence_refs:
            return PolicyViolation(
                policy_id="evidence_gate.missing_evidence",
                violation_type="confidence_without_evidence",
                severity="critical",
                message=f"Agent {agent_type} upgraded confidence to {confidence} without evidence",
                suggested_action=PolicyAction.BLOCK,
            )

        # Check output contains no secrets (data access policy)
        if self.contains_secrets(output_data):
            return PolicyViolation(
                policy_id="data_access.secret_leak",
                violation_type="secret_in_output",
                severity="critical",
                message="Output contains potential secrets",
                suggested_action=PolicyAction.SANITIZE,
            )

        return None

def enforce_policy(enforcer: PolicyEnforcer):
    """Decorator for policy enforcement."""
    def decorator(func: Callable) -> Callable:
        def wrapper(*args, **kwargs):
            # Extract context
            pool_id = kwargs.get("pool_id")
            agent_type = kwargs.get("agent_type")
            trace_id = kwargs.get("trace_id")

            # Check input policy
            input_data = kwargs.get("input_data", {})
            violation = enforcer.check_input_policy(agent_type, input_data, pool_id, trace_id)
            if violation and violation.suggested_action == PolicyAction.BLOCK:
                enforcer.logger.warning(
                    "policy.blocked",
                    pool_id=pool_id,
                    agent_type=agent_type,
                    policy_id=violation.policy_id,
                    violation_type=violation.violation_type,
                )
                raise PolicyViolationError(violation.message)

            # Execute function
            result = func(*args, **kwargs)

            # Check output policy
            violation = enforcer.check_output_policy(agent_type, result, pool_id, trace_id)
            if violation and violation.suggested_action == PolicyAction.BLOCK:
                enforcer.logger.warning(
                    "policy.blocked",
                    pool_id=pool_id,
                    agent_type=agent_type,
                    policy_id=violation.policy_id,
                    violation_type=violation.violation_type,
                )
                raise PolicyViolationError(violation.message)

            if violation and violation.suggested_action == PolicyAction.SANITIZE:
                result = enforcer.sanitize_output(result)

            return result
        return wrapper
    return decorator
```

### Pattern 4: SLO Tracking with Incident Detection

**What:** Service Level Objectives for pool execution with automated incident detection

**When to use:** Continuous monitoring of pool health, latency, accuracy, cost

**Example:**
```python
# Source: Sherlock reliability patterns + AI agent monitoring best practices
from dataclasses import dataclass
from datetime import datetime, timedelta
from typing import List, Optional
import statistics

@dataclass
class SLO:
    """Service Level Objective definition."""
    name: str
    description: str
    target_percentile: float  # e.g., 0.95 for P95
    target_value: float
    measurement_window: timedelta
    alert_threshold: float  # Percentage below target before alerting

@dataclass
class SLOViolation:
    """SLO violation event."""
    slo_name: str
    measured_value: float
    target_value: float
    violation_percentage: float
    measurement_window: timedelta
    timestamp: datetime
    affected_pools: List[str]

class SLOTracker:
    """Track and enforce SLOs for pool execution."""

    # SLO definitions for AlphaSwarm
    POOL_SLOS = {
        "pool_success_rate": SLO(
            name="pool_success_rate",
            description="Percentage of pools completing successfully",
            target_percentile=1.0,  # 100% target
            target_value=0.95,  # 95% success rate
            measurement_window=timedelta(days=7),
            alert_threshold=0.90,  # Alert if < 90%
        ),
        "verdict_accuracy": SLO(
            name="verdict_accuracy",
            description="Percentage of verdicts confirmed accurate",
            target_percentile=1.0,
            target_value=0.90,  # 90% accuracy
            measurement_window=timedelta(days=7),
            alert_threshold=0.85,
        ),
        "pool_completion_latency_p95": SLO(
            name="pool_completion_latency_p95",
            description="95th percentile pool completion time",
            target_percentile=0.95,
            target_value=300.0,  # 5 minutes (seconds)
            measurement_window=timedelta(hours=24),
            alert_threshold=360.0,  # Alert if > 6 minutes
        ),
        "cost_per_finding": SLO(
            name="cost_per_finding",
            description="Average cost per confirmed finding",
            target_percentile=0.95,
            target_value=2.0,  # $2.00 per finding
            measurement_window=timedelta(days=7),
            alert_threshold=3.0,  # Alert if > $3.00
        ),
        "bead_mttr": SLO(
            name="bead_mttr",
            description="Mean time to recovery for failed beads",
            target_percentile=0.95,
            target_value=60.0,  # 60 seconds
            measurement_window=timedelta(hours=24),
            alert_threshold=120.0,  # Alert if > 120 seconds
        ),
    }

    def __init__(self, ledger, event_store):
        self.ledger = ledger
        self.event_store = event_store
        self.logger = structlog.get_logger("alphaswarm.reliability")

    def check_slo(self, slo_name: str) -> Optional[SLOViolation]:
        """Check if SLO is violated."""
        slo = self.POOL_SLOS[slo_name]

        if slo_name == "pool_success_rate":
            return self._check_pool_success_rate(slo)
        elif slo_name == "verdict_accuracy":
            return self._check_verdict_accuracy(slo)
        elif slo_name == "pool_completion_latency_p95":
            return self._check_pool_latency(slo)
        elif slo_name == "cost_per_finding":
            return self._check_cost_per_finding(slo)
        elif slo_name == "bead_mttr":
            return self._check_bead_mttr(slo)

    def _check_pool_success_rate(self, slo: SLO) -> Optional[SLOViolation]:
        """Check pool success rate SLO."""
        # Query pools completed in measurement window
        cutoff = datetime.utcnow() - slo.measurement_window
        pools = self.event_store.query_pools(completed_after=cutoff)

        total_pools = len(pools)
        successful_pools = sum(1 for p in pools if p.status == "COMPLETE")

        if total_pools == 0:
            return None

        success_rate = successful_pools / total_pools

        if success_rate < slo.target_value:
            failed_pool_ids = [p.id for p in pools if p.status == "FAILED"]
            return SLOViolation(
                slo_name=slo.name,
                measured_value=success_rate,
                target_value=slo.target_value,
                violation_percentage=(slo.target_value - success_rate) / slo.target_value,
                measurement_window=slo.measurement_window,
                timestamp=datetime.utcnow(),
                affected_pools=failed_pool_ids,
            )

        return None

    def _check_pool_latency(self, slo: SLO) -> Optional[SLOViolation]:
        """Check P95 pool completion latency."""
        cutoff = datetime.utcnow() - slo.measurement_window
        pools = self.event_store.query_pools(completed_after=cutoff)

        latencies = [
            (p.completed_at - p.created_at).total_seconds()
            for p in pools if p.completed_at
        ]

        if not latencies:
            return None

        # Calculate P95
        p95 = statistics.quantiles(latencies, n=20)[18]  # 95th percentile

        if p95 > slo.target_value:
            slow_pools = [
                p.id for p in pools
                if p.completed_at and (p.completed_at - p.created_at).total_seconds() > slo.target_value
            ]
            return SLOViolation(
                slo_name=slo.name,
                measured_value=p95,
                target_value=slo.target_value,
                violation_percentage=(p95 - slo.target_value) / slo.target_value,
                measurement_window=slo.measurement_window,
                timestamp=datetime.utcnow(),
                affected_pools=slow_pools[:10],  # Top 10 slowest
            )

        return None

    def monitor_all_slos(self) -> List[SLOViolation]:
        """Check all SLOs and return violations."""
        violations = []
        for slo_name in self.POOL_SLOS:
            violation = self.check_slo(slo_name)
            if violation:
                self.logger.warning(
                    "slo.violated",
                    slo_name=slo_name,
                    measured_value=violation.measured_value,
                    target_value=violation.target_value,
                    violation_percentage=f"{violation.violation_percentage:.1%}",
                    affected_pools=violation.affected_pools,
                )
                violations.append(violation)
        return violations
```

### Pattern 5: Chaos Testing for Agent Systems

**What:** Systematic fault injection to test agent system resilience

**When to use:** CI/CD pipeline, pre-release testing, production validation

**Example:**
```python
# Source: ChaosEater research + ReliabilityBench patterns
from enum import Enum
from typing import Callable, Any, Optional
import random
import time

class FaultType(Enum):
    """Types of faults to inject."""
    API_TIMEOUT = "api_timeout"
    API_ERROR = "api_error"
    MALFORMED_RESPONSE = "malformed_response"
    AGENT_FAILURE = "agent_failure"
    COMMUNICATION_DELAY = "communication_delay"
    COST_SPIKE = "cost_spike"
    SCHEMA_VIOLATION = "schema_violation"

@dataclass
class ChaosExperiment:
    """Chaos experiment definition."""
    name: str
    fault_type: FaultType
    injection_rate: float  # 0.0 to 1.0 (percentage of calls)
    fault_params: Dict[str, Any]
    target_component: str  # "llm", "tool", "agent", "handoff"

class ChaosTestHarness:
    """Chaos testing harness for multi-agent orchestration."""

    def __init__(self, enabled: bool = False):
        self.enabled = enabled
        self.active_experiments: List[ChaosExperiment] = []
        self.logger = structlog.get_logger("alphaswarm.chaos")

    def add_experiment(self, experiment: ChaosExperiment):
        """Add chaos experiment."""
        self.active_experiments.append(experiment)
        self.logger.info(
            "chaos.experiment.added",
            name=experiment.name,
            fault_type=experiment.fault_type.value,
            injection_rate=experiment.injection_rate,
        )

    def should_inject_fault(self, component: str) -> Optional[ChaosExperiment]:
        """Determine if fault should be injected."""
        if not self.enabled:
            return None

        for experiment in self.active_experiments:
            if experiment.target_component == component:
                if random.random() < experiment.injection_rate:
                    return experiment
        return None

    def inject_fault(self, experiment: ChaosExperiment, context: Dict[str, Any]):
        """Inject fault based on experiment type."""
        self.logger.warning(
            "chaos.fault.injected",
            experiment=experiment.name,
            fault_type=experiment.fault_type.value,
            context=context,
        )

        if experiment.fault_type == FaultType.API_TIMEOUT:
            delay = experiment.fault_params.get("delay_seconds", 30)
            time.sleep(delay)
            raise TimeoutError(f"API timeout after {delay}s (chaos)")

        elif experiment.fault_type == FaultType.API_ERROR:
            status_code = experiment.fault_params.get("status_code", 500)
            raise APIError(f"API error {status_code} (chaos)")

        elif experiment.fault_type == FaultType.MALFORMED_RESPONSE:
            # Return corrupted JSON
            return {"error": "malformed", "data": None}

        elif experiment.fault_type == FaultType.COMMUNICATION_DELAY:
            delay = experiment.fault_params.get("delay_seconds", 5)
            time.sleep(delay)

        elif experiment.fault_type == FaultType.SCHEMA_VIOLATION:
            # Return response with wrong schema
            return {"unexpected_field": "chaos", "missing_required": True}

        elif experiment.fault_type == FaultType.COST_SPIKE:
            # Simulate 10x cost increase
            return experiment.fault_params.get("cost_multiplier", 10.0)

def with_chaos_testing(component: str):
    """Decorator for chaos testing."""
    def decorator(func: Callable) -> Callable:
        def wrapper(*args, **kwargs):
            harness = kwargs.get("chaos_harness")
            if harness:
                experiment = harness.should_inject_fault(component)
                if experiment:
                    context = {
                        "function": func.__name__,
                        "component": component,
                        "args": args,
                    }
                    return harness.inject_fault(experiment, context)

            return func(*args, **kwargs)
        return wrapper
    return decorator

def call_llm(prompt: str, model: str, chaos_harness=None):
```

### Anti-Patterns to Avoid

- **Logging prompts/responses as span attributes:** Use events instead - large payloads break backend systems (OpenTelemetry LLM Working Group recommendation)
- **No trace context propagation:** Agent chains lose causality - always propagate trace_id through context
- **Blocking guardrails everywhere:** Parallel execution for input guardrails saves latency - only block when necessary for cost control
- **SLOs without measurement:** Define concrete metrics (P95 latency, success rate) and automate measurement
- **Manual audit log querying:** Build dashboards for common queries (cost by pool, latency by agent, violations by policy)

## Don't Hand-Roll

Problems that look simple but have existing solutions:

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| Trace instrumentation | Custom span tracking | OpenTelemetry SDK | Standard exports, semantic conventions, tooling ecosystem |
| JSON logging | Custom log formatters | structlog | Context binding, processor chains, trace correlation |
| Metric aggregation | Custom counters/histograms | OpenTelemetry Metrics API | Standard metric types, exporters, backend integration |
| Guardrail validation | Ad-hoc if/else checks | guardrails-ai or OpenAI SDK patterns | Reusable validators, policy-as-code, audit trail |
| Chaos fault injection | Manual mocking | Chaos testing framework (Chaos Mesh, custom harness) | Systematic scenarios, configurable rates, measurement |
| Evidence lineage tracking | Custom provenance chains | Structured logs + trace correlation | Standard lineage pattern, query support, compliance-ready |

**Key insight:** Observability and reliability tooling has matured significantly in 2025-2026, with OpenTelemetry becoming the de-facto standard. Custom solutions miss ecosystem benefits (exporters, visualization, analysis tools) and create maintenance burden.

## Common Pitfalls

### Pitfall 1: Missing Trace Context Across Agent Handoffs

**What goes wrong:** Agent A calls Agent B, but trace context is lost - dashboards show disconnected spans instead of unified workflows

**Why it happens:** Trace context must be explicitly propagated through function arguments or context managers

**How to avoid:** Use OpenTelemetry context propagation and ensure all agent calls include `trace_id` and `parent_span_id` in metadata

**Warning signs:** Dashboard shows multiple root spans for single pool execution, unable to correlate agent decisions

### Pitfall 2: Evidence References Without Lineage

**What goes wrong:** Verdicts cite evidence IDs, but lineage to original BSKG nodes is missing - auditors can't verify claims

**Why it happens:** Evidence tracking implemented in verdict schema but not connected to graph queries or tool findings

**How to avoid:** Every evidence reference must include: source_type (bskg/tool/manual), source_id (node ID/finding ID), extraction_timestamp, extracting_agent

**Warning signs:** Audit questions "Where did this evidence come from?" have no answer, compliance failures

### Pitfall 3: Guardrails That Block Instead of Alert

**What goes wrong:** Input guardrails block all requests exceeding soft limits, killing valid workflows

**Why it happens:** Treating soft limits (warnings) same as hard limits (blocking)

**How to avoid:** Distinguish hard limits (block), soft limits (alert + log), and advisory (log only). Use PolicyAction enum.

**Warning signs:** Agents frequently blocked with "budget exceeded" even when below hard limits, false positives in policy enforcement

### Pitfall 4: SLOs Defined But Never Measured

**What goes wrong:** SLOs documented in README but no automated tracking - violations go unnoticed until production incident

**Why it happens:** SLO tracking requires continuous measurement infrastructure, not just definitions

**How to avoid:** Build SLOTracker that runs on schedule (every 5 minutes), queries event store and cost ledger, generates alerts

**Warning signs:** "We have SLOs" but no dashboard shows current state, incidents reveal SLO violations that existed for days

### Pitfall 5: Chaos Testing Only in Staging

**What goes wrong:** Production encounters fault patterns never tested - system fails catastrophically on first real API outage

**Why it happens:** Fear of "breaking production" with chaos experiments

**How to avoid:** Use canary chaos testing with low injection rates (1-5%) in production, monitor MTTR and success rate degradation

**Warning signs:** 100% staging success, 0% production resilience testing, first real failure is a crisis

### Pitfall 6: Cost Attribution Without Pool Scoping

**What goes wrong:** Cost dashboards show total spend but can't attribute to specific audit runs or beads

**Why it happens:** cost_ledger.record() called without pool_id or bead_id

**How to avoid:** Always pass pool_id and bead_id to cost tracking, use structured logs with consistent tagging

**Warning signs:** "How much did pool-abc123 cost?" question has no answer, cost optimization lacks targeting

### Ops Dashboard - Latency & Success Rate

```python
# Source: Multi-agent observability platform patterns
from dataclasses import dataclass
from datetime import datetime, timedelta
from typing import List, Dict, Any
import statistics

@dataclass
class LatencyStats:
    """Latency statistics."""
    p50: float
    p95: float
    p99: float
    mean: float
    max: float
    min: float
    sample_count: int

@dataclass
class SuccessRateStats:
    """Success rate statistics."""
    total_pools: int
    successful_pools: int
    failed_pools: int
    success_rate: float
    failure_rate: float

class OpsDashboard:
    """Operations dashboard for multi-agent orchestration."""

    def __init__(self, event_store, cost_ledger):
        self.event_store = event_store
        self.cost_ledger = cost_ledger

    def get_pool_latency_stats(self, window: timedelta = timedelta(hours=24)) -> LatencyStats:
        """Get pool completion latency statistics."""
        cutoff = datetime.utcnow() - window
        pools = self.event_store.query_pools(completed_after=cutoff)

        latencies = [
            (p.completed_at - p.created_at).total_seconds()
            for p in pools if p.completed_at
        ]

        if not latencies:
            return LatencyStats(0, 0, 0, 0, 0, 0, 0)

        sorted_latencies = sorted(latencies)
        n = len(sorted_latencies)

        return LatencyStats(
            p50=sorted_latencies[int(n * 0.50)],
            p95=sorted_latencies[int(n * 0.95)],
            p99=sorted_latencies[int(n * 0.99)] if n > 100 else sorted_latencies[-1],
            mean=statistics.mean(latencies),
            max=max(latencies),
            min=min(latencies),
            sample_count=n,
        )

    def get_success_rate_stats(self, window: timedelta = timedelta(days=7)) -> SuccessRateStats:
        """Get pool success rate statistics."""
        cutoff = datetime.utcnow() - window
        pools = self.event_store.query_pools(completed_after=cutoff)

        total = len(pools)
        successful = sum(1 for p in pools if p.status == "COMPLETE")
        failed = sum(1 for p in pools if p.status == "FAILED")

        return SuccessRateStats(
            total_pools=total,
            successful_pools=successful,
            failed_pools=failed,
            success_rate=successful / total if total > 0 else 0.0,
            failure_rate=failed / total if total > 0 else 0.0,
        )

    def get_agent_performance(self, window: timedelta = timedelta(days=7)) -> Dict[str, Dict[str, Any]]:
        """Get per-agent performance metrics."""
        cutoff = datetime.utcnow() - window
        beads = self.event_store.query_beads(created_after=cutoff)

        agent_metrics = {}
        for agent_type in ["vrs-attacker", "vrs-defender", "vrs-verifier"]:
            agent_beads = [b for b in beads if b.agent_type == agent_type]

            # Latency
            latencies = [
                (b.completed_at - b.started_at).total_seconds()
                for b in agent_beads if b.completed_at and b.started_at
            ]

            # Cost
            agent_cost = self.cost_ledger.get_agent_cost(agent_type, window)

            # Accuracy (if ground truth available)
            accuracy = self._calculate_agent_accuracy(agent_type, window)

            agent_metrics[agent_type] = {
                "total_beads": len(agent_beads),
                "mean_latency": statistics.mean(latencies) if latencies else 0,
                "p95_latency": statistics.quantiles(latencies, n=20)[18] if len(latencies) > 20 else 0,
                "total_cost_usd": agent_cost,
                "accuracy": accuracy,
            }

        return agent_metrics

    def render_dashboard_markdown(self) -> str:
        """Render ops dashboard as Markdown."""
        latency = self.get_pool_latency_stats()
        success = self.get_success_rate_stats()
        agent_perf = self.get_agent_performance()

        md = f"""# AlphaSwarm Operations Dashboard

## Pool Performance (Last 24h)

**Success Rate:** {success.success_rate:.1%} ({success.successful_pools}/{success.total_pools} pools)

**Latency:**
- P50: {latency.p50:.1f}s
- P95: {latency.p95:.1f}s
- P99: {latency.p99:.1f}s
- Mean: {latency.mean:.1f}s

## Agent Performance (Last 7d)

| Agent | Beads | Mean Latency | P95 Latency | Cost | Accuracy |
|-------|-------|--------------|-------------|------|----------|
"""
        for agent_type, metrics in agent_perf.items():
            md += f"| {agent_type} | {metrics['total_beads']} | {metrics['mean_latency']:.1f}s | {metrics['p95_latency']:.1f}s | ${metrics['total_cost_usd']:.2f} | {metrics['accuracy']:.1%} |\n"

        md += f"""
## SLO Status

{self._render_slo_status()}

## Recent Incidents

{self._render_recent_incidents()}
"""
        return md
```

## Open Questions

Things that couldn't be fully resolved:

1. **Trace Backend Selection**
   - What we know: Jaeger is simple for dev, Langfuse has LLM-native features, Grafana Tempo integrates with Prometheus
   - What's unclear: Which backend best fits AlphaSwarm's self-contained deployment model
   - Recommendation: Start with Jaeger (simpler), optionally support OTLP export for Langfuse

2. **Evidence Lineage Storage**
   - What we know: SQLite sufficient for audit logs, lineage chains can be complex graphs
   - What's unclear: Should lineage be SQLite (simple) or graph database (complex queries)?
   - Recommendation: SQLite with `evidence_lineage` table (JSON lineage_chain field), migrate to graph DB only if queries too slow

3. **Chaos Testing in Production**
   - What we know: 1-5% injection rates safe, canary testing standard practice
   - What's unclear: User tolerance for chaos-induced latency in production
   - Recommendation: Start with 1% injection rate, monitor user complaints, increase gradually

4. **SLO Alert Fatigue**
   - What we know: Too many alerts lead to ignoring, too few miss incidents
   - What's unclear: Optimal alert threshold balance for AlphaSwarm workflows
   - Recommendation: Start with conservative thresholds (only critical SLOs), tune based on incident history

5. **Dashboard Refresh Rate**
   - What we know: Real-time (WebSocket) enables live monitoring, 5-minute snapshots reduce load
   - What's unclear: Whether AlphaSwarm operations require real-time or periodic is sufficient
   - Recommendation: 5-minute snapshot for CLI dashboards, consider WebSocket only if web UI built
