# Observability Guide

This guide explains how to use AlphaSwarm's observability infrastructure for tracing, audit logging, and evidence lineage tracking.

## Overview

Phase 7.1.5 introduces production-grade observability across multi-agent orchestration:

- **Tracing**: OpenTelemetry-based span tracking for agents, tools, and handoffs
- **Audit Logging**: Structured JSON logs for verdicts, evidence usage, and policy checks
- **Evidence Lineage**: Complete provenance chains from BSKG/tool sources to verdicts

## Quick Start

### Enable Tracing

```python
from alphaswarm_sol.observability import setup_tracing, get_tracer

# Initialize tracing (call once at startup)
tracer = setup_tracing(
    service_name="alphaswarm",
    endpoint="http://localhost:4318/v1/traces",  # Optional: OTLP endpoint
)

# Get tracer for instrumentation
tracer = get_tracer("alphaswarm.orchestration")
```

### Create Spans

```python
from alphaswarm_sol.observability import create_agent_span, create_tool_span, create_handoff_span

# Agent execution span
with create_agent_span(
    agent_type="vrs-attacker",
    pool_id="pool-abc123",
    bead_id="VKG-042",
    model="claude-3-5-sonnet",
    role="attacker",
) as span:
    # Agent work here
    result = execute_agent(...)

    # Record token usage
    span.set_attribute("gen_ai.usage.input_tokens", result.input_tokens)
    span.set_attribute("gen_ai.usage.output_tokens", result.output_tokens)

# Tool execution span (child of agent)
with create_tool_span(
    tool_name="slither",
    pool_id="pool-abc123",
) as span:
    # Tool execution
    findings = run_slither(...)

# Handoff span
with create_handoff_span(
    from_agent="vrs-attacker",
    to_agent="vrs-defender",
    pool_id="pool-abc123",
    bead_id="VKG-042",
) as span:
    # Pass context to next agent
    pass
```

### Record Events

```python
from alphaswarm_sol.observability import record_input_event, record_output_event, record_error_event

# Record input (summary only, not full prompt)
record_input_event(span, input_summary="Investigating reentrancy in withdraw()", token_estimate=500)

# Record output with token usage
record_output_event(
    span,
    output_summary="Found potential reentrancy pattern",
    input_tokens=1500,
    output_tokens=800,
    cost_usd=0.05,
    finish_reason="stop",
)

# Record errors
try:
    result = call_llm(...)
except Exception as e:
    record_error_event(span, e, recoverable=True)
    raise
```

## Evidence Lineage

### Track Evidence Origin

from alphaswarm_sol.observability.lineage import LineageTracker, SourceType

tracker = LineageTracker()

# Create lineage when extracting evidence
lineage = tracker.create_lineage(
    evidence_id="ev-001",
    source_type=SourceType.BSKG,
    source_id="node_func_vault_withdraw_123",

# Add transformation steps
tracker.add_transformation(
    evidence_id="ev-001",
    transform_type="pattern_match",
    transforming_agent="vrs-attacker",

    evidence_id="ev-001",

    verification_result="confirmed",

lineage = tracker.get_lineage("ev-001")

evidence = tracker.query_by_source(SourceType.BSKG, "node_func_vault_withdraw_123")
```

## Best Practices

1. **Always propagate trace_id** through Pool/Bead metadata
2. **Don't log full prompts** in span attributes (use events with summaries)
3. **Record evidence lineage** at extraction time, not later
4. **Include trace_id** in all audit log entries for correlation
5. **Shutdown tracing** gracefully to flush pending spans

```python
from alphaswarm_sol.observability import shutdown_tracing

# At application shutdown
shutdown_tracing()
```

# Governance Guide

This guide explains how to configure and enforce governance policies for cost control, tool access, and evidence integrity.

## Overview

Phase 7.1.5 governance provides:

- **Cost Budgets**: Hard/soft limits per pool with blocking or alerting
- **Tool Access Control**: Role-based tool restrictions per agent type
- **Evidence Integrity**: Gates requiring evidence for confidence upgrades
- **Model Usage**: Approved model lists with tier routing enforcement

## Quick Start

### Enable Policy Enforcement

```python
from alphaswarm_sol.governance import PolicyEnforcer, enforce_policy
from alphaswarm_sol.observability.audit import AuditLogger
from alphaswarm_sol.metrics.cost_ledger import CostLedger

# Setup
cost_ledger = CostLedger()
audit_logger = AuditLogger()

enforcer = PolicyEnforcer(
    policies_path=Path("configs/governance_policies.yaml"),
    cost_ledger=cost_ledger,
    audit_logger=audit_logger,
)
```

### Check Policies Manually

```python
# Check before agent execution
violation = enforcer.check_input_policy(
    agent_type="vrs-attacker",
    input_data={"tools": ["slither", "mythril"]},
    pool_id="pool-abc123",
    trace_id="trace_xyz789",
)

if violation:
    print(f"Policy violated: {violation.message}")
    if violation.suggested_action == PolicyAction.BLOCK:
        raise PolicyViolationError(violation)
```

### Use Decorator

```python
from alphaswarm_sol.governance import enforce_policy

@enforce_policy(enforcer)
def execute_agent(pool_id: str, agent_type: str, input_data: dict, trace_id: str = None):
    # Policy checks happen automatically
    return run_agent(...)
```

## Policy Configuration

### Cost Budget

```yaml
# configs/governance_policies.yaml
policies:
  cost_budget:
    hard_limit_usd: 10.0      # Block when exceeded
    soft_limit_usd: 8.0       # Alert when exceeded
    per_agent_limit_usd: 3.0  # Per-agent cap
    action_on_hard_limit: BLOCK
    action_on_soft_limit: ALERT
```

**Behavior:**
- **Hard limit**: `PolicyViolationError` raised, execution blocked
- **Soft limit**: Warning logged, execution continues
- **Per-agent limit**: Individual agent capped within pool

### Tool Access

```yaml
policies:
  tool_access:
    vrs-attacker:
      allowed:
        - slither
        - mythril
        - aderyn
        - graph-query
        - file-read
      forbidden:
        - file-write
        - git-commit
        - deploy
    vrs-defender:
      allowed:
        - slither
        - graph-query
        - file-read
      forbidden:
        - mythril      # Defenders don't run symbolic execution
        - file-write
    vrs-verifier:
      allowed:
        - graph-query
        - file-read
      forbidden:
        - slither      # Verifiers cross-check, don't run tools
        - mythril
```

### Evidence Integrity

```yaml
policies:
  evidence_integrity:
    confidence_upgrade_requires_evidence: true
    min_evidence_refs_for_likely: 1
    min_evidence_refs_for_confirmed: 2
    require_provenance_chain: true
```

**Behavior:**
- `LIKELY` confidence requires ≥1 evidence reference
- `CONFIRMED` confidence requires ≥2 evidence references
- Evidence must have valid lineage chain

### Model Usage

```yaml
policies:
  model_usage:
    approved_models:
      - claude-3-5-sonnet
      - claude-3-opus
      - claude-3-haiku
      - gpt-4o
      - gpt-4o-mini
    tier_routing_required: true
    fallback_model: claude-3-haiku
```

## Handling Violations

### PolicyViolation Details

```python
@dataclass
class PolicyViolation:
    policy_id: str           # e.g., "cost_budget.hard_limit"
    violation_type: str      # e.g., "budget_exceeded"
    severity: str            # "critical", "high", "medium", "low"
    message: str             # Human-readable description
    suggested_action: PolicyAction  # ALLOW, BLOCK, SANITIZE, ALERT
    details: Dict[str, Any]  # Additional context
```

### Custom Violation Handling

```python
def handle_violation(violation: PolicyViolation, pool_id: str):
    if violation.suggested_action == PolicyAction.BLOCK:
        # Hard stop
        raise PolicyViolationError(violation)
    elif violation.suggested_action == PolicyAction.ALERT:
        # Log and continue
        logger.warning(f"Policy alert: {violation.message}")
    elif violation.suggested_action == PolicyAction.SANITIZE:
        # Clean output
        return sanitize_output(result)
```

# Reliability Guide

This guide explains how to use SLO tracking, incident detection, playbooks, and chaos testing for production reliability.

## Overview

Phase 7.1.5 reliability provides:

- **SLO Tracking**: Automated measurement of success rate, latency, cost, accuracy
- **Incident Detection**: Automatic incident creation from SLO violations
- **Playbooks**: Automated incident response with configurable steps
- **Chaos Testing**: Fault injection for resilience validation

## Quick Start

### Track SLOs

```python
from alphaswarm_sol.reliability.slo import SLOTracker, load_slos
from alphaswarm_sol.beads.event_store import EventStore
from alphaswarm_sol.metrics.cost_ledger import CostLedger

tracker = SLOTracker(
    event_store=EventStore(),
    cost_ledger=CostLedger(),
)

# Check single SLO
violation = tracker.check_slo("pool_success_rate")
if violation:
    print(f"SLO violated: {violation.slo_name} at {violation.measured_value}")

# Check all SLOs
violations = tracker.monitor_all_slos()
```

### Detect Incidents

```python
from alphaswarm_sol.reliability.incidents import IncidentDetector

detector = IncidentDetector(slo_tracker=tracker)

# Create incidents from violations
incidents = detector.detect_from_slo_violations(violations)

for incident in incidents:
    print(f"[{incident.severity.value}] {incident.title}")
```

### Execute Playbooks

```python
from alphaswarm_sol.reliability.playbooks import PlaybookExecutor, load_playbooks

def send_alert(severity, channel, message):
    # Your alerting logic
    print(f"[{severity}] {channel}: {message}")

executor = PlaybookExecutor(
    event_store=event_store,
    alert_sender=send_alert,
)

playbooks = load_playbooks(Path("configs/incident_playbooks.yaml"))
result = executor.execute(playbooks["pool_success_rate_degradation"], incident)
```

## Incident Playbooks

### Playbook Structure

```yaml
# configs/incident_playbooks.yaml
playbooks:
  pool_success_rate_degradation:
    name: "Pool Success Rate Degradation"
    description: "Respond to pool success rate dropping below threshold"
    trigger_slo: pool_success_rate
    trigger_threshold: 0.90
    steps:
      - name: "Check recent pool failures"
        action: query_event_store
        params:
          status: FAILED
          window_hours: 1
      - name: "Alert on-call"
        action: send_alert
        params:
          severity: high
          channel: slack
          message: "Pool success rate below {threshold}%"
```

### Available Actions

| Action | Description |
|--------|-------------|
| `query_event_store` | Query pools/beads by status, time window |
| `query_cost_ledger` | Query cost by agent, pool, time |
| `aggregate_errors` | Group errors by type |
| `send_alert` | Send alert via configured channel |
| `set_pool_intake_status` | Pause/resume pool intake |
| `update_policy` | Modify policy values |
| `log_message` | Log investigation message |

## Chaos Testing

### Setup Chaos Harness

```python
from alphaswarm_sol.reliability.chaos import (
    ChaosTestHarness,
    ChaosExperiment,
    FaultType,
    CHAOS_TEMPLATES,
)

harness = ChaosTestHarness(enabled=True, seed=42)

# Use pre-built template
harness.add_experiment(CHAOS_TEMPLATES["api_timeout_20pct"])

# Or create custom experiment
harness.add_experiment(ChaosExperiment(
    name="custom_fault",
    fault_type=FaultType.API_ERROR,
    injection_rate=0.15,
    fault_params={"status_code": 503},
    target_component="llm",
))
```

### Instrument Functions

```python
from alphaswarm_sol.reliability.chaos import with_chaos_testing

@with_chaos_testing("llm")
def call_llm(prompt: str, chaos_harness=None):
    # Normal LLM call
    return llm.generate(prompt)

# Faults injected based on experiment
result = call_llm("test", chaos_harness=harness)
```

### Fault Types

| Type | Description |
|------|-------------|
| `API_TIMEOUT` | Delay then timeout |
| `API_ERROR` | HTTP error response |
| `MALFORMED_RESPONSE` | Invalid JSON/schema |
| `AGENT_FAILURE` | Agent crash |
| `COMMUNICATION_DELAY` | Slow handoff |
| `COST_SPIKE` | Inflated cost |
| `SCHEMA_VIOLATION` | Wrong output format |
| `RATE_LIMIT` | Quota exceeded |

### Measure Resilience

```python
# Run chaos tests
for _ in range(100):
    try:
        result = call_llm("test", chaos_harness=harness)
    except Exception:
        harness.record_recovery(0.01)  # Track recovery time

# Get results
results = harness.get_results()
print(f"Success rate: {results.success_rate:.1%}")
print(f"MTTR: {results.mttr:.2f}s")

# Assert resilience
assert results.success_rate >= 0.75, "System not resilient enough"
```

# Trace Schema Reference

Complete specification for AlphaSwarm OpenTelemetry traces.

## Span Types

### Agent Span

```
name: agent.{agent_type}
kind: INTERNAL
```

**Attributes:**
| Attribute | Type | Description |
|-----------|------|-------------|
| `gen_ai.system` | string | LLM provider (anthropic, openai) |
| `gen_ai.request.model` | string | Model name |
| `gen_ai.operation.name` | string | Operation type (investigate, verify) |
| `alphaswarm.pool.id` | string | Pool identifier |
| `alphaswarm.bead.id` | string | Bead identifier |
| `alphaswarm.agent.type` | string | Agent type (vrs-attacker, etc.) |
| `alphaswarm.agent.role` | string | Agent role |

**Events:**
- `gen_ai.input`: Input summary and token estimate
- `gen_ai.output`: Output summary, tokens, cost, finish_reason

### Tool Span

```
name: tool.{tool_name}
kind: CLIENT
```

**Attributes:**
| Attribute | Type | Description |
|-----------|------|-------------|
| `alphaswarm.tool.name` | string | Tool name |
| `alphaswarm.pool.id` | string | Pool identifier |

### Handoff Span

```
name: agent.handoff
kind: INTERNAL
```

**Attributes:**
| Attribute | Type | Description |
|-----------|------|-------------|
| `alphaswarm.handoff.from_agent` | string | Source agent |
| `alphaswarm.handoff.to_agent` | string | Target agent |
| `alphaswarm.handoff.context_keys` | string[] | Context items passed |
| `alphaswarm.pool.id` | string | Pool identifier |
| `alphaswarm.bead.id` | string | Bead identifier |

### Guardrail Span

```
name: guardrail.{guardrail_type}
kind: INTERNAL
```

**Attributes:**
| Attribute | Type | Description |
|-----------|------|-------------|
| `alphaswarm.guardrail.id` | string | Guardrail identifier |
| `alphaswarm.guardrail.type` | string | Type (input, output, tool_access) |
| `alphaswarm.guardrail.result` | string | passed, failed, warning |
| `alphaswarm.pool.id` | string | Pool identifier |

## Usage Attributes (GenAI Conventions)

| Attribute | Type | Description |
|-----------|------|-------------|
| `gen_ai.usage.input_tokens` | int | Input token count |
| `gen_ai.usage.output_tokens` | int | Output token count |
| `gen_ai.usage.total_tokens` | int | Total tokens |
| `gen_ai.usage.cost_usd` | float | Cost in USD |

## Trace Context

Trace context propagates through:
- Pool metadata: `pool.trace_id`
- Bead metadata: `bead.trace_id`, `bead.parent_span_id`
- Function kwargs: `trace_id` parameter

# Policy Configuration Reference

Complete specification for `configs/governance_policies.yaml`.

## Schema

```yaml
version: "1.0"

policies:
  cost_budget:
    hard_limit_usd: float        # Required: Block threshold
    soft_limit_usd: float        # Required: Alert threshold
    per_agent_limit_usd: float   # Optional: Per-agent cap
    action_on_hard_limit: BLOCK  # ALLOW | BLOCK | SANITIZE | ALERT
    action_on_soft_limit: ALERT

  tool_access:
    {agent_type}:
      allowed: [string]          # Tools agent can use
      forbidden: [string]        # Tools agent cannot use
      max_calls_per_pool: int    # Optional: Rate limit

  evidence_integrity:
    confidence_upgrade_requires_evidence: bool
    min_evidence_refs_for_likely: int
    min_evidence_refs_for_confirmed: int
    require_provenance_chain: bool

  model_usage:
    approved_models: [string]
    tier_routing_required: bool
    fallback_model: string
```

## PolicyAction Values
