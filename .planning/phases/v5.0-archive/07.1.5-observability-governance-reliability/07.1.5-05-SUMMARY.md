## Decisions Made

**1. 8 comprehensive fault types**
- Covers API failures (timeout, error, rate limit), data corruption (malformed response, schema violation), agent failures, communication delays, and cost spikes
- Each type maps to real production failure modes

**2. Component-targeted injection**
- Faults target specific components (llm, tool, agent, handoff) for isolation
- Prevents cascading failures during testing
- Enables fine-grained resilience analysis

**3. Decorator pattern for integration**
- `@with_chaos_testing(component)` decorator enables fault injection without modifying function logic
- chaos_harness passed as optional kwarg
- Falls back gracefully when harness not provided

**4. Pre-built templates**
- 4 templates covering common scenarios (api_timeout_20pct, malformed_response_10pct, handoff_delay_15pct, tool_rate_limit_5pct)
- Sensible defaults for rapid adoption
- Templates include expected degradation metrics

**5. MTTR tracking**
- Recovery times recorded via harness.record_recovery()
- Aggregated in ChaosResult.mttr
- Enables degradation curve analysis
