## Decisions Made

### ORCH2-04: CLI Entry Point
- **Decision:** Create separate `alphaswarm` CLI entry point instead of `alphaswarm alphaswarm` subcommand
- **Rationale:** Product (alphaswarm/VRS) is conceptually separate from development tool (alphaswarm). Users may only install alphaswarm, not alphaswarm.

### Non-Destructive Installation
- **Decision:** Installation only creates .claude/vrs/ and .beads/, never modifies existing .claude/ root files
- **Rationale:** Respects user's existing Claude configuration. VRS content is isolated in .claude/vrs/ namespace.
- **Verification:** Test confirmed existing .claude/test.txt preserved after installation

### Metadata Tracking
- **Decision:** Store installation metadata in .vrs-meta.json (version, timestamp, skills count)
- **Rationale:** Enables version checking, upgrade detection, and health validation
- **Format:** JSON for easy programmatic access

### Auto Health Check
- **Decision:** Run health check automatically after init (unless --skip-health-check)
- **Rationale:** Immediate feedback on installation success. Users can skip for scripted installs.
