## Decisions Made

**Namespace organization:**
- Grouped related skills by prefix for discoverability
- Bead operations: vrs:bead-*
- Orchestration: vrs:orch-*
- Tools: vrs:tool-*

**Context isolation strategy:**
- All tool skills use `context: fork` to prevent main conversation pollution
- Bead management uses fork for write operations
- Orchestration uses fork for worker spawning

**Beads scope:**
- Only security-related Solidity work creates beads
- Non-security tasks return structured results without bead overhead

**Worker specialization:**
- 4 worker types: attacker (exploit construction), defender (guard finding), verifier (evidence validation), investigator (general analysis)
- Concurrency limits: max 5 subagents total, max 2 sub-orchestrators
- Workers report via bead updates, not direct returns

**Tool integration:**
- Auto-create beads for HIGH/CRITICAL findings
- Return summaries, not raw output
- Enable parallel tool execution via forked contexts
