**Key Features per Agent:**

**vkg-attacker (claude-opus-4):**
- Constructs exploit paths using BSKG behavioral signatures
- Estimates exploitability with confidence scoring
- Outputs attack steps with preconditions/postconditions
- Challenges defender claims in debate protocol

**vkg-defender (claude-sonnet-4):**
- Searches for guards using BSKG property queries
- Integrates protocol context (roles, invariants, accepted risks)
- Assesses guard strength (0.0-1.0 scale)
- Rebuts attacker preconditions with evidence

**vkg-verifier (claude-opus-4):**
- Synthesizes attacker/defender evidence (no new analysis)
- Applies decision matrix for verdict confidence
- Records dissent when strong evidence loses
- ALWAYS flags verdicts for human review

**vkg-supervisor (claude-sonnet-4):**
- Monitors bead lifecycle and agent handoffs
- Detects stuck work (log-only, no auto-intervention)
- Enforces concurrency limits (max 5 subagents, max 2 sub-orchestrators)
- Escalates to human for blocked work

**vkg-integrator (claude-sonnet-4):**
- Merges verdicts with evidence-weighted voting
- Resolves conflicts (documents all resolution reasoning)
- Deduplicates overlapping findings by location/pattern
- Generates human-readable summaries with recommendations
- Flags ALL verdicts for human confirmation

## Decisions Made

1. **Model selection by reasoning complexity:**
   - Attacker/Verifier use claude-opus-4 (complex reasoning: exploit construction, evidence synthesis)
   - Defender/Supervisor/Integrator use claude-sonnet-4 (fast coordination: guard search, workflow monitoring, result merging)

2. **Graph-first enforcement:**
   - All agents instructed to use BSKG queries and semantic operations
   - Manual code reading prohibited (graph provides structured access)

3. **Human oversight at every stage:**
   - Verifier always flags verdicts for review
   - Integrator enforces human confirmation before any action
   - Supervisor escalates uncertain/stuck work to human
   - Per PHILOSOPHY.md: "Human-in-the-loop for all security claims"

4. **Concurrency limits for resource control:**
   - Max 5 subagents per pool (prevents runaway agent spawning)
   - Max 2 sub-orchestrators per pool (limits nesting depth)
   - Token budget monitoring (warn at 80%, escalate at 95%)

5. **Log-only stuck work detection:**
   - Supervisor detects stuck work but does NOT auto-retry or intervene
   - Logs issue and escalates to human for decision
   - Prevents infinite loops and resource exhaustion
