    # VRS Installation Guide

    This guide covers installing VRS (Vulnerability Research System) in your Solidity project.

    ## Prerequisites

    - Python 3.10+
    - Slither (required): `pip install slither-analyzer`
    - Claude Code (for skill invocation)

    ## Quick Start

    1. **Install alphaswarm:**
       ```bash
       pip install alphaswarm-sol
       # or
       uv add alphaswarm-sol
       ```

    2. **Initialize VRS in your project:**
       ```bash
       cd your-solidity-project
       alphaswarm init
       ```

    3. **Run health check:**
       ```bash
       alphaswarm health-check
       ```

    ## What `init` Does

    The `alphaswarm init` command:
    - Creates `.claude/vrs/` with VRS skills
    - Creates `.beads/` for investigation tracking
    - Does NOT modify existing `.claude/` files

    ## Directory Structure

    After initialization:

    ```
    your-project/
    ├── .claude/
    │   └── vrs/
    │       ├── audit.md          # Main audit skill
    │       ├── investigate.md    # Investigation skill
    │       ├── verify.md         # Verification skill
    │       ├── debate.md         # Debate protocol skill
    │       ├── health-check.md   # Health check skill
    │       ├── bead-*.md         # Bead management skills
    │       ├── tool-*.md         # Tool execution skills
    │       └── agents/           # Agent definitions
    ├── .beads/
    │   └── index.jsonl           # Bead tracking
    └── contracts/                # Your Solidity code
    ```

    ## Reinstalling

    To update or reinstall:

    ```bash
    alphaswarm init --force
    ```

    ## Optional Tools

    VRS works best with additional tools:

    | Tool | Install | Purpose |
    |------|---------|---------|
    | Aderyn | `cargo install aderyn` | Rust-based analysis |
    | Mythril | `pip install mythril` | Symbolic execution |
    | Foundry | `curl -L https://foundry.paradigm.xyz | bash` | Test execution |

    Run `alphaswarm health-check` to see tool status.

    ## Troubleshooting

    **Skills not found:**
    ```bash
    alphaswarm health-check --verbose
    ```

    **Missing tools:**
    The health check will suggest installation commands.

    **Reinstalling:**
    Use `--force` to overwrite existing installation.
    ```

    # VRS Skills Guide

    VRS skills are Claude Code skills for Solidity security auditing.

    ## Skill Namespaces

    All VRS skills use the `vrs:` namespace:

    | Prefix | Purpose |
    |--------|---------|
    | `/vrs-audit` | Main audit orchestration |
    | `/vrs-investigate` | Deep investigation |
    | `/vrs-verify` | Multi-agent verification |
    | `/vrs-debate` | Structured debate |
    | `/vrs-bead-*` | Bead management |
    | `/vrs-tool-*` | Tool execution |
    | `/vrs-orch-*` | Orchestration |

    ## Core Skills

    ### `/vrs-audit`

    Run a full security audit on your contracts.

    **Usage:**
    ```
    /vrs-audit contracts/
    ```

    **What it does:**
    1. Builds knowledge graph with Slither
    2. Detects vulnerability patterns
    3. Creates beads for findings
    4. Spawns verification agents
    5. Produces final report

    ### `/vrs-investigate`

    Deep-dive investigation of a specific finding.

    **Usage:**
    ```
    /vrs-investigate bd-a1b2c3d4
    ```

    ### `/vrs-verify`

    Multi-agent verification with attacker/defender/verifier roles.

    **Usage:**
    ```
    /vrs-verify bd-a1b2c3d4
    ```

    ### `/vrs-health-check`

    Validate your VRS installation.

    **Usage:**
    ```
    /vrs-health-check
    ```

    ## Bead Skills

    ### `/vrs-bead-create`

    Create a new investigation bead.

    ### `/vrs-bead-update`

    Update bead status.

    ### `/vrs-bead-list`

    List beads with filtering.

    ## Tool Skills

    ### `/vrs-tool-slither`

    Run Slither analysis.

    ### `/vrs-tool-aderyn`

    Run Aderyn analysis (if installed).

    ## Context Isolation

    Most skills run with `context: fork` for isolation:
    - Tool output doesn't pollute main context
    - Each investigation gets fresh context
    - Large outputs are summarized

    ## Development vs Product Skills

    **Product skills** (shipped with VRS):
    - `/vrs-audit`, `/vrs-investigate`, `/vrs-verify`, etc.
    - Installed via `alphaswarm init`

    **Development skills** (VKG contributors only):
    - `/vrs-discover`, `/vrs-refine`, etc.
    - Pattern creation and testing
    - Only in alphaswarm repo

    # Beads Guide

    Beads are self-contained investigation packages for tracking security work.

    ## What Are Beads?

    A bead represents a single unit of security investigation work:
    - Potential vulnerability to investigate
    - Finding to verify
    - Test to write

    Beads follow the [steveyegge/beads](https://github.com/steveyegge/beads) pattern for AI-optimized task tracking.

    ## Bead Storage

    Beads are stored in `.beads/index.jsonl`:
    - JSONL format (one JSON object per line)
    - Git-friendly (append-only, line-based diffs)
    - Survives agent restarts

    ## Bead Lifecycle

    ```
    open -> in_progress -> complete
              |
              v
           blocked -> open (when blocker completes)
    ```

    ## Using Beads

    ### Create a Bead

    ```bash
    alphaswarm bead create "Check reentrancy in withdraw" -s high
    ```

    ### List Beads

    ```bash
    alphaswarm bead list
    alphaswarm bead list --status open
    alphaswarm bead list --ready  # Open and not blocked
    ```

    ### Update Status

    ```bash
    alphaswarm bead update bd-a1b2c3d4 --status in_progress
    alphaswarm bead update bd-a1b2c3d4 --status complete
    ```

    ### Show Details

    ```bash
    alphaswarm bead show bd-a1b2c3d4
    ```

    ## Bead Fields

    | Field | Description |
    |-------|-------------|
    | `id` | Unique ID (bd-{hash}) |
    | `title` | Brief description |
    | `status` | open/in_progress/complete/blocked |
    | `severity` | critical/high/medium/low |
    | `priority` | 0 = highest |
    | `pattern_id` | Pattern that triggered this |
    | `location` | File:function location |
    | `blockers` | IDs of blocking beads |
    | `work_state` | State for resumption |

    ## Beads in Skills

    Skills create and update beads automatically:

    ```
    /vrs-audit contracts/
    # Creates beads for each finding

    /vrs-investigate bd-a1b2c3d4
    # Updates bead status to in_progress, then complete
    ```

    ## JSON Output

    All bead commands support `--json`:

    ```bash
    alphaswarm bead list --json
    alphaswarm bead show bd-a1b2c3d4 --json
    ```

    ## Git Integration

    The `.beads/` directory can be:
    - **Tracked**: Share investigation state with team
    - **Ignored**: Keep local only (add to `.gitignore`)

    Recommendation: Track for audit trails, ignore for exploration.
    ```
