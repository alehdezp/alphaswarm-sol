    **bead-create.md** - Create new security investigation bead:
    ```yaml
    ---
    name: vrs-bead-create
    description: Create a new security investigation bead
    slash_command: vrs:bead-create
    context: fork
    disable-model-invocation: false
    allowed-tools: Bash(alphaswarm:*), Write
    ---
    ```
    Body:
    - Arguments: title, severity (critical/high/medium/low), pattern_id (optional), location (file:function)
    - Execution: `alphaswarm bead create "$TITLE" --severity $SEVERITY --pattern "$PATTERN_ID" --location "$FILE:$FUNCTION"`
    - Output: Returns bead ID (format: bd-{hash[:8]})
    - Note: Only security-related Solidity work creates beads

    **bead-update.md** - Update bead status:
    ```yaml
    ---
    name: vrs-bead-update
    description: Update status of an existing bead
    slash_command: vrs:bead-update
    context: fork
    disable-model-invocation: false
    allowed-tools: Bash(alphaswarm:*), Write
    ---
    ```
    Body:
    - Arguments: bead_id, status (open/in_progress/complete/blocked), notes (optional)
    - Execution: `alphaswarm bead update $BEAD_ID --status $STATUS --notes "$NOTES"`
    - For blocked status, can specify `--blocked-by $OTHER_BEAD_ID`

    **bead-list.md** - List beads with filtering:
    ```yaml
    ---
    name: vrs-bead-list
    description: List beads with optional filtering
    slash_command: vrs:bead-list
    disable-model-invocation: false
    allowed-tools: Bash(alphaswarm:*)
    ---
    ```
    Body:
    - Arguments: status filter (optional), severity filter (optional)
    - Execution: `alphaswarm bead list --status $STATUS --severity $SEVERITY`
    - Output: Table of beads with ID, title, status, severity, priority
  </action>

    ```yaml
    ---
    ```

    **orch-resume.md** - Resume interrupted work:
    ```yaml
    ---
    name: vrs-orch-resume

    slash_command: vrs:orch-resume
    context: fork
    disable-model-invocation: true
    allowed-tools: Bash(alphaswarm:*), Read
    ---
    ```
    Body:
    - Lists in_progress and blocked beads
    - Prioritizes highest priority bead for auto-resume
    - Presents others for user decision
    - Uses work_state from bead for resumption context
