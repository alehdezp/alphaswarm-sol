## Tool Permission Patterns

### File Operations
- `Read` - Read files from allowed paths
- `Write` - Create new files (architect, curator only)
- `Edit` - Modify existing files (architect, curator only)
- `Glob` - Find files by pattern
- `Grep` - Search file contents

### Command Execution (Wildcard Patterns)
- `Bash(uv run alphaswarm query*)` - Query BSKG
- `Bash(uv run alphaswarm build-kg*)` - Build knowledge graph
- `Bash(uv run alphaswarm context*)` - Context operations
- `Bash(uv run alphaswarm orchestrate*)` - Pool management
- `Bash(uv run alphaswarm vulndocs*)` - VulnDocs operations
- `Bash(uv run pytest*)` - Run tests
- `Bash(python scripts/validate_skill_schema.py*)` - Validate schemas

### MCP Tools
- `mcp__exa-search__web_search_exa` - Deep web research (researcher only)

## Data Access Policies

### Forbidden Paths (All Roles)
- `**/.env` - Environment variables
- `**/secrets/**` - Secrets directory
- `**/*key*` - Key files
- `**/*credential*` - Credential files

### Protected Files (Cannot Modify)
- `src/alphaswarm_sol/kg/builder_legacy.py`
- `src/alphaswarm_sol/beads/executor.py`

## Decisions Made

### 1. 10 Core Roles

**Decision:** Define 10 roles covering all VRS workflows

**Rationale:**
- Covers investigation (attacker, defender, verifier)
- Covers development (architect, curator, tester)
- Covers operations (orchestrator, triage)
- Covers research (researcher, auditor)
- Prevents role proliferation while maintaining granularity

### 2. Wildcard Tool Patterns

**Decision:** Use wildcard patterns for Bash commands (e.g., `Bash(uv run alphaswarm query*)`)

**Rationale:**
- Flexibility for command variations (flags, arguments)
- Prevents need to enumerate every possible command variant
- Still maintains control (only specific command prefixes allowed)
- Easier to maintain than exhaustive lists

### 3. Forbidden Paths Override Allowed

**Decision:** Forbidden path patterns take precedence over allowed patterns

**Rationale:**
- Security-first approach (deny trumps allow)
- Prevents accidental access to secrets via broad allowed patterns
- Simplifies policy definition (can use broad allowed + specific forbids)

### 4. Escalation Rules for Expensive Operations

**Decision:** Require triage validation for token_budget > 6000 and opus models

**Rationale:**
- Cost control without blocking workflows
- Cheap model (triage) validates before expensive model usage
- Prevents accidental high-cost runs
- Enables human-in-the-loop for critical decisions

### 5. Evidence Requirements Per Role

**Decision:** Enforce evidence for verification roles, optional for operational roles

**Rationale:**
- Investigation roles (attacker, defender, verifier) must provide evidence
- Operational roles (orchestrator, triage) don't make claims
- Development roles (architect) create skills, don't verify contracts
- Aligns with role responsibilities

### 6. Graph-First Requirements Per Role

**Decision:** Require graph-first for investigation roles, optional for others

**Rationale:**
- Investigation requires BSKG queries before conclusions
- Research may precede graph construction
- Development and operations don't analyze contracts directly
- Aligns with graph-first enforcement template (Plan 03)

## Security Considerations

1. **Secrets Protection:** All roles forbidden from accessing `.env`, `secrets/`, credential files
2. **Protected Files:** `builder_legacy.py` and `executor.py` cannot be modified by any role
3. **Command Injection:** Only specific bash patterns allowed, not arbitrary commands
4. **Path Traversal:** Forbidden paths checked before allowing file access
5. **Cost Control:** Escalation rules prevent expensive model usage without validation

## Anti-Patterns Prevented

### ❌ Overbroad Tool Permissions
**Blocked:** `Bash(*)` - Allows ANY bash command
**Required:** `Bash(uv run alphaswarm query*)` - Specific command pattern

### ❌ Missing Evidence Requirements
**Blocked:** `evidence_required: false` for verification roles
**Required:** `evidence_required: true` for attacker/defender/verifier

### ❌ Unbounded File Access
**Blocked:** `max_file_reads: 999999` - Effectively unbounded
**Required:** Reasonable limits based on role (20-150 reads)

### ❌ Write Access for Investigation Roles
**Blocked:** `Write` tool for attacker/defender/verifier
**Required:** Read-only access (only architect/curator can write)
