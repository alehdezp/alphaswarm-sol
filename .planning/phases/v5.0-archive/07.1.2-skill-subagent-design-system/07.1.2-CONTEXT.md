# Phase 7.1.2: Skill & Subagent Design System - Context

## Purpose
Create a production-grade skill and subagent design system for Claude Code and Codex, with strong contracts, evidence-first outputs, and graph-first reasoning. This is the standardization layer for role orchestration.

## Scope
- Skill schema v2 (role, evidence, tools, output contract, failure modes)
- Role prompts for security-focused Solidity review with creative adversarial thinking
- Graph-first reasoning templates (mandatory BSKG query blocks)
- Subagent catalog: attacker/defender/verifier/researcher/triage
- Guardrails for tool usage and data access per role
- Skill versioning, test fixtures, and deprecation policy

## Out of Scope
- New orchestration runtime (handled in 7.1.1)
- Framework adapter implementations (handled in 7.1.4)

## Key Constraints
- Prompts must enforce evidence linkage to exact code locations
- Agents must use BSKG queries before reading code
- Skills must remain token-budget aware (fit within <6k tokens)

## Desired Outcomes
- Consistent role behavior across Claude Code and Codex
- Reduced hallucinations via evidence-first contracts
- Skills are testable, versioned, and safe to evolve

## Example Role Traits
- "Secure Solidity reviewer" with adversarial + creative thinking
- "Graph-first analyst" that composes BSKG queries before reasoning
- "Verifier" that refuses claims without evidence packets
