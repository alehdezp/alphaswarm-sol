## Template Structure

The graph-first template enforces this workflow:

### 1. BSKG Queries (MANDATORY)
- Run queries BEFORE any analysis
- Document query intent (what + why)
- Use VQL, natural language, or pattern queries
- Examples: Find reentrancy candidates, access control issues, missing guards

### 2. Evidence Packet (MANDATORY)
```yaml
evidence_packet:
  finding_id: "VKG-{id}"
  evidence_items:
    - type: "{code_property|operation_sequence|missing_guard|behavioral_signature}"
      graph_node_id: "{Contract.function}"
      property_checked: "{property name}"
      code_location: "{file}:{line}"
      confidence: {0.0-1.0}
  operation_sequences:
    - signature: "{e.g., R:bal->X:out->W:bal}"
      operations: [...]
  missing_guards:
    - guard_type: "{reentrancy_guard|access_control|...}"
```

### 3. Unknowns/Gaps (MANDATORY)
- Explicitly list what's missing or unclear
- Mark taint analysis limitations, missing protocol context
- Rule: "no evidence ≠ safe" - unknowns must be documented

### 4. Conclusion (MANDATORY)
- Only after steps 1-3
- Reference evidence items by ID
- Calculate confidence with unknowns penalty
- Include caveats for gaps

## Integration Points

### Agents
- **Attacker**: Query for attack vectors → build exploit evidence
- **Defender**: Query for guards → build defense evidence
- **Verifier**: Validate evidence quality → synthesize verdict

### Skills
- **graph-retrieve**: Execute queries → build retrieval packs with evidence refs
- **Future skills**: Will reference template for consistency

### Enforcement
- Verifier validates evidence packets (ORCH-09, ORCH-10)
- Claims without graph node IDs penalized (-0.3 confidence)
- Missing unknowns section penalized (-0.2 confidence)
- UNCERTAIN verdict if BSKG queries missing

## Decisions Made

1. **Template as reference, not duplication**: Agents reference `docs/reference/graph-first-template.md` rather than duplicating full content to reduce maintenance burden

2. **Strict query-before-analysis rule**: Forbid manual code reading before BSKG queries to ensure graph-first behavior is not bypassed

3. **Evidence packet schema with graph node IDs**: Every claim must include graph node ID and code location for traceability and verification

4. **Mandatory unknowns section**: Force explicit documentation of gaps to avoid silent false negatives from missing evidence

5. **Confidence penalty for unknowns**: Quantify impact of gaps (-0.1 per unknown, -0.2 for missing protocol context)
