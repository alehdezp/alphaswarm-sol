## Decisions Made

**1. Token budget default of 6000 tokens**
- **Rationale:** Fits comfortably within 8k model context window, leaves 2k buffer for system prompts and tool outputs
- **Override guidance:** < 6k for quick checks, > 6k for complex orchestration, never exceed 10k without justification

**2. Role enum constrained to 10 core roles**
- **Roles:** attacker, defender, verifier, auditor, researcher, triage, orchestrator, architect, curator, tester
- **Rationale:** Covers all current VRS workflows while preventing role proliferation
- **Extension:** New roles require schema update and justification

**3. Evidence requirements mandatory**
- **Required subfields:** must_link_code, min_evidence_items, cite_graph_nodes
- **Optional subfields:** graph_first, require_behavioral_signature
- **Rationale:** Enforces evidence-first reasoning, prevents hallucinations

**4. Output contracts support dual schema mode**
- **External reference:** Path to JSON schema file (reusable, maintainable)
- **Inline definition:** JSON schema fragment (self-contained, single-use)
- **Rationale:** Flexibility for shared schemas vs one-off contracts

**5. Validator supports strict/warn modes**
- **Default:** Fail on invalid frontmatter
- **--strict:** Also fail on missing frontmatter
- **--warn:** Report violations but exit 0
- **Rationale:** Enables gradual migration without breaking existing workflows
