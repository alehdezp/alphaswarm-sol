# Phase 07.3.1.5 Plan 09: Failure Taxonomy & Self-Improving Loop Summary

Explicit failure taxonomy with 35 patterns, self-improving retry loop with delta tracking, and failure injection runner for recovery verification.

---

## What Was Built

### 1. Failure Taxonomy (35 patterns)

Enhanced `failure_catalog.py` with explicit failure type classification per W8 critique:

| FailureType | Description | Patterns |
|-------------|-------------|----------|
| INFRA_FAILURE | claude-code-agent-teams crash, worktree fail, disk full | 5 |
| TIMEOUT_FAILURE | command exceeded limit, stalled | 3 |
| DETECTION_FAILURE | expected vs actual mismatch | 4 |
| MISSING_MARKER | expected content not found | 3 |
| EVIDENCE_FAILURE | missing/corrupt artifacts | 4 |
| TOOL_ERROR | slither, aderyn crashed | 5 |
| CONTEXT_ERROR | protocol context failed | 3 |
| LABEL_ERROR | semantic labeling failed | 3 |
| ASSERTION_FAILURE | test logic error | 3 |
| UNKNOWN | architectural issues | 2 |

Each pattern maps to:
- **Category**: FIXABLE_IN_SESSION, FIXABLE_WITH_CODE, NOT_FIXABLE
- **Remediation**: Human-readable fix guidance
- **RerunStrategy**: SAME_SCENARIO, FRESH_WORKTREE, SUBSET, SKIP, FULL_SUITE
- **Priority**: Lower = matched first (infra=10, detection=50, assert=100)

### 2. Self-Improving Loop

Implemented `self_improving_loop.py` with:

- **Requeue logic**: Queue scenarios for targeted re-runs based on failure analysis
- **Fresh worktree support**: Create isolated git worktrees for clean reruns
- **Delta tracking**: Record improvements between iterations with evidence pack IDs
- **Human review flag**: After 3 consecutive failures on same pattern
- **E009 abort**: After 10 total retries, abort with postmortem report

Key classes:
```python
SelfImprovingLoop(run_scenario_fn, config)
  .requeue(scenario_id, failure)
  .run_iteration() -> IterationSummary
  .run_until_complete() -> list[IterationSummary]

DeltaNote(scenario_id, iteration_from, iteration_to, failure_before, failure_after, improvement, evidence_pack_id)
```

### 3. Failure Injection Runner

Created `scripts/e2e/run_failure_injection.sh` with 14 injection cases:

| Category | Cases | What's Injected |
|----------|-------|-----------------|
| tools | 3 | Missing slither, aderyn, alphaswarm |
| graph | 4 | Invalid JSON, empty file, missing nodes, missing dir |
| context | 3 | Missing pack, corrupted YAML, missing dir |
| labels | 4 | Missing overlay, corrupted JSON, missing dir, invalid refs |

Usage:
```bash
# Dry-run to see all cases
./scripts/e2e/run_failure_injection.sh --dry-run

# Run all injection tests
./scripts/e2e/run_failure_injection.sh

# Run only tool failures
./scripts/e2e/run_failure_injection.sh --inject tools
```

## Implementation Details

### Failure Pattern Priority Order

```
10-19: INFRA_FAILURE (critical infrastructure)
20-29: TIMEOUT_FAILURE (timing issues)
50-59: DETECTION_FAILURE (core validation)
60-69: MISSING_MARKER (expected content)
70-79: EVIDENCE_FAILURE (artifacts)
80-89: TOOL_ERROR (external tools)
90-94: CONTEXT_ERROR (protocol context)
95-99: LABEL_ERROR (semantic labels)
100+:  ASSERTION_FAILURE (test logic)
200+:  NOT_FIXABLE (architectural)
```

### Rerun Strategy Selection

| Strategy | When Used | Example Failure |
|----------|-----------|-----------------|
| SAME_SCENARIO | Fixable, state preserved | Missing dependency |
| FRESH_WORKTREE | Need clean state | claude-code-agent-teams session crash |
| SKIP | Not auto-fixable | Detection failure |
| FULL_SUITE | Infra-wide issue | Disk full |

### Delta Tracking Format

```json
{
  "scenario_id": "S01",
  "iteration_from": 1,
  "iteration_to": 2,
  "failure_before": "FAIL_TOOL_001",
  "failure_after": null,
  "improvement": true,
  "evidence_pack_id": "pack-001",
  "notes": "Improved: FAIL_TOOL_001 -> passed"
}
```
