## Implementation Details (v2)
- Runbook must include claude-code-agent-teams setup, worktree layout, record/diff flows, and recovery steps.
- Gates must be explicit with pass/fail thresholds (precision/recall, debug coverage, scenario coverage).
- Unified runner must execute orchestrator → gates → report → evidence collection; any gate failure stops the run.

## Strict Quality Controls
- Enforce claude-code-controller usage as a hard gate; missing markers invalidate the run.
  **Full claude-code-controller documentation:** See `.planning/testing/rules/claude-code-controller-REFERENCE.md` and `.planning/testing/rules/canonical/claude-code-controller-instructions.md`
- Validate debug-mode artifacts for every skill/agent per the debug matrix.
- Evidence packs are mandatory inputs to gate evaluation; missing packs fail the run.

## Iteration Protocol (max 10, real runs only)
- Run up to 10 real iterations per phase: run → evidence pack → gates → failure classification → fix → re-run.
- Any missing evidence pack artifacts, missing claude-code-agent-teams markers, or duration <= 5000ms is an automatic fail.
- No simulated results, no fabricated metrics, no mock detections; only real claude-code-agent-teams-driven runs count.
- After 10 failed iterations, stop and produce a postmortem with evidence pack links and remediation plan.

## Core Philosophy (Must Internalize)
- Real-world, human-operated CLI testing is mandatory; simulations and mocks are invalid for validation.
- Claude Code must operate as the human operator via `claude-code-controller` in `alphaswarm-claude-code-agent-teams-session`.
- Every workflow and subworkflow requires real claude-code-agent-teams transcripts and evidence packs.
