# Gate Failure Taxonomy

This document defines the failure categories, severity classifications, and remediation guidance for each QA gate in the `/vrs-full-testing` validation pipeline.

---

## Failure Categories

### Category Overview

| Category | Description | Typical Cause | Impact |
|----------|-------------|---------------|--------|
| **CRITICAL** | Blocks GA release | Fundamental validation failure | Must fix before release |
| **OBSERVABILITY** | Blocks GA release | Missing debug/trace artifacts | Re-run with debug enabled |
| **FUNCTIONAL** | Blocks GA release | Feature not working as designed | Code or config fix required |
| **QUALITY** | Warning only | Below optimal thresholds | Review but not blocking |
| **VALIDATION** | Blocks GA release | Test infrastructure issue | Re-run or fix harness |

---

## Gate Failure Classifications

### G01: Stage Completion (CRITICAL)

**Blocking:** Yes

**Failure Modes:**

| Failure Code | Description | Severity | Required Action |
|--------------|-------------|----------|-----------------|
| G01-F01 | Suite s1_command failed | critical | Retry - command infrastructure |
| G01-F02 | Suite s2_agent failed | critical | Retry - agent workflows |
| G01-F03 | Suite s3_context failed | critical | Retry - context detection |
| G01-F04 | Suite s4_resume failed | critical | Retry - snapshot/resume |
| G01-F05 | No suites found | critical | Redesign - manifest incomplete |

**Remediation:**

1. **Retry** (G01-F01 to G01-F04): Re-run the failed suite with `--only-suite <id>`
2. **Redesign** (G01-F05): Evidence pack manifest is malformed; regenerate

**Example Failure:**
```
GATE_FAIL: G01 - Suites failed: s2_agent: failed
  Details: { "failed_suites": ["s2_agent: failed"] }
```

---

### G02: Precision (CRITICAL)

**Blocking:** Yes

**Failure Modes:**

| Failure Code | Description | Severity | Required Action |
|--------------|-------------|----------|-----------------|
| G02-F01 | Precision < 0.85 | critical | Redesign - reduce false positives |
| G02-F02 | Precision metric missing | critical | Retry - accuracy.json not generated |
| G02-F03 | Precision = 1.0 (suspicious) | warning | Investigate - possible mock data |

**Remediation:**

1. **Redesign** (G02-F01): Analyze top false positives, refine patterns
   - Run: `uv run alphaswarm vulndocs analyze-fps --evidence <pack>`
   - Review patterns with highest FP rate
   - Add negative test cases or tighten pattern constraints

2. **Retry** (G02-F02): Ensure metrics computation step completes

3. **Investigate** (G02-F03): 100% precision is suspicious; verify real execution

**Example Failure:**
```
GATE_FAIL: G02 - Precision 0.720 < 0.85 threshold
  Details: { "top_false_positives": ["S42", "S55", "S61"] }
```

---

### G03: Recall (CRITICAL)

**Blocking:** Yes

**Failure Modes:**

| Failure Code | Description | Severity | Required Action |
|--------------|-------------|----------|-----------------|
| G03-F01 | Recall < 0.80 | critical | Redesign - reduce false negatives |
| G03-F02 | Recall metric missing | critical | Retry - accuracy.json not generated |
| G03-F03 | Recall = 1.0 (suspicious) | warning | Investigate - possible mock data |

**Remediation:**

1. **Redesign** (G03-F01): Analyze false negatives, improve coverage
   - Run: `uv run alphaswarm vulndocs analyze-fns --evidence <pack>`
   - Add missing patterns or expand detection scope
   - Check if label dependencies are satisfied (Tier-C)

2. **Retry** (G03-F02): Ensure metrics computation step completes

**Example Failure:**
```
GATE_FAIL: G03 - Recall 0.650 < 0.80 threshold
  Details: { "false_negatives": ["S12-reentrancy", "S33-access"] }
```

---

### G04: F1 Score (CRITICAL)

**Blocking:** Yes

**Failure Modes:**

| Failure Code | Description | Severity | Required Action |
|--------------|-------------|----------|-----------------|
| G04-F01 | F1 < 0.82 | critical | Redesign - balance precision/recall |
| G04-F02 | F1 metric missing | critical | Retry - accuracy.json not generated |

**Remediation:**

1. **Redesign** (G04-F01): F1 combines precision and recall
   - If precision low: focus on reducing FPs (G02 guidance)
   - If recall low: focus on reducing FNs (G03 guidance)
   - If both marginal: may need pattern overhaul

2. **Retry** (G04-F02): Ensure metrics computation step completes

**Example Failure:**
```
GATE_FAIL: G04 - F1 0.780 < 0.82 threshold
```

---

### G05: Debug Coverage (OBSERVABILITY)

**Blocking:** Yes

**Failure Modes:**

| Failure Code | Description | Severity | Required Action |
|--------------|-------------|----------|-----------------|
| G05-F01 | Debug mode disabled | critical | Retry - set ALPHASWARM_DEBUG_MODE=true |
| G05-F02 | Debug records invalid | critical | Redesign - fix debug capture code |
| G05-F03 | Missing debug records | critical | Retry - ensure all skills/agents instrumented |
| G05-F04 | Zero-duration records | critical | Redesign - mock execution detected |

**Remediation:**

1. **Retry** (G05-F01, G05-F03): Re-run with debug mode enabled
   ```bash
   export ALPHASWARM_DEBUG_MODE=true
   ./run_full_testing_all.sh
   ```

2. **Redesign** (G05-F02, G05-F04): Debug capture code needs fixes
   - Check `src/alphaswarm_sol/debug/capture.py`
   - Ensure timestamps and durations are real
   - Verify all required fields populated

**Example Failure:**
```
GATE_FAIL: G05 - Debug records invalid: 3 errors
  Details: { "validation_errors": ["E014: duration_ms=0 for vrs-attacker"] }
```

### G06: Tier-C Detection (FUNCTIONAL)

**Blocking:** Yes

**Failure Modes:**

| Failure Code | Description | Severity | Required Action |
|--------------|-------------|----------|-----------------|
| G06-F01 | No Tier-C TPs | critical | Redesign - label-dependent patterns broken |
| G06-F02 | Tier-C category missing | critical | Retry - accuracy.json missing category |

**Remediation:**

1. **Redesign** (G06-F01): Label-dependent patterns not firing
   - Verify semantic labels are being applied (`uv run alphaswarm label --verify`)
   - Check Tier-C pattern definitions in vulndocs
   - Run label-only scenarios: `--only-suite s85-s89`

2. **Retry** (G06-F02): Ensure Tier-C scenarios included in run

**Example Failure:**
```
GATE_FAIL: G06 - No Tier-C true positives detected
  Details: { "tier_c_stats": { "true_positives": 0, "total_scenarios": 5 } }
```

---

### G07: Economic Context (FUNCTIONAL)

**Blocking:** Yes

**Failure Modes:**

| Failure Code | Description | Severity | Required Action |
|--------------|-------------|----------|-----------------|
| G07-F01 | No oracle scenarios passed | critical | Redesign - oracle detection broken |
| G07-F02 | No economic scenarios included | critical | Retry - scenarios not in run |
| G07-F03 | Context pack not generated | critical | Redesign - context generation failed |

**Remediation:**

1. **Redesign** (G07-F01, G07-F03): Oracle/economic context detection broken
   - Verify context pack generation: `uv run alphaswarm context generate <contracts>`
   - Check oracle detection in `src/alphaswarm_sol/context/oracle_detector.py`
   - Run oracle scenarios: `--only-suite s21-s30`

2. **Retry** (G07-F02): Include economic scenarios in test run

**Example Failure:**
```
GATE_FAIL: G07 - No oracle scenarios passed (5 failed)
  Details: { "failed_scenarios": ["S21", "S22", "S23", "S24", "S25"] }
```

---

### G08: Agent Agreement (QUALITY)

**Blocking:** No (warning only)

**Failure Modes:**

| Failure Code | Description | Severity | Required Action |
|--------------|-------------|----------|-----------------|
| G08-F01 | Agreement < 0.90 | warning | Review - agent disagreement |
| G08-F02 | Agreement metrics missing | info | Retry - agreement.json not generated |

**Remediation:**

1. **Review** (G08-F01): Multi-agent disagreement is informational
   - Review disagreement scenarios for potential false positives
   - High disagreement may indicate unclear specifications
   - No action required for GA, but track for improvement

2. **Retry** (G08-F02): Ensure debate phase completes

**Example Warning:**
```
GATE_WARN: G08 - Agreement 0.870 < 0.90 threshold
  Details: { "disagreement_scenarios": ["S12", "S45"] }
```

---

### G09: Transcript Validity (VALIDATION)

**Blocking:** Yes

**Failure Modes:**

| Failure Code | Description | Severity | Required Action |
|--------------|-------------|----------|-----------------|
| G09-F01 | Evidence pack invalid | critical | Retry - pack generation failed |
| G09-F02 | Missing transcripts | critical | Retry - claude-code-agent-teams capture failed |
| G09-F03 | Invalid markers | critical | Redesign - marker format broken |
| G09-F04 | Pack validation errors | critical | Retry - specific validation failed |

**Remediation:**

1. **Retry** (G09-F01, G09-F02, G09-F04): Re-run with transcript capture verified
   - Ensure claude-code-agent-teams sessions persist long enough
   - Check disk space for transcript storage
   - Verify marker injection in skills

2. **Redesign** (G09-F03): Marker format needs correction
   - Check marker patterns in `full_testing_markers.yaml`
   - Verify skill output format matches expected markers

**Example Failure:**
```
GATE_FAIL: G09 - Evidence pack invalid
  Details: { "errors": ["Missing ALPHASWARM-END for S42"] }
```

---

### G10: Duration Validity (VALIDATION)

**Blocking:** Yes

**Failure Modes:**

| Failure Code | Description | Severity | Required Action |
|--------------|-------------|----------|-----------------|
| G10-F01 | Duration <= 5000ms | critical | Redesign - mock execution detected |
| G10-F02 | No scenarios found | critical | Retry - manifest incomplete |

**Remediation:**

1. **Redesign** (G10-F01): Scenario ran too fast, indicating mocked execution
   - Verify real Claude Code execution, not simulation
   - Check for hardcoded outputs bypassing execution
   - Minimum 5 seconds ensures actual LLM round-trip

2. **Retry** (G10-F02): Evidence pack manifest incomplete

**Example Failure:**
```
GATE_FAIL: G10 - 1 scenarios with invalid duration
  Details: { "invalid_scenarios": [{ "id": "S21", "duration_ms": 100 }] }
```

## Regression Failures

When running against a baseline, additional regression checks apply.

### Regression Thresholds

| Metric | Max Regression | Failure Code |
|--------|----------------|--------------|
| Precision | -5% | REG-PRECISION |
| Recall | -5% | REG-RECALL |
| F1 Score | -3% | REG-F1 |

**Remediation for Regression:**

1. Identify which scenarios regressed vs baseline
2. Check if pattern changes caused regression
3. Either fix patterns or update baseline with justification

## Decision Matrix

Given a gate failure, use this matrix to determine action:

| Gate Failed | Category | Retry? | Redesign? | Skip? |
|-------------|----------|--------|-----------|-------|
| G01 | critical | Yes (suite) | If manifest broken | Never |
| G02 | critical | If metric missing | If precision low | Never |
| G03 | critical | If metric missing | If recall low | Never |
| G04 | critical | If metric missing | If F1 low | Never |
| G05 | observability | Yes (debug off) | If capture broken | Never |
| G06 | functional | If Tier-C missing | If patterns broken | Never |
| G07 | functional | If scenarios missing | If context broken | Never |
| G08 | quality | N/A (warning) | N/A | N/A |
| G09 | validation | Yes (transcript issues) | If markers broken | Never |
| G10 | validation | If scenarios missing | If mock detected | Never |

## Automated Triage

The gate evaluator provides failure codes that can be used for automated triage:

```python
# Pseudo-code for automated triage
def triage_gate_failure(gate_id: str, failure_code: str) -> Action:
    if failure_code.endswith("-F01"):  # Primary failure
        if gate_id in ["G05", "G09"]:
            return Action.RETRY_WITH_CONFIG
        elif gate_id in ["G02", "G03", "G04", "G06", "G07"]:
            return Action.REDESIGN_PATTERNS
        else:
            return Action.INVESTIGATE
    elif failure_code.endswith("-F02"):  # Missing data
        return Action.RETRY_FULL
    else:
        return Action.MANUAL_REVIEW
```

## Failure Escalation

### Escalation Path

1. **Immediate Retry (1-2 attempts)**
   - Infrastructure failures (G01 suite fails, G09 transcripts)
   - Missing data (G02-F02, G05-F01)

2. **Configuration Fix Then Retry**
   - Debug mode disabled (G05-F01)
   - Scenarios missing (G06-F02, G07-F02)

3. **Redesign Required**
   - Metric thresholds not met (G02-F01, G03-F01, G04-F01)
   - Mock execution detected (G05-F04, G10-F01)
   - Pattern/detection broken (G06-F01, G07-F01)

4. **GA Block (No Workaround)**
   - Any blocking gate that cannot be fixed
   - Must be resolved before release
