# 07.3.1.5 Skill and Agent Coverage Map

Goal: ensure EVERY shipped skill and subagent is exercised in /vrs-full-testing with debug mode enabled.

**Enforcement:** Missing coverage is a HARD FAIL (E011). Runtime validator rejects runs with incomplete coverage.

---

## Coverage Enforcement Rules

### R1: Complete Skill Coverage (MANDATORY)

Every shipped skill MUST have at least one scenario that exercises it:
- Scenario ID listed in `covered_by` field
- At least one execution per /vrs-full-testing run
- Debug record produced per invocation

### R2: Complete Agent Coverage (MANDATORY)

Every shipped agent MUST have at least one scenario that exercises it:
- Scenario ID listed in `covered_by` field
- At least one execution per /vrs-full-testing run
- Debug record produced per invocation

### R3: Debug Mode Enforcement (MANDATORY)

All skill/agent invocations MUST:
- Set DEBUG_MODE=true in environment
- Produce debug record in evidence/debug/
- Include: inputs, tool_calls, bskg_queries, outputs

### R4: Runtime Validation (MANDATORY)

Before run start, validator checks:
- All skills in this map have `covered_by` scenarios
- All agents in this map have `covered_by` scenarios
- All scenario IDs are valid (exist in SCENARIO-MATRIX.md)

**On failure:** E011 (INCOMPLETE_SKILL_COVERAGE) blocks run start.

## Debug Record Schema (Required Fields)

Every debug record MUST include:

```json
{
  "record_id": "debug-<skill|agent>-<id>-<timestamp>",
  "type": "skill" | "agent",
  "id": "<skill_id or agent_id>",
  "scenario": "<scenario_id>",
  "invocation": {
    "timestamp": "<ISO-8601>",
    "inputs": {
      "command": "<full command>",
      "arguments": {},
      "environment": {}
    }
  },
  "execution": {
    "tool_calls": [
      {"tool": "<tool_name>", "args": {}, "result_summary": "<first 500 chars>"}
    ],
    "bskg_queries": [
      {"query": "<vql or nl query>", "result_count": 0}
    ],
    "duration_ms": 0
  },
  "outputs": {
    "status": "success" | "failure" | "timeout",
    "result_summary": "<first 1000 chars>",
    "artifacts": ["<path>"]
  },
  "validation": {
    "timestamp_valid": true,
    "duration_valid": true,
    "required_fields_present": true
  }
}
```
