# Evidence Pack Schema Specification

## Overview

An **evidence pack** is the complete record of a `/vrs-full-testing` run. It contains all artifacts required to validate, reproduce, and debug the run. Evidence packs are MANDATORY inputs to gate evaluation - missing or invalid packs fail the run.

---

## Directory Structure

```
evidence-pack-<run-id>/
├── manifest.json           # Root manifest with run metadata
├── transcripts/
│   ├── suite-s1/
│   │   ├── s01.txt         # Per-scenario claude-code-agent-teams transcript
│   │   ├── s01.replay      # claude-code-agent-teams-replay format (optional)
│   │   ├── s02.txt
│   │   └── ...
│   ├── suite-s2/
│   ├── suite-s3/
│   └── suite-s4/
├── snapshots/
│   ├── s1-snapshot.tar.gz  # Worktree state after Suite 1
│   ├── s2-snapshot.tar.gz
│   ├── s3-snapshot.tar.gz
│   └── s4-snapshot.tar.gz
├── debug/
│   ├── skills/
│   │   ├── vrs-audit-<id>.json
│   │   └── ...
│   ├── agents/
│   │   ├── vrs-attacker-<id>.json
│   │   └── ...
│   └── index.json
├── metrics/
│   ├── timing.json         # Stage and scenario timing
│   ├── tokens.json         # Token usage per agent/skill
│   ├── accuracy.json       # Precision, recall, F1
│   └── agreement.json      # Multi-agent agreement rates
├── artifacts/
│   ├── graphs/             # BSKG graphs produced
│   ├── context/            # Protocol context packs
│   ├── patterns/           # Pattern match results
│   ├── reports/            # Final audit reports
│   └── debates/            # Debate transcripts
└── ground-truth/
    ├── expected.json       # Expected findings
    └── comparison.json     # Actual vs expected
```

## Transcripts Section

### Transcript File Format

Each transcript is a plain text file with claude-code-agent-teams output:

```
[ALPHASWARM-START] S01 2026-01-30T21:30:10.000Z
$ uv run alphaswarm build-kg contracts/Vault.sol

Building knowledge graph...
  Parsing: contracts/Vault.sol
  Functions found: 8
  External calls detected: 3
  State modifications: 5

Graph saved to: .vrs/graphs/vault-graph.toon
Completeness score: 98.5%

[ALPHASWARM-END] S01 0 45000
```

### Transcript Validation Rules

| Rule | Validation | Invalidates If |
|------|------------|----------------|
| MARKER_START | Contains `[ALPHASWARM-START]` | Missing |
| MARKER_END | Contains `[ALPHASWARM-END]` | Missing |
| DURATION_VALID | End marker duration > 5000 | duration <= 5000ms |
| EXIT_CODE | End marker exit code captured | Not parseable |
| NON_EMPTY | File length > 100 bytes | Too short |

### Replay Format (Optional)

For rich replay capability, `.replay` files use claude-code-agent-teams-replay format:

```
{"time": 0.0, "type": "output", "data": "[ALPHASWARM-START] S01..."}
{"time": 0.5, "type": "output", "data": "$ uv run alphaswarm..."}
{"time": 1.2, "type": "output", "data": "Building knowledge graph..."}
...
```

## Stage Artifacts

### Artifacts Manifest

Each artifact directory contains an index:

```json
{
  "directory": "artifacts/graphs",
  "artifacts": [
    {
      "filename": "vault-graph.toon",
      "scenario": "S01",
      "created_at": "2026-01-30T21:30:45.000Z",
      "size_bytes": 45000,
      "checksum": "sha256:abc123..."
    }
  ]
}
```

### Required Artifacts by Suite

| Suite | Required Artifacts |
|-------|-------------------|
| S1 (Command) | graphs/*.toon, vulndocs validation output |
| S2 (Agent) | patterns/*.json, reports/*.md, debates/*.json |
| S3 (Context) | context/*.yaml, role/oracle detection logs |
| S4 (Resume) | Snapshot restore logs, resume state diffs |

## Invalidation Rules

Evidence packs are automatically INVALID if any of the following are true:

| Rule | Check | Result |
|------|-------|--------|
| MISSING_MANIFEST | manifest.json does not exist | Pack INVALID |
| MISSING_TRANSCRIPTS | Any scenario missing transcript file | Pack INVALID |
| MISSING_MARKERS | Any transcript missing START/END markers | Pack INVALID |
| DURATION_TOO_SHORT | Any scenario duration <= 5000ms | Pack INVALID |
| EMPTY_TRANSCRIPTS | Any transcript < 100 bytes | Pack INVALID |
| MISSING_DEBUG | debug.enabled=true but index.json missing | Pack INVALID |
| INCOMPLETE_DEBUG | Debug records count != expected | Pack DEGRADED |
| MISSING_SNAPSHOTS | Any completed suite missing snapshot | Pack DEGRADED |
| METRICS_MISSING | Any metrics/*.json missing | Pack DEGRADED |

### Pack Status

| Status | Meaning |
|--------|---------|
| VALID | All checks pass |
| DEGRADED | Some non-critical artifacts missing |
| INVALID | Critical artifacts missing, cannot use for gating |

## Integration with Gate Evaluator

Evidence packs are the primary input to the GA gate evaluator (Phase 7.3.6):

```
Evidence Pack (VALID)
        │
        ▼
┌─────────────────────────────────────┐
│  Gate Evaluator                     │
│                                     │
│  Reads: manifest.json               │
│         metrics/accuracy.json       │
│         metrics/agreement.json      │
│                                     │
│  Checks:                            │
│    - precision >= 0.85              │
│    - recall >= 0.80                 │
│    - f1 >= 0.82                     │
│    - agent_agreement >= 0.90        │
│    - all scenarios passed           │
│                                     │
│  Output: GATE_PASS | GATE_FAIL      │
└─────────────────────────────────────┘
```

## Archival and Retention

Completed evidence packs are archived:

```bash
# Archive location
.vrs/evidence-archive/<run-id>.tar.gz

# Retention policy
- PASS runs: Keep 30 days, then compress to summary-only
- FAIL runs: Keep 90 days with full artifacts
- GA release runs: Keep indefinitely
```
