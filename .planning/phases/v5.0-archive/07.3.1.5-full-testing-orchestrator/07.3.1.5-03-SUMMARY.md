# Phase 07.3.1.5 Plan 03: Orchestrator Core, Schema, and CLI Runner Summary

Implemented the core /vrs-full-testing orchestrator with schema definitions and CLI runner script.

## One-liner

Full-testing orchestrator with suite execution, evidence pack generation, and dry-run CLI

## What Was Built

### 1. Full Testing Orchestrator (`src/alphaswarm_sol/testing/full_testing_orchestrator.py`)

Complete orchestrator implementation with:

- **State Machine**: INIT -> S*_RUNNING -> S*_SNAPSHOT -> COMPLETE (or FAILED_*)
- **Worktree Management**: Create isolated git worktrees for each test run
- **claude-code-agent-teams Session**: Fixed session name `alphaswarm-claude-code-agent-teams-session` per spec
- **Suite Execution**: Sequential execution of S1-S4 suites
- **Scenario Execution**: Per-scenario execution with markers and transcript capture
- **Transcript Validation**: Validates START/END markers, duration > 5000ms, non-empty
- **Snapshot Capture**: Tarball worktree state after each suite for resume
- **Evidence Pack Generation**: Full manifest, transcripts, snapshots, metrics
- **Self-Improvement Loop**: Triggered on failures with improvement suggestions

Key classes:
- `FullTestingOrchestrator`: Main orchestrator class
- `OrchestratorState`: State machine enum
- `WorktreeInfo`: Worktree metadata
- `claude-code-agent-teamsSession`: Session tracking
- `run_full_testing()`: Convenience function

### 2. Full Testing Schema (`src/alphaswarm_sol/testing/full_testing_schema.py`)

Strict schema definitions:

- **Enums (strict)**: `Suite`, `SuiteStatus`, `EvidencePackStatus`, `ResumeStage`
- **Config**: `FullTestingConfig`, `DebugConfig`, `RetryPolicy`
- **Results**: `ScenarioResult`, `SuiteResult`, `FullTestingResult`
- **Evidence**: `TranscriptValidation`, `EvidencePackManifest`
- **Snapshots**: `SnapshotManifest`
- **Run State**: `FullTestingRun` (combined config + state for serialization)

All enums have `from_string()` methods that raise `ValueError` for unknown values.

### 3. CLI Runner (`scripts/e2e/full_testing_runner.sh`)

Bash wrapper with:

- **Flags**: `--scenario`, `--suite`, `--debug`, `--resume`, `--dry-run`
- **Record/Diff**: `--record` for golden transcripts, `--diff <run-id>` for comparison
- **Dry-run Output**: Prints exact command plan with evidence pack paths
- **Prerequisite Checks**: claude-code-agent-teams, claude, uv, git
- **Environment Variables**: Configurable session, worktree base, evidence dir

## Implementation Notes

### Orchestrator Idempotency

Retries create new worktrees rather than modifying existing ones. This ensures:
- Prior evidence packs are never mutated
- Each run can be independently replayed
- Debugging failed runs remains possible

### Transcript Validation Rules

Per the orchestrator spec, transcripts are INVALID if:
- Missing `[ALPHASWARM-START]` marker (E003)
- Missing `[ALPHASWARM-END]` marker (E004)
- Duration <= 5000ms (E005)
- Content < 100 bytes (E006)

### Evidence Pack Structure

Generated packs follow the schema:
```
evidence-pack-<run-id>/
  manifest.json           # Root manifest
  transcripts/            # Per-scenario transcripts
  snapshots/              # Suite snapshots
  debug/                  # Debug records (if enabled)
  metrics/                # timing.json, accuracy.json
```
