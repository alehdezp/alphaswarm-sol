# Debug Mode Enforcement Policy

## Overview

This document specifies the **mandatory** debug mode requirements for all production skills and agents in AlphaSwarm.sol. Debug mode is NOT optional - any E2E validation run without complete debug artifacts is **INVALID** and will **FAIL**.

---

## Enforcement Principle

**Zero-tolerance for missing debug artifacts.**

If a skill or agent executes without producing a valid debug record, the entire validation run is considered INVALID. There is no "partial" debug mode - either all artifacts are captured, or the run fails.

---

## Failure Conditions (HARD FAIL)

### F1: Missing Debug Record

**Trigger:** A skill/agent executed but no corresponding debug record exists in `evidence/debug/`

**Action:** FAIL run immediately with error code E012

**Example:**
```
ERROR E012: Missing debug record for skill 'vrs-investigate'
  Expected: evidence/debug/skills/vrs-investigate-*.json
  Found: None
  Run status: INVALID
```

### F2: Incomplete Debug Record

**Trigger:** Debug record exists but missing required fields

**Required fields (ALL mandatory):**
- `record_id` - Non-empty UUID
- `skill_or_agent` - Identifier matching execution
- `timestamp_start` / `timestamp_end` - Valid ISO-8601, start < end
- `duration_ms` - Positive integer
- `inputs.prompt` - Non-empty string
- `tool_calls` - Array (may be empty)
- `bskg_queries` - Array (may be empty)
- `outputs.final_response` - Non-empty string

**Action:** FAIL run immediately with error code E013

**Example:**
```
ERROR E013: Incomplete debug record for agent 'vrs-attacker'
  Record: evidence/debug/agents/vrs-attacker-abc123.json
  Missing fields: ['inputs.prompt', 'bskg_queries']
  Run status: INVALID
```

### F3: Zero Duration

**Trigger:** `duration_ms` is 0 or negative

**Rationale:** Indicates the skill/agent did not actually execute (mock/simulation)

**Action:** FAIL run immediately with error code E014

**Example:**
```
ERROR E014: Invalid duration in debug record for skill 'vrs-verify'
  Record: evidence/debug/skills/vrs-verify-xyz789.json
  Duration: 0 ms
  Minimum required: 1 ms
  Run status: INVALID
```

### F4: Invalid Timestamps

**Trigger:** Timestamps are invalid, zero, or in the future

**Action:** FAIL run immediately with error code E015

**Example:**
```
ERROR E015: Invalid timestamps in debug record for skill 'audit'
  timestamp_start: 1970-01-01T00:00:00Z (epoch zero - not real)
  timestamp_end: 2030-01-01T00:00:00Z (future date)
  Run status: INVALID
```

### F5: Skill/Agent Not in Debug Matrix

**Trigger:** A skill/agent executed that is not listed in `debug_mode_matrix.yaml`

**Rationale:** Every shipped skill/agent must have explicit debug requirements

**Action:** FAIL run immediately with error code E016

**Example:**
```
ERROR E016: Unknown skill/agent executed without debug specification
  Executed: 'custom-skill'
  Not found in: configs/debug_mode_matrix.yaml
  Run status: INVALID
```

### F6: Empty Content in Required Field

**Trigger:** Required field is present but empty/whitespace-only

**Action:** FAIL run immediately with error code E017

**Example:**
```
ERROR E017: Empty content in required debug field
  Record: evidence/debug/skills/vrs-audit-def456.json
  Field: outputs.final_response
  Value: "" (empty string)
  Run status: INVALID
```

## Required Debug Artifacts by Type

### Skills

| Artifact | Required | Description |
|----------|----------|-------------|
| inputs.prompt | YES | Full prompt text sent to LLM |
| inputs.prompt_tokens | YES | Token count for cost tracking |
| inputs.context_files | YES | List of files provided as context |
| tool_calls | YES | All tool invocations (Read, Bash, etc.) |
| bskg_queries | YES | All BSKG/VQL queries executed |
| outputs.final_response | YES | Complete text output |
| outputs.response_tokens | YES | Token count for cost tracking |
| duration_ms | YES | Real execution time (must be >0) |

### Agents

| Artifact | Required | Description |
|----------|----------|-------------|
| inputs.prompt | YES | Full prompt with context |
| inputs.prompt_tokens | YES | Token count |
| tool_calls | YES | All tool invocations |
| bskg_queries | YES | All graph queries (MUST have at least 1 for core agents) |
| outputs.final_response | YES | Agent's complete response |
| outputs.verdict | YES | finding / no_finding / inconclusive |
| outputs.confidence | YES | 0.0-1.0 score |
| outputs.evidence_ids | YES | Graph node/edge IDs cited |
| duration_ms | YES | Real execution time |

## Validation Pipeline

### Pre-Run Validation

Before any scenario executes:

1. **Verify debug mode enabled:**
   ```bash
   [ "$ALPHASWARM_DEBUG_MODE" = "true" ] || fail E018
   ```

2. **Verify debug directory writable:**
   ```bash
   [ -w "$ALPHASWARM_DEBUG_DIR" ] || fail E019
   ```

3. **Verify debug matrix loaded:**
   ```python
   matrix = load_yaml("configs/debug_mode_matrix.yaml")
   assert len(matrix.skills) > 0, "E020: Empty debug matrix"
   ```

### Per-Execution Validation

After each skill/agent completes:

1. **Locate debug record:**
   ```python
   record_path = find_debug_record(skill_or_agent, evidence_dir)
   if not record_path:
       fail(E012, f"Missing debug record for {skill_or_agent}")
   ```

2. **Validate required fields:**
   ```python
   required = get_required_fields(skill_or_agent, matrix)
   missing = [f for f in required if not has_valid_value(record, f)]
   if missing:
       fail(E013, f"Missing fields: {missing}")
   ```

3. **Validate duration:**
   ```python
   if record.duration_ms <= 0:
       fail(E014, f"Invalid duration: {record.duration_ms}")
   ```

### Post-Run Validation

After all scenarios complete:

1. **Cross-reference with coverage map:**
   ```python
   for skill in skill_coverage.shipped_skills:
       if skill.debug_required:
           records = find_all_records(skill.id, evidence_dir)
           if len(records) == 0:
               fail(E012, f"No debug records for shipped skill: {skill.id}")
   ```

2. **Generate debug coverage report:**
   ```python
   coverage = {
       "total_skills": len(shipped_skills),
       "skills_with_records": len(skills_with_records),
       "total_agents": len(shipped_agents),
       "agents_with_records": len(agents_with_records),
       "coverage_percent": calculate_coverage()
   }
   ```

## Error Codes Summary

| Code | Name | Description | Recovery |
|------|------|-------------|----------|
| E012 | MISSING_DEBUG_RECORD | No debug record found for execution | Verify debug mode enabled, re-run |
| E013 | INCOMPLETE_DEBUG_RECORD | Debug record missing required fields | Fix debug capture code |
| E014 | INVALID_DURATION | Duration is 0 or negative | Ensure real execution, not mock |
| E015 | INVALID_TIMESTAMPS | Timestamps invalid or fake | Fix timestamp capture |
| E016 | UNKNOWN_SKILL_AGENT | Skill/agent not in debug matrix | Add to matrix or remove execution |
| E017 | EMPTY_REQUIRED_FIELD | Required field is empty | Fix output capture |
| E018 | DEBUG_MODE_DISABLED | ALPHASWARM_DEBUG_MODE not set | Set environment variable |
| E019 | DEBUG_DIR_NOT_WRITABLE | Cannot write to debug directory | Fix permissions |
| E020 | EMPTY_DEBUG_MATRIX | Debug matrix has no entries | Fix matrix configuration |

## Partial Debug Mode: NOT ALLOWED

**There is no partial debug mode.**

The following are explicitly forbidden:

1. **Selective debug capture:** All skills/agents must capture debug info
2. **Conditional debug:** Cannot disable debug for "fast" runs
3. **Post-hoc debug generation:** Cannot generate debug records after execution
4. **Synthetic debug records:** Cannot fabricate records without real execution

If debug overhead is a concern, address it via:
- Optimizing debug capture code (target <5% overhead)
- Async debug writes
- Compressed storage

But NEVER by disabling debug capture.

## Rationale

Debug mode enforcement exists because:

1. **Reproducibility:** Failed runs must be diagnosable
2. **Evidence chain:** Findings must trace to actual BSKG queries
3. **Cost tracking:** Token usage must be auditable
4. **Validation integrity:** Metrics must come from real executions
5. **GA quality:** Cannot ship without proof of actual testing

The cost of debug capture (overhead, storage) is far less than the cost of shipping bugs that cannot be reproduced or understood.
