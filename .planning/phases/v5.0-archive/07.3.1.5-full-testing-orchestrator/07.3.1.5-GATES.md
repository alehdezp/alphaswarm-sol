# /vrs-full-testing QA Gates

This document defines the quality gates that MUST pass before a `/vrs-full-testing` run is considered successful. Gates are evaluated against the evidence pack manifest and metrics files.

---

## Gate Summary

| Gate ID | Name | Metric Source | Pass Threshold | Fail Action |
|---------|------|---------------|----------------|-------------|
| G01 | Stage Completion | manifest.json | All suites passed | Block GA |
| G02 | Precision | accuracy.json | >= 0.85 | Block GA |
| G03 | Recall | accuracy.json | >= 0.80 | Block GA |
| G04 | F1 Score | accuracy.json | >= 0.82 | Block GA |
| G05 | Debug Coverage | debug/index.json | 100% agents have debug records | Block GA |
| G06 | Tier-C Detection | accuracy.json (by_category) | >= 1 Tier-C TP | Block GA |
| G07 | Economic Context | manifest.json | >= 1 oracle/pricing scenario passed | Block GA |
| G08 | Agent Agreement | agreement.json | >= 0.90 | Warn |
| G09 | Transcript Validity | validation.* | All markers valid | Block GA |
| G10 | Duration Validity | scenarios[].duration_ms | All > 5000ms | Block GA |

---

## Gate Details

### G01: Stage Completion

**Purpose:** Ensure all test suites completed successfully.

**Metric Source:**
```json
// manifest.json
{
  "suites": {
    "s1_command": { "status": "passed" },
    "s2_agent": { "status": "passed" },
    "s3_context": { "status": "passed" },
    "s4_resume": { "status": "passed" }
  }
}
```

**Pass Condition:**
```python
all(suite["status"] == "passed" for suite in manifest["suites"].values())
```

**Fail Condition:**
- Any suite has `status != "passed"`

**Fail Action:**
- BLOCK GA
- Emit: `GATE_FAIL: G01 - Suite {suite_id} status is {status}`

---

### G02: Precision

**Purpose:** Ensure detection accuracy (minimize false positives).

**Metric Source:**
```json
// metrics/accuracy.json
{
  "overall": {
    "precision": 0.92
  }
}
```

**Pass Condition:**
```python
accuracy["overall"]["precision"] >= 0.85
```

**Fail Condition:**
- Precision < 0.85

**Fail Action:**
- BLOCK GA
- Emit: `GATE_FAIL: G02 - Precision {value} < 0.85 threshold`
- Include: Top 5 false positive scenarios by confidence

---

### G03: Recall

**Purpose:** Ensure detection coverage (minimize false negatives).

**Metric Source:**
```json
// metrics/accuracy.json
{
  "overall": {
    "recall": 0.88
  }
}
```

**Pass Condition:**
```python
accuracy["overall"]["recall"] >= 0.80
```

**Fail Condition:**
- Recall < 0.80

**Fail Action:**
- BLOCK GA
- Emit: `GATE_FAIL: G03 - Recall {value} < 0.80 threshold`
- Include: List of false negative scenarios

---

### G04: F1 Score

**Purpose:** Balanced measure of precision and recall.

**Metric Source:**
```json
// metrics/accuracy.json
{
  "overall": {
    "f1_score": 0.90
  }
}
```

**Pass Condition:**
```python
accuracy["overall"]["f1_score"] >= 0.82
```

**Fail Condition:**
- F1 < 0.82

**Fail Action:**
- BLOCK GA
- Emit: `GATE_FAIL: G04 - F1 Score {value} < 0.82 threshold`

---

### G05: Debug Coverage

**Purpose:** Ensure all agents/skills have debug records for full observability.

**Metric Source:**
```json
// manifest.json
{
  "debug": {
    "enabled": true,
    "records_count": 45,
    "validation": {
      "all_records_valid": true,
      "validation_errors": []
    }
  }
}

// debug/index.json
{
  "records": [
    { "skill_or_agent": "/vrs-audit", "status": "success" },
    { "skill_or_agent": "vrs-attacker", "status": "success" }
  ]
}
```

**Pass Condition:**
```python
manifest["debug"]["enabled"] == True and
manifest["debug"]["validation"]["all_records_valid"] == True and
len(manifest["debug"]["validation"]["validation_errors"]) == 0
```

**Required Debug Records:**
- `/vrs-audit` - For each audit invocation
- `/vrs-investigate` - For each investigation
- `/vrs-debate` - For each debate
- `vrs-attacker` - For each attacker agent run
- `vrs-defender` - For each defender agent run
- `vrs-verifier` - For each verifier agent run

**Fail Condition:**
- Debug mode not enabled
- Missing debug records for any agent invoked
- Validation errors in debug records

**Fail Action:**
- BLOCK GA
- Emit: `GATE_FAIL: G05 - Debug coverage incomplete`
- Include: List of missing/invalid records

---

### G06: Tier-C Detection

**Purpose:** Verify label-dependent (Tier-C) patterns are functional.

**Metric Source:**
```json
// metrics/accuracy.json
{
  "by_category": {
    "tier_c_labels": {
      "precision": 0.85,
      "recall": 0.80,
      "true_positives": 3,
      "false_positives": 0
    }
  }
}
```

**Pass Condition:**
```python
accuracy["by_category"].get("tier_c_labels", {}).get("true_positives", 0) >= 1
```

**Tier-C Scenarios (S85-S89):**
- S85: Missing label prevents detection
- S86: Label-dependent access control pattern
- S87: Label-dependent oracle pattern
- S88: Label-dependent invariant break
- S89: Labels present but no match (negative control)

**Fail Condition:**
- No Tier-C true positives detected
- Tier-C scenarios not included in run

**Fail Action:**
- BLOCK GA
- Emit: `GATE_FAIL: G06 - No Tier-C label-dependent detections`
- Include: Tier-C scenario results

---

### G07: Economic Context

**Purpose:** Verify oracle/pricing/economic scenarios pass.

**Metric Source:**
```json
// manifest.json
{
  "scenarios": [
    { "id": "S21", "category": "oracle", "status": "passed" },
    { "id": "S22", "category": "oracle", "status": "passed" }
  ]
}
```

**Pass Condition:**
```python
oracle_scenarios = [s for s in manifest["scenarios"] if s.get("category") in ["oracle", "pricing", "economic"]]
passed_oracle = [s for s in oracle_scenarios if s["status"] == "passed"]
len(passed_oracle) >= 1
```

**Economic Context Scenarios (S21-S30):**
- S21: Unchecked oracle staleness
- S22: Single-source oracle (manipulable)
- S23: TWAP window too small
- S24: Oracle decimals mismatch
- S25: Inverted price feed misuse
- S27: Multi-oracle median (negative control)
- S29: Chainlink feed not validated

**Fail Condition:**
- No oracle/pricing scenarios passed
- Context pack not generated for economic scenarios

**Fail Action:**
- BLOCK GA
- Emit: `GATE_FAIL: G07 - Economic context validation failed`
- Include: List of failed oracle scenarios

---

### G08: Agent Agreement

**Purpose:** Measure multi-agent debate consistency.

**Metric Source:**
```json
// metrics/agreement.json
{
  "overall_agreement_rate": 0.94,
  "by_debate": [
    { "scenario": "S01", "agreement": true },
    { "scenario": "S12", "agreement": false }
  ]
}
```

**Pass Condition:**
```python
agreement["overall_agreement_rate"] >= 0.90
```

**Fail Condition:**
- Agreement rate < 0.90

**Fail Action:**
- WARN (does not block GA)
- Emit: `GATE_WARN: G08 - Agent agreement rate {value} < 0.90`
- Include: List of disagreement scenarios for review

---

### G09: Transcript Validity

**Purpose:** Ensure all claude-code-agent-teams transcripts have valid markers.

**Metric Source:**
```json
// manifest.json
{
  "validation": {
    "all_transcripts_present": true,
    "all_markers_valid": true,
    "all_durations_valid": true,
    "pack_valid": true,
    "errors": []
  }
}
```

**Pass Condition:**
```python
manifest["validation"]["all_transcripts_present"] == True and
manifest["validation"]["all_markers_valid"] == True and
manifest["validation"]["pack_valid"] == True
```

**Required Markers:**
- `[ALPHASWARM-START] <scenario-id> <timestamp>`
- `[ALPHASWARM-END] <scenario-id> <exit-code> <duration-ms>`

**Fail Condition:**
- Missing transcripts
- Missing START or END markers
- Invalid marker format

**Fail Action:**
- BLOCK GA
- Emit: `GATE_FAIL: G09 - Transcript validation failed`
- Include: List of invalid transcripts

---

### G10: Duration Validity

**Purpose:** Ensure no mocked/simulated runs (minimum 5s execution).

**Metric Source:**
```json
// manifest.json
{
  "scenarios": [
    { "id": "S01", "duration_ms": 45000 },
    { "id": "S02", "duration_ms": 52000 }
  ]
}
```

**Pass Condition:**
```python
all(s["duration_ms"] > 5000 for s in manifest["scenarios"])
```

**Fail Condition:**
- Any scenario with `duration_ms <= 5000`

**Fail Action:**
- BLOCK GA
- Emit: `GATE_FAIL: G10 - Scenario {id} duration {duration}ms < 5000ms minimum`
- Include: All scenarios with invalid duration

---

## Gate Evaluation Order

Gates are evaluated in this order, with early exit on blocking failures:

```
1. G09: Transcript Validity (pack must be valid to evaluate)
2. G10: Duration Validity (no mocked runs)
3. G05: Debug Coverage (observability required)
4. G01: Stage Completion (all suites passed)
5. G02: Precision (accuracy metric)
6. G03: Recall (coverage metric)
7. G04: F1 Score (balanced metric)
8. G06: Tier-C Detection (label-dependent patterns)
9. G07: Economic Context (oracle/pricing)
10. G08: Agent Agreement (warning only)
```

## Gate Evaluation Script

```python
# Pseudocode for gate evaluation
def evaluate_gates(evidence_pack_path: str) -> GateResult:
    manifest = load_json(f"{evidence_pack_path}/manifest.json")
    accuracy = load_json(f"{evidence_pack_path}/metrics/accuracy.json")
    agreement = load_json(f"{evidence_pack_path}/metrics/agreement.json")
    debug_index = load_json(f"{evidence_pack_path}/debug/index.json")

    results = GateResult()

    # G09: Transcript Validity (must pass first)
    if not manifest["validation"]["pack_valid"]:
        results.fail("G09", "Evidence pack invalid")
        return results  # Early exit

    # G10: Duration Validity
    for scenario in manifest["scenarios"]:
        if scenario["duration_ms"] <= 5000:
            results.fail("G10", f"Scenario {scenario['id']} duration too short")

    # G05: Debug Coverage
    if not manifest["debug"]["validation"]["all_records_valid"]:
        results.fail("G05", "Debug records incomplete")

    # G01: Stage Completion
    for suite_id, suite in manifest["suites"].items():
        if suite["status"] != "passed":
            results.fail("G01", f"Suite {suite_id} failed")

    # G02: Precision
    if accuracy["overall"]["precision"] < 0.85:
        results.fail("G02", f"Precision {accuracy['overall']['precision']} < 0.85")

    # G03: Recall
    if accuracy["overall"]["recall"] < 0.80:
        results.fail("G03", f"Recall {accuracy['overall']['recall']} < 0.80")

    # G04: F1 Score
    if accuracy["overall"]["f1_score"] < 0.82:
        results.fail("G04", f"F1 {accuracy['overall']['f1_score']} < 0.82")

    # G06: Tier-C Detection
    tier_c = accuracy.get("by_category", {}).get("tier_c_labels", {})
    if tier_c.get("true_positives", 0) < 1:
        results.fail("G06", "No Tier-C true positives")

    # G07: Economic Context
    oracle_passed = sum(1 for s in manifest["scenarios"]
                        if s.get("category") in ["oracle", "pricing"]
                        and s["status"] == "passed")
    if oracle_passed < 1:
        results.fail("G07", "No oracle scenarios passed")

    # G08: Agent Agreement (warning only)
    if agreement["overall_agreement_rate"] < 0.90:
        results.warn("G08", f"Agreement {agreement['overall_agreement_rate']} < 0.90")

    return results
```
