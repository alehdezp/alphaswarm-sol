# Phase 07.3.1.5 Plan 12: Execution & Postmortem Docs Summary

Operator playbook, postmortem template, and automated evidence pack collection for /vrs-full-testing workflow.

---

## What Was Built

### 1. Execution Playbook (490 LOC)

Comprehensive 7-phase operator guide for running /vrs-full-testing:

| Phase | Purpose |
|-------|---------|
| 1. Environment Setup | claude-code-agent-teams session, worktree creation, evidence pack init |
| 2. Run Testing | Record mode, diff mode, recovery paths |
| 3. Snapshot Management | Suite snapshots with checksum verification |
| 4. Transcript Collection | Marker validation, claude-code-agent-teams capture |
| 5. Evidence Pack Collection | Automated artifact gathering |
| 6. Gate Evaluation | Precision/recall/F1 gate checks |
| 7. Cleanup | Archive, worktree removal, session cleanup |

**Key Features:**
- Exact claude-code-agent-teams commands for session setup and capture
- Recovery procedures for claude-code-agent-teams crash, worktree corruption, write failures
- Operator checklist for before/during/after run
- Error code reference (E003-E015)

### 2. Postmortem Template (353 LOC)

Structured failure analysis template with 9 sections:

| Section | Purpose |
|---------|---------|
| Summary | One-liner + timeline + impact |
| Failure Category | Taxonomy checkbox (E003-E015, EUNK) |
| Evidence Pack Links | Pack location + artifact integrity |
| Reproduction Steps | Minimal repro with expected vs actual |
| Root Cause Analysis | Investigation log + contributing factors |
| Fix Proposal | Code change + verification + rollback |
| Prevention | Short-term/long-term actions |
| Lessons Learned | What went well/poorly + takeaways |
| Sign-Off | Fix verification + postmortem review |

**Time Budget:** 6 hours max (30min triage, 2h investigation, 2h fix, 1h verify, 30min docs)

### 3. Evidence Pack Collector (434 LOC)

`scripts/e2e/collect_evidence_pack.sh` with:

| Feature | Description |
|---------|-------------|
| Dry-run mode | `--dry-run` lists what would be collected |
| Critical vs optional | Fails on missing critical artifacts (exit 4) |
| Transcript validation | Checks ALPHASWARM-START/END markers |
| Manifest update | Adds collection metadata |
| Colorized output | INFO/OK/WARN/ERROR with summary |

**Exit Codes:**
- 0: Success
- 1: Missing arguments
- 2: Evidence directory missing
- 3: Worktree directory missing
- 4: Missing critical artifacts
- 5: Validation failed
