# /vrs-full-testing Orchestrator Specification

## Overview

The `/vrs-full-testing` skill is the master orchestrator for AlphaSwarm.sol's E2E validation pipeline. It schedules, monitors, and coordinates all subworkflows through Claude Code sessions operating via `claude-code-controller` in the fixed session `alphaswarm-claude-code-agent-teams-session`.

**Full claude-code-controller documentation:** See `.planning/testing/rules/claude-code-controller-REFERENCE.md` and `.planning/testing/rules/canonical/claude-code-controller-instructions.md`

## Core Responsibilities

1. **Subworkflow Orchestration** - Schedule and sequence all test suites
2. **Worktree Lifecycle** - Manage isolated git worktrees for test runs
3. **Snapshot Management** - Capture stage snapshots for resume capability
4. **Evidence Collection** - Generate evidence packs for each run
5. **Failure Handling** - Classify failures and trigger targeted re-runs

## Subworkflow Graph (State Machine)

### Suites (Ordered Execution)

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                         /vrs-full-testing                                   │
│                                                                             │
│  ┌──────────────┐     ┌──────────────┐     ┌──────────────┐     ┌────────┐ │
│  │ COMMAND_SUITE│────▶│ AGENT_SUITE  │────▶│ CONTEXT_SUITE│────▶│ RESUME │ │
│  │   (Suite 1)  │     │   (Suite 2)  │     │   (Suite 3)  │     │ MATRIX │ │
│  └──────────────┘     └──────────────┘     └──────────────┘     └────────┘ │
│         │                    │                    │                  │      │
│         ▼                    ▼                    ▼                  ▼      │
│    [snapshot]           [snapshot]           [snapshot]        [snapshot]   │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### Suite Definitions

| Suite ID | Name | Focus | Entry Condition |
|----------|------|-------|-----------------|
| S1 | COMMAND_SUITE | CLI command coverage (`build-kg`, `query`, `vulndocs`, etc.) | Worktree ready |
| S2 | AGENT_SUITE | Subagent workflows (`/vrs-audit`, `/vrs-investigate`, `/vrs-debate`) | S1 passed |
| S3 | CONTEXT_SUITE | Protocol context + oracle + role detection | S2 passed |
| S4 | RESUME_MATRIX | Stop/resume, partial runs, stage skip | S3 passed |

### State Machine

```
States:
  INIT          - Orchestrator startup, worktree creation
  S1_RUNNING    - Command suite executing
  S1_SNAPSHOT   - S1 complete, snapshot captured
  S2_RUNNING    - Agent suite executing
  S2_SNAPSHOT   - S2 complete, snapshot captured
  S3_RUNNING    - Context suite executing
  S3_SNAPSHOT   - S3 complete, snapshot captured
  S4_RUNNING    - Resume matrix executing
  S4_SNAPSHOT   - S4 complete, snapshot captured
  COMPLETE      - All suites passed
  FAILED_S<N>   - Suite N failed
  RETRY_S<N>    - Retrying suite N

Transitions:
  INIT          → S1_RUNNING     : worktree created
  S1_RUNNING    → S1_SNAPSHOT    : all S1 scenarios pass
  S1_RUNNING    → FAILED_S1      : any S1 scenario fails (fail-fast mode)
  S1_SNAPSHOT   → S2_RUNNING     : snapshot confirmed
  S2_RUNNING    → S2_SNAPSHOT    : all S2 scenarios pass
  S2_RUNNING    → FAILED_S2      : any S2 scenario fails
  S2_SNAPSHOT   → S3_RUNNING     : snapshot confirmed
  S3_RUNNING    → S3_SNAPSHOT    : all S3 scenarios pass
  S3_RUNNING    → FAILED_S3      : any S3 scenario fails
  S3_SNAPSHOT   → S4_RUNNING     : snapshot confirmed
  S4_RUNNING    → S4_SNAPSHOT    : all S4 scenarios pass
  S4_RUNNING    → FAILED_S4      : any S4 scenario fails
  S4_SNAPSHOT   → COMPLETE       : evidence pack finalized
  FAILED_S<N>   → RETRY_S<N>     : retry policy allows (< max_retries)
  FAILED_S<N>   → ABORT          : retry limit exceeded
  RETRY_S<N>    → S<N>_RUNNING   : retry initiated
```

### Failure States (Explicit)

| State | Meaning | Required Artifacts |
|-------|---------|-------------------|
| FAILED_S1 | Command suite failed | Failure log, last transcript, debug record |
| FAILED_S2 | Agent suite failed | Failed scenario ID, agent debug output |
| FAILED_S3 | Context suite failed | Context pack diff, missing dependencies |
| FAILED_S4 | Resume matrix failed | Snapshot corruption indicator, stage mismatch |
| ABORT | Max retries exceeded | Postmortem with all evidence pack links |

## claude-code-agent-teams Session Orchestration

### Fixed Session Configuration

```yaml
session_name: alphaswarm-claude-code-agent-teams-session
window_layout: single-pane
shell: /bin/zsh
working_directory: <worktree-path>
```

### claude-code-controller Integration

All test execution MUST use `claude-code-controller` commands. Direct subprocess calls are PROHIBITED.

```bash
# Session lifecycle
PANE_ID=$(claude-code-controller launch "zsh")
claude-code-controller send "cd /path/to/worktree && uv run alphaswarm build-kg contracts/" --pane=$PANE_ID
claude-code-controller capture --pane=$PANE_ID  # Capture transcript after command completes
claude-code-controller send "/exit" --pane=$PANE_ID
claude-code-controller kill --pane=$PANE_ID  # Only after evidence pack is saved

# Markers (MANDATORY)
# Each command must produce visible markers:
#   [ALPHASWARM-START] <command-id> <timestamp>
#   [ALPHASWARM-END] <command-id> <exit-code> <duration-ms>
```

### Transcript Requirements

Every claude-code-agent-teams transcript MUST contain:
1. `[ALPHASWARM-START]` marker with command ID
2. `[ALPHASWARM-END]` marker with exit code and duration
3. Duration > 5000ms (runs under 5s are automatically invalid)

### Validation Rules

| Rule | Check | Action on Fail |
|------|-------|----------------|
| MARKER_PRESENT | Both START and END markers exist | Invalidate run |
| DURATION_VALID | duration_ms > 5000 | Invalidate run |
| TRANSCRIPT_NON_EMPTY | transcript.length > 100 | Invalidate run |
| EXIT_CODE_ZERO | exit_code == 0 (for pass scenarios) | Mark as failure |

## Worktree Lifecycle

### Creation

```bash
# Create isolated worktree for test run
git worktree add /tmp/alphaswarm-test-<run-id> HEAD

# Required structure after creation:
# /tmp/alphaswarm-test-<run-id>/
#   ├── contracts/           # Test contracts (symlinked or copied)
#   ├── .vrs/                # VRS artifacts directory
#   │   ├── graphs/          # Built BSKG graphs
#   │   ├── context/         # Protocol context packs
#   │   ├── patterns/        # Pattern match results
#   │   ├── agents/          # Agent outputs
#   │   ├── debate/          # Debate transcripts
#   │   └── reports/         # Final reports
#   └── evidence/            # Evidence pack directory
#       ├── transcripts/     # claude-code-agent-teams transcripts
#       ├── snapshots/       # Stage snapshots
#       └── debug/           # Debug mode artifacts
```

### Snapshot Points

Snapshots capture worktree state at suite completion for resume capability.

```yaml
snapshot_manifest:
  run_id: <uuid>
  suite_completed: S1 | S2 | S3 | S4
  timestamp: <iso-8601>
  worktree_path: /tmp/alphaswarm-test-<run-id>
  files_checksum:
    .vrs/graphs: <sha256>
    .vrs/context: <sha256>
    .vrs/patterns: <sha256>
    .vrs/agents: <sha256>
    .vrs/debate: <sha256>
  resume_command: "/vrs-full-testing --resume <run-id> --from S<N+1>"
```

### Cleanup

Worktrees are preserved on failure for debugging. Cleanup is explicit:

```bash
# Manual cleanup after investigation
git worktree remove /tmp/alphaswarm-test-<run-id> --force

# Auto-cleanup only after COMPLETE state AND evidence pack archived
```

## Fail-Fast and Retry Policies

### Fail-Fast Mode (Default)

```yaml
fail_fast:
  enabled: true
  scope: suite  # Fail the entire suite on first scenario failure
  capture_on_fail:
    - transcript: true
    - debug_record: true
    - stack_trace: true
```

### Retry Policy

```yaml
retry_policy:
  max_retries_per_suite: 3
  max_total_retries: 10
  backoff:
    initial_delay_ms: 1000
    max_delay_ms: 30000
    multiplier: 2.0

  # Retry boundaries (when to stop vs. when to auto-rerun)
  auto_retry_conditions:
    - transient_network_error
    - claude-code-agent-teams_session_crash
    - timeout_exceeded

  no_retry_conditions:
    - assertion_failure        # Logic error, needs investigation
    - missing_dependency       # Environment issue
    - invalid_ground_truth     # Test data issue
    - detection_mismatch       # Actual finding vs. expected
```

### Retry State Tracking

```yaml
retry_state:
  run_id: <uuid>
  suite: S2
  attempt: 2
  previous_failures:
    - attempt: 1
      error: timeout_exceeded
      transcript_link: evidence/transcripts/s2-attempt1.txt
      timestamp: <iso-8601>
  max_attempts_remaining: 1
```

## Scenario Execution

### Per-Scenario Flow

```
1. Load scenario from SCENARIO-MATRIX.md (e.g., S01)
2. Prepare test contracts in worktree
3. Execute via claude-code-controller:
   a. claude-code-controller send "[ALPHASWARM-START] S01 $(date -u +%Y-%m-%dT%H:%M:%SZ)"
   b. claude-code-controller send "uv run alphaswarm <command>"
   c. Wait for completion
   d. claude-code-controller send "[ALPHASWARM-END] S01 $? $(duration)"
   e. claude-code-controller capture > evidence/transcripts/s01.txt
4. Validate transcript markers
5. Compare output to expected (ground truth)
6. Record result in evidence pack
7. If fail + fail-fast: STOP and transition to FAILED_S<N>
8. If pass: continue to next scenario
```

### Scenario Categories (from SCENARIO-MATRIX.md)

| Category | Scenario Range | Primary Suites |
|----------|----------------|----------------|
| Reentrancy | S01-S08 | S1, S2 |
| Access Control | S09-S16 | S1, S2 |
| Oracle/Pricing | S17-S24 | S2, S3 |
| Upgradeability | S25-S30 | S1, S2 |
| Flash Loans | S31-S36 | S2, S3 |
| Governance | S37-S42 | S2 |
| ERC Standards | S43-S48 | S1, S2 |
| Math/Precision | S49-S54 | S1, S2 |
| Cross-Contract | S55-S58 | S2, S3 |
| MEV | S59-S61 | S3 |
| Parser Edge | S62-S65 | S1 |
| Tier-C Labels | S66-S70 | S2 |
| Workflow Resume | S71-S72 | S4 |

```
    └── accuracy.json      # Precision/recall metrics
```

## Invocation

### Basic Usage

```bash
/vrs-full-testing                        # Run all suites, fresh worktree
/vrs-full-testing --only-suite S1        # Run only command suite
/vrs-full-testing --scenarios S01,S05,S17 # Run specific scenarios
/vrs-full-testing --resume <run-id>      # Resume from last snapshot
/vrs-full-testing --resume <run-id> --from S3  # Resume from specific suite
```

## Validation Checklist

Before a run is considered VALID:

- [ ] Worktree created and initialized
- [ ] All scenarios executed via claude-code-controller (not subprocess)
- [ ] All transcripts contain ALPHASWARM-START and ALPHASWARM-END markers
- [ ] All scenario durations > 5000ms
- [ ] Evidence pack manifest.json present and complete
- [ ] No missing transcript files
- [ ] Debug mode artifacts present for all agents invoked
- [ ] Snapshot checksums match expected state

## Error Codes

| Code | Meaning |
|------|---------|
| E001 | Worktree creation failed |
| E002 | claude-code-agent-teams session could not be created |
| E003 | Missing ALPHASWARM-START marker |
| E004 | Missing ALPHASWARM-END marker |
| E005 | Duration below minimum (< 5000ms) |
| E006 | Transcript empty or missing |
| E007 | Ground truth file missing |
| E008 | Detection mismatch |
| E009 | Retry limit exceeded |
