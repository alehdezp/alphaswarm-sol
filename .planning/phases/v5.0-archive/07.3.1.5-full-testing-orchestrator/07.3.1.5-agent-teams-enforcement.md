# claude-code-controller Enforcement Policy

## Purpose

This policy declares that all /vrs-full-testing runs are **INVALID** if they do not use claude-code-controller for command execution. This is a hard gate that cannot be bypassed.

**Full claude-code-controller documentation:** See `.planning/testing/rules/claude-code-controller-REFERENCE.md` and `.planning/testing/rules/canonical/claude-code-controller-instructions.md`

---

## Policy Statement

**ALL test execution in /vrs-full-testing MUST use claude-code-controller commands.**

Direct subprocess calls, manual shell invocations, or any execution path that bypasses claude-code-agent-teams session isolation will result in immediate run invalidation.

---

## Rationale

1. **Isolation** - claude-code-agent-teams sessions provide process isolation and prevent test interference
2. **Capture** - Transcript capture requires claude-code-agent-teams pane access
3. **Markers** - ALPHASWARM-START/END markers require claude-code-agent-teams session context
4. **Reproducibility** - Consistent execution environment across all runs
5. **Debug** - Pane capture enables post-mortem debugging

---

## Acceptable Evidence Signals

A valid /vrs-full-testing run MUST produce the following claude-code-controller evidence:

### Required Markers (All MUST Be Present)

| Marker | Pattern | Purpose |
|--------|---------|---------|
| SESSION_CREATE | `claude-code-agent-teams new-session` or `claude-code-controller launch` | Proves session was created |
| ALPHASWARM_START | `[ALPHASWARM-START] <id> <timestamp>` | Command execution began |
| ALPHASWARM_END | `[ALPHASWARM-END] <id> <exit_code> <duration_ms>` | Command execution completed |
| PANE_CAPTURE | `claude-code-agent-teams capture-pane` or `claude-code-controller capture` | Transcript captured |
| SESSION_DESTROY | `claude-code-agent-teams kill-session` or `claude-code-controller kill` | Session cleaned up |

### Socket Isolation (Required for Parallel Runs)

Each run MUST create a unique socket file:

```
Socket pattern: /tmp/vrs-{run_id}.sock
Socket creation: claude-code-agent-teams -S /tmp/vrs-{run_id}.sock new-session -d -s alphaswarm-claude-code-agent-teams-session
```

Evidence of socket ID MUST appear in:
1. Session creation command
2. All subsequent claude-code-agent-teams commands
3. Session destruction command

## Fail Conditions

The following conditions MUST cause the run to fail immediately:

### E003: Missing ALPHASWARM-START Marker

**Condition:** Any transcript file lacks `[ALPHASWARM-START]` pattern.

**Action:** Invalidate run, report error code E003.

```yaml
error:
  code: E003
  message: "Missing ALPHASWARM-START marker"
  transcript_file: "<path>"
  resolution: "Ensure all commands are wrapped with ALPHASWARM markers via claude-code-controller"
```

### E004: Missing ALPHASWARM-END Marker

**Condition:** Any transcript file lacks `[ALPHASWARM-END]` pattern.

**Action:** Invalidate run, report error code E004.

```yaml
error:
  code: E004
  message: "Missing ALPHASWARM-END marker"
  transcript_file: "<path>"
  resolution: "Command may have crashed or timed out without proper termination"
```

### E005: Duration Below Minimum

**Condition:** Any command duration < 5000ms.

**Action:** Invalidate run, report error code E005.

```yaml
error:
  code: E005
  message: "Duration below minimum threshold"
  scenario_id: "<id>"
  actual_duration_ms: <N>
  minimum_required_ms: 5000
  resolution: "Runs under 5s indicate mock or simulated execution"
```

### E016: Missing claude-code-agent-teams Session Evidence

**Condition:** No evidence of claude-code-agent-teams session creation in run artifacts.

**Action:** Block run from starting, report error code E016.

```yaml
error:
  code: E016
  message: "No claude-code-agent-teams session creation evidence"
  resolution: "Run must use claude-code-controller launch to establish session"
```

### E017: Socket ID Missing or Invalid

**Condition:** Socket file pattern not found or socket ID inconsistent across commands.

**Action:** Invalidate run, report error code E017.

```yaml
error:
  code: E017
  message: "Socket isolation not enforced"
  expected_pattern: "/tmp/vrs-{run_id}.sock"
  resolution: "All claude-code-agent-teams commands must use consistent socket via -S flag"
```

### E018: Direct Subprocess Detected

**Condition:** Evidence of subprocess.run, os.system, or similar direct execution without claude-code-agent-teams context.

**Action:** Invalidate run, report error code E018.

```yaml
error:
  code: E018
  message: "Direct subprocess execution detected"
  evidence: "<pattern found>"
  resolution: "All commands must be executed via claude-code-controller, not direct subprocess"
```

## Validation Sequence

The validator runs this sequence for every evidence pack:

```
1. CHECK: Session creation evidence exists
   FAIL: E016 if missing

2. CHECK: Socket ID present and consistent
   FAIL: E017 if missing or inconsistent

3. FOR EACH transcript file:
   a. CHECK: [ALPHASWARM-START] marker present
      FAIL: E003 if missing

   b. CHECK: [ALPHASWARM-END] marker present
      FAIL: E004 if missing

   c. CHECK: Duration from END marker >= 5000ms
      FAIL: E005 if below threshold

4. CHECK: Session destruction evidence exists
   WARN: W001 if missing (not a hard fail, but flagged)

5. CHECK: No subprocess patterns detected
   FAIL: E018 if detected

RESULT: PASS only if all checks pass
```

---

## Bypass Prevention

### Prohibited Patterns

The validator MUST scan for and reject these patterns:

```python
# Direct execution patterns that bypass claude-code-agent-teams
PROHIBITED_PATTERNS = [
    r"subprocess\.run\([^)]*alphaswarm",
    r"subprocess\.Popen\([^)]*alphaswarm",
    r"os\.system\([^)]*alphaswarm",
    r"os\.popen\([^)]*alphaswarm",
    r"asyncio\.subprocess",
    r"shell=True.*alphaswarm",
]
```

### Evidence Integrity

All transcripts MUST:
1. Originate from `claude-code-agent-teams capture-pane` output
2. Contain continuous output (no gaps or splices)
3. Have timestamps that increase monotonically
4. Match the run_id in their content

## Policy Enforcement Points

| Phase | Enforcement |
|-------|-------------|
| Run Start | E016 check (session creation) |
| Each Scenario | E003, E004, E005 checks |
| Run Complete | E017 check (socket consistency) |
| Evidence Validation | E018 check (subprocess detection) |

---

## Exception Policy

**There are NO exceptions.** All runs MUST pass all claude-code-controller enforcement checks.

If a scenario cannot be tested via claude-code-controller due to technical limitations, that scenario MUST be:
1. Documented as untestable
2. Excluded from coverage requirements
3. Reviewed and approved by project owner
