# Phase 07.3.1.5 Plan 05: Golden Transcript Capture and Diffing Summary

**One-liner:** Golden transcript infrastructure for regression detection with record/diff modes and marker-based validation for all subworkflows.

## What Was Built

### 1. Golden Transcript Scenarios (full_testing_golden.yaml)

Defines 6 golden scenarios for baseline capture and regression testing:

| Scenario ID | Description | Suites |
|-------------|-------------|--------|
| golden-s1-command-suite | Command suite full baseline | s1_command |
| golden-s2-agent-suite | Agent suite full baseline | s2_agent |
| golden-s3-context-suite | Context suite full baseline | s3_context |
| golden-s4-resume-matrix | Resume matrix full baseline | s4_resume |
| golden-quick-s1 | Quick command suite (CI) | s1_command (S01 only) |
| golden-quick-s2 | Quick agent suite (CI) | s2_agent (S09 only) |

Each scenario includes:
- **Record config**: Baseline directory, artifact types (transcript, markers, hashes)
- **Diff config**: Mode (structural/exact/semantic), ignore patterns, tolerance rules
- **Normalization patterns**: Timestamp, run ID, duration replacement

### 2. Expected Markers (full_testing_markers.yaml)

Defines 21+ markers across 4 subworkflows plus 13 error markers:

| Suite | Markers | Key Checks |
|-------|---------|------------|
| command_suite | 6 | build-kg, query (NL/VQL2), vulndocs, context, label |
| agent_suite | 5 | audit, investigate, debate, patterns, evidence |
| context_suite | 5 | context gen, oracle, roles, doc-conflict, invariants |
| resume_matrix | 5 | snapshot, resume, partial, skip, verify |
| error_markers | 13 | E001-E013 infrastructure, marker, test, resource errors |

Artifact validation checks for each suite ensure required outputs exist.

### 3. Golden Runner Script (run_full_testing_golden.sh)

CLI wrapper supporting:
- `--record <scenario>`: Capture new golden baseline
- `--diff <scenario>`: Compare against recorded baseline
- `--list`: Show available scenarios
- `--validate`: Validate config files
- `--dry-run`: Preview without execution
- `--force`: Overwrite existing baseline
- `--verbose`: Detailed output

Features:
- Transcript normalization for non-deterministic outputs
- Fail-fast on first diff mismatch
- Evidence pack path emission on failure
- Worktree isolation for each run
