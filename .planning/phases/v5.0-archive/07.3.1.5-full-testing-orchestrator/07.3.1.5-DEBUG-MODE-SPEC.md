# Debug Mode Contract Specification

## Overview

Every skill and subagent shipped to production MUST implement **debug mode** - a structured capture mechanism that records all inputs, intermediate steps, and outputs for post-hoc analysis and failure triage.

Debug mode is MANDATORY for all E2E validation. Runs without debug artifacts are invalid.

---

## Required Fields

### Core Debug Record Schema

```yaml
debug_record:
  # Identification
  record_id: string           # UUID, unique per invocation
  skill_or_agent: string      # e.g., "/vrs-audit", "vrs-attacker"
  version: string             # Skill/agent version
  timestamp_start: string     # ISO-8601 start time
  timestamp_end: string       # ISO-8601 end time
  duration_ms: integer        # Total execution time

  # Inputs
  inputs:
    prompt: string            # Full prompt text sent to LLM
    prompt_tokens: integer    # Token count of prompt
    context_files: list       # Files provided as context
    parameters: object        # Any parameters/flags passed

  # Tool Calls
  tool_calls:
    - tool_id: string         # Unique ID for this call
      tool_name: string       # e.g., "Read", "Bash", "Grep"
      timestamp: string       # When called
      parameters: object      # Parameters passed to tool
      result_summary: string  # Truncated result (first 500 chars)
      result_tokens: integer  # Token count of result
      duration_ms: integer    # Tool execution time
      error: string | null    # Error message if failed

  # BSKG/VQL Queries
  bskg_queries:
    - query_id: string        # Unique ID for this query
      query_type: string      # "vql" | "natural_language" | "pattern"
      query_text: string      # The query string
      timestamp: string       # When executed
      graph_file: string      # Which graph was queried
      result_count: integer   # Number of matches
      result_ids: list        # List of node/edge IDs matched
      duration_ms: integer    # Query execution time

  # Intermediate Artifacts
  intermediate_artifacts:
    - artifact_id: string     # Unique ID
      artifact_type: string   # "graph", "context_pack", "pattern_match", etc.
      path: string            # Path to artifact file
      checksum: string        # SHA-256 of artifact
      created_at: string      # Timestamp

  # Outputs
  outputs:
    final_response: string    # Full text output
    response_tokens: integer  # Token count of response
    structured_output: object # Parsed structured output (if any)
    verdict: string | null    # For agents: "finding" | "no_finding" | "inconclusive"
    confidence: float | null  # Confidence score 0.0-1.0
    evidence_ids: list        # IDs of evidence cited

  # Metadata
  metadata:
    model: string             # LLM model used
    temperature: float        # Temperature setting
    max_tokens: integer       # Max tokens setting
    cost_usd: float           # Estimated cost
    parent_record_id: string | null  # If spawned by another agent
    run_id: string            # Parent run ID from orchestrator
```

## Activation

### Environment Variable

```bash
export ALPHASWARM_DEBUG_MODE=true
export ALPHASWARM_DEBUG_DIR=/path/to/evidence/debug
```

### CLI Flag

```bash
# Note: --debug flag is aspirational - not yet implemented in CLI
uv run alphaswarm build-kg contracts/ --verbose  # Current: use verbose for diagnostics
/vrs-audit contracts/  # Skill invocation (debug mode via ALPHASWARM_DEBUG_MODE env var)
```

### Programmatic

```python
from alphaswarm_sol.debug import enable_debug_mode, get_debug_recorder

enable_debug_mode(output_dir="evidence/debug")

recorder = get_debug_recorder()
with recorder.record_invocation("/vrs-investigate", inputs={"prompt": "..."}) as record:
    # ... skill execution ...
    record.add_tool_call(...)
    record.add_bskg_query(...)
    record.set_output(...)
```

## Validation Rules

Debug records are validated before inclusion in evidence pack:

| Rule | Check | Action on Fail |
|------|-------|----------------|
| RECORD_ID_PRESENT | record_id is non-empty UUID | Reject record |
| TIMESTAMPS_VALID | start < end, both valid ISO-8601 | Reject record |
| DURATION_POSITIVE | duration_ms > 0 | Reject record |
| PROMPT_PRESENT | inputs.prompt is non-empty | Reject record |
| TOOL_CALLS_LOGGED | tool_calls array exists (may be empty) | Reject record |
| BSKG_QUERIES_LOGGED | bskg_queries array exists (may be empty) | Reject record |
| OUTPUT_PRESENT | outputs.final_response is non-empty | Reject record |

## Error Handling

### Graceful Degradation

If debug recording fails, it MUST NOT block the main execution:

```python
try:
    recorder.add_tool_call(...)
except DebugRecordingError as e:
    logger.warning(f"Debug recording failed: {e}")
    # Continue execution, mark run as "debug_incomplete"
```

### Incomplete Debug Records

Runs with incomplete debug records are flagged:

```json
{
  "debug": {
    "status": "incomplete",
    "missing_records": ["vrs-defender-rec-003"],
    "reason": "Agent terminated unexpectedly"
  }
}
```

## Performance Considerations

| Metric | Target | Measured By |
|--------|--------|-------------|
| Overhead per tool call | < 5ms | timestamp delta |
| Overhead per query | < 10ms | timestamp delta |
| Record serialization | < 50ms | end-of-invocation |
| Total debug overhead | < 5% of run time | aggregated |

Debug mode MUST NOT significantly impact execution time. If overhead exceeds 5%, the implementation requires optimization.
