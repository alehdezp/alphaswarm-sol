# /vrs-full-testing Operational Runbook

This runbook documents the complete workflow for running the `/vrs-full-testing` orchestrator, including prerequisites, claude-code-agent-teams setup, worktree lifecycle, execution modes, and debugging.

## Prerequisites

### Required Tools

| Tool | Version | Check Command | Install |

| claude-code-agent-teams | >= 3.0 | `claude-code-agent-teams -V` | `brew install claude-code-agent-teams` |

## claude-code-agent-teams Session Setup

### Session Name

The orchestrator uses a **fixed session name**: `alphaswarm-claude-code-agent-teams-session`

This ensures consistency across runs and enables resume capability.

### Creating the Session

```bash
# Kill existing session (if any)
claude-code-agent-teams has-session -t alphaswarm-claude-code-agent-teams-session 2>/dev/null && \
    claude-code-agent-teams kill-session -t alphaswarm-claude-code-agent-teams-session

# Create new session
claude-code-agent-teams new-session -d -s alphaswarm-claude-code-agent-teams-session -c /path/to/project

# Attach to monitor
claude-code-agent-teams attach-session -t alphaswarm-claude-code-agent-teams-session
```

### Session Commands

| Action | Command |
|--------|---------|
| Create session | `claude-code-agent-teams new-session -d -s alphaswarm-claude-code-agent-teams-session` |
| Attach | `claude-code-agent-teams attach-session -t alphaswarm-claude-code-agent-teams-session` |
| Detach | `Ctrl+b d` (from inside session) |

## Worktree Creation and Snapshotting

### Worktree Lifecycle

```
1. CREATE: git worktree add /tmp/alphaswarm-test-<run-id> HEAD
2. INIT: Copy/symlink test contracts, create .vrs directory
3. EXECUTE: Run scenarios in worktree
4. SNAPSHOT: Capture state after each suite
5. CLEANUP: git worktree remove (only after evidence pack saved)
```

### Creating a Worktree

```bash
# Generate run ID
RUN_ID="run-$(date +%Y%m%d-%H%M%S)-$(head -c 3 /dev/urandom | xxd -p)"

# Create worktree
git worktree add /tmp/alphaswarm-test-$RUN_ID HEAD

# Initialize structure
cd /tmp/alphaswarm-test-$RUN_ID
mkdir -p .vrs/{graphs,context,patterns,agents,debate,reports}
mkdir -p evidence/{transcripts,snapshots,debug,metrics}
```

### Snapshot Points

Snapshots are captured after each suite completion:

```bash
# After Suite 1 (Command)
tar -czf evidence/snapshots/s1-snapshot.tar.gz .vrs/

# After Suite 2 (Agent)
tar -czf evidence/snapshots/s2-snapshot.tar.gz .vrs/

# After Suite 3 (Context)
tar -czf evidence/snapshots/s3-snapshot.tar.gz .vrs/

# After Suite 4 (Resume)
tar -czf evidence/snapshots/s4-snapshot.tar.gz .vrs/
```

### Cleanup

```bash
# Only after evidence pack is complete and archived
git worktree remove /tmp/alphaswarm-test-$RUN_ID --force
```

## Execution Modes

### Record Mode (Capture Golden Baselines)

Use when establishing baseline transcripts for regression detection:

```bash
# Via CLI runner
./scripts/e2e/full_testing_runner.sh --record --scenario S01

# Via Claude Code directly
/vrs-full-testing --scenarios S01 --debug --record
```

Output:
- Transcript saved to `.vrs/evidence/.golden-ref`
- Debug artifacts captured
- Baseline for future diff comparison

### Diff Mode (Compare Against Baseline)

Use for regression detection against recorded baselines:

```bash
# Via CLI runner
./scripts/e2e/full_testing_runner.sh --diff run-20260130-abc123 --scenario S01

# Compare two evidence packs
diff -r evidence/evidence-pack-baseline evidence/evidence-pack-current
```

Output:
- Diff report showing changes
- Evidence pack paths for investigation

### Standard Execution

```bash
# Run all suites
./scripts/e2e/full_testing_runner.sh

# Run specific suite
./scripts/e2e/full_testing_runner.sh --suite s1_command

# Run specific scenarios
./scripts/e2e/full_testing_runner.sh --scenario S01,S05,S17

# With debug mode
./scripts/e2e/full_testing_runner.sh --debug

# Resume from prior run
./scripts/e2e/full_testing_runner.sh --resume run-20260130-abc123
```

## Bootstrap Detection Checklist

**CRITICAL:** Before running full E2E suites, verify basic detection works with a single known-vulnerable contract.

### Step 1: Choose Bootstrap Scenario

Use S01 (DVD Naive Receiver) - classic reentrancy, well-documented ground truth.

```bash
# Dry-run first
./scripts/e2e/full_testing_runner.sh --dry-run --scenario S01
```

### Step 2: Execute Bootstrap

```bash
./scripts/e2e/full_testing_runner.sh --scenario S01 --debug
```

### Step 3: Verify Detection

Check the evidence pack for:

| Check | Expected | Failure Action |
|-------|----------|----------------|
| Graph built | `.vrs/graphs/naive-receiver.toon` exists | Check slither, solc version |
| Pattern matched | `reentrancy-classic` in patterns/ | Check VQL query, pattern YAML |
| Finding generated | Critical reentrancy in report | Check agent prompts, debate |
| Debug records | `evidence/debug/index.json` present | Check debug mode enabled |
| Markers valid | `[ALPHASWARM-START]` and `[ALPHASWARM-END]` | Check claude-code-agent-teams capture |
| Duration > 5s | `duration_ms > 5000` | Check for mocked execution |

### Step 4: Proceed or Diagnose

- **If bootstrap passes:** Proceed to full suite execution
- **If bootstrap fails:** Do NOT run full suites. Debug the specific failure first.

## E2E Suite Order

Suites MUST execute in order due to dependencies:

### Suite 1: Command Suite (S01-S25)

Focus: CLI command coverage (`build-kg`, `query`, `vulndocs`, etc.)

```bash
./scripts/e2e/full_testing_runner.sh --suite s1_command
```

Baseline Scenarios (MUST pass):
- S01: DVD Naive Receiver (reentrancy)
- S02: DVD Side Entrance (cross-function reentrancy)
- S06: Safe Reentrancy Guard (negative control)

### Suite 2: Agent Suite (S26-S55)

Focus: Subagent workflows (`/vrs-audit`, `/vrs-investigate`, `/vrs-debate`)

```bash
./scripts/e2e/full_testing_runner.sh --suite s2_agent
```

Requires: S1 passed (graphs and patterns exist)

Baseline Scenarios (MUST pass):
- S31: DVD Backdoor (uninitialized proxy)
- S39: DVD Unstoppable (flash-loan price manipulation)

### Suite 3: Context Suite (S56-S75)

Focus: Protocol context + oracle + role detection

```bash
./scripts/e2e/full_testing_runner.sh --suite s3_context
```

Requires: S1 and S2 passed (agents can be invoked)

Key Scenarios:
- S61: Rebasing token integration (Lido-style)

### Suite 4: Resume Matrix (S76-S95)

Focus: Stop/resume, partial runs, failure injection

```bash
./scripts/e2e/full_testing_runner.sh --suite s4_resume
```

Requires: S1, S2, S3 passed (full pipeline available)

Key Scenarios:
- S90: Partial run (graph only) then resume
- S93: Timeout injection recovery
- S95: Missing file error handling

## Debugging via Evidence Pack

### Locating Evidence

```bash
# Default location
EVIDENCE_DIR=".vrs/evidence/evidence-pack-<run-id>"

# Contents
ls -la $EVIDENCE_DIR
# manifest.json       - Run metadata and results
# transcripts/        - claude-code-agent-teams transcripts per scenario
# snapshots/          - Worktree state after each suite
# debug/              - Skill and agent debug records
# metrics/            - Timing, tokens, accuracy
# artifacts/          - Graphs, context packs, reports
# ground-truth/       - Expected vs actual comparison
```

### Debug Mode Investigation

When debug mode is enabled (`--debug`), each skill/agent produces detailed records:

```bash
# View debug index
cat $EVIDENCE_DIR/debug/index.json | jq '.records'

# View specific skill debug
cat $EVIDENCE_DIR/debug/skills/vrs-audit-<record-id>.json | jq '.tool_calls'

# View agent BSKG queries
cat $EVIDENCE_DIR/debug/agents/vrs-attacker-<record-id>.json | jq '.bskg_queries'
```

### Debug Record Fields

| Field | Use For |
|-------|---------|
| `inputs.prompt` | Verify correct prompt sent |
| `tool_calls` | Check tool usage sequence |
| `bskg_queries` | Verify graph-first reasoning |
| `outputs.verdict` | Check agent conclusion |
| `outputs.evidence_ids` | Verify evidence linkage |
| `metadata.cost_usd` | Track token spend |

### Common Debug Scenarios

**1. Detection missed (false negative)**

```bash
# Check if graph was built correctly
cat $EVIDENCE_DIR/artifacts/graphs/<contract>-graph.toon

# Check if pattern matched
grep -r "reentrancy" $EVIDENCE_DIR/artifacts/patterns/

# Check agent debate
cat $EVIDENCE_DIR/artifacts/debates/<scenario>.json
```

**2. False positive (detection on safe contract)**

```bash
# Check negative control in ground truth
cat $EVIDENCE_DIR/ground-truth/expected.json | jq '.[] | select(.expected_finding == false)'

# Check defender response
cat $EVIDENCE_DIR/debug/agents/vrs-defender-*.json | jq '.outputs'
```

**3. Execution timeout**

```bash
# Check timing metrics
cat $EVIDENCE_DIR/metrics/timing.json | jq '.scenarios | to_entries | .[] | select(.value > 120000)'

# Check for slow tools
cat $EVIDENCE_DIR/debug/*/vrs-run-slither-*.json | jq '.tool_calls[].duration_ms'
```

**4. Transcript marker issues**

```bash
# Search for markers
grep -l "ALPHASWARM-START" $EVIDENCE_DIR/transcripts/*/*.txt
grep -l "ALPHASWARM-END" $EVIDENCE_DIR/transcripts/*/*.txt

# Check for missing markers
for f in $EVIDENCE_DIR/transcripts/*/*.txt; do
    if ! grep -q "ALPHASWARM-START" "$f"; then
        echo "Missing START marker: $f"
    fi
    if ! grep -q "ALPHASWARM-END" "$f"; then
        echo "Missing END marker: $f"
    fi
done
```

## Recovery Procedures

### Crashed Session Recovery

```bash
# Check if session exists
claude-code-agent-teams has-session -t alphaswarm-claude-code-agent-teams-session && echo "Session exists"

# If crashed, check for worktree
ls /tmp/alphaswarm-test-*/

# Resume from last snapshot
./scripts/e2e/full_testing_runner.sh --resume <run-id> --from S2
```

### Corrupt Evidence Pack

```bash
# Validate manifest
cat $EVIDENCE_DIR/manifest.json | jq '.' || echo "Invalid JSON"

# Check for missing files
jq -r '.scenarios[].transcript_path' $EVIDENCE_DIR/manifest.json | while read path; do
    [[ -f "$EVIDENCE_DIR/$path" ]] || echo "Missing: $path"
done
```

### Worktree Stuck

```bash
# Force remove stuck worktree
git worktree remove /tmp/alphaswarm-test-<run-id> --force

# Prune worktree refs
git worktree prune
```
