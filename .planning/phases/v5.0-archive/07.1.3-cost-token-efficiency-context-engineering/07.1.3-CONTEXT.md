# Phase 7.1.3: Cost/Token Efficiency & Context Engineering - Context

## Purpose
Reduce run cost without sacrificing detection precision by applying modern context engineering, model routing, and evidence compression.

## Scope
- Context budgets per bead/pool
- Progressive disclosure and retrieval-first context loading
- Multi-tier model routing (validator -> specialist)
- Evidence packing (TOON + minimal JSON envelopes)
- Query/memo caching for graph and tool results
- Prompt/tool description compression
- Cost dashboards and enforcement policies

## Out of Scope
- New detection patterns (handled elsewhere)
- Framework interop (handled in 7.1.4)

## Key Constraints
- Maintain GA target precision/recall
- Keep evidence-first requirements intact
- Preserve BSKG query usage

## Desired Outcomes
- 35-50% token reduction on standard runs
- Stable precision/recall with lower cost
- Predictable budget ceilings per run

## Validation
- Cost deltas measured across standard and adversarial corpora
- No regression in Tier A/B precision gates
