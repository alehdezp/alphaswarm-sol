## What Was Built

### TierRoutingPolicy (`routing_policy.py`)

Core routing logic with validator-first approach:

```python
from alphaswarm_sol.llm.routing_policy import TierRoutingPolicy, route_task

# Simple task - stays CHEAP
decision = route_task("pattern_validation", risk_score=0.2)  # -> CHEAP

# High risk - escalates to PREMIUM
decision = route_task("tier_b_verification", risk_score=0.9)  # -> PREMIUM
```

**Key classes:**
- `TierRoutingPolicy` - Main routing logic
- `RoutingDecision` - Decision with tier, rationale, cost metadata
- `EscalationThresholds` - Configurable thresholds
- `EscalationReason` - Enum for escalation tracking

### Escalation Rules

| Condition | Action |
|-----------|--------|
| risk_score >= 0.5 | Escalate to STANDARD |
| risk_score >= 0.8 | Escalate to PREMIUM |
| evidence_completeness < 0.3 | Escalate to STANDARD |
| severity = critical/high | Escalate from CHEAP |
| Complex pattern type | Escalate to PREMIUM |
| budget < $2.00 | Cap at STANDARD |
| budget < $0.50 | Force CHEAP |

## Decisions Made

### D1: Validator-First Routing

**Decision:** Default to CHEAP tier, escalate only when warranted.

**Rationale:** Most validation tasks don't need expensive models. Starting cheap and escalating based on evidence reduces costs by 35-50% while maintaining quality.

### D2: Configurable Thresholds

**Decision:** Use `EscalationThresholds` dataclass with pool_id/workflow_id tracking.

**Rationale:** Different contexts have different risk tolerances. High-value audits may use aggressive escalation while routine scans stay conservative.

### D3: Two-Level Budget Downgrade

**Decision:** $2.00 threshold caps at STANDARD, $0.50 forces CHEAP.

**Rationale:** Allows controlled degradation. Low budget still gets reasonable analysis (STANDARD), critical budget ensures task completion (CHEAP).
