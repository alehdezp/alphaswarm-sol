# Phase 7.1.3 Plan 05: Prompt Linting & Tool Compression Summary

**One-liner:** Prompt linting with 5 rules (oversized, duplicate, missing-constraint, unused-tool, size) + tool description compression with phrase abbreviations for 10-20% token savings.

Add prompt linting and tool description compression to reduce prompt size and detect wasteful context.

## What Was Built

### Prompt Linting Module

**File:** `src/alphaswarm_sol/llm/prompt_lint.py` (~500 LOC)

- `PromptLintReport` - Structured report with violations, token impact, summary
- `LintViolation` - Individual violation with severity, location, suggestion
- `LintRule` - Base class for extensible rules
- 5 default rules:
  - `OversizedSectionRule` - Flags code blocks >2000 chars, metadata >1000 chars
  - `DuplicateContextRule` - Detects repeated evidence IDs, file paths
  - `MissingConstraintRule` - Warns on missing output schema or evidence requirement
  - `UnusedToolRule` - Flags tool references not in allowed list
  - `PromptSizeRule` - Error if exceeds budget, warn if >80%
- `lint_prompt()` - Main entry point for prompt linting
- `create_linter()` - Factory for custom rule selection

### Tool Description Compression

**File:** `src/alphaswarm_sol/tools/description_compress.py` (~350 LOC)

- `ToolDescriptionCompressor` - Configurable compression with stats tracking
- Phrase abbreviations for common verbose descriptions
- Aggressive mode for tight token budgets
- `compress_tool_description()` - Single tool compression
- `compress_tool_descriptions()` - Batch compression with budget
- `compress_for_context()` - Minimal context string generation

### Integration

**Modified:** `src/alphaswarm_sol/llm/subagents.py`
- `_build_prompt()` now returns lint report
- `SubagentResult.prompt_lint_report` field added
- Lint warnings logged at WARNING level

**Modified:** `src/alphaswarm_sol/tools/registry.py`
- `get_tools_for_context(max_tokens)` - Get compressed tool list
- `get_tools_context_string(max_chars)` - Get minimal context string

## Decisions Made

1. **Non-blocking lint by default** - Lint reports available but don't block execution
2. **Five default rules** - Cover common patterns without over-engineering
3. **Phrase abbreviation approach** - Preserve meaning while reducing tokens
