## What Was Built

### ClaudeCodeRunner (runner.py, 450 LOC)

Wraps the Claude Code CLI (`claude -p`) for programmatic invocation:

```python
runner = ClaudeCodeRunner(project_root=Path("."))
result = runner.run_analysis(
    prompt="Analyze contracts/ for reentrancy",
    allowed_tools=["Bash(uv run alphaswarm*)", "Read", "Glob"],
    json_schema=FINDINGS_SCHEMA,
    model="claude-sonnet-4"
)
```

**Features:**
- `--allowedTools` for pre-approved tool execution (no human intervention)
- `--output-format json` for structured parsing
- `--json-schema` for output validation
- `--resume` for session continuation
- Cost tracking from output metadata
- Duration measurement

### OutputParser (output_parser.py, 350 LOC)

Extracts structured findings and calculates accuracy:

```python
findings = OutputParser.parse_findings(result.structured_output)
accuracy = OutputParser.calculate_accuracy(findings, ground_truth)
# Returns: precision, recall, f1_score, severity-weighted recall
```

**Features:**
- `ParsedFinding` dataclass with pattern/severity/location/confidence
- Pattern and location fuzzy matching
- Severity-weighted recall calculation
- Findings grouping and filtering

### AgentSpawner (agent_spawner.py, 400 LOC)

Parallel agent orchestration using ThreadPoolExecutor:

```python
spawner = AgentSpawner(runner, max_workers=4)
results = spawner.run_model_comparison(
    prompt="Analyze for vulnerabilities",
    allowed_tools=["Read", "Glob"],
    models=["claude-opus-4", "claude-sonnet-4", "claude-haiku-4"]
)
```

**Features:**
- `AgentTask` dataclass with metadata support
- `AgentResult` with success/error tracking
- Progress callback support
- `run_model_comparison()` for model benchmarking
- `run_contract_batch()` for parallel contract analysis
- `run_vuln_class_sweep()` for focused vulnerability checking
- Aggregate statistics calculation

### ScenarioLoader (scenario_loader.py, 350 LOC)

YAML-based test scenario configuration:

```yaml
# scenarios/reentrancy/config.yaml
name: "Classic Reentrancy"
category: reentrancy
contracts:
  - path: corpus/reentrancy/vuln-001.sol
    has_vulnerability: true
    ground_truth:
      - pattern: reentrancy-classic
        severity: critical
        location: withdraw
prompt_template: "Analyze {contract_path} for reentrancy"
allowed_tools: ["Bash(uv run alphaswarm*)", "Read"]
```

**Features:**
- `ContractCase` dataclass with ground truth
- `TestScenario` dataclass with isolation settings
- Recursive scenario discovery
- Category-based filtering
- Path validation
- Scenario creation helper

## Key Insight

Skills (`/vrs-test-full`) don't work in headless mode. The autonomous testing harness bypasses skills and invokes Claude Code directly via subprocess with:

1. **Prompt descriptions** instead of skill invocations
2. **--allowedTools** for pre-approved tool execution
3. **--output-format json** for structured parsing
4. **ThreadPoolExecutor** for parallel agent spawning
