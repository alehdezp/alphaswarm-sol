**vrs-test-conductor.md:**
- name: VRS Test Conductor
- role: test_conductor
- model: claude-opus-4 (complex coordination)
- Purpose: Orchestrates complete test runs including health checks, component tests, integration tests, E2E tests, model comparison, mutation tests, adversarial tests, gap analysis, and report generation
- Must NOT execute tests directly - delegates to other agents
- Monitors test progress across all testing agents
- Enforces quality gates (precision >=85%, recall_critical >=95%, recall_high >=85%)
- Generates TestMetrics summary at completion
- Output format: JSON with test_orchestration_result schema

**vrs-corpus-curator.md:**
- name: VRS Corpus Curator
- role: corpus_curator
- model: claude-sonnet-4 (balance reasoning/cost)
- Purpose: Maintains corpus integrity, validates ground truth, categorizes new test cases
- Validates corpus.db schema compliance
- Ensures ground truth labels are from verified sources
- Categorizes contracts (recent-audits, mutations, adversarial, safe)
- Tracks corpus versions
- Output format: JSON with corpus_validation_result schema

**vrs-mutation-tester.md:**
- name: VRS Mutation Tester
- role: mutation_tester
- model: claude-haiku-4 (mechanical, low reasoning)
- Purpose: Generates contract mutations that preserve vulnerability semantics
- Mutation types: identifier renaming, code reordering, pattern variation, structural changes
- Validates each mutation preserves expected vulnerability (or marks as safe variant)
- Generates 10x variants per known pattern
- Output format: JSON with mutation_result schema containing variants list

Each agent file must include:
- YAML frontmatter (name, role, model, description)
- Role explanation section
- Core principles section
- Input context section (what the agent receives)
- Output format section with JSON schema
- Step-by-step execution framework
- Notes on responsibilities and constraints

**vrs-regression-hunter.md:**
- name: VRS Regression Hunter
- role: regression_hunter
- model: claude-sonnet-4 (pattern matching)
- Purpose: Detects accuracy degradation between test runs
- Compares current TestMetrics against baseline
- Identifies which patterns/categories regressed
- Bisects changes to find root cause
- Generates regression report with affected tests
- Output format: JSON with regression_analysis_result schema containing regressions list, affected_patterns, suspected_cause

**vrs-benchmark-runner.md:**
- name: VRS Benchmark Runner
- role: benchmark_runner
- model: claude-haiku-4 (mechanical execution)
- Purpose: Executes benchmark configurations against corpus
- Runs quick/standard/thorough benchmark modes
- Collects precision, recall, F1 metrics per model and task type
- Records execution time, tokens used, cost
- Updates rankings.yaml with EMA scores
- Output format: JSON with benchmark_result schema containing metrics dict, model_scores, cost_breakdown

**vrs-gap-finder.md:**
- name: VRS Gap Finder
- role: gap_finder
- model: claude-opus-4 (complex reasoning)
- Purpose: Identifies coverage holes and improvement priorities
- Analyzes which vulnerability classes are under-tested
- Finds false positive patterns
- Identifies agent reasoning failures
- Categorizes gaps: detection, false_positive, technical, agent, scalability, ux
- Outputs prioritized gap inventory
- Output format: JSON with gap_analysis_result schema containing gaps list with category, severity, root_cause, roadmap_item

Each agent file must include:
