## Key Components

### AuditTester
- Executes 5 audit phases in order: VKG_BUILD, PATTERN_DETECTION, BEAD_GENERATION, VERIFICATION, REPORT_GENERATION
- Supports CLI, direct, and mock invocation modes
- Checks expectations: finding count, patterns, precision, recall
- Integrates with OrchestratorTester for verification phase

### BenchmarkRunner
- Runs benchmark suites (quick, standard, thorough)
- Calculates precision, recall, F1 by severity class
- Supports parallel scenario execution via ThreadPoolExecutor
- Model comparison with F1-based ranking

### BlindTester
- 5 isolation levels: NONE, BASIC, STANDARD, STRICT, MAXIMUM
- Contract anonymization (name replacement, comment stripping)
- Corpus segment filtering (post-cutoff contracts)
- Generic blind prompts without vulnerability hints

### RealWorldSimulator
- Time budget enforcement (strict_timeout)
- Attempt and tool call limits
- Accuracy requirement validation
- Network isolation support

## Decisions Made

- **5-phase workflow:** Granular phases allow testing specific pipeline stages in isolation
- **Mock mode default:** E2E tests run in mock mode for speed (4.59s for 60 tests)
- **Isolation level presets:** BlindConfig.from_level() provides consistent presets per isolation level
- **F1-based ranking:** Model comparison ranks by F1 score for balanced precision/recall
