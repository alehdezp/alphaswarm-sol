# Phase 7.1: Autonomous Testing Research

**Researched:** 2026-01-22
**Question:** Can Claude Code test itself autonomously, or is human-in-the-loop required?

## Executive Summary

**YES, fully autonomous testing IS possible** using Claude Code's programmatic features:

1. **`claude -p` (print/headless mode)** - Non-interactive execution
2. **`--output-format json`** - Structured, parseable results
3. **`--allowedTools`** - Pre-approve tools without human intervention
4. **Python subprocess** - Invoke Claude Code from test harness
5. **Agent SDK** - Native Python/TypeScript integration for complex orchestration

**However**, there are important constraints that change our testing strategy significantly.

## Key Findings

### 1. Headless Mode Capabilities

```bash
# Basic headless invocation
claude -p "Analyze this contract for reentrancy" --output-format json

# With pre-approved tools (no human approval needed)
claude -p "Run the test suite" --allowedTools "Bash(pytest*),Bash(uv run*),Read,Glob"

# With structured output schema
claude -p "Extract vulnerabilities from analysis" \
  --output-format json \
  --json-schema '{"type":"object","properties":{"findings":[...]}}'
```

### 2. What WORKS Autonomously

| Capability | How | Notes |
|------------|-----|-------|
| Run agents via subprocess | `subprocess.run(["claude", "-p", ...])` | Full JSON output parsing |
| Pre-approve tools | `--allowedTools "Bash,Read,Edit"` | Wildcards: `Bash(git*)` |
| Structured output | `--json-schema '{...}'` | Schema validation built-in |
| Continue conversations | `--resume $session_id` | Cross-run context |
| Custom system prompts | `--system-prompt` or `--append-system-prompt` | Agent personas |
| Spawn subagents | Task tool from within Claude Code | Parallel work |

### 3. What DOES NOT WORK in Headless Mode

| Limitation | Workaround |
|------------|------------|
| **Skills** (`/vrs-audit`, `/commit`) | Describe the task in natural language instead |
| **Built-in commands** (`/help`, `/clear`) | Not needed for testing |
| **Interactive prompts** | Pre-approve with `--allowedTools` |
| **Human approval gates** | Pre-define allowed actions |

**Critical Insight:** Skills are user-invoked features, NOT programmatic. For autonomous testing, we describe what we want done rather than invoking skills.

### 4. Python Integration Pattern

```python
import subprocess
import json
from pathlib import Path

class ClaudeCodeRunner:
    """Programmatic Claude Code invocation for testing."""

    def run_analysis(
        self,
        prompt: str,
        allowed_tools: list[str] | None = None,
        system_prompt: str | None = None,
        json_schema: dict | None = None,
        timeout_seconds: int = 300
    ) -> dict:
        """Run Claude Code in headless mode."""
        cmd = ["claude", "-p", prompt, "--output-format", "json"]

        if allowed_tools:
            cmd.extend(["--allowedTools", ",".join(allowed_tools)])

        if system_prompt:
            cmd.extend(["--system-prompt", system_prompt])

        if json_schema:
            cmd.extend(["--json-schema", json.dumps(json_schema)])

        result = subprocess.run(
            cmd,
            capture_output=True,
            text=True,
            timeout=timeout_seconds,
            cwd=self.project_root
        )

        if result.returncode != 0:
            raise RuntimeError(f"Claude Code failed: {result.stderr}")

        return json.loads(result.stdout)

    def spawn_subagent(
        self,
        agent_type: str,
        task: str,
        model: str = "sonnet"
    ) -> dict:
        """Spawn a subagent for parallel work."""
        prompt = f"""
        Use the Task tool to spawn a {agent_type} agent:

        Task: {task}
        Model: {model}

        Return the agent result in JSON format.
        """
        return self.run_analysis(
            prompt,
            allowed_tools=["Task"],
            json_schema={
                "type": "object",
                "properties": {
                    "agent_id": {"type": "string"},
                    "result": {"type": "object"},
                    "status": {"type": "string"}
                }
            }
        )
```

### 5. Agent SDK Alternative

The official **Agent SDK** provides native Python integration:

```python
from claude_agent_sdk import Agent, Task

# Create agent programmatically
agent = Agent(
    model="claude-sonnet-4",
    system_prompt="You are a security auditor...",
    tools=["Read", "Glob", "Grep", "Bash(uv run*)"]
)

# Execute task
result = await agent.run("Analyze contracts/ for reentrancy vulnerabilities")

# Access structured output
findings = result.structured_output
```

**Advantages over subprocess:**
- Native async support
- Built-in tool approval callbacks
- Streaming responses
- Better error handling
- Session management

## Revised Testing Strategy

### Original Approach (07.1 Plans): Skill-Based
```
/vrs-test-full  →  spawns vrs-test-conductor  →  runs tests
```
**Problem:** Skills don't work in headless mode.

### New Approach: Prompt-Based Autonomous Testing

```
Python test harness
    │
    ├── subprocess.run(["claude", "-p", "..."])
    │   └── Pre-approved tools, JSON output
    │
    ├── Parse structured results
    │   └── TestMetrics extraction
    │
    └── Evaluate against ground truth
        └── Update rankings, track gaps
```

### Test Execution Flow

```python
# tests/test_autonomous_pipeline.py
import pytest
from true_vkg.testing.harness import ClaudeCodeRunner, ScenarioLoader

@pytest.fixture
def runner():
    return ClaudeCodeRunner(project_root=Path("."))

@pytest.fixture
def scenarios():
    return ScenarioLoader.load_all("src/true_vkg/testing/scenarios/")

class TestAutonomousPipeline:
    """Fully autonomous Claude Code testing - no human intervention."""

    def test_reentrancy_detection(self, runner, scenarios):
        """Test reentrancy detection accuracy."""
        scenario = scenarios["reentrancy"]

        for contract in scenario.contracts:
            prompt = scenario.prompt_template.format(
                contract_path=contract.path
            )

            result = runner.run_analysis(
                prompt=prompt,
                allowed_tools=scenario.allowed_tools,
                json_schema=scenario.json_schema,
                timeout_seconds=scenario.timeout_seconds
            )

            # Check detection accuracy
            expected = contract.has_vulnerability
            actual = result["has_vulnerability"]

            assert actual == expected, (
                f"Detection mismatch for {contract.path}: "
                f"expected={expected}, got={actual}"
            )

    def test_model_comparison(self, runner, scenarios):
        """Compare model accuracy across scenarios."""
        models = ["claude-opus-4", "claude-sonnet-4", "claude-haiku-4"]
        results = {}

        for model in models:
            metrics = self._run_benchmark(runner, scenarios, model)
            results[model] = metrics

        # All models should meet minimum precision
        for model, metrics in results.items():
            assert metrics.precision >= 0.70, f"{model} precision too low"

    def test_agent_spawning(self, runner):
        """Test that subagents can be spawned programmatically."""
        result = runner.run_analysis(
            prompt="""
            Use the Task tool to spawn an Explore agent to find all
            files containing 'withdraw' function definitions.
            Return the file list.
            """,
            allowed_tools=["Task", "Read", "Glob"],
            json_schema={
                "type": "object",
                "properties": {
                    "files": {"type": "array", "items": {"type": "string"}}
                }
            }
        )

        assert "files" in result
        assert isinstance(result["files"], list)
```

## Real-World Solidity Assessment Simulation

### Isolation Requirements

To test like "real-world" without solution leakage:

```yaml
# scenarios/blind_audit/config.yaml
name: "Blind Audit Simulation"
description: |
  Test agents on contracts they haven't seen in training.
  No hints about vulnerability type or location.

isolation:
  # Don't include vulndocs in context
  exclude_knowledge: true
  # Don't provide expected vulnerability class
  blind_prompt: true
  # Use contracts from post-training-cutoff audits
  corpus_segment: "recent-audits"  # Post-Jan 2025

prompt_template: |
  You are auditing {contract_path} for a security firm.

  Analyze this contract for ALL potential vulnerabilities.
  Do not make assumptions - use the VKG to identify issues.

  For each finding:
  1. Describe the vulnerability
  2. Provide exact code location
  3. Assess severity and exploitability
  4. Suggest remediation

# Agent doesn't know what it should find
expected_findings_hidden: true  # Only test harness knows ground truth
```

### Adversarial Testing

```python
class TestAdversarialScenarios:
    """Test agent robustness against deceptive contracts."""

    def test_false_positive_traps(self, runner):
        """Contracts that LOOK vulnerable but are safe."""
        traps = [
            "corpus/adversarial/fake-reentrancy.sol",  # Has guard in modifier
            "corpus/adversarial/fake-overflow.sol",    # SafeMath used
            "corpus/adversarial/deceptive-names.sol",  # withdraw() is actually safe
        ]

        for contract in traps:
            result = runner.run_analysis(
                prompt=f"Find vulnerabilities in {contract}",
                allowed_tools=["Bash(uv run true-vkg*)", "Read"],
                json_schema={"type": "object", "properties": {
                    "findings": {"type": "array"}
                }}
            )

            # Should NOT flag false positives
            critical_findings = [
                f for f in result["findings"]
                if f.get("severity") in ["critical", "high"]
            ]
            assert len(critical_findings) == 0, (
                f"False positive in {contract}: {critical_findings}"
            )

    def test_obfuscated_vulnerabilities(self, runner):
        """Vulnerabilities hidden by unusual coding patterns."""
        obfuscated = [
            "corpus/adversarial/split-reentrancy.sol",  # Split across functions
            "corpus/adversarial/proxy-reentrancy.sol",  # Via delegatecall
            "corpus/adversarial/callback-reentrancy.sol", # Via callback
        ]

        for contract in obfuscated:
            result = runner.run_analysis(...)

            # Should STILL find the vulnerability
            assert result["has_vulnerability"], (
                f"Missed obfuscated vuln in {contract}"
            )
```

## Conclusion

**Autonomous testing IS possible** with Claude Code, but requires:

1. **Prompt-based invocation** instead of skills
2. **Pre-approved tools** via `--allowedTools`
3. **Structured JSON output** via `--output-format json`
4. **Python harness** to orchestrate and evaluate
5. **Scenario definitions** with ground truth for comparison

The agents can absolutely test themselves without human intervention. The key insight is that we bypass the skill system and invoke Claude Code directly with carefully crafted prompts that describe what we want done.
