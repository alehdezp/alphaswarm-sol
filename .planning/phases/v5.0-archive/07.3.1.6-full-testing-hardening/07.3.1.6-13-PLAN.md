## Evidence Requirements
- Any live run must use demo claude-code-agent-teams session label `vrs-demo-{workflow}-{timestamp}`.
- Evidence packs must include proof tokens per `docs/PHILOSOPHY.md` (graph hash, node/edge counts, agent spawn proof, debate proof, detection proof).
- Graph-first rule enforced: BSKG/VQL queries must run before conclusions for any reasoning step.
- claude-code-agent-teams runs must use Claude Code as the subject inside claude-code-agent-teams (launch shell, then send `claude`); no direct shell runs or simulated outputs.
- Anti-fabrication checks required for claude-code-agent-teams transcripts:
  - Minimum duration: 30s for agent runs (120s if designated E2E).
  - Minimum transcript length: 100 lines (200 lines if designated E2E).
  - Transcript must include tool/agent markers (alphaswarm, pattern-forge/refine, agent names).
  - Perfect metrics (e.g., 100%/100%) are invalid and must be flagged.

## Reasoning
When evidence does not match known patterns, the framework must capture anomalies and turn them into validated pattern candidates. The value is in expanding detection coverage from hard cases that current patterns miss.

## Purpose
Define discovery triggers, logging, and routing to pattern-forge or refine workflows, then validate with a live run that produces an actionable candidate.

## Why
B10.O is blocking because novel pattern discovery is required to grow detection coverage from hard cases. Without this, the pattern library becomes stale.
