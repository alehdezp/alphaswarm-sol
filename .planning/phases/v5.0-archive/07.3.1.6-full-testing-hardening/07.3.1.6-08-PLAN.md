Create a hard-case library with protocol categories and scenario entries:

For each hard case, include:
- Category tag (Lending, DEX, Staking, Vault, Oracle, Governance, Bridge)
- Difficulty justification: WHY is this hard? What makes it challenging?
- Provenance: Where does this case come from? (audit report, bug bounty, exploit)
- Required reasoning: What kind of analysis is needed? (cross-function, economic, temporal)
- Expected challenge: What should the system struggle with?
- Attack vector description: What's the actual vulnerability?

Required protocol categories (minimum 7):
1. Lending (flash loans, liquidations, interest rate manipulation)
2. DEX (sandwich attacks, price manipulation, LP attacks)
3. Staking (reward manipulation, unstaking vulnerabilities)
4. Vault (share inflation, deposit/withdraw reentrancy)
5. Oracle (price manipulation, stale data, TWAP attacks)
6. Governance (flash loan governance, vote manipulation)
7. Bridge (cross-chain replay, message verification)

Exclude easy-to-detect patterns like:
- Simple reentrancy without cross-function complexity
- Missing access control on obvious admin functions
- Unchecked return values without economic context
  </action>
