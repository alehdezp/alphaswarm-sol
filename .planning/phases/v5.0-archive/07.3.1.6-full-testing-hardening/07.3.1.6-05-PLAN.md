## Reasoning
These deliverables turn workflow testing into auditable proof. Without outcome-based validation, we only know files exist, not that they deliver their promised value.

## Purpose
Lock in an evidence-backed baseline where:
- Coverage map honestly shows what's tested vs not
- Docs match reality (not aspiration)
- Parallel runs are isolated
- Repeat runs are safe
- Graph corruption is detectable

## Why
Mechanical checks (file exists, contains keyword) create false confidence. Outcome tests prove the capability works in practice.

- Task 3: Document locking policy, verify by running parallel audits and checking isolation
- Task 4: Document idempotency, verify by running commands twice and checking for corruption
- Task 5: Define graph integrity rules, verify by corrupting a graph and checking detection
