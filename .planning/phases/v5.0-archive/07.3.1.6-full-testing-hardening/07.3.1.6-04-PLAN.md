**Reasoning**
Tier B/C scenarios test complex reasoning and context-dependent vulnerabilities. Without proving the system can actually detect these bugs (not just claim to), the framework has no credibility as a security tool. This plan requires demonstration on known vulnerabilities, not fabricated metrics.

**Purpose**
Establish a repeatable Tier B/C harness with:
- A VQL library that enables meaningful graph queries
- Scenario templates that capture real vulnerability complexity
- Evidence requirements that force reasoning through the graph
- Proof of detection on known complex vulnerabilities

**Why**
Phase 7.3.5 benchmarking and 7.3.6 continuous validation depend on Tier B/C scenarios that prove real capability. Without demonstrating detection of known bugs, all metrics are unverifiable.

**How To Implement**
1. Define a VQL library with queries that surface security-relevant code paths.
2. Create Tier B/C scenario templates requiring provenance, context, and counterfactuals.
3. Embed reasoning requirements that tie findings to graph nodes.
4. Execute a first Tier B/C scenario on a contract with a KNOWN vulnerability and demonstrate detection.
5. Add Tier C pattern ranking validation methodology (precision/recall) to the evaluation guide.
