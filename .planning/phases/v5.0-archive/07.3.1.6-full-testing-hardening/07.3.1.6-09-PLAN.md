<task id="T1" type="auto">
  <name>Formalize the pattern evaluation protocol with quality focus</name>
  <files>
    .planning/testing/guides/guide-pattern-evaluation.md
    .planning/testing/templates/PATTERN-EVALUATION-TEMPLATE.md
  </files>
  <action>
Define the evaluation protocol to require:

1. VQL Query Execution
   - Mandatory graph queries before conclusions
   - Query results must be cited in reasoning
   - Missing queries = evaluation incomplete

2. Tool Usage Evidence
   - Slither, Aderyn, Mythril runs when applicable
   - Tool findings cross-referenced with pattern matches
   - Tool disagreements must be explained

3. Self-Critique Requirements
   - Explicit "what could I be wrong about?" section
   - Identify potential false positives proactively
   - Acknowledge limitations and blind spots
   - Self-critique must influence confidence level

4. False-Positive Analysis
   - Every match must be validated, not just counted
   - Suppressed findings require documented rationale
   - Rationale must cite specific code/context evidence

5. Improvement Guidance
   - Evaluation must produce actionable recommendations
   - Recommendations target specific pattern weaknesses
   - "How to make this pattern better" section required

Create a template that captures:
- evidence_ids: list of graph node IDs supporting the finding
- false_positives: explicit false-positive analysis with suppression rationales
- self_critique: acknowledged limitations and uncertainties
- tool_evidence: tool outputs and how they influenced evaluation
- reasoning: step-by-step reasoning chain with evidence citations
- improvements: actionable recommendations for pattern improvement
- verdict: final verdict with confidence and caveats
  </action>

  <done_when>
    - File .planning/testing/guides/guide-pattern-evaluation.md exists
    - Protocol requires VQL, tools, self-critique, FP analysis, improvements
    - Template exists with all required fields
    - Each field has clear documentation of what it must contain
  </done_when>
</task>
