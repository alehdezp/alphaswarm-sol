# 07.3.2-00 Summary: Proof Token Spec, Evidence Pack Schema, Gates

## What Was Done

### Task 1: Proof Token Specification
Defined proof tokens for all 6 critical workflow stages with anti-fake guarantees.

**Stages Covered:**
- `stage.graph_build` — graph_hash, node_count, edge_count, graph_builder_version
- `stage.context_pack` — context_hash, context_sources
- `stage.pattern_match` — pattern_pack_version, pattern_match_count
- `stage.agent_spawn` — agent_id, agent_role, vql_queries, graph_nodes_queried
- `stage.debate` — finding_id, attacker_claim_id, defender_claim_id, verifier_verdict_id
- `stage.report` — report_path, report_hash

**Required Fields (All Proofs):**
- proof_id, run_id, suite_id, stage_id, claude-code-agent-teams_session_id
- command, duration_ms, transcript_path, transcript_hash, nonce, timestamp
- tool_versions, config_hash, graph_schema_version, proof_schema_version

**Hash Chain:** `nonce → transcript → transcript_hash → proof_token → evidence_pack_manifest`

**Verification Rules:**
- Nonce must appear in transcript
- Transcript hash must match captured file
- Stage timestamps must be strictly ordered
- Graph hash must match canonical hash
- All references must exist in evidence pack

### Task 2: Evidence Pack Schema
Defined canonical layout and required artifacts for validation runs.

**Layout:**
```
.vrs/validation/runs/<run_id>/
  manifest.json
  proofs/
  transcripts/
  metrics/
  evidence_pack/
```

**Required Sections (Manifest):**
- run_id, suite_id, command, duration_ms, claude-code-agent-teams_session_id
- transcript_hashes, proof_tokens, debug_artifacts, tool_outputs
- ground_truth_refs, hash_manifest, schema_version

**Invalidation Rules:**
- Missing/empty required artifacts
- Transcript hash mismatch
- Missing/invalid proof tokens
- Missing debug artifacts for required skills/agents
- Tool outputs missing without skip reason
- Duration <= 5000ms

### Task 3: Gate Definitions (G0-G7)
Defined 8 validation gates with explicit pass/fail criteria.

| Gate | Purpose | Key Pass Criteria |
|------|---------|-------------------|
| G0 Preflight | Environment check | claude-code-controller, CLI version, dataset hash, worktree |

**Full claude-code-controller documentation:** See `.planning/testing/rules/claude-code-controller-REFERENCE.md` and `.planning/testing/rules/canonical/claude-code-controller-instructions.md`
| G1 Evidence Integrity | Execution proof | Schema valid, all proof tokens, hash match, nonce |
| G2 Graph Soundness | BSKG integrity | Graph hash match, node/edge counts, operations |
| G3 Ground Truth | External truth | Coverage >= thresholds, provenance recorded |
| G4 Mutation Robustness | Adversarial | Detection rate >= target, clean negative controls |
| G5 Consensus & Variance | Stability | Multi-seed variance within tolerance |
| G6 Regression Baseline | No regression | No precision/recall drop, no new critical FPs |
| G7 Continuous Health | Pipeline | Nightly/weekly pass, trend report generated |
