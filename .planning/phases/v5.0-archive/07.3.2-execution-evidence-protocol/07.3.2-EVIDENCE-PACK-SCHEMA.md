# 07.3.2 Evidence Pack Schema (v1)

## Purpose
Evidence packs are the **canonical output** for all validation runs. No evidence pack → no valid run.

## Canonical Layout
```
.vrs/validation/runs/<run_id>/
  manifest.json
  proofs/
  transcripts/
  metrics/
  evidence_pack/
```

## Required Sections (Manifest)
- `run_id`
- `suite_id`
- `command`
- `duration_ms`
- `claude-code-agent-teams_session_id`
- `transcript_hashes`
- `proof_tokens`
- `debug_artifacts`
- `tool_outputs`
- `ground_truth_refs`
- `hash_manifest`
- `schema_version`

## Required Artifacts
- **Transcripts:** claude-code-agent-teams capture files + hashes
- **Proof Tokens:** per-stage proof JSON files
- **Debug Artifacts:** inputs, tool calls, BSKG/VQL queries, outputs
- **Tool Outputs:** SARIF or JSON outputs or explicit skip reason
- **Ground Truth Manifest:** URLs + hashes + access dates

## Minimal JSON Example (Manifest)
```json
{
  "run_id": "run-20260130-0001",
  "suite_id": "full-testing",
  "command": "/vrs-audit contracts/",
  "duration_ms": 62000,
  "claude-code-agent-teams_session_id": "alphaswarm-claude-code-agent-teams-session",
  "schema_version": "1.0.0",
  "transcript_hashes": {
    "graph": "sha256:...",
    "context": "sha256:...",
    "pattern": "sha256:...",
    "agents": "sha256:...",
    "debate": "sha256:...",

  },
  "proof_tokens": [
    "proofs/proof-graph.json",
    "proofs/proof-context.json",
    "proofs/proof-pattern.json",

  },
  "ground_truth_refs": "evidence_pack/ground_truth/manifest.yaml",
  "hash_manifest": "evidence_pack/hash_manifest.json"
}
```

## Invalidation Rules
Evidence pack is invalid if:
- Any required artifact is missing or empty
- Transcript hashes do not match
- Proof tokens are missing or invalid
- Debug artifacts are absent for required skills/agents
- Tool outputs missing without explicit skip reason
- Duration <= 5000ms

## Versioning
- `schema_version` must increment on any structural change
- Validators must reject unknown versions unless explicitly permitted
