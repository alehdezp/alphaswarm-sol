# AlphaSwarm Testing Gap Analysis: Why Current Phases Don't Prove It Works

## Executive Summary

**The Fundamental Problem:** Phase 7.3 and its sub-phases test "test infrastructure existence" rather than "product functionality."

The testing model assumes:
```
Write test script → Run script → Check output exists → PASS
```

But the reality is:
```
Write test script → Script exists → Mark complete → Never actually ran → No evidence product works
```

## Root Cause Analysis

### Gap 1: No Real Claude Code Session Testing

**What should happen:**
```
Open Claude Code → Run /vrs-audit contracts/ → Observe 7 stages → See findings
```

**What testing does:**
```
Create script that WOULD run /vrs-audit → Check script exists → PASS
```

**The fix needed:** Actually run Claude Code sessions with observable output.

**Full claude-code-controller documentation:** See `.planning/testing/rules/claude-code-controller-REFERENCE.md` and `.planning/testing/rules/canonical/claude-code-controller-instructions.md`

### Gap 2: No Agent Spawning Validation

**What should happen:**
```
/vrs-audit → Stage 5 → Spawn vrs-attacker → Agent queries BSKG → Produces finding
```

**What testing assumes:**
```
Stage 5 code exists → Therefore agents spawn
```

**Evidence needed but missing:**
- Agent spawn timestamps
- Graph nodes queried by agents
- Agent reasoning chains
- Tool calls made by agents

### Gap 3: No BSKG Query Evidence

**What should happen:**
```
Agent investigates finding → Queries graph: "FIND functions WHERE has_reentrancy = true"
                          → Gets node: func_0x1234_withdraw
                          → Includes in evidence packet
```

**What we have:**
- No query logs
- No node references in findings
- No evidence agents actually use the graph

### Gap 4: No Multi-Agent Debate Evidence

**What should happen:**
```
Attacker: "This is exploitable because X"
Defender: "But guard exists at Y"
Verifier: "Checking X vs Y... Verdict: CONFIRMED"
```

**What we have:**
- Debate protocol code exists
- No debate transcripts
- No evidence debate ever ran

### Gap 5: No Economic Context Integration

**What should happen:**
```
Stage 3 → Exa MCP search → "lending protocol exploits 2024"
        → Returns: Euler hack, Compound vulnerability
        → Context pack includes relevant exploits
```

**What we have:**
- Exa MCP documented in skill
- No evidence Exa was ever called
- No context packs with external research

### Gap 6: No End-User Perspective Testing

**What developer testing does:**
```
From dev repo → With CLAUDE.md → With all context → Test script
```

**What user would do:**
```
uv tool install alphaswarm → cd my-project → claude → /vrs-audit contracts/
```

**Gap:** User has no CLAUDE.md, no .planning/, no developer knowledge.

## The Meta-Problem: Testing Tests, Not Product

Current phases verify:
1. ✓ Test scripts exist
2. ✓ Test infrastructure is substantive
3. ✓ Ground truth fixtures created
4. ✓ Skill file has content

Current phases DON'T verify:
1. ✗ Product actually runs
2. ✗ Vulnerabilities actually detected
3. ✗ Agents actually spawn and reason
4. ✗ Graph actually gets queried
5. ✗ Debate actually happens
6. ✗ Report actually contains valid findings

## What Real-World Validation Requires

### Non-Negotiable Evidence

| Evidence Type | How to Get It | Why It Matters |
|--------------|---------------|----------------|
| Execution transcript | claude-code-agent-teams capture of Claude Code session | Proves 7 stages ran |
| Graph node IDs | Logged queries during audit | Proves graph used |
| Agent spawn logs | Agent lifecycle timestamps | Proves agents ran |
| Debate transcript | Attacker/Defender/Verifier exchange | Proves debate happened |

```
BEFORE: "Scripts exist therefore product works"

```

## Required Paradigm Shift

### Old Model: Script-Based Testing
```
1. Create test script
2. Run script (subprocess, claude-code-controller)
3. Check outputs exist
4. Parse outputs for expected markers
5. Report pass/fail
```

**Problem:** If product doesn't produce outputs, tests fail but don't tell you why.

### New Model: Evidence-Based Validation
```
1. Run actual product (Claude Code with /vrs-audit)
2. Collect execution evidence (not just outputs)
3. Validate evidence chain:
   - Did graph build? (Show node count, sample nodes)
   - Did agents spawn? (Show spawn timestamps, agent IDs)
   - Did queries run? (Show VQL queries, results)
   - Did debate occur? (Show claim/counterclaim/verdict)
   - Did detection match ground truth? (Show finding vs expected)
4. If any evidence missing → Debug that specific stage
5. Fix root cause, not test harness
```

---

## Conclusion

**Phase 7.3 tested the test infrastructure, not the product.**

The next phases must:
1. Force real execution with observable evidence
2. Debug actual failures (not just report them)
3. Fix product, not just tests
4. Iterate until product demonstrably works
5. Capture evidence that proves it works

This requires a fundamental shift from "does the file exist?" to "did the operation succeed with evidence?"
