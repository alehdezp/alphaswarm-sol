# 07.3.2 Proof Token Spec (v1)

## Purpose
Proof tokens provide **non-fakeable evidence** that each critical workflow stage executed in a real claude-code-agent-teams-driven Claude Code session.

A proof token is **invalid** if any required field is missing, if the transcript hash does not match, or if the nonce challenge is not echoed in the claude-code-agent-teams transcript.

## Required Fields (All Proofs)
- `proof_id`
- `run_id`
- `suite_id`
- `stage_id`
- `claude-code-agent-teams_session_id`
- `command`
- `duration_ms`
- `transcript_path`
- `transcript_hash`
- `nonce`
- `timestamp`
- `tool_versions`
- `config_hash`
- `graph_schema_version`
- `proof_schema_version`

## Hash Chain (Anti-Fake)
```
nonce → transcript → transcript_hash → proof_token → evidence_pack_manifest
```
If any link is missing or mismatched, the run is invalid.

## Stage Coverage
Proof tokens are required for:
- `stage.graph_build`
- `stage.context_pack`
- `stage.pattern_match`
- `stage.agent_spawn`
- `stage.debate`
- `stage.report`

## Stage-Specific Fields

### Graph Build Proof
Additional fields:
- `graph_hash`
- `node_count`
- `edge_count`
- `graph_builder_version`

**Non-fakeable rule:** `graph_hash` must match the canonical hash for the contract source + builder + tool versions.

### Context Pack Proof
Additional fields:
- `context_hash`
- `context_sources` (URLs + access dates)

**Non-fakeable rule:** context references must exist in the evidence pack and match the hash.

### Pattern Match Proof
Additional fields:
- `pattern_pack_version`
- `pattern_match_count`

**Non-fakeable rule:** count must match findings referenced in evidence pack.

### Agent Spawn Proof
Additional fields:
- `agent_id`
- `agent_role`
- `vql_queries`
- `graph_nodes_queried`

**Non-fakeable rule:** queried nodes must exist in the graph and be referenced in outputs.

### Debate Proof
Additional fields:
- `finding_id`
- `attacker_claim_id`
- `defender_claim_id`
- `verifier_verdict_id`

**Non-fakeable rule:** evidence chain must link to graph node IDs.

### Report Proof
Additional fields:
- `report_path`
- `report_hash`

**Non-fakeable rule:** report hash must match the report content stored in the evidence pack.

## Verification Rules (Summary)
- **Nonce rule:** nonce must appear in transcript.
- **Hash rule:** transcript hash must match captured transcript.
- **Temporal rule:** stage timestamps must be strictly ordered.
- **Graph rule:** graph hash must match canonical hash.
- **Evidence rule:** proof token references must exist in evidence pack.

## Failure Outcomes
Any invalid proof token is an automatic **G1 Evidence Integrity** failure and invalidates the run.
