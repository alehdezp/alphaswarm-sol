# 07.3.2 Gate Definitions (G0–G7)

## G0 Preflight
**Purpose:** Ensure claude-code-agent-teams + toolchain + dataset integrity.

**Full claude-code-controller documentation:** See `.planning/testing/rules/claude-code-controller-REFERENCE.md` and `.planning/testing/rules/canonical/claude-code-controller-instructions.md`

**Pass Criteria:**
- claude-code-controller available
- alphaswarm CLI version matches expected
- dataset hash matches manifest
- worktree isolation confirmed

**Fail Conditions:**
- missing claude-code-agent-teams
- version mismatch
- dataset hash mismatch

---

## G1 Evidence Integrity
**Purpose:** Prove execution actually happened.

**Pass Criteria:**
- evidence pack schema validates
- proof tokens exist for every stage
- transcript hashes match
- nonce appears in transcripts

**Fail Conditions:**
- any missing proof token
- transcript hash mismatch
- missing nonce

---

## G2 Graph Soundness
**Purpose:** Ensure BSKG is real and consistent.

**Pass Criteria:**
- graph hash matches canonical hash
- node/edge counts non-zero
- required operations detected

**Fail Conditions:**
- graph hash mismatch
- missing operations

---

## G3 Ground Truth Coverage
**Purpose:** Ensure external ground truth coverage per category.

**Pass Criteria:**
- coverage >= thresholds per category
- provenance recorded

**Fail Conditions:**
- category below threshold
- missing provenance

---

## G4 Mutation Robustness
**Purpose:** Prove detection across adversarial variants.

**Pass Criteria:**
- mutation detection rate >= target
- negative controls remain clean

**Fail Conditions:**
- below threshold detection
- false positives on safe variants

---

## G5 Consensus & Variance
**Purpose:** Ensure multi-run stability.

**Pass Criteria:**
- multi-seed variance within tolerance
- attacker/defender/verifier disagreement below threshold

**Fail Conditions:**
- high variance or instability

---

## G6 Regression Baseline
**Purpose:** Prevent regression vs locked baseline.

**Pass Criteria:**
- no drop in precision/recall
- no new critical false positives

**Fail Conditions:**
- regression beyond tolerance

---

## G7 Continuous Health
**Purpose:** Validate continuous pipeline health.

**Pass Criteria:**
- nightly + weekly suites pass
- trend report generated

**Fail Conditions:**
- pipeline failures or missing reports
