# Phase 07.3.2 Plan 02: Validation Validators Summary

**One-liner:** Proof token validator, evidence pack validator, and G1/G2 gate enforcement with fail-fast behavior and machine-readable output.

## What Was Built

### 1. Proof Token Validator (`scripts/e2e/validate_proof_tokens.py`)

Validates proof tokens according to 07.3.2-PROOF-TOKEN-SPEC.md:

| Check | Description |
|-------|-------------|
| Structure | All required fields present and non-empty |
| Hashes | Transcript hash matches actual content (sha256) |
| Nonce | Nonce echoed in transcript |
| Temporal | Stages in correct execution sequence |
| Cross-refs | Graph node references exist (optional) |

**Key functions:**
- `validate_proof_tokens(proofs_dir, transcripts_dir, graph_path, strict)` -> `ValidationResult`
- `validate_single_proof_token(token_path, transcript_path)` -> `(is_valid, errors)`

### 2. Evidence Pack Validator (`scripts/e2e/validate_evidence_pack.py`)

Validates evidence packs according to 07.3.2-EVIDENCE-PACK-SCHEMA.md:

| Component | Validation |
|-----------|------------|
| Manifest | Required fields, schema version, duration > 5s |
| Transcripts | All stages present, hashes match content |
| Proof Tokens | Files exist, contain valid JSON |
| Debug Artifacts | If referenced, must exist |
| Tool Outputs | Present or have explicit skip reason |

**Key functions:**
- `validate_pack(evidence_dir, strict, check_hash_integrity)` -> `PackValidationResult`
- `validate_pack_completeness(evidence_dir)` -> `(is_complete, missing_items)`

### 3. G1/G2 Gate Validator (`scripts/e2e/validate_gate_g1_g2.py`)

Enforces gates from 07.3.2-GATES.md:

**G1 Evidence Integrity:**
- G1.1: Evidence pack schema validates
- G1.2: Proof tokens exist for every stage
- G1.3: Transcript hashes match
- G1.4: Nonces appear in transcripts

**G2 Graph Soundness:**
- G2.1: Graph hash matches canonical (or is well-formed)
- G2.2: Node/edge counts non-zero
- G2.3: Semantic operations detected

**Key functions:**
- `validate_gates_g1_g2(evidence_dir, graph_path, expected_graph_hash)` -> `GateValidationResult`

## Usage Examples

```bash
# Validate proof tokens
uv run python scripts/e2e/validate_proof_tokens.py proofs/ --transcripts transcripts/

# Validate evidence pack
uv run python scripts/e2e/validate_evidence_pack.py .vrs/validation/runs/run-001/

# Enforce G1/G2 gates
uv run python scripts/e2e/validate_gate_g1_g2.py .vrs/validation/runs/run-001/ --json

# Strict mode (warnings become errors)
uv run python scripts/e2e/validate_proof_tokens.py proofs/ --strict
```
