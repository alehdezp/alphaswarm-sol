## Implementation Details

### Gate Aggregator (1126 LOC)

The `validate_all_gates.py` script provides:

- **G0 Preflight**: claude-code-agent-teams availability, alphaswarm CLI, evidence structure
- **G1/G2 Integration**: Calls `validate_gate_g1_g2.py` and parses JSON output
- **G3 Ground Truth**: Calls `validate_ground_truth.py` with manifest path
- **G4 Mutation**: Reads metrics from evidence pack
- **G5 Variance**: Reads multiseed variance results
- **G6 Regression**: Reads baseline comparison metrics
- **G7 Continuous**: Reads CI status from evidence pack

Output formats:
- JSON report: `<output>/<run_id>_gates.json`
- Markdown report: `<output>/<run_id>_gates.md`

### CLI Wrapper (381 LOC)

The `validate_gates_cli.sh` shell script:

```bash
# Basic usage
./validate_gates_cli.sh .vrs/validation/runs/run-001/

# Dry-run to preview command
./validate_gates_cli.sh --dry-run evidence/

# With custom config
./validate_gates_cli.sh run-001 --config configs/gate_runner.yaml
```

Features:
- Color-coded output (PASS/FAIL/WARN)
- Dry-run mode showing command plan
- Automatic default config detection
- Report path printing on completion

### Gate Runner Config (149 LOC)

The `configs/gate_runner.yaml` defines:

```yaml
gates:
  - gate_id: G0
    enabled: true
    fail_fast: true  # Infrastructure must pass
  - gate_id: G1-G7
    enabled: true

required_gates: [G0, G1, G2, G3]

thresholds:
  mutation_detection_rate: 0.75
  max_variance_coefficient: 0.15
  min_agent_agreement_rate: 0.80
```
