# Phase 7.3.2: Execution Evidence Protocol

## Purpose

Force every product claim to have observable, non-fakeable evidence of actual execution.

---

## The Problem It Solves

Current testing checks "did file get created?" instead of "did the product work?"

Example:
```python
# BAD: Current approach
if Path(".vrs/findings/verdicts.json").exists():
    PASS  # But file could be empty or fake!

# GOOD: Evidence-based approach
verdicts = json.load(Path(".vrs/findings/verdicts.json"))
assert len(verdicts) > 0
assert verdicts[0]["finding_id"].startswith("bskg_")  # Must reference graph
assert verdicts[0]["evidence_chain"] is not None
assert verdicts[0]["execution_proof"] matches graph_hash
```

## Core Innovation: Execution Proof Tokens

Every major operation must produce a unique "proof token" that can ONLY exist if the operation actually ran correctly.

### Proof Token Types

#### 1. BSKG Build Proof
```json
{
  "proof_type": "bskg_build",
  "contract_hash": "sha256(contract_source)",
  "node_count": 245,
  "edge_count": 892,
  "sample_nodes": ["func_0x1234_withdraw", "func_0x5678_deposit"],
  "graph_hash": "sha256(graph_content)",
  "timestamp": "2026-01-30T10:00:00Z",
  "verification": "graph_hash == expected_from_contract_hash"
}
```

**Why it can't be faked:**
- `contract_hash` is deterministic from source
- `graph_hash` must match expected value for that contract
- If graph is empty/fake, hash won't match

#### 2. Agent Spawn Proof
```json
{
  "proof_type": "agent_spawn",
  "agent_id": "vrs-attacker-20260130-100123",
  "agent_type": "vrs-attacker",
  "spawn_timestamp": "2026-01-30T10:01:23Z",
  "graph_nodes_queried": ["func_0x1234_withdraw"],
  "vql_queries_executed": ["FIND functions WHERE has_reentrancy_pattern = true"],
  "tool_calls": ["alphaswarm query", "Read file"],
  "findings_produced": ["finding_001"],
  "termination_timestamp": "2026-01-30T10:02:45Z"
}
```

**Why it can't be faked:**
- `graph_nodes_queried` must be valid nodes from BSKG
- `vql_queries_executed` must return matching results
- Timestamps must show reasonable duration

#### 3. Debate Proof
```json
{
  "proof_type": "multi_agent_debate",
  "finding_id": "finding_001",
  "rounds": [
    {
      "round": 1,
      "attacker_claim": {
        "text": "Function withdraw() is vulnerable to reentrancy",
        "evidence_nodes": ["func_0x1234_withdraw"],
        "graph_properties_cited": ["has_external_call_before_state_write: true"]
      },
      "defender_response": {
        "text": "ReentrancyGuard is inherited but not applied",
        "evidence_nodes": ["contract_Vault"],
        "graph_properties_cited": ["inherits_reentrancy_guard: true", "function_has_nonreentrant: false"]
      },
      "verifier_analysis": {
        "text": "Attacker is correct. Guard exists but not applied to this function.",
        "evidence_weighed": "attacker 2 points, defender 1 point",
        "verdict": "CONFIRMED"
      }
    }
  ],
  "final_verdict": "CRITICAL - Reentrancy confirmed"
}
```

**Why it can't be faked:**
- Each claim cites specific graph nodes
- Graph properties must actually exist in graph
- Verifier must reference both attacker and defender claims

#### 4. Detection Proof
```json
{
  "proof_type": "vulnerability_detection",
  "finding_id": "finding_001",
  "ground_truth_id": "DVD-NR-001",
  "detection_method": "pattern_match + agent_investigation + debate",
  "evidence_chain": {
    "pattern_matched": "reentrancy-classic",
    "graph_node": "func_0x1234_flashLoan",
    "source_location": "NaiveReceiverLenderPool.sol:42-58",
    "behavioral_signature": "R:bal→X:out→W:bal",
    "agent_confirmation": "vrs-attacker-20260130-100123",
    "debate_verdict": "CONFIRMED"
  },
  "ground_truth_match": true,
  "is_true_positive": true
}
```

**Why it can't be faked:**
- Must match external ground truth
- Evidence chain must trace through graph
- All referenced entities must exist

## Validation Rules

### Rule 1: No Empty Proofs
```python
def validate_proof(proof: dict) -> bool:
    # Every proof must have timestamp
    assert "timestamp" in proof

    # Every proof must reference graph nodes
    if proof["proof_type"] != "bskg_build":
        assert len(proof.get("graph_nodes_referenced", [])) > 0

    # Graph build proof must have non-zero counts
    if proof["proof_type"] == "bskg_build":
        assert proof["node_count"] > 0
        assert proof["edge_count"] > 0

    return True
```

### Rule 2: Cross-Reference Validation
```python
def validate_cross_references(proofs: list) -> bool:
    bskg_proof = find_proof(proofs, "bskg_build")
    agent_proofs = find_proofs(proofs, "agent_spawn")

    # All agent-referenced nodes must exist in graph
    for agent in agent_proofs:
        for node in agent["graph_nodes_queried"]:
            assert node in bskg_proof["sample_nodes"] or node_exists_in_graph(node)

    return True
```

### Rule 3: Temporal Consistency
```python
def validate_temporal_order(proofs: list) -> bool:
    # BSKG must be built before agents spawn

        assert agent_time > bskg_time

    debate_proofs = find_proofs(proofs, "multi_agent_debate")

        # ... validate debate after relevant agents

    return True
```

### Rule 4: Ground Truth Matching
```python
def validate_ground_truth(detection_proofs: list, ground_truth: dict) -> dict:
    true_positives = 0
    false_positives = 0
    false_negatives = set(ground_truth.keys())

    for detection in detection_proofs:
        gt_id = detection.get("ground_truth_id")
        if gt_id in ground_truth:
            true_positives += 1
            false_negatives.discard(gt_id)
        else:
            false_positives += 1

    return {
        "precision": true_positives / (true_positives + false_positives),
        "recall": true_positives / (true_positives + len(false_negatives)),
        "true_positives": true_positives,
        "false_positives": false_positives,
        "false_negatives": len(false_negatives)
    }
```

```
┌─────────────────────────────────────────────────────────────┐
│                    EXECUTION CYCLE                          │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. RUN /vrs-audit on test contract                         │
│     ↓                                                       │
│  2. COLLECT all proof tokens                                │
│     ↓                                                       │
│  3. VALIDATE proofs against rules                           │
│     │                                                       │
│     ├── All proofs valid? ────────────→ PASS               │
│     │                                                       │
│     └── Proof invalid/missing ────→ 4. INVESTIGATE         │
│                                          ↓                  │
│                                     5. FIX root cause       │
│                                          ↓                  │
│                                     6. GOTO 1 (re-run)      │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### Investigation Protocol

When a proof is missing or invalid:

```yaml
Missing BSKG Build Proof:
  investigate:
    - Did alphaswarm build-kg run?
    - Did Slither succeed?
    - Is contract path correct?
  fix_targets:
    - Setup stage
    - Slither configuration
    - Path resolution

Missing Agent Spawn Proof:
  investigate:
    - Did Stage 5 execute?
    - Were agents actually spawned?
    - Did agents crash on startup?
  fix_targets:
    - Agent spawn logic in skill
    - Agent configuration
    - Error handling

Missing Debate Proof:
  investigate:
    - Did swarm mode run?
    - Did all 3 agents participate?
    - Did debate protocol execute?
  fix_targets:
    - Mode detection
    - Agent communication
    - Debate orchestration

Invalid Detection Proof:
  investigate:
    - Does finding match ground truth?
    - Is evidence chain complete?
    - Are graph references valid?
  fix_targets:
    - Pattern definitions
    - Agent reasoning
    - Evidence collection
```

## Exit Criteria

Phase 7.3.2 is COMPLETE when:

1. **All proof types validated** for at least one full audit run
2. **Cross-reference validation** passes (all nodes exist)
3. **Temporal consistency** verified (correct order)
4. **At least one ground truth match** (true positive detected)
5. **Metrics not suspicious** (precision < 100%, recall < 100%)

Phase CANNOT be marked complete if:
- Any proof type is missing
- Any proof fails validation
- No ground truth matches
- Metrics are 0% or 100% (suspicious)
