## Reasoning
The original plan did not validate workspace isolation or collision behavior, which can silently corrupt evidence. This plan adds explicit collision checks and parallel safety validation.

## Purpose
- Enable isolated, repeatable workflow tests.
- Support debug inspection of failed runs.
- Enforce claude-code-agent-teams evidence collection in workspace runs.
- Prevent state leakage across parallel tests.

## Why
- Parallel workflow tests require isolation to avoid state leakage.
- Rollback allows fast iteration without rerunning graph build.
- Debug preservation is essential for diagnosing failures.
- Collision detection prevents corrupt evidence packs.

## How To Implement

### T1: Extend workspace script to support synthetic repo fixtures
**Goal:** Add `create-from-fixture` and checkpoint rollback support for tests/fixtures/repos using `jj workspace add` and `jj edit`.

### T2: Implement snapshot + debug preservation policy
**Goal:** Ensure .vrs artifacts can be preserved on failure or debug mode.

### T3: Add isolation and collision checks
**Goal:** Validate .vrs paths and parallel workspace safety.

### T4: Update workflow docs and tooling
**Goal:** Ensure users can discover how to run workspace harness.

### T5: claude-code-controller smoke test in a workspace
**Goal:** Produce evidence that workspace runs are usable.
