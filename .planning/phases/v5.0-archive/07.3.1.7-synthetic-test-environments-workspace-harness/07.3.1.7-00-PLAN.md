## Reasoning
The original plan lacked explicit quality gates and reproducibility constraints, which would allow low-quality or inconsistent repos to slip in. This plan fixes that by defining measurable quality thresholds and a deterministic template/seed model.

## Purpose
- Standardize synthetic repo structure.
- Guarantee three scenarios per workflow.
- Enforce quality gates and reproducibility.
- Prevent misuse of synthetic expectations as external ground truth.

## Why
- Repeatable workflows require deterministic repo structure and checkpoints.
- Scenario coverage is meaningless without explicit success criteria.
- Quality gates prevent junk repos that would degrade validation.
- Synthetic repos are for workflow validation, not external benchmark claims.

## How To Implement

### T1: Inventory existing fixtures and repo shapes
**Goal:** Identify reusable patterns, avoid duplication, and classify existing synthetic shapes.

### T2: Define standardized repo template + manifest schema
**Goal:** Ensure every repo has consistent structure, checkpoints, and expectations metadata.

### T3: Define synthetic repo quality gates
**Goal:** Enforce minimum vulnerability diversity, decoy signals, safe controls, and graph/compile viability.

### T4: Build workflow coverage map with 3 scenarios per workflow
**Goal:** Guarantee scenario completeness and explicit observable criteria.

### T5: Tooling update for synthetic repo workflow
**Goal:** Document how to generate and validate synthetic repos in `.planning/TOOLING.md`.
