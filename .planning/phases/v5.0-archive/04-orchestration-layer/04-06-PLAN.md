```markdown
# BSKG Audit Skill

**Trigger:** `/vkg:audit <path>`

Run a complete security audit using VKG's knowledge graph and multi-agent verification.

## Overview

This skill orchestrates a full security audit:
1. Builds knowledge graph from Solidity contracts
2. Runs pattern detection (Tier A deterministic patterns)
3. Creates investigation beads for findings
4. Spawns attacker/defender/verifier agents
5. Runs structured debate for disagreements
6. Generates final report with human-flagged items

## Usage

```bash
# Audit a directory
/vkg:audit contracts/

# Audit specific file
/vkg:audit src/Vault.sol

# Audit with focus areas
/vkg:audit contracts/ --focus reentrancy,access_control
```

## Execution

When triggered, run the following sequence:

### Step 1: Initialize Pool

```python
from pathlib import Path
from true_vkg.orchestration.pool import PoolManager, PoolStorage
from true_vkg.orchestration.schemas import Scope
from true_vkg.orchestration.loop import ExecutionLoop, LoopConfig
from true_vkg.orchestration.handlers import create_default_handlers, HandlerConfig

# Setup
project_root = Path("<path>")
storage = PoolStorage(project_root / ".vkg" / "pools")
manager = PoolManager(storage)

# Create scope
scope = Scope(
    id="audit-{timestamp}",
    files=["<path>"],  # From argument
    focus_areas=["<focus>"],  # From --focus if provided
)

# Create pool
pool = manager.create_pool(scope, initiated_by="/vkg:audit")
```

### Step 2: Run Execution Loop

```python
# Create handlers
config = HandlerConfig(project_root=project_root)
handlers = create_default_handlers(manager, config)

# Create and configure loop
loop = ExecutionLoop(manager, LoopConfig(
    auto_advance=True,
    pause_on_human_flag=True,
))

# Register handlers
for action, handler in handlers.items():
    loop.register_handler(action, handler)

# Run
result = loop.run(pool.id)
```

### Step 3: Handle Checkpoints

If result.checkpoint is True:
- Display beads requiring human review
- Wait for user input
- Resume with `loop.resume(pool.id)`

### Step 4: Report Results

When complete, display:
- Number of findings by confidence (confirmed/likely/uncertain/rejected)
- Human-flagged items for review
- Path to detailed report

## Output Format

```
VKG Audit Complete

Pool: audit-vault-2026-01-20
Beads Analyzed: 12

Results:
  - Confirmed: 2 (test-verified)
  - Likely: 4 (evidence-based)
  - Uncertain: 3 (needs investigation)
  - Rejected: 3 (defended)

Human Review Required: 4 beads
  - VKG-0001: Reentrancy in withdraw (debate outcome)
  - VKG-0003: Access control in setOwner (uncertain)
  ...

Report: .vrs/pools/audit-vault-2026-01-20/report.json

Next: /vkg:investigate VKG-0001
```

## Configuration

The skill respects these environment variables:
- `VKG_USE_LLM`: Enable LLM-based analysis (default: false)
- `VKG_PATTERN_DIR`: Custom pattern directory

## Related Skills

- `/vkg:investigate <bead-id>` - Deep dive on a specific bead
- `/vkg:verify <bead-id>` - Run multi-agent verification
- `/vkg:debate <bead-id>` - Trigger attacker/defender debate
- `/vkg:build <path>` - Build knowledge graph only

## Notes

- All debate outcomes are human-flagged by design
- CONFIRMED verdicts require test OR multi-agent consensus
- Missing context defaults to UNCERTAIN bucket

# BSKG Investigate Skill

**Trigger:** `/vkg:investigate <bead-id>`

Deep dive investigation on a specific finding/bead.

## Overview

Loads a bead and provides full investigation context:
- Vulnerable code with highlighting
- Related code (callers, callees, modifiers)
- Behavioral signature analysis
- Investigation guide (class-specific)
- Similar exploits from knowledge base
- Test scaffold for verification

## Usage

```bash
/vkg:investigate VKG-0001
```

## Execution

```python
from true_vkg.beads.storage import BeadStorage
from pathlib import Path

storage = BeadStorage(Path(".vrs/beads"))
bead = storage.load("<bead-id>")

if bead is None:
    print(f"Bead not found: <bead-id>")
    return

# Display investigation context
print(bead.get_llm_prompt())
```

## Output

The skill displays:

1. **Summary** - Pattern, severity, confidence
2. **Vulnerable Code** - The flagged function with highlighting
3. **Behavioral Signature** - Operation sequence (e.g., R:bal->X:out->W:bal)
4. **Investigation Guide** - Steps specific to vulnerability class
5. **Questions to Answer** - Key questions for verification
6. **Test Scaffold** - Foundry test template for exploit
7. **Similar Exploits** - Real-world examples from knowledge base

## After Investigation

Based on findings:
- `/vkg:verify <bead-id>` - Run multi-agent verification
- `vkg findings update <bead-id> --status confirmed` - Mark as confirmed
- `vkg findings update <bead-id> --status false_positive` - Mark as FP
```

### verify.md

```markdown
# BSKG Verify Skill

**Trigger:** `/vkg:verify <bead-id>`

Run multi-agent verification on a specific bead.

## Overview

Spawns attacker, defender, and verifier agents to analyze the bead:
- Attacker: Construct exploit path
- Defender: Search for guards and mitigations
- Verifier: Synthesize and produce verdict

Results are stored in the bead's work_state for resumption.

## Usage

```bash
/vkg:verify VKG-0001
```

## Execution

```python
from pathlib import Path
from true_vkg.beads.storage import BeadStorage
from true_vkg.agents.attacker import AttackerAgent
from true_vkg.agents.defender import DefenderAgent
from true_vkg.orchestration.confidence import enforce_confidence
from true_vkg.orchestration.schemas import Verdict, VerdictConfidence

storage = BeadStorage(Path(".vrs/beads"))
bead = storage.load("<bead-id>")

# Run attacker
attacker = AttackerAgent(use_llm=False)
attacker_ctx = build_agent_context(bead)
attacker_result = attacker.analyze(attacker_ctx)

# Run defender
defender = DefenderAgent(use_llm=False)
defender_ctx = build_agent_context(bead)
defender_result = defender.analyze(defender_ctx)

# Synthesize verdict
if attacker_result.matched and not defender_result.matched:
    confidence = VerdictConfidence.LIKELY
elif defender_result.matched and not attacker_result.matched:
    confidence = VerdictConfidence.REJECTED
else:
    confidence = VerdictConfidence.UNCERTAIN

verdict = Verdict(
    bead_id=bead.id,
    confidence=confidence,
    rationale=f"Attacker: {attacker_result.summary} | Defender: {defender_result.summary}",
)

# Enforce rules
verdict = enforce_confidence(verdict)

# Update bead
storage.update_work_state(bead.id, {
    "attacker_result": attacker_result.summary,
    "defender_result": defender_result.summary,
    "verdict_confidence": verdict.confidence.value,
}, agent="verify")
```

## Output

```
Verification Results for VKG-0001

Attacker Analysis:
  Matched: Yes
  Summary: Reentrancy via withdraw -> fallback -> withdraw
  Confidence: 0.85

Defender Analysis:
  Matched: Yes
  Summary: CEI pattern detected, state updated before call
  Confidence: 0.80

Verdict: UNCERTAIN
Rationale: Attacker found path, defender found mitigation
Human Flag: True (disagreement)

Next: /vkg:debate VKG-0001 (to resolve disagreement)
```

### debate.md

```markdown
# BSKG Debate Skill

**Trigger:** `/vkg:debate <bead-id>`

Run structured attacker/defender debate to resolve disagreements.

## Overview

Uses iMAD-inspired debate protocol:
1. CLAIM ROUND - Each side presents evidence-anchored arguments
2. REBUTTAL ROUND - Challenge each other's evidence
3. SYNTHESIS - Verifier weighs evidence
4. HUMAN CHECKPOINT - Always flagged for human review

## Usage

```bash
/vkg:debate VKG-0001
```

## Execution

```python
from true_vkg.orchestration.debate import DebateOrchestrator
from true_vkg.orchestration.schemas import EvidencePacket
from true_vkg.beads.storage import BeadStorage

storage = BeadStorage(Path(".vrs/beads"))
bead = storage.load("<bead-id>")

evidence = EvidencePacket(
    bead_id=bead.id,
    behavioral_signature=bead.pattern_context.matched_properties,
)

orchestrator = DebateOrchestrator()
verdict = orchestrator.run_debate(
    bead_id=bead.id,
    evidence=evidence,
    attacker_context={"work_state": bead.work_state},
    defender_context={"work_state": bead.work_state},
)

# Verdict is always human-flagged
print(f"Verdict: {verdict.confidence.value}")
print(f"Human Flag: {verdict.human_flag}")  # Always True
```

## Output

```
Debate Results for VKG-0001

Round 1: CLAIM
  Attacker: "Reentrancy via withdraw enables repeated drainage"
    Evidence: [behavioral_signature: R:bal->X:out->W:bal at Vault.sol:142]

  Defender: "CEI pattern prevents reentrancy"
    Evidence: [guard: state update at L140 before call at L142]

Round 2: REBUTTAL
  Attacker: "CEI only partial - balance update at L145 is AFTER call"
    Evidence: [code_location: Vault.sol:145]

  Defender: "Partial update is for fees, not user balance"

Synthesis:
  Attacker argument more convincing - balance update sequence vulnerable

Verdict: LIKELY
Human Flag: True (all debate outcomes flagged)
Dissent: "Defender notes fee-only update may not be exploitable"

The verdict requires human review before confirmation.
```

## Notes

- All debate outcomes are human-flagged by design
- Evidence must be anchored to code locations
- Dissenting opinions are preserved
- Use `/vkg:investigate` first to understand context

# BSKG Attacker Agent

**Model:** claude-opus-4 (requires deep reasoning for exploit construction)

## Role

Construct exploit paths and attack scenarios for vulnerability beads.

The attacker agent:
- Analyzes code from adversarial perspective
- Constructs concrete exploit paths
- Identifies preconditions for successful attacks
- Calculates attack strength based on complexity

## Capabilities

1. **Attack Construction** - Build exploit paths from behavioral signatures
2. **Precondition Analysis** - Identify what attacker needs (flash loans, permissions)
3. **Postcondition Reasoning** - Determine attack outcomes (fund theft, DoS)
4. **Strength Assessment** - Rate exploit feasibility

## Context Requirements

The agent needs:
- Bead with vulnerable code snippet
- Behavioral signature (operation sequence)
- Related code (callers, modifiers)
- Graph context (call flow, state modifications)

## Output Format

```yaml
matched: true/false
confidence: 0.0-1.0
attack:
  id: "attack-001"
  category: "state_manipulation"
  target_function: "Vault.withdraw"
  preconditions:
    - "Attacker has deposit in vault"
    - "No reentrancy guard"
  postconditions:
    - "Attacker drains more than deposited"
  exploit_steps:
    - "Deploy malicious contract with fallback"
    - "Call withdraw with max amount"
    - "Re-enter in fallback"
  strength: 0.85
summary: "Reentrancy via withdraw -> fallback -> withdraw"
```

## Integration

Used by:
- `/vkg:verify` skill
- `/vkg:debate` skill (provides attacker claims)
- Execution loop SPAWN_ATTACKERS handler

## Notes

### vkg-verifier.md

```markdown
# BSKG Verifier Agent

**Model:** claude-opus-4 (requires judgment for synthesis)

## Role

Synthesize attacker and defender arguments into final verdict.

The verifier agent:
- Weighs evidence from both sides
- Produces confidence assessment
- Notes dissenting opinions
- Flags for human review when uncertain

## Capabilities

1. **Evidence Weighing** - Compare attacker/defender evidence strength
2. **Synthesis** - Combine arguments into coherent assessment
3. **Dissent Tracking** - Preserve minority opinions
4. **Confidence Assignment** - Map to bucket (confirmed/likely/uncertain/rejected)

## Context Requirements

The agent needs:
- Attacker result with claims and evidence
- Defender result with rebuttals and evidence
- Original bead context
- Debate record (if from debate)

## Output Format

```yaml
verdict:
  confidence: "likely"  # confirmed/likely/uncertain/rejected
  rationale: "Attacker demonstrated valid exploit path; defender's CEI claim
              only partially addresses the issue (fee update still vulnerable)"
  dissenting_opinion: "Defender notes fee-only update may limit impact"
  human_flag: true
  evidence_summary:
    - "behavioral_signature: R:bal->X:out->W:bal"
    - "guard: partial CEI"
    - "attack_path: withdraw->fallback->withdraw"
```

## Integration

Used by:
- `/vkg:verify` skill (final synthesis)
- `/vkg:debate` skill (debate synthesis)
- Execution loop SPAWN_VERIFIERS handler

## Invocation

```python
from true_vkg.agents.verifier import VerifierAgent

agent = VerifierAgent()
result = agent.synthesize(attacker_result, defender_result, context)
```

## Confidence Rules

Per PHILOSOPHY.md:
- **CONFIRMED**: Requires test OR multi-agent consensus
- **LIKELY**: Strong evidence + analysis
- **UNCERTAIN**: Default bucket for missing context
- **REJECTED**: Defender rebuttal accepted

## Notes

- Uses opus for nuanced judgment
- Never synthesizes new analysis (only weighs existing)
- Always flags debate outcomes for human review
