## Decisions Made

**1. CLI-Based Bead Creation**
- Use `uv run alphaswarm beads generate` and `uv run alphaswarm findings create` instead of direct file writes
- Ensures proper bead ID generation, schema validation, and index updates
- Prevents corrupt artifacts from partial creation

**2. Evidence Chain Validation Requirements**
- Finding beads require: >= 1 code location, >= 2 reasoning steps, >= 1 VQL query
- Confidence level must be explicit with reasoning
- Graph-based detection mandatory (no VQL queries = rejected finding)
- Rationale: Ensures findings are verifiable and traceable

**3. Model Tier Assignments**
- Haiku 4.5 for VQL help (fast reference, < 500 tokens)
- Sonnet 4.5 for bead creation (balanced quality/cost)
- Opus 4.5 for cleanup/critical decisions only
- Rationale: Cost optimization while maintaining quality

**4. Bidirectional Context Linking**
- Finding beads link to context beads via `context_bead_id`
- Context beads updated with `finding_bead_ids` list
- Enables provenance tracking and context retrieval during verification
