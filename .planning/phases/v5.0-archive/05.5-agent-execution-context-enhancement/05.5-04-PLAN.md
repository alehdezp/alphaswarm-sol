# /vrs-create-bead-context-merge

**Purpose:** Create a bead from verified context-merge output.

**Model:** Sonnet 4.5

**When to invoke:**
- After context-merge agent completes merge
- After context-merge verifier passes validation
- Before spawning vuln-discovery agent

## Workflow

### Phase 1: Validate Inputs

1. Confirm merge result exists and succeeded:
   - `merge_result.success == True`
   - `merge_result.bundle` is populated

2. Confirm verification result exists and passed:
   - `verification_result.valid == True`
   - Note any `verification_result.warnings`

3. Extract required fields:
   - `vulnerability_class` from bundle
   - `protocol_name` from bundle
   - `target_scope` from bundle
   - `pool_id` if orchestrating in a pool

### Phase 2: Create Bead via CLI

Run the bead creation command:

```bash
uv run alphaswarm context create-bead \
  --vuln-class "{vulnerability_class}" \
  --protocol "{protocol_name}" \
  --scope "{target_scope_json}" \
  --bundle-file "{bundle_yaml_path}" \
  --verification-score "{score}" \
  --pool-id "{pool_id}" \
  --output-format yaml
```

This command:
1. Loads the verified context bundle
2. Creates ContextMergeBead with generated ID
3. Saves to `.vrs/beads/{pool_id}/{bead_id}.yaml`
4. Returns bead ID and path

### Phase 3: Verify Creation

1. Confirm bead file exists at returned path
2. Validate bead YAML structure
3. Report bead ID to orchestrator

## Input Requirements

| Field | Required | Source |
|-------|----------|--------|
| vulnerability_class | Yes | merge_result.bundle |
| protocol_name | Yes | merge_result.bundle |
| target_scope | Yes | merge_result.bundle |
| bundle | Yes | merge_result.bundle (serialized) |
| verification_score | Yes | verification_result.quality_score |
| pool_id | No | Orchestrator context |

## Output Format

```yaml
status: success
bead_id: CTX-a1b2c3d4e5f6
bead_path: .vrs/beads/audit-2026-01/CTX-a1b2c3d4e5f6.yaml
vulnerability_class: reentrancy/classic
protocol_name: MyProtocol
verification_score: 0.95
warnings: []
```

## Error Handling

| Error | Action |
|-------|--------|
| Merge failed | Report error, do NOT create bead |
| Verification failed | Report error, do NOT create bead |
| CLI error | Report error with stderr output |
| Filesystem error | Report error, retry once |

## Integration Points

**Before:** Context-merge agent + verifier complete
**After:** Orchestrator reads bead, spawns vuln-discovery agent

**Deletion protection:** Sonnet agents CANNOT delete beads without Opus confirmation.
```

2. Verify context bead exists and is valid

### Phase 3: Create Finding Bead via CLI

```bash

  --context-bead-id "{context_bead_id}" \
  --vuln-class "{vulnerability_class}" \
  --severity "{severity}" \
  --confidence "{confidence}" \
  --code-locations "{locations_json}" \
  --vulndoc-ref "{vulndoc_path}" \
  --reasoning "{reasoning_text}" \
  --vql-queries "{queries_json}" \
  --pool-id "{pool_id}" \
  --output-format yaml
```

### Phase 4: Update Context Bead

After finding bead created:
1. Add finding bead ID to context bead's `finding_bead_ids`
2. Save updated context bead

### Phase 5: Report to Orchestrator

- Bead ID
- Severity
- Confidence
- Summary

# /vrs-vql-help

**Purpose:** On-demand VQL syntax and query examples for agents.

**Model:** Haiku 4.5 (fast, cost-efficient)

**When to invoke:**
- Agent needs to formulate VQL query
- Agent unsure of VQL syntax
- Agent wants example queries for vulnerability class

## Query Syntax Reference

### Basic Structure

```
FIND {target} WHERE {conditions}
```

**Targets:** functions, contracts, state_variables

**Conditions:** property comparisons, operations, labels

### Property Operators

| Operator | Example |
|----------|---------|
| = | `visibility = public` |
| in | `visibility in [public, external]` |
| AND | `writes_state AND is_payable` |
| OR | `has_modifier OR has_require` |
| NOT | `NOT has_reentrancy_guard` |

### Semantic Operation Checks

```sql
-- Has specific operation
FIND functions WHERE has_operation(TRANSFERS_VALUE_OUT)

-- Has all operations
FIND functions WHERE has_all_operations([READS_BALANCE, WRITES_BALANCE])

-- Operation sequence (order matters)
FIND functions WHERE operation_before(TRANSFERS_VALUE_OUT, WRITES_BALANCE)
```

### Label Checks (Tier C)

```sql
-- Has label
FIND functions WHERE has_label(state_mutation.balance_update)

-- Missing label (negation)
FIND functions WHERE missing_label(access_control.reentrancy_guard)
```

## Example Queries by Vulnerability Class

### Reentrancy

```sql
FIND functions WHERE
  visibility in [public, external] AND
  has_all_operations([TRANSFERS_VALUE_OUT, WRITES_USER_BALANCE]) AND
  operation_before(TRANSFERS_VALUE_OUT, WRITES_USER_BALANCE) AND
  NOT has_reentrancy_guard
```

### Access Control

```sql
FIND functions WHERE
  visibility in [public, external] AND
  has_operation(MODIFIES_CRITICAL_STATE) AND
  NOT has_access_gate
```

### Oracle Manipulation

```sql
FIND functions WHERE
  has_operation(READS_ORACLE) AND
  has_operation(TRANSFERS_VALUE_OUT) AND
  NOT has_price_validation
```

### Flash Loan

```sql
FIND functions WHERE
  has_all_operations([READS_BALANCE, WRITES_BALANCE]) AND
  is_callback_target
```

## Response Format

3. Common pitfalls to avoid
