## Decisions Made

**1. Quality Loop Retry Strategy**
- Retry up to 3 times on quality errors with verification feedback
- Abort immediately on schema errors (missing vulndoc) without retries
- Rationale: Schema errors cannot be fixed by retrying, quality errors can be improved with feedback

**2. Async Orchestration Pattern**
- SubCoordinator uses async/await with asyncio.to_thread for parallel execution
- ContextMergeAgent.execute() is synchronous but called via asyncio.to_thread
- Rationale: Allows parallel execution of blocking operations without thread pool exhaustion

**3. Concurrency Control**
- Default max_parallel=10 to prevent resource exhaustion
- Controlled via asyncio.Semaphore for fine-grained task scheduling
- Rationale: Balance between parallelism and system resource limits

**4. Timeout Handling**
- Default timeout=300s per class to prevent runaway tasks
- Uses asyncio.wait_for to enforce timeout
- Rationale: Prevents a single slow merge from blocking entire batch

**5. Sub-Coordinator Responsibility**
- Ends after context beads created (does not spawn vuln-discovery)
- Rationale: Clear separation of concerns - SubCoordinator handles context prep, parent orchestrator handles discovery
