## Decisions Made

**Conservative risk defaults:**
- Unknown exposure assumes present=True
- Better false positives than missed vulnerabilities
- Per 05.5-CONTEXT.md: "Conservative default: Unknown exposure = assume present"

**Context structure:**
- System prompt = vulndoc reasoning_template (HOW to think)
- User context = protocol pack risk profile (WHAT to analyze)
- One vuln class per bundle (fresh context, focused reasoning)

**Token budget management:**
- Target: 2-4k tokens for efficiency
- Max: 6k tokens before trimming
- Trim order: drop graph_patterns first, then vql_queries, then truncate reasoning_template to 80%
- Never drop risk_profile or target_scope

**Risk category mapping:**
- `reentrancy/*` → access_risks, timing_risks
- `oracle/*` → oracle_risks, integration_risks
- `access-control/*` → access_risks, governance_risks
- `flash-loan/*` → liquidity_risks, timing_risks
- `upgrade/*` → upgrade_risks, access_risks
- Default: all categories with present=True

**Risk extraction logic:**
- Oracle risks: detected via offchain_inputs with input_type="oracle"
- Liquidity risks: detected via protocol_type (lending, dex, vault) and flash loan mentions
- Access risks: detected via privileged role capabilities (pause, upgrade, admin, owner, mint)
- Upgrade risks: detected via upgrade roles and proxy/implementation mentions
- Integration risks: detected via external_protocol/bridge offchain inputs
- Timing risks: detected via protocol_type (dex, auction, lending) and MEV-related terms
- Economic risks: detected via incentives and tokenomics_summary presence
- Governance risks: detected via governance section and governance-related roles
