## Decisions Made

**1. VQL execution errors are fatal**
- Rationale: If VQL queries fail, the entire discovery process is invalid
- Implementation: Removed inner try-catch around VQL execution, let errors propagate
- Impact: Context bead correctly marked as FAILED when VQL execution fails

**2. Confidence level mapping**
- confirmed → 0.95, likely → 0.80, uncertain → 0.60, rejected → 0.20
- Rationale: Explicit numeric thresholds for filtering and prioritization
- Maps to VulnerabilityBead confidence field (0.0-1.0)

**3. Evidence chain in bead metadata**
- Stored as `metadata["evidence_chain"]` dictionary
- Rationale: Preserves full evidence chain without extending VulnerabilityBead schema
- Enables complete traceability for findings

**4. Pool-based directory structure**
- Findings saved to `{beads_dir}/{pool_id}/findings/{bead_id}.json`
- Context beads saved to `{beads_dir}/{pool_id}/context/{bead_id}.json`
- Rationale: Organizes beads by pool for orchestration layer
