# Phase 5.8: Repository Cleanup & Hygiene - Context

**Gathered:** 2026-01-27 (updated from discussion)
**Status:** Ready for planning
**Depends on:** Phase 5.7 (VulnDocs Knowledge Consolidation)

<domain>
## Phase Boundary

Clean up `.true_vkg/` folder, scan repository for duplicated/outdated documentation, relocate needed content to proper locations, remove junk, and generate a hygiene report with remaining cleanup opportunities.

Scope exclusions:
- Active planning artifacts (ROADMAP.md, STATE.md, PROJECT.md, phase plans)
- Code quality issues (unused imports, dead code - different concern)

</domain>

<decisions>
## Implementation Decisions

### Cache & Artifact Organization
- **Delete regenerable caches** (crawl_cache, filtered_cache, llm_cache) - no relocation needed, just delete
- `.vrs/cache/` only for regenerable files that could be useful across projects (git-ignored)
- Valuable, non-regenerable content goes to appropriate locations (not cache)

### Discovery Folder Handling
- **Keep** `swc-to-vkg-mapping.yaml` → move to `vulndocs/.meta/references/swc-mapping.yaml` (curated reference linking SWC IDs to vulndocs categories)
- **Delete** tracking files (coverage-matrix.yaml, dvdefi-validation.yaml) - these are Phase 1 snapshots; Phase 7 will regenerate fresh metrics dynamically
- **Delete** execution artifacts (logs, progress reports, crawl sessions, raw crawl content) - knowledge was extracted to vulndocs

### Duplicate & Documentation Handling
- Focus on `.md` files and documentation
- `.planning/` is the source of truth (most recently updated)
- **Sync** outdated content: update `docs/` to match `.planning/` where content overlaps
- **Move** polished/finished content from `.planning/reference/` to `docs/`
- **Keep** only active planning artifacts in `.planning/`: ROADMAP.md, STATE.md, PROJECT.md, phase plans, config.json

### Script Preservation
- **Audit** `.true_vkg/scripts/` against existing `scripts/`
- **Merge** useful functionality not already in `scripts/alphaswarm_crawl.py`
- **Delete** superseded/duplicate scripts

### Hygiene Report Contents
- Remaining cleanup opportunities (files that could be cleaned but weren't touched)
- Documentation gaps (areas where docs are missing or incomplete)
- Technical debt items (things that work but could be improved)
- NOT code quality issues (unused imports, dead code)

### Claude's Discretion
- Cache directory structure under `.vrs/cache/`
- Hygiene report file location (likely `.planning/reports/` or `docs/maintenance/`)

</decisions>

<specifics>
## Specific Ideas

- "Make sure everything is not double-checked and regenerated" - avoid static tracking files that go stale; prefer dynamic generation
- Coverage metrics should be computed fresh via CLI (`alphaswarm vulndocs coverage`), not maintained as static YAML
- The goal is cleaner, more understandable codebase with single sources of truth

</specifics>

<deferred>
## Deferred Ideas

- **Automated docs sync skill** - Create skills/subagents that automatically update `docs/` from completed `.planning/` phases to keep documentation in sync. This is new functionality, not cleanup - belongs in a future phase.

</deferred>
