# 07.3 Validation Program v2 (Unified Engine + Suites + Ops)

## Purpose
Create the highest-integrity validation pipeline possible for AlphaSwarm.sol, grounded in **real claude-code-agent-teams-driven execution**, non-fakeable proof tokens, and external ground truth. This program is designed to maximize vulnerability detection quality while preventing regressions and false confidence.

## Core Philosophy (Non-Negotiable)
- **Real-world execution only**: all interactive tests must run via `claude-code-controller` with Claude Code as the operator.
  - **Full claude-code-controller documentation:** See `.planning/testing/rules/claude-code-controller-REFERENCE.md` and `.planning/testing/rules/canonical/claude-code-controller-instructions.md`
- **Evidence-first**: every claim must be backed by evidence packs, proof tokens, and transcripts.
- **No simulated tests**: any run without claude-code-agent-teams transcript + proof tokens is invalid.
- **External ground truth**: precision/recall are only computed against external sources.
- **No perfect metrics**: 100% precision/recall triggers investigation, not success.

## Program Map

## Unified Engine Contract
All suites must use the same execution contract:
- **Input**: scenario manifest + suite config
- **Process**: claude-code-agent-teams-driven `/vrs-audit` or `/vrs-full-testing` run
- **Output**: evidence pack + proof tokens + gate results
- **Artifacts**: `.vrs/validation/runs/<run_id>/...`

## Canonical Output Layout

```
.vrs/validation/runs/<run_id>/
  proofs/               # proof tokens per stage
  transcripts/          # claude-code-agent-teams captures + hashes
  metrics/              # precision/recall + gate outcomes
  manifest.json         # canonical manifest with hashes
  evidence_pack/         # stage artifacts + debug outputs
```

## Gate Model (G0–G7)
- **G0 Preflight**: claude-code-agent-teams available, tools installed, dataset hash, CLI version
- **G1 Evidence Integrity**: proof tokens + schema validation
- **G2 Graph Soundness**: graph hash + property/ops coverage
- **G3 Ground Truth Coverage**: per-category coverage thresholds
- **G4 Mutation Robustness**: mutation matrix pass rate
- **G5 Consensus & Variance**: multi-run stability, agent agreement
- **G6 Regression Baseline**: no drop vs locked baseline
- **G7 Continuous Health**: scheduled pipeline running and healthy

## Evidence Pack Requirements
Every run must include:
- claude-code-agent-teams transcript + hash
- proof tokens for each critical stage
- debug artifacts per skill/agent
- tool outputs or explicit skip reasons
- timing metadata and duration

## Anti-Fake Mechanisms
- **Nonce challenge** per stage; must appear in claude-code-agent-teams transcript.
- **Hash chain**: transcript hash → proof token → evidence pack manifest.
- **Cross-reference validation** between proof tokens and graph nodes.
- **Temporal validation** ensures correct stage ordering.

## Self-Improvement Loop
1. Run suite via claude-code-agent-teams + evidence pack.
2. Validate proof tokens + gates.
3. Classify failures (infra/tool/logic/evidence/ground-truth).
4. Apply fix and re-run (new worktree, new evidence pack).
5. Record delta + update baseline only after G6 pass.

## Failure Taxonomy (Required)
- **Infra Failure**: claude-code-agent-teams/tools missing
- **Evidence Failure**: missing proof token or transcript
- **Graph Failure**: hash mismatch, missing operations
- **Tool Failure**: external tool error or missing output
- **Logic Failure**: wrong detection vs ground truth
- **Variance Failure**: unstable multi-run results

## Additional Quality Amplifiers
- **Mutation Matrix**: generate adversarial variants to force reasoning depth.
- **Negative Controls**: safe contracts to enforce FP ceilings.
- **Needle-in-Haystack**: 1 bug in 50 safe cases.
- **Consensus Stress**: force attacker/defender disagreement tracking.

## Success Criteria (Program-Level)
- Every workflow and subworkflow can be executed via claude-code-agent-teams without manual hacks.
- Evidence packs and proof tokens validate with zero missing artifacts.
- External benchmarks show clear differentiation (unique finds, FP reduction).
- Continuous pipeline catches regressions before users do.
