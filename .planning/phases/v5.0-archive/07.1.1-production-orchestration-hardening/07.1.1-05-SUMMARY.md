# Phase 07.1.1 Plan 05: Worktree Isolation Summary

**One-liner:** Git worktree isolation for parallel agents with per-agent working directories, PropulsionEngine integration, and Codex CLI runtime support.

---

## What Was Built

### Task 1: WorktreeManager for Isolated Agent Runs
- Created `src/alphaswarm_sol/orchestration/worktree.py` with:
  - `WorktreeManager` class for git worktree lifecycle management
  - Allocate worktrees under `.vrs/worktrees/{pool_id}/{agent_id}`
  - Track metadata (pool_id, agent_id, base_ref, created_at) in JSON
  - `release()`, `cleanup_pool()`, and `cleanup_stale()` helpers
  - Fail-fast validation if git unavailable or not in repo
- Exported from `alphaswarm_sol.orchestration` module

### Task 2: Worktree Integration in Agent Spawning
- Extended `AgentConfig` with optional `workdir` field
- Extended `PropulsionConfig` with:
  - `enable_worktree_isolation: bool` (default: False)
  - `pool_id: Optional[str]` for worktree grouping
- `PropulsionEngine` now:
  - Initializes `WorktreeManager` when isolation enabled
  - Allocates per-agent worktree in `_get_agent_config()`
  - Releases worktree in `_on_task_done()` callback
  - Provides `cleanup_worktrees()` method for pool cleanup

### Task 3: Runtime Workdir Support
- Updated `CodexCLIRuntime`:
  - Added `_get_working_dir(agent_config)` method
  - `execute()` passes workdir to subprocess via `cwd` parameter
  - Config workdir overrides runtime default working directory
- Created comprehensive test suite in `tests/agents/test_worktree_isolation.py`:
  - 32 tests covering all worktree isolation scenarios
  - Tests for AgentConfig.workdir field
  - Tests for WorktreeManager allocate/release/cleanup
  - Tests for PropulsionEngine integration
  - Tests for Codex CLI runtime workdir support
  - Tests for parallel agent isolation
