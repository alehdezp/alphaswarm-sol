# Phase 07.1.1 Plan 03: Work Queue and Backpressure Summary

**One-liner:** Durable work queue with JSONL persistence, max_in_flight concurrency limits, and backpressure that pauses pools when queues are full.

## What Was Built

### 1. WorkQueue Module (`queue.py`)

A persistent work queue with concurrency control:

- **WorkItem**: Dataclass with pool_id, bead_id, action, payload_hash, enqueued_at
- **WorkItemStatus**: Enum (QUEUED, RESERVED, COMPLETED, FAILED, DEAD)
- **QueueLimits**: Configuration for max_in_flight, max_queue_depth, per_action_limits
- **QueueSnapshot**: Point-in-time state with counts and capacity indicators
- **BackpressureError**: Exception raised when queue is at capacity

API:
```python
queue = WorkQueue(Path('.vrs/pools/test-pool'), limits)
queue.enqueue(item)           # Add work, raises BackpressureError if full
queue.reserve(max_items=5)    # Get items up to limit
queue.ack(item_id)            # Mark complete
queue.nack(item_id, retry=True)  # Mark failed, optionally requeue
queue.snapshot()              # Get queue stats
```

### 2. ExecutionLoop Integration

Updated loop.py with queue-aware execution:

- **enqueue()**: Queue route decisions instead of immediate execution
- **_process_queue()**: Reserve and execute work items respecting limits
- **_apply_backpressure()**: Pause pool when queue full, record reason in metadata
- **get_queue_snapshot()**: Expose queue state for monitoring

Configuration:
```python
config = LoopConfig(
    use_queue=True,
    queue_limits=QueueLimits(max_in_flight=10, max_queue_depth=100),
)
loop = ExecutionLoop(manager, config=config)
```

### 3. RouteDecision Enhancement

Added deterministic payload_hash property for queue deduplication:

```python
decision = RouteDecision(
    action=RouteAction.SPAWN_ATTACKERS,
    target_beads=["b1", "b2"],
)
print(decision.payload_hash)  # SHA256[:16] of action + sorted beads
```

## Key Design Decisions

1. **JSONL Storage**: Append-only log under `.vrs/pools/{pool_id}/queue.jsonl` for durability without database dependencies.

2. **Reserve/Ack/Nack Protocol**: Items are reserved (leased) for processing, then acknowledged or negatively acknowledged. Stale reservations (>10 min) are automatically re-reserved.

3. **Per-Action Limits**: Fine-grained concurrency control per action type (e.g., limit attackers separately from defenders).

4. **Backpressure via Pool Status**: When queue is full, pool is paused with `metadata["backpressure"] = True` and `metadata["pause_reason"] = "backpressure"`.
