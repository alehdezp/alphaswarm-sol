## Decisions Made

1. **Non-RETRY actions execute immediately** - Actions like SKIP, PAUSE, ABORT, QUARANTINE don't use retry logic since they're immediate responses, not retry-based recovery
2. **STATE_CORRUPTION is always CRITICAL** - KeyError, AttributeError, IndexError indicate corrupted state requiring immediate abort
3. **Failure history capped at 20 entries** - Prevents unbounded metadata growth while maintaining audit trail
4. **Per-pool recovery attempts** - Tracking attempts per pool (not globally) allows proper retry bounds
