## Decisions Made

- **Event ordering:** Sort by (timestamp, event_id) for deterministic replay regardless of file append order
- **Fallback behavior:** If no events exist, fall back to loading from pool snapshot for backward compatibility
- **Strict validation:** Optional mode that compares replayed state against current snapshot and reports mismatches
- **Seed parameter:** Explicit random seed for any nondeterministic operations (currently sets random.seed)
- **Event store location:** Pool events stored at `.vrs/pools/{pool_id}/events/pool.jsonl` alongside bead events
