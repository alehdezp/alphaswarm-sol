# Phase 7.1.1: Production Orchestration Hardening - Context

## Purpose
Move the multi-agent orchestration layer from "works in tests" to "safe and reliable in production" by adding durability, idempotency, and concurrency control. This phase is the foundation for predictable, repeatable agent runs at scale.

## Scope
- Event-sourced bead state (append-only, replayable history)
- Idempotency keys for all tool/agent actions
- Queueing and backpressure to prevent run explosions
- Deterministic replay for audit and regression
- Worktree isolation policy for parallel agents
- Failure taxonomy with automated recovery playbooks

## Out of Scope
- New vulnerability detection logic
- Changes to VulnDocs content
- Major SDK integrations (handled in 7.1.4)

## Key Constraints
- Preserve graph-first reasoning (BSKG queries remain mandatory)
- Keep bead artifacts deterministic for replay
- No modifications to builder_legacy.py or executor.py

## Desired Outcomes
- Every bead run can be reconstructed from log only
- No duplicated side effects under retries
- Parallelism is safe and observable
- Recovery is automated and testable

## Primary Risks
- Hidden state in external tools or models
- Non-idempotent operations in tools
- Cross-agent file conflicts

## Validation
- Deterministic replays match original outputs
- Chaos tests do not corrupt bead state
- Failure modes are categorized and recoverable
