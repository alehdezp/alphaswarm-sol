# Phase 5.9 Plan 08: v2-Compliant Pattern Outputs Summary

**One-liner:** Pattern results packaging with clause matrix, evidence links, unknowns budget gating, and v2 contract validation at packaging boundary.

## What Was Built

### 1. v2 Result Data Structures (results_v2.py)

Created comprehensive v2-compliant result packaging structures:

- **ResultsV2**: Top-level output structure with interface_version, build_hash, timestamp, query, summary, findings, and omissions
- **FindingV2**: v2 finding with clause_matrix, evidence_refs, evidence_missing, omissions, and insufficient_evidence flag
- **ClauseMatrix**: Truth table for all pattern clauses with matched/failed/unknown tracking
- **ClauseResult**: Individual clause evaluation result with evidence and omission links
- **EvidenceMissing**: Record of missing evidence with reason codes

### 2. PatternResultPackager Class

Transforms raw pattern engine findings to v2 format:

- Builds clause_matrix from pattern engine explain data
- Generates deterministic evidence IDs using interface_contract utilities
- Applies unknowns budget gating (excess unknowns -> insufficient_evidence flag)
- Validates output at packaging boundary (fail fast in strict mode)
- Calculates confidence scores based on clause evaluation status

### 3. Clause Matrix Generation

Full truth-table style output for each finding:

```python
clause_matrix = ClauseMatrix()
clause_matrix.add("all:visibility.in", ClauseStatus.MATCHED, evidence=[...])
clause_matrix.add("ops_all:has_all_operations:...", ClauseStatus.MATCHED, evidence=[...])
clause_matrix.add("none:has_reentrancy_guard.eq", ClauseStatus.MATCHED)
```

Each clause includes:
- Clause identifier (group:property.op format)
- Status (matched/failed/unknown)
- Evidence references (for matched clauses)
- Omission references (for unknown clauses)

### 4. Unknowns Budget Gating

Leverages UnknownsBudgetGate from interface_contract:

```python
budget = UnknownsBudget(max_ratio=0.3, max_absolute=2)
packager = PatternResultPackager(unknowns_budget=budget)
# Findings with >30% unknowns or >2 unknown clauses marked insufficient_evidence=True
```

### 5. Module Exports

Added to queries/__init__.py:
- ResultsV2, FindingV2, ClauseMatrix, ClauseResult
- PatternResultPackager
- package_pattern_results (convenience function)

## Key Implementation Details

### Clause ID Generation

Clauses are identified by their condition group and specifics:

| Condition Type | Example Clause ID |
|----------------|-------------------|
| Property (all) | `all:visibility.in` |
| Property (any) | `any:is_payable.eq` |
| Property (none) | `none:has_reentrancy_guard.eq` |
| Operation (all) | `ops_all:has_all_operations:TRANSFERS_VALUE_OUT,WRITES_USER_BALANCE` |
| Sequence order | `ops_all:sequence_order:TRANSFERS_VALUE_OUT->WRITES_USER_BALANCE` |
| Edge | `edge:CALLS` |
| Path | `path:INHERITS_FROM` |
| Tier B | `tier_b:risk_tags:high` |

### Confidence Calculation

```python
confidence = (matched / total) - (unknown / total) * 0.5
```

- Base confidence from matched clause ratio
- Penalty of 50% of unknown ratio
- Result clamped to [0.0, 1.0]

### Evidence Collection

Evidence refs are collected from:
1. Clause-level evidence (in clause_matrix)
2. Finding-level aggregation (deduplicated by snippet_id)

For legacy patterns without evidence, `evidence_missing` is populated with `LEGACY_NO_EVIDENCE` reason.
