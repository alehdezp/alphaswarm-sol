### 2. Expanded taint.py (725 lines)

New taint analysis types and analyzer:

**TaintSource enum:**
- `CALL_TARGET_CONTROL` (CRITICAL) - User controls call destination
- `EXTERNAL_RETURN` (HIGH) - Return from external call
- `USER_INPUT` (HIGH) - Function parameters
- `ORACLE` (HIGH) - Price feeds
- `ENVIRONMENT` (MEDIUM) - msg.sender, msg.value
- `STORAGE_ALIASED` (LOW) - Tainted storage read

**TaintSink enum:**
- `CALL_TARGET` (CRITICAL)
- `EXTERNAL_CALL_VALUE` (CRITICAL)
- `STORAGE_WRITE` (HIGH)
- `ARITHMETIC` (MEDIUM)
- `COMPARISON` (LOW)

**TaintSanitizer enum:**
- `BOUNDS_CHECK` (affects ARITHMETIC)
- `WHITELIST_CHECK` (affects CALL_TARGET)
- `SAFE_MATH` (affects ARITHMETIC)
- `TYPE_CAST` (affects ARITHMETIC)
- `OWNERSHIP_CHECK` (context only)
- `ZERO_CHECK` (weak)

**TaintAvailability:**
- `available`: Is analysis available?
- `confidence`: 0.0-1.0 (calibrated)
- `reason`: Explanation
- Factory methods: `full()`, `aliased()`, `delegatecall()`, etc.

**TaintResult:**
- `is_tainted`: Direct result
- `is_unknown`: True if unavailable (NOT safe!)
- `is_safe`: True only if available AND not tainted
- `risk_level`: Highest source risk

**TaintAnalyzer:**
- Analyzes function/variable taint
- Detects delegatecall/assembly blockers
- Tracks storage taint with aliasing

## Decisions Made

### D-0906-01: Direct-then-Aliased Taint Propagation

**Decision:** Use two-tier taint propagation:
1. **Direct taint** (Tier 1): Immediate uses, confidence 1.0
2. **Aliased taint** (Tier 2): Through storage, confidence 0.7

**Rationale:** Direct taint is precise. Storage aliasing introduces uncertainty, so confidence is reduced to signal this to LLM agents.

### D-0906-02: Availability Confidence Calibration

**Decision:** Fixed confidence values for known conditions:

| Condition | available | confidence |
|-----------|-----------|------------|
| Full CFG, direct | true | 1.0 |
| Storage aliased | true | 0.7 |
| External call | true | 0.5 |
| Delegatecall | false | 0.0 |
| Inline assembly | false | 0.0 |

**Rationale:** Calibrated values allow LLM agents to make confidence-weighted decisions.

### D-0906-03: Missing Taint Yields Unknown, Not Safe

**Decision:** `TaintResult.is_unknown` indicates unavailable analysis. `is_safe` requires both `available=True` AND `is_tainted=False`.

**Rationale:** False safety claims are worse than unknown results. LLM agents must check availability before making safety claims.
