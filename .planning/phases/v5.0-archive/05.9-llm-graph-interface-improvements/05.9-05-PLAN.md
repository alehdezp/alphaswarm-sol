<research_references>
**Algorithm:** Use Cooper-Harvey-Kennedy iterative dominance algorithm (simpler and faster than Lengauer-Tarjan for typical CFG sizes <30k nodes). Reference: "A Simple, Fast Dominance Algorithm" - Cooper et al., 2001.

**NetworkX:** Consider using networkx.algorithms.dominance if available, or implement iterative algorithm directly on Slither CFG nodes.

**Critical Pitfall (PITFALLS.md):** "CFG Order != Dominance Order" - Current `compute_ordering_pairs()` uses CFG traversal order which produces "(A, B) in pairs" meaning "A appears before B in CFG." This does NOT mean "A always executes before B." Conditional branches create paths where the "earlier" operation may never execute.
</research_references>

  <files>src/alphaswarm_sol/kg/dominance.py, docs/reference/dominance.md</files>
  <action>
Create `dominance.py` module implementing:

1. **DominanceAnalyzer class:**
   - Constructor takes Slither CFG (from function.nodes)
   - `compute_dominators()` using Cooper-Harvey-Kennedy iterative algorithm:
     ```
     dom[entry] = {entry}
     for all other nodes n: dom[n] = all_nodes
     while changes:
       for each node n (except entry):
         dom[n] = {n} ∪ (∩ dom[p] for p in predecessors(n))
     ```
   - `compute_post_dominators()` using reverse CFG
   - `dominates(a, b) -> bool`: Does node a dominate node b?
   - `post_dominates(a, b) -> bool`: Does node a post-dominate node b?

2. **OrderingRelation enum:**
   ```python
   class OrderingRelation(Enum):
       ALWAYS_BEFORE = "always_before"      # A dominates B
       SOMETIMES_BEFORE = "sometimes_before" # A precedes B on some path
       NEVER_BEFORE = "never_before"        # A is not before B on any path
       UNKNOWN = "unknown"                  # Cannot determine (truncated CFG, etc.)
   ```

3. **PathQualifiedOrdering result:**
   ```python
   @dataclass
   class PathQualifiedOrdering:
       relation: OrderingRelation
       confidence: float  # 0.0-1.0
       reason: Optional[str] = None  # Why unknown, if applicable
   ```

   - Entry/exit points cannot be determined -> emit unknown

   - Loop with unresolvable iteration count -> emit unknown for loop-carried dependencies

Create `docs/reference/dominance.md` documenting:
