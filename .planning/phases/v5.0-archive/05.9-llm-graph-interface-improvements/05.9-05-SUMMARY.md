## What Was Built

### Task 1: Dominance Tree Computation

Created `src/alphaswarm_sol/kg/dominance.py` (1046 lines) implementing:

**DominanceAnalyzer class:**
- `compute_dominators()` - Iterative Cooper-Harvey-Kennedy algorithm
- `compute_post_dominators()` - Reverse CFG for post-dominance
- `dominates(a, b)` / `post_dominates(a, b)` - Query methods
- `immediate_dominator()` / `immediate_post_dominator()` - Idom queries
- `compute_ordering(a, b)` - Full path-qualified ordering

**OrderingRelation enum:**
```python
ALWAYS_BEFORE    # A dominates B - A on all paths to B
SOMETIMES_BEFORE # A precedes B on some paths
NEVER_BEFORE     # A not before B on any path
UNKNOWN          # Cannot determine (external, unreachable, etc.)
```

**PathQualifiedOrdering result type:**
- `relation: OrderingRelation`
- `confidence: float` (0.0-1.0)
- `reason: Optional[str]`
- `evidence: Optional[Dict]`

**CFGNodeInfo abstraction** - Decouples from Slither CFG

**Documentation (418 lines):**
- Algorithm overview
- Unknown emission criteria
- Guard dominance semantics
- Multi-modifier chain handling
- Examples with vulnerable/safe patterns

## Critical Regression Test

The `TestCFGOrderVsDominanceOrderRegression` class tests the PITFALLS.md bug:

```python
def test_conditional_write_before_call(self):
    """CRITICAL: conditional write does NOT always precede call."""
    # CFG: if(cond) { write } call
    # CFG order says: write before call -- WRONG
    # Dominance says: SOMETIMES_BEFORE (not ALWAYS)
```

This catches the naive CFG ordering bug that was causing false negatives.
