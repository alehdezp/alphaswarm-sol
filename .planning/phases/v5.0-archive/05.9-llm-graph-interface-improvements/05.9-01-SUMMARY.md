# Phase 05.9 Plan 01: Graph Interface Contract v2 Spec and Validator

## One-Liner

ABI-style Graph Interface Contract v2 with semver, JSON Schema validation, deterministic evidence IDs, and v1-to-v2 compatibility shim.

## What Was Built

### 1. Graph Interface Contract v2 Specification

Created `docs/reference/graph-interface-v2.md` (615 lines) defining:

- **ABI versioning rules**: Major version for breaking changes, minor for additive, patch for fixes
- **Core schema**: Root structure, Finding, EvidenceRef, OmissionLedger
- **Clause matrix rules**: Status definitions, invariants, clause ID format
- **Evidence determinism**: SHA256-based evidence IDs tied to build hash
- **Coverage score formula**: Weighted node capture ratio with thresholds
- **Unknowns budget gating**: Ratio/absolute limits and critical clause protection
- **Debug slice mode**: Bypasses pruning for diagnostic purposes
- **Validation rules**: Required fields, type validation, semantic validation
- **Reason codes**: Evidence missing reasons and omission reasons
- **Migration guide**: v1 to v2 breaking changes and shim behavior

### 2. JSON Schema

Created `schemas/graph_interface_v2.json` (443 lines) encoding:

- All required/optional fields with types
- Enum constraints for status, severity, reason codes
- Pattern validation for build_hash (12 hex chars), evidence ID (EVD-xxxxxxxx)
- Coverage score bounds (0.0-1.0)
- Nested definitions for Finding, EvidenceRef, OmissionLedger, etc.
- anyOf constraint: findings must have evidence_refs OR evidence_missing

### 3. Validator Module

Created `src/alphaswarm_sol/llm/interface_contract.py` (955 lines) implementing:

**Classes:**
- `GraphInterfaceValidator`: Loads schema, validates outputs, semantic checks
- `UnknownsBudgetGate`: Checks unknowns ratio/absolute/critical clause limits
- `V1ToV2CompatibilityShim`: Transforms v1 outputs with explicit defaults

**Dataclasses:**
- `UnknownsBudget`: Budget configuration
- `EvidenceRef`, `ClauseMatrixEntry`, `CutSetEntry`, `OmissionLedger`: Output building blocks

**Enums:**
- `SliceMode`, `ClauseStatus`, `Severity`, `EvidenceMissingReason`, `OmissionReason`

**Functions:**
- `generate_evidence_id()`: Deterministic SHA256-based ID generation
- `generate_build_hash()`: Build hash from source content
- `validate_output()`: Schema + semantic validation
- `validate_and_serialize()`: Fail-fast serialization gate
- `transform_v1_to_v2()`: Compatibility shim entry point

## Key Schema Elements

```yaml
# Root structure
interface_version: "2.0.0"
build_hash: "<sha256-12>"
timestamp: "<ISO8601>"
query: { kind, id, source }
summary: { nodes, edges, findings, coverage_score, omissions_present, unknowns_count }
findings: [ <Finding> ]
omissions: { coverage_score, cut_set, excluded_edges, slice_mode }

# Finding structure
id, pattern_id, severity, confidence
matched_clauses, failed_clauses, unknown_clauses
clause_matrix: [ { clause, status, evidence_refs, omission_refs } ]
evidence_refs: [ { file, line, node_id, snippet_id, build_hash } ]
evidence_missing: [ { reason, clause, details } ]
omissions: { coverage_score, cut_set, excluded_edges, slice_mode }
```

## Validation Pipeline

1. **Schema validation** via jsonschema Draft 2020-12
2. **Semantic validation**:
   - Version compatibility check
   - Build hash consistency across evidence refs
   - Clause list completeness (matched + failed + unknown = all clauses)
   - Clause matrix alignment with lists
   - Evidence linkage (matched clauses must have evidence)
   - Omission linkage (unknown clauses must have reasons)
   - Coverage score bounds (0.0-1.0)
3. **Unknowns budget gate** (optional):
   - Ratio limit (default 30%)
   - Absolute limit (default 2)
   - Critical clause protection

## Decisions Made

| ID | Decision | Rationale |
|----|----------|-----------|
| IF-01 | ABI versioning with semver | Enables migration paths for breaking changes |
| IF-02 | Deterministic evidence IDs | Reproducible references across sessions |
| IF-03 | Dual unknowns limits (ratio + absolute) | Prevents both sparse and dense unknown findings |
| IF-04 | Deprecation warnings in shim | Encourages native v2 generation |
| IF-05 | Required clause matrix | Truth table enables per-clause evidence linking |
