# Phase 05.9 Plan 02: Ops Taxonomy Registry Summary

Canonical ops taxonomy registry with deprecation system, legacy aliases, and SARIF-normalized name coverage.

## What Was Built

### 1. Ops Taxonomy Registry (`src/alphaswarm_sol/kg/taxonomy.py`)

A single source of truth for semantic operations, edge types, and their mappings:

- **20 canonical operations** with full metadata (category, short code, risk, pattern tags)
- **25 canonical edge types** with risk scores and operation mappings
- **Deprecation system** with versioning (deprecated_in, sunset_in, replacement, migration_rule)
- **Legacy alias resolution** with deterministic mapping to canonical names
- **SARIF alias resolution** for tool output normalization

Key classes:
- `OpDefinition`: Operation definition with aliases, SARIF names, pattern tags
- `EdgeDefinition`: Edge type definition with aliases and risk scores
- `DeprecationInfo`: Deprecation metadata with sunset policy
- `OpsTaxonomyRegistry`: Central registry with resolution methods

### 2. Operations Integration (`src/alphaswarm_sol/kg/operations.py`)

Registry-backed resolution functions:

- `resolve_operation_name()`: Resolves any format (canonical, alias, short code, SARIF)
- `get_operation_from_name()`: Returns SemanticOperation enum
- `validate_operation_names()`: Validates lists, returns (valid, invalid)
- `get_operation_pattern_tags()`: Returns pattern tags for an operation
- `get_operation_risk_base()`: Returns base risk score

### 3. Comprehensive Test Suite (`tests/test_ops_taxonomy.py`)

54 tests covering:
- Taxonomy version and structure
- All 20 operations match SemanticOperation enum
- Short codes match OP_CODES mapping
- Deprecated aliases have replacements and migration rules
- Legacy aliases resolve deterministically
- SARIF-normalized names resolve correctly
- Pattern validation detects invalid ops
- Edge resolution and validation
- Serialization and category filtering

### 4. Documentation Update (`docs/reference/operations.md`)

- Added taxonomy version and registry overview
- Documented all 20 semantic operations with short codes
- Listed deprecated aliases with migration guidance
- Added SARIF integration section
- Added pattern tags reference table
