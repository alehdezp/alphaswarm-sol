<research_references>
**Critical Pitfall (PITFALLS.md):** "External-Return Taint Aliasing Ambiguity" - When `uint price = oracle.getPrice()`, does `price` taint storage writes? What if `price` is stored, then read later? Aliasing rules become complex. Prevention: Start with direct taint, add storage aliasing rules explicitly, document in TAINT_RULES.md.

**Design Decision:** The aliasing strategy must be explicitly documented as a DESIGN CHOICE, not implicit behavior. Two strategies:
1. **Direct taint:** Return value taints immediate uses only
2. **Aliased taint:** Return value propagates through storage/memory assignments

This plan documents and implements the chosen strategy with availability flags for ambiguous cases.
</research_references>

  <action>
Create canonical taint rules document covering:

1. **Taint Sources (ranked by risk):**
   ```markdown
   ## Taint Sources

   | Source Type | Risk Level | Example | Notes |
   |-------------|------------|---------|-------|
   | External return | HIGH | `oracle.getPrice()` | Untrusted external data |
   | Call-target control | CRITICAL | `user_address.call()` | User controls call target |
   | User input | HIGH | `function f(uint x)` | Direct parameter taint |
   | Environment | MEDIUM | `msg.sender`, `msg.value` | Context-dependent |
   | Oracle | HIGH | `chainlink.latestAnswer()` | External price feeds |
   | Storage read | LOW | `balances[user]` | Only if storage was tainted |
   ```

2. **Aliasing Strategy (EXPLICIT DESIGN CHOICE):**
   ```markdown
   ## Aliasing Strategy

   This implementation uses **Direct-then-Aliased** taint propagation:

   ### Direct Taint (Default)
   - External return values taint IMMEDIATE uses only
   - `uint price = oracle.getPrice(); use(price);` -> `price` is tainted at `use()`
   - Does NOT automatically propagate through storage

   ### Aliased Taint (Explicit Tracking)
   - When tainted value is written to storage: `storage[key] = tainted_value`
   - Storage slot is marked as tainted
   - Subsequent reads from that slot propagate taint
   - Aliased taint has LOWER confidence (0.7 vs 1.0)

   ### Delegatecall Propagation
   - Delegatecall to external contract: ALL storage potentially tainted
   - Availability = false (cannot track external contract's behavior)
   - Pattern: Mark as UNKNOWN, not SAFE
   ```

3. **Sink Definitions:**
   ```markdown
   ## Taint Sinks

   | Sink Type | Severity | Pattern |
   |-----------|----------|---------|
   | External call value | CRITICAL | `.call{value: TAINTED}()` |
   | Call target | CRITICAL | `TAINTED_ADDR.call()` |
   | Storage write | HIGH | `balances[x] = TAINTED` |
   | Arithmetic | MEDIUM | `a + TAINTED` (overflow risk) |
   | Comparison | LOW | `if (TAINTED > x)` |
   ```

4. **Sanitizers:**
   ```markdown
   ## Sanitizers

   | Sanitizer | Effect | Example |
   |-----------|--------|---------|
   | Bounds check | Removes overflow taint | `require(x < MAX)` |
   | Ownership check | Context-specific | `require(msg.sender == owner)` |
   | SafeMath | Arithmetic sanitizer | `x.add(y)` |
   | Explicit cast | Type narrowing | `uint8(x)` |
   ```

5. **Availability Flag Calibration:**
   ```markdown
   ## Availability Flags

   | Condition | availability | confidence | Reason |
   |-----------|--------------|------------|--------|
   | Direct taint, full CFG | true | 1.0 | Complete analysis |
   | Aliased taint, tracked storage | true | 0.7 | Storage aliasing uncertainty |
   | External call in path | true | 0.5 | External effects unknown |
   | Delegatecall present | false | 0.0 | Cannot track external storage |
   | Inline assembly | false | 0.0 | Opaque to analysis |
   | Loop with dynamic bound | true | 0.6 | Iteration-dependent taint |
   ```

   ```python
   class TaintSource(Enum):
       USER_INPUT = "user_input"           # Function parameters
       ENVIRONMENT = "environment"          # msg.sender, msg.value, block.*
       EXTERNAL_RETURN = "external_return"  # Return from external call
       CALL_TARGET_CONTROL = "call_target_control"  # User controls call address
       ORACLE = "oracle"                    # Price feed, external data
       STORAGE_ALIASED = "storage_aliased"  # Tainted storage read
   ```

2. **TaintResult with availability:**
   ```python
   @dataclass
   class TaintAvailability:
       available: bool          # Is taint analysis available?
       confidence: float        # 0.0-1.0
       reason: Optional[str]    # Why unavailable or low confidence

   @dataclass
   class TaintResult:
       is_tainted: bool
       sources: list[TaintSource]
       path: list[str]          # Taint propagation path
       availability: TaintAvailability
       sanitizers_applied: list[str]

       @property
       def is_unknown(self) -> bool:
           """True if taint analysis is unavailable."""
           return not self.availability.available
   ```

3. **Sink and sanitizer tracking:**
   ```python
   class TaintSink(Enum):
       EXTERNAL_CALL_VALUE = "external_call_value"
       CALL_TARGET = "call_target"
       STORAGE_WRITE = "storage_write"
       ARITHMETIC = "arithmetic"
       COMPARISON = "comparison"

   class TaintSanitizer(Enum):
       BOUNDS_CHECK = "bounds_check"
       OWNERSHIP_CHECK = "ownership_check"
       SAFE_MATH = "safe_math"
       TYPE_CAST = "type_cast"
   ```

   - Implement aliased taint for storage writes/reads
   - Track delegatecall as availability=false
   - Apply sanitizers at check points
