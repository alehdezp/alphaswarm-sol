# Phase 5.9 Plan 07: Unified Slicing Pipeline Summary

Unified slicing pipeline with 5 stages (PPR, extraction, property slicing, policy filtering, omission injection) and debug mode that bypasses pruning with annotated omissions.

## Objective Achievement

Unified all slicing paths into one deterministic pipeline so all agents receive consistent, omission-aware context.

## What Was Built

### Task 1: Unified Slicing Pipeline (slicer.py)

**New classes and functions:**

1. **PipelineConfig** - Configuration dataclass:
   - `slice_mode`: "standard" or "debug"
   - `context_mode`: "strict", "standard", or "relaxed" for PPR
   - `category`: Vulnerability category for property filtering
   - `max_nodes`: Maximum nodes in subgraph
   - `role_budget`: Optional role-specific token budget
   - Factory methods: `for_role(role)`, `debug(**kwargs)`

2. **PipelineResult** - Result dataclass:
   - `graph`: SlicedGraph output
   - `omissions`: OmissionLedger with coverage metadata
   - `stats`: Pipeline execution statistics
   - `config`: Configuration used

3. **UnifiedSlicingPipeline** - Main pipeline class:
   - 5-stage processing: PPR seed selection, subgraph extraction, property slicing, policy filtering, omission injection
   - `slice(focal_nodes, config)`: Main entry point
   - `slice_for_role(focal_nodes, role, category)`: Role-specific slicing
   - `slice_debug(focal_nodes, category)`: Debug mode convenience

4. **Convenience functions:**
   - `slice_graph_unified()`: Primary entry point for all LLM-facing slicing
   - `slice_graph_for_agent()`: Role-optimized slicing

### Task 2: Debug Slice Mode + Pipeline Tests (context_policy.py, tests)

**New context_policy.py additions:**

1. **SliceMode** enum - STANDARD/DEBUG values
2. **SlicePolicyConfig** - Debug mode configuration
3. **SlicedContextResult** - Debug-aware result with omissions metadata
4. **SliceAwareContextPolicy** - Policy with debug mode support
5. **get_slice_aware_policy()** - Factory function

**New test file (test_slicing_pipeline.py):**

- 27 tests covering:
  - PipelineConfig: defaults, role presets, debug config
  - UnifiedSlicingPipeline: standard/debug mode, stages, stats
  - Slicing equivalence: router/LLM/CLI consistency
  - Debug mode behavior: bypasses max_nodes, annotates slice_mode
  - SliceAwareContextPolicy: debug mode integration
  - Omissions metadata: slice_mode in serialization

## Decisions Made

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Pipeline architecture | 5-stage unified pipeline | Ensures consistency across all surfaces |
| Debug mode behavior | Bypasses pruning, annotates omissions | Enables diagnosis without changing flow |
| Role budgets | Config presets per agent type | Attacker needs more context than classifier |
| Slice mode tracking | In OmissionLedger.slice_mode | Single source of truth for mode |
