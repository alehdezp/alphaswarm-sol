## Decisions Made

1. **18 lint rules organized in 7 categories:**
   - Required fields (PCP001-PCP004): errors for missing core fields
   - Determinism (PCP010-PCP012): errors for RAG/name heuristics violations
   - Evidence (PCP020-PCP024): warnings for missing witnesses/anti-signals
   - Ordering (PCP030-PCP031): warnings for sequence patterns without ordering variants
   - Unknowns (PCP040-PCP041): warnings for non-explicit unknowns policy
   - Guard taxonomy (PCP050-PCP051): info for taxonomy consistency
   - Evidence requirements (PCP070-PCP071): info for empty node/edge types

2. **Severity level approach:**
   - High/critical patterns get warnings (not errors) for missing witnesses/anti-signals
   - This avoids blocking adoption while encouraging best practices
   - Strict mode can promote warnings to errors for CI enforcement

3. **PCP file convention:** `*.pcp.yaml` extension distinguishes PCP files from pattern files in same directory
