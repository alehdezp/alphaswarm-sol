### Task 2: Phase 5.10 Quality Benchmark

Created `scripts/benchmarks/phase_510_quality.py` (~600 LOC):

- **DeterminismResult/ShuffledOrderResult/SlicingMetrics** dataclasses for structured results
- **QualityReport** aggregating determinism checks, shuffled-order checks, slicing metrics
- **Phase510QualityBenchmark** class:
  - `run()` executes full benchmark with configurable determinism runs and shuffle seed
  - `_run_determinism_checks()` validates PCP, evidence ID, manifest serialization
  - `_run_shuffled_order_checks()` validates batch, manifest, fork-rank shuffle stability
  - `_run_slicing_benchmark()` reuses `kg/slicing_benchmark` for token reduction metrics
  - `_compute_precision_recall_deltas()` and `_compute_novelty_yield()` for quality metrics
  - `_compute_evidence_entropy()` for evidence diversity measurement
- **CLI interface** with `--output`, `--check-determinism`, `--ci`, `--verbose` options
- **`phase_510_quality()` exported function** for programmatic invocation

## Key Assertions

1. **Bit-identical output:** Same graph + PCP version produces identical JSON output
2. **Evidence ID stability:** `EVD-xxxxxxxx` format stable across rebuilds
3. **Unknown handling:** UNKNOWN status never upgrades without evidence
4. **Omission surfacing:** Pruned graphs always have non-empty omission ledgers
5. **Shuffled-input invariance:** Random order input produces identical sorted output
6. **Cache versioning:** Different graph hashes produce different cache keys
7. **Fork-then-rank:** Highest confidence wins, contradictions veto
