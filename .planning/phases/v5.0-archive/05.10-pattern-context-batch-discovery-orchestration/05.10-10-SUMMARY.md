## Decisions Made
- **Delta ID from content hash**: Delta IDs are computed from type+target+evidence+payload, making them deterministic regardless of creation order or batch source
- **Ordering key priority**: FINDING_ADD < EVIDENCE_ADD < CONFIDENCE_CHANGE < FINDING_UPDATE < VERDICT_ADD ensures consistent merge order
- **Conflict detection criteria**: Same target with disjoint evidence, confidence gap > 0.3, or conflicting verdicts triggers quarantine
- **Full delta preservation**: Both conflicting deltas preserved in quarantine for audit trail and resolver agent review
