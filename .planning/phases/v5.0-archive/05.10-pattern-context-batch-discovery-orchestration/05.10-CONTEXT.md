# Phase 5.10 Context: Pattern Context + Batch Discovery Orchestration

## Problem Statement
Pattern discovery accuracy and creativity are capped by thin context packs, weak evidence gating, and brittle slicing. Sequential execution wastes cost. Findings can conflict under parallel execution due to non-deterministic merges. Missing context is incorrectly interpreted as safety. Tier B/C output lacks explicit unknown handling and guard-aware counter-signals.

## Non-Negotiables
- **No RAG or semantic search**. Context must be deterministic: PCP + graph slice + protocol pack only.
- **Graph-first** analysis; no name heuristics.
- **Evidence-first**: every claim must include evidence refs or be marked unknown.
- **Append-only merges** with single-writer deterministic ordering.
- **Unknown != false**. Missing signals remain unknown and can trigger controlled expansion.

## Core Concepts (Updated)
- **PCP v2**: Deterministic Pattern Context Pack with anti-signals, witnesses, evidence weights, guard taxonomy, ordering variants, and economic placeholders.
- **Witnesses**: Minimal proof subgraph (required) + negative witness (must-not-exist).
- **Pattern-scoped slices**: Edge-closure of required ops with explicit omission lists.
- **Escalation ladder**: Cheap pass → expand slice → verify pass; budget enforced before any expensive call.
- **Batch orchestration**: Shared cacheable prefix, fork/gather scouts, verifier ranking, and append-only deltas.
- **Creative loop (Tier B only)**: Near-miss mining, mutation, counterfactual probes, anomaly motifs.
- **Skill contracts**: Skills must document when to run, when to reuse cached PCPs, and when protocol context is required vs optional.

## Design Risks to Neutralize
1. **Evidence ambiguity**: Without stable IDs, evidence gates are non-reproducible.
2. **Slice blindness**: Pattern-scoped slices can hide context if omissions are not explicit.
3. **Over-pruning**: Hard-fail on missing ops blocks discovery; must allow controlled expansion.
4. **Merge conflicts**: Concurrent updates require CRDT-like deterministic ordering and conflict quarantine.
5. **Stale cache**: Prefix caching must include graph hash + PCP version.

## Success Criteria
- PCP v2 schema adopted with deterministic defaults and validation.
- Slices are bit-identical for identical inputs and include omission lists.
- Batch cost reduces measurably vs sequential while maintaining or improving precision/recall.
- Creative loop surfaces Tier-B hypotheses without violating evidence gates.
- Merge pipeline is idempotent and conflict-free.

## PCONTEXT Acceptance Checks (01..11)
- **PCONTEXT-01:** PCP v2 schema + validator + v1 shim exist; JSON schema export matches model; missing required ops/unknowns are rejected or flagged.
- **PCONTEXT-02:** Authoring guide + lint rules enforce determinism, unknowns policy, and required fields; templates include anti-signals/witnesses.
- **PCONTEXT-03:** Evidence IDs stable across rebuilds; witness + negative witness extraction deterministic; IDs include graph hash.
- **PCONTEXT-04:** Slicer produces edge-closed subgraphs; omission lists are explicit and preserved in manifests; ordering is deterministic.
- **PCONTEXT-05:** Expansion triggers on multi-hop separation; semantic dilation increases coverage predictably; suspicion-weighted escalation obeys budget.
- **PCONTEXT-06:** Budget governor enforces cheap→verify→deep gates; delta packing includes only new evidence; protocol context inclusion is recorded.
- **PCONTEXT-07:** Batch orchestration groups by cost/complexity; prefix cache keyed by graph hash + PCP version; fork-then-rank produces deterministic winners.
- **PCONTEXT-08:** Role contracts enforce input/output schemas; diversity policy yields distinct reasoning paths; skill docs state when to run + context gating.
- **PCONTEXT-09:** Creative loop outputs remain Tier-B; near-miss/mutation/counterfactual probes are deterministic and marked unknown unless evidenced.
- **PCONTEXT-10:** Merge pipeline uses append-only deltas with deterministic ordering; conflicts quarantined; replays are idempotent.
- **PCONTEXT-11:** Quality suite reports precision/recall deltas, novelty yield, evidence entropy, budget Pareto; regression checks are enforced.

## Minimal Creative Discovery Loop (Token-Bounded)
- **Inputs:** PCP v2 + slice + protocol context (if gated) + budget.
- **Steps:** near-miss mining → ordered/mutated variants → counterfactual probes → anomaly motifs.
- **Outputs:** Tier-B hypotheses only; confidence cannot exceed evidence-gated thresholds.
- **Budget rule:** halt at budget ceiling; emit "unknown" when evidence insufficient.

## Out of Scope
- Bulk PCP population across all patterns (Phase 7.2).
- Economic reasoning engine (Phase 5.11).
- UI or UX enhancements.
