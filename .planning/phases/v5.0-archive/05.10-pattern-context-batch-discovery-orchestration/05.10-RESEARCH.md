### Pattern 1: Content-Addressable Evidence IDs

**What:** Generate stable, deterministic evidence IDs using content hashing.

**When to use:** Any evidence reference that must be reproducible across graph rebuilds.

**Example:**
```python
# Source: Content-addressable storage principles + existing Evidence class
import hashlib
from dataclasses import dataclass
from typing import Optional

def evidence_id_for(
    graph_hash: str,
    file: str,
    line_start: Optional[int],
    line_end: Optional[int],
    semantic_op: Optional[str] = None,
) -> str:
    """Generate deterministic evidence ID.

    Components are joined in canonical order and hashed.
    """
    components = [
        graph_hash,
        file,
        str(line_start or 0),
        str(line_end or 0),
    ]
    if semantic_op:
        components.append(semantic_op)

    # Canonical serialization: sorted, joined with null separator
    canonical = "\x00".join(components)
    return hashlib.sha256(canonical.encode()).hexdigest()[:16]
```

### Pattern 3: Append-Only Merge with Deterministic Ordering

**What:** Merge concurrent findings using append-only deltas with lexicographic ordering by evidence ID.

**When to use:** Combining batch discovery results without conflicts.

**Example:**
```python
# Source: CRDT principles (G-Set with deterministic ordering)
from dataclasses import dataclass, field
from typing import FrozenSet

@dataclass(frozen=True)
class FindingDelta:
    """Immutable finding delta for append-only merging."""
    evidence_id: str
    finding_id: str
    claim_type: str  # "hypothesis", "verification", "contradiction"
    confidence: float
    source_agent: str
    timestamp_epoch_ms: int

def merge_deltas(
    existing: FrozenSet[FindingDelta],
    new: FrozenSet[FindingDelta],
) -> tuple[FrozenSet[FindingDelta], list[tuple[FindingDelta, FindingDelta]]]:
    """Merge deltas with conflict detection.

    Returns:
        - Merged set (union)
        - List of conflicts (same evidence_id, different claims)
    """
    merged = existing | new

    # Detect conflicts: same evidence_id, different claims
    by_evidence = {}
    conflicts = []
    for delta in merged:
        if delta.evidence_id in by_evidence:
            existing_delta = by_evidence[delta.evidence_id]
            if existing_delta.claim_type != delta.claim_type:
                conflicts.append((existing_delta, delta))
        else:
            by_evidence[delta.evidence_id] = delta

    return merged, conflicts
```

### Pattern 4: Fork-Then-Rank Orchestration

**What:** Spawn parallel scouts with diverse reasoning modes, then rank/verify results.

**When to use:** Batch pattern discovery where diverse perspectives improve coverage.

**Example:**
```python
# Source: Google ADK multi-agent patterns, Microsoft AutoGen patterns
@dataclass
class ScoutResult:
    hypotheses: list[dict]
    reasoning_mode: str  # "operation-first", "guard-first", "invariant-first"
    confidence: float

async def fork_then_rank(
    pattern: PatternContextPackV2,
    graph_slice: SliceResult,
    scouts: list[Callable],  # Parallel scout functions
    verifier: Callable,
) -> list[VerifiedFinding]:
    """Fork to multiple scouts, then rank and verify.

    1. Fork: Run scouts in parallel with diverse reasoning modes
    2. Gather: Collect all hypotheses
    3. Rank: Sort by confidence + evidence completeness
    4. Verify: Top-N hypotheses go to verifier
    """
    # Run scouts in parallel
    scout_results = await asyncio.gather(*[
        scout(pattern, graph_slice) for scout in scouts
    ])

    # Flatten and rank hypotheses
    all_hypotheses = []
    for result in scout_results:
        for hyp in result.hypotheses:
            hyp["reasoning_mode"] = result.reasoning_mode
            all_hypotheses.append(hyp)

    # Sort by confidence (deterministic tie-break by evidence_id)
    ranked = sorted(
        all_hypotheses,
        key=lambda h: (-h["confidence"], h["evidence_ids"][0] if h["evidence_ids"] else ""),
    )

    # Verify top-N
    verified = []
    for hyp in ranked[:10]:  # Budget-limited
        result = await verifier(hyp, pattern, graph_slice)
        if result.verdict != "rejected":
            verified.append(result)

    return verified
```

### Anti-Patterns to Avoid

- **Non-deterministic iteration:** Never iterate over dict without sorted() - use `sorted(d.items())` or `sorted(d.keys())`
- **Missing evidence context:** Never allow claims without evidence_ids or explicit "unknown" markers
- **Silent omission:** Never drop context without recording what was omitted in a manifest
- **Hash instability:** Never include timestamps or random values in evidence IDs
- **Implicit safety:** Never interpret missing signals as "safe" - always mark as "unknown"

## Common Pitfalls

### Pitfall 1: Evidence ID Instability

**What goes wrong:** Evidence IDs change between graph rebuilds, breaking caching and reproducibility.

**Why it happens:** Including non-deterministic components (timestamps, dict iteration order, object hashes).

**How to avoid:**
1. Use only content-derived components: file path, line numbers, semantic operation
2. Include graph hash as version component
3. Sort all list/dict inputs before hashing
4. Test with two sequential builds, assert IDs match

**Warning signs:** Cache misses on unchanged code, flaky tests

### Pitfall 2: Silent Context Drops

**What goes wrong:** Agents assume missing context means "safe" when it actually means "unknown".

**Why it happens:** Protocol context omission not recorded, missing ops not flagged.

**How to avoid:**
1. Every slice must include `omission_list` and `missing_required_ops`
2. Every context pack must include `context_included: {protocol: true/false, rationale: "..."}`
3. Unknowns policy enforced at schema level: `missing_required: "unknown"` not `false`

**Warning signs:** False negatives on patterns that should match

### Pitfall 3: Non-Deterministic Merge Ordering

**What goes wrong:** Batch results differ based on execution order, breaking reproducibility.

**Why it happens:** Dict/set iteration order, async completion order, timestamp-based sorting.

**How to avoid:**
1. Sort by evidence_id (content-derived, deterministic)
2. Use frozen sets for immutable delta collections
3. Deterministic tie-breaker in all sorts
4. Test with shuffled input order, assert identical output

**Warning signs:** Different results on retry, flaky integration tests

### Pitfall 4: PCP Field Trimming

**What goes wrong:** Token budget enforcement trims PCP fields, causing pattern evaluation failures.

**Why it happens:** Naive token reduction targets all content equally.

**How to avoid:**
1. PCP fields are never trimmed - they are the pattern definition
2. Only graph slices are reduced (fewer nodes/edges)
3. Budget governor has separate budgets: `pcp_max_tokens` vs `context_max_tokens`

**Warning signs:** Pattern match failures after context packing

### Pitfall 5: Conflict Resolution Loops

**What goes wrong:** Contradicting deltas cause infinite resolution loops or data loss.

**Why it happens:** No quarantine mechanism, resolver agent creates new conflicts.

**How to avoid:**
1. Quarantine conflicting deltas (preserve both, mark as conflict)
2. Resolver agent output goes to separate "resolution" channel
3. Idempotent merge: `merge(merge(A, B), B) == merge(A, B)`
4. Audit trail for all conflict resolutions

**Warning signs:** Growing conflict queues, resolution timeouts

from pydantic import BaseModel, Field, ConfigDict
from typing import Optional

class DeterminismConfig(BaseModel):
    """Determinism constraints for PCP."""
    no_rag: bool = True
    no_name_heuristics: bool = True
    serialization_order: list[str] = ["pcp", "slice", "protocol", "evidence"]
    hash_seed: str = "graph_hash + pcp_version + pattern_id"

class BudgetConfig(BaseModel):
    """Token budget configuration."""
    pcp_max_tokens: int = 800
    context_max_tokens: int = 2200
    cheap_pass_tokens: int = 1200
    verify_pass_tokens: int = 1800
    deep_pass_tokens: int = 2400

class WitnessConfig(BaseModel):
    """Witness evidence configuration."""
    minimal_required: list[str] = Field(default_factory=list)
    negative_required: list[str] = Field(default_factory=list)

class UnknownsPolicy(BaseModel):
    """Explicit unknowns handling policy."""
    missing_required: str = "unknown"
    missing_optional: str = "unknown"
    missing_anti_signal: str = "unknown"

class PatternContextPackV2(BaseModel):
    """PCP v2 schema with deterministic defaults."""

    model_config = ConfigDict(
        extra="forbid",
        json_schema_extra={"$schema": "https://json-schema.org/draft/2020-12/schema"}
    )

    id: str
    version: str = "2.0"
    pattern_id: str
    name: str
    summary: str
    scope: str = "Function"

    determinism: DeterminismConfig = Field(default_factory=DeterminismConfig)
    budget: BudgetConfig = Field(default_factory=BudgetConfig)
    evidence_weights: dict[str, list[str]] = Field(default_factory=lambda: {
        "required": [], "strong": [], "weak": [], "optional": []
    })
    witness: WitnessConfig = Field(default_factory=WitnessConfig)
    op_signatures: dict = Field(default_factory=dict)
    anti_signals: list[dict] = Field(default_factory=list)
    guard_taxonomy: list[dict] = Field(default_factory=list)
    risk_envelope: dict = Field(default_factory=dict)
    economic_placeholders: dict = Field(default_factory=dict)
    unknowns_policy: UnknownsPolicy = Field(default_factory=UnknownsPolicy)

    @classmethod
    def from_v1(cls, v1_data: dict) -> "PatternContextPackV2":
        """Migrate PCP v1 to v2 with explicit defaults."""
        return cls(
            id=f"pcp-{v1_data.get('pattern_id', 'unknown')}",
            pattern_id=v1_data.get("pattern_id", "unknown"),
            name=v1_data.get("name", ""),
            summary=v1_data.get("summary", ""),
            op_signatures=v1_data.get("op_signatures", {}),
            # ... additional field mappings with explicit defaults
        )

### Budget Governor

```python
# Source: Existing agents/context/types.py patterns
from dataclasses import dataclass
from enum import Enum

class PassType(Enum):
    CHEAP = "cheap"
    VERIFY = "verify"
    DEEP = "deep"

@dataclass
class BudgetPolicy:
    """Budget policy for escalation ladder."""
    cheap_pass_tokens: int = 1200
    verify_pass_tokens: int = 1800
    deep_pass_tokens: int = 2400
    hard_limit: int = 6000

    def get_budget_for_pass(self, pass_type: PassType) -> int:
        """Get token budget for pass type."""
        budgets = {
            PassType.CHEAP: self.cheap_pass_tokens,
            PassType.VERIFY: self.verify_pass_tokens,
            PassType.DEEP: self.deep_pass_tokens,
        }
        return budgets[pass_type]

    def should_escalate(
        self,
        current_pass: PassType,
        coverage_score: float,
        has_unknowns: bool,
    ) -> Optional[PassType]:
        """Determine if escalation is warranted.

        Escalation ladder: cheap -> verify -> deep
        Trigger: coverage < threshold AND unknowns present
        """
        if current_pass == PassType.DEEP:
            return None  # Already at max

        if coverage_score < 0.7 and has_unknowns:
            if current_pass == PassType.CHEAP:
                return PassType.VERIFY
            elif current_pass == PassType.VERIFY:
                return PassType.DEEP

        return None
```
