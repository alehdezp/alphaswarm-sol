## Decisions Made

1. **PCP reserved tokens (500) are never trimmed**: Protocol Context Pack fields are critical for analysis and should never be reduced to fit budget. Only graph slices are trimmed when budget is exceeded.

2. **Delta packing threshold**: Use delta packing when it provides >30% reduction compared to full context. Below this threshold, the overhead of delta computation doesn't justify the savings.

3. **Escalation rules**: The escalation path follows unknown -> expand -> verify -> deep -> manual. This ensures progressive context enrichment with budget control.

4. **Deterministic evidence ordering**: Evidence IDs are always sorted to ensure reproducible outputs across runs. This is important for debugging and testing.
