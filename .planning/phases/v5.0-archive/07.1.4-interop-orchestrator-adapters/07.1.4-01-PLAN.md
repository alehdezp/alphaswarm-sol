Create the adapters module with the abstract base class that all orchestrator adapters must implement.

   - `name`: Adapter identifier (e.g., "agents-sdk", "langgraph")
   - `capabilities`: Set of AdapterCapability enums
   - `evidence_mode`: How evidence is preserved ("inline", "bead", "external")
   - `trace_propagation`: Trace propagation mode ("header", "context", "none")
   - `max_handoff_depth`: Maximum handoff chain depth (default 10)

2. Define `HandoffContext` dataclass:
   - `source_agent`: Agent ID handing off
   - `target_agent`: Agent ID receiving
   - `bead_id`: Optional VulnerabilityBead ID being transferred
   - `evidence_snapshot`: Dict of evidence at handoff point
   - `trace_id`: Trace correlation ID
   - `parent_span_id`: Parent span for trace continuity
   - `timestamp`: ISO timestamp

3. Define `TraceContext` dataclass:
   - `trace_id`: Unique trace identifier
   - `span_id`: Current span ID
   - `parent_span_id`: Parent span (for nesting)
   - `operation`: Operation name
   - `attributes`: Dict of trace attributes
   - `events`: List of trace events with timestamps

4. Define `OrchestratorAdapter` abstract base class with methods:
   - `async execute_agent(config: AgentConfig, messages: List[Dict]) -> AgentResponse`
   - `async handoff(ctx: HandoffContext) -> HandoffResult`
   - `async spawn_with_trace(config: AgentConfig, task: str, trace: TraceContext) -> AgentResponse`
   - `get_capabilities() -> Set[AdapterCapability]`
   - `supports(capability: AdapterCapability) -> bool`
   - `export_trace(trace: TraceContext) -> Dict[str, Any]` (for interop)
   - `import_trace(data: Dict[str, Any]) -> TraceContext` (for interop)
   - `preserve_evidence(bead: VulnerabilityBead, ctx: HandoffContext) -> VulnerabilityBead`

5. Define `HandoffResult` dataclass:
   - `success`: bool
   - `target_response`: Optional AgentResponse
   - `evidence_preserved`: bool
   - `trace_continued`: bool
   - `errors`: List of error messages

Import types from existing runtime base and beads schema. Ensure evidence contracts are NOT modified during handoffs - add validation in `preserve_evidence`.

<task type="auto">
  <name>Task 2: Create capability matrix and feature detection</name>
  <files>src/alphaswarm_sol/adapters/capability.py</files>
  <action>
Create the capability matrix that defines what features each adapter supports.

1. Define `AdapterCapability` enum with values:
   - `TOOL_EXECUTION`: Can execute tools/functions
   - `TOOL_CONVERSION`: Can convert between tool formats
   - `MEMORY_SHARED`: Supports shared memory across agents
   - `MEMORY_THREAD`: Supports thread-local memory
   - `MEMORY_PERSISTENT`: Supports persistent checkpointing
   - `TRACE_PROPAGATION`: Can propagate trace context
   - `TRACE_EXPORT_OTEL`: Can export OpenTelemetry traces
   - `TRACE_EXPORT_CUSTOM`: Can export custom trace format
   - `HANDOFF_SYNC`: Supports synchronous handoffs
   - `HANDOFF_ASYNC`: Supports async handoffs
   - `GUARDRAILS`: Supports input/output guardrails
   - `COST_TRACKING`: Tracks token/cost usage
   - `BEAD_REPLAY`: Can replay bead state for resumption
   - `GRAPH_FIRST`: Enforces BSKG query requirements

2. Define `CapabilityMatrix` class:
   - `__init__(adapter_name: str, capabilities: Set[AdapterCapability])`
   - `supports(cap: AdapterCapability) -> bool`
   - `supports_all(caps: Set[AdapterCapability]) -> bool`
   - `supports_any(caps: Set[AdapterCapability]) -> bool`
   - `missing_for(required: Set[AdapterCapability]) -> Set[AdapterCapability]`
   - `to_dict() -> Dict[str, bool]` (for serialization)

3. Define `ADAPTER_CAPABILITIES` constant dict mapping adapter names to CapabilityMatrix:
   ```python

           AdapterCapability.GUARDRAILS,
           AdapterCapability.COST_TRACKING,
       }),
       "langgraph": CapabilityMatrix("langgraph", {
           AdapterCapability.TOOL_EXECUTION,
           AdapterCapability.MEMORY_PERSISTENT,

           AdapterCapability.HANDOFF_ASYNC,
           AdapterCapability.BEAD_REPLAY,
       }),
       "autogen": CapabilityMatrix("autogen", {
           AdapterCapability.TOOL_EXECUTION,
           AdapterCapability.MEMORY_THREAD,
           AdapterCapability.HANDOFF_SYNC,
           AdapterCapability.HANDOFF_ASYNC,
       }),
       "crewai": CapabilityMatrix("crewai", {
           AdapterCapability.TOOL_EXECUTION,
           AdapterCapability.MEMORY_SHARED,
           AdapterCapability.HANDOFF_SYNC,
       }),
       "beads-gastown": CapabilityMatrix("beads-gastown", {
           AdapterCapability.MEMORY_PERSISTENT,
           AdapterCapability.BEAD_REPLAY,
           AdapterCapability.GRAPH_FIRST,
       }),
   }
   ```

4. Add `get_capability_matrix(adapter_name: str) -> CapabilityMatrix` helper function.
