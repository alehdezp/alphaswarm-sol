## Technical Decisions Made

### 1. VRS Agent Nodes in State Graph
**Decision:** Create fixed nodes for each VRS role (attacker, defender, verifier) with conditional routing

**Context:** LangGraph requires predefined nodes and edges for state graph compilation

**Impact:**
- ✅ Predictable routing logic
- ✅ Easier debugging and visualization
- ⚠️ Adding new agent roles requires graph modification

### 2. JSONL Storage for Checkpoints
**Decision:** Use JSONL append-only log with `latest.json` cache per bead

**Context:** Need git-friendly checkpoint storage that preserves investigation history

**Storage Structure:**
```
.langgraph_checkpoints/
  {bead_id}/
    checkpoints.jsonl  # Append-only log
    latest.json        # Fast access cache
```

**Impact:**
- ✅ Git-friendly diffs (line-by-line changes)
- ✅ Append-only semantics prevent history loss
- ✅ Human-readable for debugging
- ⚠️ Slower random access (must scan JSONL)

### 3. Bead Hash Validation
**Decision:** Store SHA256 hash of serialized bead data with each checkpoint

**Context:** Need to detect checkpoint corruption or bead state divergence during replay

**Impact:**
- ✅ Strong integrity guarantee
- ✅ Minimal storage overhead
- ⚠️ Cannot identify which field changed (only that something changed)

### 4. Replay Chain Reconstruction
**Decision:** Use parent checkpoint links to form linked list chain

**Context:** Need deterministic replay from initial state to target checkpoint

**Chain Structure:**
```
ckpt-1 (attacker) ← ckpt-2 (defender) ← ckpt-3 (verifier)
(parent: None)      (parent: ckpt-1)     (parent: ckpt-2)
```

**Impact:**
- ✅ Explicit chain without timestamp assumptions
- ✅ Efficient traversal (no sorting required)
- ⚠️ Broken parent links break replay

## Known Limitations

1. **LangGraph Optional Dependency:** Adapter only available if langgraph installed
   - Graceful fallback with `HAS_LANGGRAPH` flag
   - Import errors handled in tests with `pytest.mark.skipif`

2. **Mock Agent Execution in Tests:** Tests use mock AgentExecutor
   - Full integration tests require real LangGraph graph execution
   - Consider adding integration tests in Phase 07.3

3. **Checkpoint Cleanup:** Manual cleanup required
   - `cleanup_old_checkpoints()` must be called explicitly
   - Consider background cleanup job for production

4. **Replay Verification:** Replay tests verify state loaded, not full re-execution
   - Full replay validation requires graph re-execution
   - Consider adding replay correctness tests

## Lessons Learned

1. **TypedDict for State Schema:** LangGraph requires TypedDict for state, not dataclass
   - Provides type hints without runtime overhead
   - Compatible with LangGraph's state validation

2. **Conditional Edges Require Return Value Mapping:** LangGraph conditional edges need explicit mapping
   - Return value from `_route_next_agent` must match edge mapping keys
   - Use string constants to avoid typos

3. **Checkpointer Protocol Differences:** LangGraph checkpoint protocol differs from simple save/load
   - `BaseCheckpointSaver` requires `get`/`put`/`list` methods
   - `RunnableConfig` passed to all methods for context

4. **JSONL for Git-Friendly Storage:** Append-only JSONL provides clean git diffs
   - Each checkpoint is a single line
   - Git shows line-by-line changes, not full file rewrites
