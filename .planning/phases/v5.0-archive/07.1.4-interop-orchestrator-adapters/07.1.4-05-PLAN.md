Create the Beads/Gas Town adapter for git-backed orchestration.

1. Define `BeadsGasTownConfig(AdapterConfig)`:
   - `repo_path`: Path to git repository
   - `branch_prefix`: Prefix for bead branches (default "bead/")
   - `auto_commit`: bool (default True)
   - `commit_message_template`: str (default "bead: {bead_id} - {action}")
   - `enable_worktree`: bool (default True for isolation)

2. Define `GitBackedBead` class:
   - `__init__(bead: VulnerabilityBead, repo_path: Path, branch: str)`:
     - Store bead state in git branch
     - Track changes via commits

   - `save() -> str`:
     - Write bead to YAML/JSON in repo
     - Commit with message
     - Return commit hash

   - `load(commit_hash: str) -> VulnerabilityBead`:
     - Checkout specific commit
     - Load bead from file
     - Return bead instance

   - `get_history() -> List[Dict[str, Any]]`:
     - Return commit history for bead branch
     - Include timestamps, actions, commit hashes

   - `replay_to(commit_hash: str) -> VulnerabilityBead`:
     - Reset to specific commit
     - Return bead at that state

3. Implement `BeadsGasTownAdapter(OrchestratorAdapter)`:
   - `__init__(config: BeadsGasTownConfig)`:
     - Verify git repo exists
     - Initialize worktree manager

   - `async execute_agent(config: AgentConfig, messages: List[Dict]) -> AgentResponse`:
     - Create worktree for isolated execution
     - Execute agent in worktree
     - Commit results
     - Return AgentResponse

   - `async handoff(ctx: HandoffContext) -> HandoffResult`:
     - Create new branch for target agent work
     - Transfer bead state via git
     - Execute target agent
     - Merge or preserve branch history
     - Return HandoffResult with commit hash

   - `async replay_bead(bead_id: str, to_commit: str) -> VulnerabilityBead`:
     - Load GitBackedBead
     - Replay to specified commit
     - Return bead at that state

   - `get_capabilities() -> Set[AdapterCapability]`:
     - Return capabilities including BEAD_REPLAY, GRAPH_FIRST

   - `preserve_evidence(bead: VulnerabilityBead, ctx: HandoffContext) -> VulnerabilityBead`:
     - Commit evidence snapshot
     - Verify via git diff
     - Return bead with commit_hash in metadata

4. Add git helpers:
   - `_create_bead_branch(bead_id: str) -> str`
   - `_commit_bead_state(bead: VulnerabilityBead, message: str) -> str`
   - `_merge_bead_branch(source: str, target: str) -> bool`

Create the unified adapter registry for adapter management and discovery.

1. Implement `AdapterRegistry`:
   - `_adapters`: Dict[str, Type[OrchestratorAdapter]] (class storage)
   - `_instances`: Dict[str, OrchestratorAdapter] (instance cache)

   - `register(name: str, adapter_class: Type[OrchestratorAdapter]) -> None`:
     - Register adapter class by name

   - `get(name: str, config: Optional[AdapterConfig] = None) -> OrchestratorAdapter`:
     - Return cached instance or create new
     - Validate config against adapter requirements

   - `list_adapters() -> List[str]`:
     - Return registered adapter names

   - `get_capability_comparison() -> Dict[str, Dict[str, bool]]`:
     - Return capability matrix for all adapters

   - `find_adapter_for_capabilities(required: Set[AdapterCapability]) -> List[str]`:
     - Return adapters supporting all required capabilities

2. Create module-level registry singleton:
   ```python
   _registry = AdapterRegistry()

   def get_adapter(name: str, config: Optional[AdapterConfig] = None) -> OrchestratorAdapter:
       return _registry.get(name, config)

   def register_adapter(name: str, adapter_class: Type[OrchestratorAdapter]) -> None:
       _registry.register(name, adapter_class)
   ```

3. Auto-register all adapters:
