# Phase 7.1.4: Interop & Orchestrator Adapters - Context

## Purpose
Enable interoperability with modern orchestration ecosystems to validate AlphaSwarm’s workflow semantics across external runtimes and to reuse best-in-class infrastructure (tracing, persistence, deployment).

## Scope
- Adapter interface and capability matrix
- Agents SDK + Codex MCP adapter
- LangGraph swarm adapter
- AutoGen / Magentic-One adapter
- CrewAI adapter
- Beads/Gas Town adapter for git-backed orchestration
- Agentwise / Agentrooms integration notes

## Out of Scope
- Replacing the native orchestrator
- Production deployment for third-party platforms

## Key Constraints
- Preserve evidence-first outputs and BSKG query requirements
- Maintain deterministic bead histories across adapters

## Desired Outcomes
- Portable orchestration runs across at least three frameworks
- Shared trace semantics and handoff alignment
