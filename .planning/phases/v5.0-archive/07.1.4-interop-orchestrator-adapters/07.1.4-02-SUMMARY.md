## Decisions Made

1. **Guardrails filter tools at input, not output**
   - Rationale: Prevents execution of disallowed tools rather than detecting after
   - Implementation: `_apply_input_guardrails` filters tool list before SDK execution

2. **Evidence validation uses snapshot comparison**
   - Rationale: Lightweight validation without deep object comparison
   - Implementation: Snapshot key fields (id, class, severity, confidence, operations) in HandoffContext

3. **Codex adapter uses temp files for bead state**
   - Rationale: CLI can't share memory, needs serialization format
   - Implementation: JSON files in workdir/beads/ for bead transfer

4. **Trace context via environment variables in CLI**
   - Rationale: CLI execution doesn't have access to Python objects
   - Implementation: Set TRACE_ID, SPAN_ID, PARENT_SPAN_ID env vars

5. **Config dataclasses use field factories for defaults**
   - Rationale: Allows instantiation without explicit capabilities argument
   - Implementation: `capabilities: Set[Any] = field(default_factory=lambda: ...)`
