# Phase 7.1.4: Adapter Runtime Wiring - Gap Research

**Researched:** 2026-01-29
**Domain:** Multi-runtime agent execution and adapter integration
**Confidence:** HIGH

## Summary

Phase 7.1.4 created adapter interfaces (AgentsSdkAdapter, LangGraphAdapter, AutoGenAdapter, CrewAIAdapter, BeadsGasTownAdapter) but the `execute_agent` methods return mock/placeholder responses. The adapters need to be wired to existing AlphaSwarm runtimes to enable actual LLM execution.

**Key Finding:** Adapters reference non-existent `AgentExecutor` class. The correct pattern is to use `create_runtime()` factory and call `runtime.execute()` directly.

**Primary Blocker:** Test fixtures use incorrect `PatternContext` schema (using `operations`, `behavioral_signature`, `anti_patterns_checked` instead of actual schema fields).

## Existing Runtime Architecture

### Runtime Factory Pattern

AlphaSwarm has a production-ready multi-runtime system:

**Location:** `src/alphaswarm_sol/agents/runtime/factory.py`

**RuntimeType Enum:**
```python
class RuntimeType(str, Enum):
    OPENCODE = "opencode"      # OpenCode SDK (default, 75-95% cost reduction)
    CLAUDE_CODE = "claude_code" # Claude Code CLI (subscription)
    CODEX = "codex"            # Codex CLI (subscription)
    ANTHROPIC = "anthropic"    # Anthropic API (expensive, legacy)
    OPENAI = "openai"          # OpenAI API (expensive, legacy)
    AUTO = "auto"              # Smart routing
```

**Factory Function:**
```python
def create_runtime(
    sdk: Union[str, RuntimeType] = "auto",
    config: Any = None,
    **kwargs: Any,
) -> AgentRuntime
```

**Default behavior:** AUTO → OPENCODE (cost-optimized)

### AgentRuntime Interface

**Location:** `src/alphaswarm_sol/agents/runtime/base.py`

**Core Methods:**
```python
class AgentRuntime(ABC):
    async def execute(
        self,
        config: AgentConfig,
        messages: List[Dict[str, Any]],
    ) -> AgentResponse

    async def spawn_agent(
        self,
        config: AgentConfig,
        task: str,
    ) -> AgentResponse

    def get_model_for_role(self, role: AgentRole) -> str

    def get_usage(self) -> Dict[str, Any]
```

**Execute Signature:**
- **Input:** `AgentConfig` (role, system_prompt, tools, max_tokens, temperature, timeout_seconds)
- **Input:** `messages: List[Dict[str, Any]]` (OpenAI format: `[{"role": "user", "content": "..."}]`)
- **Output:** `AgentResponse` (content, tool_calls, tokens, cost, latency)

**Spawn Agent Signature:**
- **Input:** `AgentConfig`, `task: str`
- **Output:** `AgentResponse`
- **Purpose:** Context-fresh agent for single task (PHILOSOPHY.md pattern)

## Wiring Pattern (Code Examples)

### Pattern 1: Direct Runtime Wiring (AgentsSdkAdapter)

**For OpenAI Agents SDK adapter:**
```python
class AgentsSdkAdapter(OrchestratorAdapter):
    def __init__(self, config: AgentsSdkConfig):
        super().__init__(config)
        self.sdk_config = config

        # Get API key
        self.api_key = config.api_key or os.getenv("OPENAI_API_KEY")
        if not self.api_key:
            raise ValueError("OpenAI API key required")

        # Load guardrail policies
        self.guardrail_policies: Dict[str, Any] = {}
        if config.guardrails_enabled:
            self.guardrail_policies = self._load_guardrail_policies(
                Path(config.guardrail_policy_path)
            )

        # Trace context storage
        self._trace_storage: Dict[str, TraceContext] = {}

        # Create internal runtime for OpenAI Agents SDK
        from alphaswarm_sol.agents.runtime import create_runtime, RuntimeType

        self._internal_runtime = create_runtime(
            sdk=RuntimeType.OPENAI,
            api_key=self.api_key,
        )

    async def execute_agent(
        self, config: AgentConfig, messages: List[Dict[str, Any]]
    ) -> AgentResponse:
        # Apply input guardrails
        filtered_tools = self._apply_input_guardrails(
            config.role.value, config.tools
        )

        # Create modified config with filtered tools
        execution_config = AgentConfig(
            role=config.role,
            system_prompt=config.system_prompt,
            tools=filtered_tools,
            max_tokens=config.max_tokens,
            temperature=config.temperature,
            timeout_seconds=config.timeout_seconds,
            metadata=config.metadata,
        )

        # Execute via internal runtime
        response = await self._internal_runtime.execute(execution_config, messages)

        # Apply output guardrails
        if not self._validate_output_guardrails(config.role.value, response):
            raise ValueError(
                f"Response failed output guardrails for role {config.role.value}"
            )

        # Add adapter metadata
        response.metadata["adapter"] = "agents-sdk"
        response.metadata["guardrails_applied"] = self.sdk_config.guardrails_enabled
        response.metadata["tools_filtered"] = len(config.tools) - len(filtered_tools)

        return response
```

### Pattern 2: Default Runtime Wiring (LangGraphAdapter)

**For LangGraph adapter (doesn't need specific SDK):**
```python
class LangGraphAdapter(OrchestratorAdapter):
    def __init__(self, config: LangGraphConfig):
        if not HAS_LANGGRAPH:
            raise ImportError(
                "langgraph is not installed. Install with: pip install langgraph"
            )

        super().__init__(config)
        self.config: LangGraphConfig = config

        # Initialize checkpointer
        from .checkpointer import BeadCheckpointer

        self.checkpointer = (
            BeadCheckpointer(Path(config.persistence_path))
            if config.enable_replay
            else None
        )

        # Create internal runtime (default cost-optimized)
        from alphaswarm_sol.agents.runtime import create_runtime

        self._internal_runtime = create_runtime(sdk="auto")  # Defaults to opencode

        # Create state graph
        self.state_graph = VrsStateGraph(self)
        self.compiled_graph = self.state_graph.compile()

    async def execute_agent(
        self, config: AgentConfig, messages: List[Dict[str, Any]]
    ) -> AgentResponse:
        # Direct execution via internal runtime
        return await self._internal_runtime.execute(config, messages)

    async def handoff(self, ctx: HandoffContext) -> HandoffResult:
        # Validate handoff depth
        if ctx.handoff_depth >= self.config.max_handoff_depth:
            return HandoffResult(
                success=False,
                errors=[
                    f"Max handoff depth ({self.config.max_handoff_depth}) exceeded"
                ],
            )

        try:
            # Update state for handoff
            state_update = {
                "current_agent": ctx.source_agent,
                "next_agent": ctx.target_agent,
                "evidence": ctx.evidence_snapshot,
                "trace_context": {
                    "trace_id": ctx.trace_id,
                    "parent_span_id": ctx.parent_span_id,
                    "handoff_depth": ctx.handoff_depth,
                },
            }

            # Create checkpoint if enabled
            checkpoint_id = None
            if self.checkpointer:
                vrs_state: VrsState = {
                    "bead": {},
                    "messages": [],
                    "current_agent": ctx.source_agent,
                    "evidence": ctx.evidence_snapshot,
                    "trace_context": state_update["trace_context"],
                    "checkpoint_id": str(uuid.uuid4()),
                    "verdict_reached": False,
                    "next_agent": ctx.target_agent,
                }
                checkpoint_id = self.checkpointer.save(vrs_state, ctx.source_agent)

            # Execute target agent via internal runtime
            target_config = AgentConfig(
                role=AgentRole(ctx.target_agent) if ctx.target_agent in AgentRole.__members__.values() else AgentRole.DEFENDER,
                system_prompt="",  # Would be populated from catalog
                tools=[],
                metadata={"handoff_context": ctx.to_dict()},
            )
            target_response = await self._internal_runtime.execute(target_config, [])

            return HandoffResult(
                success=True,
                target_response=target_response,
                evidence_preserved=True,
                trace_continued=True,
                metadata={"checkpoint_id": checkpoint_id},
            )

        except Exception as e:
            return HandoffResult(
                success=False, errors=[f"Handoff failed: {str(e)}"]
            )
```

### Pattern 3: Spawn with Trace

**Generic spawn_with_trace implementation:**
```python
async def spawn_with_trace(
    self, config: AgentConfig, task: str, trace: TraceContext
) -> AgentResponse:
    # Create child span
    child_span_id = str(uuid.uuid4())

    # Add trace context to config metadata
    config.metadata["trace_context"] = trace.to_dict()
    config.metadata["task"] = task

    # Execute agent via internal runtime
    messages = [{"role": "user", "content": task}]
    response = await self._internal_runtime.execute(config, messages)

    # Attach trace metadata to response
    response.metadata["trace_id"] = trace.trace_id
    response.metadata["span_id"] = child_span_id
    response.metadata["parent_span_id"] = trace.span_id

    return response
```

## Open Questions

**Q1: Should adapters create one runtime instance per adapter, or one per agent execution?**

**Recommendation:** One runtime instance per adapter (in `__init__`), reused across executions. Rationale:
- Runtimes track usage statistics
- Creating runtime is fast but not free
- Context-fresh execution is at agent level (new messages), not runtime level

**Q2: Should adapters support runtime selection via config?**

**Recommendation:** Yes for flexibility. Add optional `runtime_type` field to adapter configs:
```python
@dataclass
class AgentsSdkConfig(AdapterConfig):
    # ...
    runtime_type: str = "openai"  # Override default
```

**Q3: How should adapters handle runtime failures?**

**Recommendation:** Bubbles up exceptions to caller. Fallback logic should be at orchestration layer, not adapter layer. Adapters are thin wrappers, orchestration owns retry/fallback policy.

**Q4: Should adapters use idempotency features from Phase 07.1.1-02?**

**Recommendation:** Not initially. Idempotency is orchestration concern, adapters should remain stateless. If needed, orchestration layer can set `idempotency_key` in AgentConfig and use `runtime.spawn_idempotent()`.
