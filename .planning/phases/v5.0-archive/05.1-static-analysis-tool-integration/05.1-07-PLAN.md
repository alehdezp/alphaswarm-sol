# /vkg:coordinate-tools

Analyze a Solidity project and create an optimal tool execution strategy.

**Model:** sonnet-4.5 (coordination decisions require reasoning)
**Load:** on-demand
**Phase 5.2 Ready:** Hook-compatible interface

## Purpose

Analyze project characteristics and decide:
1. Which tools to run
2. In what order (parallel groups)
3. With what configuration

## When to Use

- Starting a new audit
- After significant code changes
- When previous tool run had issues

## When NOT to Use

- Quick single-file analysis (use specific tool directly)
- Re-running same analysis (cached results available)

## Invocation

```python
from true_vkg.tools.coordinator import ToolCoordinator

coordinator = ToolCoordinator()
analysis = coordinator.analyze_project(project_path)
strategy = coordinator.create_strategy(analysis, available_tools)
print(coordinator.explain_strategy(strategy))
```

## Decision Logic

### Project Analysis Indicators

| Indicator | Detection Method | Affects |
|-----------|------------------|---------|
| Proxy patterns | delegatecall, ERC1967 | +mythril, +halmos |
| Complex math | mulDiv, FullMath | +mythril, +halmos |
| Value transfers | .transfer, .call{value | +echidna, +foundry |
| Oracle usage | priceFeed, getPrice | +slither oracles |
| External calls | .call( | +reentrancy focus |

### Tool Selection Matrix

| Project Type | Baseline | Add If Complex | Add If Value |
|--------------|----------|----------------|--------------|
| Simple ERC20 | slither, aderyn | semgrep | - |
| DeFi Protocol | slither, aderyn, semgrep | mythril, halmos | echidna |
| Upgradeable | slither, aderyn | mythril | halmos |
| Oracle Consumer | slither, aderyn | mythril | - |

### Parallel Execution Groups

Group 1 (Fast): slither, aderyn, semgrep (< 2 min each)
Group 2 (Slow): mythril, halmos (5-10 min each)
Group 3 (Fuzzing): echidna, foundry (10+ min)

## Output Format

Strategy includes:
- tools_to_run: List of tool names
- parallel_groups: List of parallel execution groups
- tool_configs: Per-tool configuration
- estimated_time: Expected total time
- rationale: Why these tools were selected

## Integration

Feed strategy to ToolExecutor:
```python
from true_vkg.tools.executor import ToolExecutor

executor = ToolExecutor()
results = executor.execute_strategy(strategy, project_path)
```

## Effectiveness Tracking

Log tool selections and outcomes for skill improvement:
- Which tools found what categories
- False positive rates by tool
- Execution times by project size

## Failure Handling

If analysis fails:
1. Log detailed error
2. Return safe default strategy (baseline tools only)
3. Note tools to skip in rationale

# /vkg:run-slither

Run Slither static analysis with VKG-optimized configuration.

**Model:** haiku-4.5 (tool running, output parsing)
**Load:** on-demand
**Phase 5.2 Ready:** Hook-compatible interface

## Purpose

Run Slither on a Solidity project with:
- VKG-optimized detector configuration
- Library path filtering
- Output normalization to BSKG findings

## When to Use

- First-pass static analysis (fast, comprehensive)
- Checking for common vulnerability patterns
- Every audit (baseline tool)

## When NOT to Use

- Already have cached Slither results for unchanged code
- Targeting symbolic execution bugs (use Mythril)
- Need property-based testing (use Echidna)

## Invocation

```bash
# CLI
vkg tools run slither /path/to/project

# Python
from true_vkg.tools.executor import ToolExecutor
from true_vkg.tools.config import get_optimal_config

executor = ToolExecutor()
config = get_optimal_config("slither")
result = executor.execute_tool("slither", config, project_path)
```

## Configuration

VKG-optimized defaults (skip low-value detectors):

```yaml
exclude_detectors:
  - naming-convention     # Style, not security
  - solc-version         # Build config, not vuln
  - pragma               # Build config
  - external-function    # Gas optimization
  - assembly             # Often intentional
exclude_severity: [informational]
filter_paths: ["lib/", "node_modules/", "test/"]
timeout: 120
```

## Output Interpretation

### High Confidence Detectors (likely real)

| Detector | BSKG Pattern | Action |
|----------|-------------|--------|
| reentrancy-eth | reentrancy-classic | Create HIGH bead |
| arbitrary-send-eth | access-control-permissive | Create HIGH bead |
| suicidal | selfdestruct-unprotected | Create CRITICAL bead |
| controlled-delegatecall | delegatecall-injection | Create CRITICAL bead |

### Medium Confidence (verify with VKG)

| Detector | BSKG Pattern | Action |
|----------|-------------|--------|
| uninitialized-state | uninitialized-state | Verify with graph |
| locked-ether | locked-funds | Check if intentional |
| timestamp | timestamp-dependence | Context-dependent |

### Known False Positives

- tx-origin in modern protocols (often legitimate)
- block-timestamp with wide windows (acceptable)
- solc-version for multi-version support

## Error Handling

| Error | Recovery |
|-------|----------|
| Timeout | Increase to 180s, exclude test/ |
| Compilation | Check Solidity version, foundry.toml |
| No output | Verify project structure |

## Integration with VKG

Slither findings feed through:
1. SlitherAdapter (parse JSON)
2. SARIF normalization
3. Deduplication
4. Bead creation (confidence=uncertain)
```

# Tool Selection Decision Guide

Flowchart for choosing which tools to run.

## Quick Decision Tree

  |
  +---> Oracle usage? --> Enable Slither oracle detectors
  |
  v
[Time budget?]
  |
  +---> <10 min --> Skip symbolic execution
  |
  +---> 10-30 min --> Add Mythril
  |
  +---> >30 min --> Full suite including fuzzing

## Tool Selection by Project Type

### Simple Token (ERC20/721)

**Run:** Slither, Aderyn
**Skip:** Mythril (overkill), Echidna (no complex state)
**Time:** 2-3 minutes

### DeFi Protocol (DEX, Lending)

**Run:** Slither, Aderyn, Semgrep, Mythril, Echidna
**Skip:** Halmos (unless formal verification needed)
**Time:** 15-30 minutes

### Upgradeable Contract

**Run:** Slither, Mythril, Halmos
**Focus:** Delegatecall, storage layout, proxy patterns
**Time:** 10-20 minutes

### Oracle Consumer

**Run:** Slither (with oracle detectors), Aderyn
**Focus:** Price manipulation, stale data, decimal handling
**Time:** 5-10 minutes

### Governance Contract

**Run:** Slither, Aderyn, Echidna
**Focus:** Access control, timelocks, voting mechanisms
**Time:** 10-15 minutes

## Tool Overlap & Redundancy

| Pattern | Best Tool | Avoid Running |
|---------|-----------|---------------|
| Reentrancy | Slither | Semgrep (lower precision) |
| Overflow | Mythril | Slither (if SafeMath) |
| Access Control | Slither | - |
| Delegatecall | Mythril | Slither (add only) |

## Cost/Benefit Analysis

| Tool | Runtime | Findings/Min | Precision |
|------|---------|--------------|-----------|
| Slither | 30-60s | 5-10 | 40% |
| Aderyn | 20-40s | 3-5 | 50% |
| Semgrep | 30-60s | 10-20 | 25% |
| Mythril | 5-10m | 1-3 | 80% |
| Echidna | 10-30m | 0-2 | 90% |
| Halmos | 5-15m | 0-1 | 95% |

**Insight:** For time-constrained audits, Slither + Aderyn provides best coverage per minute.

## Integration with BSKG Patterns

When tools have high precision for a pattern, BSKG may skip running its own pattern:

| BSKG Pattern | Tool Coverage | BSKG Skip? |
|-------------|---------------|-----------|
| reentrancy-classic | Slither 85%, Mythril 90% | Yes |
| access-control-permissive | Slither 80% | Yes |
| business-logic-violation | None | Never skip |
| economic-manipulation | None | Never skip |

"""Hook interfaces for Phase 5.2 SDK integration.

These interfaces define how tool skills will interact with the SDK's
hook system. Implementing now ensures Phase 5.1 work is SDK-ready.

The actual hook implementation (queues, routing, propulsion) will be
built in Phase 5.2. This module defines the contracts.
"""

from abc import ABC, abstractmethod
from dataclasses import dataclass, field
from datetime import datetime
from enum import Enum
from pathlib import Path
from typing import Any, Callable, Dict, List, Optional, Union
import asyncio


class HookPriority(str, Enum):
    """Priority levels for hook queue."""
    CRITICAL = "critical"
    HIGH = "high"
    NORMAL = "normal"
    LOW = "low"
    BACKGROUND = "background"


class ToolRunStatus(str, Enum):
    """Status of a tool run request."""
    PENDING = "pending"
    RUNNING = "running"
    COMPLETED = "completed"
    FAILED = "failed"
    TIMEOUT = "timeout"
    CACHED = "cached"


@dataclass
class ToolRunRequest:
    """Request to run a tool via hook system.

    This is what an agent submits to the hook queue.
    """
    tool: str
    project_path: Path
    config: Optional[Dict[str, Any]] = None
    priority: HookPriority = HookPriority.NORMAL
    timeout: int = 300
    use_cache: bool = True
    # Metadata for tracking
    request_id: str = ""
    requester: str = ""  # Agent ID
    created_at: datetime = field(default_factory=datetime.now)


@dataclass
class ToolRunResult:
    """Result from a tool run via hook system.

    This is what the hook returns to the requesting agent.
    """
    request_id: str
    tool: str
    status: ToolRunStatus
    findings: List[Any] = field(default_factory=list)  # List[VKGFinding]
    execution_time: float = 0.0
    error: Optional[str] = None
    from_cache: bool = False
    partial: bool = False
    completed_at: datetime = field(default_factory=datetime.now)

    def to_dict(self) -> Dict[str, Any]:
        """Serialize for hook transmission."""
        return {
            "request_id": self.request_id,
            "tool": self.tool,
            "status": self.status.value,
            "findings_count": len(self.findings),
            "execution_time": self.execution_time,
            "error": self.error,
            "from_cache": self.from_cache,
            "partial": self.partial,
        }


class ToolHook(ABC):
    """Abstract base for tool hook implementations.

    Phase 5.2 will implement concrete versions:
    - LocalToolHook: Direct execution
    - QueuedToolHook: Via message queue
    - DistributedToolHook: Across workers
    """

    @abstractmethod
    async def submit(self, request: ToolRunRequest) -> str:
        """Submit a tool run request.

        Returns: request_id for tracking
        """
        pass

    @abstractmethod
    async def get_result(
        self,
        request_id: str,
        timeout: Optional[float] = None
    ) -> ToolRunResult:
        """Wait for and retrieve result.

        Args:
            request_id: From submit()
            timeout: Max wait time in seconds

        Returns:
            ToolRunResult when available

        Raises:
            TimeoutError: If timeout exceeded
        """
        pass

    @abstractmethod
    async def get_status(self, request_id: str) -> ToolRunStatus:
        """Check status without blocking."""
        pass

    @abstractmethod
    async def cancel(self, request_id: str) -> bool:
        """Cancel a pending request."""
        pass


class LocalToolHook(ToolHook):
    """Direct execution hook for Phase 5.1 compatibility.

    Runs tools directly without queueing. Provides same interface
    as future queued implementations.
    """

    def __init__(self):
        self._results: Dict[str, ToolRunResult] = {}
        self._pending: Dict[str, asyncio.Task] = {}

    async def submit(self, request: ToolRunRequest) -> str:
        """Execute tool directly, store result."""
        from .executor import ToolExecutor
        from .config import get_optimal_config

        request_id = request.request_id or f"local-{id(request)}"

        async def _run():
            executor = ToolExecutor()
            config = request.config or get_optimal_config(request.tool)
            result = executor.execute_tool(
                request.tool,
                config,
                request.project_path
            )
            return ToolRunResult(
                request_id=request_id,
                tool=request.tool,
                status=ToolRunStatus.COMPLETED if result.success else ToolRunStatus.FAILED,
                findings=result.findings,
                execution_time=result.execution_time,
                error=result.error,
                from_cache=result.from_cache,
                partial=result.partial,
            )

        self._pending[request_id] = asyncio.create_task(_run())
        return request_id

    async def get_result(
        self,
        request_id: str,
        timeout: Optional[float] = None
    ) -> ToolRunResult:
        """Wait for task completion."""
        if request_id in self._results:
            return self._results[request_id]

        if request_id not in self._pending:
            raise ValueError(f"Unknown request: {request_id}")

        task = self._pending[request_id]
        try:
            result = await asyncio.wait_for(task, timeout=timeout)
            self._results[request_id] = result
            del self._pending[request_id]
            return result
        except asyncio.TimeoutError:
            return ToolRunResult(
                request_id=request_id,
                tool="",
                status=ToolRunStatus.TIMEOUT,
                error="Request timed out",
            )

    async def get_status(self, request_id: str) -> ToolRunStatus:
        """Check if task is done."""
        if request_id in self._results:
            return self._results[request_id].status
        if request_id in self._pending:
            task = self._pending[request_id]
            return ToolRunStatus.COMPLETED if task.done() else ToolRunStatus.RUNNING
        return ToolRunStatus.PENDING

    async def cancel(self, request_id: str) -> bool:
        """Cancel pending task."""
        if request_id in self._pending:
            self._pending[request_id].cancel()
            del self._pending[request_id]
            return True
        return False


# Convenience functions for skill authors
def get_tool_hook() -> ToolHook:
    """Get the configured tool hook.

    Returns LocalToolHook for Phase 5.1.
    Phase 5.2 will return SDK-configured hook.
    """
    return LocalToolHook()


async def run_tool(
    tool: str,
    project_path: Path,
    config: Optional[Dict[str, Any]] = None,
    priority: HookPriority = HookPriority.NORMAL,
) -> ToolRunResult:
    """Convenience function to run a tool via hook.

    Example:
        result = await run_tool("slither", Path("./contracts"))
        if result.status == ToolRunStatus.COMPLETED:
            for finding in result.findings:
                print(finding.title)
    """
    hook = get_tool_hook()
    request = ToolRunRequest(
        tool=tool,
        project_path=project_path,
        config=config,
        priority=priority,
    )
    request_id = await hook.submit(request)
    return await hook.get_result(request_id)
