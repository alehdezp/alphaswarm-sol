  <name>Task 1: Echidna Adapter</name>
  <files>src/true_vkg/tools/adapters/echidna_adapter.py</files>
  <action>
Create adapter for Echidna property-based fuzzing results.

**Echidna output format:**
Echidna outputs to stdout/stderr with test results. Key formats:
- `echidna_xxx: failed!` -> Property violation found
- `echidna_xxx: passed!` -> Property holds
- JSON output with `--format json`: contracts, seeds, coverage data

**EchidnaAdapter class:**

Property violation to finding mapping:
```python
PROPERTY_TO_PATTERN = {
    "echidna_revert": "dos-revert",
    "echidna_no_reentrant": "reentrancy-classic",
    "echidna_balance_invariant": "arithmetic-overflow",
    # VKG-generated properties map back to patterns
}
```

**Methods:**
- parse_output(stdout: str, stderr: str) -> List[Dict]
- parse_json(output: str) -> List[Dict] (for --format json)
- to_vkg_findings(raw: List[Dict]) -> List[VKGFinding]
- to_sarif(raw: List[Dict]) -> Dict

**Finding structure from Echidna:**
- Property name (e.g., echidna_no_overflow)
- Counterexample (call sequence that violates property)
- Contract/function where violation occurred
- Seed for reproduction

Severity: high (property violation indicates exploitable bug)
Confidence: 0.95 (fuzzing found actual violation)

Handle both text and JSON output formats.

<task type="auto">
  <name>Task 2: Foundry Adapter</name>
  <files>src/true_vkg/tools/adapters/foundry_adapter.py</files>
  <action>
Create adapter for Foundry (forge) test results.

**Foundry output formats:**
- `forge test --json` outputs JSON with test results
- `forge test -vvv` outputs detailed traces

**JSON structure:**
```json
{
  "test_name": {
    "status": "Success|Failure|Skipped",
    "reason": "...",
    "counterexample": {...},
    "logs": [...],
    "decoded_logs": [...]
  }
}
```

**FoundryAdapter class:**

Test naming convention to pattern mapping:
```python
TEST_PREFIX_TO_PATTERN = {
    "test_RevertIf_": None,  # Expected failures, not bugs
    "testFuzz_": None,  # Fuzz tests, handle separately
    "testFail_": None,  # Expected to fail
    "invariant_": "invariant-violation",  # Invariant break = bug
    "test_exploit_": "known-exploit",  # Exploit PoC
}
```

**Methods:**
- parse_json(output: str) -> List[Dict]
- to_vkg_findings(raw: List[Dict]) -> List[VKGFinding]
- to_sarif(raw: List[Dict]) -> Dict
- extract_trace_info(logs: List[str]) -> Dict (parse stack trace)
- get_failure_location(reason: str, logs: List) -> Tuple[str, int] (file, line)

**Filtering logic:**
- Only report UNEXPECTED failures (not testFail_, not RevertIf_)
- Invariant failures are high severity
- Counterexamples from fuzz tests are high confidence

Confidence: 1.0 for invariant violations (reproducible)
Severity: Based on test name hints (exploit_ = critical)
  </action>
  <verify>
```bash
python -c "from true_vkg.tools.adapters.foundry_adapter import FoundryAdapter; print('Foundry adapter loaded')"
```
  </verify>
  <done>Foundry adapter parses forge test JSON, identifies unexpected failures as findings.</done>
</task>
