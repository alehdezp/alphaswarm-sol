## Model Tier Rationale

### Why Sonnet for Coordination
- Project analysis requires understanding code patterns
- Strategy decisions have consequences (wrong tools waste time)
- Explaining rationale needs reasoning capability
- Pattern skip decisions are nuanced

### Why Haiku for Execution
- Mechanical: run command, capture output
- No interpretation of findings needed
- Cost efficiency for many tool runs
- Fast inference for responsive execution
