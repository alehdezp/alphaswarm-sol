**4.6 Multi-Agent Adversarial Verification Protocol (~500 words)**
Frame as "Adversarial Verification Protocol" per 06-CONTEXT.md.

Agent roles:
- **Attacker (Opus):** Constructs exploit paths, identifies preconditions
- **Defender (Sonnet):** Searches for guards, invariants, mitigations
- **Verifier (Opus):** Cross-checks evidence, arbitrates conflicts

Debate protocol (iMAD-inspired):
1. Attacker produces claim with exploit narrative
2. Defender produces counterclaim with guard evidence
3. Verifier checks both sides, runs tests if available
4. Integrator issues verdict with rationale

Evidence anchoring requirement: ALL claims must reference code locations. No unsupported assertions.

Confidence buckets: confirmed, likely, uncertain, rejected
