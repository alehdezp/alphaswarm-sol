# AlphaSwarm.sol

**AI-native smart contract security analysis powered by behavioral knowledge graphs.**

AlphaSwarm.sol is a multi-agent Vulnerability Reasoning System (VRS) that detects smart contract vulnerabilities through semantic operations and behavioral signatures, not syntactic pattern matching.

## Key Features

- **Behavioral Detection**: Identify vulnerabilities by what code *does*, not what it's called
- **Multi-Agent Verification**: Attacker, defender, and verifier agents debate findings
- **Evidence-Linked**: Every finding includes exact code locations and reasoning
- **Protocol Context**: Economic reasoning for logic bug detection

pip install alphaswarm-sol

alphaswarm query "functions that transfer value without access control"
```

```
Source Code --> BSKG Builder --> Pattern Engine --> Beads --> Agent Pool --> Verdicts

# First Audit

This guide walks through analyzing a simple contract with AlphaSwarm.sol.

## Sample Contract

Create `contracts/Vault.sol`:

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.0;

contract Vault {
    mapping(address => uint256) public balances;

    function deposit() external payable {
        balances[msg.sender] += msg.value;
    }

    function withdraw(uint256 amount) external {
        require(balances[msg.sender] >= amount, "Insufficient");
        (bool success, ) = msg.sender.call{value: amount}("");
        require(success, "Transfer failed");
        balances[msg.sender] -= amount;  // State update after external call!
    }
}
```

## Build Knowledge Graph

```bash
alphaswarm build-kg contracts/
```

This creates a Behavioral Security Knowledge Graph (BSKG) with:
- Function properties (visibility, modifiers, operations)
- Semantic operations (TRANSFERS_VALUE_OUT, WRITES_USER_BALANCE)
- Behavioral signatures (operation ordering)

## Query for Vulnerabilities

```bash
# Natural language query
alphaswarm query "functions that make external calls before updating state"

# VQL query
alphaswarm query "FIND functions WHERE has_operations([TRANSFERS_VALUE_OUT, WRITES_USER_BALANCE]) AND NOT has_reentrancy_guard"
```

## Run Pattern Detection

```bash
alphaswarm detect --patterns reentrancy
```

## Next Steps

- Learn about [behavioral signatures](../architecture.md)
- Explore [CLI commands](../reference/cli.md)

# CLI Reference

AlphaSwarm.sol provides two CLI entry points: `alphaswarm` (full) and `aswarm` (shorthand).

## Global Options

```bash
alphaswarm --version    # Show version
alphaswarm --help       # Show help
```

## Commands

### build-kg

Build a Behavioral Security Knowledge Graph from Solidity sources.

```bash
alphaswarm build-kg <path> [OPTIONS]

# Examples
alphaswarm build-kg contracts/
alphaswarm build-kg contracts/ --with-labels
alphaswarm build-kg contracts/ --output graph.json
```

**Options:**
- `--with-labels`: Enable semantic labeling (LLM-assisted)
- `--output, -o`: Output path for graph JSON
- `--config`: Path to config file

### query

Query the knowledge graph using VQL or natural language.

```bash
alphaswarm query <query> [OPTIONS]

# Examples
alphaswarm query "public functions that write state"
alphaswarm query "FIND functions WHERE visibility = public AND writes_state"
```

**Options:**
- `--graph, -g`: Path to graph JSON
- `--format`: Output format (table, json, yaml)

### detect

Run pattern detection on a knowledge graph.

```bash
alphaswarm detect [OPTIONS]

# Examples
alphaswarm detect --patterns reentrancy
alphaswarm detect --tier a
alphaswarm detect --all
```

**Options:**
- `--patterns, -p`: Specific patterns to run
- `--tier`: Pattern tier (a, b, c, all)
- `--graph`: Path to graph JSON

### orchestrate

Manage multi-agent audit pools.

```bash
alphaswarm orchestrate <command> [OPTIONS]

# Subcommands
alphaswarm orchestrate start <pool-id> --scope contracts/
alphaswarm orchestrate status <pool-id>
alphaswarm orchestrate beads <pool-id>
alphaswarm orchestrate pause <pool-id>
alphaswarm orchestrate resume <pool-id>
```

### tools

Manage external analysis tools.

```bash
alphaswarm tools status          # Show installed tools
alphaswarm tools run <path>      # Run tools on contracts
alphaswarm tools dedupe <file>   # Deduplicate findings
```
```

# Architecture

AlphaSwarm.sol is built around the Behavioral Security Knowledge Graph (BSKG) and multi-agent verification.

## Core Components

### BSKG (Behavioral Security Knowledge Graph)

Unlike structural knowledge graphs that encode AST or CFG relationships, BSKG captures *behavioral* properties:

- **Semantic Operations**: What code does (TRANSFERS_VALUE_OUT, CHECKS_PERMISSION)
- **Behavioral Signatures**: Operation ordering (`R:bal->X:out->W:bal`)
- **Security Properties**: 50+ derived properties per function

### Pattern Engine

Three-tier pattern system:

| Tier | Detection | Verification | Example |
|------|-----------|--------------|---------|
| A | Graph-only | Deterministic | Missing reentrancy guard |
| B | LLM-assisted | Exploratory | Complex access control |
| C | Label-dependent | Semantic reasoning | Policy mismatch |

### Multi-Agent Verification

Findings are verified through structured debate:

1. **Attacker Agent** constructs exploit paths
2. **Defender Agent** finds guards and mitigations
3. **Verifier Agent** arbitrates with evidence cross-checking

All claims must reference specific code locations (evidence anchoring).

## Data Flow

```
Source Code
    |
    v
Slither Parser --> AST, CFG, call graph
    |
    v
BSKG Builder --> 50+ security properties per function
    |           --> Semantic operations (name-agnostic)
    |           --> Behavioral signatures
    v
Pattern Engine --> Tier A/B/C matching
    |
    v
Bead Generator --> Investigation packages
    |
    v
Agent Pool --> Attacker/Defender/Verifier
    |
    v
Verdict --> Evidence-linked finding
```
```

# Philosophy

AlphaSwarm.sol is built on eight core pillars. See the full [PHILOSOPHY.md](https://github.com/[org]/alphaswarm-sol/blob/main/docs/PHILOSOPHY.md) for details.

## Core Pillars

1. **Behavior over Names**: Detect what code does, not what it's called
2. **Semantic Operations**: Abstract behavioral vocabulary for detection
3. **Two-Tier Patterns**: Deterministic (Tier A) and exploratory (Tier B/C)
4. **Multi-Agent Verification**: Adversarial debate for robust findings
5. **Beads Orchestration**: Self-contained investigation packages
6. **Tool Orchestration**: Integrate external tools with smart deduplication
7. **VulnDocs Knowledge**: LLM-optimized vulnerability documentation
8. **Economic Context**: Protocol context for logic bug detection

## Key Insight

> "Names lie, behavior doesn't."

Traditional tools match patterns like `withdraw()` or `transfer()`. AlphaSwarm matches behavioral signatures like `R:bal->X:out->W:bal` (read balance, external call, write balance) - the reentrancy pattern regardless of function names.
```
