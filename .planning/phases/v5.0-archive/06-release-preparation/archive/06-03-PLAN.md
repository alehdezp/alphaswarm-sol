  <action>
Create Dockerfile at project root:

```dockerfile
# AlphaSwarm.sol CLI Docker Image
# Multi-stage build for minimal production image

# ============================================
# Stage 1: Build dependencies with uv
# ============================================
FROM python:3.12-slim AS builder

# Install uv for fast dependency resolution
COPY --from=ghcr.io/astral-sh/uv:latest /uv /uvx /bin/

WORKDIR /app

# Install dependencies first (cached layer)
RUN --mount=type=cache,target=/root/.cache/uv \
    --mount=type=bind,source=uv.lock,target=uv.lock \
    --mount=type=bind,source=pyproject.toml,target=pyproject.toml \
    uv sync --frozen --no-install-project --no-dev

# Copy source and install project
COPY . /app
RUN --mount=type=cache,target=/root/.cache/uv \
    uv sync --frozen --no-dev

# ============================================
# Stage 2: Runtime (minimal)
# ============================================
FROM python:3.12-slim

LABEL org.opencontainers.image.title="AlphaSwarm.sol"
LABEL org.opencontainers.image.description="AI-native smart contract security analysis"
LABEL org.opencontainers.image.source="https://github.com/[org]/alphaswarm-sol"
LABEL org.opencontainers.image.licenses="MIT"

# Install minimal runtime dependencies
RUN apt-get update && apt-get install -y --no-install-recommends \
    git \
    && rm -rf /var/lib/apt/lists/*

# Copy virtual environment from builder
COPY --from=builder /app/.venv /app/.venv

# Add venv to PATH
ENV PATH="/app/.venv/bin:$PATH"

# Create non-root user for security
RUN useradd -m -s /bin/bash alphaswarm
USER alphaswarm

# Working directory for audits
WORKDIR /audit

# Default command
ENTRYPOINT ["alphaswarm"]
CMD ["--help"]
```

Key features:
- Multi-stage build keeps image small
- uv for fast dependency installation
- Non-root user for security
- Caches uv downloads for faster rebuilds
- Only includes runtime dependencies

```dockerfile
# AlphaSwarm.sol Agent Docker Image
# Optimized for multi-agent orchestration

# ============================================
# Stage 1: Build (same as CLI)
# ============================================
FROM python:3.12-slim AS builder

COPY --from=ghcr.io/astral-sh/uv:latest /uv /uvx /bin/

WORKDIR /app

RUN --mount=type=cache,target=/root/.cache/uv \
    --mount=type=bind,source=uv.lock,target=uv.lock \

COPY . /app
RUN --mount=type=cache,target=/root/.cache/uv \
    uv sync --frozen --no-dev

# ============================================
# Stage 2: Agent Runtime
# ============================================
FROM python:3.12-slim

LABEL org.opencontainers.image.title="AlphaSwarm.sol Agent"
LABEL org.opencontainers.image.description="AlphaSwarm.sol optimized for agent integration"
LABEL org.opencontainers.image.source="https://github.com/[org]/alphaswarm-sol"

# Minimal runtime - no interactive features needed
RUN apt-get update && apt-get install -y --no-install-recommends \
    git \
    && rm -rf /var/lib/apt/lists/*

COPY --from=builder /app/.venv /app/.venv

ENV PATH="/app/.venv/bin:$PATH"

# Agent mode configuration
ENV ALPHASWARM_AGENT_MODE=1
ENV ALPHASWARM_OUTPUT_FORMAT=json
ENV ALPHASWARM_NO_INTERACTIVE=1

# Non-root user
RUN useradd -m -s /bin/bash alphaswarm
USER alphaswarm

WORKDIR /audit

ENTRYPOINT ["alphaswarm"]
```

Key differences from CLI Dockerfile:
- ALPHASWARM_AGENT_MODE=1 for non-interactive behavior
- ALPHASWARM_OUTPUT_FORMAT=json for machine parsing
- ALPHASWARM_NO_INTERACTIVE=1 to disable prompts

2. Create .github/workflows/docker.yml:

```yaml
name: Docker

on:
  push:
    tags:
      - 'v*'
  workflow_dispatch:

env:
  REGISTRY: ghcr.io
  IMAGE_NAME: ${{ github.repository }}

jobs:
  build-and-push:
    name: Build and Push
    runs-on: ubuntu-latest
    permissions:
      contents: read
      packages: write
    strategy:
      matrix:
        include:
          - dockerfile: Dockerfile
            suffix: ""
          - dockerfile: Dockerfile.agent
            suffix: "-agent"
    steps:
      - uses: actions/checkout@v4

      - name: Set up Docker Buildx
        uses: docker/setup-buildx-action@v3

      - name: Log in to Container Registry
        uses: docker/login-action@v3
        with:
          registry: ${{ env.REGISTRY }}
          username: ${{ github.actor }}
          password: ${{ secrets.GITHUB_TOKEN }}

      - name: Extract metadata
        id: meta
        uses: docker/metadata-action@v5
        with:
          images: ${{ env.REGISTRY }}/${{ env.IMAGE_NAME }}${{ matrix.suffix }}
          tags: |
            type=semver,pattern={{version}}
            type=semver,pattern={{major}}.{{minor}}
            type=raw,value=latest,enable={{is_default_branch}}

      - name: Build and push
        uses: docker/build-push-action@v6
        with:
          context: .
          file: ${{ matrix.dockerfile }}
          push: true
          tags: ${{ steps.meta.outputs.tags }}
          labels: ${{ steps.meta.outputs.labels }}
          cache-from: type=gha
          cache-to: type=gha,mode=max
```

This workflow:
- Triggers on version tags and manual dispatch
- Builds both CLI and agent images
- Pushes to ghcr.io (GitHub Container Registry)
- Tags with semver (v0.5.0 -> 0.5.0, 0.5, latest)
- Uses GitHub Actions cache for fast builds
