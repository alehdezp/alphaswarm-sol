<task type="auto">
  <name>Task 2: Execute Tier 1+2 crawl and extraction via subagent</name>
  <files>.true_vkg/crawl_cache/*.md, knowledge/vulndocs/categories/**/*.md, .true_vkg/discovery/state.yaml</files>
  <action>
    Spawn `knowledge-aggregation-worker` subagent with the following instructions:

    **Mission:** Process all 12 Tier 1+2 sources following the DISCOVER > REASON > LEARN > EVOLVE pipeline.

    **For each source:**
    1. CRAWL - Download content via crawl4ai Docker (follow Solidity-related links at Claude's discretion)
    2. FILTER - Remove non-Solidity content (can spawn crawl-filter-worker for parallel filtering)
    3. EXTRACT - Identify vulnerability patterns using 7-component extraction:
       - Identification Signals (operations, signatures, properties)
       - Remediation Signals (fix approaches, safe patterns)
       - Check Patterns (manual + automated)
       - Pattern Signals (Tier A deterministic, Tier B LLM)
       - Code Examples (vulnerable + safe, SEMANTIC ONLY)
       - Similar Scenarios (variations, related vulns)
       - Real-World Exploits (name, date, description)
    4. REASON - Apply decision framework:
       - ACCEPT: New value not in VulnDocs (1+ occurrence)
       - MERGE: Enhances existing entry (new delta)
       - CREATE_SPECIFIC: New variant (1+ occurrence)
       - CREATE_SUBCATEGORY: New variant class (3+ occurrences)
       - CREATE_CATEGORY: New attack mechanism (5+ occurrences)
       - REJECT: Redundant or off-topic
    5. INTEGRATE - Update VulnDocs following TEMPLATE.md:
       - core-pattern.md < 100 lines
       - Code examples < 15 lines each
       - Semantic-only (no variable names except library methods)
       - Include: vulnerable pattern, safe pattern, BSKG properties, fix steps
    6. UPDATE - Mark source complete in tier-1-2-sources.yaml

    **Critical Rules:**
    - NEVER use variable/function names (only operations like `TRANSFERS_VALUE_OUT`)
    - Skip Solidity < 0.8 vulnerabilities
    - Include all severities (informational, gas can become critical in context)
    - Merge same semantic signature regardless of source
    - Retry 3x on failure, then skip and log

    **Source Order (by priority):**
    1. Solodit (aggregated findings - most comprehensive)
    2. Code4rena reports
    3. Sherlock HIGH/CRITICAL
    4. Remaining sources

    **Output:**
    - Crawled content in `.true_vkg/crawl_cache/`
    - Updated VulnDocs in `knowledge/vulndocs/categories/`
    - Progress in `.true_vkg/discovery/tier-1-2-sources.yaml`
    - Novel discoveries in `.true_vkg/discovery/novel_findings.yaml`
  </action>
  <verify>
    - All 12 sources in tier-1-2-sources.yaml marked as complete or skipped-with-reason
    - VulnDocs categories updated with new patterns
    - No variable names in any core-pattern.md files (grep verification)
    - Discovery state.yaml updated with session progress
  </verify>
  <done>All 12 Tier 1+2 sources crawled, patterns extracted, VulnDocs updated with semantic-only content</done>
</task>
