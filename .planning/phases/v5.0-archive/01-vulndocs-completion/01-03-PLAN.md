    **Secureum:**
    - Comprehensive bootcamp content
    - Extract vulnerability checklist items
    - Map to semantic operations

    **YouTube sources:**
    - Use WebFetch or transcript extraction
    - Focus on code examples shown in videos
    - Extract the "teaching pattern" - how they explain the vuln

    **TIER 6 - CTF Platforms:**

    **CRITICAL:** Extract BOTH the vulnerability pattern AND the solution approach.

    **DamnVulnerableDeFi (highest priority):**
    - 13 challenges covering major DeFi vulnerability classes
    - Extract: challenge name, vulnerability type, solution pattern
    - Map to existing VulnDocs categories
    - Note: BSKG already benchmarks against DVDeFi

    **Ethernaut:**
    - Classic Solidity security challenges
    - Some may be pre-0.8 - skip those per context rules
    - Focus on patterns that apply to modern Solidity

    **For each CTF challenge:**
    ```yaml
    extraction:
      challenge_name: "..."
      vulnerability_class: "reentrancy|access-control|etc"
      vulnerable_pattern: "semantic code pattern"
      exploit_approach: "how attacker exploits"
      solution_insight: "what makes this solvable"
      vkg_mapping:
        operations: [TRANSFERS_VALUE_OUT, ...]
        signature: "R:bal->X:out->W:bal"
    ```

    **Processing order:**
    1. SWC Registry (foundational reference)
    2. DamnVulnerableDeFi (benchmark alignment)
    3. samczsun blog (novel patterns)
    4. Secureum (comprehensive checklist)
    5. Remaining sources

    **Quality rules:**
    - Researcher blogs often have excellent "why" explanations - capture these
    - Educational content may be more verbose - extract only patterns
    - CTF solutions should map to semantic operations, not solution code
    - Skip pre-0.8 Solidity content
