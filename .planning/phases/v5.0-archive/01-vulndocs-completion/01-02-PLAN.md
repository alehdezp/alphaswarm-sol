  <files>.true_vkg/crawl_cache/*.md, knowledge/vulndocs/categories/**/*.md, .true_vkg/discovery/state.yaml</files>
  <action>
    Spawn `knowledge-aggregation-worker` subagent with the following instructions:

    **Mission:** Process all 19 Tier 3 audit firm sources following DISCOVER > REASON > LEARN > EVOLVE.

    **Source-specific handling:**

    **GitHub repositories (Trail of Bits, Spearbit, Cyfrin):**
    - Clone/fetch repository contents
    - Process markdown/PDF audit reports
    - Extract findings from structured reports
    - Note: PDF extraction may require special handling

    **Blog sources (OpenZeppelin, Dedaub, Zellic, etc.):**
    - Crawl main blog index
    - Follow links to security-relevant posts
    - Extract vulnerability writeups, research articles
    - Skip non-security content (announcements, hiring, etc.)

    **For each extracted finding:**
    1. EXTRACT using 7-component framework:
       - Identification Signals (operations, signatures, properties)
       - Remediation Signals (fix approaches from audit recommendations)
       - Check Patterns (manual checks auditors perform)
       - Pattern Signals (deterministic vs reasoning-required)
       - Code Examples (from audit report code snippets)
       - Similar Scenarios (related findings in same audit)
       - Real-World Context (affected protocols, if disclosed)

    2. REASON with decision framework:
       - MERGE if enhances existing VulnDocs (likely for common vulns)
       - CREATE_SPECIFIC for novel audit-discovered variants
       - ACCEPT for new insights on existing patterns
       - REJECT for duplicates or non-generalizable findings

    3. INTEGRATE following TEMPLATE.md:
       - Audit firm insights often provide deeper "why" explanations
       - Include auditor-recommended checks in detection section
       - Reference audit methodology when it aids detection

    **Special considerations for audit content:**
    - Auditors use precise technical language - preserve semantic meaning
    - Audit recommendations often include edge cases - capture these
    - Some findings are protocol-specific - generalize to pattern IF possible
    - High severity findings get priority extraction

    **Processing order:**
    1. Trail of Bits publications (comprehensive, well-structured)
    2. OpenZeppelin blog (standards expertise)
    3. Consensys Diligence (methodology leader)
    4. Spearbit/Zellic (elite researcher content)
    5. Remaining sources by priority tier

    **Output:**
    - Crawled/cloned content in `.true_vkg/crawl_cache/`
    - Updated VulnDocs in `knowledge/vulndocs/categories/`
    - Progress in `.true_vkg/discovery/tier-3-sources.yaml`
  </action>
