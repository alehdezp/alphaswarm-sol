## Summary

Phase 5.7 consolidates three separate knowledge locations (`patterns/`, `knowledge/vulndocs/`, `vulndocs/`) into a unified `vulndocs/` structure. The research inventoried all current content, mapped integration points, and identified the migration approach.

**Current state:**
- **patterns/**: 146 YAML pattern files across 7 directories (core/, semantic/, label_patterns/, investigation/, semgrep/, lens/, root lens files)
- **knowledge/vulndocs/**: 20 categories with ~97 subcategories containing index.yaml and core-pattern.md files
- **vulndocs/**: Framework-only structure from Phase 5.4 (`.meta/templates/`, `.meta/instructions/`, README.md)

**Primary recommendation:** Migrate all content to unified `vulndocs/` with patterns co-located inside vulnerability folders per the Phase 5.4 design. This requires updating 16+ skills/agents and 15+ source files.

## Common Pitfalls

### Pitfall 1: Breaking Pattern Quality with YAML Formatting
**What goes wrong:** Multiline content in patterns (descriptions, prompts) gets corrupted during migration
**Why it happens:** Automated YAML writers can change formatting
**How to avoid:** Read and write patterns using raw YAML, preserve original formatting
**Warning signs:** Pattern descriptions become single-line or lose indentation

### Pitfall 2: Orphaned Imports in Skills/Agents
**What goes wrong:** Skills/agents reference old paths (`patterns/`, `knowledge/vulndocs/`)
**Why it happens:** Many markdown files with hardcoded paths
**How to avoid:** Comprehensive grep for all path references, update systematically
**Warning signs:** 404 errors in skill execution, "file not found" in agent logs

### Pitfall 3: Missing Pattern-to-VulnDoc Links
**What goes wrong:** Patterns migrated without `vulndoc:` field linking back
**Why it happens:** Legacy patterns don't have this field
**How to avoid:** Add `vulndoc: {category}/{subcategory}` to every migrated pattern
**Warning signs:** Validation warnings about missing vulndoc field

### Pitfall 4: CLI Hardcoded Paths
**What goes wrong:** CLI commands still reference old `patterns/` or `knowledge/vulndocs/`
**Why it happens:** Path constants in source files
**How to avoid:** Search all Python files for path references
**Warning signs:** CLI commands failing after migration

### Pitfall 5: Test Fixtures Using Old Paths
**What goes wrong:** Tests fail because they reference old directory structure
**Why it happens:** Test files have hardcoded paths
**How to avoid:** Update test fixtures alongside migration
**Warning signs:** Test failures in CI after migration

### Pitfall 6: Duplicate Category Names
**What goes wrong:** `knowledge/vulndocs/categories/reentrancy/` and pattern `reentrancy/` conflict
**Why it happens:** Parallel development created overlapping structures
**How to avoid:** Merge content carefully, prefer `knowledge/vulndocs/` as source of truth
**Warning signs:** Directory exists errors during migration

## Code Examples

### Pattern Migration Script Pattern
```python
from pathlib import Path
import yaml

def migrate_pattern(source: Path, target_vulndoc: str, vulndocs_root: Path) -> Path:
    """Migrate a pattern file to vulndocs structure.

    Args:
        source: Original pattern file path
        target_vulndoc: Target vulndoc path (e.g., "reentrancy/classic")
        vulndocs_root: Root vulndocs directory

    Returns:
        New pattern file path
    """
    # Read original pattern (preserve formatting)
    with open(source, 'r') as f:
        content = f.read()

    # Parse to add vulndoc field
    data = yaml.safe_load(content)
    data['vulndoc'] = target_vulndoc

    # Create target directory
    target_dir = vulndocs_root / target_vulndoc / "patterns"
    target_dir.mkdir(parents=True, exist_ok=True)

    # Write to new location
    target = target_dir / source.name
    with open(target, 'w') as f:
        yaml.dump(data, f, default_flow_style=False, allow_unicode=True, width=120)

    return target
```

### Skill Path Update Pattern
```python
def update_skill_paths(skill_path: Path) -> None:
    """Update path references in skill markdown file."""
    with open(skill_path, 'r') as f:
        content = f.read()

    # Path replacements
    replacements = [
        ('patterns/', 'vulndocs/'),
        ('knowledge/vulndocs/categories/', 'vulndocs/'),
        ('vulndocs/.meta/templates/', 'vulndocs/.meta/templates/'),
        ('vulndocs/.meta/instructions/', 'vulndocs/.meta/instructions/'),
    ]

    for old, new in replacements:
        content = content.replace(old, new)

    with open(skill_path, 'w') as f:
        f.write(content)
```
