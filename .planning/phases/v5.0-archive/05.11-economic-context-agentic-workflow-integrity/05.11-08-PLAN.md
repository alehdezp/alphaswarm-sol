  <name>Task 1: Implement Red Agent with MCTS-style attack synthesis</name>
  <files>src/alphaswarm_sol/agents/adversarial/__init__.py, src/alphaswarm_sol/agents/adversarial/red_agent.py</files>
  <action>
Build the Red Team attack agent:

1. RedAgent class:
   - synthesize_attack(finding, protocol_state, budget) -> AttackPlan
   - Uses LLM for move generation (what transactions are possible)
   - Uses GATE for attack viability scoring
   - Prunes branches where attack EV < 0

2. MCTS-inspired exploration:
   - ExplorationNode: protocol state + available actions
   - Selection: UCB1 algorithm for promising nodes
   - Expansion: LLM generates possible next transactions
   - Simulation: estimate exploit outcome
   - Backpropagation: update node values

3. AttackPlan dataclass:
   - exploit_path: List[Transaction] (ordered steps)
   - expected_profit: Decimal (from GATE)
   - success_probability: float
   - required_capital: Decimal (flash loans, etc.)
   - mev_vulnerability: bool (can be frontrun?)
   - time_constraints: Dict (timing windows)

4. Integration:
   - Use GATE's AttackPayoffMatrix for scoring
   - Use CEG's causal paths for exploit structure
   - Budget-limited exploration (max iterations, max depth)

5. Create agents/adversarial/__init__.py with exports.
  </action>

<task type="auto">
  <name>Task 2: Implement Blue Agent and Judge Agent</name>
  <files>src/alphaswarm_sol/agents/adversarial/blue_agent.py, src/alphaswarm_sol/agents/adversarial/judge_agent.py</files>
  <action>
Build the Blue Team and Judge agents:

1. BlueAgent class (blue_agent.py):
   - generate_defense(attack_plan, mitigation_db) -> DefensePlan
   - Propose patches that break attack causal chain
   - Cost analysis: implementation cost vs prevented loss
   - Invariant synthesis: suggest require() statements

2. DefensePlan dataclass:
   - patches: List[PatchRecommendation] (code changes)
   - mitigations: List[Mitigation] (configs, guards)
   - cost_estimate: Decimal (implementation effort)
   - effectiveness: float (% of attack prevented)
   - side_effects: List[str] (potential downsides)

3. PatchRecommendation dataclass:
   - file_path, line_range, description
   - code_snippet: suggested code
   - rationale: why this blocks the attack

4. JudgeAgent class (judge_agent.py):
   - evaluate(attack_plan, defense_plan, ground_truth) -> Verdict
   - Score attack: creativity, feasibility, economic viability
   - Score defense: effectiveness, cost-efficiency, completeness
   - Determine winner: did defense adequately counter attack?

5. Verdict dataclass:
   - red_score: float (0-100)
   - blue_score: float (0-100)
   - winner: "red" | "blue" | "draw"
   - novel_findings: List[Finding] (new vulnerabilities found)
   - reasoning: str (explanation of verdict)
  </action>
