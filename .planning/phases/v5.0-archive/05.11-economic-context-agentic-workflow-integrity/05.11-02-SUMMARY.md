## Implementation Details

### Task 1: ContextLinker with Causal Chain Support

Created `src/alphaswarm_sol/context/linker.py` (~800 LOC):

**Core Classes:**

1. **LinkRecord**: Function <-> economic context link with evidence tracking
   - `source`: PACK (explicit), SEMANTIC (inferred), HEURISTIC (guessed)
   - `has_two_key_evidence`: Checks if link has 2+ independent evidence refs
   - `can_surface`: Only true if not quarantined AND has two-key evidence
   - `quarantined`: Hidden from agents until corroborated

2. **CausalChainLink**: Exploitation chain with probability and counterfactuals
   - `root_cause_id`: Starting vulnerability
   - `exploit_steps`: Ordered list of steps to financial loss
   - `probability_chain`: Cumulative probability (product of step probabilities)
   - `counterfactual_blocks`: Mitigations that would break the chain

3. **ContextLinker**: Main linker class
   - `register_explicit_link()`: Pack mappings (authoritative)
   - `register_semantic_link()`: Inferred from ops (quarantined)
   - `register_heuristic_link()`: Low confidence (always quarantined)
   - `trace_causal_chain()`: BFS traversal of CAUSES/ENABLES/AMPLIFIES edges
   - `counterfactual_query()`: Check if mitigation would break chain
   - `inject_overlay_edges()`: Prepare edges for DomainKG overlay

### Task 2: Contract Passports and Invariant Registry

**Contract Passports** (`passports.py`, ~500 LOC):

1. **CrossProtocolDependency**: External protocol dependency
   - `dependency_type`: oracle, liquidity, governance, collateral, bridge, keeper
   - `criticality`: 1-10 scale (10 = critical)
   - `failure_cascade_impact`: What happens if dependency fails
   - `affected_functions`: Functions that depend on this protocol

2. **ContractPassport**: Per-contract economic summary
   - `economic_purpose`: What this contract does economically
   - `assets_handled`: Assets under custody/control
   - `cross_protocol_dependencies`: External protocol deps
   - `invariant_ids`: References to InvariantRegistry
   - `systemic_risk_score`: Derived from dependency centrality

3. **PassportBuilder**: Generate passports from context pack and KG

**Invariant Registry** (`invariant_registry.py`, ~640 LOC):

1. **InvariantSource**: DECLARED, MINED, HYBRID
   - DECLARED: From official docs, specs, governance
   - MINED: Discovered through trace analysis, fuzzing
   - HYBRID: Both declared and confirmed by mining

2. **InvariantRecord**: Individual invariant with source and confidence
   - `would_prevent_exploits`: List of known exploits this would block
   - `validate_trace()`: Record validation results and adjust confidence

3. **InvariantRegistry**: Central registry
   - `register_declared()`: From documentation
   - `register_mined()`: From trace analysis
   - `compare_mined_vs_declared()`: Detect discrepancies for review

**Documentation:**

Created `docs/reference/contract-passport.md` with full schema reference,
systemic risk scoring algorithm, and integration examples.

### Task 3: Policy Diff and Cross-Protocol Awareness in Routing

Updated `src/alphaswarm_sol/routing/router.py` (~230 LOC added):

1. **PolicyDiff**: Policy mismatch between expected and actual access control
   - `mismatch_type`: "missing_check", "wrong_role", "extra_permission"
   - `provenance`: declared, inferred, hypothesis
   - `confidence`: Higher for declared expectations

2. **AgentContext Extensions**:
   - `passport`: ContractPassport attached at routing time
   - `policy_diffs`: List of policy mismatches
   - `cross_protocol_dependencies`: External deps for focal nodes
   - `systemic_risk_score`: From passport

3. **ContextSlicer Extensions**:
   - `compute_policy_diff()`: Compare dossier policy against code
   - `_get_cross_protocol_dependencies()`: Extract deps for focal nodes
   - `_attach_economic_context()`: Attach passport, diffs, deps to context
   - `strict_mode`: If True, reject candidates without passports

4. **Agent-Specific Attachment**:
   - Classifier: Gets systemic risk score only (minimal overhead)
   - Attacker: Gets full economic context (enables attack reasoning)
   - Defender: Gets policy diffs and passport (guards reasoning)
   - Verifier: Gets passport and systemic risk for prioritization
