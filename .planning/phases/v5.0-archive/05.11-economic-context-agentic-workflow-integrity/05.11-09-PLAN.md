  <name>Task 1: Implement invariant mining from transaction traces</name>
  <files>src/alphaswarm_sol/economics/invariants/__init__.py, src/alphaswarm_sol/economics/invariants/miner.py, src/alphaswarm_sol/economics/invariants/patterns.py</files>
  <action>
Build the invariant mining infrastructure:

1. InvariantMiner class (miner.py):
   - mine_from_traces(contract, traces) -> List[InvariantCandidate]
   - Analyzes transaction execution traces for patterns
   - Uses Trace2Inv-style pattern matching
   - Filters noise with statistical confidence

2. Invariant pattern types (patterns.py):
   - MappingUpperBound: balance[user] <= totalSupply
   - MappingLowerBound: balance[user] >= 0
   - CallValueUpperBound: msg.value <= maxDeposit
   - StateTransitionConstraint: state can only go A->B->C
   - VariableRelation: reserveA * reserveB >= k (AMM)
   - SumInvariant: sum(balances) == totalSupply
   - MonotonicProperty: counter only increases

3. InvariantCandidate dataclass:
   - id: unique identifier
   - type: InvariantPattern
   - expression: string representation
   - confidence: float (0-1, based on trace coverage)
   - supporting_traces: int (how many traces support this)
   - counterexample_traces: int (violations found)
   - would_prevent_exploits: List[str] (known exploit IDs)

4. Statistical filtering:
   - Minimum trace coverage threshold (default 90%)
   - Counterexample tolerance (default 0%)
   - Confidence decay for edge cases

5. Create economics/invariants/__init__.py with exports.
  </action>


<task type="auto">
  <name>Task 2: Implement invariant synthesis and validation</name>
  <files>src/alphaswarm_sol/economics/invariants/synthesis.py</files>
  <action>
Build invariant synthesis and exploit validation:

1. InvariantSynthesizer class:
   - synthesize_require(invariant) -> RequireStatement
   - Generate Solidity require() from invariant pattern
   - Include revert message with invariant description
   - Handle complex invariants (relations, sums)

2. RequireStatement dataclass:
   - code: string (the require() statement)
   - insertion_point: (file, function, position)
   - invariant_id: reference to source invariant
   - gas_overhead: estimated additional gas cost

3. Exploit validation:
   - validate_against_exploits(invariant, exploit_db) -> ValidationResult
   - Test: would this invariant have prevented known exploits?
   - For each exploit, simulate with invariant enforced
   - Track: prevented_count, missed_count, false_alarm_count

4. Discrepancy detection:
   - compare_with_declared(mined, declared) -> List[Discrepancy]
   - Find invariants mined but not declared (missing safety)
   - Find invariants declared but not mined (dead code or untested)
   - Flag discrepancies for review

5. Integration with InvariantRegistry:
   - Register mined invariants with source=MINED
   - Link validation results to registry entries
  </action>
