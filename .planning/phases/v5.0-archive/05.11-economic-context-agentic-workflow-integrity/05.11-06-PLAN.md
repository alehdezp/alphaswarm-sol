  <name>Task 1: Implement attack payoff matrix computation</name>
  <files>src/alphaswarm_sol/economics/gate/__init__.py, src/alphaswarm_sol/economics/gate/attack_synthesis.py</files>
  <action>
Build the core attack synthesis engine:

1. AttackSynthesisEngine class:
   - compute_attack_ev(vulnerability, protocol_state, gas_price, tvl) -> AttackPayoffMatrix
   - Model each vulnerability as a 3-player game:
     * Attacker: maximize profit (exploit vs honest behavior)
     * Protocol: minimize losses (guards, timelocks, monitoring)
     * MEV Searchers: frontrun/backrun the attack (compete for MEV)

2. AttackPayoffMatrix dataclass:
   - attacker_strategies: List[AttackStrategy] (exploit paths)
   - protocol_strategies: List[ProtocolDefense] (guards, pauses)
   - mev_strategies: List[MEVStrategy] (frontrun, backrun, sandwich)
   - payoff_tensor: 3D array of payoffs for each strategy combination
   - dominant_strategies: Dict mapping player to dominant strategy

3. Expected value computation:
   - expected_profit = (success_prob * profit) - gas_cost - mev_extraction_loss
   - Include: slippage, oracle update costs, flashloan fees
   - Consider: timelock delays, cooldown periods, rate limits

4. Create economics/gate/__init__.py with exports.

<task type="auto">
  <name>Task 2: Implement Nash equilibrium solver and incentive analysis</name>
  <files>src/alphaswarm_sol/economics/gate/nash_solver.py, src/alphaswarm_sol/economics/gate/incentive_analysis.py</files>
  <action>
Implement game-theoretic solving:

1. NashEquilibriumSolver class (nash_solver.py):
   - solve_nash_equilibrium(payoff_matrix) -> NashResult
   - Support both pure and mixed strategy equilibria
   - Use Lemke-Howson algorithm for 2-player subgames
   - Approximate for 3-player with iterated best response

2. NashResult dataclass:
   - attacker_strategy: optimal attack path (or "honest")
   - protocol_strategy: optimal defense configuration
   - mev_strategy: expected MEV behavior
   - attacker_payoff: expected profit at equilibrium
   - convergence_prob: probability of reaching equilibrium
   - is_attack_dominant: True if attack dominates honest behavior

3. BlockingCondition dataclass:
   - condition_type: TIMELOCK | RATE_LIMIT | MEV_PROTECTION | GUARD
   - threshold: value that blocks attack (e.g., "timelock > 1 day")
   - effect: how much it reduces attacker payoff

4. IncentiveAnalyzer class (incentive_analysis.py):
   - analyze_incentives(protocol_state) -> IncentiveReport
   - Identify Proof-of-Behavior-style incentive structures
   - Flag when honest behavior is NOT dominant
   - find_blocking_conditions() to suggest mitigations

Integration: Nash results feed into rationality_gate.py for filtering.
