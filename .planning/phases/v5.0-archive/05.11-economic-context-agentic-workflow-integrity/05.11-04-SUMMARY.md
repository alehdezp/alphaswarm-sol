## Key Implementations

### EconomicRiskScorer (risk.py)

Comprehensive risk scoring with three components:

1. **Base Risk (0-10)**: VAR + PRIV + OFFCHAIN + GOV + INCENTIVE
2. **Game-Theoretic Adjustment (1-2x)**: Attack EV multiplier
3. **Causal Amplification (1-3x cap)**: From exploitation chains
4. **Systemic Risk (1-1.5x)**: Cross-protocol cascade factor

```python
# Total score calculation
adjustment = (attack_ev_score - 1.0) * 0.3 + (loss_amplification - 1.0) * 0.4 + (systemic_factor - 1.0) * 0.3
total_score = min(10.0, base_score * (1.0 + adjustment))
```

### CausalAnalyzer (causal_analysis.py)

Analyzes AMPLIFIES edges for loss amplification:

- Flash loan: 100x
- Leverage: 10x
- Oracle manipulation: 20x
- Token mint: 1000x
- Reentrancy: 10x

Multipliers combined with capped additive formula to prevent explosion.

### SystemicRiskScorer (systemic_risk.py)

Cross-protocol systemic risk from passports:

- **DependencyCentrality**: Degree, betweenness, protocol role
- **CascadeRisk**: Affected protocols, TVL at risk, cascade depth
- Components: oracle dependency, bridge dependency, governance concentration

### EconomicProbeRunner (economic_probes.py)

Seven probes for exploitability analysis:

1. **profitability_probe**: Is there clear payoff?
2. **control_path_probe**: Can attacker reach control?
3. **assumption_break_probe**: What if assumption fails?
4. **governance_capture_probe**: Can governance enable abuse?
5. **value_at_risk_probe**: Max extractable value?
6. **counterfactual_probe** (NEW): What if X guard existed?
7. **cascade_probe** (NEW): What protocols affected?

Probe failures produce UNKNOWN or EXPANSION_REQUIRED, not false positives.

### FindingRiskMetadata (output.py)

Risk metadata attached to findings in orchestrator reports:

```python
@dataclass
class FindingRiskMetadata:
    economic_risk: float  # Total score 0-10
    attack_ev_score: float  # Game-theoretic multiplier
    loss_amplification: float  # Causal amplification
    systemic_risk: float  # Cross-protocol factor
    risk_breakdown: Dict  # Full breakdown
    priority: str  # critical/high/medium/low/minimal
```
