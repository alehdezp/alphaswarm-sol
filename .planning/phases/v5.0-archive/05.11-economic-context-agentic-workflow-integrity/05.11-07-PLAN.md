  <name>Task 1: Implement causal exploitation graph builder</name>
  <files>src/alphaswarm_sol/economics/causal/__init__.py, src/alphaswarm_sol/economics/causal/exploitation_graph.py</files>
  <action>
Build the causal exploitation graph infrastructure:

1. CausalNode types:
   - RootCause: the fundamental vulnerability (missing_access_control, oracle_manipulation)
   - ExploitStep: intermediate exploitation steps (unauthorized_call, state_manipulation)
   - FinancialLoss: the financial outcome (treasury_drain, token_theft, liquidation_abuse)
   - Amplifier: factors that increase loss (flash_loan, leverage, liquidity_depth)

2. CausalEdge types (extending schema.py types):
   - CAUSES: direct causation (root -> exploit step)
   - ENABLES: prerequisite (state A enables exploit B)
   - AMPLIFIES: multiplies loss (flash loan amplifies attack)
   - BLOCKS: mitigation breaks chain (guard prevents step)

3. CausalExploitationGraph class:
   - add_node(node: CausalNode)
   - add_edge(source, target, edge_type, probability, evidence_refs)
   - build_from_vulnerability(vulnerability_id) -> CausalExploitationGraph
   - get_all_paths(root_cause, financial_loss) -> List[CausalPath]

4. CausalPath dataclass:
   - nodes: List[CausalNode] (ordered path)
   - edges: List[CausalEdge] (connecting edges)
   - cumulative_probability: product of edge probabilities
   - amplification_factor: product of AMPLIFIES multipliers

5. Create economics/causal/__init__.py with exports.

<task type="auto">
  <name>Task 2: Implement counterfactual query engine</name>
  <files>src/alphaswarm_sol/economics/causal/counterfactual_engine.py, src/alphaswarm_sol/economics/causal/loss_estimation.py</files>
  <action>
Build counterfactual reasoning capabilities:

1. CounterfactualEngine class (counterfactual_engine.py):
   - query(graph, counterfactual) -> CounterfactualResult
   - Supports queries like:
     * "If timelock_on_setAdmin existed, would chain break?"
     * "If reentrancy_guard existed, what paths remain?"
     * "If oracle had TWAP, what amplification is possible?"

2. CounterfactualQuery dataclass:
   - condition: the hypothetical change (guard_exists, param_changed)
   - target_edge: which edge to evaluate blocking
   - scope: which paths to analyze

3. CounterfactualResult dataclass:
   - chain_blocked: bool (does counterfactual break chain?)
   - loss_reduction: percentage (how much loss is prevented?)
   - remaining_paths: List[CausalPath] (alternative attack paths)
   - minimal_fix: the smallest change that blocks all paths

4. LossEstimator class (loss_estimation.py):
   - estimate_loss(causal_path, protocol_state) -> LossEstimate
   - Account for: TVL exposure, rate limits, caps, cooldowns
   - Apply amplification factors from AMPLIFIES edges
   - estimate_counterfactual_loss(path, counterfactual) -> LossEstimate
  </action>
