# Phase 05.11 Plan 09: Automated Invariant Mining and Synthesis (AIMS) Summary

**One-liner:** Trace2Inv-style invariant mining from transaction traces with require() synthesis and exploit validation.

## What Was Built

Implemented the Automated Invariant Mining and Synthesis (AIMS) system that discovers protocol invariants from transaction traces and generates defensive `require()` statements that would have prevented past exploits.

### Components Created

1. **InvariantMiner** (`miner.py`, ~560 LOC)
   - Mine invariants from transaction traces
   - Trace2Inv-style pattern matching
   - Statistical confidence scoring
   - Counterexample tracking
   - State transition inference

2. **Pattern Library** (`patterns.py`, ~720 LOC)
   - 10 pattern types: MappingUpperBound, MappingLowerBound, CallValueUpperBound, StateTransitionConstraint, VariableRelation, SumInvariant, MonotonicProperty, RatioBound, DifferenceBound, Equality
   - Each pattern: check_trace(), extract_expression(), to_require_condition()
   - Common patterns predefined for ERC20-style tokens

3. **InvariantSynthesizer** (`synthesis.py`, ~600 LOC)
   - RequireStatement synthesis with gas estimation
   - Exploit validation against known exploit database
   - Discrepancy detection (mined vs declared)
   - InvariantRegistry integration

4. **Tests** (`test_invariant_mining.py`, 39 tests)
   - Pattern validation tests
   - Mining tests with ERC20 and state machine traces
   - Synthesis and exploit validation tests
   - Registry integration tests

5. **Documentation** (`automated-invariant-mining.md`, ~600 lines)
   - Complete API reference
   - Pattern type explanations with examples
   - Mining algorithm walkthrough
   - Configuration options

## Key Design Decisions

### AIMS-01: Pattern Library
10 Trace2Inv-style pattern types covering:
- Balance/mapping bounds (upper, lower)
- Conservation laws (sum invariants)
- Monotonicity (nonce, counters, timestamps)
- State machines (valid transitions)
- Relations (AMM constant product, ratios)

### AIMS-02: Confidence Scoring
Statistical confidence based on:
- Base confidence from pattern type (e.g., non-negative uint = 0.95)
- Support ratio (supporting traces / total)
- Counterexample decay (each counterexample reduces confidence)
- Sample size boost (100+ traces = 10% boost)

Default thresholds:
- min_trace_coverage: 0.9 (90%)
- counterexample_tolerance: 0.0 (none)
- min_confidence: 0.7

### AIMS-03: Require Synthesis
Position determination by pattern type:
- **Pre**: CallValueUpperBound, StateTransition
- **Post**: SumInvariant, MonotonicProperty, VariableRelation
- **Both**: MappingBounds

Gas estimation includes SLOAD (~2100), comparisons (~3), arithmetic (~5).

### AIMS-04: Discrepancy Detection
Four discrepancy types with severity:
- MISSING_DECLARED (high): Mined but not in docs - safety gap
- MISSING_MINED (medium): Declared but not mined - untested
- CONFLICT (critical): Contradictory invariants
- CONFIDENCE_GAP (high): Declared high, mined low
