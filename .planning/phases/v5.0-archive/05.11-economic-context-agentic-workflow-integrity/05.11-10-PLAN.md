  <files>src/alphaswarm_sol/economics/composability/__init__.py, src/alphaswarm_sol/economics/composability/protocol_graph.py</files>
  <action>
Build the DeFi protocol dependency infrastructure:

1. ProtocolNode dataclass:
   - protocol_id: unique identifier (e.g., "aave-v3", "compound-v3")
   - tvl: Decimal (total value locked)
   - category: LENDING | DEX | ORACLE | BRIDGE | DERIVATIVE
   - chains: List[str] (deployed chains)
   - governance: GovernanceInfo (timelock, multisig, dao)

2. DependencyEdge dataclass:
   - source: protocol_id
   - target: protocol_id
   - dependency_type: ORACLE | LIQUIDITY | COLLATERAL | GOVERNANCE | BRIDGE
   - criticality: float (1-10)
   - failure_propagation_time: timedelta
   - tvl_at_risk: Decimal (if source fails, target loses this)

3. ProtocolDependencyGraph class:
   - add_protocol(node: ProtocolNode)
   - add_dependency(edge: DependencyEdge)
   - build_from_passports(passports: List[ContractPassport]) -> graph
   - get_dependencies(protocol_id) -> List[DependencyEdge]
   - get_dependents(protocol_id) -> List[DependencyEdge] (who depends on this)

4. Graph analysis methods:
   - compute_centrality() -> Dict[protocol_id, float] (eigenvector centrality)
   - find_critical_paths() -> List[CriticalPath] (high-impact failure chains)
   - detect_circular_dependencies() -> List[Cycle]

5. Seed data for major protocols:
   - Oracle dependencies: Chainlink feeds for Aave, Compound, Synthetix
   - Liquidity dependencies: Uniswap pools for many protocols
   - Collateral dependencies: stETH, wstETH, rETH cross-protocol

6. Create economics/composability/__init__.py with exports.

<task type="auto">
  <name>Task 2: Implement cascade failure simulator</name>
  <files>src/alphaswarm_sol/economics/composability/cascade_simulator.py, src/alphaswarm_sol/economics/composability/systemic_scorer.py</files>
  <action>
Build cascade simulation and systemic scoring:

1. CascadeSimulator class (cascade_simulator.py):
   - simulate_failure(trigger_protocol, failure_type) -> CascadeResult
   - Model failure propagation through dependency edges
   - Track: affected protocols, timeline, TVL impacted

2. CascadeScenario dataclass:
   - trigger: protocol_id + failure_type
   - failure_type: ORACLE_MANIPULATION | EXPLOIT | GOVERNANCE_ATTACK | LIQUIDITY_CRISIS
   - market_conditions: Dict (ETH price, gas, etc.)

3. CascadeResult dataclass:
   - trigger_protocol: protocol_id
   - affected_protocols: List[protocol_id] (ordered by impact time)
   - total_tvl_at_risk: Decimal
   - propagation_timeline: List[(time, protocol, status)]
   - critical_bottlenecks: List[protocol_id] (single points of failure)

4. Cascade propagation rules:
   - ORACLE failure: affects all dependents within oracle staleness window
   - LIQUIDITY crisis: affects dependents with > 30% liquidity from source
   - EXPLOIT: affects collateral dependents immediately
   - Propagation stops when criticality < threshold

5. SystemicScorer class (systemic_scorer.py):
   - compute_systemic_score(protocol_id) -> SystemicRiskScore
   - Combines: centrality, cascade_tvl_at_risk, dependency_count
   - score_finding(finding, protocol_graph) -> finding with systemic_risk

6. SystemicRiskScore dataclass:
   - score: float (0-10)
   - centrality_component: float
   - cascade_component: float
   - dependency_component: float
   - critical_paths: List[CriticalPath] (why score is high)
  </action>
