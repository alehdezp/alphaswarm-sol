# Phase 05.11 Plan 10: Cross-Protocol Composability Risk Model Summary

**One-liner:** DeFi ecosystem dependency graph with cascade simulation, systemic scoring, and GATE-filtered multi-protocol attack discovery.

## What Was Built

### 1. Protocol Dependency Graph (`protocol_graph.py`)
- **ProtocolNode**: Protocol metadata (TVL, category, chains, governance)
- **DependencyEdge**: Typed dependencies (ORACLE, LIQUIDITY, COLLATERAL, GOVERNANCE, BRIDGE)
- **ProtocolDependencyGraph**: Full graph with centrality analysis
- **Seed data**: 10+ major DeFi protocols (Chainlink, Aave, Compound, Uniswap, Curve, Lido, etc.)
- **Features**: Eigenvector centrality, critical path detection, cycle detection

### 2. Cascade Failure Simulator (`cascade_simulator.py`)
- **CascadeSimulator**: Propagates failures through dependency edges
- **FailureType**: 7 types (oracle manipulation, exploit, governance attack, liquidity crisis, bridge hack, depeg, insolvency)
- **MarketConditions**: Stressed vs normal conditions affect cascade speed
- **CascadeResult**: Full timeline, TVL at risk, bottleneck identification
- **Propagation rules**: Type-specific (oracle fast, governance slow, etc.)

### 3. Systemic Risk Scorer (`systemic_scorer.py`)
- **SystemicScorer**: Combines centrality + cascade + dependencies into 0-10 score
- **CrossProtocolAttackPath**: Multi-step attacks with economic viability
- **GATE integration**: Filters paths by EV > 0 (reduces ~30% false positives)
- **score_finding()**: Enrich findings with systemic risk metadata

## Architecture

```
ProtocolDependencyGraph
    |
    +-- ProtocolNode (TVL, category, governance)
    +-- DependencyEdge (type, criticality, propagation time)
    +-- compute_centrality() -> eigenvector centrality
    +-- find_critical_paths() -> high-impact chains
    |
    v
CascadeSimulator
    |
    +-- simulate_failure(trigger, failure_type, market_conditions)
    +-- Propagation rules by dependency type
    +-- CascadeResult (timeline, TVL at risk, bottlenecks)
    |
    v
SystemicScorer
    |
    +-- compute_systemic_score() -> 0-10 with breakdown
    +-- discover_cross_protocol_attacks() -> List[CrossProtocolAttackPath]
    +-- GATE integration for economic viability filtering
```

## Decisions Made

1. **Centrality Algorithm**: Eigenvector centrality via power iteration (standard for network importance)
2. **Cascade Model**: BFS with criticality threshold and depth limiting
3. **Propagation Times**: Type-specific (oracle 30min, liquidity 2hr, collateral 15min, governance 24hr)
4. **Score Composition**: Centrality (0-3) + Cascade (0-4) + Dependency (0-3) = 0-10
5. **GATE Integration**: Filter paths where EV <= 0 as LOW_PRIORITY
