**One-liner:** Token savings tests document TOON's 25-40% reduction for uniform arrays and its limitations with nested structures; CLAUDE.md updated with format guidelines.

## What Was Built

### 1. Token Savings Validation Tests (tests/test_toon_token_savings.py)

**11 tests across 7 test classes documenting TOON behavior:**

| Class | Tests | Purpose |
|-------|-------|---------|
| `TestUniformArraySavings` | 3 | Validate >25% savings for uniform arrays |
| `TestSimpleMetadataBehavior` | 1 | Document behavior with flat metadata |

**Key findings validated:**
- Uniform arrays: 26-38% token reduction
- Large arrays (100 items): 30%+ reduction
- Mixed schema arrays: May INCREASE tokens (-24% in tests)
- JSON-inside-TOON envelope: <15% overhead (preserves benefits)

## Token Savings Analysis

From test output (using tiktoken cl100k_base):

| Data Structure | JSON Tokens | TOON Tokens | Reduction |
|----------------|-------------|-------------|-----------|
| Evidence (20 items) | 450 | 303 | 32.7% |
| Edges (50 items) | 690 | 507 | 26.5% |
| Evidence (100 items) | 2012 | 1314 | 34.7% |
| Mixed nodes (3 items) | 90 | 112 | -24.4% (increase) |
| Real graph (NoAccessGate) | 5097 | 5809 | -14.0% (increase) |
| JSON-inside-TOON envelope | 5097 | 5183 | 1.7% overhead |

**Key insight:** TOON excels for uniform arrays but struggles with heterogeneous/nested data. The JSON-inside-TOON envelope approach in store.py is the correct solution.
