# Phase 5.6.1 Plan 02: GraphStore TOON Integration Summary

**One-liner:** GraphStore now defaults to TOON format output with 24% token reduction while maintaining JSON fallback for full data preservation; CLI supports --format flag.

## What Was Built

### 1. GraphStore TOON Support (src/alphaswarm_sol/kg/store.py)
- **save() method changes:**
  - Added `format: Literal["toon", "json"] = "toon"` parameter
  - Default output is now `.toon` extension
  - Uses `toon_dumps()` for TOON, `json.dumps()` for JSON
  - Timestamped files use correct extension per format
  - Added comprehensive docstring

- **load() method changes:**
  - Auto-detects format from file extension
  - Tries `graph.toon` first, falls back to `graph.json`
  - Uses `toon_loads()` for .toon files, `json.loads()` for .json
  - Added comprehensive docstring

### 2. CLI Format Flag (src/alphaswarm_sol/cli/main.py)
- Added `--format/-f` option to `build-kg` command
- Default value: `"toon"` (LLM-optimized)
- Alternative: `"json"` (legacy, full preservation)
- Validation: rejects invalid format values
- Output message shows format used

### 3. Comprehensive Test Suite (tests/test_store.py)
- **19 tests** across 6 test classes (329 lines):
  - `TestGraphStoreToonFormat` - default format, explicit formats, round-trips
  - `TestGraphStoreAutoDetect` - TOON preference, JSON fallback
  - `TestGraphStoreOverwrite` - timestamped files, overwrite behavior
  - `TestGraphStoreToonContent` - format version, timestamp, size comparison
  - `TestGraphStoreNodeEdgePreservation` - JSON full preservation, TOON structure

### Documented Limitations

**TOON Nested Property Flattening**
- **Discovery:** During Task 3 test development
- **Issue:** TOON library flattens nested dicts inside arrays (tabular encoding)
- **Example:** `properties: {visibility: public}` becomes flat fields
- **Resolution:** Documented in tests; JSON format recommended for full preservation
- **Files affected:** tests/test_store.py (documented in TestGraphStoreNodeEdgePreservation docstring)

Real-world test with NoAccessGate.sol:
