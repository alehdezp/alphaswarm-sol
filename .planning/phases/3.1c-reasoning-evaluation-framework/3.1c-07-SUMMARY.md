## Decisions Made

- **Debate threshold at 10pt**: Plan specified >10pt disagreements trigger debate (IMP-14), not the 15pt in original model docstring. Updated model docstring and implementation.
- **Resolution literals changed**: `average`/`debate` renamed to `consensus`/`escalated` for clearer semantics. `consensus` = scores converged within threshold after rebuttal. `escalated` = unresolved, uses lower score with `unreliable=True`.
- **Session isolation via UUID**: Each dual evaluation generates fresh uuid-based session IDs (`eval-a-{hex}`, `eval-b-{hex}`) ensuring no session state leakage between evaluators.
- **Escalated uses conservative scoring**: When debate fails to resolve, the lower of the two scores is used with `unreliable=True` -- this ensures overestimation bias is avoided.
