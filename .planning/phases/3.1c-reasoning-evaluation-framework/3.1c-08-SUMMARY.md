## Decisions Made

- **Staleness check dual approach**: Uses `staleness_marker.json` file for fixture testability alongside file mtime for live sessions. Fixtures in git always have recent mtime, so the marker provides a testable staleness signal.
- **Post-scoring mode filtering**: Mode-aware filtering runs as a post-scoring pass that sets `applicable=False` on unavailable dimensions, rather than preventing grader invocation. This preserves full scoring data while ensuring `effective_score()` excludes inapplicable dimensions.
- **AND gate in run() not BaselineManager**: The three-condition baseline gate is implemented in the runner's `run()` method (not delegated to BaselineManager) because the runner owns the SessionValidityManifest and status information that BaselineManager doesn't have.
- **fcntl.flock for parallel writes**: BaselineManager writes use `fcntl.flock(LOCK_EX)` for safe parallel headless session writes, with `max_parallel_headless=1` default.
