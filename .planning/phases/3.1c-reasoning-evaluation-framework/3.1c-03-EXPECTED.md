# Expected Results: 3.1c-03 — ObservationParser Adapter

**Plan:** 3.1c-03
**Wave:** 2
**Depends on:** 3.1c-01 (models), 3.1c-02 (TranscriptParser + hooks)
**Files modified:** `tests/workflow_harness/lib/observation_parser.py`, `tests/evaluation/test_observation_parser.py`

---

## Fundamental Deliverable

This plan delivers a refactored `ObservationParser` as a thin adapter (~40 LOC) over `TranscriptParser`, with `tool_use_id`-based tool-call pairing and session-scoped observation loading, so that Plans 04, 07, and 08 can consume parsed observation data from a single authoritative data source without inheriting the LIFO pairing bug or cross-session contamination risk. Without it, downstream plans consume either fabricated observation data or data paired by the wrong mechanism, making every score computed by the graph value scorer and reasoning evaluator suspect. Success means `ObservationParser` delegates all extraction to `TranscriptParser.to_observation_summary()`, its entire implementation fits in ~40 LOC, real transcripts from Plan 02 parse with zero `parse_errors`, and tool calls are paired by `tool_use_id` with LIFO only as a fallback. Failure means the class still implements its own JSONL extraction logic, reads all sessions indiscriminately, or produces pairing errors on sessions with parallel tool calls.

## Drift Detection

### OK — LIFO fallback kept with different warning text

**Scenario:** Executor implements `tool_use_id` pairing as primary but retains LIFO fallback. The warning message says "tool_use_id absent, using positional fallback" instead of the plan's exact phrasing "LIFO fallback."

**Why OK:** The must_have says "replaced with tool_use_id-based pairing." The plan task says "Fall back to LIFO only when tool_use_id absent (with warning, not crash)." A graceful fallback is explicitly required. Warning text variation is immaterial.

**Action:** Verify the fallback triggers only when `tool_use_id` is absent, not as the default path.

---

### OK — Adapter is 45 LOC instead of exactly 40

**Scenario:** The refactored `observation_parser.py` has 46 non-blank, non-comment lines due to the staleness guard, precompact snapshot handler, and tool failure handler each adding a few lines beyond the 40 LOC estimate.

**Why OK:** The ~40 LOC constraint is an architectural signal (thin adapter, not a full extractor), not a hard line count requirement. 45-50 LOC implementing the full feature set is aligned. 120 LOC would indicate bloat.

**Action:** Confirm no inline extraction logic exists. Count is a proxy for "thin adapter," not a literal gate.

---

### DRIFT — session_id filter implemented as post-load filter, not path selection

**Scenario:** The constructor accepts `session_id` and `_load()` reads all `*.jsonl` files, then filters records to only those matching the session ID. Tests pass because the filtering is correct.

**Why DRIFT:** The must_have explicitly says "O(1) lookup." Iterating all files and filtering is O(n) in the number of session files. The plan task specifies: "reads only `{session_id}.jsonl`." The performance contract is violated even if behavior is correct.

**Action:** Request correction. The fix is to construct `{session_id}.jsonl` path and open only that file. Filtering approach is a misread of the spec.

---

### DRIFT — GroundTruthAdapter validation tests use only synthetic fixture, no real transcript

**Scenario:** `test_observation_parser.py` validates the adapter with hand-crafted JSONL fixtures that are structured to match expected schema. No real transcript files from Plan 02 are loaded.

**Why DRIFT:** The must_have truth "Parse real transcripts with zero errors" is not met. The plan explicitly calls out validation on "real transcripts from Plan 02." If Plan 02's hook output has schema variations, hand-crafted fixtures will not catch them.

**Action:** Request that at least one test loads a real `*.jsonl` file from `tests/workflow_harness/fixtures/real_sessions/` and asserts `parse_errors == 0`.

---

### STOP — ObservationParser still implements independent JSONL extraction

**Scenario:** The executor adds `to_observation_summary()` delegation as an optional or secondary path, but the class still contains its own record-level iteration, field mapping, and event-type dispatch. The file is 120+ LOC. `TranscriptParser` is imported but only called in a `if use_transcript_parser:` branch that is False by default.

**Why STOP:** The Fundamental Deliverable is "thin adapter over TranscriptParser." An adapter that keeps its own extraction engine is not an adapter — it is a parallel implementation. Plans 04, 07, 08 would continue consuming data from two divergent sources. The architecture decision (P13-IMP-12) — that ObservationParser produces overlapping but inferior data to TranscriptParser — is explicitly unresolved by this approach.

**Action:** Escalate. Do not proceed. The entire purpose of this plan is to eliminate the parallel extraction path, not augment it.

## Reviewer Guidance

**Priority 1 — Verify the adapter is genuinely thin.** Open `observation_parser.py` and check for `json.loads()`, direct record field iteration, and event-type dispatch logic in the class body. Any of these signals that the plan's primary goal (eliminate the parallel extraction engine) was not achieved. The delegation call `TranscriptParser(...).to_observation_summary()` must be the extraction path, not a supplement.

**Priority 2 — Verify tool_use_id pairing is the primary path, not the fallback.** The old code at line 161 used a LIFO stack. Check that no list-used-as-stack exists as the default pairing logic. LIFO must be the exception (absent `tool_use_id`), not the rule.

**Priority 3 — Verify session_id produces O(1) file access.** Read `_load()` and confirm it constructs a single `{session_id}.jsonl` path rather than globbing and filtering. This is the difference between a session-safe implementation and one that silently reads all sessions.

**Priority 4 — Verify real transcript validation tests exist and assert parse_errors == 0.** Synthetic fixtures can be structured to match expectations regardless of bugs. The must_have is only met if at least one test uses an actual Plan 02 transcript file and asserts zero parse errors.

**Priority 5 — Verify all four ~4 call sites were updated.** Run `grep -r "ObservationParser(" tests/ src/` mentally (or actually) against the diff. Any call site that omits `session_id` after the constructor change is a missed atomic migration (P13-ADV-6-01).
