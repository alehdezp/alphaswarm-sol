## Decisions Made

1. **Literal type for InfrastructureFailureType** — Pydantic serializes Literal types cleanly to JSON; Enum requires custom handling. All consumers use string matching anyway.

2. **CalibrationConfig/Result as separate models** — Plan 12 needs to verify that calibration inputs match across runs while outputs vary. Merging them would require filtering on every comparison.

3. **behavioral_signature as a string field on DimensionScore** — Simple word-set overlap is sufficient for disambiguation_check(). Plan 06 may refine this, but the schema pre-condition is in place.

4. **Self-evaluation guard as model_validator** — Rejects at construction time, not at evaluation runtime. This prevents EvaluationConfig from ever being in an invalid state.

5. **EvaluatorDisagreement.delta as auto-computed model_validator** — Ensures delta is always consistent with a_score and b_score without manual calculation.

6. **SKILL.md stub intentionally incomplete** — 6 [TO-FILL: Plan 09] markers preserve the cross-plan contract. Plan 09 fills budget, kill-file, circuit breaker, sub-wave ordering, pre-flight activation, and intra-stage failure policy.
