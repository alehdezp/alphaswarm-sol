## Post-Review Corrections (63f537bd)

Three alignment issues identified by reviewer and corrected:

### Correction 1: Wire PostToolUseFailure compliance credit to affect ordering

**Issue:** The sentinel `"attempted"` was appended to `bskg_timestamps` but the ordering logic uses `bskg_indices`. With `bskg_indices` empty, `earliest_bskg_idx = inf`, so no Read call could satisfy the ordering check. The compliance credit had zero effect.

Additionally, the early return `if not bskg_events: return False, 0.0` at line 297-298 exited before the PostToolUseFailure handling code was reached.

**Fix:** (a) Moved PostToolUseFailure handling before the `bskg_events` early-exit check so failed queries are processed even when `bskg_query_events` is empty. (b) Added sentinel index `0` to `bskg_indices` (not just `bskg_timestamps`) so the ordering logic at line 327 computes `earliest_bskg_idx = 0` instead of `inf`.

**EXPECTED.md criterion:** 10 -- "Such events increment graph-first compliance counters (agent attempted a query)."

### Correction 2: Wire `_get_result_utilization_penalty()` into `score()`

**Issue:** `_get_result_utilization_penalty()` was defined but never called from `score()` or any helper. The penalty had no effect on the final score.

**Fix:** Added call to `_get_result_utilization_penalty(obs_summary)` in `score()` after citation rate computation. The penalty is subtracted from `citation_rate` before dimension building. Added integration assertion to `test_result_utilization_penalty` verifying the penalty is reflected in `result.citation_rate`.

**EXPECTED.md criterion:** 10 (improvement) -- "Such events reduce `result_utilization` score (no result was returned to utilize)."

### Correction 3: Fix `test_failed_query_counts_for_compliance` behavioral assertion

**Issue:** Test asserted only `isinstance(result, GraphValueScore)` -- an existence-only check with no signal about whether the compliance credit path works.

**Fix:** Rewrote test to assert `result.graph_first_compliant is True` and `result.graph_first_score > 0.0` when a failed `alphaswarm query` precedes all `Read` calls. This gives the test behavioral meaning per EXPECTED.md: "A unit test asserts both behaviors for a transcript containing a failed query."
