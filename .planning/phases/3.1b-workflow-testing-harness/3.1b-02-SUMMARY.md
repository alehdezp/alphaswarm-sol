## Decisions Made
- ToolCall new fields all have defaults (None/dict()) so existing constructors `ToolCall(tool_name=..., tool_input=..., tool_result=..., index=...)` continue working unchanged
- BSKGQuery citation heuristic: 20-char substring matching on first 200 chars of result_snippet against subsequent Write/SendMessage tool_input. Documented as best-effort with expected false negatives when agents paraphrase
- `get_messages_between()` raises IndexError on out-of-bounds rather than silently clamping, matching the API contract specification
- OutputCollector.collect() signature uses explicit keyword args (not claude_result from API contract) for clarity and testability without needing mock ClaudeCodeResult objects
