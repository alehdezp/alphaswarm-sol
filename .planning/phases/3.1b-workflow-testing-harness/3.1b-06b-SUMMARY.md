## Decisions Made
1. **Manual creation over corpus-generator agent:** The corpus-generator agent has 120s timeout limitations. Creating 18 projects with 5-10 patterns each required manual crafting for quality and diversity.
2. **Two-pass ground truth enrichment:** Initial generation used generic pattern IDs (e.g., "missing-access-control"). Two Python enrichment scripts mapped these to specific catalog entries (auth-001, auth-002, etc.) and added underrepresented category patterns.
3. **Jujutsu standalone init:** Used `jj init` (not `jj git init --colocate`) to create `.jj` without `.git`, preventing nested git repo conflicts with the parent examples/testing repo.
4. **Deferred solc compilation:** Build verification scripts exist but solc is not installed on the development machine. Scripts will work when solc is available.
