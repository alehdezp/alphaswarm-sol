## Decisions Made
1. EvaluationConfig.contract is optional (None default) and accepts any string -- 3.1c validates existence at evaluation time, not parse time. This avoids circular dependencies.
2. JSON Schema validation is advisory (warn, never block) to avoid breakage during rapid iteration.
3. ModelGrader shells out to `claude --print -p` consistent with the project's execution model (no direct API calls).
4. Unified GraderConfig model covers both code and model grader types with shared optional fields, rather than separate discriminated union types.
5. Score range: ModelGrader returns 0-100 integer scores which are stored as-is in GradeResult.score (int). No normalization — 0-100 scale used throughout, consistent with 3.1c's evaluation scale.
