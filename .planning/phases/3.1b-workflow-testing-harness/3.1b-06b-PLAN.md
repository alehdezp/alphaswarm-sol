Use the corpus-generator agent (from 3.1b-06a) and generate_project.py to produce the
seed corpus. Follow this composition plan:

**Batch 1 — Workflow-testing projects (3 projects, Category: None/vanilla):**
- Focus on pipeline testing, not adversarial detection
- 5-7 mixed vulnerability categories per project
- Simple, well-named contracts for basic harness validation
- Projects: `workflow-basic-defi-01`, `workflow-token-staking-02`, `workflow-governance-03`

**Batch 2 — Tier A: Multi-pattern DeFi (5 projects, Category A):**
- 5-8 vulnerabilities per project combining access-control + reentrancy + oracle + logic + dos
- Name obfuscation applied (misleading function/variable names)
- Projects: `defi-lending-protocol-01`, `dex-amm-pool-02`, `yield-aggregator-03`,
  `flash-loan-vault-04`, `staking-rewards-05`

**Batch 3 — Tier A+B: Complex protocols (3 projects, Category A+B):**
- 7-10 vulnerabilities per project with cross-contract + temporal + state machine patterns
- Both name AND protocol complexity obfuscation
- Projects: `cross-chain-bridge-01`, `governance-timelock-02`, `multi-token-vault-03`

**Batch 4 — Tier B: Cross-contract (2 projects, Category B):**
- 5-7 vulnerabilities split across multiple contracts, proxy patterns, inheritance
- Protocol complexity obfuscation only (names can be obvious)
- Projects: `proxy-delegate-system-01`, `inherited-token-hierarchy-02`

**Batch 5 — False-positive controls (3 projects, Category C):**
- 0 real vulnerabilities (safe code with dangerous-looking patterns)
- Honeypot inversion techniques applied
- Projects: `safe-complex-governance`, `safe-defi-lending`, `safe-token-mechanics`

**Batch 6 — Adversarial stress-tests (2 projects, Category A+B+C):**
- 8-10 patterns per project, all obfuscated, maximum difficulty
- Projects: `stress-test-max-obfuscation-01`, `stress-test-protocol-maze-02`

**For each project, invoke:**
```bash
python examples/testing/scripts/generate_project.py \
    --patterns {comma-separated-pattern-ids} \
    --category {A|B|C|AB|ABC} --tier {1|2|3} \
    --output examples/testing/corpus/{project-name}/ --verify
```

**After generation, for each project verify:**
1. `solc --bin contracts/*.sol` compiles without errors (both vulnerable and safe variants)
2. `ground-truth.yaml` exists with 5-10 findings (or 0 for FP controls)
3. `project-manifest.yaml` exists with correct metadata
4. `build-verification.sh` runs successfully
5. Initialize Jujutsu: `cd {project-dir} && jj git init && jj commit -m "initial seed corpus"`

**Coverage validation after all projects:**
```bash
python examples/testing/scripts/audit-vulndocs-coverage.py --corpus examples/testing/corpus/
```
Must show:
- All 5 major categories (access-control, reentrancy, oracle, dos, logic) have >= 2 projects
- All 3 adversarial categories (A, B, C) have >= 3 projects
- >= 3 false-positive control projects
- >= 75 unique patterns across all projects
- Every non-FP project has >= 5 distinct vulnerability patterns

**Quality spot-check:** Manually review at least 3 projects:
- Obfuscation is non-trivial (can't immediately spot vulnerability from names)
- Multi-pattern projects have realistic inter-function data flow
- Safe variants are genuinely safe (not commented-out vulnerabilities)
- Project structure looks like real DeFi codebase
