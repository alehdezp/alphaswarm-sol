**Part B — Infrastructure smoke test:**

Create `tests/workflow_harness/test_infrastructure_smoke.py` with tests that exercise
the FULL data flow through all infrastructure components:

```python
"""End-to-end infrastructure smoke test for Phase 3.1b.

This test validates that all infrastructure components built in 3.1b
work together. It exercises: scenario loading, workspace setup (with Jujutsu),
hooks, transcript parsing, output collection, and grading.

Works WITHOUT Companion. Uses native CLI or synthetic data where real CLI
invocation is impractical in CI.
"""
```

**Test structure (at least 6 tests):**

1. `test_scenario_loading_roundtrip` — Load health-check-basic.yaml, verify all fields
   parsed correctly including evaluation_guidance, graders, steps.

2. `test_transcript_parser_with_bskg_extraction` — Parse a synthetic transcript JSONL
   containing alphaswarm Bash commands. Verify BSKGQuery extraction works, ToolCall
   has timestamp, graph_citation_rate() returns a float.

3. `test_output_collector_produces_collected_output` — Feed parsed transcript into
   OutputCollector.collect(). Verify CollectedOutput has all fields populated.
   Verify summary() produces readable text.

4. `test_code_grader_scores_output` — Run CodeGrader on CollectedOutput with
   expected string matches. Verify GradeResult has correct passed/score/reason.

5. `test_hook_infrastructure_is_ready` — Verify WorkspaceManager.install_hooks()
   exists and accepts a list. Verify .vrs/observations/ directory convention.
   (Does NOT require running a real hook — just verifies the plumbing.)

6. `test_jujutsu_workspace_methods_exist` — Verify WorkspaceManager has
   create_workspace, forget_workspace, rollback, list_workspaces, snapshot_operation.
   (Mock subprocess to avoid requiring real jj in CI.)

7. `test_corpus_project_validity` — Pick one corpus project from
   examples/testing/corpus/, verify ground-truth.yaml exists and parses,
   project-manifest.yaml exists, contracts directory exists with .sol files.

8. `test_full_pipeline_synthetic` — Full pipeline with synthetic data:
   Load scenario -> create synthetic transcript -> parse -> collect ->
   grade -> assert GradeResult.passed. This is the "all components together" test.

**IMPORTANT:** Use synthetic/mock data for CLI-dependent steps. The smoke test must
be runnable without a live Claude session. The human verification checkpoint (Task 2)
tests with real CLI execution.

**Markers:** Use `@pytest.mark.smoke` for all tests in this file. Add to pytest markers
in conftest.py if not already registered.
