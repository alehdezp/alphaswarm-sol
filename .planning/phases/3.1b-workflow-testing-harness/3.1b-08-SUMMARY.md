## Decisions Made

1. **ToolCall backward compatibility via defaults** - New fields (timestamp, duration_ms, content_block) all have None/dict defaults so existing constructors don't break.
2. **BSKGQuery result_snippet length** - 2000 chars (4x ToolCall's 500) to preserve enough graph output for citation detection.
3. **cited_in_conclusion as heuristic** - Substring matching on first 200 chars of result. False negatives expected when agents paraphrase. 3.1c-04 treats this as signal, not ground truth.
4. **Dual hook format** - HookConfig dataclass (preferred) alongside existing tuple format during transition. Both accepted by install_hooks().
5. **Scoring scale resolution** - 0-100 integer is canonical. Any 3.1b 0.0-1.0 float usage converts at the 3.1c boundary.
6. **Separate Pydantic/dataclass forms** - EvaluationGuidanceConfig (Pydantic, in scenario DSL) and EvaluationGuidance (dataclass, in output_collector) serve different purposes. Conversion happens at the boundary.
