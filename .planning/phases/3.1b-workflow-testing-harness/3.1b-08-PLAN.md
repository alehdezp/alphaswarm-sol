- `AgentObservation` dataclass: `agent_id`, `agent_type`, `transcript`, `bskg_queries`, `messages_sent`, `messages_received`

- `CollectedOutput` dataclass: all fields including `failure_notes: str`
- `EvaluationGuidance` dataclass: `reasoning_questions: list[str]`, `hooks_if_failed: list[str]`, `run_gvs: bool`, `run_reasoning: bool`

**hooks-and-workspace.md** — Full signatures for:
- `WorkspaceManager.install_hooks(hook_scripts: list[HookConfig])` signature, where `HookConfig` = `(event_type: str, script_path: Path, timeout: int)`
- `_ensure_hook()` deduplication logic
- `.vrs/observations/` directory convention: file naming (`{session_id}.jsonl`), JSONL record schema
- Jujutsu workspace API: `create_workspace(name: str) -> Path`, `forget_workspace(name: str)`, `rollback(op_id: str)`, `list_workspaces() -> list[str]`
- SendMessage capture format: `{agent_id, recipient, content, timestamp, message_type}`

**scenario-dsl.md** — Full schemas for:
- `evaluation:` block: `contract: str`, `run_gvs: bool`, `run_reasoning: bool`
- `evaluation_guidance:` block: `reasoning_questions: list[str]`, `hooks_if_failed: list[str]`
- `post_run_hooks:` field: `list[str]` (hook script paths)
- `modes: list[Literal["single", "team"]]`
- `team_config:` block: `roles`, `model`, `team_ground_truth`
- Grader types: `code` (string match, regex, schema) and `model` (AI judge)
