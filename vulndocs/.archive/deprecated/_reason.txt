ARCHIVED: Patterns requiring semantic/LLM analysis beyond tier_a capability
==========================================================================

These 39 patterns reference properties that require deep semantic analysis,
cross-contract reasoning, or protocol-specific knowledge that cannot be
implemented as deterministic builder properties. They are NOT suitable for
tier_a matching.

Every pattern here has ALL its tier_a property conditions referencing
non-existent builder properties, meaning the pattern ALWAYS returns zero
findings -- it silently pretends to check for something it cannot detect.

These patterns should be redesigned as:
- Tier B (risk-tag-based) patterns with LLM-assisted classification
- Tier C (semantic-label-based) patterns
- Full agent investigation beads (attacker/defender/verifier)

They should NOT be restored as tier_a patterns because the underlying
properties cannot be deterministically computed from Solidity AST alone.

Moved: 2026-02-08 (Phase 2.1-04 Pattern Triage)

- Governance semantics: vote-without-snapshot, quorum checks, timelock checks,
