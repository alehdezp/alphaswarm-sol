# Research Plan: Project-to-Publication Pipeline

## Main Question
How to streamline turning project artifacts (code, planning docs, research, experiments) into LinkedIn posts and blog posts — investigating Claude Code skills, plugins, documentation patterns, and automation tools.

## Subtopics

### 1. Claude Code Skills for Content Generation
- Existing community skills for writing/publishing
- How to build a custom `/publish` or `/blog-post` skill
- Skills that mine git history, CLAUDE.md, or planning docs into summaries
- Agent-based content generation patterns

### 2. Developer Blogging Tools & Automation
- Tools that turn code/docs into blog posts (dev.to, Hashnode, Ghost integrations)
- LinkedIn API for automated posting
- Static site generators with CI/CD publishing
- AI-assisted content pipelines (from project artifacts to drafts)

### 3. Documentation-to-Content Patterns
- "Build in public" best practices
- How to structure project docs so they're already half-written blog posts
- Research paper to blog post conversion patterns
- Changelog/ADR/decision log to content pipelines

## Expected Synthesis
A concrete recommendation for:
1. A Claude Code skill that generates blog/LinkedIn drafts from project artifacts
2. Documentation structure tips that make content creation easier
3. Tool recommendations for the publishing pipeline
